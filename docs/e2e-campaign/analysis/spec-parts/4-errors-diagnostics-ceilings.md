## DEFERRED / out-of-scope (Cluster 4)
- **LSP wire-protocol transport of diagnostics** is out of scope for loom 1.0; loom delivers diagnostics only through `loom-system-note` and transient-toast channels and does not define an LSP-protocol transport for `details.diagnostics`. (docs/spec_topics/diagnostics.md §Non-goals)
- **`loom/typecheck/*` build-time `tsc` brands** are out of scope for the runtime Code registry — no registry row, no delivery channel. (docs/spec_topics/diagnostics/diagnostic-shape.md:61)
- **User-defined error types beyond `QueryError`** are out of scope for loom 1.0. (docs/spec_topics/errors-and-results/error-model.md §"`Result` as a user-visible type")
- **Deferred `QueryError` extensions (user-defined error type; `BinderError` as a variant)** ride the ERR-15 discriminator-type-openness seam and are deferred to a minor revision. (docs/spec_topics/errors-and-results/queryerror-variants.md:25)
- **Per-call timeouts** (`timeout:` field) are deferred to a later release; the field is rejected at parse time. (docs/spec_topics/cancellation.md §"Per-call timeouts"; docs/spec_topics/hard-ceilings/ceiling-invariants-and-audit.md §no-additional-ceilings NOCEIL-1)
- **Pre-flight token-count check / token budget** is deferred. (docs/spec_topics/hard-ceilings/ceiling-invariants-and-audit.md §no-additional-ceilings NOCEIL-2)
- **Deferred JSON-mode fallback for typed queries** on providers outside the loom 1.0 typed-query support set. (docs/spec_topics/diagnostics/code-registry-load.md:54)
- **`match` guards and rest patterns** are deferred to a future release (parse-time rejection). (docs/spec_topics/diagnostics/code-registry-parse.md:74, :75)
- **Division/modulo by zero, integer overflow, explicit author-driven panics** are deliberately excluded from the loom 1.0.0 panic catalogue (division/modulo by zero yield IEEE-754 `Infinity`/`NaN`; no `panic(...)` builtin); adding any is a registry-level change. (docs/spec_topics/diagnostics/code-registry-runtime.md §"Division by zero…"; docs/spec_topics/errors-and-results/error-model.md §runtime-panics)
- **Routing obligation / four-axis closure for future ceilings** is a spec-editor governance obligation (GOV-30 lock-step, GOV-15 release gate), not a runtime-testable behaviour — inspection only. (docs/spec_topics/hard-ceilings/ceiling-invariants-and-audit.md §audit-methodology, §release-gate)
- **`loom/load/binder-model-not-strict-capable`** does not fire in production under the loom 1.0 Pi-SDK pin (`Model<Api>.strictCapable` is absent); the W-level `loom/load/binder-model-strict-capability-unknown` fires instead. (docs/spec_topics/diagnostics/code-registry-load.md:30, :31)
- **`loom/host/session-swap-instance-survived`** is dormant on every conformant Pi minor (Pi rebinds a fresh instance on session-only swaps); it is a fail-fast tripwire that fires only on a lifecycle-contract violation. (docs/spec_topics/diagnostics/code-registry-host.md:13)

# `H5a` — REQ-ID / diagnostic-code closing-gate automation

**Convention.** [`conventions.md`](./conventions.md) — *REQ-ID discipline*, *Diagnostic message anchors*, and the [`coverage-matrix.md`](./coverage-matrix.md) closure obligation.

**Adds.** The CI gate that reconciles the spec REQ-ID set and the diagnostics code registry against the plan's [`coverage-matrix.md`](./coverage-matrix.md) and the asserting tests — failing on an unmapped executable REQ-ID, a coverage-matrix-mapped numbered REQ-ID with no citing test in the test corpus, a registry code with no asserting test, an asserted code absent from the registry, a retired/live ID clash, or a per-prefix numbering hole. The `no-broad-catch` lint allow-list reconciliation arm is owned by [`H5c`](./H5c-broad-catch-allow-list-gate.md). The standing **transitive-completeness** plan-structural arm — which reconciles each [`coverage-matrix.md`](./coverage-matrix.md) closing-leaf cell against [`H5b`](./H5b-warn-only-canary.md)'s `Deps.` — is owned by [`H5d`](./H5d-transitive-completeness-gate.md). The un-anchored normative-MUST text-scan arm — spanning the un-enumerated-MUST, `<new>`-placeholder-MUST, and un-rowed-page sub-recognisers and their seeded fixtures — is owned by [`H5e`](./H5e-un-anchored-must-gate.md). At `H5a` the gate's pass/fail is evaluated against the seeded fixtures below; the live-corpus unmapped-executable-REQ-ID failure mode — the live spec REQ-ID set reconciled against the live [`coverage-matrix.md`](./coverage-matrix.md) — does not gate `npm test` at this leaf and first becomes binding at the loom 1.0 release gate activated by [`H6a`](./H6a-live-corpus-activation.md), per [`conventions.md`](./conventions.md) *REQ-ID discipline*. The live-corpus un-anchored-MUST failure mode — the live `spec_topics/**` normative MUST/MUST-NOT set reconciled against the same table, recognised by the un-anchored normative-MUST text-scan arm [`H5e`](./H5e-un-anchored-must-gate.md) owns — shares that posture: it is exercised against [`H5e`](./H5e-un-anchored-must-gate.md)'s seeded fixtures and first becomes binding at the same [`H6a`](./H6a-live-corpus-activation.md) release-gate activation. The live-corpus numbered-REQ-ID citing-test failure mode — the live [`coverage-matrix.md`](./coverage-matrix.md) numbered-REQ-ID mappings reconciled against the citing tests in the live test corpus — shares that posture too: it is exercised against the seeded fixtures here and first becomes binding at the same [`H6a`](./H6a-live-corpus-activation.md) release-gate activation. The gate machinery also exposes a **warn-only live-corpus mode** that runs the same three live-corpus reconciliation surfaces — the live spec REQ-ID set and the live numbered-REQ-ID citing-test surface this leaf owns, and the live `spec_topics/**` normative-MUST surface the [`H5e`](./H5e-un-anchored-must-gate.md) arm contributes — but emits each gap as a CI finding without failing the build; the [`H5b`](./H5b-warn-only-canary.md) canary consumes this mode to pre-flight live coverage immediately before [`H6a`](./H6a-live-corpus-activation.md)'s hard-fail flip. The seeded gate fixtures this leaf owns — the fixture spec REQ-ID with no [`coverage-matrix.md`](./coverage-matrix.md) row and the fixture coverage-matrix mapping whose numbered REQ-ID has no citing test — are permanent by design (they exist to fail their gate arm) and so reside under a dedicated test-fixtures root outside `docs/spec_topics/**` and outside the live test corpus; the gate's live-mode path selection reconciles the live spec/test corpus exclusive of that fixtures root, so no seeded fixture is ever scanned as live coverage. The [`H5e`](./H5e-un-anchored-must-gate.md) un-anchored-MUST arm and the [`H5c`](./H5c-broad-catch-allow-list-gate.md) broad-catch allow-list arm place their own seeded violation fixtures under the same fixtures root.

**Tests.**
- `Convention:` (*REQ-ID discipline*) the gate fails when a fixture spec REQ-ID has no coverage-matrix row.
- `Convention:` (*Diagnostic message anchors*) the gate fails when a test asserts a diagnostic code absent from the registry, and when a registry code has no asserting test.
- `Convention:` (*REQ-ID discipline*) the gate excludes the `loom/typecheck/*` build-time brand-string prefix from registry reconciliation.
- `Convention:` (*REQ-ID discipline* — citing tests) the gate fails when a fixture coverage-matrix-mapped numbered REQ-ID has no test citing that REQ-ID inline in the seeded test corpus, and passes when every mapped REQ-ID has a citing test; the recogniser is a best-effort inline-`PREFIX-N`-citation scan over the test sources and certifies citing-test existence, not the assertion's semantic fidelity.

**Deps.** `H4a`

**Ships when.** `npm test` runs the closing gate green against the seeded no-violation fixture — which includes a seeded fixture coverage-matrix mapping whose numbered REQ-ID has a citing test in the seeded test corpus — and red against each seeded violation fixture this leaf owns, including a seeded fixture spec REQ-ID with no [`coverage-matrix.md`](./coverage-matrix.md) row, a seeded fixture coverage-matrix mapping whose numbered REQ-ID has no citing test in the seeded test corpus, and a seeded registry code with no asserting test, all residing under the same dedicated test-fixtures root outside `docs/spec_topics/**` and outside the live test corpus.

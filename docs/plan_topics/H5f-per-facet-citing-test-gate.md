# `H5f` — Per-facet citing-test closing-gate arm (multi-leaf coverage-matrix rows)

**Convention.** [`conventions.md`](./conventions.md) — *REQ-ID discipline* (the per-facet citing-test surface for multi-leaf rows) and the [`coverage-matrix.md`](./coverage-matrix.md) *Multi-leaf-row per-facet citing tests* convention.

**Adds.** The CI gate arm that reconciles each multi-leaf [`coverage-matrix.md`](./coverage-matrix.md) row's facet partition against the citing tests in the test corpus, run as part of the unified closing-gate machinery [`H5a`](./H5a-closing-gate-automation.md) establishes. For every row whose *Closing leaf(s)* cell lists two or more leaves — across **both** the *Numbered REQ-IDs* table and the *Code-keyed obligation areas (no numbered REQ-IDs)* table — the arm derives the row's **facet partition** by the same rule the [loom 1.0 release-time residue inspection](./conventions.md#release-time-residue-inspection) item 7 fixes: a listed leaf whose immediately-following parenthetical annotation contains the literal token `co-witness` is dropped, and every remaining listed leaf is a **facet** of the row's obligation. It then asserts that each remaining facet leaf carries its own **facet-naming citing test** — a test that cites both the row's subject inline (the numbered REQ-ID for a *Numbered REQ-IDs* row, or the `cka-<n>` *Token* for a *Code-keyed obligation areas* row) and that facet's closing-leaf-ID inline, per the *Multi-leaf-row per-facet citing tests* convention on [`coverage-matrix.md`](./coverage-matrix.md) — and reddens on any remaining facet leaf with no such citing test in the corpus. Like the [`H5a`](./H5a-closing-gate-automation.md) per-REQ-ID citing-test scan it is a **best-effort inline-citation existence scan**: it certifies that a facet-naming citing test exists for each facet, not that the cited test's assertion is semantically faithful to the facet's obligation — that per-facet faithfulness stays a TDD and self-review obligation under the per-phase ritual and the residual item-7 sweep. It reuses the existing backtick-delimited leaf-ID tokens as facet keys and adds no token to the closing-leaf cell, so the [`coverage-matrix.md`](./coverage-matrix.md) *Closing-leaf column authoring* rule is unaffected. This arm closes the per-facet gap the H5a per-REQ-ID scan (one citing test anywhere satisfies a whole REQ-ID), the [`H5d`](./H5d-transitive-completeness-gate.md) per-cell at-least-one transitive-completeness arm, and the [`H5e`](./H5e-un-anchored-must-gate.md) un-anchored-MUST enumeration arm each leave open — including the code-keyed rows (e.g. `cka-33`'s subagent-`AgentSession.abort()` per-site facet at `V9o`) the diagnostic-code↔asserting-test parity gate gives no incidental coverage. Because it reads the live test corpus, the live-corpus per-facet citing-test failure mode does **not** gate `npm test` at this leaf; it is exercised against the seeded fixtures below and first becomes binding at the loom 1.0 release gate activated by [`H6a`](./H6a-live-corpus-activation.md), the same footing flip that activates [`H5a`](./H5a-closing-gate-automation.md)'s per-REQ-ID citing-test surface, and is the surface [`H5b`](./H5b-warn-only-canary.md)'s warn-only canary pre-flights. The arm scans the same pinned test-source corpus the [`H5a`](./H5a-closing-gate-automation.md) per-REQ-ID citing-test scan reads. Its seeded gate fixtures — a multi-leaf coverage-matrix fixture row one of whose facet leaves has no facet-naming citing test in the seeded test corpus, and a companion no-violation fixture where every facet leaf does — are permanent by design (they exist to fail their gate arm) and reside under the same dedicated test-fixtures root [`H5a`](./H5a-closing-gate-automation.md) places its seeded gate fixtures under (outside `docs/spec_topics/**` and outside the live test corpus), so the gate's live-mode path selection never scans a seeded fixture as live coverage.

**Tests.**
- `Convention:` (*REQ-ID discipline* — per-facet citing tests) the gate fails when a seeded multi-leaf coverage-matrix fixture row has a facet leaf (a non-co-witness leaf in its derived facet partition) with no test citing both that row's REQ-ID or `cka-<n>` token and that facet's closing-leaf-ID inline in the seeded test corpus, and passes when every facet leaf of every seeded multi-leaf fixture row has such a facet-naming citing test.
- `Convention:` (*REQ-ID discipline* — facet partition) the gate derives the facet partition from the closing-leaf cell alone — dropping a leaf whose immediately-following parenthetical annotation contains the literal `co-witness` token and keeping every other listed leaf — asserted against a seeded fixture row mixing a co-witness-annotated leaf (excluded, no per-facet citing test required) with two facet leaves (each required to carry its own facet-naming citing test).
- `Convention:` (*REQ-ID discipline* — best-effort existence) the recogniser is a best-effort inline-citation scan over the test sources and certifies per-facet citing-test existence, not the assertion's semantic fidelity.

**Deps.** `H5a`

**Ships when.** `npm test` runs the per-facet citing-test arm green against the seeded no-violation fixture — a multi-leaf coverage-matrix fixture row every facet leaf of which carries a facet-naming citing test in the seeded test corpus, including a row whose co-witness-annotated leaf is correctly excluded from the facet partition — and red against a seeded violation fixture in which one facet leaf of a multi-leaf row carries no facet-naming citing test, all residing under the same dedicated test-fixtures root outside `docs/spec_topics/**` and outside the live test corpus.

# `V18d-T` — Pi version-bump runtime-evidence acceptance gate and revert path (tests)

**Spec.** [`../spec_topics/pi-integration-contract/version-bump-intro.md`](../spec_topics/pi-integration-contract/version-bump-intro.md), [`../spec_topics/pi-integration-contract/version-bump-triggers.md`](../spec_topics/pi-integration-contract/version-bump-triggers.md), [`../spec_topics/pi-integration-contract/host-prerequisites.md`](../spec_topics/pi-integration-contract/host-prerequisites.md).

**Adds.** Failing tests for the paired `V18d` implementation leaf.

**Tests.**
- Runtime-evidence acceptance gate (output (c) of [`version-bump-triggers.md`](../spec_topics/pi-integration-contract/version-bump-triggers.md)): a failing test asserts the gate is composed correctly — that the [`H4a`](./H4a-factory-shell-and-harness.md) end-to-end harness is driven against the bumped pin and that a green surface-inventory run alone does not satisfy acceptance — exercised against a feature-free `H4a` harness double rather than an integrated `.loom`. The test fails red precisely because the gate is unwired, not because any integrated feature is missing; the integrated `.loom` (typed query + tool loop + invoke + schema validation + binder + cancellation) belongs to the impl-time acceptance run on [`V18d`](./V18d-version-bump-acceptance.md). The assertion's coverage is bounded by the `H4a` double's fidelity to the bumped pin.
- Revert / rollback verification (the merge-blocking MUST [`version-bump-triggers.md`](../spec_topics/pi-integration-contract/version-bump-triggers.md) carries — a bump whose runtime-evidence run is red MUST NOT be merged at the candidate pin): a failing test asserts the revert sequence is observable — that after a simulated red run the restore reverts step 4's pin edit and [`V18c`](./V18c-version-bump-checklist.md)'s static build-time gates are re-run against the restored prior pin, exercised against the [`H4a`](./H4a-factory-shell-and-harness.md) harness double and the [`V18c`](./V18c-version-bump-checklist.md) static-gate doubles (whose gate definitions [`V18c`](./V18c-version-bump-checklist.md) / `V18c-T` own and which are not redefined here). The test covers both revert signals: the complete-revert signal (every re-run gate double reports green → the revert is observed complete) and the incomplete-revert signal (a co-edited operand left at its candidate value — a capability-probe constant, the `engines.node` literal, or the reason-snapshot entry — reddens at least one re-run gate double → the revert must be widened to restore that operand). The test fails red precisely because the revert verification is unwired, not because any gate double is missing. The assertion's coverage is bounded by the `H4a` / `V18c`-gate doubles' fidelity to the prior pin.

**Deps.** `H4a`

**Ships when.** The tests above exist, compile, and fail red for the intended reason.

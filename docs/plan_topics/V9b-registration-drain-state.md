# `V9b` — Registration steps and reload-wiring seams

**Spec.** [`../spec_topics/pi-integration-contract/registration-steps.md`](../spec_topics/pi-integration-contract/registration-steps.md), [`../spec_topics/implementation-notes.md`](../spec_topics/implementation-notes.md), [`../spec_topics/pi-integration-contract/host-interfaces-core.md`](../spec_topics/pi-integration-contract/host-interfaces-core.md) (model-registry surface).

**Adds.** Registration steps 1–5 (`registerFlag('loom')` → walk; pending list; `session_start` collision + `registerCommand`; `session_shutdown` subscribe; watcher + `LoomRegistry` build-aside-publish swap). The `LoomRegistry` drain-state contract is owned by [`V9m`](./V9m-drain-state-contract.md). `V9b` is the single declaration site of the test-only `ReloadFailureInjector` interface for the whole **watcher-time reload failure-injection seam** — all three arms (registry-swap, `.loom`/`.warp` re-parse, settings re-merge) inject through the one `injectReloadFailure` method declared here. `V9b` itself wires the registry-swap arm (`loom/runtime/registry-swap-failed`) and the `.loom`/`.warp` re-parse arm; [`V10d`](./V10d-reload-debounce.md) contributes the settings-re-merge arm against this same interface. A caller supplies a synthetic registry-swap or re-parse failure to `ReloadFailureInjector.injectReloadFailure` and the seam routes it onto the `loom-system-note` surfacing path without standing up a live watcher; `V4e` imports the interface from `V9b` (via `Deps`) to exercise `ERR-7`. The **model-reference-matcher production wiring point** is also owned here: at the load pass, `V9b` constructs the concrete model-reference matcher — Pi's `findExactModelReferenceMatch` (`host-interfaces-core.md#model-registry-pin`) run against `ctx.modelRegistry.getAvailable()`, the shared resolution contract [`V11a`](./V11a-binder-model-resolution.md) owns — and injects it into the frontmatter parser through the injection seam [`V6a`](./V6a-frontmatter-contract.md) defines, so `V6a`'s load-time `model:` resolution (`loom/load/model-unresolved`) and `V11a`'s binder-model resolution bind one matcher.

**Tests.**
- [registration-steps.md — registry swap](../spec_topics/pi-integration-contract/registration-steps.md) (PIC area): looms discovered are registered; the swap is atomic (build-aside, then publish); a failed swap fires `loom/runtime/registry-swap-failed`.
- [implementation-notes.md — Static-resolution load pass](../spec_topics/implementation-notes.md) (IMPL area): the in-process re-parse path drops the per-pass cache entry for the changed file and every transitive `.warp` importer as part of the `LoomRegistry` swap.

**Deps.** `V9b-T`, `V9a`, `V10a`, `V8e`, `V6a` (the load-pass wiring injects the concrete model-reference matcher into `V6a`'s parser seam; the watcher swap consumes the `FileWatcher` seam `V8e` owns)

**Ships when.** `npm test` registers looms, swaps atomically (build-aside then publish), fires `loom/runtime/registry-swap-failed` on a failed swap, and drops the per-pass cache entry for the changed file and its transitive `.warp` importers on the swap.

# `H5e` — Un-anchored normative-MUST text-scan closing-gate arm

**Convention.** [`conventions.md`](./conventions.md) — *REQ-ID discipline* (the un-anchored-obligation surface) and the [`coverage-matrix.md`](./coverage-matrix.md) closure obligation.

**Adds.** The CI gate arm that reconciles the un-anchored normative MUST/MUST-NOT obligations on the spec corpus against [`coverage-matrix.md`](./coverage-matrix.md)'s *Code-keyed obligation areas (no numbered REQ-IDs)* table, run as part of the unified closing-gate machinery [`H5a`](./H5a-closing-gate-automation.md) establishes. It fails when a `spec_topics/**` page carries a normative MUST/MUST-NOT that carries no numbered `PREFIX-N` REQ-ID and no `loom/...` registry code and is absent from that table with a closing leaf; the gate does not mechanically exclude named cross-leaf seam obligations, so a seam-contract MUST surfaces as a candidate and is dispositioned via a [`coverage-matrix.md`](./coverage-matrix.md) closing leaf or the [`governance.md` GOV-15](../spec_topics/governance/source-language-stability.md#gov-15) reviewer-inspection residue. The arm spans three sub-recognisers — facets of the one text-scan, not separate arms: the **un-enumerated-MUST** recogniser, the **`<new>`-placeholder-MUST** recogniser (an un-anchored MUST mapped only to a `<new>` coverage-matrix row naming no real closing leaf, treated identically to an un-enumerated MUST), and the **un-rowed-page** recogniser. The recogniser scans the non-narrative `spec_topics/**` pages — those whose row in the [REQ-ID prefix table](../spec_topics/governance/req-id-prefix-table-active-a.md) is not the byte-exact `(no IDs — narrative)` cell per [`governance.md` GOV-3](../spec_topics/governance/req-id-prefix-table-active-a.md#gov-3) — and additionally reddens on any `spec_topics/**` page absent from that table altogether — other than a [GOV-24](../spec_topics/governance/req-id-prefix-table-active-b.md#gov-24) table-of-contents hub stub — as an un-rowed-page residue defect, per [`conventions.md`](./conventions.md) *REQ-ID discipline*. The live-corpus un-anchored-MUST failure mode — the live `spec_topics/**` normative MUST/MUST-NOT set reconciled against the live [`coverage-matrix.md`](./coverage-matrix.md) — does not gate `npm test` at this leaf; it is exercised against the seeded fixtures below and first becomes binding at the loom 1.0 release gate activated by [`H6a`](./H6a-live-corpus-activation.md), the same footing flip that activates [`H5a`](./H5a-closing-gate-automation.md)'s live surfaces, and is the surface [`H5b`](./H5b-warn-only-canary.md)'s warn-only canary pre-flights. Its seeded gate fixtures — the un-enumerated-MUST fixture page, the `<new>`-placeholder-MUST fixture page (an un-anchored MUST mapped only to a `<new>` coverage-matrix row naming no real closing leaf), and the un-rowed-page fixture (a non-hub-stub `spec_topics/**`-shaped page absent from the fixture prefix table) — are permanent by design (they exist to fail every gate arm) and reside under the same dedicated test-fixtures root [`H5a`](./H5a-closing-gate-automation.md) places its seeded gate fixtures under (outside `docs/spec_topics/**` and outside the live test corpus), so the gate's live-mode path selection never scans a seeded fixture as live coverage.

**Tests.**
- `Convention:` (*REQ-ID discipline* — un-anchored obligations) the gate fails when a fixture spec page carries a normative MUST/MUST-NOT with no `PREFIX-N` REQ-ID and no registry code that is absent from the [`coverage-matrix.md`](./coverage-matrix.md) *Code-keyed obligation areas (no numbered REQ-IDs)* table with a closing leaf, and passes when every such fixture MUST is enumerated with a closing leaf.
- `Convention:` (*REQ-ID discipline* — un-anchored obligations, `<new>` placeholder) the gate fails when a fixture spec page's un-anchored normative MUST/MUST-NOT maps only to a `<new>` placeholder coverage-matrix row whose closing-leaf column names no real leaf (treated identically to an un-enumerated MUST, mirroring the un-enumerated-MUST fixture above), and passes when that fixture MUST is enumerated with a real closing leaf; like the un-enumerated-MUST recogniser it is exercised against the seeded fixtures here and first becomes binding at the [`H6a`](./H6a-live-corpus-activation.md) release-gate activation.
- `Convention:` (*REQ-ID discipline* — un-rowed page) the gate reddens when a seeded non-hub-stub `spec_topics/**`-shaped fixture page is absent from the fixture [REQ-ID prefix table](../spec_topics/governance/req-id-prefix-table-active-a.md) altogether (an un-rowed-page residue defect), and stays green both when that fixture page carries a prefix-table row and when an un-rowed fixture page is a [GOV-24](../spec_topics/governance/req-id-prefix-table-active-b.md#gov-24) table-of-contents hub stub (hub stubs are excluded from the un-rowed-page defect).

**Deps.** `H5a`

**Ships when.** `npm test` runs the un-anchored-MUST text-scan arm green against the seeded no-violation fixture — which includes a fixture page whose un-anchored normative MUST is enumerated in the *Code-keyed obligation areas* table with a closing leaf and a seeded [GOV-24](../spec_topics/governance/req-id-prefix-table-active-b.md#gov-24) hub-stub fixture page that is itself absent from the fixture prefix table yet stays green because hub stubs are excluded from the un-rowed-page residue defect — and red against each seeded violation fixture, including a seeded un-enumerated-MUST fixture page that carries a `PREFIX-N`-less, registry-code-less normative MUST mapped to no closing leaf, a seeded `<new>`-placeholder-MUST fixture page whose un-anchored MUST maps only to a `<new>` placeholder coverage-matrix row naming no real closing leaf, and a seeded un-rowed-page violation fixture (a non-hub-stub `spec_topics/**`-shaped page absent from the fixture prefix table) residing under the same dedicated test-fixtures root outside `docs/spec_topics/**` as the other seeded fixtures.

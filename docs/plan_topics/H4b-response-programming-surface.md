# `H4b` — Response-programming surface

**Convention.** [`conventions.md`](./conventions.md) (phase categories — end-to-end harness).

**Adds.** The shared **response-programming surface** that extends [`H4a`](./H4a-factory-shell-and-harness.md)'s minimal response-emission capability — the base [`H4a`](./H4a-factory-shell-and-harness.md)'s session-double fidelity contract self-checks against — into the full scripted-injection surface: the single input-side API a test uses to script what the in-process double does — that gives a test an observable way to: (a) script the sequence of assistant turns the double emits and the streamed fragments per turn; (b) script the `tool_use` result the double returns for a given tool call, including an `isError: true` tool-result and a mixed-success parallel `tool_use` batch (the `V14b` tool-calls vector); (c) script the binder/provider-call response or failure fed to the binder path, including a malformed-envelope failure and a transport-class failure (the malformed-envelope and transport-class definitions and per-invocation retry budget per [`determinism-cancellation-failure.md` §"Failure-class taxonomy"](../spec_topics/binder/determinism-cancellation-failure.md#failure-class-taxonomy), envelope schema per [`binder-bypass-and-envelope.md`](../spec_topics/binder/binder-bypass-and-envelope.md)) that drive the `V11f` per-class retry budget; (d) force a `tool_loop.max_rounds` round-exhaustion — the `tool_loop_exhausted` outcome per [`queryerror-variants.md` ERR-19](../spec_topics/errors-and-results/queryerror-variants.md#err-19) and the `max_rounds` round semantics per [`frontmatter-fields-b-and-templates.md` FRNT-1](../spec_topics/frontmatter/frontmatter-fields-b-and-templates.md#frnt-1) (the `V13c` ceiling #2 vector); and (e) inject an abort/cancellation at a chosen point in the drive — pre-call, during an in-flight provider call, and during a budgeted retry (the `V11f` / `V17a` vectors). This response-programming surface is the single shared scripting contract every (a)–(e)-driven harness leaf consumes — `H7a`, `V11f`, `V13c`, `V13d`, `V13f`, `V14b`, and `V17a` — so each scripts against one API rather than inventing a per-leaf surface; the named injection points above are the content the contract must define, while the API's exact shape is the implementer's to finalise. The modeled-behaviour injection categories — (f) completed-invoke-child and (g) subagent-mode callee — are split into the follow-on harness leaf [`H4c`](./H4c-modeled-behaviour-surface.md), which depends on this leaf and extends this surface for the `V4c` / `V4f` / `V9i` / `V9o` vectors that consume them.

**Tests.**
- `Convention:` (end-to-end harness) the self-check asserts the response-programming surface drives the double **deterministically** — replaying the same scripted assistant turns, `tool_use` results, binder responses/failures, `tool_loop.max_rounds` round-exhaustion, and cancellation-injection points yields the same observable transcript on every run.
- `Convention:` (end-to-end harness) the self-check asserts the **functional effect** of each response-programming-surface injection-point category — that each scripted input yields its expected observable, not merely a stable one — so the seam contract is verified at this owning leaf rather than only transitively by downstream consumers. The self-check asserts, for the categories `Adds` enumerates: (a) scripted assistant turns and the per-turn streamed fragments appear in the transcript in the scripted order; (b) a scripted `isError: true` tool-result surfaces as an error tool-result in the transcript, and a mixed-success parallel `tool_use` batch lowers each sibling call's outcome independently; (c) a scripted malformed-envelope failure and a scripted transport-class failure each drive the observable binder failure / retry path; (d) a scripted `tool_loop.max_rounds` round-exhaustion produces the round-exhaustion observable; and (e) an abort injected at each chosen point — pre-call, during an in-flight provider call, and during a budgeted retry — surfaces the cancellation observable. The assertions are phrased in terms of the observable each scripted input must yield; the surface's API shape stays the implementer's to finalise.

**Deps.** `H4a`

**Ships when.** `npm test` exercises the response-programming surface through [`H4a`](./H4a-factory-shell-and-harness.md)'s harness and the self-check passes — including the determinism gate and the per-category functional-effect assertions (a)–(e).

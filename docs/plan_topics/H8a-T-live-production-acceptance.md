# `H8a-T` — Live production end-to-end acceptance (tests)

**Convention.** [`conventions.md`](./conventions.md) (phase categories — end-to-end harness; the [live-host acceptance pair exception](#horizontal-leaves--single-task)). Narrative spec references for the implementer: [`extension-bootstrap-and-per-loom.md`](../spec_topics/pi-integration-contract/extension-bootstrap-and-per-loom.md), [`registration-steps.md`](../spec_topics/pi-integration-contract/registration-steps.md), [`discovery.md`](../spec_topics/discovery.md).

**Adds.** An **opt-in live-host acceptance suite** that loads the *shipped* extension the way Pi loads it — through the real [`extensions/index.ts`](../../extensions/index.ts) entry, not the [`H4a`](./H4a-factory-shell-and-harness.md) in-memory fixture-supply — discovers a real `.loom` file from a real on-disk discovery source, and drives it against a **live provider/model**. It closes no new spec REQ-ID; it verifies the live composition the double-backed gates ([`H4a`](./H4a-factory-shell-and-harness.md), [`H7a`](./H7a-integration-acceptance.md), [`V18d`](./V18d-version-bump-acceptance.md)) never exercise, and that the [manual real-host smoke](./real-host-smoke-gate.md) covers only by hand. It is deliberately small and token-bounded (trivial prompts with deterministic sentinels, a single small structured query, a fast/cheap model), spending real tokens but keeping cost and flakiness low. The suite lands as a **dedicated runner excluded from the default `npm test`** (e.g. an `npm run test:live` script) so ordinary CI neither burns tokens nor requires network/credentials; it **fails loudly** when its live-provider precondition is unmet (per *No silent test skipping*), never silently passing. These tests red today because the production composition root supplies `fixtures: []` — the default export runs no discovery walk, so no `.loom`-derived slash command is ever registered by the shipped extension.

**Tests.**
- `Convention:` (live-host acceptance) **discovery → registration.** With the shipped extension loaded through its real `extensions/index.ts` entry (not the `H4a` fixture-supply), a `.loom` written under the project discovery source `<cwd>/.pi/looms/` registers a live slash command named for its filename stem — exercising the real [`V10a`](./V10a-discovery-walk.md) walk over the real [`V8b`](./V8b-filesystem-seam.md) `PiFileSystem` and the [`V9b`](./V9b-registration-reload-wiring.md) `session_start` → `pi.registerCommand` step. Reds today because the shipped default export registers no discovered command.
- `Convention:` (live-host acceptance) **prompt-mode turn against a live model.** Invoking that registered command drives exactly one real prompt-mode turn against a live provider/model and the assistant response contains the fixture's deterministic sentinel string — the end-to-end injection promise ([`M`](./M-minimal-slash-command.md)'s prompt-mode drive) exercised against a real model rather than [`H4b`](./H4b-response-programming-surface.md)'s scripted double. Reds today because no command exists to invoke.
- `Convention:` (live-host acceptance) **alternate discovery source.** A `.loom` discovered via a second real source — the `--loom <dir>` CLI source or the global `~/.pi/agent/looms/` source ([`V10a`](./V10a-discovery-walk.md)/[`V10c`](./V10c-settings-merge.md) over `PiFileSystem`) — also registers a live slash command, proving discovery is source-general and not wired to a single hardcoded path.
- `Convention:` (live-host acceptance) **typed-query lowering (bounded).** A single small schema-typed `@`-query resolves through the real binder-model resolver ([`V11a`](./V11a-binder-model-resolution.md)) and a live structured-output model and yields a value that **validates against its declared schema** ([`V5d`](./V5d-reject-gate.md) schema lowering/validation — structural validity, not exact content) — the typed-reply promise exercised against a real model, which the double-backed gates and the manual smoke otherwise confirm only structurally or against the in-process double. Reds today because no command exists to invoke.

**Deps.** `H4a`, `H4b`, `M`, `V8b`, `V9b`, `V10a`, `V10c`, `V11a`, `V5d`, `V13b` — the discovery, registration, prompt-mode, real-`FileSystem`-seam, binder-model, and schema-lowering machinery must already exist so the suite reds for the *intended* reason (the production composition root is unwired) rather than on an absent seam.

**Ships when.** The opt-in live suite (added by this leaf, invoked by a dedicated runner such as `npm run test:live` and **excluded from the default `npm test`**) exists, compiles, and — run in an environment with a configured live provider (the `loom-leaf-implementer`'s environment satisfies this) — *fails red for the intended reason*: no `.loom`-derived slash command registers because the shipped extension's production composition root supplies no discovered fixtures, so each test above reds on the missing command / absent turn, not on a credential, network, setup, or harness throw. When no provider is configured the runner fails loudly naming the missing precondition (never a silent skip); the intended-reason red is observed with a provider configured.

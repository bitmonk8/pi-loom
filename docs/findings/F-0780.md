## Finding

The **Engine value model (non-checked invariant)** paragraph in `docs/spec_topics/runtime-value-model.md` (§ *JavaScript engine assumptions*) closes with the clause "no entry of the always-log set is guaranteed to fire on a violation (see [Pi Integration Contract — Runtime event channel](./pi-integration-contract.md) for the always-log carve-out clause)." The link text claims to point at *Runtime event channel*, but the target `pi-integration-contract.md` is the parent page, which under the post-size-cap split is a one-screen table-of-contents file — it neither defines "always-log set" nor carries the carve-out clause. A reader following the link must already know that the real target lives in the sibling sub-page `pi-integration-contract/runtime-event-channel.md` (where the term is defined under the **Runtime event channel** subsection — the same anchor the *Engine-assumption carve-out* paragraph on that page co-locates with).

The paragraph is also the first and only use of the term on the page, so an isolation reader of the runtime-value-model section gets no inline gloss either: the link resolves to a TOC, the term is not defined locally, and the glossary entry (which does define it) is two hops away.

The fix is two small edits in the same sentence: retarget the cross-reference at the section that actually defines and carves out the always-log set, and prepend a one-clause inline gloss so an isolation reader can resolve the term without leaving the page.

## Solution Space

**Shape:** single

### Recommendation

In `docs/spec_topics/runtime-value-model.md`, **Engine value model (non-checked invariant)** paragraph, rewrite the trailing parenthetical to (a) add a one-clause inline gloss and (b) retarget the cross-reference at the subsection that actually defines and carves out the term:

> …and no entry of the *always-log set* — the closed set of `QueryError` kinds, binder failure causes, and `loom/runtime/*` panic codes whose runtime occurrence emits exactly one `loom-system-note` (see [Pi Integration Contract — Runtime event channel](./pi-integration-contract/runtime-event-channel.md#runtime-event-channel) for the full definition and the engine-assumption carve-out clause) — is guaranteed to fire on a violation.

Edge cases the implementer must watch:

- The `#runtime-event-channel` fragment is the explicit `<a id="…">` anchor at line 40 of `pi-integration-contract/runtime-event-channel.md` (the **Runtime event channel** bold lead-in). Verify the anchor exists verbatim before committing; if the file uses a different anchor scheme for that section, use whatever anchor that page actually exposes.
- The companion **Engine-assumption carve-out** paragraph (line 44 of the same file) lives directly under the *Runtime event channel* anchor, so a single anchor jump lands the reader within one screen of both the definition and the carve-out clause; no second cross-reference is needed.
- Do not alter the inline-gloss content beyond the closed-set enumeration above — the glossary entry (`docs/spec_topics/glossary.md` `always-log set`) is the canonical short-form; the gloss must mirror it byte-faithfully on the three constituent token classes (`QueryError` kinds, binder failure causes, `loom/runtime/*` panic codes) and the exactly-once emission clause, so the two surfaces stay in lock-step.
- The `*always-log set*` italicisation in the rewritten sentence matches the term-definition convention used elsewhere on the page (e.g. the **Equality (`==`)** italicised cross-references); keep the italics so the term is visually distinguishable from the surrounding prose.

## Consequence

**Severity:** advisory

An implementer reading the JavaScript-engine-assumptions section in isolation cannot resolve "always-log set" without leaving the page, and the in-text link sends them to a TOC rather than to the carve-out clause it promises. The defect does not cause two implementers to produce divergent behaviour — the underlying carve-out semantics are unambiguous once located — but it forces an unnecessary navigation step on every isolation reader and silently mis-advertises a definitional link as a target-anchor link.

## Problem

In `host-prerequisites.md`, the "Pi-side slash-handler promise lifecycle (consumption posture)" paragraph (anchor `id="pi-slash-handler-promise-lifecycle-presupposition"`) opens by stating its purpose: recording its six presuppositions so a future Pi change surfaces against a named presupposition rather than passing silently, and it closes by confirming version-bump item (ad) re-checks presupposition (vi). Presupposition (vi) itself, however, ends with the clause that a future macrotask-deferred flip "could let an Esc during a compute-bound loop be missed at the next iteration without surfacing against any named presupposition." Presupposition (vi) *is* the named presupposition for that failure mode and item (ad) re-checks it on every Pi minor, so the closing clause directly contradicts the paragraph's stated design intent. The sibling presupposition (v), written in the same construction, correctly stops at describing the silent degradation without a contradictory tail.

## Issue introduction
**Verdict:** single-commit

**Introducing commit:** `fdc5115` — "pi-loom spec: resolve \"Esc-cancellation assumes single-macrotask abort-flip\"" (2026-06-06)

**History:** The defect token "without surfacing against any named presupposition" first enters the corpus at `fdc5115`, the commit that authored presupposition (vi). `git log -S` confirms no earlier occurrence of the phrase in `docs/`. The same commit's diff shows the phrase pre-existed verbatim in the *problem statement* of the finding it resolved ("an Esc during a compute-bound loop could be missed at the next iteration without surfacing against any named presupposition"), where it correctly described the absence of a guarding presupposition. When the fix added presupposition (vi) and the matching bump-checklist item (ad), the problem-statement wording was copied into (vi)'s body, where it now contradicts the guard it helped create. A later commit (`605a3a7`, F-1393) relocated the presuppositions into `host-prerequisites.md` but carried the clause across unchanged; the relocation did not introduce or alter the defect.

## Solution approach

Rewrite presupposition (vi)'s closing clause in `host-prerequisites.md` so it no longer denies the guard the presupposition provides, mirroring presupposition (v)'s construction: end at the degradation description ("...could let an Esc during a compute-bound loop be missed at the next iteration") and delete the contradictory tail "without surfacing against any named presupposition."

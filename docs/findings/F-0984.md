## Problem

The `Path` production in the `system:` interpolation slot is `Path := Ident ('.' Ident)*` — bare identifiers only, with no narrowing operator. The *Path grammar* paragraph in `frontmatter/frontmatter-fields-b-and-templates.md` correctly states that discriminated unions terminate the path, then immediately undermines it: "A path that descends into an arm of a discriminated union without a discriminator narrowing is rejected at parse time — loom 1.0 has no narrowing in this slot." The "without a discriminator narrowing" qualifier names an exception the grammar makes unreachable, implying some narrowing syntax exists when it does not. The same implicature recurs in the `loom/parse/system-interp-bad-field` Cause cell in `diagnostics/code-registry-parse.md` via the phrase "or descends into an array or un-narrowed discriminated union".

## Issue introduction

**Verdict:** single-commit

**Introducing commits:** `428fcb7` (2026-05-04, "pi-loom spec: resolve \"system: interpolation: path grammar, stringification, and parse errors unspecified\"")

**History:** `git log -S 'discriminator narrowing' -- docs/ spec_topics/` resolves the *Path grammar* paragraph's first appearance to `428fcb7` (which authored the entire `system:` interpolation block in the then-monolithic `spec_topics/frontmatter.md`, including both the sentence "A path that descends into an arm of a discriminated union without a discriminator narrowing is rejected at parse time — V1 has no narrowing in this slot." and the `loom/parse/system-interp-bad-field` "or attempts to descend into an array or un-narrowed discriminated union" parenthetical). Subsequent commits did not introduce the defect: `31ff060` (2026-05-08) rehomed the file under `docs/spec_topics/` without touching the paragraph; `f5e89f4` (2026-06-04) sharded the file into `frontmatter-fields-b-and-templates.md` carrying the prose unchanged; `00681e0` rewrote the same paragraph to fix the unrelated wire-name namespace bug (F-0700) but preserved the "without a discriminator narrowing" clause verbatim. The defect has therefore been present continuously since `428fcb7` and survives all later edits to the same paragraph.

## Solution approach

Rewrite the rejection clause in the *Path grammar* paragraph of `frontmatter/frontmatter-fields-b-and-templates.md` to state positively that a discriminated-union value terminates the path: any `.Ident` step whose parent is a discriminated-union type is `loom/parse/system-interp-bad-field`. Align the `loom/parse/system-interp-bad-field` Cause cell in `diagnostics/code-registry-parse.md` by removing the `un-narrowed` qualifier from "or descends into an array or un-narrowed discriminated union". Keep any forward-compatibility note about a future richer sublanguage in the [Future Considerations](../future-considerations.md) entry already cross-referenced from the paragraph.

## Solution constraints

- The `Path` BNF production is correct; out of scope — do not modify the grammar.

## Problem

Three editorial sites inline a fixed GOV-N enumeration that no commit relocks when the governed set changes, so all three have drifted out of parity with the authoritative live/retired tables on `governance/anchor-scheme-and-retired.md`. The Appendix Governance bullet in `session-model-and-appendix.md` and its byte-identical copy in `governance.md`'s preamble both summarise the set as "GOV-1, GOV-3 through GOV-9, GOV-15, GOV-16, GOV-30, GOV-31; GOV-2, GOV-10–GOV-14 retired" — wrongly listing GOV-4 as active and omitting GOV-21's retirement and successors. The glossary `GOV-N (governance rule)` entry states the live range runs "currently running through `GOV-23`", now stale by eight IDs (GOV-24…GOV-31), and contradicts its own adjacent directive to cite `governance.md` rather than a fixed upper bound.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `1d773d6` (2026-06-04, retired GOV-4 + added GOV-24); `8e3ccb4` (2026-06-05, retired GOV-21, split into GOV-25…GOV-29); `96cfc08` (2026-06-04, the glossary "currently running through `GOV-23`" literal figure); `fe8aa5c` (2026-06-05, GOV-30, GOV-31).
**History:**

*Appendix / governance.md preamble copies:* `52219c4` (2026-05-06) introduced the Governance bullet on `spec.md`. `f5e89f4` (2026-06-04) moved it into `session-model-and-appendix.md`. `1d773d6` retired GOV-4 by appending to the Retired table but did not update the Appendix bullet's "GOV-3 through GOV-9" range or the identical preamble sentence, nor add GOV-24 (the "GOV-4 listed as active" half). `8e3ccb4` retired GOV-21 likewise without propagating (the "GOV-21 omitted" half). `fe8aa5c` and `5a772e3` (2026-06-05) DID update both copies in lock-step when retiring GOV-12/14 — the discipline `1d773d6`/`8e3ccb4` failed. `81bf52a` (2026-06-11) relocated the file as a pure move.

*Glossary entry:* `git log -L` on the entry shows `96cfc08` rewrote the bullet to the current prose and introduced the "currently running through `GOV-23`" parenthetical, accurate when it landed. Within ~28 hours three commits added `GOV-24`…`GOV-31` to the governance pages without touching the glossary line; nothing locks the glossary parenthetical to the live range, so the figure went stale on the first such commit (`1d773d6`) and grew staler.

All three drifts are missed lock-steps between an inline editorial snapshot and the authoritative governance tables, not defects in any single commit.

## Solution approach

Delete the fixed GOV-N enumeration in the `session-model-and-appendix.md` Appendix Governance bullet and in `governance.md`'s preamble paragraph, replacing each with a brief role description plus forward-links to the authoritative live (REQ-ID prefix table) and retired ([Retired REQ-IDs](./governance/anchor-scheme-and-retired.md#retired-req-ids)) tables. Delete the glossary `GOV-N (governance rule)` entry's "currently running through `GOV-23`" figure, leaving its existing directive to consult `governance.md` for the authoritative range. Do not substitute a fresher figure for any deleted range — a replacement value rearms the same drift.

## Solution constraints

- Out of scope: the companion inline-label-prefix sentence (`HC3`/`NOCEIL`/`SNK`) in `governance.md`'s preamble — a different aggregator (same-cluster F-1331).
- The Appendix bullet and the `governance.md` preamble copy must both delegate to the tables; leaving one enumerative and the other delegating is itself a consistency defect.

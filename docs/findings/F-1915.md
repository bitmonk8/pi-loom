## Problem

`V2e` (Wire-name translation boundary) declares three behaviours in its **Adds** field — an inbound post-AJV loom-side rebuild, inbound enum-tag reattach via the `V5f`-produced per-schema sidecar, and outbound wire-name JSON — all three of which `runtime-value-model.md` makes normative. `V2e`'s single **Tests** bullet asserts only the inbound name rebuild and the defaults-bypass case; neither outbound serialisation nor inbound enum-tag reattach is exercised by any vector, and the bullet is mirrored verbatim in `V2e-T`. Because `V2e`'s **Ships when** references only "the wire-translation vectors above" and the `cka-2`/`H5f` per-facet citing-test gate is satisfied by that lone bullet, the ship gate and coverage gate both stay green while two spec-normative behaviours ship unexercised.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `c6a664e` (2026-06-10, "pi-loom plan: build/update plan for spec.md + review")
**History:** The wire-name translation Tests bullet was authored in `c6a664e` on the then-combined `V2c` leaf ("Runtime value model, equality, and wire-name translation"), covering only the inbound rebuild + defaults-bypass case while that leaf's Adds already listed outbound JSON and enum-tag reattach — so the assertion gap dates to the bullet's first authoring. Commit `d4f6e5c` (2026-06-17, "split V2c value model from wire-name translation boundary") moved the bullet verbatim into the newly created `V2e` leaf without broadening it, preserving the gap rather than introducing it.

## Solution approach

Add **Tests** vectors to `V2e` covering the two unasserted behaviours — outbound serialisation of a loom value with a renamed object-schema field to wire-named JSON before AJV, and inbound enum-tag reattach of a validated model-output enum value via the `V5f` sidecar so the rebuilt value compares equal to the loom-side literal and unequal cross-enum (`docs/spec_topics/runtime-value-model.md`, *Wire-name translation* → *Outbound* and *Inbound* clause (b) + *Equality*). Mirror the new vectors into `V2e-T` per the paired-leaf TDD ritual. `V2e`'s **Ships when** ("asserts the wire-translation vectors above") extends automatically and needs no edit.

## Solution constraints

- Out of scope: the spec is read-only (it already fully specifies both behaviours); `V5f` remains the sidecar owner `V2e` consumes; `V2c` keeps the value-model `JSON.stringify`/equality vectors.
- Do not retarget the `cka-2` coverage-matrix closing-leaf cell.

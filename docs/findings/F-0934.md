## Finding

`V9g`'s **Ships when** field restates, near-verbatim, the entire CNCL-4 assertion already specified in the leaf's CNCL-4 **Tests** bullet. Both passages carry the same load-bearing tokens: `signal.reason instanceof Error`, the byte-exact message `"loom cancelled by session shutdown"`, the `signal.reason === source.reason` contract reference, and the "at a downstream checkpoint" qualifier — roughly 60–80 words duplicated between the two fields of one leaf.

Per `conventions.md` §Leaf format, **Ships when** is meant to state "a concrete, externally observable change" — an acceptance criterion — while the **Tests** bullets are the binding per-REQ-ID specification. Carrying the full test-assertion text in both places makes Ships-when a second copy of the test spec rather than a terse exit-gate criterion, and creates a drift hazard: a future edit to the CNCL-4 Tests bullet (the authoritative source) must be hand-mirrored into Ships-when or the two silently disagree.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `9081369` (2026-06-10, Thomas Andersen — "pi-loom plan: resolve \"CNCL-4 session-shutdown reason facet not authored red or gated\""), `1640cdb` (2026-06-16, Thomas Andersen — "pi-loom plan: resolve \"V9g CNCL-4 test bullet ambiguous 'this' signal.reason shape\"")
**History:** `git log -S 'signal.reason === source.reason'` and `git show` on `docs/plan_topics/V9g-session-shutdown.md` show the duplication was born and then deepened across two plan-fix commits. `9081369` first added the CNCL-4 facet to both the Tests bullet and the Ships-when field, with Ships-when already carrying a substantial restatement (synthesised `Error`, byte-exact message, downstream checkpoint). `1640cdb` then expanded both fields in lockstep — adding `signal.reason instanceof Error` and the `signal.reason === source.reason` contract reference to each — producing the present near-verbatim ≈80-word overlap. The duplication is therefore not present-since-inception of the leaf but an artefact of two successive edits that each touched both fields rather than pointing one at the other.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V9g-session-shutdown.md`, edit the **Ships when** field so its CNCL-4 clause is a reference pointer to the leaf's CNCL-4 Tests bullet instead of a copy of that bullet's assertion. Strike the inlined assertion text — `each in-flight \`loomAbort.signal.reason\` is the synthesised \`Error\` object itself (\`signal.reason instanceof Error\` with \`signal.reason.message\` byte-exact \`"loom cancelled by session shutdown"\`), per \`cancellation.md\` CNCL-4's \`signal.reason === source.reason\` contract, at a downstream checkpoint` — and replace it with a short reference to the facet, e.g. `…and the \`CNCL-4\` session-shutdown reason facet`. The CNCL-4 Tests bullet remains the single authoritative specification of the asserted behaviour; Ships-when names the observable gate. Leave the other Ships-when clauses (per-step isolation, the await cap, the wrapped host emissions) unchanged.

## Consequence

**Severity:** cosmetic

No implementer is misled today — both fields state the same thing. The cost is maintenance drift: the duplicated ≈80-word block must be edited in lockstep with the CNCL-4 Tests bullet, and a future single-sided edit would leave the exit-gate criterion silently inconsistent with the authoritative test spec.

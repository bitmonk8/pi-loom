## Finding

The *loom 1.0 seam — Pi-owned subagents collision source set* blockquote in `host-interfaces-services.md` carries a normative MUST whose subject is the runtime's internal source representation rather than its externally-observable behaviour:

> The set of `SlashCommandSource` values the `session_start` handler treats as collision candidates (…) **MUST be defined as a single named set inside the runtime, consulted by the collision check via membership test rather than open-coded as three separate string comparisons or a hard-coded `switch`**.

A conformant runtime that open-codes the three `source === "prompt" | "extension" | "skill"` comparisons inline produces a behaviourally indistinguishable collision check: the same pending looms are dropped, the same `loom/runtime/cross-format-collision` diagnostics fire, and the same survivors register. The MUST is therefore not a conformance obligation — it is a maintainability convention about how the runtime is factored so that the deferred *Pi-owned subagents exposed as enumerable slash commands* extension can widen one declaration site rather than auditing every comparison site. Phrased as a MUST under GOV-18 arm (a), it binds the implementation target on a property no conformance test can witness.

The companion seam blockquote one shard over — *loom 1.0 seam — typed-query supported provider set* in `conversation-drive.md` — used to carry the same "MUST be exposed as a single named runtime constant" wording and was already demoted to a non-normative implementation note in the spec edit titled *"Demote over-prescriptive and non-testable MUSTs on unobservable implementation structure"*. The subagents-collision-source-set seam carries the same defect and was missed in that pass; the precedent for the fix is verbatim adjacent in the corpus.

## Solution Space

**Shape:** single

### Recommendation

Demote the structural MUST in the *loom 1.0 seam — Pi-owned subagents collision source set* blockquote to a non-normative implementation note, following the shape `8362824` applied to the *typed-query supported provider set* sibling in `conversation-drive.md`. Preserve the behavioural framing (the set's *content* — `"prompt"`, `"extension"`, `"skill"` — and the forward-compatibility statement that the deferred subagents extension widens it to a fourth `"subagent"` arm) since both carry real obligations; only the source-shape sentence is the defect.

Concretely, replace the existing wording

> … MUST be defined as a single named set inside the runtime, consulted by the collision check via membership test rather than open-coded as three separate string comparisons or a hard-coded `switch`.

with a two-sentence split that keeps the behavioural content normative and moves the factoring to an implementation note, e.g.:

> The set of `SlashCommandSource` values the `session_start` handler treats as collision candidates is `{"prompt", "extension", "skill"}` under `@earendil-works/pi-coding-agent` at the [loom 1.0 Pi-SDK pin](./host-prerequisites.md#pi-sdk-pin). The deferred *Pi-owned subagents exposed as enumerable slash commands* extension in [Future Considerations](../future-considerations.md) widens that set to four arms (adding a `"subagent"` entry) when a future Pi minor extends `SlashCommandSource` accordingly; widening the set is the only loom 1.0 surface that perturbs, and the asymmetric loser-drops collision rule defined in [Discovery](../discovery.md) is preserved by construction. *Non-normative implementation note.* loom 1.0 expects the set to be exposed as a single named runtime constant consulted via membership test (rather than open-coded as three string comparisons or a `switch`) so there is one source of truth to widen; this single-constant factoring is a maintainability convention, not an observable conformance point.

Preserve both `<a id="loom-1-0-seam-pi-owned-subagents-collision-source-set">` and `<a id="v1-seam-pi-owned-subagents-collision-source-set">` anchors on the blockquote label line so the three live cross-references from `registration-steps.md`, `host-prerequisites.md`, and `surface-extensions.md` survive byte-equal under GOV-25 / GOV-27. The blockquote label *loom 1.0 seam — Pi-owned subagents collision source set* must stay, since `surface-extensions.md` counts it among the 12 typed/structural seams under GOV-31.

Edge case for the implementer: the [related cruft finding](#related-findings) targeting `conversation-drive.md`'s inline seam blockquotes proposes deleting that file's *typed-query supported provider set* blockquote outright; if both fixes land, this fix needs no rework, because this finding only edits the subagents-collision-source-set blockquote's body and does not depend on the sibling blockquote remaining in `conversation-drive.md`. The reverse is also true. The two findings co-resolve on the structural-MUST vector but operate on different files.

## Consequence

**Severity:** cosmetic

A MUST whose violation no conformance test can witness invites two failure modes: a reviewer rejecting an otherwise-conformant implementation on a non-observable factoring grievance, and the inverse — the MUST being silently ignored, training readers to treat the corpus's normative keywords as advisory. Both are reputational rather than behavioural; the runtime's externally-observable collision behaviour is unaffected either way.

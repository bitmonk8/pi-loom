## Problem

DISC-5's "Per-package resolution" paragraph says that when no plain (un-prefixed) include pattern is present, "the starting set is every path under the package root", but does not state whether that enumeration is recursive (every transitive descendant) or flat (only immediate children). The rest of the file is uniformly non-recursive — the `looms/` fallback, the global walks, and DISC-5's own directory-match rule are each explicitly non-recursive — so the unqualified phrase reads as ambiguous rather than deliberate. The two readings produce materially different registered sets for a package shipping nested `.loom` files, and they determine whether the `!`/`+`/`-` override steps act over a recursive or flat set. The ambiguity propagates to `loomPaths`, whose entry schema delegates its glob contract back to DISC-5.

The depth qualifier the corpus uses elsewhere was not carried into the new sentence; the gap is a missed parity edit inside an otherwise-thorough fix.

## Solution approach

Clarify the DISC-5 (`#disc-5`) starting-set sentence to state the enumeration depth explicitly, scanning the package root non-recursively so the default starting set is the package root's immediate children — matching the global non-recursion rule already governing the `looms/` fallback and DISC-5's own directory matches. Note that nested `.loom` files enter the set only via an explicit recursive plain include such as `**/*.loom`. The `### loomPaths entry schema › Glob patterns and exclusions` block delegates its glob contract to DISC-5, so it needs no separate depth edit once DISC-5 states it.

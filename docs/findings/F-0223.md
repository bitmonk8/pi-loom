## Finding

`H7a` commits a **permitted-code list** alongside the fixture `.loom` and describes it as "the union of `loom/...` codes the slices in **Deps** … *can* emit … a **superset** of [the golden diagnostics list], so every code the deterministic run emits is permitted and the in-process gate and the real-host smoke **cannot disagree** on a code the fixture path actually emits." Two distinct guarantees are asserted here, and neither is backed by a mechanism.

First, the `golden ⊆ permitted` superset relation — the basis of the "cannot disagree" claim — is established only *by construction* in prose. Both the committed golden diagnostics list and the committed permitted-code list are checked-in artifacts, but no test asserts containment between them. A maintainer who adds a code to the golden diagnostics list (because the fixture path changed) without adding it to the permitted list silently breaks the relation: `H7a`'s in-process gate still passes (it exact-matches the golden list against the run), so `npm test` stays green while the "cannot disagree" property no longer holds.

Second, the union's *completeness* — that the permitted list captures every code each `Deps` slice can emit — is a manual, per-slice provenance obligation ("kept current under the same per-Deps-slice provenance obligation"), not a mechanically-verifiable property. There is no per-slice emittable-code registry to check against. If the manually-derived union omits a code a slice can legitimately emit, a benign emission of that code during the manual real-host smoke fails pass criterion (e) of `H4a` (`emitted ⊆ permitted`), which scores as a confirmed behavioural-divergence finding and a merge blocker — a false positive driven by a stale union rather than a real divergence.

## Solution Space

### Recommendation

Apply two edits to `docs/plan_topics/H7a-integration-acceptance.md`, both internal to the plan leaf files (the spec is read-only for this fix).

First, gate the `golden ⊆ permitted` subset relation. Add to `H7a`'s in-process Tests a verifiable assertion that the committed golden diagnostics list is contained in the committed permitted-code list. Both are checked-in artifacts alongside the fixture `.loom`: the gate reads both committed files and asserts containment — it must not re-derive either from a pipeline run. This turns "the in-process gate and the real-host smoke cannot disagree on a code the fixture path actually emits" into a gated property that reddens `npm test` the moment a maintainer adds a golden code without permitting it. The check compares the committed golden-diagnostics-list codes against the committed permitted-code-list codes — the two checked-in files — not against live-run output; read the committed artifacts, do not reconstruct them.

Second, soften the union-completeness wording. Revise the permitted-code-list Adds prose and Tests bullet so the "union of `loom/...` codes the slices in **Deps** … *can* emit" / "kept current under the same per-Deps-slice provenance obligation" language explicitly flags the union as a manually-maintained, best-effort set whose completeness is not mechanically verified — so a code outside the permitted set on the live smoke may indicate a stale union (a maintenance gap) rather than a real divergence. Align the parenthetical restatements in `H4a`'s criterion (e) and `H6a`'s evidence-record item to this softened wording. This sets correct expectations for the manual-smoke owner and the `H6a` release-gate auditor, who must exercise judgment when an emitted code falls outside the permitted set; the completeness gap remains unmechanized because no per-slice emittable-code registry exists to check the union against.

## Consequence

**Severity:** correctness

The plan promises a guarantee ("the in-process gate and the real-host smoke cannot disagree") that no gate enforces: a golden code outside the permitted set passes `npm test` undetected and surfaces only as a confusing manual-smoke result, while an incomplete manual union spuriously fails the smoke (a merge blocker) on a benign live emission. Two reasonable implementers diverge — one mechanizes the cheap subset check, the other trusts the prose and ships the relation ungated.

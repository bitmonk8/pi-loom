## Finding

`H7a` commits a **permitted-code list** alongside the fixture `.loom` and describes it as "the union of `loom/...` codes the slices in **Deps** … *can* emit … a **superset** of [the golden diagnostics list], so every code the deterministic run emits is permitted and the in-process gate and the real-host smoke **cannot disagree** on a code the fixture path actually emits." Two distinct guarantees are asserted here, and neither is backed by a mechanism.

First, the `golden ⊆ permitted` superset relation — the basis of the "cannot disagree" claim — is established only *by construction* in prose. Both the committed golden diagnostics list and the committed permitted-code list are checked-in artifacts, but no test asserts containment between them. A maintainer who adds a code to the golden diagnostics list (because the fixture path changed) without adding it to the permitted list silently breaks the relation: `H7a`'s in-process gate still passes (it exact-matches the golden list against the run), so `npm test` stays green while the "cannot disagree" property no longer holds.

Second, the union's *completeness* — that the permitted list captures every code each `Deps` slice can emit — is a manual, per-slice provenance obligation ("kept current under the same per-Deps-slice provenance obligation"), not a mechanically-verifiable property. There is no per-slice emittable-code registry to check against. If the manually-derived union omits a code a slice can legitimately emit, a benign emission of that code during the manual real-host smoke fails pass criterion (e) of `H4a` (`emitted ⊆ permitted`), which scores as a confirmed behavioural-divergence finding and a merge blocker — a false positive driven by a stale union rather than a real divergence.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** 052b019 — pi-loom plan: resolve "Real-host smoke pass criterion (e) names a permitted code set with no committed source" (2026-06-11, Thomas Andersen)
**History:** `H7a` was created earlier (e7e51cc), but the committed permitted-code list and its "superset" / "cannot disagree" wording, together with the manually-maintained per-Deps-slice provenance obligation and no mechanical containment check, were introduced as a unit in 052b019 when criterion (e) was given a committed source. Later edits (542395a widened the `Deps` union; eaa5000 touched the same line) preserved the overclaim without adding a gate.

## Solution Space

**Shape:** multiple

The finding carries two independent obligations: enforcing the subset relation (mechanizable) and correcting the completeness overclaim (a wording fix). They touch different fields and are resolved by different means, so they are presented separately. The spec is read-only for this fix — both obligations are internal to the plan leaf files.

### Option A — Gate the `golden ⊆ permitted` subset relation

**Approach.** Convert the by-construction "superset" / "cannot disagree" claim into a checked property in `H7a`'s in-process `npm test` gate.

**Plan edits.** In `docs/plan_topics/H7a-integration-acceptance.md`, add to `H7a`'s in-process Tests a verifiable assertion that the committed golden diagnostics list is contained in the committed permitted-code list. Both are checked-in artifacts alongside the fixture `.loom`, so the gate reads both committed files and asserts containment — it does not re-derive either from a run. This is what makes "the in-process gate and the real-host smoke cannot disagree on a code the fixture path actually emits" a gated property rather than a prose claim.

**Spec edits.** None.

**Pros.** Cheap (both inputs already committed); reddens `npm test` the moment a maintainer adds a golden code without permitting it; directly enforces the narrower "cannot disagree" guarantee.

**Cons.** Does not address completeness of the broader union (codes a slice can emit but the deterministic fixture does not).

**Risks.** None material; it is a containment check over two committed files.

### Option B — Soften the union-completeness overclaim

**Approach.** State that the union's completeness is a best-effort, manually-maintained provenance obligation rather than a mechanically-verified complete set.

**Plan edits.** In `docs/plan_topics/H7a-integration-acceptance.md`, revise the permitted-code-list Adds prose and Tests bullet so the "union of `loom/...` codes the slices in **Deps** … *can* emit" / "kept current under the same per-Deps-slice provenance obligation" language explicitly flags the union as a manually-maintained, best-effort set whose completeness is not mechanically verified — so a code outside the permitted set on the live smoke may indicate a stale union (a maintenance gap) rather than a real divergence. Optionally align the parenthetical restatements in `H4a`'s criterion (e) and `H6a`'s evidence-record item to the softened wording.

**Spec edits.** None.

**Pros.** Removes the completeness overclaim; sets correct expectations for the manual-smoke owner and the `H6a` release-gate auditor.

**Cons.** Leaves the completeness gap unmechanized — inherent, since no per-slice emittable-code registry exists to check the union against.

**Risks.** The smoke owner must exercise judgment when an emitted code falls outside the permitted set.

### Recommendation

Both obligations are required. Address **Option A** first — the mechanical subset assertion is bounded and lands on a stable baseline — then apply **Option B**'s wording softening on top. Edge case the implementer must watch: Option A's check compares the committed golden-diagnostics-list codes against the committed permitted-code-list codes (the two checked-in files), not against live-run output; it must read the committed artifacts, not reconstruct them from a pipeline run.

## Consequence

**Severity:** correctness

The plan promises a guarantee ("the in-process gate and the real-host smoke cannot disagree") that no gate enforces: a golden code outside the permitted set passes `npm test` undetected and surfaces only as a confusing manual-smoke result, while an incomplete manual union spuriously fails the smoke (a merge blocker) on a benign live emission. Two reasonable implementers diverge — one mechanizes the cheap subset check, the other trusts the prose and ships the relation ungated.

## Problem

`ERR-7` (watcher-time reload failures) pins the channel contract
(`loom-system-note`, `triggerTurn: false`, no final value, not subject to
cancellation) and its two arms — registry-swap failure
(`loom/runtime/registry-swap-failed`) and re-parse / re-merge diagnostic
(a re-emitted `loom/load/*` / `loom/parse/*` batch) — but neither
surfacing paragraph pins the operator-visible `content` string the
`loom-system-note` carries on the wire. The re-parse arm is implicitly
covered by the parse / load batch row of the per-variant
`display` / `content` table at
`runtime-event-channel.md#system-note-details-shapes`; the registry-swap
arm has no covering row (the only near match is the runtime-panic row,
whose `"loom /<name> aborted: <message>"` template requires an owning
invocation that a watcher-time swap lacks). Two implementers can therefore
ship visibly different bytes for the same registry-swap failure while
passing every existing assertion, and GOV-15's 1.x source-language
`content`-string equivalence has no left-hand side to compare against.

## Issue introduction

**Verdict:** present-since-inception (with a missed-opportunity refinement)
**Introducing commits:**
- `f5e89f4` (2026-06-04, "docs: enforce 100KB hard / 30KB recommended size cap on spec set") — first commit on the post-split layout to introduce the `<a id="err-7">` list item, as `**ERR-7.** watcher-time reload failures (per [Discovery](../discovery.md))`. No content template was specified at this point; the list item simply delegated to a then-non-existent target section.
- `d488eaa` (2026-06-06, "pi-loom spec: resolve \"ERR-7 defining anchor + collision payload field\"") — added the `<a id="watcher-time-reload-failures">` section in `discovery/package-and-settings.md` with the two-arm split (`Registry-swap failure` / `Re-parse / re-merge diagnostic`) and pinned the channel contract (`loom-system-note`, `triggerTurn: false`, no final value, not subject to cancellation). This commit was the natural opportunity to also pin the `content` template — parallel to the `Runtime panics` paragraph in the same `error-model.md` file, which pins `"loom /<name> aborted: <message>"` — but did not.

**History:** `git log --follow -- docs/spec_topics/errors-and-results/error-model.md` shows `f5e89f4` as the file's origin; `git log -L '/err-7/,+3:docs/spec_topics/errors-and-results/error-model.md'` confirms the `ERR-7` list item has carried no content-template clause across both touches. `git log --follow -- docs/spec_topics/discovery/package-and-settings.md` plus `git show d488eaa -- docs/spec_topics/discovery/package-and-settings.md` confirm `d488eaa` added the elaborating section with channel-contract pins and no content-template pin. The defect is therefore not the result of a regression — it has been the steady state of the `ERR-7` surface since the surface existed — but its persistence past `d488eaa` is the durable signal: that commit established the surrounding contract and would have been the in-context place to add the template.

## Solution approach

Pin a normative `content` template for both `ERR-7` arms at the
`#watcher-time-reload-failures` section in
`discovery/package-and-settings.md`, and add forward-cross-references from
the `ERR-7` list item at `error-model.md#err-7`. Close the per-variant
`display` / `content` table at
`runtime-event-channel.md#system-note-details-shapes` over the
watcher-time variants so the registry-swap arm is no longer covered only
by the panic row. The registry-swap arm's template is authored against the
failing path the `loom/runtime/registry-swap-failed` registry row already
names; the re-parse arm reuses the serialised diagnostic-line `content`
the existing parse / load batch row already defines.

## Solution constraints

- Out of scope: whether the re-parse / re-merge arm also emits
  `loom/runtime/registry-swap-failed` — a sibling finding owns that
  firing decision.
- When adding watcher-time rows to the per-variant table at
  `runtime-event-channel.md#system-note-details-shapes`, leave the
  `details.recovery` (fourth-shape) row to the sibling finding that
  closes it.

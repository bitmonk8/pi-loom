## Problem

The cross-format collision-candidate set is pinned by one blockquote seam in `registration-steps.md`, dual-anchored `#loom-1-0-seam-pi-owned-subagents-collision-source-set` (canonical arm) and `#v1-seam-pi-owned-subagents-collision-source-set` (back-compat alias). Three in-corpus cross-references — `registration-steps.md` step 3, `host-prerequisites.md` prerequisite (c-ii), and `future-considerations/surface-extensions.md`'s *Carrier* line — all link to the `#v1-seam-*` alias arm, violating GOV-25's canonical-arm-citation requirement. Their link text also diverges: two render "loom 1.0 cross-format collision source set" while `surface-extensions.md` renders "Pi Integration Contract — loom 1.0 seam: Pi-owned subagents collision source set", and neither matches the seam heading, so a reader cannot tell from link text that all three resolve to the same set.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `7a6cf96` (2026-05-07), `2e0dad2` (2026-05-29)
**History:** The corpus is git-tracked. The collision-source-set seam and its descriptive cross-reference link text ("…cross-format collision source set") were authored together in `7a6cf96` ("V1 seam-preservation MUSTs hidden inside the deferred-features narrative page"); the link text was a paraphrase rather than the heading verbatim from inception, so the link-text divergence is present-since-inception for that vector. The same-day prose rename `a6d82db` ("Prose V1 / V1.0 / V1.x rename to canonical spelling at non-closure callsites") relabelled heading prose `V1` → `loom 1.0` without aligning the cross-reference link text. The canonical-arm-citation gap was introduced by the dual-anchor migration `2e0dad2` ("HTML anchor dual-anchor authoring for v1- sites"), which added the canonical `loom-1-0-seam-*` arm alongside the legacy `v1-seam-*` arm per GOV-25–GOV-29 but did not repoint the pre-existing `#v1-seam-*` cross-references onto the new canonical arm. The defect is therefore the interaction of inception authoring with the later dual-anchor pass, not a single regressing commit.

## Solution approach

Repoint the three cross-references (`registration-steps.md` step 3, `host-prerequisites.md` prerequisite (c-ii), `surface-extensions.md` *Carrier* line) from `#v1-seam-pi-owned-subagents-collision-source-set` to the canonical `#loom-1-0-seam-pi-owned-subagents-collision-source-set` arm, satisfying GOV-25. Normalise the three link texts to a single form aligned with the seam heading.

## Solution constraints

- Preserve both `<a id>` anchors on the seam itself; the `v1-*` arm is a permanent back-compat alias under GOV-26 and must not be removed (retirable only under the narrow GOV-28 discharge).

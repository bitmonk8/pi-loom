## Finding

The `PIC-12` and `PIC-20` Tests bullets in both `V8d` (the `Clock`/`IdSource` implementation leaf) and its paired `V8d-T` test leaf restate the full `// allow-ambient:` comment-mechanism: the same-line comment format (`// allow-ambient: <primitive> — <seam>`), the registry-absence claim ("that comment is itself the allow-list entry; there is no separate registry"), the `H3a` identifier-keyed scan surface (`Date.now`, `performance.now`, `Date.prototype.getTime`, `setTimeout`, `clearTimeout`), and the indirect-form exclusion ("indirect forms are not mechanically detected and are owned by the *Per-phase TDD ritual* self-review step"). Every one of these clauses is already owned by `conventions.md` under the *No globals, statics, singletons* rule, which is the canonical home for the allow-list representation, the scan's enumerated timing-primitive list, and the conceded indirect-read residue.

The result is convention exposition embedded inside what should be a checkable test assertion. The actual assertion each bullet makes is small — for `PIC-12`, that no `src/**` module directly references a banned timing primitive outside the `WallClock` adapter; for `PIC-20`, that `IdSource.newInvocationId()` is the only direct `crypto.randomUUID` site — but a reader must extract it from a paragraph of mechanism prose that belongs elsewhere. The same prose is also carried in `H3a`'s Tests/Adds, so the mechanism description is currently maintained by hand in three locations.

## Solution Space

**Shape:** single

### Recommendation

In `V8d-T-clock-id-seams.md` and `V8d-clock-id-seams.md`, reduce the `PIC-12` and `PIC-20` Tests bullets to the observable assertion plus a citation to `conventions.md` *No globals, statics, singletons* and the corresponding spec REQ-ID, and strike the embedded mechanism exposition.

- `PIC-12` bullet (both leaves): keep the assertion that an architectural test finds no direct ambient timing reference in `src/**` outside the `WallClock` adapter, and cite `conventions.md` and `PIC-12`. Strike the `// allow-ambient: <primitive> — Clock` comment-format spelling, the "that comment is itself the allow-list entry; there is no separate registry" parenthetical, the enumerated `Date.now`/`performance.now`/`Date.prototype.getTime`/`setTimeout`/`clearTimeout` scan surface, and the indirect-form-exclusion clause — all already owned by `conventions.md`.
- `PIC-20` bullet (both leaves): keep the assertion that `IdSource.newInvocationId()` is the only direct `crypto.randomUUID` site, and cite `conventions.md` and `PIC-20`. Strike the same comment-format, registry-absence, scan, and indirect-form clauses.

Do not re-home the struck prose into `V8d` `Adds` — `conventions.md` already owns it, and copying it into `Adds` would create a fourth maintained copy. `V8d` `Adds` already states the "allow-list contract is not split" rationale for keeping `Clock` and `IdSource` together; a single pointer to `conventions.md` there is sufficient if any reference is wanted.

Edge case for the implementer: the trimmed PIC-12 assertion wording should stay consistent with however the related finding *"V8d — indirect ambient-access covered only by non-reproducible self-review"* resolves — if that finding narrows the enforced criterion to direct references only, the kept assertion text should say so rather than implying full indirect coverage.

## Consequence

**Severity:** advisory

The mechanism description lives in three hand-maintained places (`conventions.md`, `H3a`, `V8d`/`V8d-T`); a future edit to the convention must be mirrored by hand into the test bullets or they silently drift out of sync with their owner. No implementer is blocked, but the PIC-12/PIC-20 bullets read as convention exposition rather than an observable assertion, forcing a reader to reconstruct what the test actually checks.

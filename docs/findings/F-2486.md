## Problem

Several cross-references in the diagnostics cluster that pin teardown-emission timing relative to the `session_shutdown` sub-steps link to the bare cluster index `../pi-integration-contract.md` while their link text names a section — "**Per-step isolation** rule", "Extension entry point", "**idempotent** clause", "System notes" — that no longer exists on that page. `pi-integration-contract.md` is now a pure table-of-contents; the named normative prose lives in child topic files (`session-shutdown-semantics.md`, `extension-bootstrap-and-per-loom.md`, `drain-state-contract.md`, etc.). An implementer following these links lands on a contents list and must hunt across the PIC child files for the sub-step definition the row defers to. The affected links include, at minimum: in `code-registry-host.md`, the `session-shutdown-runtime-degraded` row's "Per-step isolation rule" and "idempotent clause" links and the "Extension entry point" links repeated across the four `session-shutdown-*` rows; in `diagnostic-shape.md`, the "System notes" / "Renderer registration" links; and in `placeholder-rendering-b.md`, the "Extension-bootstrap SDK failures" link.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** `f5e89f4` ("docs: enforce 100KB hard / 30KB recommended size cap on spec set")
**History:** `git log -S '**Per-step isolation** rule](../pi-integration-contract.md)'` over `docs/spec_topics/diagnostics/code-registry-host.md` returns `f5e89f4` as the sole introducing commit, and the link is unchanged since. `git show --stat f5e89f4` shows that commit split the 576KB monolithic `pi-integration-contract.md` into an index + 30 children and the 152KB `diagnostics.md` into an index + 7 children (including `code-registry-host.md`, created at +12 lines in that commit). Before the split, `pi-integration-contract.md` was the full page that contained the "Per-step isolation", "Extension entry point", "idempotent", and "System notes" sections, so the sibling links resolved. The split relocated that prose into child topic files but left the diagnostics-cluster links pointing at the now-emptied index. A later commit `c3de47a` ("relocate universal session_shutdown teardown to session-shutdown-semantics.md; sweep spec + plan citations") swept some citations but did not touch these anchorless index links.

## Solution approach

Repoint each anchorless `](../pi-integration-contract.md)` reference in the diagnostics cluster to the child topic file (plus anchor) that now owns the named section, so each teardown-seam reference resolves to one normative definition. The "Per-step isolation" links resolve to `../pi-integration-contract/session-shutdown-semantics.md` (add a stable anchor on the **Per-step isolation** paragraph — none exists today); "Extension entry point" to `../pi-integration-contract/extension-bootstrap-and-per-loom.md#extension-entry-point`; the "**idempotent** clause" to `../pi-integration-contract/drain-state-contract.md#idempotent-handler-clause`; "Extension-bootstrap SDK failures" to `../pi-integration-contract/extension-bootstrap-and-per-loom.md`; and "System notes" / "Renderer registration" to the child topic file that now owns the renderer-registration prose, adding a stable anchor there if absent. Where a target anchor does not yet exist, add an `<a id="...">` anchor on the owning paragraph and link to it.

## Solution constraints

- `../pi-integration-contract.md` is a pure table-of-contents index — resolve the links by retargeting to the child topic files; do not re-add prose or section anchors to the index.

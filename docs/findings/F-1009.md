## Finding

`docs/spec_topics/query/query-failure-and-repair.md` opens with a top-level `## Options surface` section whose entire body is a single blockquote tagged `> **loom 1.0 seam — per-call timeout / per-query overrides.**`. The body is explicit that the record it describes is "the record carrying per-call configuration **not surfaced to authors in loom 1.0**" — i.e. an internal runtime struct reserved as a forward-compatibility seam, not an author-visible options surface. The heading therefore promises an author-configurable options surface and delivers a non-normative implementation-stability note about an unobservable internal struct.

The placement compounds the heading miscue. The file's title is "Query failure and repair" and its other `##`-level sections are `Failure modes` and `Schema-validation respond-repair` — both concretely about how queries fail and how the runtime recovers. A seam-reservation note for an internal options record is not a failure-mode topic, and elevating it to peer status with `Failure modes` overstates its weight in the page's information hierarchy. The same pattern elsewhere in the corpus — the analogous seams on tool-calls and invocation pages — appears either as an inline `> **loom 1.0 seam — …**` blockquote (`tool-calls.md` line 42) or as a `> INV-3 …` blockquoted REQ-ID under a non-options section (`invocation.md` line 46). The query seam is the only one of the three pinned under a bare `## Options surface` H2 with no surrounding prose context.

The mis-placement also breaks one of the two callsites that point here. `docs/spec_topics/future-considerations/surface-extensions.md` cites "Query — Options surface" twice but links to `../query.md` (the parent index page, which has no `Options surface` section — only a `## Contents` list pointing at the four sub-pages). The third callsite (`docs/findings/F-0641.md`) cites the correct page with the correct anchor — `query/query-failure-and-repair.md#options-surface` — so any rename/demote must preserve a stable anchor that incoming references can be repointed to.

## Solution Space

**Shape:** single

### Recommendation

Demote the section to an inline blockquote matching the `tool-calls.md` and `invocation.md` precedents, in place — do not retitle, do not relocate to a different page. Concretely:

1. Delete the `## Options surface` H2 line in `docs/spec_topics/query/query-failure-and-repair.md`.
2. Move the existing seam blockquote underneath the first paragraph of `## Failure modes` (the paragraph beginning "A query never throws."), placed before the second paragraph ("The canonical declaration of `QueryError` …"). This co-locates the per-call-timeout / per-query-overrides seam with the only other seam on the page (the `loom 1.0 seam — pre-flight token-count nullability` blockquote inside `### Detection of `ContextOverflowError``), keeping both forward-compatibility callouts under the same failure-modes umbrella.
3. Add an HTML anchor immediately before the relocated blockquote so the incoming `query/query-failure-and-repair.md#options-surface` link from `docs/findings/F-0641.md` still resolves:
   ```html
   <a id="options-surface"></a><a id="v1-seam-query-options-surface"></a>
   ```
   Keep the second anchor too if the `v1-seam-…` convention is preferred — every other surface-extensions-cited seam on the page already double-anchors `loom-1-0-seam-…` and `v1-seam-…`.
4. Repoint both `*Anchored at:* [Query — Options surface](../query.md)` cites in `docs/spec_topics/future-considerations/surface-extensions.md` (lines 19 and 25) to `../query/query-failure-and-repair.md#options-surface`. This is the same fix the broken-link audit referenced in the `f5e89f4` commit message acknowledged as outstanding ("2 dangling refs remain").

Edge cases for the implementer:
- The `F-0641.md` reference uses `#options-surface` as the anchor — preserve that exact slug.
- Do not delete the `loom 1.0 seam — per-call timeout / per-query overrides` heading text inside the blockquote — it is what GOV-30 / GOV-31 use to count the 12 forward-compatibility seams (`overview-and-orientation.md` line 45, `governance/req-id-prefix-table-active-b.md` GOV-31). The convention is that the `> **loom 1.0 seam — <name>.**` opener inside the blockquote is what carries the inventory weight, not the H2 wrapper.
- Do not introduce a REQ-ID (no `<QRY-N>`) — none of the other surface-extension seams on this page carry one, and inventing one would alter the GOV-31 prefix tally.

## Consequence

**Severity:** cosmetic

Two readers' confusions, both minor: (1) an implementer scanning the failure-and-repair page for failure-modes content hits a section whose heading promises an unrelated author-facing options surface; (2) the two existing `surface-extensions.md` cross-references already mis-resolve to `query.md` (no such anchor) and a maintainer renaming the section without repointing them would silently keep the links broken. Neither produces wrong runtime behaviour.

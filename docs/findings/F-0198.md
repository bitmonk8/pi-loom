## Problem

Several `coverage-matrix.md` rows map one numbered REQ-ID to two or more closing leaves where each leaf closes a **distinct facet** of the obligation rather than re-closing the whole rule (`DISC-4`, `CIO-1`, `CIO-5`, `ERR-17`, `ERR-19`, `PIC-17, PIC-18`, `CNCL-4`). Neither closing-gate arm reconciles per-facet coverage: the H5a citing-test scan certifies a citing test exists for each mapped REQ-ID (one anywhere in the corpus satisfies it), and the H5d transitive-completeness arm requires only AT LEAST ONE listed leaf to be reachable and documents that a multi-leaf cell "stays green as long as one listed leaf is present." Consequently a split REQ-ID can ship with one named facet fully tested and the other facet carrying no citing test at all, with both gates green. Partial coverage passes as full coverage.

## Issue introduction

**Verdict:** present-since-inception (of the citing-test gate)
**Introducing commits:** `953e3fa` (2026-06-10, "pi-loom plan: resolve \"closing gate checks numbered-REQ-ID matrix mapping, not test assertion\""); reinforced by `617a3ad` (2026-06-16, "pi-loom plan: resolve \"H5a bundles gate arms spanning four data sources\"")
**History:** `git log -S "certifies that a citing test exists" -- docs/plan_topics/conventions.md` shows the citing-test reconciliation entered the corpus in a single commit, `953e3fa`. From that first introduction it was scoped per-REQ-ID ("certifies that a citing test exists for each mapped REQ-ID"), never per-facet — the joint-completeness gap is intrinsic to the granularity the gate has always had, not a regression a later edit introduced. The multi-leaf split rows it fails to fully reconcile (e.g. `DISC-4`, `ERR-17`) predate that commit in `coverage-matrix.md`. `617a3ad` later split the transitive-completeness arm into `H5d` with an explicit at-least-one multi-leaf rule ("stays green as long as one listed leaf is present"), entrenching the same per-row-not-per-facet semantics by design. No commit ever narrowed either arm to per-facet granularity.

## Solution approach

Add a checklist item to the `conventions.md` §"Loom 1.0 release-time residue inspection" enumeration (alongside items 1–6) covering a split-row joint-completeness sweep, clean when every facet of every multi-leaf `coverage-matrix.md` row carries its own distinct facet-naming citing test on that facet's closing leaf. Give the item a rule for deriving each row's facet partition (parenthetical facet annotations where present; otherwise the closing-leaf set minus any co-witness leaf). Add a pointing clause to the §REQ-ID discipline citing-test paragraph noting that per-REQ-ID citing-test existence does not certify per-facet completeness for multi-leaf rows, routing that residue to the new item.

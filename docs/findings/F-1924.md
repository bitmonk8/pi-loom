## Problem

`V9b` is named the single declaration site for the test-only `ReloadFailureInjector` interface, but its `Adds` field names `injectReloadFailure` without stating its contract: no parameter shape, no discriminator routing a single call onto one of the three failure arms (registry-swap, `.loom`/`.warp` re-parse, settings-re-merge), no per-arm payload, and no return type. Two consumer leaves bind against this same shape — `V10d` contributes the settings-re-merge arm and explicitly does not declare its own interface, and `V4g` imports the interface from `V9b` to exercise `ERR-7`. Because the single declaration site never fixes the shape, the `V10d` and `V4g` implementers would each invent a divergent signature, discriminator, and payload, producing a seam whose three arms do not compose through one method.

## Issue introduction

**Verdict:** present-since-introduction
**Introducing commits:** `d64dce5` (`pi-loom plan: resolve "Watcher-time reload failure-injection seam under-specified and ungated"`, 2026-06-11)
**History:** The named `ReloadFailureInjector.injectReloadFailure` interface was coined in `d64dce5`, which replaced an unnamed "watcher-time reload failure-injection seam" prose phrase with the named test-only interface and method. That commit introduced the method name but never pinned its parameter shape, arm discriminator, per-arm payload, or return type. Later refining commits repeatedly touched the seam and its consumers without closing the gap: `fdce2a2` (made `V9b` the single declaration site and added `V10d`'s settings-re-merge arm), `f14e929` (split `V10c` into settings-merge and reload-debounce leaves), and `324813d` (split the `ERR-7` scenario out of `V4e`'s routing checks). The signature gap has existed continuously since the interface was first named.

## Solution approach

In `V9b`'s `Adds` field, pin the `ReloadFailureInjector.injectReloadFailure` contract its two consumers bind against: the parameter (a synthetic failure descriptor), a closed discriminator over exactly the three arms `{ registry-swap, `.loom`/`.warp` re-parse, settings-re-merge }`, the per-arm failure payload, and the return type. State which arm each leaf contributes or exercises (`V9b` wires registry-swap and re-parse, `V10d` contributes settings-re-merge, `V4g` exercises `ERR-7`) so the discriminator's three values are unambiguous. The exact TypeScript form of the descriptor and return type is the implementer's.

## Solution constraints

- Out of scope: the `V10d` settings-re-merge arm and `V4g`'s `ERR-7` test wiring — `V9b` is the sole declaration site and consumers bind against it without declaring their own interface.
- The spec is read-only for this fix.

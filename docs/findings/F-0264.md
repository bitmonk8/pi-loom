## Problem

The category-8 Tests bullet in `V7c-placeholder-rendering.md` and its paired
`V7c-T-placeholder-rendering.md` — "The category-8 host-derived tail keeps
prefix/suffix byte-identical with an implementation-defined tail." — is
self-contradictory: it calls the host-derived part a "tail" yet asserts a
byte-identical "prefix/suffix". Spec §8 in `placeholder-rendering-b.md`
(*Host-derived freeform-tail placeholders*) pins the canonical row
`loom/load/host-incompatible` (`node-floor`) with the implementation-defined
segment interpolated mid-string and a byte-identical suffix after, so "tail"
mislocates the boundary. The bullet also leaves three authoring questions
unanswered — which §8 row to exercise, what to mock the host string to, and
which assertion form to use — and §8 classifies a full-*Message*-string
equality assertion against a category-8 row as non-conformant, so an author
reading only the bullet could write the forbidden assertion shape.

## Issue introduction

narrowed it.

## Solution approach

Rewrite the category-8 Tests bullet in both `V7c-placeholder-rendering.md` and
`V7c-T-placeholder-rendering.md` so it names a concrete §8 row, fixes the
mocked host-supplied value, and pins the spec's anchored partial-match
assertion form. Use the spec's canonical row `loom/load/host-incompatible`
(`node-floor`) and its normative vector
`^host incompatible \(node-floor\): observed (.*), required >=22\.19\.0$`,
sourced from §8 and the *Category 8 prefix/suffix anchoring* edge case in
`placeholder-rendering-b.md`. Drop the "tail" framing: the host-derived value
is interpolated between a byte-identical prefix and a byte-identical suffix,
and a full-*Message*-string equality assertion is non-conformant for
category-8 rows.

## Solution constraints

- Out of scope: `placeholder-rendering-b.md` and all spec topics — the spec is
  read-only for this fix; it is the source of truth for the §8 contract.

## Problem

The `loom/load/missing-source` row in the `loom/load/*` registry table (`docs/spec_topics/diagnostics/code-registry-load.md`) carries `E/W` in the *Sev* column. Per the `diagnostic-shape.md#column-legend` definition, `E/W` asserts a severity resolved per case to either `E` or `W` by the row's own trigger prose. For this row the trigger admits only two outcomes — `error` for explicit references and no diagnostic at all for conventional locations ("warning never") — confirmed by the discovery failure-modes table (`discovery/discovery-sources.md`, DISC-2), whose "Missing path" cells are all `error` or `silent`. The `W` arm is therefore unreachable; the only severity this code can resolve to is `E`.

## Issue introduction

**Verdict:** introduced-by-commit
**Introducing commit:** `5b772b6` — "pi-loom spec: resolve \"Discovery source failure-mode policy is unspecified\"" (Mon May 4 2026)
**History:** The corpus is git-tracked. `git log -S 'warning never'` and `git log -S '| \`loom/load/missing-source\` | E/W |'` both bottom out at `5b772b6`, the commit that first authored the discovery-source diagnostic rows together with the per-source `E/W` legend (then in `spec_topics/diagnostics.md`; relocated verbatim into `diagnostics/code-registry-load.md` by the `f5e89f4` size-cap split and the `a37d4ef` legend relocation, neither of which altered the row's `E/W` value). In that same commit the row simultaneously gained the `E/W` token and the "warning never" trigger clause: `missing-source` was given the shared `E/W` shorthand alongside its genuinely-bivalent siblings (`unreadable-source`, `wrong-type-source`) even though its own trigger admits no warning resolution. The contradiction has been present since the row's inception; no later commit introduced or widened it.

## Solution approach

Change the *Sev* column of the `loom/load/missing-source` row from `E/W` to `E`. The existing trigger prose already states the per-source disposition and needs no change.

## Solution constraints

- Out of scope: the sibling rows `loom/load/unreadable-source` and `loom/load/wrong-type-source`, whose triggers genuinely resolve to either severity, and the `column-legend` `E/W` definition itself, which is sound.

## Finding

The Clock bullet in `implementation-notes.md` (and the parallel clause in PIC-12 at `host-interfaces-services.md`) carries a normative `MUST NOT` enumerated as a list of host symbols:

> The runtime MUST NOT call `Date.now`, `performance.now`, `Date.prototype.getTime`, or the global `setTimeout` / `clearTimeout` outside the `WallClock` adapter.

That obligation is phrased as a source-code shape (a set of identifiers a call expression may not name), not as runtime-observable behaviour. No black-box conformance test against the loom runtime can distinguish a build that violates the list from one that satisfies it, because the observable determinism guarantee the ban exists to protect — monotonic `now()`, fake-injectable `wallNow()`, deterministic deadline + registration firing under `FakeClock.advance(ms)`, per-runtime independence — is already pinned by the `Clock` / `FakeClock` interface contract one paragraph earlier. A conforming implementation that routes one stray `Date.now()` call through some other deterministic wrapper would satisfy every observable consequence the spec relies on, yet fail this MUST on inspection alone.

The spec itself acknowledges the gap: the only verification mechanism it cites — "the reference implementation enforces this ban with a build-time grep-test" — is explicitly labelled non-normative. A normative MUST whose sole compliance witness is a non-normative project convention is not a conformance obligation; it is an implementation note miscast as one.

## Solution Space

**Shape:** single

### Recommendation

Demote the host-symbol enumeration to a non-normative implementation convention at both sites (`implementation-notes.md` Clock bullet and PIC-12 in `host-interfaces-services.md`). Keep the determinism contract normative by stating it in observable terms:

- Normative (replace the symbol-list MUST): "All wall-clock and timer access by the runtime MUST go through the injected `Clock` seam. Production wiring is the `WallClock` adapter; tests use `FakeClock`, whose `advance(ms)` deterministically fires due timers in deadline + registration order with the semantics already given in this section."
- Non-normative (move the symbol list and the grep-test together): "*Implementation convention.* The reference implementation routes `Date.now`, `performance.now`, `Date.prototype.getTime`, and the global `setTimeout` / `clearTimeout` through the `WallClock` adapter only, and a build-time grep-test enforces that convention; both the symbol list and the grep-test are implementation choices, not conformance points."

Edge cases the implementer must watch:

- The `WallClock` exemption text ("the grep-test ban below already exempts the `WallClock` adapter, so its `wallNow()` delegation to `Date.now()` is permitted") in PIC-12 currently leans on the MUST being normative; after demotion, the exemption sentence should be retitled as part of the convention block, not a carve-out from a normative ban.
- The companion finding "implementation-notes.md — Clock DI-seam MUST NOT misplaced" wants this same constraint moved to PIC-12. If both findings are taken, do the demotion first at PIC-12 (the owner site) and have the `implementation-notes.md` bullet reference PIC-12 instead of restating; this avoids carrying the over-prescriptive MUST through a copy step.
- The `H3a` / `V8d` architectural test that scans `src/**` for direct references to the banned identifiers remains a legitimate project convention; the plan should re-attribute its rationale from "spec PIC-12 MUST" to the new convention block, but the test itself does not need to change.

## Consequence

**Severity:** advisory

A second loom-runtime implementation that respects the `Clock`/`FakeClock` interface and its observable determinism is fully interoperable with everything else in the spec yet may be judged non-conformant on this single MUST because of a single un-routed `Date.now()` reading wall-clock through a different deterministic wrapper. The risk is editorial overreach, not divergent behaviour — the determinism guarantee is already carried elsewhere, so the implementation surface that matters stays nailed down whether or not this MUST stands.

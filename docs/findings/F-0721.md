## Problem

The Clock `MUST NOT` clause in `implementation-notes.md` (and the parallel clause in PIC-12 at `host-interfaces-services.md`) bans a set of host symbols (`Date.now`, `performance.now`, `Date.prototype.getTime`, the global `setTimeout` / `clearTimeout` outside the `WallClock` adapter). That is a source-code shape, not runtime-observable behaviour: no black-box conformance test can distinguish a violating build from a conforming one, because the determinism guarantee the ban protects is already pinned by the `Clock` / `FakeClock` interface contract one paragraph earlier. The only verification mechanism the spec cites — the build-time grep-test — is itself labelled non-normative. A normative MUST whose sole compliance witness is a non-normative project convention is an implementation note miscast as a conformance obligation.

## Issue introduction

**Verdict:** present-since-introduction (refined and re-affirmed since)
**Introducing commits:**
- `bc50e69` (2026-05-05) — "pi-loom spec: resolve 'H2 Clock seam — watcher debounce, scanPackagesTimeoutMs, RuntimeEvent occurred_at'" first authored the `**Clock.**` bullet in `spec_topics/implementation-notes.md` with the symbol-list `MUST NOT … outside the WallClock adapter; H2 ships a grep-test that enforces this.`
- `9183961` (2026-06-04) — "pi-loom spec: resolve 'RuntimeEvent.occurred_at source clock contradicts the Clock.now() monotonic pin'" extended the same clause by adding `wallNow()` (no change to the MUST shape).
- `8362824` (2026-06-04) — "pi-loom spec: resolve 'Demote over-prescriptive and non-testable MUSTs on unobservable implementation structure'" reviewed the broader class of mechanism-shaped MUSTs and explicitly chose to *keep* this ban normative, only relabelling the grep-test mechanism as non-normative. The current "the reference implementation enforces this ban with a build-time grep-test; the grep-test mechanism is non-normative" wording in both `implementation-notes.md` and `host-interfaces-services.md` (PIC-12) dates from that commit.

**History:** the symbol-list MUST has been in the spec since the Clock seam first landed; the verification-mechanism caveat was added when a prior reviewer flagged the same class of defect but stopped short of demoting this particular MUST. The present finding revisits that 8362824 decision.

## Solution approach

Demote the host-symbol enumeration to a non-normative implementation convention at both the `implementation-notes.md` Clock bullet and PIC-12 (`#pic-12` / `#clock--fakeclock-interface` in `host-interfaces-services.md`), and keep the determinism contract normative by restating it in observable terms grounded in the existing `Clock` / `FakeClock` interface. Retitle the `WallClock` `wallNow()` / `Date.now()` exemption sentence in PIC-12 as part of that convention block rather than as a carve-out from a normative ban.

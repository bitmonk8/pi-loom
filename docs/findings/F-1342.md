## Finding

The glossary entry for `GOV-N (governance rule)` describes the live range as "currently running through `GOV-23`, with some IDs retired". The governance pages now coin and use `GOV-24` … `GOV-31`, so the literal upper bound is wrong by eight IDs. The entry's own next clause already tells readers to "cite `governance.md` for the authoritative range rather than a fixed upper bound" — i.e. the parenthetical figure is exactly the kind of fixed upper bound the entry warns against. As long as the figure stays, the glossary contradicts both the governance pages and its own advice, and any future GOV addition reopens the drift.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `96cfc08` (2026-06-04, the literal figure); `1d773d6` (2026-06-04, GOV-24), `8e3ccb4` (2026-06-05, GOV-25…GOV-29), `fe8aa5c` (2026-06-05, GOV-30, GOV-31) for the additions that made it stale.
**History:** `git log -L` on the entry shows `96cfc08` rewrote the bullet to the current prose and introduced the "currently running through `GOV-23`" parenthetical, accurate when it landed. Within ~28 hours three commits added `GOV-24`…`GOV-31` to the governance pages without touching the glossary line; nothing locks the glossary parenthetical to the live range, so the figure went stale on the first such commit (`1d773d6`) and grew staler. The defect is a missed lock-step between the glossary entry and the governance prefix tables, not a defect in any single commit.

## Solution Space

**Shape:** single

### Recommendation

Delete the concrete "currently running through `GOV-23`" figure from the parenthetical, leaving only the directive that readers consult `governance.md` for the authoritative range. The entry already names the Retired REQ-IDs table and tells readers to cite `governance.md`, so removing the figure preserves every useful piece of information while eliminating the drift surface. Do not replace the figure with an updated one (`GOV-31`) — that rearms the same drift mechanism, and the entry already argues against fixed upper bounds. One-line edit, touches no other file.

## Consequence

**Severity:** advisory

A reader auditing the GOV namespace from the glossary stops at `GOV-23` and never discovers `GOV-24`…`GOV-31`; the contradiction with the entry's own "cite `governance.md`" advice undermines its credibility, and the drift recurs on every future GOV addition. No implementer-facing obligation breaks.

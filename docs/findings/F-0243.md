## Problem

The **Ships when** clause of leaf `V4a` (`docs/plan_topics/V4a-match-result.md`) reads "a `match` whose arms cover none of the six pattern forms raises the runtime `loom/runtime/match-error` panic". Making the *arms* the subject of "cover" admits a structural reading — a `match` whose arm patterns use none of the six recognised forms — rather than the intended runtime reading where a scrutinee value at runtime falls through every arm. The spec frames this trigger as a scrutinee fall-through at runtime, and the leaf's own **Tests** bullet already uses the correct framing, so the gate is phrased more loosely than the test it gates. An author reading only **Ships when** could build a test exercising a structural property of the arm set and still claim the gate satisfied.

## Issue introduction

**Verdict:** introduced-by-prior-fix
**Introducing commits:** `516fd86f437cab9dfd9f7200f9e9107ca8344b1d` (2026-06-11, "pi-loom plan: resolve \"V4a Ships-when names match exhaustiveness vs no-static-check\"")
**History:** `V4a-match-result.md` was created at `c6a664e` with the **Ships when** clause reading simply "…and `match` exhaustiveness." Commit `516fd86`, resolving a prior review finding that the gate failed to distinguish runtime exhaustiveness from the absence of a static check, expanded the clause to its current "a `match` whose arms cover none of the six pattern forms raises the runtime `loom/runtime/match-error` panic (loom 1.0 performs no static exhaustiveness check)" — correctly adding the no-static-check qualifier but introducing the arms-as-subject ambiguity flagged here. The line has not changed semantically since (`8c5da7b` only added the `loom/parse/` prefix to a neighbouring code). This is a defect introduced while fixing an earlier defect in the same clause.

## Solution approach

Rewrite the **Ships when** clause in `docs/plan_topics/V4a-match-result.md` so the runtime scrutinee value (a value that matches no arm) is the subject that triggers the panic, matching the spec's canonical wording and the leaf's own **Tests** bullet. Retain the existing "(loom 1.0 performs no static exhaustiveness check)" parenthetical.

## Solution constraints

- Out of scope: the spec is read-only for this fix; it already states the canonical runtime-scrutinee trigger.

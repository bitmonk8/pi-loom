## Finding

The session-only degraded-state cluster coins and consistently uses **"session-only reasons"** (also "session-only triplet", "the three *session-only reasons*") as the stable name for the `{"new","resume","fork"}` group — the reasons under which Pi swaps the user session without tearing the extension runtime down. The complement group `{"quit","reload"}` — the reasons whose runtime the host tears down regardless — has no corresponding coined term anywhere in the normative text.

As a result, prose names the complement inconsistently across the cluster: the bare literal `{"quit","reload"}` in the *Predicate split* table and the `capturedEventReason ∉ {"quit","reload"}` seam (`session-only-degraded-state.md`); the adjective phrase "always-tear-down reasons" in acceptance-criterion (c) and "always-tear-down branch" in the *Emergent fast-path* (`session-only-degraded-state.md`) and acceptance-criterion (a) (`drain-state-contract.md`); and the descriptive clause "every shutdown whose runtime the host will tear down regardless" in the `"shutting-down"`-arm definition (`session-shutdown-semantics.md`). None of these is registered as a term of art, so the partition that is the load-bearing axis of the whole degraded-state branch is named asymmetrically: one half is a coined noun phrase, the other is a rotating set of literals and ad-hoc descriptions.

The defect is purely terminological — the behavioural predicate is pinned exactly by the literal set `{"quit","reload"}`, so no implementer is at risk of diverging on which reasons tear down. The cost is reading and maintenance burden: a reader cannot grep one canonical name for the complement, and a future editor has no single term to keep consistent.

## Issue introduction
**Verdict:** single-commit
**Introducing commits:** 52b9101 — pi-loom spec: resolve "T24 — Fork-reason watcher closure leaves the extension in an unspecified, silently degraded state" (2026-05-11, Thomas Andersen)
**History:** The session-only degraded-state machinery entered the corpus in 52b9101 (then in the monolithic spec, later split into `spec_topics/` by f5e89f4 / 81bf52a). That commit coined "session-only reasons"/"session-only triplet" for the `{"new","resume","fork"}` group while referring to the `{"quit","reload"}` complement only via the bare literal and the descriptive "always-tear-down half". The naming asymmetry was present from the moment the concept was coined; no later commit introduced or widened it.

## Solution Space

**Shape:** single

### Recommendation

Adopt a symmetric coined term — **"teardown reasons"** (or "always-teardown reasons") — for the `{"quit","reload"}` group. Introduce it at the same site where "session-only reasons" is first defined in `session-only-degraded-state.md`, defining it as the partition complement (the reasons whose runtime the host tears down after the handler returns). Then use it consistently in:

- the *Predicate split* table and the `capturedEventReason ∉ {"quit","reload"}` seam description in `session-only-degraded-state.md`;
- acceptance criterion (c) ("Negative — always-tear-down reasons") and the *Emergent fast-path* "always-tear-down branch" in `session-only-degraded-state.md`;
- the `"shutting-down"`-arm definition in `session-shutdown-semantics.md` ("every shutdown whose runtime the host will tear down regardless — `"quit"`, `"reload"`, …");
- acceptance criterion (a)'s "always-tear-down branch" in `drain-state-contract.md`.

The coined term is naming sugar over the partition; the normative `{"quit","reload"}` literal stays the source of truth at the predicate sites (the term names the set, it does not replace the literal where the predicate is pinned). Edge case to preserve: the snapshot-failure and unknown-reason paths land on the degraded (non-teardown) arm via the `∉ {"quit","reload"}` complement check even though their reasons are not literally session-only, so the new term must be defined as the literal complement, not as a synonym for "session-swap".

## Consequence

**Severity:** advisory

The two-partition structure is the load-bearing axis of the degraded-state branch, yet only one half is named. Implementers still produce correct behaviour (the predicate is pinned by the literal set), but the inconsistent naming of the complement imposes a reading burden and gives future editors no canonical term to keep aligned as the cluster evolves.

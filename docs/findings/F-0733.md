## Problem

Two sibling files under `docs/spec_topics/` both carry an H1 beginning "Overview and …" and a `## Overview` first child section: `overview.md` ("Overview and Conceptual Model") and `overview-and-orientation.md` ("Overview and orientation"). The two pages are not peers — `overview-and-orientation.md` is the reading-order front door (the first entry in `spec.md`'s table of contents), while `overview.md` owns canonical conceptual definitions (the *loom* file unit, prompt vs. subagent mode, query-and-await, Scope of a loom file) reached only via deep-links. Neither the heading text nor the file basenames distinguish the two: `overview` is a strict prefix of `overview-and-orientation`, so file-pickers and link-completion show them adjacently with near-identical labels, and a reader landing on either page from a deep-link cannot infer which is subordinate.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `fecb504` (2026-05-04, "Split spec.md and plan.md into per-topic / per-phase files") created `spec_topics/overview.md` with the H1 "Overview and Conceptual Model"; `f5e89f4` (2026-06-04, "docs: enforce 100KB hard / 30KB recommended size cap on spec set") created `spec/overview-and-orientation.md` with the H1 "Overview and orientation" as a child of `spec.md` in a separate directory; `81bf52a` (2026-06-11, "docs: collapse spec/ section pages into spec_topics/") moved `overview-and-orientation.md` into `spec_topics/`, making the two files siblings for the first time.
**History:** Neither of the creating commits is itself defective: each H1 was unambiguous in its original location, and the `spec/` vs `spec_topics/` split kept the two pages in distinct architectural roles. The collision is the joint consequence of (a) `f5e89f4` picking "Overview and orientation" without checking the existing `spec_topics/overview.md` heading and (b) `81bf52a` collapsing the two directories without renaming either page to remove the resulting sibling overlap. `git log -S "Overview and orientation"` shows only the two move/split commits touch the string; no later commit revisits the naming.

## Solution approach

Rename `overview.md` to a non-overlapping H1 and basename so the page reads as structurally distinct from `overview-and-orientation.md` — for example retitle the H1 from "Overview and Conceptual Model" to "Conceptual Model" and rename the file to `conceptual-model.md`, which breaks both the heading collision and the `overview`/`overview-and-orientation` stem-collision in one move. Update every inbound link site that targets `overview.md` — `language-and-architecture.md`'s Reading order, `glossary.md`, `pi-integration-contract/subagent.md`, and the `governance/req-id-prefix-table-active-b.md` prefix-table row — and the plan-side path citations.

## Solution constraints

- Preserve the deep-link anchors `#scope-of-a-loom-file`, `#code-and-model`, and `#query-and-await` across the rename — inbound cross-references consume them.

## Finding

Two sibling files under `docs/spec_topics/` both carry an H1 that begins "Overview and …": `overview-and-orientation.md` ("# Overview and orientation") and `overview.md` ("# Overview and Conceptual Model"). Both also contain a `## Overview` H2 as their first child section, so heading text alone cannot tell a reader which page is the orientation entry point and which is the conceptual deep-dive.

The two pages are not peers in the architecture. `spec.md`'s top-level table of contents links `overview-and-orientation.md` as the first reading-order entry; it is the front door. `overview.md` is reached indirectly — via `language-and-architecture.md` Reading order ("[Overview and Conceptual Model](./overview.md) — what a loom is, query-and-await, prompt vs. subagent mode."), via `glossary.md` ("See: [Overview — Scope of a loom file](./overview.md)"), and via `pi-integration-contract/subagent.md` — yet it owns canonical conceptual definitions (the *loom* file unit, prompt vs. subagent mode, query-and-await, Scope of a loom file) that several other pages reference by anchor. A reader who lands on either page from a deep-link cannot infer the subordinate relationship from the title, and a corpus-wide grep for "Overview" yields two collisions per page.

The collision is structural rather than verbal: the file basenames also stem-collide (`overview` is a strict prefix of `overview-and-orientation`), so editor file-pickers and link-completion tools show the two files adjacently with near-identical labels.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `fecb504` (2026-05-04, "Split spec.md and plan.md into per-topic / per-phase files") created `spec_topics/overview.md` with the H1 "Overview and Conceptual Model"; `f5e89f4` (2026-06-04, "docs: enforce 100KB hard / 30KB recommended size cap on spec set") created `spec/overview-and-orientation.md` with the H1 "Overview and orientation" as a child of `spec.md` in a separate directory; `81bf52a` (2026-06-11, "docs: collapse spec/ section pages into spec_topics/") moved `overview-and-orientation.md` into `spec_topics/`, making the two files siblings for the first time.
**History:** Neither of the creating commits is itself defective: each H1 was unambiguous in its original location, and the `spec/` vs `spec_topics/` split kept the two pages in distinct architectural roles. The collision is the joint consequence of (a) `f5e89f4` picking "Overview and orientation" without checking the existing `spec_topics/overview.md` heading and (b) `81bf52a` collapsing the two directories without renaming either page to remove the resulting sibling overlap. `git log -S "Overview and orientation"` shows only the two move/split commits touch the string; no later commit revisits the naming.

## Solution Space

**Shape:** multiple

### Option A — Rename `overview.md`'s heading (and filename) to a non-overlapping title

**Approach.** Retitle `overview.md`'s H1 from "Overview and Conceptual Model" to "Conceptual Model" and rename the file to `conceptual-model.md`. Update the four inbound link sites (`language-and-architecture.md` Reading order, `glossary.md` `loom (file unit)`, `pi-integration-contract/subagent.md`, `governance/req-id-prefix-table-active-b.md` prefix-table row) and the three plan-side citations (`M-minimal-slash-command.md`, `M-T-minimal-slash-command.md`, plus any other path mentions). Anchors inside the file (`#scope-of-a-loom-file`, `#code-and-model`, `#query-and-await`) are preserved.

**Spec edits.** H1 retitle in `overview.md`; file rename; ~7 link path/text rewrites across `docs/spec_topics/` and `docs/plan_topics/`.
**Pros.** Eliminates both the heading collision and the filename stem-collision in one move. Each page's title now states its sole purpose. No content reshuffle.
**Cons.** File rename is the largest blast radius among the options (touches plan files, not just spec). Loses the explicit "Overview" word from the deep-dive page's title, though the H2 `## Overview` remains.
**Risks.** A missed citation produces a broken link; the `docs/_tools/check_links.py` validator already used in `f5e89f4`/`81bf52a` catches this.

### Option B — Retitle only the H1, keep the filename

**Approach.** Change `overview.md`'s H1 from "Overview and Conceptual Model" to "Conceptual Model" and leave the filename as `overview.md`. Update the three inbound link sites whose link **text** says "Overview …" (`language-and-architecture.md`, `glossary.md`, `subagent.md`) to use "Conceptual Model" instead. The prefix-table row and plan-side `[`../spec_topics/overview.md`]` references are unchanged.

**Spec edits.** H1 retitle in `overview.md`; ~3 link-text rewrites (paths untouched).
**Pros.** Smallest diff. No file rename, no plan touched. Solves the heading collision; pages that grep on titles get distinct hits.
**Cons.** The filename stem-collision (`overview` is a prefix of `overview-and-orientation`) survives, so file-pickers and shell tab-completion still show the two pages adjacently with near-identical labels.
**Risks.** Minimal; link-text rewrites are local.

### Option C — Merge `overview.md` into `overview-and-orientation.md`

**Approach.** Fold `overview.md`'s Conceptual Model section into `overview-and-orientation.md` as a new H2 (e.g. `## Conceptual model`) between the existing `## Overview` and `## Orientation`. Delete `overview.md`. Repoint all inbound links (including anchored ones like `#scope-of-a-loom-file`) to the merged page.

**Spec edits.** Append ~50 lines to `overview-and-orientation.md`; delete `overview.md`; rewrite ~7 inbound links and 3 anchored deep-links; update the `governance/req-id-prefix-table-active-b.md` row.
**Pros.** Single canonical orientation page; eliminates the collision by construction. Matches the architecture spec.md actually links (only `overview-and-orientation.md` is reachable from the TOC).
**Cons.** Pushes `overview-and-orientation.md` closer to the 30KB recommended size cap that `f5e89f4` enforced — the conceptual-model content is ~3KB and the host page is already dense; would warrant a size check before merging. Largest content edit.
**Risks.** Anchors `#scope-of-a-loom-file`, `#code-and-model`, `#query-and-await` are deep-linked from multiple files; each must survive the move byte-for-byte (preserve `<a id="…">` markers or add them at the merged location). The `docs/_tools/fingerprint.py` anchor-preservation tooling that `f5e89f4` introduced exists for exactly this case.

### Recommendation

Take **Option A**. The finding's underlying defect is structural (two sibling files whose names and titles read as the same concept), so the durable fix is to make the two pages structurally distinct — rename both the heading and the filename. Option B leaves the filename stem-collision in place, which is half the defect, and Option C is more invasive than the problem warrants and risks pushing the orientation page over the corpus's own 30KB size budget. Implementer notes: preserve the `<a id="scope-of-a-loom-file">`, `<a id="code-and-model">`, and `<a id="query-and-await">` anchors during the file rename; run `docs/_tools/check_links.py` before commit; the prefix-table row in `governance/req-id-prefix-table-active-b.md` is easy to miss because it lists the file with no IDs.

## Consequence

**Severity:** advisory

Readers arriving at either page from a deep-link cannot tell from the heading which is the entry point and which is the subordinate concept page, and grep on "Overview" silently double-matches. No normative content is ambiguous; navigation and discoverability degrade.

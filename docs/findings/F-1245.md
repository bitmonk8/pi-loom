## Finding

The single cross-cutting governance artefact defined in `docs/plan_topics/real-host-smoke-gate.md` — the non-automated pre-merge gate that drives `H7a`'s committed multi-feature fixture `.loom` against a live Pi host — is referred to under four different names across the plan corpus:

- **`manual real-host smoke governance protocol`** — `real-host-smoke-gate.md` (title-body, twice), `H4a` (the scope-exclusion note), and `plan.md`'s **Release gate** section.
- **`manual real-host smoke run`** — `real-host-smoke-gate.md` (the concrete execution), and `H6a` (link text and `manual real-host smoke acceptance` prose).
- **`pre-merge real-host smoke gate`** — `H6a`'s **Ships when** block, as the link text on a link to the same `real-host-smoke-gate.md` target.
- **`real-host smoke gate`** (canonical, matching the filename) — the `real-host-smoke-gate.md` H1 title.

The drift is sharpest inside `H6a`: two distinct link texts — `[**manual real-host smoke run**]` and `[pre-merge real-host smoke gate]` — point at the same `./real-host-smoke-gate.md` target within one leaf, so a reader cannot tell from the link text whether they denote one artefact or several.

There is a defensible two-level distinction buried in the drift — the *standing gate/obligation* versus *one execution* of it (`real-host-smoke-gate.md` itself separates "this note defines the … governance protocol" from "the concrete … gate … is a manual real-host smoke run"). The defect is that the head noun for the standing concept is not held constant ("governance protocol" / "pre-merge … gate" / "gate"), so the legitimate gate-vs-run layering is obscured rather than expressed.

## Solution Space

**Shape:** single

### Recommendation

Standardise the standing concept on **`real-host smoke gate`** (the head noun that matches the filename `real-host-smoke-gate.md`), qualified adjectivally (`manual` / `pre-merge`) where context needs it, and reserve **`run`** strictly for one execution of that gate (e.g. "a run of the real-host smoke gate" / "the recorded run"). This keeps the legitimate gate-vs-run distinction while collapsing the synonyms for the standing artefact.

Concrete edits:

- `docs/plan_topics/real-host-smoke-gate.md`: replace the two `manual real-host smoke governance protocol` occurrences with `manual real-host smoke gate`. Keep `manual real-host smoke run` where it names the concrete execution event.
- `docs/plan.md` Release gate paragraph: change `The **manual real-host smoke governance protocol** the bump/merge contributor runs …` to `The **manual real-host smoke gate** the bump/merge contributor runs …`.
- `docs/plan_topics/H4a-factory-shell-and-harness.md`: change `The manual real-host smoke governance protocol that gates a double-vs-real divergence …` to `The manual real-host smoke gate that gates a double-vs-real divergence …`.
- `docs/plan_topics/H6a-live-corpus-activation.md`: make both links to `./real-host-smoke-gate.md` that denote the standing gate use the link text `real-host smoke gate` — in particular change the **Ships when** link text `[pre-merge real-host smoke gate](./real-host-smoke-gate.md)` to `[real-host smoke gate](./real-host-smoke-gate.md)`. Retain `manual real-host smoke run` only where the text refers to the executed-and-recorded run instance (the evidence record).
- `docs/plan_topics/V18d-version-bump-acceptance.md`: `real-host-smoke` as a compound modifier (`real-host-smoke behavioural-divergence finding`) is consistent with the canonical head noun and may be left as-is, or expanded to `real-host smoke gate` for symmetry — implementer's call.

Cross-shard addendum (from the source artefact): the canonical name is already confirmed by the `real-host-smoke-gate.md` H1 title `Manual real-host smoke gate`.

## Consequence

**Severity:** advisory

A reader who greps for one name (e.g. `real-host smoke gate`) misses the references filed under the other three, and an auditor following `H6a`'s two divergent link texts cannot tell whether "run", "gate", and "governance protocol" name one artefact or three. The gate's behaviour is fully specified, so no implementer produces a divergent leaf; the cost is navigational and review friction around the release-gate evidence record.

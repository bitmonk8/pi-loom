## Problem

`H1a` describes the `typescript` devDependency two incompatible ways. The **Adds** field says the `tsc` provider is "pinned to the TypeScript floor of `>=5.4.0`," which reads as a single fixed declared range, while the **Tests** field asserts the `typescript` key "is present and its declared range satisfies the TypeScript floor of `>=5.4.0`" — accepting any floor-satisfying range. An implementer cannot tell whether to declare the exact floor string or any satisfying range (e.g. `^5.6.0`), so the two readings produce different `package.json` pins across checkouts. The divergence has teeth because `host-prerequisites.md` §`tsc-toolchain-floor` confirmed the grep-able `loom/typecheck/session-shutdown-reason-snapshot` brand-string surfacing only at the `>=5.4.0` floor.

## Issue introduction

**Verdict:** introduced-by-commit
**Introducing commits:** `1abc125` (2026-06-16, "pi-loom plan: resolve \"H1a omits the typescript devDependency and its >=5.4.0 floor\"")
**History:** The plan corpus is git-tracked. `git log -S` confirms both phrasings — the Adds "pinned to the TypeScript floor" and the Tests "satisfies the TypeScript floor" — first entered `docs/plan_topics/H1a-scaffold-and-toolchain.md` in the same commit `1abc125`, the leaf-edit that added the `typescript` devDependency and its floor. The mismatched predicates were present at the inception of the typescript pin; no later edit introduced the divergence.

## Solution approach

Rewrite the `typescript` clause in `H1a`'s **Adds** field so it describes a range that satisfies the floor `>=5.4.0` rather than a fixed-equality pin, aligning it with the **Tests** predicate (which already reads "satisfies the TypeScript floor") and with the floor semantics `host-prerequisites.md` §`tsc-toolchain-floor` owns.

## Solution constraints

- The floor literal `>=5.4.0` stays sourced from `host-prerequisites.md` §`tsc-toolchain-floor`; do not restate it as a fresh plan-side constant.
- Out of scope: the spec is read-only for this fix; do not edit `host-prerequisites.md`.

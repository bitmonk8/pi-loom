## Finding

`V4f`'s **Ships when** field carries two sentences. The first states the observable shipping criterion: `npm test` proves the enumerated `ERR-13` cases hold — a completed callee stays final after a downstream terminal event across the sampled authoring sites, with no side effect unwound and no compensating turn injected. The second sentence restates the architectural no-rollback rationale ("it rests on the runtime containing no compensating/rollback path, and the tests witness it on the enumerated cases rather than exhaustively").

That rationale is already stated twice elsewhere in the same leaf: in **Adds** ("The no-rollback guarantee is architectural: the runtime contains no compensating path, so a completed callee's side effect survives a downstream terminal event by construction rather than by an undo being suppressed") and at the tail of the **ERR-13 Tests** bullet ("The guarantee rests on the runtime having no compensating/rollback path, so the tests witness it on these enumerated authoring sites rather than proving the absolute absence of a compensating path exhaustively").

The **Ships when** field's purpose is the observable shipping criterion; the third restatement adds no observable condition and is pure rationale duplication.

The redundancy was authored with the leaf; no later commit introduced or amplified it.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V4f-no-rollback.md`, strike the final sentence of the **Ships when** field — "The no-rollback guarantee is architectural: it rests on the runtime containing no compensating/rollback path, and the tests witness it on the enumerated cases rather than exhaustively." — leaving the field as its first sentence only ("`npm test` proves the enumerated `ERR-13` cases hold — a completed callee stays final after a downstream terminal event across the sampled authoring sites, with no side effect unwound and no compensating turn injected."). The architectural rationale remains stated in **Adds** and the sampled-witness caveat remains stated at the end of the **ERR-13 Tests** bullet, so no content is lost.

## Consequence

**Severity:** cosmetic

The duplicated rationale carries no implementer-facing divergence: the observable shipping criterion is fully stated by the first Ships-when sentence, and the architectural rationale is already authoritatively stated in Adds. The redundant sentence only adds maintenance surface (three copies of the same rationale drift independently on edit).

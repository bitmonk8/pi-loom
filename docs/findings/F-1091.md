## Problem

`V9l` and `V9l-T` specify the `new`/`resume`/`fork` session-only degraded-state branch unconditionally in their **Adds** and **Tests** — the `markRuntimeDegraded` tag write, `"degraded-needs-reload"` transition, `loom/host/session-shutdown-runtime-degraded` emission, recovery-path prohibition, and predicate split — describing only the surviving-registry (branch-reachable) implementation. `host-prerequisites.md` clause (a) records an open contradiction at the loom 1.0 Pi-SDK pin with two dispositive verdicts: `provably-differs` (branch reachable → implement) and `governed-by-rebind` (branch unreachable → rework spec-side). The leaf is correctly gated non-pickable (its `V18c` **Deps** edge and the `plan.md` §Blocked obligations entry), and its **Ships when** carries the `branch reachable` / `branch unreachable` split, but the **Adds**/**Tests** bodies never state that the entire described branch is written against the `provably-differs` verdict. An implementer picking the leaf up after the block clears could build the described branch even when clause (a) resolved `governed-by-rebind`, shipping a path the spec has determined is unreachable.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** 55ed974 — pi-loom plan: resolve "degraded-state obligations green over open spec contradiction" (2026-06-12, plan author)
**History:** The spec recorded clause (a) as an open contradiction earlier, in 7447d1e (2026-06-10, "Session-only degraded-state presupposition (a) contradicts Pi's documented lifecycle"). The `V9l` / `V9l-T` leaves were then created in 55ed974 (2026-06-12) already against that known-open contradiction; from their first commit the Adds/Tests describe the surviving-registry branch unconditionally while only the Ships-when carries the `branch reachable` / `branch unreachable` split, so the disposition the body is written against was never stated. The later 9c42f2d (2026-06-16) relocated the inline fallback block to plan.md §Blocked obligations but did not add the missing disposition label to the leaf body.

## Solution approach

In the **Adds** field of `V9l` and `V9l-T`, state that the described degraded-state branch is the deliverable only under the `provably-differs` verdict of `host-prerequisites.md` clause (a) — as recorded by the `version-bump-step2.md` item (a) editorial-review checklist — and that a `governed-by-rebind` verdict instead invokes the branch-unreachable fallback in `plan.md` §Blocked obligations rather than this leaf body. Use the `provably-differs` / `governed-by-rebind` verdict vocabulary the spec's clause (a) and `version-bump-step2.md` item (a) already define, keying the leaf's reachability to the recorded determination.

## Solution constraints

- Out of scope: the spec-side resolution of the clause-(a) contradiction, owned by `V18c`'s editorial-review checklist item (a).
- Leave the leaf's non-pickable gating unchanged — the `V18c` **Deps** edge and the `plan.md` §Blocked obligations entry stay as-is.

## Finding

The V15 leaf family and the spec both write infra-side invoke failures in full struct-literal form — `InvokeInfraError { cause: "validation", ... }`, `{ cause: "load_failure" }`, `{ cause: "panic" }`, `{ cause: "return_validation" }` — naming the sub-discriminator field `cause` explicitly. This matches the canonical schema declaration in `spec_topics/errors-and-results/queryerror-variants.md` (the `cause:` field) and the glossary's `cause` entry, which fixes `cause` as the uniform sub-discriminator field name across every `QueryError` variant.

V14a and V15a (and their paired `-T` leaves) instead use a field-eliding shorthand: `InvokeInfraError{validation}` (V14a, V14a-T) and `InvokeInfraError{load_failure}` (V15a, V15a-T). The braces carry only the sub-arm value with no field name, so a reader cannot tell from the leaf alone whether `{validation}` refers to `cause`, `kind`, or some other field — particularly relevant because `InvokeInfraError` carries both a `kind` discriminant (`"invoke_infra"`) and a `cause` sub-discriminant, and the shorthand value is a `cause` value, not a `kind` value.

This is a notation inconsistency only; the underlying contract is identical and unambiguous once cross-referenced to the spec or a sibling leaf. The shorthand appears at six sites across the four leaves.

## Solution Space

**Shape:** single

### Recommendation

Convert the six shorthand occurrences to the canonical full struct-literal form, naming the `cause` field, so V14a/V15a match the spec's `queryerror-variants.md` declaration and the V15b/V15f/V15j sibling leaves:

- `docs/plan_topics/V14a-tool-calls.md` — Tests bullet (`.loom`-callable failure) and Ships when: replace `InvokeInfraError{validation}` with `InvokeInfraError { cause: "validation" }` (both occurrences).
- `docs/plan_topics/V14a-T-tool-calls.md` — Tests bullet (`.loom`-callable failure): replace `InvokeInfraError{validation}` with `InvokeInfraError { cause: "validation" }`.
- `docs/plan_topics/V15a-invocation-core.md` — `INV-1` Tests bullet and Ships when: replace `InvokeInfraError{load_failure}` with `InvokeInfraError { cause: "load_failure" }` (both occurrences).
- `docs/plan_topics/V15a-T-invocation-core.md` — `INV-1` Tests bullet: replace `InvokeInfraError{load_failure}` with `InvokeInfraError { cause: "load_failure" }`.

The paired impl/`-T` leaves duplicate the bullet text, so apply the identical edit to both members of each pair to keep them in sync. No spec edit is required — the spec already uses the target form.

## Consequence

**Severity:** cosmetic

An implementer reading V14a or V15a in isolation must cross-reference the spec or a sibling V15 leaf to learn that the elided field is `cause`; the contract is recoverable and every sibling leaf and the spec spell it out consistently, so no implementation divergence is likely. Pure notation drift.

## Problem

The **Runtime event channel** section of `runtime-event-channel.md` pins an "exactly once per occurrence" emission guarantee for the always-log set across every disposition path (author-matched, discarded via `let _ =`, propagated via `?` to a non-top-level handler, cascaded unhandled to the slash-dispatch boundary), plus the cascade-twin behaviour under **Deduplication and lifetime rules** (one origin emission plus a verbatim boundary re-emission with byte-identical `RuntimeEvent`). All of this is stated only as running normative prose. No disposition path carries an acceptance-criteria block or a structured pass/fail fixture, so a conformance tester must hand-derive the expected emission count, `display` value, and payload-equality assertion for the central observability contract, with no canonical fixture against which two implementations can be cross-checked. It is the only major obligation in the section with no testable encoding.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `4ccc072` (2026-05-07, "Masked ceilings…") and `7598f50` (2026-05-07, "CIO-6 hard-ceiling co-fire…") coined "cascade twin" in the hard-ceiling/PIC content; `78a2f94` (2026-06-04, "Misplaced sections within the pi-integration-contract pages") introduced both "top-level cascade" and the "`display: false` cascades" phrasing; `d75f2a8` (2026-06-06, "Runtime-event channel occurrence/origin, observable PIC-1 (e), mask-domain placement") consolidated all three senses onto `runtime-event-channel.md` and added the hyphenated "cascade-twin re-emission"; `0033898` (2026-06-09, "subagent-mode top-level Err at slash boundary unsurfaced") elaborated the subagent-private cascade prose.
**History:** No single commit introduced the overload. Three "cascade"-flavoured terms were coined in separately-authored fixes (2026-05-07 → 2026-06-09), each addressing its own concern without reconciling the word against the others; the collision became durable once `d75f2a8` brought all three senses onto one page. The fix therefore addresses the term-system on the page, not any single commit.

## Solution approach

Add an acceptance-criteria block to the **Runtime event channel** section, adjacent to the exactly-once guarantee and the cascade-twin rule, carrying one structured fixture per disposition path: author-matched `Err`, `let _ =` discard, non-top-level `?`-propagation, and unhandled cascade to the slash-dispatch boundary in prompt mode. Each fixture pins the expected emission count, the `display` value, and — for the cascade twin — the byte-identical `RuntimeEvent` payload-equality assertion (including `occurred_at` and `masked`). Keep the criteria consistent with the existing `(kind, query_site, message, occurred_at)` dedup tuple under **Deduplication and lifetime rules** so the twin collapses under it.

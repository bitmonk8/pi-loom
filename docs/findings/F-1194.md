## Problem

The §7 "Identifier-shaped" sub-rule in `placeholder-rendering-b.md` renders `<callee>`, `<enum>`, `<schema>`, `<method>`, `<model>`, `<provider>`, `<slash-name>`, `<slug>`, `<invocation-id>`, `<uuid>`, `<name1>`, `<name2>`, and `<X>` "in identifier form per [Lexical — Identifiers](../lexical.md)" — a citation to the strict production `[A-Za-z_][A-Za-z0-9_]*` (no hyphens, no leading digit). Several of these placeholders carry values that cannot satisfy that production: `<slash-name>` / `<slug>` admit hyphens and leading digits (`code-review`, `42-things`) per the slash-name regex in `discovery/discovery-sources.md`; `<model>` / `<provider>` / `<method>` are host-supplied ids (`claude-3-5-sonnet`); and `<uuid>` / `<invocation-id>` are hyphenated 8-4-4-4-12 hex, as the same bullet states two clauses later. Under DIAG-4's byte-identical contract this is material: one implementer reads "identifier form" as "emit the source bytes verbatim", another as "reject or escape values that don't match the production" — the two readings produce different bytes for every panic message interpolating these tokens.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** f930818 — pi-loom spec: resolve "placeholder rendering exemption open-ended" (2026-05-07, Thomas Andersen)
**History:** The "Identifier-shaped" sub-rule was introduced verbatim in its current form by f930818, which authored the eight-category placeholder-rendering surface — that commit placed `<slash-name>`, `<slug>`, `<invocation-id>`, `<uuid>`, `<model>`, `<provider>`, `<method>` together under "rendered unquoted in identifier form per [Lexical — Identifiers]" without reconciling them against the broader slash-name regex (`[a-z0-9][a-z0-9_-]*`) already pinned in discovery or the hyphenated UUID form named later in the same bullet. Subsequent edits (`bf7c69e` on 2026-06-05, `f5e89f4` on 2026-06-04) refined the `<list>` row's verbatim wording and re-split the page but never revisited the sub-rule itself; the conflict has been latent in the spec from the moment §7 was first authored.

## Solution approach

Rewrite the §7 "Identifier-shaped" sub-rule so its rendering rule reflects the actual byte shapes the placeholders carry rather than citing [Lexical — Identifiers](../lexical.md) as the umbrella form for all of them. Render each value verbatim from its producing site — the canonical-key form for `<slash-name>` / `<slug>` (per [Discovery — Filename validity](../discovery/discovery-sources.md)), the host-supplied id for `<model>` / `<provider>` / `<method>`, and the lowercase 8-4-4-4-12 hex form for `<uuid>` / `<invocation-id>` — retaining the Lexical citation only for the placeholders that genuinely conform (`<enum>`, `<schema>`, `<name1>`, `<name2>`, `<X>`). Add a §7 test vector exercising a hyphenated slash name end-to-end so the verbatim reading is fixed at conformance-test time.

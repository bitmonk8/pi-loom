## Problem

The per-loom `ToolDefinition.label` derivation in `extension-bootstrap-and-per-loom.md` (§Per-loom registration) says the label is "the loom file's basename", but its two worked examples strip the `.loom` extension (`summarise.loom` → `"Summarise"`). In Node/POSIX convention a basename includes the extension, so no clause removes it: a literal reading of the prose yields `"Summarise.loom"` while the examples yield `"Summarise"`. The `label` is observer-facing — the human-readable name a model and operator see for every tool-exposed loom — so two implementers diverge on every such label with no load-time or runtime error to catch it.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** `cb43deb` (2026-05-04, "pi-loom spec: resolve \"ToolDefinition is mis-shaped: missing label, and parameters must be a TypeBox TSchema\"")
**History:** Commit `cb43deb` first added the `label` field-derivation bullet to the per-loom registration paragraph (then in `spec_topics/pi-integration-contract.md`, later relocated to `extension-bootstrap-and-per-loom.md` by `fecb504`'s split), introducing both halves of the contradiction in the same edit: the prose word "basename" and the worked examples `summarise.loom → "Summarise"` / `code-review.loom → "Code-Review"`, the latter case-corrected to `"Code-review"` by `6d8b472` (2026-06-05, "resolve 'label derivation rule and example capitalisation disagreement'") without touching the basename / extension-strip disagreement. No earlier revision carries the rule; subsequent edits to the surrounding paragraph (`841a085`, `bf85092`, `79ea4e8`, `9568906`, `6a42c1a`, `4b4382f`) did not touch this bullet's wording. The defect has been present since the rule first entered the corpus.

## Solution approach

Clarify the `label` derivation bullet in `extension-bootstrap-and-per-loom.md`'s §Per-loom registration so the `.loom`-extension-stripping step is explicit, resolving the disagreement in favour of the worked examples. Match the precise phrasing already used for the frontmatter default-name rule in `frontmatter-fields-a.md` ("the file's basename without the `.loom` extension").

## Finding

`patch-skew-degradation.md` is presented in the PIC table of contents as "Patch skew degradation" — its sole topic, by both filename and TOC label, is the patch-skew window between a user's installed `@earendil-works/pi-coding-agent` patch and loom's snapshot. In practice the file carries far more than that: after the patch-skew contract paragraph it contains (a) the complete normative spec of the five-sub-step `session_shutdown` handler (sub-step 1 drain + tag init, sub-step 2 in-flight stamp + `loomAbort.abort()`, sub-step 3 bounded `Promise.allSettled` over `disposeBarrier`, sub-step 4 watcher close, sub-step 5 listener detach), and (b) the **Per-step isolation** paragraph with its closed eight-entry `details.call` label set and the interstitial-seam `markRuntimeDegraded` swallow rule. Both run unconditionally for every `event.reason` — they are not patch-skew-specific behaviour.

An implementer building the `"quit"` / `"reload"` teardown path, or auditing the Per-step isolation contract, has no incentive to open a file labelled "patch skew degradation": the topic name does not advertise the surface. The mismatch is invisible from the PIC index page and from cross-references that cite `patch-skew-degradation.md §session_shutdown sub-step N`; downstream artefacts such as the plan's coverage-matrix row `cka-31`, plan leaves `V9g`/`V9g-T`/`V17a`/`V9a`, the `code-registry-host.md` row anchor `#runtime-degraded-emission-condition`, and the `cancellation.md` cross-link from `session-only-degraded-state.md` all key on that misplaced location. The defect is purely structural — every observable behaviour is fully specified — but every reader-driven discovery path leads away from where the rules actually live, and every co-edit of the teardown surface lands in a file whose name disclaims it.

**History:** `5e8b499` introduced the five-sub-step `session_shutdown` handler as step 4 of the monolithic `spec_topics/pi-integration-contract.md` "Extension entry point" sequence, correctly nested under registration steps. `75a9626` then inserted the **Patch-skew degradation contract** paragraph immediately above the handler prose in the same file (the introductory sentence of the patch-skew paragraph ends with "The handler runs in fixed order regardless of `event.reason`:" followed by the five sub-steps). At that point the two paragraphs were physically adjacent inside one monolithic file with no internal heading boundary between them. `f5e89f4` (the 100KB-cap split commit) created `patch-skew-degradation.md` as a new file by mechanically partitioning that monolithic source; because no heading separated the patch-skew prose from the teardown-sequence prose that followed it, the splitter swept both — plus the trailing **Per-step isolation** paragraph and the interstitial-seam swallow rule — into the single new child file. The misplacement is therefore not authored as such; it is an emergent artefact of (i) two unrelated edits landing without a section break between them and (ii) a later mechanical splitter that respected only existing heading boundaries.

## Solution Space

**Shape:** multiple

### Option A — Extract to a dedicated `session-shutdown-handler.md`

**Approach.** Create a new child page under `docs/spec_topics/pi-integration-contract/` whose declared scope is exactly the five-sub-step handler, the Per-step isolation rule, and the interstitial-seam swallow rule. Move the prose verbatim from `patch-skew-degradation.md`; preserve every `<a id="...">` anchor (`substep-1-drain-flag`, `substep-1-init-drain-state-tag`, `substep-1-act-order`, `substep-1-shutting-down-arm`, `substep-1-unset-tag-fallback`, `substep-1-degraded-needs-reload-arm`, `interstitial-seam-tag-write-swallow`) at the same fragment id on the new page. `patch-skew-degradation.md` retains the patch-skew contract paragraph and the fixture-obligation-categories paragraph only, with a forward-reference at the point where the teardown sequence is invoked ("the full five-sub-step teardown" → link to new page). Add the new page to the PIC index TOC immediately after `unknown-reason-rule.md` (preserving the natural read order).

**Spec edits.** New file created; `patch-skew-degradation.md` shortened; PIC index updated; cross-references rewritten in `session-only-degraded-state.md`, `drain-state-contract.md`, `unknown-reason-rule.md`, `diagnostic-emission-isolation.md`, `code-registry-host.md`, `cancellation.md`, plan leaves `V9a`/`V9g`/`V9g-T`/`V17a`, and `coverage-matrix.md` row `cka-31`.

**Pros.** The new page's name maps one-to-one onto its contents, restoring the structural-lens invariant. `patch-skew-degradation.md` ends up scoped to its TOC label. Anchor renaming is unnecessary (preserve fragment ids verbatim); only the file portion of every link changes.

**Cons.** Adds one more child page to the PIC subdirectory (currently 30). Every inbound cross-reference must be rewritten in lockstep with the move.

**Risks.** A missed inbound reference fails the link-check tooling under `docs/_tools/check_links.py` (already gated by repository CI). Anchor preservation is mechanical and low-risk.

### Option B — Relocate into `session-shutdown-semantics.md`

**Approach.** Move the teardown sequence, Per-step isolation, and interstitial-seam swallow into `session-shutdown-semantics.md`, which already houses session-swap behaviour and the **Factory-ordering pin** for the same handler. Reorganise `session-shutdown-semantics.md` into a two-part structure: "Handler sequence" (the imported prose) and "Session-swap behaviour for in-flight invocations" (its current contents). `patch-skew-degradation.md` shrinks to the patch-skew contract proper.

**Spec edits.** `session-shutdown-semantics.md` grows by ~17 KB (still well under the 30 KB recommended cap). All anchors preserved. Cross-references updated as in Option A but pointing into the expanded file.

**Pros.** No new file; all `session_shutdown` handler material lives under one roof; the file name already matches the topic. Co-locates the **Factory-ordering pin** with the handler it constrains.

**Cons.** `session-shutdown-semantics.md` becomes a heavier page covering two distinct concerns (handler internals + session-swap obligations). The two-part structure relies on internal headings rather than file separation for navigation. File-size budget is still respected but the file moves from ~11 KB to ~28 KB, near the recommended cap.

**Risks.** Any future edit on either half competes for the same file's surface and may push it over the 30 KB recommended cap, re-triggering a split (and a re-occurrence of this same class of defect).

### Recommendation

Option A. The teardown handler is itself a normative surface large enough (~15 KB of prose plus eight closed `details.call` labels and the interstitial-seam swallow rule) to warrant its own page, and it has more cross-references inbound than `patch-skew-degradation.md` does — it is the topic, not a sub-section of another topic. Performing the work in order: first add the new file with prose moved and all anchors preserved; second, in a separate commit, rewrite every inbound cross-reference and the plan-side citations (`V9a`, `V9g`, `V9g-T`, `V17a`, `coverage-matrix.md` `cka-31`); third, run `docs/_tools/check_links.py` to confirm no dangling refs. Implementer edge cases: the `// allow: cka-31 — patch-skew-degradation.md` exemption-comment example baked into `V9g-session-shutdown.md` is a runtime-enforced lint-allow-list token that the closing gate resolves against `coverage-matrix.md`; updating the matrix row's filename reference is sufficient — no source-code edits are required for that lint to keep resolving — but the leaf prose's worked example must be updated in the same commit.

## Consequence

**Severity:** advisory

A reader scanning the PIC index for the `session_shutdown` teardown contract walks past `patch-skew-degradation.md` and lands instead on `session-shutdown-semantics.md` (which covers only session-swap obligations) or `session-only-degraded-state.md` (which covers only the `{"new","resume","fork"}` branch). The structural pin (the Per-step isolation closed `details.call` set; the interstitial-seam swallow rule) is then discoverable only by chasing inbound cross-references from sibling files. Implementation work proceeds — the plan leaves' citations resolve at the file level — but future spec edits that touch the teardown surface land in a file whose declared scope disclaims them, and the spec corpus's structural lens fails on every readout.

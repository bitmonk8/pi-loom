## Problem

PIC-22 at `id="pic-22"` in `docs/spec_topics/pi-integration-contract/subagent.md` places the parenthetical `(for any fixed N ≥ 3)` inside the MUST clause, so it scopes the normative obligation as well as the conformance-witness sentence. This admits two readings: the MUST applies only at N ≥ 3 (a runtime may serialise two parallel subagent spawns and remain conformant), or the MUST applies to every parallel set and N ≥ 3 is merely the chosen test cardinality. The first reading lets a conforming runtime ship the admission-cap-at-N=2 behaviour the neighbouring `id="no-invocation-cap"` "no admission cap, priority queue, or scheduler" claim forbids, and the N = 3 conformance fixture cannot witness the divergence.

## Issue introduction

**Verdict:** single-commit-introduced
**Introducing commits:** `7afe9bb` ("pi-loom spec: resolve \"no-invocation-cap MUST NOT is unobservable\"", 2026-05-31)
**History:** Pre-`7afe9bb`, the `#no-invocation-cap` paragraph in `docs/spec_topics/pi-integration-contract.md` carried only the structural MUST NOT clause ("MUST NOT introduce an admission cap, a priority queue, or any scheduler …"); there was no parallel-tool-call witness. Commit `7afe9bb` rewrote the clause into an observable obligation and introduced the wording "when the model emits N (for any fixed N ≥ 3) subagent-mode `.loom` callables … the runtime MUST initiate `createAgentSession` for all N invocations before any one of the N returns", placing `(for any fixed N ≥ 3)` inside the MUST. The subsequent 100KB-cap shard (`f5e89f4`, 2026-06-04) moved the paragraph from `pi-integration-contract.md` to `pi-integration-contract/subagent.md` verbatim, and commit `7a3b907` (2026-06-10, "subagent parallel-initiation MUST has no closing leaf") split the combined paragraph into a separate PIC-22 block and added the `<a id="pic-22">` anchor — the defective phrasing was preserved through both moves and never edited.

## Solution approach

Restructure the PIC-22 sentence at `id="pic-22"` so the MUST is unqualified over parallel subagent-mode `.loom` invocations in a single assistant turn, and the `N ≥ 3` (or `N = 3`) cardinality appears only on the conformance-witness sentence that follows. Preserve the cross-link to `../tool-calls.md#concurrency` and the remainder of the PIC-22 paragraph from the "rule constrains the runtime only" sentence onward.

## Solution constraints

- If the witness cardinality is changed away from N = 3, the `V9i`/`V9i-T` plan-leaf test bullets and `coverage-matrix.md` that pin that value must change in lock-step.
- Out of scope: the structural "no admission cap, priority queue, or scheduler" claim owned by the neighbouring `id="no-invocation-cap"` sentence — do not restate it inside PIC-22.

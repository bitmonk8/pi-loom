## Finding

PIC-22 in `docs/spec_topics/pi-integration-contract/subagent.md` reads:

> when the model emits N (for any fixed N ≥ 3) subagent-mode `.loom` callables as parallel tool calls in a single assistant turn, the runtime MUST initiate `createAgentSession` for all N invocations before any one of the N returns.

The parenthetical `(for any fixed N ≥ 3)` is placed inside the MUST clause itself, so it scopes the normative obligation as well as the conformance witness sentence that follows. Two readings are syntactically available:

1. **Obligation-scoping reading.** The MUST applies only when N ≥ 3; for N = 2 the runtime is free to serialise. Under this reading, a runtime that ran two parallel subagent spawns through a serialising queue but allowed three or more to run concurrently would be conformant.
2. **Witness-scoping reading.** The MUST applies for every N ≥ 2 (any parallel set); the `N ≥ 3` is the chosen conformance-test cardinality and is not a limit on the rule itself. Under this reading, the same serialise-two-spawns runtime is non-conformant.

The disposition PIC-22 is the observable witness for — the surrounding paragraph's "no admission cap, no priority queue, no scheduler" structural claim — supports only the witness-scoping reading: an admission cap that activates at N = 2 is plainly a cap. The embedded `(for any fixed N ≥ 3)` reads the other way and lets a conforming runtime ship the very behaviour the surrounding paragraph forbids. The plan's acceptance text reinforces the slip: `V9i`/`V9i-T` already pin the **test** at exactly `N = 3`, so the witness cardinality is committed elsewhere and does not need to be embedded in the MUST.

The fix is structural: split the sentence so the MUST is unqualified over parallel invocations and the `N ≥ 3` (or `N = 3`) appears only on the conformance-witness sentence that follows.

## Solution Space

**Shape:** single

### Recommendation

Split the current sentence so the MUST is unqualified and the witness cardinality lives on its own sentence. Replace the PIC-22 paragraph's opening with substantially this shape (preserve the existing cross-link to `../tool-calls.md#concurrency` and the rest of the paragraph — the "rule constrains the runtime only" sentence onwards — unchanged):

> This disposition is observable at the parallel-tool-call surface (see [Tool Calls — Concurrency](../tool-calls.md#concurrency)): when the model emits two or more subagent-mode `.loom` callables as parallel tool calls in a single assistant turn, the runtime MUST initiate `createAgentSession` for every such invocation before any one of them returns. The conformance fixture uses N = 3: against a fake `AgentSession` whose `sendUserMessage` blocks until explicitly released, a conformant runtime has created all three sessions and entered each session's `sendUserMessage` before any blocked call is released, whereas a runtime that admitted invocations behind a cap, queue, or scheduler would leave at least one session uncreated.

Edge cases the implementer must watch:

- The obligation is keyed on "parallel tool calls in a single assistant turn", not on a numerical threshold; do not reintroduce an `N ≥ k` qualifier on the MUST in any subsequent edit.
- The conformance fixture cardinality (N = 3) is independently committed in the `V9i`/`V9i-T` leaves' test bullets and in `coverage-matrix.md`; either keep that value or, if widened, widen the leaves' acceptance text in lock-step. Lowering it to N = 2 weakens the witness (a single-slot queue passes a two-invocation fixture trivially), so N ≥ 3 remains the right test cardinality.
- The neighbouring `<a id="no-invocation-cap"></a>` sentence already pins the structural "no admission cap, priority queue, or scheduler" claim; do not duplicate that claim into the PIC-22 MUST when rewording — PIC-22's role is to make the existing claim observable, not to restate it.

## Consequence

**Severity:** correctness

Two reasonable implementers diverge on the N = 2 case: one ships a serialise-at-2-but-not-at-3 runtime and reads PIC-22 as conformant; the other reads the surrounding "no admission cap or scheduler" framing as definitive and rejects the same runtime. The conformance fixture at N = 3 cannot witness the disagreement, so the divergence ships undetected. The defect frustrates the very property PIC-22 was added to make observable.

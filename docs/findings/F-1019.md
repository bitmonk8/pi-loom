## Problem

The five-row *Per-boundary surfaces* table under `## Depth Enforcement` → **Error shape** in `docs/spec_topics/schema-subset.md` restates carrier shape, destination, and routing for surfaces whose normative owners live elsewhere, without declaring whether the table is itself normative or merely an informative restatement. Its closing sentence cross-links the GOV-30 informative orientation aggregator `overview-and-orientation.md#hard-runtime-ceilings` rather than the normative per-boundary table at `hard-ceilings/ceilings-3-and-4.md#ceiling-4-table`, and the word "mirrors" leaves both readings open. A drift surface already exists: the `<ajv-summary>` placeholder clause on schema-subset row #4 (`params`) is not present on the corresponding row of the hard-ceilings table. Because the per-row wire contract (`schema_keyword`, message, carrier `cause`) is part of the pinned on-wire contract, a silent inter-table divergence on these rows is a correctness defect.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** `1750848` (2026-05-07) — *pi-loom spec: resolve "Ceiling #4 routing class collapses four boundaries whose destinations differ"*
**History:** Commit `1750848` introduced both the five-row *Per-boundary surfaces* table under **Error shape** in `schema-subset.md` and the closing sentence "*this section's table mirrors it with the depth-walk-specific carrier details*", whose cross-link at that time was `../spec.md#hard-runtime-ceilings`. The same commit added the parallel five-row table under the four-item hard-ceilings list on `spec.md`. The "mirrors" wording — and the lack of any normative-vs-informative tag on the new schema-subset table — is therefore present-since-inception of the table itself; no later commit weakened the framing. Commit `81bf52a` (2026-06-11, the corpus reorganisation into `docs/spec_topics/`) rewrote the cross-link target from `../spec.md#hard-runtime-ceilings` to `./overview-and-orientation.md#hard-runtime-ceilings`, but only as a path rewrite — both the source and target sites of the "mirrors" cross-reference moved together and the authority ambiguity was unaffected. Subsequent ceiling-#4 work (notably the per-boundary table's relocation onto `hard-ceilings/ceilings-3-and-4.md` and the addition of the *Five-site list co-edit obligation* paragraph on `ceiling-invariants-and-audit.md`) added a third authority site for the same content without retiring the schema-subset duplicate or tagging it.

## Solution approach

Establish `hard-ceilings/ceilings-3-and-4.md#ceiling-4-table` as the sole normative owner of ceiling #4's per-boundary destination/surface contract, and label the schema-subset.md *Error shape* table as an informative depth-walk-specific restatement. Rewrite the closing "mirrors it" sentence to state that disposition and retarget its cross-link from `overview-and-orientation.md#hard-runtime-ceilings` to `hard-ceilings/ceilings-3-and-4.md#ceiling-4-table`. Reconcile the `<ajv-summary>` placeholder clause by moving it to the `params` row of `#ceiling-4-table`, or deleting it from schema-subset.md if that row already carries it.

## Problem

Under `bind_context: session`, the runtime sends a bounded most-recent slice of the caller's session transcript to whatever provider backs the resolved binder model. That slice is not a sanitised projection: BNDR-7c renders `loom-system-note` custom messages with `display: false` into the transcript, and BNDR-7e renders assistant `ToolCall.arguments` verbatim — so operator-only diagnostics and full tool-call argument JSON leave the caller's data path when the binder runs. The binder model resolves independently of the caller's model (`bind_model:` → `looms.binderModel`) and may use a different provider, which the non-normative cost guidance actively encourages. No spec text on this page or in the `## Trust boundary` non-functional bullet — which speaks only to host-process privilege, not data destination — acknowledges this cross-provider data-egress flow.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `83802db` (2026-05-04, "spec: collapse to single typed argument style; add LLM-driven slash-command binder")
**History:** The binder feature itself, the `bind_context: session` axis, and the cost guidance steering authors toward a cheaper tier-2 model all landed in the same commit (`git log --reverse -S "bind_context" -- '*.md'` and `git log --reverse -S "Haiku" -- '*.md'` both bottom out at `83802db`). That commit message lists "`bind_context` (none | session, default `none`)" and "cost note" as siblings under a single bullet, with no data-boundary statement accompanying either. Subsequent edits (the `binder-model-and-context.md` split at `f5e89f4`, the BNDR-7 transcript reference renderings at `1a41db2`, the automatic-context-escalation seam at `0594eea`) further fleshed out the transcript surface — making the egress more concrete (custom-message `display: false` rendering, verbatim `[tool-call …]` lines) — without ever adding the missing disposition. The defect is not the result of a later interaction; the omission has been continuously present since the binder section was first authored.

## Solution approach

Add a normative author/operator-responsibility disposition to the `## Binder context` section stating that under `bind_context: session` the transcript slice sent to the binder model includes the unsanitised content the [Compact-transcript format](../../docs/spec_topics/binder/binder-model-and-context.md#compact-transcript-format-normative) renders — `display: false` system notes (BNDR-7c) and verbatim `ToolCall.arguments` JSON (BNDR-7e) — and that the resolved binder-model provider may differ from the caller's, making the destination an author/operator responsibility. Add a forward-link from the non-normative cost note under `## Binder model` to that disposition, acknowledging that the cheaper models it recommends are typically reached through a different provider. Bound the disposition to the non-bypass path (bypass-eligible looms per [Binder bypass](../../docs/spec_topics/binder/binder-bypass-and-envelope.md#binder-bypass) route no caller content) and note it is per-invocation per the [automatic-context-escalation seam](../../docs/spec_topics/binder/binder-model-and-context.md#loom-1-0-seam-automatic-context-escalation).

## Solution constraints

- Out of scope: do not add a load-time diagnostic or a provider-equality gate — a cross-provider binder model is a legitimate configuration the cost guidance steers authors toward; the gap is disclosure, not enforcement.
- Out of scope: do not broaden `## Trust boundary` in `overview-and-orientation.md` (it owns the host-process-privilege posture); a forward cross-reference from the new disposition to it is the only permitted touch of that section.

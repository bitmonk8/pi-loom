## Problem

The *Substitution limitation (acknowledged soft surface)* sub-bullet under *Stale sub-kinds* in `docs/spec_topics/pi-integration-contract/audit-failures.md` is a single long paragraph whose only normative content is its closing clause: the audit is **not owed an additional discriminator** for this surface. Everything before that clause is design rationale — an illustrative `pi.alpha → pi.beta` rewrite example plus an explanation that per-line scope cannot machine-validate the substituted symbol — and the middle "mitigation" sentence merely restates the contributor obligation already defined at step 2(b) branch (4). Inside this dense normative taxonomy block the rationale adds reading load and risks a reader treating the descriptive prose as a mechanical obligation.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:**
- `e5cd514` (2026-05-09) — *pi-loom spec: resolve "T35 SDK capability inventory closed-set negative-direction audit"*
**History:** The *Stale-citation limitation (acknowledged soft surface)* sub-bullet, the trailing "widening the audit to validate citation resolution would require attaching a registry of valid REQ-IDs and PIC anchors to the audit input, which is a PIC edit deferred past V1" clause, and the *Substitution limitation* / *Generated-output-in-src limitation* siblings in the same shape were all introduced in commit `e5cd514` when the inventory-closure audit narrative first landed on `docs/spec_topics/pi-integration-contract.md` (`git log -G "registry of valid REQ"` returns `e5cd514` as the first add; the subsequent commits `9e53613`, `a6d82db`, `f5e89f4` are corpus-direction sweeps, prose `V1`→`loom 1.0` renames, and the size-cap shard split that moved the bullet from `pi-integration-contract.md` into `pi-integration-contract/audit-failures.md` — none of them touched the implementation-sketch clause). The defect therefore entered with the section itself; no later commit reshaped it. The split into a separate `audit-failures.md` file did not introduce the issue, only relocated it.

## Solution approach

Rewrite the *Substitution limitation (acknowledged soft surface)* sub-bullet to its normative core: a single statement that substituting one out-of-inventory surface for a different one on a marked line is not mechanically detectable by the audit and is not owed an additional discriminator. Delete the illustrative `pi.alpha → pi.beta` example and the per-line-scope / free-text-justification explanatory clause. Preserve a brief mitigation reference forward-linking to step 2(b) branch (4) (`version-bump-step2b.md#bump-step-2b-stale-rewrite`).

## Solution constraints

- The mitigation reference must remain recognisable: the *Stale-citation limitation* sibling sub-bullet on this page cross-references "the same contributor obligation … that mitigates the *Substitution limitation* above", and that link must continue to resolve.

## Problem

`V13c` bundles two distinct deliverable classes in one leaf: the query tool loop itself (free-phase round advancement, the typed two-phase forced-respond turn, `max_rounds:0`, and the untyped `tool_loop_exhausted` path) and a depth-6 ceiling-#4 co-fire live-carrier vector that pulls in `V5e`, `V16a`, and the `RuntimeEvent` shape owned by `V9d`. The plan elsewhere gives each ceiling-#4 depth-6 live carrier its own dedicated leaf (`V14e`, `V15j`, each depending on its feature core plus `V5e`); the typed-query-response row alone is folded inline, so the decomposition is applied inconsistently across structurally identical routing rows. The inline vector also asserts the `masked` enumeration on a `V9d`-owned `RuntimeEvent` while `V13c` carries no `Deps` edge on `V9d`, leaving that assertion ungrounded.

Delete the depth-6 co-fire Tests bullet from `V13c-query-tool-loop.md` and `V13c-T-query-tool-loop.md` and drop the now-unused `V5e` Deps entry from both, retaining the loop mechanics, `max_rounds:0`, untyped exhaustion, and the `ERR-13`/`V4f` delegated witness on `V13c`.

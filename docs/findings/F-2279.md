## Problem

The glossary's `no-params bypass` / `single-string bypass` pair entry pins both static trigger predicates, both warning codes (`loom/load/bind-echo-without-params`, `loom/parse/bind-echo-on-bypass`), and the no-params-before-single-string check-ordering invariant. The glossary charter restricts each entry to a one-paragraph descriptive reminder and reserves normative binder-bypass behaviour for the canonical binder page. The duplicated routing rule is a maintenance hazard: a later one-sided change leaves the glossary asserting stale law, and the fuller entry invites an implementer to read the glossary as authoritative.

## Issue introduction
**Verdict:** multi-commit-interaction
**Introducing commits:** `b4fbfac` (glossary charter), `8168ea8`, `9aeb318`, `94e292f`, `c45de81`, `22619b0`, `0e2eb05`, `51ae8f1`, `ace24a0`
**History:** The descriptive-only charter ("Each entry is descriptive — a one-paragraph reminder … the canonical page wins") was established at the glossary's inception in `b4fbfac` ("resolve 'no central glossary'"). The four entries then accreted normative content across a sequence of later "resolve …" commits, each of which placed a routing or predicate rule into the glossary entry rather than confining it to the canonical page: `8168ea8` ("T30 ceiling #4 model-driven row trichotomy carve-out") created the `query-terminating` and `in-loop` anchored entries already carrying ceiling-#4 routing detail, with `ace24a0` and `51ae8f1` extending them; `9aeb318` / `94e292f` / `c45de81` built up the `no-params` / `single-string` bypass trigger, warning-code, and ordering detail; `22619b0` and `0e2eb05` added the slot-accounting bookkeeping and CIO-4 cross-references. The scope violation is therefore the cumulative product of the inception-set charter interacting with many subsequent content-placement decisions, not a single defective commit.

## Solution approach

Narrow the `no-params bypass` / `single-string bypass` glossary entry to a definitional stub naming the pair as two static binder-bypass conditions decided at loom-load time, both suppressing the binder and constructing no envelope, with a forward-link to `binder/binder-bypass-and-envelope.md#binder-bypass`. Delete the trigger predicates, the two warning codes, and the check-ordering rule; the canonical page already carries them (items 1–2 and the no-params-before-single-string sentence).

## Solution constraints

- Out of scope: the `query-terminating` / `in-loop` entries (owned by F-2276 / F-2275) and the `tool-call round slot accounting` entry — edit only the `no-params bypass` / `single-string bypass` entry.
- MUST NOT strip normative content from the glossary entry that is not already present on the canonical binder page — confirm the predicates, both warning codes, and the ordering rule survive canonically before deleting them.

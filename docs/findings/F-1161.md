## Problem

Four executable REQ-IDs each bundle multiple independently-testable obligations under one anchor, defeating the cite-by-ID contract GOV-1 / GOV-3 / GOV-8 / GOV-9 deliver: DISC-1 (positive seam-routing plus two distinct negative prohibitions), DISC-6 (global walk caps, per-read deadline race, wholesale disable), FRNT-1 (`tool_loop.max_rounds` semantics plus parse-time range rejection), and PIC-8 (four sequential restore-failure MUSTs). A reviewer or test that fails one half of a bundle cannot cite a narrower identifier than the whole. Separately, the DISC-1 rule body cross-references the `pi-integration-contract.md` hub page rather than PIC-13's defining anchor; per GOV-9 every cross-page reference to a normative rule MUST resolve as a `#prefix-n` fragment of the depended-upon REQ-ID, so the hub link paraphrases past that contract.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:**
- `a110739` (2026-06-04) — "pi-loom spec: resolve 'discovery.md carries zero DISC-N REQ-IDs'" — coined DISC-1 and DISC-6 in their current bundled form.
- `d4ac81b` (2026-06-04) — "pi-loom spec: resolve 'PIC sections beyond Probe-wide invariants — missing REQ-ID anchors'" — coined PIC-8 in its current four-MUST bundled form.
- `4af6fb3` (2026-06-07) — "pi-loom spec: resolve 'Ceiling #1/#2 enforcement obligations carry no REQ-IDs'" — coined FRNT-1 across the `tool_loop` paragraph including its range-rejection clause.

**History:** `git log -S '<a id="disc-1"></a>'`, `git log -S '<a id="pic-8"></a>'`, and `git log -S '<a id="frnt-1"></a>'` each surface exactly one coining commit (plus the later size-cap split commit `f5e89f4`). Each of the three coining commits is a *bulk REQ-ID coinage pass* closing an upstream "carries no REQ-IDs" finding — the work item was framed as "give these paragraphs anchors", not "decompose them per GOV-8 atomicity expectations". The defect is the interaction: each commit honoured GOV-22 (coin an anchor on every defining obligation site) but none consulted GOV-8 *Split* hygiene against the bundled paragraph it was anchoring, and there was no upstream convention at the time that pinned "one anchor per independently-testable MUST" as a constraint on coinage. The DISC-1 hub-page xref is co-resident in `a110739` — DISC-1 was coined and back-linked in the same commit, with the existing hub-page link inherited verbatim rather than retargeted to PIC-13.

## Solution approach

Split each of the four bundled REQ-IDs per GOV-8 *Split* into one sub-ID per independently-testable obligation — DISC-1 and the DISC-1 xref retarget on `discovery/discovery-sources.md`, DISC-6 on `discovery/package-and-settings.md`, FRNT-1 on `frontmatter/frontmatter-fields-b-and-templates.md`, PIC-8 on `pi-integration-contract/tool-registration-lifetime.md` — retiring each parent ID on its page's `## Retired REQ-IDs` table, appending fresh IDs per GOV-3, and updating the matching `coverage-matrix.md` row in the same commit. Retarget the DISC-1 rule-body cross-reference from the `pi-integration-contract.md` hub link to PIC-13's defining anchor (`#pic-13`). Repoint inbound `#disc-1` / `#disc-6` / `#frnt-1` / `#pic-8` fragment links across the corpus (including under `plan_topics/`) in the same diff as each split so none dangles when its parent ID retires.

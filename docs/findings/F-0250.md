## Finding

The **Deps** field of both `V4e` (Pre-evaluation failures) and its paired
`V4e-T` tests leaf carries a long inline parenthetical describing the
`ReloadFailureInjector` seam decomposition — who declares the interface
(`V9b`), which arms each dependency contributes (`V9b`: registry-swap and
`.loom`/`.warp` re-parse; `V10d`: settings-re-merge), and the fact that
`ERR-7`'s synthetic failure is injected through that one interface. That same
information is already stated, in fuller form, inside the `ERR-7` **Tests**
bullet of each leaf.

The leaf format (`conventions.md` *Leaf format*, **Deps** rule) defines the
**Deps** field as a list of leaf IDs that must be complete first — bare IDs,
ranges where contiguous, comma-separated otherwise. It does not provide for
rationale prose. The parenthetical is therefore cruft in a field that should
hold only the dependency list, and it duplicates a narrative the `ERR-7`
bullet already owns.

Because the two surfaces restate the same seam ownership, they can drift: a
later edit to the `ERR-7` bullet that does not also update the Deps
parenthetical leaves a reader unable to tell which surface is authoritative.
The Deps parenthetical also reaches inside itself to name `V4e` (a
self-reference) and the contributing leaf IDs as prose, which is exactly the
kind of free text the bare-ID Deps convention exists to keep out of the field.

## Issue introduction

**Verdict:** introduced-by-later-edit (multi-commit amplification)
**Introducing commit:** `66acde6` — "pi-loom plan: resolve 'ERR-7 test injects at an undefined channel seam'"
**Amplifying commits:** `b2d5a1f`, `d64dce5`, `fdce2a2`, `f14e929`
**History:** The plan corpus is git-tracked. At leaf creation (`c6a664e`) the
`V4e` Deps field was a bare list (`V4e-T, V9a, V6a, V11f, V10a, V16a`) with no
parenthetical, and the `ERR-7` Tests bullet was a single line
("watcher-reload failure routes pre-eval."). Commit `66acde6` added `V9b`/`V10c`
to Deps and, in the same commit, attached the seam-decomposition parenthetical
to the Deps field while simultaneously expanding the `ERR-7` bullet with the
matching seam ownership — creating the duplication at that point. Subsequent
resolve commits (`b2d5a1f`, `d64dce5`, `fdce2a2`, `f14e929`) reworded and
grew both surfaces in parallel (introducing the `ReloadFailureInjector`
interface naming and the later `V10c`→`V10d` rename), keeping the two copies in
lockstep but never collapsing them.

## Solution Space

**Shape:** single

### Recommendation

Strike the parenthetical rationale from the **Deps** field of both leaves,
leaving the bare leaf-ID list per the `conventions.md` Deps rule.

- In `docs/plan_topics/V4e-pre-evaluation-failures.md`, the **Deps** line
  becomes the leaf IDs only:
  `**Deps.** `V4e-T`, `V9a`, `V6a`, `V11f`, `V10a`, `V16a`, `V9b`, `V10d``
- In `docs/plan_topics/V4e-T-pre-evaluation-failures.md`, the **Deps** line
  becomes:
  `**Deps.** `V9a`, `V6a`, `V11f`, `V10a`, `V16a`, `V9b`, `V10d``

The seam decomposition the parenthetical carried (which leaf declares the
`ReloadFailureInjector` interface and which arms `V9b`/`V10d` contribute) is
already present in the `ERR-7` Tests bullet, so no information is lost by the
deletion. Do not edit the spec — this fix is internal to the two plan leaf
files. Keep the leaf IDs themselves unchanged.

Note for the implementer: findings on the same `V4e` Deps/Adds surface decide
*where* the seam decomposition should ultimately live (Adds vs the Tests
bullet). This fix only removes the redundant copy from the Deps field; it does
not move the surviving narrative, so it composes with whichever of those
decisions lands.

## Consequence

**Severity:** cosmetic

If shipped unfixed nothing breaks for the leaf implementer — both surfaces
currently agree. The cost is maintenance: two copies of the same seam
decomposition can drift out of sync under later edits, and the rationale prose
violates the bare-leaf-ID Deps convention that keeps the field
machine-readable for the pickability predicate.

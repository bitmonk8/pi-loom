## Problem

The Depth Enforcement "Error shape." per-boundary table in `docs/spec_topics/schema-subset.md` numbers its five enforcement-point rows `#1`…`#5` in the first column. Inside the same table and the paragraph that follows it, unqualified `ceiling #N` references mean the *hard ceiling* of that ordinal, not the table row of that ordinal — row #2's "exhausts under ceiling #2" means hard ceiling #2 (`tool_loop.max_rounds`), and row #4's "routes through ceiling #3's no-retry classification" means hard ceiling #3, neither of which relates to the like-numbered row. The bare references collide lexically with the table's own row numbering, so a first-time reader cannot tell they reach out to the hard-ceilings list rather than to a sibling row. The parallel table in `hard-ceilings/ceilings-3-and-4.md` does not number its rows, so the collision is unique to schema-subset.md.

## Issue introduction

**Verdict:** present-since-inception (of the per-boundary table structure)
**Introducing commits:** `1750848` (2026-05-07) — *pi-loom spec: resolve "Ceiling #4 routing class collapses four boundaries whose destinations differ"*
**History:** `git log -S "routes through ceiling #3"` on `docs/spec_topics/schema-subset.md` (and its pre-rename path `spec_topics/schema-subset.md`) returns a single commit, `1750848`. That commit is the one that converted the prior single-paragraph "Error shape." prose into the per-boundary table with rows numbered `#1`…`#5` and that simultaneously introduced the `ceiling #3's no-retry classification` phrase in row #4 and the `exhausts under ceiling #2` phrase in row #2. The collision between the table's row-`#N` labels and the unqualified hard-`ceiling #N` references therefore has existed for the entire lifetime of the table — no later edit either widened or narrowed it. The May 2026 commit was a refactor whose goal was to split the conflated routing-class summary across five boundaries; it did not consider whether the new row labels would lexically collide with the hard-ceiling references the migrated prose already contained.

## Solution approach

Clarify the bare `ceiling #N` references in the Depth Enforcement "Error shape." table and the paragraph that follows so each names its hard ceiling explicitly instead of colliding with the table's row-`#N` labels. Row #2's "ceiling #2" is hard ceiling #2 (`tool_loop.max_rounds`, anchored at `#cio-4` in `hard-ceilings/ceilings-3-and-4.md`); row #4's "ceiling #3's no-retry classification" is hard ceiling #3's `#hc3-c` sub-obligation. Anchor the first bare `ceiling #N` occurrence in the follow-up paragraph to the same hard-ceiling targets.

## Solution constraints

- Do not renumber or rename the table's `#1`–`#5` row labels; they are consumed by the `#cio-3` "five sites, in the same order as the per-boundary table" cross-reference in `hard-ceilings/ceilings-3-and-4.md` and by the leading-paragraph "five enforcement points" enumeration.

## Problem

`cancellation.md` §*Race semantics — swallowing-handler attachment on every abandonable Promise* carries an un-anchored normative `MUST` (GOV-22 residue): the runtime MUST keep a swallowing handler attached to every abandonable Pi-returned Promise so a late post-checkpoint rejection is absorbed across all three side channels. Per `conventions.md` *REQ-ID discipline*, every such un-anchored MUST on a non-narrative `spec_topics/**` page must appear as one row in the `## Code-keyed obligation areas (no numbered REQ-IDs)` table of `coverage-matrix.md`, mapped to its closing leaves. No matrix row maps this obligation, while the sibling un-anchored cancellation MUST (§*Forwarding-listener throw*) was added to that table at `V17b`, so the omission is asymmetric. The obligation closes at `V17a`, `V14a`, `V13f`, `V15a`, and `V9i`.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** 2ebe288 — pi-loom spec: resolve "Post-cancel late Promise settlement: unhandledRejection discard" (2026-05-07, spec author)
**History:** The swallowing-handler attachment obligation entered `cancellation.md` as an un-anchored normative MUST in 2ebe288; d9f42d0 (2026-05-08, "T42 Race semantics — split four cancellation MUSTs into per-obligation paragraphs") reshaped the race-semantics MUSTs into per-obligation paragraphs but left this one un-anchored. The `coverage-matrix.md` Code-keyed table never carried a row for it (a pickaxe over `swallowing-handler` and `abandonable` returns no matrix commit), even though the sibling forwarding-listener-throw cancellation MUST was added to that table on 2026-06-17 (9a131a3) — so the gap is an omission present continuously since the obligation entered the spec.

## Solution approach

Add one row to the `## Code-keyed obligation areas (no numbered REQ-IDs)` table in `coverage-matrix.md` enumerating the `cancellation.md` swallowing-handler attachment / three-side-channel suppression MUST, mirroring the existing forwarding-listener-throw row at `V17b`. Map the obligation's closing-leaf cell to `V17a`, `V14a`, `V13f`, `V15a`, `V9i`.

- The spec is read-only for this fix: do not anchor the MUST with a new `CNCL-n` REQ-ID or otherwise edit `cancellation.md`.

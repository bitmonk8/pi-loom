## Problem

`frontmatter-fields-a.md`'s `#argument-hint-binder-only` paragraph is the declared canonical owner of the slash-command autocomplete contract for the shared `description` and `argument-hint` frontmatter fields, and it directs that the architecture-hub page forward-link to it rather than restating the contract. The slash-command discovery opening bullet in `pi-integration.md` honours this for `argument-hint` (it forward-links the owner anchor) but still restates the `description` half inline ("The `description` field from frontmatter is what populates the autocomplete entry..."). This is a single-source-of-truth violation: the hub page restates behaviour the owner page already pins, which a future edit to the owned definition would leave silently stale.

## Issue introduction

**Verdict:** present-since-introduction
**Introducing commits:** `53f5831` ("spec: rewrite query primitive, system prompts, and schema syntax", 2026-05-03) introduced the unsourced `description` ‚Üí autocomplete claim in its current form when it first added the slash-command-discovery bullet (`"The description and argument-hint from frontmatter populate the autocomplete entry."`). `1e103d5` ("spec: rescope argument-hint to binder grounding only (Decision #7)", 2026-05-05) then asymmetrically split the bullet ‚Äî giving `argument-hint` a fully-cited SDK-gap framing (forwarded to Future Considerations and the `frontmatter-fields-a.md` autocomplete-contract anchor) while *retaining* the `description` half as a bare behavioural assertion, which is the precise asymmetry this finding flags.
**History:** `git log -L '/Slash-command discovery/,+1:docs/spec_topics/pi-integration.md'` and `git log -G 'populate the autocomplete entry'` show a four-commit chain (`fecb504` ‚Üí `d6cbb37` ‚Üí `1e103d5` ‚Üí `4aef0e2` ‚Üí `4da6c6c`): the bullet originated in `53f5831` against the top-level `spec.md`, was carried into the split `spec_topics/pi-integration.md` by `fecb504`, and the description-vs-argument-hint split was performed in `1e103d5` (the only commit that touched the rendering language); subsequent commits only adjusted the discovery-sources clause and the `argument-hint` cross-reference. No commit ever added a host-behaviour anchor for `description`. The defect is therefore present-since-introduction in `53f5831` and made asymmetric (and more conspicuous) by `1e103d5`'s targeted rescope of the sibling clause.

## Solution approach

Rewrite the autocomplete-contract sentences in `pi-integration.md`'s slash-command discovery opening bullet so the `description` half is folded into the existing forward-link to `frontmatter-fields-a.md#argument-hint-binder-only`, pointing the hub page at the owner definition for both shared fields rather than restating the `description` behaviour inline.

## Solution constraints

- Out of scope: the slash-command discovery and `.warp`-exclusion prose in the same bullet ‚Äî edit only the `description`/`argument-hint` autocomplete-contract sentences.

## Finding

The `cka-23` row partitions its obligation into four facet sub-labels, each a `cka-23.<facet>` token cited inline by the matching `V9b` / `V9b-T` *Tests* bullets. Three of the four render multi-word facet names with a hyphen separator (`cka-23.note-empty`, `cka-23.note-rename`) or are single words (`cka-23.swap`); the fourth, `cka-23.getcommands`, is a two-word name (`get` + `commands`, after the `pi.getCommands()` API) written as a run-together compound with no separator.

The inconsistency is purely lexical: the token is a stable facet key shared across `docs/plan_topics/coverage-matrix.md` (the `cka-23` row), `docs/plan_topics/V9b-registration-reload-wiring.md`, and `docs/plan_topics/V9b-T-registration-reload-wiring.md`. Because the token is matched literally across those three files, the divergent spelling is harmless to execution but breaks the readable convention the sibling facets establish.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** c1fa860 — pi-loom plan: resolve "registration-steps.md V9b row bundles three obligation areas with no sub-IDs" (2026-06-18, Thomas Andersen)
**History:** The four `cka-23.<facet>` sub-labels were introduced together in c1fa860, the commit that split the `cka-23` row into independently-verifiable facets. `cka-23.getcommands` was authored without a hyphen in that same commit while its siblings `cka-23.note-empty` / `cka-23.note-rename` received one; the inconsistency has existed since the tokens first appeared and was not introduced by any later edit.

## Solution Space

**Shape:** single

### Recommendation

Rename the facet token `cka-23.getcommands` to `cka-23.get-commands` everywhere it appears, so it matches the hyphen-separated form of its sibling facets:

- `docs/plan_topics/coverage-matrix.md` — in the `cka-23` row's *Token* / facet prose, change `facet \`cka-23.getcommands\`` to `facet \`cka-23.get-commands\``.
- `docs/plan_topics/V9b-registration-reload-wiring.md` — in the getCommands() snapshot read-only Spec bullet, change `facet \`cka-23.getcommands\`` to `facet \`cka-23.get-commands\``.
- `docs/plan_topics/V9b-T-registration-reload-wiring.md` — in the corresponding Spec bullet, make the same change.

All three occurrences must change together so the token stays a single consistent key across the coverage-matrix row and the paired `V9b` / `V9b-T` leaves.

## Consequence

**Severity:** cosmetic

The compound token is harder to read and breaks the hyphen-separated convention its three siblings follow; no implementer would diverge on behaviour, since the token is matched as a literal string and the facet's meaning is fixed by its prose. The cost is readability and convention consistency only.

## Finding

Two separate `## Acceptance criteria` enumerations cover the same `LoomRegistry.readDrainState` drain-state surface, and both open their fixture lists with the bare letters `(a)`, `(b)`, … :

- `drain-state-contract.md` → **Acceptance criteria — `(drained: true, tag: undefined)` → arm (b)** carries two fixtures `(a)` reachability and `(b)` arm-(b) routing, scoped to `reason ∈ {"quit","reload"}`.
- `session-only-degraded-state.md` → **Acceptance criteria** carries thirteen fixtures `(a)`…`(m)`, scoped to `reason ∈ {"new","resume","fork"}` plus the read-failure and unknown/snapshot-failure arms.

The two label spaces overlap on `(a)` and `(b)` with no qualifying prefix or registered identifier to tell them apart. The overlap is not hypothetical: cross-page references already cite these fixtures by bare letter — `host-prerequisites.md` cites "acceptance criterion (g)", `session-shutdown-semantics.md` cites "branch (k) under *Acceptance criteria*", and `session-only-degraded-state.md` cites "(i)/(j)" — so a reader who encounters "branch (b)" or "(a)/(b)" cannot determine from the letter alone which file's enumeration owns it. Compounding the local ambiguity, the `readDrainState` arms themselves are labelled `(a)`/`(b)`/`(c)` by PIC-29, so within `drain-state-contract.md` the string `(b)` denotes both an arm and a fixture.

This is purely an identifier-namespace defect; both enumerations are individually correct and the underlying drain-state behaviour is fully specified. The spec corpus has already met the identical hazard elsewhere and resolved it with a prefix: `patch-skew-degradation.md` labels its fixtures `(P-a)`/`(P-b)`/`(P-c)` with a `"P-"` prefix "deliberately distinct … so the three adjacent enumerations cannot collide on cross-reference." The drain-state fixtures never received the same treatment.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** f5e89f4 (`docs: enforce 100KB hard / 30KB recommended size cap on spec set`, 2026-06-04), 8008b91 (`pi-loom spec-plan: fix F-1217 — add acceptance-criteria/fixture blocks for four Shard-08 MUSTs`, 2026-06-25)
**History:** Both `drain-state-contract.md` and `session-only-degraded-state.md` were created by the spec-split commit f5e89f4. The session-only acceptance set was established on the new page shortly after (its throwing-`readDrainState` fixtures (l)/(m) were added by 2775382, 2026-06-05, resolving "Two throwing-readDrainState MUSTs have no acceptance criteria"). The cross-file partition was created much later by 8008b91 (`git show 8008b91 -- drain-state-contract.md` confirms the `## Acceptance criteria — \`(drained: true, tag: undefined)\` → arm (b)` heading was net-new in that commit), which added a second acceptance-criteria section in `drain-state-contract.md` for the arm-(b) MUST without consolidating it with, or cross-indexing it against, the pre-existing session-only set. The defect is the interaction: neither commit is wrong in isolation, but together they leave the single PIC-29 arm enumeration with two unconnected fixture homes and no index.

## Solution Space

**Shape:** multiple

### Option A — Consolidate into one enumeration

- **Approach:** Move the `drain-state-contract.md` `(a)`/`(b)` fixtures into `session-only-degraded-state.md`'s `## Acceptance criteria` so a single enumeration owns every `readDrainState`-arm fixture; leave a cross-reference anchor where the block was.
- **Spec edits:** Delete the *Acceptance criteria — `(drained: true, tag: undefined)` → arm (b)* section from `drain-state-contract.md`; append its two fixtures (renumbered into the `(a)`…`(m)` sequence, e.g. `(n)`/`(o)`) in `session-only-degraded-state.md`; repoint the `drain-state-contract.md:23` self-reference.
- **Pros:** One conformance checklist for the whole drain-state machine; only one label space to police.
- **Cons:** Relocates `reason ∈ {"quit","reload"}` (always-tear-down) fixtures onto a page titled and scoped for session-only reasons; the consolidated fixtures fall under that page's *Implementation gate*, which blocks fixtures `(a)`–`(m)` until the clause-(a) open contradiction resolves to `provably-differs`.
- **Risks:** Couples the currently non-gated arm-(b) reachability tests (`V9m-T`) to an unrelated blocked obligation, and contradicts the sibling-file gate boundary already flagged separately (see Related Findings).

### Option B — Disambiguate labels, keep enumerations co-located

- **Approach:** Keep each enumeration with its normative owner, but give the `drain-state-contract.md` fixtures a distinct prefix mirroring the corpus's own `P-` precedent on `patch-skew-degradation.md`, and qualify the bare cross-references.
- **Spec edits:** Relabel the two `drain-state-contract.md` fixtures with a non-colliding prefix (e.g. `(DSC-a)`/`(DSC-b)`, chosen so it collides with neither the `session-only` `(a)`…`(m)` space nor PIC-29's arm letters `(a)`/`(b)`/`(c)`); update the `drain-state-contract.md:23` self-reference; qualify the bare cross-references in `host-prerequisites.md`, `session-shutdown-semantics.md`, and `session-only-degraded-state.md` to name the owning file/section.
- **Pros:** Preserves co-location of fixtures with PIC-29 (their normative owner) and the gate boundary (`V9m-T` stays non-gated, `V9l-T` stays gated); consistent with the existing `P-` disambiguation pattern.
- **Cons:** Two enumerations remain; the prefix must be chosen carefully to avoid the arm-letter overload.
- **Risks:** If GOV-16 stable-inline-label machinery is invoked for the prefix it is heavier than warranted; a plain local prefix (not a registered inline-label class) suffices since these are list labels, not cross-page-citable obligation IDs.

### Recommendation

Adopt **Option B**. Disambiguate the two `drain-state-contract.md` fixtures with a distinct local prefix and qualify the bare cross-references. This keeps the always-tear-down `(quit/reload)` reachability fixtures co-located with PIC-29 and out of the session-only page's implementation gate, and follows the prefix-disambiguation convention the corpus already adopted for `patch-skew-degradation.md`. Edge cases for the fixer: the prefix must not collide with PIC-29's arm letters `(a)`/`(b)`/`(c)` nor with the `session-only` `(a)`…`(m)` space; the section heading's "→ arm (b)" (a `readDrainState` arm, not a fixture) stays as-is; and a plain list-label prefix is sufficient — do not register it as a GOV-16 inline-label class.

## Consequence

**Severity:** advisory

An implementer or auditor tracking conformance fixtures by bare letter can mis-map a cited criterion to the wrong file (the cross-references "acceptance criterion (g)", "branch (k)", "(i)/(j)" are unqualified), and within `drain-state-contract.md` the token `(b)` is overloaded between a `readDrainState` arm and a fixture. No runtime behaviour diverges; the cost is interpretive and citation-integrity risk.

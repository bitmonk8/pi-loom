## Problem

In `version-bump-triggers.md` step 5, the *PIC owns the routing semantics* sentence frames its first normative MUST around a human-judgement predicate — each failure output must be "unambiguously routable by a reader" to trigger (i) or trigger (ii) "without an out-of-band translation table". Because the spec deliberately does not enumerate the discriminator strings (the implementation owns the literal strings), no conformance test or auditor can mechanically decide whether a given failure output is ambiguous or unambiguous to a reader. The second MUST defines its satisfaction by reference to the first ("satisfies that routability obligation"), so it inherits the same unverifiable predicate. The enforceable properties the spec needs are already present nearby — orthogonal, individually-discriminable failures, a stable per-failure discriminator, and per-sub-trigger negative fixtures — but the MUST is phrased around subjective reader-routability instead of those concrete properties.

## Solution approach

Rewrite the *PIC owns the routing semantics* obligation in `version-bump-triggers.md` step 5 so the normative MUST keys on observable discriminator properties rather than on reader judgement: each failure carries exactly one stable test-output discriminator value, and that value maps the failure to either trigger [(i)](#bump-step-5-trigger-i) or trigger [(ii)](#bump-step-5-trigger-ii). Demote the "unambiguously routable by a reader … without an out-of-band translation table" clause to a non-normative design-intent note. Keep the rewrite consistent with the adjacent "orthogonal, individually-discriminable failures" MUST and the per-sub-trigger negative fixtures.

## Solution constraints

- Out of scope: pinning a discriminator-string vocabulary; the "implementation owns the literal strings" delegation remains intact.
- Preserve the existing brand-string carve-out: `loom/typecheck/session-shutdown-reason-snapshot` remains the `tsc`-emitted discriminator for trigger (ii)'s type-equality sub-trigger and is not reframed as implementation-owned.

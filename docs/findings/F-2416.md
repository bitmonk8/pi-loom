## Problem

The *Infra-aborted-run carve-out* under `id="audit-non-empty-scan-canary"` tells CI parsers to exclude infra-aborted runs identified "by the presence of one or more infrastructure-failure records". But the *Pre-emission termination carve-out* under `id="audit-infrastructure-failure-handling"` permits a non-zero-exit run that emits zero records (OOM-kill, host shutdown, mid-write panic, torn-pipe write failure). Such a run carries no infra record, so the record-presence predicate fails to classify it as infra-aborted. A conforming CI parser then falsely flags the missing canary record as a canary-obligation violation, defeating the carve-out's purpose.

## Solution approach

Extend the *Infra-aborted-run carve-out*'s identification rule under `id="audit-non-empty-scan-canary"` so that a zero-record run is also excluded from the canary obligation alongside a run carrying ≥1 `infra`-class record. The branch to cover is a run with zero emitted records and a non-zero `npm test` exit. Forward-link the *Pre-emission termination carve-out* under `id="audit-infrastructure-failure-handling"` as the path this branch covers.

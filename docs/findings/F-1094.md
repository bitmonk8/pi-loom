## Problem

The manual real-host smoke gate is the sole backstop against the in-process Pi session double diverging from a live host on its four fidelity-contract axes (streamed-token order relative to `ctx.waitForIdle()`, single-turn prompt-mode append, the `pi.on` cancel-forward subscription, cancellation propagation); the `npm test` surface never exercises a live host. Pass/fail criterion (d) in `real-host-smoke-gate.md` requires only that the four axes "hold" as the single composite `H7a` fixture happens to exercise them, with no requirement that each axis be witnessed individually — so an axis the composite path does not stress can pass without ever being checked. The gate is also purely pre-merge and contributor-driven, leaving no post-release detection path for a divergence that escapes it.

## Issue introduction

**Verdict:** introduced-by-decision, refined by later commits

**Introducing commits:**
- `07403da` (2026-06-10) — "resolve T20 (Branch A) — H4a double fidelity is an editorial-review presupposition, not a real-host backstop; unpark" — introduced the "cannot detect the double diverging from real Pi" / "loom 1.0 has no mechanical real-host fidelity gate" language into `H4a`, deliberately accepting the no-mechanical-gate posture this finding is about.
- `e7f14dd` (2026-06-11) — "resolve 'Real-host fidelity of the session double has no reproducible detection point'" — carried the no-mechanical-gate statement into `H6a` and pruned the corresponding review entry.
- `15eb44c` (2026-06-11) — "resolve 'real-host divergence pre-merge gate (posture i)'" — layered the manual pre-merge smoke gate onto `H4a`/`H6a`/`V18d` as the single backstop.
- `fd22a3d` (2026-06-17) — added `docs/plan_topics/real-host-smoke-gate.md`, the standalone governance note that now houses the single-composite-fixture manual gate.

**History:** The gap is intrinsic to the T20 Branch A design decision (`07403da`) that classified session-double fidelity as an editorial-review presupposition rather than a mechanical real-host backstop. The subsequent commits (`e7f14dd` → `15eb44c` → `fd22a3d`) added a single manual smoke run over one composite fixture as the only backstop, but never added per-axis pre-release coverage or any post-release detection — so the residual risk this finding names has been present continuously since the manual-gate backstop was first introduced.

## Solution approach

Rewrite criterion (d) in `real-host-smoke-gate.md` so the live run must individually witness each of the four session-double fidelity-contract axes, stating that a composite fixture that does not exercise an axis does not satisfy it. Mirror the per-axis witnessing requirement in `H6a`'s Ships-when evidence record so the recorded result attests each axis individually rather than a single composite pass. Where the composite `H7a` fixture cannot stress all four axes, extend or add a fixture so every axis is individually exercised by the live run.

## Solution constraints

- Out of scope: the deliberately-accepted residual posture that loom 1.0 has no mechanical real-host fidelity gate and no post-release detection — do not author an automated gate, CI hook, or post-release telemetry.
- Extending or adding a fixture MUST NOT alter `H7a`'s in-process double exact-match golden criterion, which is correct against the deterministic double.

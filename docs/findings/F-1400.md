## Problem

`V5f` / `V5f-T`'s `Adds.` names five lowering mechanisms, but three of them have no asserting `Tests.` bullet and are absent from `Ships when.`: per-query `$defs` pruning, the `__inline_<slug>` hoist with auto `$ref` rewrite, and the `integer`-vs-`number` discriminator sourced from the JSON-Schema `type` of the enclosing `const`/`enum` position. Each can be implemented wrong and ship green: a missing prune ships oversized request payloads, a missing hoist/`$ref` rewrite breaks inline-schema references, and a runtime-integrality discriminator yields a different canonical form. The integer-vs-number gap is the sharpest — the canonical schema hash is part of the on-disk and on-wire contract, so a discriminator sourced from runtime integrality rather than schema `type` produces a divergent canonical form, a different slug, and a silently broken contract that no current bullet catches.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** cb8737a — pi-loom plan: resolve "Systemic leaf over-bundling across the plan corpus" (2026-06-12, Thomas Andersen); 4807958 — pi-loom plan: resolve "renderer input contract: integer-vs-number discriminator origin" (2026-06-17, Thomas Andersen)
**History:** `V5f` was created in cb8737a when it was split out of `V5d`; that first commit already named the `__inline_<slug>` hoist/`$defs`/`$ref` and per-query `$defs` pruning in `Adds.` while supplying `Tests.` bullets only for slug-collision, the canonical hash, and the sidecar — so the pruning and hoist/`$ref` assertions have been absent since the leaf's inception. The integer-vs-number discriminator-origin sentence was added later by 4807958, whose diff appended the schema-`type`-sourced wording to `Adds.` but added no matching `Tests.` bullet, so its assertion was never present.

## Solution approach

Add three `Tests.` bullets to both `V5f-T-subset-lowering-hash.md` and `V5f-subset-lowering-hash.md` (tests leaf first, per the paired TDD ritual) asserting: per-query `$defs` pruning (cite `schema-subset.md` Lowering Algorithm step 4), the anonymous-inline-schema hoist into `$defs/__inline_<slug>` with use-site `$ref` rewrite (cite Lowering Algorithm step 2 + step 3 reference-emission rule), and the schema-`type`-sourced `integer`-vs-`number` discriminator (cite `schema-subset.md` § Canonical schema hash). Extend each leaf's `Ships when.` enumerated set to include the prune, hoist/`$ref`-rewrite, and schema-`type`-sourced number-rendering assertions.

## Solution constraints

- The spec is read-only; `schema-subset.md` already pins all three behaviours, so this fix adds no spec obligation and is internal to the two `V5f` plan leaves.

## Problem

The `## Binder model` opening paragraph negates a "tier-2" default tier that the resolution chain never defines and coins an unexplained "cost premise". Both terms were orphaned when the built-in default fallback was removed, leaving undefined jargon in the resolution paragraph. This makes the paragraph harder to read but does not change behaviour.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `0fa81fa` (jargon), `e64c7a3` (BNDR-12 testability)
**History:** The corpus is git-tracked. The "tier-2 model" descriptor entered with the binder at its inception (`83802db`, 2026-05-04) as a *positive* term — the third-tier "built-in default (cheap tier-2 model)" fallback. `0fa81fa` (2026-05-05, "pin binder model resolution") removed that built-in default ("**no further fallback**") but retained a now-antecedent-less *negated* reference ("no 'tier-2' default") and coined "cost premise" in the same edit, leaving both terms undefined. The "re-entrant per loom turn" wording first appeared as a *non-normative* escalation-seam note; `e64c7a3` (2026-06-28, "fix F-2094 — promote binder re-entrancy constraint to BNDR-12 MUST") promoted that note to the normative BNDR-12 MUST, which is the edit that introduced the untestable within-invocation second-call clause.

## Solution approach

Rewrite the parenthetical in the `## Binder model` opening paragraph in plain terms: state that resolution stops at `looms.binderModel` with no further fallback, and that the loom's own `model:` is deliberately not consulted because a separate `bind_model:` lets authors run binder calls on a cheaper structured-output-capable model than the loom body uses. Drop the quoted "tier-2" reference and the "cost premise" coinage; the existing non-normative note already supplies the cost rationale and example models.

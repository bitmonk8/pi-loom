## Problem

The closing paragraph of `### File-extension namespace` in `docs/spec_topics/discovery/discovery-sources.md` — beginning "This check is a point-in-time observation, not a guarantee." — names every property that disqualifies it as a normative statement (no REQ-ID, no per-leaf test obligation, no diagnostic) and then states its purpose as recording a past namespace-clearance decision. Sitting under a `### ` heading in a normative topic file, it imports decision-log prose into the implementation-target corpus, which [GOV-18 arm (b)](docs/spec_topics/governance/corpus-direction-and-scope.md#gov-18-arm-b) reserves for spec invariants. No other normative `###` subsection in the file carries a self-declared no-obligation disclaimer of this form.

## Issue introduction

**Verdict:** single-commit-introduction
**Introducing commits:** `5c697d0` ("pi-loom spec: resolve \"spec.md paragraph 2 — .loom / .warp file contract rewrite\"", 2026-05-06)
**History:** Commit `5c697d0` introduced the entire `### File-extension namespace` subsection at the then-`spec_topics/discovery.md` path, including the flagged closing paragraph verbatim aside from one sentence ("If a future Pi-ecosystem package adopts the same extensions, this section is the place to document the resolution.") that was later removed in `37b0428` ("Namespace-clearance subsection: missing --loom flag clearance and dated/editorial chrome", 2026-06-05) as part of trimming "dated/editorial chrome". The earlier review pass that produced `37b0428` recognised the chrome problem in the same paragraph and trimmed it partially without removing the self-declaration that drives the present finding; the residual self-declaration paragraph has been carried unchanged since then. Subsequent commits on the file (`f5e89f4` split, then `9dfbcee`/`7e440c9`/`d488eaa`/`f2a82ad`/`4a10cde`/`da4bd57`) touched DISC-3 / DISC-4 / ERR-7 only and did not interact with this paragraph.

## Solution approach

Delete the closing paragraph of `### File-extension namespace` — the paragraph beginning "This check is a point-in-time observation, not a guarantee." The `#file-extension-namespace` anchor sits on the preceding `### ` heading and is preserved, so the inbound cross-references that consume it remain valid.

## Solution constraints

- Out of scope: the three preceding paragraphs of the subsection (`.loom`/`.warp` clearance, `--loom` flag clearance, "no central file-extension registry" observation) — leave them unchanged.

## Problem

The *Sequential by default* rule in `conventions.md` gates its Promise-combinator exemption on the calling leaf's `Spec.` field citing a concurrency obligation. Horizontal leaves carry a `Convention.` field instead of a `Spec.` field, and several contribute `src/**` production code (`H3a`, `H4a`, `H5a`), so the rule's scope reaches them while its exemption path does not. The rule leaves the horizontal-leaf case undefined: either no combinator is ever permitted to such a leaf, or an unstated `Convention.`-field path exists. The sibling *Specific exception types only* rule resolved the analogous "no spec obligation to cite" problem with a structural token independent of `Spec.`, making the asymmetry conspicuous.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `fecb504` (2026-05-04, "Split spec.md and plan.md into per-topic / per-phase files" — first authoring of the *Sequential by default* rule as "No `Promise.all` / `Promise.race` outside slices that have a documented spec reason"); hardened to the explicit `Spec.`-field coupling around `c6a664e` (2026-06-10, "build/update plan for spec.md + review") and `75b6a9b` (2026-06-10, "resolve 'Sequential by default carve-out admits only a numbered REQ-ID'", which extended the cited obligation from a REQ-ID to a REQ-ID-or-code-keyed-area but left the `Spec.`-field coupling untouched).
**History:** `git log -S`/`-G` over `docs/plan_topics/conventions.md` shows the rule keyed its exemption to a spec reason from the file's first authoring (`fecb504`), and the `Convention.`-field-for-horizontal-leaves concept coexisted in the same *Leaf format* rule throughout. No commit ever closed the gap between the two: every later refinement (`c6a664e`, `75b6a9b`, `d31eeef`) tightened the obligation/scope of the cited token but never addressed the horizontal-leaf field-coupling. The defect is original to the rule, not introduced by a specific later edit.

## Solution approach

Clarify the *Sequential by default* rule in `conventions.md` so its exemption path explicitly addresses the horizontal-leaf class: state that the combinator exemption requires a `Spec.`-field citation and is therefore unavailable to horizontal leaves, whose `src/**` production code must be sequential. Point a horizontal leaf that genuinely needs concurrency at the *Spec drift* rule as the resolution path. Word the clarification so it does not contradict the existing `Adds.`-field clause, which presumes a `Spec.`-field citation accompanies any exempted combinator.

## Solution constraints

- Out of scope: leaf edits, the `H2a` closing-gate, and `coverage-matrix.md` — the conservative sequential reading already holds and no horizontal leaf currently uses a combinator in `src/**`.

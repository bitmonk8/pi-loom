## Problem

`session-only-degraded-state.md` carries RFC-2119 normative tokens across six defining-obligation sites — *Drain-state tag write*, *Recovery path and prohibitions*, *Diagnostic emission — Isolation*, *Inline triplet is normative*, *Predicate split — Seam minimality*, and *Co-emission rules* — none of which carries a `PIC-N` (or any) REQ-ID anchor; the page exposes only two narrative anchors. The closing `## Acceptance criteria` section enumerates fixture obligations `(a)`–`(m)`, each prose-cross-referencing the obligation it audits, but no fixture can target a stable identifier because none exists. AC items `(i)`/`(j)`/`(k)` assert obligations defined on the same page, so coining REQ-IDs without repointing the AC list leaves the testability gap open.

## Issue introduction

**Verdict:** indeterminate
**Introducing commits:** none identified
**History:** `session-only-degraded-state.md` was first added by commit `f5e89f4` (2026-06-04) as part of the split of the monolithic `pi-integration-contract.md`; that commit retained the source page's acceptance-grid and both anchors verbatim. Walking `git log -G` against the inlined `## Acceptance criteria` block on the pre-split file shows that block was already present without REQ-ID locators in every commit that touched it (most relevantly `2775382` which authored items `(l)`/`(m)`); the obligations and their acceptance criteria have co-existed without REQ-IDs since the section's inception, before GOV-22 was introduced (`4f588e0`, 2026-06-03). The defect is best classified as pre-rule corpus residue that GOV-22 absorbed under its "defect set drains progressively as pages are edited" carve-out, not an introduction by any single commit.

## Solution approach

Add `PIC-N` REQ-ID anchors at each defining-obligation site on `session-only-degraded-state.md` — *Drain-state tag write*, *Recovery path and prohibitions*, *Diagnostic emission — Isolation*, *Inline triplet is normative*, *Predicate split — Seam minimality*, and *Co-emission rules* — allocated under the already-registered `PIC` prefix per GOV-3. Repoint the `## Acceptance criteria` items so each fixture cites the REQ-ID(s) it audits via inline cross-reference rather than reaching in by paragraph keyword.

## Solution constraints

- Out of scope: do not coin REQ-IDs for obligation sites on other PIC pages or for un-edited sites on this page — GOV-22's coinage obligation fires only on sites this commit touches, and the systemic PIC-N coinage is owned by F-1344..F-1351.

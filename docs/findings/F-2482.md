## Problem

`diagnostic-shape.md`'s key-switch sentence requires the `loom-system-note` renderer to switch on which `details` key is present, but resolves the dependency only as "specified under 'System notes' and 'Runtime event channel' in [Pi Integration Contract](../pi-integration-contract.md)". That target is the PIC table-of-contents aggregator, which contains none of the named content. The `RuntimeEvent` payload shape, the Group-A always-log enumeration, and the four-shape key enumeration all live one hop further in `pi-integration-contract/runtime-event-channel.md`, where the Group-A list and the `RuntimeEvent` type block carry no `<a id>` anchors. A renderer author following the link lands on an index, must guess the child page, then scan it for anchorless bold labels.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** f5e89f4 — docs: enforce 100KB hard / 30KB recommended size cap on spec set (2026-06-04, sharding commit)
**History:** Before f5e89f4 the diagnostics content lived in `docs/spec_topics/diagnostics.md` and the sentence linked `[Pi Integration Contract](./pi-integration-contract.md)`, which at that time resolved to a single content page that actually contained the "System notes" and "Runtime event channel" sections. The f5e89f4 size-cap commit simultaneously split the diagnostics page into `diagnostics/diagnostic-shape.md` and split `pi-integration-contract.md` into 30 child shards — relocating those two sections (and the `RuntimeEvent` type / Group-A enumeration) into `pi-integration-contract/runtime-event-channel.md` while turning the parent into a table-of-contents aggregator. The link text was not updated to target the new shard or its anchors, so the cross-reference that was precise pre-split became a pointer to an index from that commit onward.

## Solution approach

In `pi-integration-contract/runtime-event-channel.md`, add stable anchors for the Group-A enumeration and the `RuntimeEvent` type block (the four-shape key enumeration already carries `#system-note-details-shapes`). Rewrite the key-switch sentence's `[Pi Integration Contract](../pi-integration-contract.md)` reference in `diagnostic-shape.md` to forward-link those anchors in the owning shard — covering both the "System notes" and "Runtime event channel" labels it names — rather than the table-of-contents aggregator.

## Solution constraints

- Do not duplicate the `RuntimeEvent` shape or the Group-A enumeration into `diagnostic-shape.md`; `pi-integration-contract/runtime-event-channel.md` remains their normative owner — keep the dependency as a cross-reference.

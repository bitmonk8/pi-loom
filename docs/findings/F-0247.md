## Finding

The `Adds.` field of the `V4c` leaf is two sentences. The first is a proper descriptive `Adds.` line — it states what the leaf introduces (the closed success/failure/cancelled trichotomy, the partial-append contract, turn-grain finality). The trailing second sentence is not a description of what the leaf builds; it is design rationale explaining *why* the no-rollback guarantee holds: "The no-rollback guarantee is architectural: the runtime contains no compensating path, so a completed callee's side effect survives a downstream terminal event by construction rather than by an undo being suppressed."

Both [`leaf-template.md`](docs/plan_topics/leaf-template.md) and [`conventions.md`](docs/plan_topics/conventions.md) §Leaf format specify `Adds.` as one descriptive sentence naming what the leaf introduces, and explicitly note that `Adds.` prose is non-binding by default — the binding obligations live in the `Tests.` REQ-ID bullets and the `Ships when` gate. The architectural rationale is therefore redundant in `Adds.`: the no-rollback behaviour is already bound by the `ERR-9`/`ERR-13` test bullets, and its normative home is the cited spec page [`error-model.md`](docs/spec_topics/errors-and-results/error-model.md).

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** e8f0236 — pi-loom plan: resolve "V4c-T/V4c assert no-rollback over later-slice surfaces" (2026-06-11, Thomas Andersen)
**History:** The leaf was created in c6a664e with a single-sentence `Adds.` field. A later plan-review-fix commit, e8f0236, appended the "The no-rollback guarantee is architectural: …" rationale sentence to `Adds.` while resolving a different finding about over-claimed no-rollback test surfaces; the rationale prose was added to the descriptive field rather than to the test bullets or the spec, introducing the misplacement.

## Solution Space

**Shape:** single

### Recommendation

Strike the trailing rationale sentence from the `Adds.` field of `docs/plan_topics/V4c-terminal-outcomes.md`, leaving only the descriptive first sentence. The text to remove is:

> The no-rollback guarantee is architectural: the runtime contains no compensating path, so a completed callee's side effect survives a downstream terminal event by construction rather than by an undo being suppressed.

The first sentence ("The closed success/failure/cancelled trichotomy … symmetric across cancellation and `?`-propagation.") stays as the leaf's `Adds.` line. The no-rollback guarantee remains bound by the `ERR-9` ("no compensating turn is injected") and `ERR-13` test bullets and is documented normatively at [`error-model.md` §No rollback](docs/spec_topics/errors-and-results/error-model.md); no relocation into the leaf is needed, so nothing else in `V4c` changes.

## Consequence

**Severity:** cosmetic

`Adds.` is descriptive and non-binding by convention, so the extra rationale sentence does not change what an implementer builds; it is organisational cruft that bloats the field and duplicates rationale already owned by the spec and the test bullets. No implementer divergence results from leaving it.

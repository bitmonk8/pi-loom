## Problem

`invocation.md`'s *Resolution* section defines discovery-root containment as a **segment-boundary** predicate (a resolved callee path is within a root only when it equals the root or begins with the root followed by a single path separator), and `id="inv-1"` pins this as a MUST across the load-time check and the invocation-time re-check. The sibling-prefix escape that makes the single-separator guard load-bearing — `/proj/foo-bar/evil.loom` passing a `startsWith("/proj/foo")` test while sitting outside root `/proj/foo` — lives only in explanatory prose; it carries no normative test-vector status. INV-1 conformance phrasing is therefore satisfiable by a suite that exercises a fully-outside path and never the discriminating sibling-prefix vector. A `startsWith`-based implementation missing the single-separator guard could violate the INV-1 MUST yet pass such a suite, leaving the security-relevant escape case unguarded by conformance testing.

## Issue introduction
**Verdict:** single-commit
**Introducing commits:** bd635ce
**History:** The segment-boundary containment predicate and its `/proj/foo-bar/evil.loom` sibling-prefix attack example were both added to `docs/spec_topics/invocation.md` by commit bd635ce (2026-06-07), which resolved an earlier finding ("Discovery-root containment predicate undefined at invoke-path-escape site"). `git log -S` confirms the `foo-bar/evil.loom`, `startsWith`, and `segment-boundary containment` tokens appear only in that commit; the same diff added the predicate as explanatory prose and the INV-1 seam but pinned no normative test vector. The testability gap thus entered the corpus together with the rule it describes — the prior commit had no containment predicate at all, so there was nothing to test before bd635ce.

## Solution approach

Pin the segment-boundary predicate's discriminating cases as normative test vectors under `id="inv-1"`, using the corpus's existing "normative test vectors; conformance tests MUST match" convention (as the diagnostics placeholder-rendering subsections do). Require the sibling-prefix escape vector specifically — a resolved callee path of the form `<root>-sibling/evil.loom` against active root `<root>` — so a bare-`startsWith` implementation fails red.

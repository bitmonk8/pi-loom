## Problem

The step-2 editorial-review checklist in `version-bump-step2.md` closes each SHOULD-level audit item across the (f)–(aq) range with the same escalation sentence stating the obligation is SHOULD-level only at loom 1.0 and escalates to a build-time pin once Pi exposes a typed contract that step 2(a)'s surface-inventory probe can mechanically verify. The sentence appears 37 times verbatim, varying only by the typed-contract noun (ordering, turn-liveness, partition, or bare "typed contract") that each item's body already names. This is a single escalation policy expressed once per item rather than once for the group; the per-item repetition carries no per-item information, and an editor changing the policy must edit 37 synchronised sites.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** 8c930fe — pi-loom spec: resolve "T22c — Pi version-bump procedure has no step for the session-binding contract" (2026-05-17, Thomas Andersen); 2123dc9 — fix F-0560 — surface V14b parallel-tool-mode presupposition via new bump checklist item (an) (2026-06-21, Thomas Andersen); 077afb5 — fix F-0806 — tools-allowlist suppression pinned to new bump checklist item (am) (2026-06-20, Thomas Andersen); db69f2b — fix F-0981 — promote loom-flag-namespace … item (ao) (2026-06-22, Thomas Andersen); bef37f3 — fix F-1228 — pin registry-persistence presupposition … item (ap) (2026-06-25, Thomas Andersen)
**History:** Commit 8c930fe (2026-05-17) introduced the SHOULD-level escalation sentence with a single item carrying it. The duplication then accreted incrementally: each later commit that added a new presupposition item to the checklist copied the same closing sentence verbatim (count grew 1→2→4→5 by 2026-06-04 and reached 36 at HEAD as items through (ap) were added one fix at a time). No single commit created the ~35× repetition; it is the cumulative product of the per-item authoring pattern.

## Solution approach

Hoist the escalation clause into a single framing paragraph covering the (f)–(aq) item block, generalising the per-item typed-contract noun to a corresponding typed contract (the specific contract is already named in each item's body). Delete the trailing escalation sentence from each item (f)–(aq) that carries it. Rewrite the step-2 preamble's "per each item's SHOULD-level scope and escalation clause" reference to point at the single framing paragraph instead of per-item escalation clauses.

## Solution constraints

- Out of scope: item (i) (`#bump-checklist-provider-overflow-wording`) — it carries a distinct escalation form (a provider-fixture CI-gate post-loom-1.0.0 follow-up), not the hoisted clause; do not delete or absorb it.

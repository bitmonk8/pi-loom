## Problem

The opening paragraph of `pi-integration-contract/conversation-drive.md` (anchor `#no-extra-mediation`) is a cross-version trust/security NFR: it states the runtime interposes no privilege layer, pins a `MUST NOT introduce additional access channels` rule across all loom 1.x releases, declares widening/narrowing a major-version concern, and fixes the host-side denial-surface rule (silent success on denial forbidden). None of this is conversation-drive material — it is the runtime's trust disposition. An operator or security reviewer scanning `overview-and-orientation.md § Non-functional requirements` will not navigate into a page titled "Conversation drive" to find a cross-version `MUST NOT`; the orientation **Trust boundary** bullet treats the rule as forward-linked normative content owned elsewhere, but forward-links into this misplaced paragraph rather than owning it.

## Issue introduction

**Verdict:** single-commit-introduction
**Introducing commits:** `52219c4` ("Address all 41 spec-review findings", 2026-05-06)
**History:** `git log -S "No additional access channels"` and `-S "no-extra-mediation"` both first show the paragraph and anchor in `52219c4`, which added the block (and the `<a id="no-extra-mediation"></a>` anchor) to the then-monolithic `docs/spec_topics/pi-integration-contract.md` as part of the *Trust-boundary cluster* of fixes (commit message lists "Host-side denial surface forward-linked" and "Callable set explicitly distinguished from host-process sandbox"). At that commit the block was already inserted under the Conversation-drive section of the PIC rather than under an NFR / trust-posture heading. The later file-split commit `f5e89f4` ("docs: enforce 100KB hard / 30KB recommended size cap on spec set", 2026-06-04) carved the PIC into a topic directory and carried the paragraph + anchor verbatim into the new `pi-integration-contract/conversation-drive.md`, preserving the original placement. No subsequent commit has touched the placement choice.

## Solution approach

Move the **No additional access channels** paragraph out of `conversation-drive.md` into `overview-and-orientation.md § Non-functional requirements`, attaching it to the existing **Trust boundary** bullet, and carry the `<a id="no-extra-mediation"></a>` anchor with it. Repoint the inbound cross-references that cite the anchor — in `overview-and-orientation.md`, `future-considerations/model-changes-and-non-goals.md`, `tool-calls.md`, and the `V14d` / `V14d-T` plan leaves — to `overview-and-orientation.md#no-extra-mediation`. Consolidate the orientation bullet's existing paraphrase against the relocated normative text so the `MUST NOT` lives in exactly one place.

## Solution constraints

- Preserve the anchor id `no-extra-mediation` unchanged across the move; do not rename it.
- Preserve the denial-surface clause verbatim (thrown / `isError: true` → `Err(QueryError { kind: "code_tool", cause: "execution" })`; silent success forbidden) — it is the spec pin the `V14d` / `V14d-T` plan leaves consume.
- Per GOV-30 (aggregator-vs-source lock-step), do not leave both a normative copy and a paraphrase of the rule; the relocated text is the single source of truth.

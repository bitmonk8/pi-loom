## Problem

The *Slash-command entry* bullet under **Forwarding into `loomAbort`** in `cancellation.md` asserts that "an `agent_end` event reporting a user-cancelled turn aborts `loomAbort`", but the runtime is given no way to recognise such an event: `AgentSessionEvent`'s terminal `agent_end` variant is pinned in `host-interfaces-core.md` only for its `messages` field and its `willRetry: false` shape, with no field, value, or `stopReason` documented as the user-cancellation indicator. Two implementers — one treating every terminal `agent_end` as cancellation, one gating on a guessed indicator — diverge silently, and CNCL-4's byte-exact `"loom cancelled by agent_end"` reason then fires under different conditions per build. Separately, the same bullet's parenthetical claims the slash-command handler is dispatched only from an idle, non-turn context, asserted as Pi-documented fact with no anchor; the related `operator-always-present-presupposition` pins operator presence but not the dispatch-from-idle predicate. Without that anchor, a Pi minor that dispatched a slash handler mid-turn would silently regress the load-bearing "tolerate `undefined` `ctx.signal`" obligation with no surface to fail against.

**History:** `git log -G "an \`agent_end\` event reporting"` and `git log -G "which is exactly when the slash-command handler fires"` on `docs/spec_topics/cancellation.md` (with its pre-move path `spec_topics/cancellation.md`) both first surface at `8395c03`. That commit rewrote the prior `**Signal sources:**` bullet list into the current *Signal source* / *Forwarding into `loomAbort`* prose; the rewrite introduced both the bare-prose `agent_end`-driven clause (with no detection predicate) and the unanchored parenthetical "which is exactly when the slash-command handler fires" in the same diff hunk. Neither clause has been touched by any subsequent commit — `git log --follow` against the file shows ten later edits to neighbouring CNCL-1…6 anchors, the forwarding-listener throw trap, the `complete()` reason-forwarding, and the swallowing-handler attachment, but no edit to the two defect sites themselves. The defect is therefore localised to one commit and one rewrite hunk; CNCL-4's byte-exact `"loom cancelled by agent_end"` reason added later (commit `d9f42d0`) consumed the predicate without supplying one.

## Solution approach

Clarify the `agent_end`-driven trigger in `cancellation.md`'s *Slash-command entry* bullet by specifying a detection predicate grounded in the loom 1.0 Pi-SDK pin's `agent_end` payload, and pin the same indicator (or record its absence) in `host-interfaces-core.md` alongside the existing terminal-`agent_end`-variant pin at `#messages-chronological-order-presupposition`. If the pinned payload carries no cancellation indicator, the trigger reduces to a terminal `agent_end` whose `ctx.signal` reads aborted — state which baseline holds so CNCL-4's reason rule is grounded against it. Separately, anchor the idle-non-turn-context dispatch premise by adding a named presupposition under `#pi-slash-handler-promise-lifecycle-presupposition` in `host-interfaces-core.md`, and replace the unanchored parenthetical in the cancellation.md bullet with a forward-link to that anchor.

## Finding

The slash-command-entry bullet under **Forwarding into `loomAbort`** asserts two distinct things that neither this page nor any forward-linked PIC page actually pins:

1. *`agent_end`-driven trigger without a detection predicate.* The text says "equivalently, an `agent_end` event reporting a user-cancelled turn aborts `loomAbort`", but it gives the runtime no way to recognise such an event. `AgentSessionEvent`'s terminal `agent_end` variant is pinned in [`pi-integration-contract/host-interfaces-core.md`](../spec_topics/pi-integration-contract/host-interfaces-core.md) only for its `messages: AgentMessage[]` field (chronological-order / leading-`user`-message presuppositions) and its terminal `willRetry: false` shape (presupposition (iv) of the slash-handler promise-lifecycle paragraph); no field, value, `stopReason`, or shape on the payload is documented as the user-cancellation indicator. Two implementers — one that treats every terminal `agent_end` as cancellation, one that gates on a guessed `stopReason: "cancelled"` or `reason` field — will diverge silently, and CNCL-4's byte-exact `"loom cancelled by agent_end"` reason then fires in different conditions on different builds. The `Suggested fix` margin of "or forward-link a PIC section" does not apply: no PIC section currently documents this predicate.

2. *Unanchored idle-non-turn-context premise.* The parenthetical "Pi documents `ctx.signal` as `undefined` in idle, non-turn contexts (which is exactly when the slash-command handler fires)" splits into two sub-claims. The first sub-claim *is* anchored — host-interfaces-core.md quotes the verbatim JSDoc "current abort signal, or undefined when the agent is not streaming" — but the second sub-claim, that the slash-command handler is dispatched *only* from such an idle/non-turn context, is asserted as Pi-documented fact with no anchor. The nearest related presupposition, `operator-always-present-presupposition` in [`host-prerequisites.md`](../spec_topics/pi-integration-contract/host-prerequisites.md), pins operator presence inside an interactive TUI session but does not pin the dispatch-from-idle predicate. Without that anchor, a Pi minor that dispatched a slash handler mid-turn would silently regress the load-bearing "tolerate `undefined`" obligation with no surface to fail against.

Both defects sit in the same bullet but resolve independently — one demands a new detection-predicate specification (or an admission that the trigger is identifying *terminal* `agent_end` regardless of cancellation, paired with the matching CNCL-4 wording), the other demands one new named presupposition (or forward-link).

## Solution Space

**Shape:** multiple

This finding bundles two independent obligations — a missing detection predicate and a missing presupposition anchor — that touch the same bullet but resolve through unrelated edits. They are decomposed into one option per obligation so the fixer can land them sequentially.

### Option A — Specify the `agent_end`-driven detection predicate

**Approach.** Decide whether the trigger is (a) any terminal `agent_end` for the user-session turn the slash handler is currently driving, or (b) a specific payload-shape predicate (e.g. `stopReason: "cancelled"` if such a value exists on `StopReason` at the pin, or a dedicated `cancelled: true` / `reason: "user"` field). The decision is gated by what `AgentSessionEvent`'s terminal `agent_end` variant actually carries at the loom 1.0 Pi-SDK pin (`dist/core/agent-session.d.ts`), which the spec already pins for `messages` and `willRetry: false` but not for any cancellation indicator.

**Spec edits.** In `cancellation.md`'s slash-command-entry bullet, replace "an `agent_end` event reporting a user-cancelled turn aborts `loomAbort`" with the predicate decided above, naming the payload field/value verbatim (or, if the predicate is "any terminal `agent_end` whose `ctx.signal` reads aborted from inside the handler", state that explicitly so the trigger is plainly redundant with the `ctx.signal`-aborted trigger and CNCL-4's synthesised-reason rule degrades to fallback). Add a parallel pin to `host-interfaces-core.md` under the `AgentSessionEvent` paragraph naming the same field/value (or noting its absence). Add a per-bump checklist row to `version-bump-step2.md` mirroring item (j) so a Pi minor that renames or restructures the cancellation indicator surfaces at bump time. Adjust `V17a`'s CNCL-4 reason-identity test and `V9c`'s `pi.on` cancel-forward wiring so they assert against the named predicate.

**Pros.** Removes the silent divergence between implementers; gives CNCL-4's byte-exact synthesised-reason rule a deterministic firing condition; adds one named version-bump audit step.

**Cons.** Requires reading the loom 1.0 Pi-SDK pin's `dist/core/agent-session.d.ts` to confirm what the cancellation indicator actually is (or that none exists, in which case the predicate degrades to the trivial form).

**Risks.** If the SDK has no cancellation indicator on `agent_end` at the pin, the only sound predicate is "terminal `agent_end` while `ctx.signal.aborted`", which collapses the `agent_end`-driven trigger into the `ctx.signal`-aborted trigger and makes the bullet's "equivalently, …" clause editorially redundant — the fixer must either delete the clause or reframe it as a belt-and-braces backstop, not silently keep the wording.

### Option B — Anchor the idle-non-turn-context premise

**Approach.** Add the "slash-command handler is dispatched only from an idle/non-turn context" claim as a named presupposition under [Pi-side slash-handler promise lifecycle (consumption posture)](../spec_topics/pi-integration-contract/host-interfaces-core.md#pi-slash-handler-promise-lifecycle-presupposition), with its own anchor (e.g. `slash-handler-idle-dispatch-presupposition`), and replace the unanchored parenthetical in `cancellation.md` with a forward link to it.

**Spec edits.** Insert one new presupposition paragraph at the appropriate point in the (i)–(vi) sequence in `host-interfaces-core.md` (this is presupposition (vii); the lifecycle paragraph already enumerates six and is the natural home). The paragraph names the source of truth (the `ExtensionContext.signal` JSDoc *"current abort signal, or undefined when the agent is not streaming"* is the symptom; the dispatch-from-idle predicate is the unstated cause) and records the failure mode (a Pi minor that dispatched a slash handler mid-turn would route the loom's "tolerate `undefined`" obligation past every signal-defined assumption). Replace the parenthetical in `cancellation.md` with `(see [slash-handler idle-dispatch presupposition](./pi-integration-contract/host-interfaces-core.md#slash-handler-idle-dispatch-presupposition))`. Add one new per-bump checklist row to `version-bump-step2.md` parallel to item (j) / (v).

**Pros.** Brings the claim into the existing presupposition framework so a future Pi change surfaces against a named anchor; mechanical, low-risk edit; mirrors the established pattern.

**Cons.** Adds one more presupposition to an already-long list (the lifecycle paragraph currently runs to six).

**Risks.** Very low — the presupposition is genuinely the load-bearing one for the surrounding `MUST tolerate undefined` obligation, and naming it strengthens rather than complicates the consumption-posture surface.

### Recommendation

Address **Option B first**, then **Option A**. Option B is a pure citation/anchor addition with no semantic decision; landing it first stabilises the presupposition framework that Option A's new version-bump row will plug into. Option A then lands against a clean baseline and can take whichever predicate the pinned `agent_end` payload actually supports without re-litigating the idle-dispatch premise. The fixer must, in Option A, also revisit `V17a`'s CNCL-4 Tests bullet so the synthesised-reason test asserts the predicate landed in the spec, not the legacy "any terminal `agent_end`" implicit reading.

## Consequence

**Severity:** correctness

Two reasonable implementers will diverge on when the `agent_end`-driven trigger fires: one wires every terminal `agent_end` to `loomAbort.abort(new Error("loom cancelled by agent_end"))`, surfacing spurious `cancelled` results on every clean turn-end; the other gates on a guessed `stopReason` and misses real user cancellations whose payload differs. The unanchored idle-dispatch claim is a slower-burn correctness risk — a future Pi minor that dispatched a slash handler mid-turn would silently invalidate the "tolerate `undefined`" obligation without surfacing against any named presupposition.

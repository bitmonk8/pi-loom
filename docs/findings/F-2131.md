## Problem

The normative *Column legend* (anchor `id="column-legend"` in `diagnostic-shape.md`) defines the closed *Phase* vocabulary for all four registry tables. Four glosses name a pipeline stage — `lex`, `parse`, `type`, `load` — but the fifth glosses `runtime` as "panic during execution", naming a single failure mode rather than the stage. The `loom/runtime/*` namespace it governs carries many non-panic codes (the `internal-error` runtime-defect surface and the delivery / rebuild / registration / teardown / restore-failure family), and the namespace's own section heading already reflects this. A spec author adding a new non-panic runtime diagnostic and consulting the gloss as the normative definition finds no stage that fits, and may mis-assign `load` or conclude no phase label applies.

## Issue introduction
**Verdict:** multi-commit-interaction
**Introducing commits:** `839db71` (coined the legend gloss); `11b9199` (first non-panic runtime code, which made the gloss too narrow); broadened by `1e2a6c7`, `9d7b1b0`, `0b195eb` and later accretion.
**History:** The *Phase* legend — including `runtime (panic during execution)` verbatim — was introduced whole in `839db71` ("resolve 'Diagnostic codes are declared as namespaces but never assigned to specific errors'", 2026-05-04), when the `loom/runtime/*` table held *only* panic codes (match-error, index-out-of-bounds, null-member-access, null-index-access, missing-object-key) under the section title "runtime panics"; at that point the gloss was accurate. The gloss became too narrow when the first non-panic runtime code (`system-note-delivery-failed`) was added in `11b9199`, and the gap widened as further non-panic codes accreted (`subagent-dispose-failure` in `1e2a6c7`, the `internal-error` runtime-defect reframe in `9d7b1b0`, `watcher-terminated` in `0b195eb`, and others). The section heading itself was updated to acknowledge "runtime-defect surface, and delivery failures", but the legend gloss was never carried along. The defect is the interaction between the inception gloss and the subsequent namespace expansion, not any single commit.

## Solution approach

Rewrite the `runtime` *Phase* gloss in the *Column legend* (anchor `id="column-legend"`) so it names the execution stage, parallel to the other four `lex` / `parse` / `type` / `load` stage glosses, rather than the single "panic during execution" failure mode.

## Solution constraints

- Out of scope: the *Phase* value on any `loom/runtime/*` (or other) registry row — every current row is already correctly phased; only the legend gloss changes.

## Finding

`V4f-T`'s `ERR-13` Tests bullet enumerates the six no-rollback authoring sites and the seams each is driven through, then appends a sentence that justifies the test methodology rather than asserting anything testable: "The guarantee rests on the runtime having no compensating/rollback path, so the tests witness it on these enumerated authoring sites rather than proving the absolute absence of a compensating path exhaustively." A reader scanning a test-case description expects the assertion and its drive mechanism, not the design argument for why an enumerated sample suffices.

The rationale already has an authoritative home. `V4f`'s `Adds` states the same architectural claim ("The no-rollback guarantee is architectural: the runtime contains no compensating path, so a completed callee's side effect survives a downstream terminal event by construction rather than by an undo being suppressed"), and `V4f`'s own `ERR-13` bullet and `Ships when` repeat it verbatim. The sentence on `V4f-T` is therefore duplicated narrative: it adds no test obligation the implementer can act on, and it widens the surface a future edit must keep mutually consistent across `V4f` and `V4f-T`.

The defect has therefore existed in this leaf file since its inception; no later edit introduced it.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V4f-T-no-rollback.md`, strike the trailing rationale sentence from the `ERR-13` Tests bullet — delete the text beginning "The guarantee rests on the runtime having no compensating/rollback path, so the tests witness it…" through "…proving the absolute absence of a compensating path exhaustively." so the bullet ends after the seam-and-checkpoint clause ("…together with the `V17c` checkpoint set, not the live `V14a`/`V13c`/`V15a` surfaces."). The same architectural rationale is already carried in `V4f`'s `Adds`, so removal leaves the claim owned in one place and loses nothing. Do not edit `V4f-no-rollback.md` — its `Adds` is the canonical home and must remain.

## Consequence

**Severity:** cosmetic

If shipped unfixed, the `V4f-T` implementer reads a methodology justification inside a test-vector description that imposes no testable obligation, and the architectural rationale now lives in three places (`V4f` Adds, `V4f` ERR-13/Ships-when, `V4f-T` ERR-13). The duplication is a divergence surface: a later change to how the guarantee is argued must be applied to every copy or one silently goes stale.

## Finding

`V16a`'s `Adds.` field opens with the seam description and then appends a defensive disambiguation clause: after the positive normative statement that the arbitration function "is imported and *consulted* directly at each call site, never constructor-injected," the text continues "and is therefore a categorically different thing from the spec's construction-time–injected host DI seams (the PIC-10…PIC-16 seams in `host-interfaces-services.md`, modelled by `V8a`–`V8e`); the No-globals/statics/singletons rule must not be misread as a directive to wire this stateless pure function as an injected interface or a module-level singleton."

That trailing clause is design rationale and a misreading warning, not a constraint that adds anything beyond "never constructor-injected." `conventions.md` §"Leaf format" defines `Adds.` as **one sentence — what the leaf introduces**, and explicitly **descriptive by default**: `Adds.` prose does not bind the implementer; binding comes from the `Tests.` REQ-ID bullets and the `Ships when` gate. The Class-2-vs-construction-time-DI-seam contrast is exactly the kind of architecture-shape rationale the convention treats as non-binding illustration, so it has no operative effect where it sits and inflates the field well past its one-sentence contract.

The positive statement ("imported and consulted directly at each call site, never constructor-injected") already pins the seam's wiring shape unambiguously; the appended "must not be misread" warning is belt-and-suspenders prose whose proper home — if preservation is wanted — is a `conventions.md` ADR on the no-globals rule's scope, not this leaf's `Adds.` field.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V16a-ceiling-order-masked.md`, edit the `Adds.` field to strike the rationale/warning clause while retaining the positive normative statement. Delete the run beginning "and is therefore a categorically different thing from the spec's construction-time–injected host DI seams" through "…or a module-level singleton." The sentence should terminate at "never constructor-injected." so it reads:

> Here "seam" means a **Class-2 named cross-leaf seam** per [`conventions.md`](./conventions.md) — the arbitration function is imported and *consulted* directly at each call site, never constructor-injected.

If the no-globals-scope rationale is worth preserving, relocate it to a `conventions.md` ADR/note on the no-globals/statics/singletons rule's scope (the natural home for cross-leaf architecture rationale), not back into any leaf's `Adds.`. The relocation is optional; the strike alone resolves the finding. The spec is read-only for this fix; no `Tests.`, `Deps.`, or `Ships when` text changes.

## Consequence

**Severity:** cosmetic

The clause violates the one-sentence-descriptive `Adds.` contract and adds reading overhead, but it misleads no implementer: per `conventions.md` §"Leaf format", `Adds.` prose is non-binding, the positive "never constructor-injected" statement already fixes the wiring shape, and the warning (if read) only reinforces correct behaviour. Shipping it unfixed degrades plan tidiness, nothing operative.

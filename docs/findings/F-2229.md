## Problem

The *Recovery path and prohibitions* section of `session-only-degraded-state.md` pins the "edits silently no-op" guarantee on an exhaustive set of MUST NOTs (no un-drain, no re-subscribe, no polling, no other self-recovery path) and explicitly states the guarantee is delivered by that prohibition, not by the implicit absence of other paths. No acceptance criterion exercises that guarantee. The *Acceptance criteria* fixtures (a)–(m) cover diagnostic counts, drain-state arms, idempotent re-entry, and the `readDrainState` fail-safe sites, but none observes the post-degradation editor-edit path. An implementation that grew a polling fallback or a re-subscribe path — exactly what the prohibition forbids — would satisfy every existing fixture and pass the suite.

## Issue introduction
**Verdict:** single-commit
**Introducing commits:** 52b9101 — pi-loom spec: resolve "T24 — Fork-reason watcher closure leaves the extension in an unspecified, silently degraded state" (2026-05-11, Thomas Andersen)
**History:** The degraded-state branch — the exhaustive recovery-path prohibition, the "edits silently no-op" guarantee, and acceptance criteria (a)–(k) — was authored as a single block in commit 52b9101 inside `pi-integration-contract.md`; that initial criteria set never included a fixture exercising the prohibition. The split into `session-only-degraded-state.md` (f5e89f4, 2026-06-04, size-cap split) carried the gap unchanged, and the later addition of criteria (l)/(m) for the `readDrainState` fail-safe sites did not close it. The defect has therefore been present since the branch was first specified.

## Solution approach

Add an acceptance criterion to the *Acceptance criteria* section of `session-only-degraded-state.md` exercising the post-degradation edit path. After a degraded-state shutdown per criteria (a)–(k), the fixture writes a change to a `.loom` or settings file, issues a slash invocation, and asserts the command returns the degraded note (`"loom /<name>: extension degraded; /reload to recover"`) rather than a loom result reflecting the change, and that the injected watcher seam trace shows no watcher construction and no further `discoveryWatcher.close()` / `settingsWatcher.close()` after the handler returns.

## Solution constraints

- Pin the slash-note assertion on the non-throwing tag-write arm: the throwing interstitial-seam tag-write arm (criterion (k)) returns `"shutting down"`, so an unconditional `"degraded; /reload to recover"` assertion would clash with (k).
- Out of scope: the edit-without-slash-invocation case (accepted residual gap 2, same-cluster with F-2228); this criterion verifies the prohibition with a subsequent slash invocation only.

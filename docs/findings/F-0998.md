## Problem

The single normative paragraph that enumerates the complete loom 1.0 routing of every `execute()` outcome at the code-side `<name>(args)` boundary lives only in `docs/spec_topics/pi-integration-contract/host-interfaces-core.md` as the `*Outcome routing summary.*` paragraph, on a page whose title signals host-shape pinning rather than author-facing surface contract. The author-observable contract — what `<name>(args)` returns for each disposition of `execute()` — belongs topically in `docs/spec_topics/tool-calls.md`, which today carries only fragments (the `**Failures.**` `CodeToolError.cause` enum and the `*Outcome enumeration (normative).*` off-`CodeToolError` block) and defers to the PIC page for the lowering rule. A reader landing on `tool-calls.md` cannot answer what each `execute()` disposition returns without chasing a cross-link into the PIC page. The split also fragments anchoring across two coverage tokens (`cka-13` on `tool-calls.md`, `cka-46` on `host-interfaces-core.md`) in two files.

## Issue introduction

**Verdict:** introduced-by-commit
**Introducing commits:** `11ef3b9` (2026-05-07) — *pi-loom spec: resolve "Tool failure modes beyond throw / isError: true unspecified"*
**History:** `git log -G "Outcome routing summary" --diff-filter=A` shows commit `11ef3b9` as the sole content-bearing introduction of the `*Outcome routing summary.*` paragraph; that commit added the paragraph to `spec_topics/pi-integration-contract.md` immediately after the envelope-lowering rules in §*Tool execution from loom code*. The same commit, in the same diff, added the `*Outcome enumeration (normative).*` block to `spec_topics/tool-calls.md` — listing the three off-`CodeToolError` outcomes and explicitly deferring to "the lowering rule in [Pi Integration Contract — Tool execution from loom code]". The placement split was therefore introduced in a single editorial act, not by drift. Subsequent commits only refined the prose: `2ebe288` (post-cancel `unhandledRejection` discard) and `bf85092` (`<underlying-error string truncated>` re-wording) edited the existing paragraph in place, and `f5e89f4` migrated `pi-integration-contract.md` to the `pi-integration-contract/host-interfaces-core.md` subdirectory layout without moving the paragraph between topical homes. No earlier revision placed any equivalent of the per-disposition routing summary in `tool-calls.md`; the file the finding flags as the natural home has never carried it.

## Solution approach

Move the per-disposition `execute()` outcome routing summary out of `host-interfaces-core.md`'s `*Outcome routing summary.*` paragraph into `tool-calls.md` under a new `id="outcome-routing"` anchor in the `**Failures.**` region alongside the `*Outcome enumeration (normative).*` block, so the by-`cause` author-observable routing is single-sourced where authors land. Leave the envelope-shape and lowering-mechanics prose on `host-interfaces-core.md`'s `id="tool-execution-from-loom-code"` section and add a forward-link there to the new anchor. Re-point `docs/plan_topics/coverage-matrix.md`'s `cka-46` by-cause-routing citation to the new `tool-calls.md` anchor while keeping `host-interfaces-core.md` cited for the envelope-walking mechanics, and reconcile its exclusion clause so the routing surface no longer splits across `cka-46` and `cka-13`. Update the `V14a` plan leaves' `**Spec.**` citations to match.

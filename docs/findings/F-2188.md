## Problem

`docs/spec_topics/discovery/package-and-settings.md` contains six "at the top of this file" self-references that no longer resolve to anything in that file. The content they point at — the extension-ownership framing paragraph, the non-recursive / byte-exact `*.loom` discovery rules, and the DISC-2 failure-modes table — lives in the sibling file `discovery-sources.md`, not at the top of `package-and-settings.md` (whose top is the `## Package discovery` heading and the "Roots scanned" list). A reader following any of these references finds none of the named content and must already know it lives in the sibling file to proceed.

## Issue introduction
**Verdict:** multi-commit-interaction
**Introducing commits:** f5e89f4 — docs: enforce 100KB hard / 30KB recommended size cap on spec set (2026-06-04, spec author); 1cb54c7 — pi-loom spec: resolve "Glob !/+/- precedence and matcher engine unspecified for pi.looms / loomPaths" (2026-06-06, spec author)
**History:** In the pre-split monolith `docs/spec_topics/discovery.md` the framing paragraph, the non-recursion / byte-exact rules, and the failure-modes table all sat at the top of the same file, so the five "at the top of this file" self-references were accurate (confirmed via `git show f5e89f4^:docs/spec_topics/discovery.md`). Commit f5e89f4 split that file into `discovery-sources.md` (which received those anchors) and `package-and-settings.md` (which kept DISC-5/DISC-6 and the settings section), carrying the five self-references over verbatim without repointing them and thereby turning all five stale. Commit 1cb54c7 then expanded DISC-5 and added a sixth instance (the byte-exact `*.loom` reference), replicating the now-stale pattern at a new site.

## Solution approach

Rewrite each of the six "at the top of this file" references in `package-and-settings.md` as an anchored cross-link into `discovery-sources.md`, the sibling that owns the targeted content. The framing-paragraph reference (`## Package discovery` intro) targets `#file-extension-namespace`; the two failure-modes-table references (DISC-6 and the settings trailing paragraph) target `#disc-2`; the three non-recursion / byte-exact references (DISC-5 ×2 and the settings Directory-entries bullet) target the "Discovery is non-recursive and matches only `*.loom`" paragraph in `discovery-sources.md`. That paragraph currently carries no stable anchor — add one before linking to it.

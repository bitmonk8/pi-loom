## Finding

`session-only-degraded-state.md` opens with one paragraph-level blockquote declaring that every obligation on the page is blocked until host-prerequisites clause (a) resolves to the `provably-differs` verdict. Below that line the page reads as ordinary normative spec — 13 lettered acceptance criteria, a state-independent tag-write rule, a predicate split, co-emission rules, an emergent fast-path, and three named residual gaps — none of which carry any visible "contingent on (a)" marker. An implementer who scrolls past the first blockquote (or arrives at the page via a deep link to e.g. `#session-only-reason-degraded-state`) sees only the body, has no in-line reminder that the obligations are gated, and may treat the criteria as immediately implementable.

The bigger gap is at the *consumer* sites in `drain-state-contract.md`. That page defines `LoomRegistry.markRuntimeDegraded` (the only writer that produces the `"degraded-needs-reload"` tag value), defines `readDrainState` arm (c) (the only reader that consumes it to front the degraded-arm slash note), and discusses the all-three-throw corner case and the `readDrainState`-throw re-entry case — all of which are mechanically reachable *only* on the session-only branch the gate blocks. None of these clauses carries a cross-reference to the implementation gate, so an implementer working through `V9m` (which is not itself blocked in `plan.md`) sees `markRuntimeDegraded` and arm (c) as normal first-class contract surfaces and wires them up, unaware that the only branch that exercises them is contingent.

The result is that the gate's authority is spec-shaped but not spec-spread: it gates `V9l` correctly via `plan.md`'s Blocked-obligations register, but the consumer surfaces in `drain-state-contract.md` (built by `V9m`, not `V9l`) provide no in-band signal that they are only reachable from blocked machinery.

## Solution Space

**Shape:** multiple

The finding decomposes into two independent obligations addressed at different files; each can be loop-fixed in isolation. Resolve them in the order below so the second obligation's edits land on a stable framing baseline.

### Option A — Propagate the gate to consumer sites in `drain-state-contract.md`

**Approach.** Add a brief implementation-gate cross-reference at each clause in `drain-state-contract.md` that defines or consumes a surface reachable only on the blocked branch — `LoomRegistry.markRuntimeDegraded`'s bullet, `LoomRegistry.readDrainState` arm (c), and the all-three-throw and `readDrainState`-throw re-entry corner cases that explicitly cross to `session-only-degraded-state.md`. The marker can be a single italicised sentence at the end of each clause: *"This surface is exercised only on the session-only degraded branch, which is gated per [Host prerequisites for the degraded-state branch, clause (a)](./host-prerequisites.md#degraded-state-host-prerequisites)."*

**Spec edits.**
- `drain-state-contract.md` — append the one-sentence gate cross-reference to (a) the `LoomRegistry.markRuntimeDegraded` bullet under *Methods*, (b) `readDrainState` arm (c) in the *Methods* paragraph, and (c) the *all-three-throw corner case* and *Read-failure fallback* re-entry sub-paragraphs of the *idempotent* clause where they cite the session-only edge-case bullet.

**Pros.** Localised, low-risk edits. Each gate reference points at the existing authoritative gate site rather than restating its semantics. Aligns the in-spec signal with the existing `plan.md` Blocked-obligations register so `V9m` authors see the gate when they read the surfaces they own.

**Cons.** Four small sentences sprinkled across a long file; future edits to the gate wording must sweep these sites (the cross-reference uses the anchor, so wording drift is bounded).

**Risks.** Minimal. The cross-reference uses the existing anchor `degraded-state-host-prerequisites`, which is stable. No normative semantics added.

### Option B — Demote the body of `session-only-degraded-state.md` below its own gate

**Approach.** Restructure the page so the body sits visually under the gate rather than beside it. Either (i) move all body sections under a single H2 such as `## Contingent on clause (a) — degraded-state contract` directly under the top-of-page blockquote, so deep-link arrivals at any anchor still see "Contingent on clause (a)" in the section breadcrumbs; or (ii) prefix the first sentence of each H2 with a one-clause reminder ("*Contingent on clause (a).* The page presents …"). Pick (i) unless an anchor that the cross-spec corpus depends on lives at H1 level.

**Spec edits.**
- `session-only-degraded-state.md` — wrap the existing body (everything below the top-of-page blockquote) under a single H2 marked contingent on clause (a), preserving every existing anchor; the `<a id="session-only-reason-degraded-state">` anchor at the top of the original first paragraph must remain valid and inbound-link-target-stable.

**Pros.** Restores the establish-before-solve invariant within the page itself: every deep-link arrival sees the gate's contingency as a structural part of the section it landed in. Costs one H2 rewrite; no normative content changes.

**Cons.** Carries a non-trivial inbound-link-target audit (the section header changes; any reference to `session-only-degraded-state.md` that pinned the body to its current section heading needs a sweep). Page diff is wider than Option A's diff.

**Risks.** Anchor-stability regression if the existing `<a id>` hooks are not preserved verbatim during the restructure. Run an inbound-reference sweep across `docs/` for every `session-only-degraded-state.md#…` link before merging.

### Recommendation

Land Option A first; it is the higher-value half of the fix (it closes the gap at the consumer-surface sites where the absence of the gate marker can actually mislead `V9m` implementers) and its diff is small enough that the inbound-reference sweep and the gate-wording-drift risk are negligible. Land Option B as a separate edit once Option A is in; the page-internal restructure is lower-stakes — its only consumers are readers already on the page — and benefits from the smaller diff of being separately reviewable. The implementer should preserve every existing `<a id>` hook on `session-only-degraded-state.md` during Option B and run a `grep -rn "session-only-degraded-state.md#" docs/` sweep before merging.

## Consequence

**Severity:** correctness

`V9m` ships ahead of `V9l` and implements `markRuntimeDegraded` plus `readDrainState` arm (c) per `drain-state-contract.md` as written, with no in-band signal that the only writer-callsite and the only reader-callsite that consume the `"degraded-needs-reload"` tag value live on the blocked branch. The clause-(a) resolution may rule the branch unreachable as written, in which case `V9m`'s implementation of those two surfaces becomes contradicted code that must be reworked alongside the spec rework `plan.md`'s pre-designed fallback names — discovered late rather than at design time. The same risk extends to readers arriving at `session-only-degraded-state.md` via deep link, who never see the top-of-page gate.

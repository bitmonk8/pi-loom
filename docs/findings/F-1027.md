## Problem

`session-only-degraded-state.md` declares its implementation gate once, in a top-of-page blockquote stating every obligation on the page is blocked until host-prerequisites clause (a) resolves to `provably-differs`. The page body below that line — 13 lettered acceptance criteria, the predicate split, co-emission rules, and named residual gaps — carries no in-line "contingent on (a)" marker, so a reader scrolling past the blockquote or arriving via a deep link to an in-page anchor sees only unconditional normative spec. The same gate is invisible at the consumer surfaces in `drain-state-contract.md`: `LoomRegistry.markRuntimeDegraded`, `readDrainState` arm (c), and the all-three-throw / `readDrainState`-throw re-entry cases are mechanically reachable only on the gated session-only branch, yet none cross-references the gate. An implementer building those surfaces (e.g. under a plan leaf that is not itself blocked) wires them as first-class contract surfaces, unaware the only branch that exercises them is contingent.

## Issue introduction

**Verdict:** multi-commit-interaction (the gating became page-wide but stopped at the page boundary)

**History:** Before `b8f004f` no implementation-gate marker existed anywhere; the host-prerequisites clause-(a) contradiction was recorded only on the originating page. `b8f004f` propagated the gate to the SM-level peers of the blocked obligations and to the canonical contradictions register, and added the top-of-page blockquote on `session-only-degraded-state.md` itself. `git log --all -S "Implementation gate" -- docs/spec_topics/pi-integration-contract/drain-state-contract.md` returns no commits: the gate-propagation pass did not visit `drain-state-contract.md`, and the body of `session-only-degraded-state.md` below the top-of-page gate was not restructured by either commit (the 13 acceptance criteria, predicate split, and other normative clauses pre-date both fixes). The current shape is therefore the *intersection* of two complete passes — F-0719 propagated the gate to SM-level peers and to the contradictions register, F-0718/F-0823 sharpened its phrasing — but neither pass propagated it to the writer/reader surfaces in `drain-state-contract.md` that actually realise the blocked branch, and neither pass demoted the body of `session-only-degraded-state.md` below its own gate.

## Solution approach

Add forward-links to the implementation gate at the `drain-state-contract.md` clauses that define or consume surfaces reachable only on the blocked session-only branch — the `LoomRegistry.markRuntimeDegraded` bullet, `readDrainState` arm (c), and the all-three-throw and `#read-failure-fallback` re-entry sub-paragraphs of the `#idempotent-handler-clause` — pointing at `host-prerequisites.md#degraded-state-host-prerequisites`. Restructure the body of `session-only-degraded-state.md` (everything below the top-of-page gate blockquote) so it sits structurally under its own gate, such that a deep-link arrival at any in-page anchor still surfaces the contingency.

## Solution constraints

- Preserve every existing `<a id="…">` hook in `session-only-degraded-state.md` verbatim through the restructure, including `<a id="session-only-reason-degraded-state">`; inbound link targets must remain valid.
- The added gate cross-references reference the existing gate only and introduce no new normative semantics.

## Problem

The `H4b` leaf heading reads `Response-programming surface (scripted-injection contract)`, naming one artefact two ways: the primary `response-programming surface` and the parenthetical `scripted-injection contract`. The identical dual-name string appears in both the H1 of `docs/plan_topics/H4b-response-programming-surface.md` and the `H4b` link text in `docs/plan.md` (Horizontal phases link list). The leaf body uses `response-programming surface` throughout and never the parenthetical token, and no equivalence statement ties `scripted-injection contract` back to it. A reader encountering the parenthetical may go looking for a separate contract artefact that does not exist.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** fd6169b
**History:** The `H4b` leaf was created in `fd6169b` (2026-06-12), the commit that split `H4a` into `H4a` + `H4b`. That commit added both the leaf file — with the H1 `# `H4b` — Response-programming surface (scripted-injection contract)` — and the `docs/plan.md` link line carrying the identical parenthetical. `git log -S 'Response-programming surface (scripted-injection contract)' -- docs/plan_topics/H4b-response-programming-surface.md docs/plan.md` reports `fd6169b` as the sole introducing commit. The dual-name heading has been present from the leaf's first revision; no later commit added an equivalence statement to the body.

## Solution approach

Remove the parenthetical `(scripted-injection contract)` from the `H4b` heading so `Response-programming surface` is the sole name, editing both the H1 of `docs/plan_topics/H4b-response-programming-surface.md` and the `H4b` link text in `docs/plan.md` (Horizontal phases link list) and leaving the link target unchanged. Alternatively, retain a single primary heading name and add an inline equivalence the first time the surface is named in the H4b body, stating that the response-programming surface is this leaf's scripted-injection contract. Either way the parenthetical must not be left standing alone in the heading with no body anchor.

## Problem

The `V11c` and `V11c-T` leaves both carry the bullet "The envelope is a relaxed copy (`maxLength:500` as a model budget, not a user cap); the bypass path skips the LLM call.", and `V11c`'s **Adds** repeats the compression "the relaxed copy with `maxLength:500`". The phrasing names neither the schema being copied nor the relaxation that makes the copy "relaxed", and it parenthetically binds `maxLength:500` to the relaxed copy as if that cap were the relaxation. Per `binder-bypass-and-envelope.md` §"Binder envelope schema" these are two unrelated things: the `ok` arm's `args` is a copy of the lowered `params` schema whose *only* relaxation is dropping defaulted fields from `required` (types unchanged, still satisfying the subset including `additionalProperties: false`), while `maxLength: 500` is a binder-model budget on the `message` string and each `candidates[i]`. A `V11c-T` author working from the leaf bullet alone could assert `maxLength:500` on the `args` params copy and omit the defaulted-fields-dropped-from-`required` assertion.

The defect dates to the leaves' first authoring, not to a later edit.

## Solution approach

In both `V11c` and `V11c-T`, rewrite the "relaxed copy" Tests bullet so the two concepts are stated separately: name the `ok` arm's `args` source schema (a copy of the lowered `params` schema) and its actual relaxation (defaulted fields removed from `required`, types unchanged, still satisfying the subset), and separately attach `maxLength: 500` to the `message`/`candidates[i]` strings as a binder-model budget rather than to the relaxed copy. Fix the `V11c` **Adds** parenthetical so it no longer implies `maxLength` is the relaxation. Ground both against `binder-bypass-and-envelope.md` §"Binder envelope schema".

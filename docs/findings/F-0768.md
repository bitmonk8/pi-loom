## Problem

Inside the *single-source-page* sub-clause of GOV-30 (`req-id-prefix-table-active-b.md`, anchor `gov-30`), the navigation-pointer-cross-link parenthetical declares that a heading-rename which silently breaks an aggregator-side cross-link "is a documentation defect, not a normative defect", and then in the same parenthetical writes the obligation with an uppercase `SHOULD` ("the PR that renames the heading SHOULD repoint each affected aggregator-side cross-link in the same commit"). The token matches no code-span or quotation fence and so reads as RFC-2119 — a reading GOV-22 reinforces by enumerating uppercase `SHOULD` as a normative-modal token marking a defining-obligation site. The paragraph thus simultaneously claims the obligation is non-normative and writes it in the corpus's own normative-modal vocabulary, leaving a maintainer unable to tell whether a missed repoint is a spec defect or housekeeping outside normative scope.

## Issue introduction

**Verdict:** present-since-introduction
**Introducing commits:** `0e12140` (`pi-loom spec: resolve "Non-goals are not consolidated into a single section"`, 2026-05-08)
**History:** The navigation-pointer-cross-link parenthetical was introduced wholesale in `0e12140`, which rewrote the predecessor rule (then `GOV-12`) on `docs/spec_topics/governance.md` to add the *single-source-page* shape. `git log -G "SHOULD repoint each affected aggregator"` and `git log -G "documentation defect, not a normative defect"` agree on `0e12140` as the first commit carrying either phrase; `git show 0e12140^:docs/spec_topics/governance.md` confirms neither phrase exists in the parent. The contradiction — uppercase RFC-2119 `SHOULD` adjacent to the explicit "not a normative defect" disclaimer — was therefore present in the very first version of this sub-clause. Commit `fe8aa5c` (2026-06-05) renumbered the host rule from GOV-12 to GOV-30 (and split out the integer-count invariant as GOV-31), and `f5e89f4` (2026-06-04) relocated the paragraph from `docs/spec_topics/governance.md` to `docs/spec_topics/governance/req-id-prefix-table-active-b.md` during the 100KB-cap split; neither commit altered the contradictory wording, both preserved it verbatim.

## Solution approach

Downcase the navigation-pointer occurrence of `SHOULD` to `should` in the GOV-30 single-source-page parenthetical, so the clause "the PR that renames the heading SHOULD repoint each affected aggregator-side cross-link" reads as advisory editorial guidance consistent with the adjacent "documentation defect, not a normative defect" framing.

## Solution constraints

- The genuinely-normative `SHOULD` later in the same paragraph ("reviewers SHOULD flag it on the PR that introduced the drift") must remain uppercase; touch only the navigation-pointer occurrence.

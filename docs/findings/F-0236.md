## Problem

`V2d` adds a single canonical number-rendering pure function shared by `V5f`
(canonical-hash recipe) and `V11h` (binder argument echo), and splits its
behaviour by static type: BNDR-4 renders `integer` as canonical base-10, BNDR-5
renders `number` as shortest round-tripping fixed-point. A JS `number` carries no
`integer`-vs-`number` distinction at runtime, and `V2d` disclaims any dependency
on the runtime value model or schema declarations, so the discriminator must be
an explicit input — but neither `V2d` nor either call site pins the renderer's
input contract or where each caller obtains the discriminator. With the contract
unpinned, the two call sites can thread the tag differently (or infer it from
value integrality rather than static type), diverging for large-magnitude or
non-safe-integer values. Because `V5f` hashes the binder-number-rendered
canonical form, this leaves the canonical-hash bytes — and the derived slug,
dedup, and collision detection — unspecified across conforming implementations.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `c6a664e` (2026-06-10, "pi-loom plan: build/update plan for spec.md + review")
**History:** The `V2d` renderer leaf was created in the genesis plan commit
`c6a664e` with the clause "no dependency on the runtime value model or schema
declarations" and the per-kind BNDR-4 / BNDR-5 split, but never pinned the
renderer's input contract or the discriminator's origin at each call site.
`cb8737a` ("resolve 'Systemic leaf over-bundling across the plan corpus'")
re-pointed the cited callers from `V5d`/`V11d` to `V5f`/`V11h` but did not add
the contract. The gap has existed unchanged since the leaf's inception.

## Solution approach

In `V2d`'s **Adds**, pin the renderer's input contract: the function receives the
numeric value together with an explicit integer/number kind discriminator and
selects BNDR-4 versus BNDR-5 from that discriminator rather than from runtime
integrality. State the discriminator's origin at each call site — `V11h` from the
bound value's static type via the value model (`V2c`), `V5f` from the JSON-Schema
`type` of the enclosing `const`/`enum` position. Record the caller-side
obligation in the `V5f` and `V11h` **Adds** so both sites cite the shared contract
instead of re-deriving the tag.

## Problem

The *Hoist obligation* clause under PIC-25 on `diagnostic-emission-isolation.md` mandates a pre-hoisted local string binding and forbids the catch arm from re-reading from the partially-constructed structured object or from `entry.shutdownReason`. Only the catch-arm emission's `details.event.reason` value is observable; the hoist is one conforming mechanism for producing it, not the requirement itself. Pinning the mechanism as a MUST needlessly forbids equivalent conforming implementations and grows the normative surface conformance fixtures must track.

## Solution approach

Rewrite PIC-25's normative requirement to state the observable property: the two-token catch-arm emission MUST carry the same `details.event.reason` value the structured payload would have carried — including the `"<unreadable>"` sentinel on the sub-step-2 stamp-throw residual-gap path — even when the serialiser throws mid-construction. Demote the "hoist into a local before the wrapped sequence / do not re-read from `entry.shutdownReason`" prose to a non-normative implementation note presenting the hoist as the recommended way to satisfy that property.

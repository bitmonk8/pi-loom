## Finding

`docs/spec_topics/hard-ceilings/ceiling-invariants-and-audit.md` carries two phrases that locate the CIO-N rules (or related ceiling-4 material) on this page when in fact every CIO-1 … CIO-6 anchor — and the *Per-boundary destination/surface table (ceiling #4)* the worked-consequences bullets cite — lives in the sibling file `ceilings-3-and-4.md`:

- The *Worked consequences* lede (line 5) says "each illustrates one or more `CIO-N` rules **above**"; the two depth-6 bullets directly under it (lines 7–8) additionally reference "the per-boundary table **above**" for ceiling #4.
- The *Five-site list co-edit obligation* paragraph (line 47) tells future spec-editors they "MUST update the five-site enumeration in CIO-3 **above (in this page)**." Neither CIO-3 nor the per-boundary table appears anywhere in this page; both are in `ceilings-3-and-4.md` (anchors `#cio-3`, `#ceiling-4-table`, section anchor `#ceiling-interaction-order`).

A reader following any of these deictics has to traverse to a different file. A spec-editor reading the co-edit MUST can interpret "in this page" two ways — that CIO-3 is in this page (and miss the cross-file edit), or that the cross-file edit is required (and ignore the misleading parenthetical) — and the rule's audit-time function (catching a future ceiling #4 row change that fails to co-edit CIO-3) silently relaxes.

## Solution Space

**Shape:** single

### Recommendation

Replace the deictics with explicit cross-file anchors into `ceilings-3-and-4.md`. Concretely, in `docs/spec_topics/hard-ceilings/ceiling-invariants-and-audit.md`:

- *Worked consequences* lede: change "each illustrates one or more `CIO-N` rules above" to point at the section anchor — "each illustrates one or more of the `CIO-N` rules in [Hard Runtime Ceilings — Interaction between ceilings](./ceilings-3-and-4.md#ceiling-interaction-order)."
- *Depth-6 tool-call args mid-loop* and *Depth-6 forced respond at `max_rounds`* bullets: change "the per-boundary table above" to "the [per-boundary destination/surface table (ceiling #4)](./ceilings-3-and-4.md#ceiling-4-table)".
- *Five-site list co-edit obligation* paragraph: replace "in CIO-3 above (in this page)" with "in [CIO-3](./ceilings-3-and-4.md#cio-3)". Drop the parenthetical "(in this page)" outright — it is not just imprecise but false.

Implementer notes:
- Use the same relative-link form (`./ceilings-3-and-4.md#<anchor>`) the file already uses elsewhere (e.g. the existing link in the *Five-site list* paragraph to `#ceiling-4-table`); no need to introduce index-page hops.
- The *Five-site list* paragraph's leading link "Ceiling #4's per-boundary table at [Per-boundary destination/surface table (ceiling #4)](./ceilings-3-and-4.md#ceiling-4-table) **above**" carries a third "above" that is similarly stale; treat it the same way (drop "above", keep the anchor) for consistency.
- Anchors `#cio-3`, `#ceiling-4-table`, and `#ceiling-interaction-order` already exist in `ceilings-3-and-4.md` (verified at lines 41, 17, 33 of that file); no anchor creation needed.
- Run `docs/_tools/check_links.py` after the edit to confirm no link breakage.

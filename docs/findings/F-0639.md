## Problem

`docs/spec_topics/hard-ceilings/ceiling-invariants-and-audit.md` carries deictic phrases that locate the CIO-N rules and the ceiling #4 per-boundary table on this page: the *Worked consequences* lede says the bullets illustrate `CIO-N` rules "above"; the two depth-6 bullets reference "the per-boundary table above"; and the *Five-site list co-edit obligation* paragraph tells spec-editors to update "CIO-3 above (in this page)". Every CIO-1 … CIO-6 anchor and the per-boundary table actually live in the sibling file `ceilings-3-and-4.md` (anchors `#cio-3`, `#ceiling-4-table`, `#ceiling-interaction-order`). A reader following any deictic traverses off-file, and the co-edit MUST's "(in this page)" parenthetical is false — a spec-editor can misread it as asserting CIO-3 is local and miss the cross-file edit, dulling the rule's audit-time function.

## Issue introduction

**Verdict:** present-since-split (single introducing commit)
**Introducing commits:** `f5e89f4f` (2026-06-04, *docs: enforce 100KB hard / 30KB recommended size cap on spec set*)
**History:** Before `f5e89f4f`, all of CIO-1 … CIO-6, the per-boundary table, the *Worked consequences* bullets, and the *Five-site list co-edit obligation* paragraph lived together in a single `hard-ceilings.md` file (introduced together in `1c7f5508`, *Hard ceilings block does load-bearing definitional work inside informative orientation*, 2026-05-07; the *Five-site* paragraph was added shortly after in `5d14719e`, *CIO-3 enumerates four AJV boundaries; ceiling #4 table has five*, 2026-05-07). In both wordings, "above" and "(in this page)" were locally correct. `f5e89f4f` split the page mechanically by anchor count to enforce the 100 KB / 30 KB size cap, moving CIO-1 … CIO-6 and the ceiling-4 table to `ceilings-3-and-4.md` while leaving the worked-consequences and audit/co-edit prose in `ceiling-invariants-and-audit.md`; the deictic words ("above", "in this page") were not rewritten as part of that split. A follow-up commit `fe8aa5c7` (2026-06-05) re-touched the *Five-site list co-edit obligation* line for an unrelated GOV-12→GOV-30 rename without revisiting the now-stale "(in this page)". No commit since has repaired any of the three deictics.

## Solution approach

Rewrite each stale deictic in `ceiling-invariants-and-audit.md` as an explicit cross-file link into `ceilings-3-and-4.md`: the *Worked consequences* lede to `#ceiling-interaction-order`, the two depth-6 bullets' per-boundary-table references to `#ceiling-4-table`, and the *Five-site list co-edit obligation* paragraph's CIO-3 reference to `#cio-3`. Drop the false "(in this page)" parenthetical, and drop the residual "above" the *Five-site* paragraph's leading `#ceiling-4-table` link carries. The cited anchors already exist in `ceilings-3-and-4.md`.

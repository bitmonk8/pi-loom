## Finding

The `## Code registry` heading in `code-registry-parse.md` does not describe the section's actual content. The section body contains (a) a single orientation paragraph explaining the four-page split and (b) the `### loom/parse/*` subsection table — i.e., it covers the parse namespace only, not the full registry. The heading "Code registry" implies the complete registry. The three sibling registry pages (`code-registry-load.md`, `code-registry-runtime.md`, `code-registry-host.md`) carry no equivalent `## Code registry` heading and proceed directly to their `###` subsection.

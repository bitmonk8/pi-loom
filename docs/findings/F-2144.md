## Problem

In the *Per-cause caller surfaces* index table in `error-model.md`, the **Hard-ceiling exhaustion** row's `invoke` parent cell forwards to the per-ceiling routing class at [`spec.md` — Hard ceilings](../overview-and-orientation.md#hard-runtime-ceilings), but the slash-caller and operator cells both read the bare word `same` with no stated antecedent. `same` admits two incompatible readings — "same as the `invoke parent` cell" (the aggregator it points to is keyed by ceiling and routing class, not by per-audience surface) and "same as the Panic row above" (correct only for ceiling #1, since ceilings #2–#4 surface as a recoverable `Err`, a load-time system note, and a boundary-dependent destination respectively). Hard-ceiling exhaustion is not a single routing class, so one bare `same` cell cannot stand for four divergent per-audience surfaces; the row points slash-caller and operator readers at no resolvable destination.

## Issue introduction
**Verdict:** single-commit-introduction
**Introducing commits:** `52219c4` (2026-05-06, "Address all 41 spec-review findings")
**History:** Before `52219c4` the per-cause caller-surface mapping lived as prose in the pre-split monolithic spec, and that prose explicitly stated that "Exhausting a runtime limit is *not* a single routing class," enumerating the divergent per-ceiling surfaces inline (`invoke`-chain depth = panic; `tool_loop` = `ToolLoopExhaustedError`; JSON-depth = `ValidationError`; binder cap = slash-binding-time failure). Commit `52219c4` refactored that prose into the tabular *Per-cause caller surfaces* index. The refactor preserved a per-ceiling pointer in the `invoke parent` cell of the Hard-ceiling exhaustion row but compressed the slash-caller and operator columns to a bare `same`, dropping the per-ceiling differentiation the prior prose carried. The two `same` cells have not been edited since: `39045df` and `f5e89f4` only repointed the link inside the `invoke parent` cell during the `spec.md` anchor rename and the `spec_topics/` split. The defect therefore entered the corpus at `52219c4` as a tabularisation regression.

## Solution approach

Rewrite the Hard-ceiling exhaustion row's slash-caller and operator cells in the *Per-cause caller surfaces* table of `error-model.md`, replacing each bare `same` with explicit per-ceiling pointers and stating that hard-ceiling exhaustion is not a single routing class. Route each audience to the per-ceiling surface owner: ceiling #1 to the **Runtime panics** / Panic-row surfaces, ceiling #2 to `ToolLoopExhaustedError` ([`queryerror-variants.md#err-19`](./queryerror-variants.md#err-19)), ceiling #3 to the binder load-time failure-mode template ([`determinism-cancellation-failure.md#failure-mode-templates-normative`](../binder/determinism-cancellation-failure.md#failure-mode-templates-normative)), and ceiling #4 to the per-boundary destination/surface table ([`ceilings-3-and-4.md#ceiling-4-table`](../hard-ceilings/ceilings-3-and-4.md#ceiling-4-table)).

## Problem

The two trailing paragraphs on `host-prerequisites.md` — *Extension ES-module declaration* (anchor `#extension-esm-type-module`) and *Extension entry point* — describe loom's own extension manifest (`package.json#type`, `extensions/index.ts`, `package.json#pi.extensions`), not a Pi-side host prerequisite. They belong on the parent integration page `pi-integration.md`, which already lists slash-command discovery and the file-watcher contract for the same audience, or on a dedicated extension-manifest topic page. Their placement on `host-prerequisites.md` mixes loom-owned manifest content into a page scoped to Pi-side prerequisites.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:**
- `4f32428` (2026-05-04) — "resolve 'Extension entry point path contradicts package.json#pi.extensions'" — first introduced the *Extension entry point* paragraph on what became `host-prerequisites.md`.
- `ce48e09` (2026-06-22) — "fix F-0961 — relocate ESM type:module obligation to host-prerequisites" — relocated the ES-module declaration into `host-prerequisites.md`, finalising the misplacement.

**History:** Split from F-1143 (shard-07 placement cluster) under the finding-atomicity invariant; this is the standalone extension-manifest relocation obligation.

## Solution approach

Move both the *Extension ES-module declaration* and *Extension entry point* paragraphs off `host-prerequisites.md` onto `pi-integration.md` or a dedicated extension-manifest topic page. Preserve the `#extension-esm-type-module` anchor id on the relocated paragraph so its inbound citations stay resolvable. Re-point the anaphors that reference "the **Extension entry point** above" — PIC-18 on `conversation-drive.md`, `binder-inference.md`, and the *System notes* / factory-bind paragraphs — to the new location.

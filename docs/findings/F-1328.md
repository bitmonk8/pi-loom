## Problem

GOV-16's *Per-page inline-label prefix table* in `docs/spec_topics/governance/stable-inline-labels.md` registers the `HC3` prefix against `hard-ceilings.md`, and the *Identifier classes used on this page* paragraph at the top of `docs/spec_topics/hard-ceilings.md` claims the `HC3-a` … `HC3-e` markers appear "below" and that the `HC3` prefix is "registered on this page". But `hard-ceilings.md` is a hub stub carrying no `HC3-*` anchor sites — every `HC3-a` through `HC3-e` anchor and every inbound `#hc3-*` cross-link resolves on `docs/spec_topics/hard-ceilings/ceilings-3-and-4.md`. Three GOV-16 obligations key off the registered `Page` cell — *GOV-1 dual-form layout* per-page progressive normalisation, the `## Retired inline labels` placement, and *Cross-link form* resolution — so the stale attribution misdirects each obligation to the empty parent.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** f5e89f4 — docs: enforce 100KB hard / 30KB recommended size cap on spec set (2026-06-04, Thomas Andersen)
**History:** GOV-16 and its per-page inline-label prefix table were coined in c4ad105 (2026-05-08), at which point `hard-ceilings.md` was a single file that physically contained the `HC3-a` … `HC3-e` anchor block, so the row `| hard-ceilings.md | HC3 |` was correct. f5e89f4 split that file under the 100KB cap, moving the `HC3-*` anchor block into the newly-created child `hard-ceilings/ceilings-3-and-4.md`, but in the same commit it carried forward both the GOV-16 table row pointing at `hard-ceilings.md` and the *Identifier classes used on this page* prose claiming "the `HC3` … prefix is registered on this page" — neither of which the mechanical split-and-repoint pass updated. The `SNK` row added by 27b2576 and the GOV-4→GOV-24 rename in 1d773d6 left the stale `HC3` and `NOCEIL` rows untouched.

## Solution approach

Rename the `HC3` row's `Page` cell in GOV-16's *Per-page inline-label prefix table* from `hard-ceilings.md` to `hard-ceilings/ceilings-3-and-4.md`, the file that owns the `HC3-*` anchor sites. Rewrite the `HC3-*` portion of the *Identifier classes used on this page* paragraph at the top of `docs/spec_topics/hard-ceilings.md` as a forward-reference to `hard-ceilings/ceilings-3-and-4.md` as the owning page, replacing the false "below" and "registered on this page" local claims.

## Solution constraints

- Out of scope: the `NOCEIL` row and the `NOCEIL-*` portion of the *Identifier classes used on this page* paragraph (owned by F-1329).
- Treat the `Page`-cell correction as a GOV-7 *Rename* in place; do not model it as a *Delete*-plus-*Add* (the `HC3` prefix is not retired).

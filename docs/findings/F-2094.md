## Problem

In `binder-model-and-context.md`, the re-entrancy / no-caching constraint for the binder-invocation path — the binder input record and resolved binder-model handle are constructed afresh on every binder call with no cached state — is stated only as prose inside the `> **loom 1.0 seam — automatic context escalation.**` blockquote after BNDR-10. It carries no MUST keyword and no REQ-ID, unlike every other binder obligation on the page (BNDR-7 through BNDR-11). The blockquote's future-work framing signals optional / rationale reading, so an implementer scanning for normative obligations can reasonably skip it and cache the binder input record or the resolved binder-model handle across calls. A cached implementation can then serve a stale `bind_context` snapshot on a second binder call within a loom's lifetime.

A later edit (before `528cef4`, 2026-06-03) reshaped that paragraph into the `> **loom 1.0 seam — automatic context escalation.**` blockquote, moving the constraint into visually-non-normative form; `528cef4` added the dual `<a id>` anchors. `0594eea` then appended the "this re-entrancy invariant is a settled loom 1.0 binder-path property … The normative pin above therefore commits loom 1.0 only to the binder-path re-entrancy property" sentences — an attempt to settle the normativity question that declared the property normative while leaving it inside the blockquote, without extracting it to a MUST/REQ-ID. The structural defect (a binding constraint with no normative marker) has therefore been present since the property was first authored; subsequent edits preserved rather than introduced it. The file's later relocation (`f5e89f4` split of `binder.md` into the topic subdirectory) is a pure move and carries the defect forward unchanged.

## Solution approach

Promote the re-entrancy / no-caching constraint out of the seam blockquote into a first-class numbered MUST obligation placed alongside BNDR-10 and BNDR-11; the next free identifier under the registered `BNDR` prefix is BNDR-12. Demote the remaining blockquote to a non-normative forward-pointer to the deferred automatic-context-escalation extension.

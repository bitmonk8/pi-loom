## Problem

The first `SLSH-2` Tests bullet in both `V12a-slash-dispatch.md` and `V12a-T-slash-dispatch.md` opens with a verifiable streaming-ordering assertion but then trails two non-assertion clauses onto the same bullet: a design-rationale fragment explaining why the test runs through the in-process Pi session double (whose `ctx.waitForIdle()`-vs-streaming ordering is fixed by the `H4a` session-double fidelity contract), and a governance caveat recording that the modelled real-Pi behaviour is an unpinned host presupposition audited per bump-checklist item (ak). Neither clause is a test assertion, and the bump-checklist already owns and links the governance obligation. Separately, `V12a` Adds describes the dispatch path with no indication that the `SLSH-2` ordering assertion has a fidelity ceiling against a double rather than a real host, so the design rationale is lodged in a test bullet while Adds is silent on it.

## Issue introduction

**Verdict:** introduced-by-specific-edit (over-correction while resolving an earlier finding)
**Introducing commits:** `85fc906` (2026-06-10, "pi-loom plan: resolve \"streamed-token-before-waitForIdle() ordering has no version-bump checklist item\"") for the governance caveat; `eeb0014` (2026-06-10, plan build/park commit) for the design-rationale fragment.
**History:** `git log -S 'unpinned host presupposition'` and `git log -S 'audited by editorial review per'` over both leaf files return exactly `85fc906`; `git show 85fc906 -- docs/plan_topics/V12a-slash-dispatch.md` confirms that commit appended the "The real-Pi behaviour this models … audited by editorial review per bump-checklist item (ak)" sentence to the `SLSH-2` bullet as the fix for the prior "streamed-token-before-waitForIdle() ordering has no version-bump checklist item" finding — i.e. that earlier fix closed the missing-checklist gap but embedded the governance caveat directly in the test bullet, creating this placement defect. `git log -S 'session-double fidelity contract'` over the same files returns `eeb0014`, the plan build/park commit that introduced the design-rationale fragment. The defect is therefore a two-commit accretion: the older rationale fragment plus the later governance over-correction, both landing in the Tests bullet.

## Solution approach

In both `V12a-slash-dispatch.md` and `V12a-T-slash-dispatch.md`, narrow the first `SLSH-2` Tests bullet to its ordering assertion only. Move the session-double fidelity-ceiling rationale into the `V12a` Adds field and mirror it into `V12a-T` Adds, where the design choice belongs. Strike the governance caveat from the Tests bullet; because `version-bump-step2.md#bump-checklist-user-visible-streaming-ordering` already owns and links the obligation, removal is sufficient — if a plan-side pointer is wanted, place it in `V12a` Ships when rather than in the assertion.

## Solution constraints

- Keep the two follow-up `SLSH-2` bullets (the `Err`-after-partial-text path and the mid-stream-cancellation path) unchanged.
- Apply the trim identically across `V12a` and `V12a-T` so the paired leaves stay in sync.

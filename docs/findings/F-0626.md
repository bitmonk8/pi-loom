## Finding

The Binder-model paragraph in `docs/spec_topics/binder/binder-model-and-context.md` opens:

> Resolved at loom-load time from a two-step chain: `bind_model:` frontmatter field, then the loom-extension setting `looms.binderModel` in `settings.json` (read per [Settings file reads](../discovery/package-and-settings.md#settings-file-reads); not a Pi-recognised setting).

The tail-attached parenthetical bundles two clauses with two different antecedents. The "read per [Settings file reads]" half legitimately describes `settings.json` (the file is read by the loom-extension settings discovery path). The "not a Pi-recognised setting" half is meant to qualify the key `looms.binderModel` — disclaiming that the key is owned by the loom extension, not the Pi host. As written, both halves grammatically attach to the nearest noun phrase (`settings.json`), so a reader can plausibly conclude the whole `settings.json` file is loom-specific, or that the file is itself somehow not Pi-recognised. The intended meaning — that the *key* is extension-owned — is the load-bearing claim (it justifies the rest of the resolution chain and the hot-reload story), so leaving it grammatically dangling is a real clarity defect, not a cosmetic one.

## Solution Space

**Shape:** single

### Recommendation

Split the parenthetical so each half sits next to its actual antecedent. Rewrite the opening sentence as:

> Resolved at loom-load time from a two-step chain: `bind_model:` frontmatter field, then the loom-extension setting `looms.binderModel` (not a Pi-recognised setting) in `settings.json` (read per [Settings file reads](../discovery/package-and-settings.md#settings-file-reads)).

This places the ownership disclaimer immediately after the key it qualifies and leaves the file-read pin attached to `settings.json` where it belongs. Edge cases for the implementer: preserve the existing link target and anchor verbatim; do not touch any other occurrence of the phrase elsewhere in the corpus (the only other live occurrence is this same sentence — no parallel rewording is needed).

## Consequence

**Severity:** cosmetic

A careful reader recovers the intended meaning from context (the surrounding paragraph and the linked Settings file reads section), so no implementer is meaningfully misled. The disclaimer is a provenance note about who owns the key, not a behavioural rule; misreading it does not change the load-time resolution algorithm or any diagnostic surface.

## Problem

In `docs/spec_topics/expressions.md` the `string` stdlib row `replace(from, to)` specifies all-occurrences substitution and its own parenthetical concedes the behaviour matches host `String.prototype.replaceAll`. JavaScript's `String.prototype.replace(str, str)` replaces only the first occurrence, which is the prior art a loom author will carry into the language. A loom author writing `s.replace("X", "Y")` expecting first-only substitution instead gets every occurrence replaced, and the mismatch produces wrong values rather than a parse-time or type-time diagnostic. The behaviour is well-pinned by the five reference vectors; the defect is the surface name, which carries the wrong host-language association.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `53f5831` (2026-05-03, "spec: rewrite query primitive, system prompts, and schema syntax")
**History:** The `string` stdlib surface was first enumerated in `53f5831`, which introduced the row `replace(from, to)` together with the "Replaces all occurrences" semantics in the same edit — the JS-name / non-JS-semantics pairing has been present since the stdlib's first appearance. The file was split out of `spec.md` into `spec_topics/expressions.md` in `fecb504` with no semantic change. Later edits refined the row's wording (`8984e3d` removed historical-rationale prose, `ccab317` added the `$`-sequence and empty-`from` clauses, `d292b46` added the reference-vector table, `3b4cd71` added the explicit "matching host `String.prototype.replaceAll`" parenthetical) but none touched the method name. `3b4cd71` is the commit that makes the divergence textually self-evident, but the defect predates it.

## Solution approach

Rename the `replace(from, to)` row in `docs/spec_topics/expressions.md` — together with its reference-vector table caption, the five vector expressions, and the following prose — to `replaceAll`, and drop the now-redundant `String.prototype.replaceAll` parenthetical. Update the `split` / `replace` cross-reference in `docs/spec_topics/hard-ceilings/ceiling-invariants-and-audit.md` to name `replaceAll`. The section heading anchor `#built-in-methods-and-properties` is stable, so internal cross-references survive the rename.

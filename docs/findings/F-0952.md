## Problem

`V11j` owns one half of a two-part cancellation contract — handling an abort that lands during the binder's in-flight provider call (initial attempt or budgeted retry) — while `V17c` owns the pre-call `binder-call` checkpoint abort. That division is a normative implementation boundary, yet it is stated only inside a mid-sentence parenthetical of `V11j`'s sole Tests bullet, duplicated verbatim into `V11j-T`. `V11j`'s **Adds** field describes the forwarding mechanism (`ctx.signal` → `options.signal`, the cancelled-binder note) but never states the `V17c` boundary, so an implementer reading **Adds** for the leaf's scope sees the mechanism but not the boundary that separates it from `V17c`.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** 66b3815 — pi-loom plan: resolve "V11f bundles binder retry budget with in-flight cancellation forwarding" (2026-06-16, Thomas Andersen)
**History:** Both `V11j-binder-call-cancellation.md` and `V11j-T-binder-call-cancellation.md` were created in 66b3815 (`git log --follow --diff-filter=A` reports that commit as the sole creation point). From inception the V11j/V17c scope demarcation lived in the Tests bullet and the Adds field never carried it. The later commit 5b95143 (2026-06-18) only retargeted the demarcation's owner reference from `V17a` to `V17c` and added `V17c` to `Deps` (`git show 5b95143` touches the Tests bullet and Deps line, not Adds); it preserved the placement rather than introducing it. The defect is therefore present at the leaf's inception.

## Solution approach

Add the `V11j`↔`V17c` boundary to the **Adds** field of `docs/plan_topics/V11j-binder-call-cancellation.md`: that `V11j` handles an abort landing during the binder's in-flight provider call (initial attempt and each budgeted retry), whereas the pre-call `binder-call` checkpoint abort is owned by `V17c`. Then reduce the Tests-bullet parenthetical in both `V11j` and `V11j-T` to its assertable content, carrying the existing PIC-10 / `cancellation.md` *Granularity* citations to whichever home retains the boundary statement.

## Solution constraints

- Out of scope: `V17c`'s own leaf files need no change, and the spec is read-only for this fix.
- Keep the `V11j` and `V11j-T` Tests bullets in agreement after trimming (paired-leaf convention, `conventions.md`).

## Problem

The active-set gating window (the `snapshot → setActiveTools → query → finally` restore bracket on the prompt path) is introduced and owned by `V9c`. Both `V9f` and `V9f-T` exercise that window — `V9f-T`'s `PIC-8` test drives the step-4 finally-restore throw and `PIC-19` drives the step-1/step-2 snapshot/swap-install throw — yet neither leaf names `V9c` in its `Deps.` (`V9f` declares `V9f-T`, `V9a`, `V5d`; `V9f-T` declares `V9a`, `V5d`). `V9f-T` therefore becomes pickable as soon as `V9a` and `V5d` complete, before `V9c` introduces the window as a production symbol. Picked in that window, its `PIC-8`/`PIC-19` tests reference a window step with no production target, so the suite reds for a compile/missing-symbol reason that fails the "fail red for the intended reason" gate and blocks honest tagging of `V9f-T-complete`.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** c6a664e (2026-06-10, "pi-loom plan: build/update plan for spec.md + review")
**History:** Both `V9f-tool-registration-lifetime.md` and `V9f-T-tool-registration-lifetime.md` were created in c6a664e already carrying the `PIC-8` step-4 restore-throw test against the active-set gating window and a `Deps.` line that omits `V9c` (`V9f-T`, `V9a`, `V5d` for V9f; `V9a`, `V5d` for V9f-T). The dependency-on-V9c-without-declaring-it defect is thus original to the leaf. Commit 8bd82be (2026-06-17, "resolve 'one window concept named three ways across V9c/V9c-T/V9f'") later made the V9c ownership explicit in V9f's `Adds.` prose ("defined in `V9c`") but did not introduce the missing edge — it pre-dates that clarification.

## Solution approach

Add `V9c` to the `Deps.` of `V9f-T` (the load-bearing edge — its `PIC-8`/`PIC-19` tests reference the V9c-owned window step, so its pickability must wait on `V9c-complete`). Also add `V9c` to the `Deps.` of `V9f` so the Deps DAG stays self-describing, since `V9f`'s own `Adds.` and tests cite the V9c window directly. Reference the existing leaf by its exact ID `V9c`.

## Solution constraints

- Out of scope: `V9c` itself is read-only for this fix — the window definition already exists there.
- The spec is read-only for this fix.

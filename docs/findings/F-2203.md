## Problem

QRY-14's `max_rounds: 0` boundary case carries a byte-level requirement: the runtime right-trims any trailing U+000A bytes the rendered prompt text carries before inserting the single-U+000A separator, keeping the prompt-to-instruction gap exactly one line feed even when the template leaves extra trailing blank lines. The requirement is mechanical and fully specified, but no test vector exercises it. The only normative vector in this topic page — the depth-6 worked example at `worked-example-depth-6-forced-respond` — uses the prompt text `Probe.`, which carries no trailing newlines and never reaches the right-trim branch. An implementation that omits the right-trim step emits a second leaked U+000A yet passes every stated test, leaving the byte divergence undetected until a downstream provider-transcript comparison surfaces it.

## Issue introduction
**Verdict:** single-commit
**Introducing commits:** `ccde46d` ("pi-loom spec: resolve 'max_rounds: 0 user-turn separator disagrees with Follow-up turn templates'", 2026-06-03)
**History:** The QRY-14 boundary case originally separated the prompt body from the instruction wording by *two* U+000A line feeds (a deliberate blank-line paragraph break). `ccde46d` tightened the separator to a single U+000A and, in the same edit, added the right-trim sentence to guarantee the prompt-to-instruction gap is exactly one line feed even when the rendered prompt ends in extra blank lines. The byte-level right-trim requirement and its untested status were therefore born together in `ccde46d`; no accompanying normative test vector was added. Subsequent commits (`f5e89f4` split the section into `query-tool-loop.md` under the size-cap reorg; `716a13b` coined the `QRY-*` REQ-IDs) relocated and re-labelled the text without altering the requirement or adding a vector. Later appearances of the token are spec-review store/triage commits, not corpus edits.

## Solution approach

Add a normative test vector to `query-tool-loop.md`, modelled on the existing depth-6 worked example, that exercises the right-trim branch: a typed query with `tool_loop.max_rounds: 0` whose rendered prompt text (after newline-trim and dedent) ends in two trailing U+000A bytes. Pin the expected user-message body so the assertion verifies exactly one U+000A separates the last non-newline prompt byte from the first byte of the forced-respond-turn instruction wording. Cover the two-trailing-newline case and state the gap is one line feed across the zero-, one-, and two-trailing-newline inputs.

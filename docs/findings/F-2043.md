## Finding

The entry contains the sentence "The rename does NOT affect any code identifiers, file extensions, diagnostic codes, frontmatter field names, or wire formats — it is purely terminology." This is a migration-scope boundary claim: it defines what a terminology rename does and does not touch across the implementation surface. Migration-scope statements of this kind belong in governance or migration-guide notes, not in a term definition. Embedded here, it implies the glossary is also a change-management authority, a further undeclared role (see Finding 1).

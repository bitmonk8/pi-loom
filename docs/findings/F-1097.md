## Problem

`H4c`'s **Ships when** criterion ("`npm test` exercises the modeled-behaviour surface … including the determinism gate and the per-category functional-effect assertions (f)–(g)") folds three independently-failable sub-criteria — the determinism gate, assertion (f) (a scripted invoke-child completion surfacing as a completed-invoke-child outcome), and assertion (g) (a scripted subagent-mode callee surfacing as a subagent-loom outcome) — into one compound pass/fail. The `Adds` and `Tests` bodies enumerate (f) and (g) separately, but the Ships-when line names only the bundled range, so a reviewer reasoning from it records "H4c passed/failed" rather than which sub-criterion held. This corpus carries no plan index, so the missing addressability can only be supplied within the leaf.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** 1063255 — pi-loom spec-plan: fix F-0476 — split H4b (a)-(e) from (f)/(g) into new H4c leaf, retarget consumers (2026-06-20, Thomas Andersen)
**History:** `H4c-modeled-behaviour-surface.md`'s only commit is `1063255`, which created the leaf by splitting `H4b`'s (f)/(g) categories out; the bundled "per-category functional-effect assertions (f)–(g)" Ships-when wording was authored in that first commit. The pattern is inherited from `H4b`, where the equivalent bundled form ("assertions (a)–(g)", then "(a)–(e)" after the split) has been present since `fd6169b` (2026-06-12, the commit that first split `H4b` out of `H4a`). The defect is thus present from the H4c leaf's inception, carried over from the pre-split H4b design.

## Solution approach

Rework `H4c`'s **Ships when** criterion so its three bundled sub-criteria — the determinism gate, assertion (f), and assertion (g) — are each individually identifiable, so a failing run is attributable to a named sub-criterion rather than to the leaf as a whole. Carry the (f)/(g) and determinism distinctions the `Tests` bullets already draw into the Ships-when condition, keeping the addressability inside the leaf since the corpus has no plan index to register per-assertion sub-IDs.

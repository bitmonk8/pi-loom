## Finding

`V4c` (Terminal outcomes and partial-append) explicitly defers the *live-surface* confirmation of `ERR-8` to `V9i`: `V4c-T`'s `ERR-12` bullet exercises ERR-8 inside a subagent loom "via the `H4a` harness modelling a subagent-mode callee — **not** the live `V9i` surface," and `coverage-matrix.md`'s `ERR-8 … ERR-12` row names `V9i` as the "live subagent-surface ERR-8 co-witness." `V9i` is therefore the authoritative closing leaf for the live-surface arm of ERR-8.

That obligation appears only in the `V9i` **implementation** leaf's Tests list: "`ERR-8` (delegated live-carrier witness for `V4c`'s ERR-8/ERR-12 deferral): a mid-stream cancellation driven inside the real subagent `AgentSession` does not mutate the subagent session's committed turns." The paired tests leaf `V9i-T` carries no such bullet — its Tests enumerate `PIC-9`, `PIC-22`, the isolated-session check, the `return.md` final-value contract, and the two `subagent-*` codes, but not ERR-8. `V9i`'s `Ships when` likewise names only `dispose()`-in-`finally` idempotency.

Under the per-phase TDD ritual (`conventions.md` §Per-phase TDD ritual), the failing assertion for every REQ-ID a feature introduces is authored in the `<id>-T` tests task and must "fail red for the intended reason" before `<id>-T-complete` is tagged; the implementation loop then turns those red tests green. Because the delegated ERR-8 witness is absent from `V9i-T` and unnamed in `V9i`'s `Ships when`, an implementer following the ritual never authors the live-surface assertion red and can tag `V9i-complete` with the observable gate satisfied — while the cancellation-non-mutation behaviour on the real subagent `AgentSession` is never verified anywhere in the corpus.

## Solution Space

**Shape:** single

### Recommendation

Give the delegated ERR-8 live-surface witness a red-first home and an observable gate, both in the `V9i` pairing:

- In `docs/plan_topics/V9i-T-subagent-isolation.md`, add a `Tests` bullet mirroring the impl leaf's ERR-8 obligation, cited inline as `ERR-8`: a mid-stream cancellation driven inside the real subagent `AgentSession` does not mutate the subagent session's committed turns (the live-surface confirmation `V4c` defers from the `H4a` double). If that test references `V4c`-owned symbols to compile, add `V4c` to `V9i-T`'s `Deps` so the tests task matches the impl leaf's existing `V4c` edge and stays pickable in the correct order.
- In `docs/plan_topics/V9i-subagent-isolation.md`, extend `Ships when` so the ERR-8 live-surface non-mutation outcome is an observable criterion alongside the existing `dispose()`-in-`finally` idempotency assertion — e.g. assert that the live subagent session's committed turns are unmodified under mid-stream cancellation.

Edge case: if the `V9i` lifecycle/parallel-spawn split (see Related Findings) is adopted, the delegated ERR-8 witness moves to the new parallel-spawn leaf; land that split first so the red-first bullet and the `Ships when` clause are authored on the final owning leaf rather than relocated afterward.

## Problem

The introductory preamble of `docs/spec_topics/future-considerations.md` closes with a cross-link whose text names the section *Provider compatibility for typed queries* but whose target is `./pi-integration-contract.md`, the PIC hub page. The hub no longer owns that section — it now lives in `docs/spec_topics/pi-integration-contract/conversation-drive.md` as the `**Provider compatibility for typed queries.**` paragraph, which carries no dedicated anchor. A reader following the link lands on the section-less hub and must hunt through the sub-page tree for the referenced material.

## Issue introduction
**Verdict:** multi-commit-interaction
**Introducing commits:** `dd1965e` (link authored, then-correct); `f5e89f4` (page split that falsified it)
**History:** `dd1965e` (2026-05-06, "pi-loom spec: resolve \"V1 design constraints filed under the deferrals inventory\"") introduced the future-considerations preamble paragraph carrying the cross-link `[Pi Integration Contract — Provider compatibility for typed queries](./pi-integration-contract.md)`. At that point the named section lived on the monolithic `pi-integration-contract.md`, so the link resolved correctly. `f5e89f4` (2026-06-04, "docs: enforce 100KB hard / 30KB recommended size cap on spec set") split the PIC hub into sub-pages — moving the *Provider compatibility for typed queries* section out of the hub and into `pi-integration-contract/conversation-drive.md` — without repointing the inbound preamble link, which still targeted the now-section-less hub. The defect entered the corpus at `f5e89f4` as a stale inbound reference surviving the page split, not as an originally-wrong link.

## Solution approach

Repoint the `future-considerations.md` preamble cross-link from `./pi-integration-contract.md` to `./pi-integration-contract/conversation-drive.md`. Add an HTML anchor on the `**Provider compatibility for typed queries.**` paragraph in `conversation-drive.md` and append the matching fragment to the link so it deep-links to that paragraph rather than dropping the reader at the sub-page top.

- Out of scope: the matching cross-link in `future-considerations/surface-extensions.md`, owned by F-2244.

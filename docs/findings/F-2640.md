## Problem

Several non-normative notes explicitly labelled "not an observable conformance point" sit embedded inside normative prose. Specifically: the `ToolDefinition` / `defineTool` construction-helper note in `extension-bootstrap-and-per-loom.md`, the single-provider-constant note in `conversation-drive.md`, and the *Non-normative editorial convention* / *(cont.)* paragraphs in `drain-state-contract.md`. These maintainability notes carry no observable conformance content yet clutter the normative contract, obscuring what an implementer must actually satisfy.

## Solution approach

Delete the non-normative maintainability notes from the spec body: the `ToolDefinition` / `defineTool` construction note in `extension-bootstrap-and-per-loom.md`, the single-provider-constant note in `conversation-drive.md`, and the *Non-normative editorial convention* / *(cont.)* paragraphs in `drain-state-contract.md`. Where a note carries a genuine sweep obligation an editor must honour — such as the four-method-rename sweep in `drain-state-contract.md` — demote it to a contributor-facing Implementation Notes entry rather than deleting it outright.

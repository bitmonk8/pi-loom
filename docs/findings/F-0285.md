## Problem

The `BNDR-9` test bullet in `V11b` and `V11b-T` enumerates the forbidden transcript-unsafe token as the code span `` `": "` `` — four characters: quote, colon, space, quote — but its inline annotation `(U+003A U+0020)` and the surrounding "two-byte sequence" prose describe exactly two code points (`:` and space). The code span and its annotation contradict each other. The authoritative spec (`BNDR-9` in `docs/spec_topics/binder/binder-model-and-context.md` and the `loom/runtime/custom-type-unsafe` row in `docs/spec_topics/diagnostics/code-registry-runtime.md`) uses the un-quoted `: ` separator, so the quotation marks in the plan leaf are spurious. A test author reading the bullet literally would assert rejection of the 4-character quote-wrapped token rather than the 2-byte `: ` separator, producing the opposite acceptance/rejection result for inputs containing `: ` but no surrounding quotes.

## Issue introduction

**Verdict:** single-commit
**Introducing commit:** `fe694dd` (2026-06-10) — "pi-loom plan: resolve \"BNDR-9 forbidden-character enumeration is ambiguous about the colon-space token\""
**History:** The plan corpus is git-tracked. Before `fe694dd`, both `V11b` and `V11b-T` enumerated the forbidden set as `` (containing `\n`/`\r`/`]`/`: `) `` — the colon-space code span `` `: ` `` carried no surrounding quotes (`git show fe694dd^:docs/plan_topics/V11b-bind-context-transcript.md`). The edit that aimed to *disambiguate* that token is the edit that introduced the contradiction: `fe694dd` rewrote the bullet to "the two-byte sequence `": "` (U+003A U+0020)", adding the `(U+003A U+0020)` annotation and "two-byte sequence" wording (correct) while simultaneously wrapping the code span in quotation marks (incorrect), turning a 2-byte span into a 4-character one. The same one-line substitution landed in `V11b` and `V11b-T` in that single commit (`git show fe694dd --stat`). No later commit touched the bullet; the contradiction has been present in every revision since `fe694dd`.

## Solution approach

In the `BNDR-9` bullet of both `V11b-bind-context-transcript.md` and `V11b-T-bind-context-transcript.md`, replace the code span `` `": "` `` with `` `: ` `` so the forbidden token matches its own `(U+003A U+0020)` annotation, the "two-byte sequence" prose, and the spec's BNDR-9 / `loom/runtime/custom-type-unsafe` wording. Apply the identical substitution to both members of the paired leaf.

## Solution constraints

- The spec is read-only for this fix — it already carries the correct un-quoted `: ` form.
- `V11b` and `V11b-T` are a paired leaf; the corrected bullet must be identical in both.

## Problem

`H4a`'s **Adds** defines an "in-memory fixture-supply mechanism" that supplies fixture `.loom` content to the extension in memory, with the invariant that harness-driven discovery reads it "rather than the real filesystem — no ambient `src/**` filesystem read, no `FileSystem` seam (`V8b`) dependency, keeping the `H3a` ambient-access scan clean." It never pins the architectural injection point: what entry the factory exposes in harness mode and the channel by which discovery is told to read the in-memory content instead of walking the filesystem. The only production route through which discovery reads files is the `FileSystem` seam — `V10a` reads exclusively through it and the `H3a` seam set contains no discovery-source seam — so the mechanism asserts fixture content reaches discovery while bypassing the only seam discovery reads through, with no stated alternative. `M`, `M-T` (both `Deps: H4a`), and `H7a` all consume this mechanism with no defined loading route, so implementers diverge: an in-memory `FileSystem`-seam fake breaks the leaf's own invariant, while a harness-only discovery override diverges from the production seam path `M` is meant to validate.

That commit resolved an earlier finding that `M`'s happy-path discovery had no defined source under the ambient-access ban; it introduced the "in-memory fixture-supply mechanism" wording into `H4a` and the matching consumer notes in `M`/`M-T`, asserting the mechanism bypasses both ambient reads and the `FileSystem` seam, but never specified the injection point by which harness content reaches discovery. The present gap was introduced together with the mechanism in that single commit.

## Solution approach

In `H4a`'s **Adds**, name the injection affordance the factory exposes in harness mode — a harness-supplied discovery source the factory accepts, distinct from the `H3a` seam set — and state that harness-driven discovery reads its fixture content from this named affordance rather than the filesystem. In `H7a`'s **Adds**, state that the integrated-run multi-feature fixture reaches the pipeline through this same `H4a` affordance. `M` and `M-T` can then cite the named affordance in their existing happy-path discovery text.

## Solution constraints

- The injection affordance must not be an in-memory implementation of the `FileSystem` seam (`V8b`); `H4a`'s "no `FileSystem` seam dependency / clean `H3a` ambient-access scan" invariant must remain intact.
- The spec is read-only for this fix — author the affordance entirely in the plan leaves.

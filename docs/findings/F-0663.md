## Finding

GOV-17's normative cross-reference definition turns on a single external citation: "a CommonMark link (per the CommonMark Spec §6.3 Links; subsequent uses of 'Markdown link' in this rule denote a CommonMark link)". The CommonMark Spec is a versioned, living document (0.29, 0.30, 0.31, 0.31.2, …); its section numbering and the edge cases of its link grammar (e.g. autolinks vs. inline links, angle-bracket destinations, reference-style normalization, fragment handling) have shifted between revisions. A bare "§6.3 Links" cite does not identify *which* revision's link grammar GOV-17 normatively borrows.

Two maintainers can therefore audit the same Markdown construct against different CommonMark revisions, classify it differently as "Markdown link" / "not a Markdown link", and reach different verdicts on GOV-17's MUST-NOT (cross-references into a dependent). Because the whole `[text](path)` / `[text]: path` recognition machinery — and by extension the dependent check, the operational `grep -nE` aid, and the resolution-relative-to-the-linking-document rule — is built on this citation, every downstream invariant in GOV-17 inherits the same ambiguity.

Note that GOV-17 *does* version-pin its other external citations (RFC 3986 §3.3/§3.4/§3.5), which are by construction immutable; CommonMark is the sole unpinned dependency.

## Solution Space

**Shape:** single

### Recommendation

Replace the bare "CommonMark Spec §6.3 Links" with a version-pinned citation. Preferred form: "per the CommonMark Spec, version 0.31.2 (2024-01-28), §6.3 Links", with the spec URL captured as a permalink to that revision (CommonMark publishes per-version permalinks at `https://spec.commonmark.org/0.31.2/`). Apply the same pin to the parenthetical "subsequent uses of 'Markdown link' in this rule denote a CommonMark link" — the term gets its definition from the same revision.

Edge cases the editor should watch:

- Confirm "§6.3 Links" is the correct section number in the chosen revision (0.31.2 currently uses §6.3 for Links under the Inlines chapter; verify before committing). If a future revision renumbers, the pin protects against silent drift.
- GOV-17's other external cite (RFC 3986 §3.3/§3.4/§3.5) is already stable by virtue of RFC immutability and needs no change.
- Do not introduce a *floating* "or later" qualifier ("CommonMark 0.31.2 or later"); that reintroduces the same ambiguity the pin is meant to remove. If the spec ever wants to track a newer CommonMark, bump the pin explicitly.
- The pin is editorial and self-binding under GOV-18 arm (b); it requires no implementation work outside the spec corpus.

## Consequence

**Severity:** advisory

Two reviewers can in principle reach different GOV-17 verdicts on the same commit if they consult different CommonMark revisions; in practice §6.3 "Links" has been structurally stable across recent revisions, so divergence is unlikely in normal operation. The cost of leaving it unpinned is auditability — the spec cannot point at a single byte-stable artefact for its link grammar.

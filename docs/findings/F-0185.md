## Finding

The `## Blocked obligations` section of `docs/plan.md` carries a single entry for `V9l` (session-only degraded-state branch). Its opening clause is already normative and self-contained: `V9l` is blocked on the host-prerequisites clause (a) **open contradiction** against Pi's documented teardown-and-rebind extension lifecycle at the loom 1.0 Pi-SDK pin, with the deciding resolution owned by version-bump editorial-review checklist item (a) and tracked in plan terms by `V18c`'s **Deps** edge.

Immediately after that clause the entry inserts a parenthetical explaining the technical mechanism of the contradiction: "(Pi reloads-and-rebinds the extension for the swapped-in session, so the drained `LoomRegistry` the branch depends on does not survive)". This is design-rationale — it explains *why* the cited lifecycle and the branch contradict each other. It adds no constraint on the implementer beyond what the normative clause and the cited spec anchors already carry: the block state, the gating Deps edge, and the pre-designed fallback are all stated independently of this explanation. The detailed mechanism lives properly in the cited spec topics (`host-prerequisites.md`, `version-bump-step2.md`), and duplicating its substance inline gives a second surface that can drift from the spec's account of the contradiction.

## Issue introduction

A later commit (9c42f2d, "V9l fallback path belongs in §Blocked obligations") reworded only the fallback sentence of the same bullet and left the parenthetical untouched, so the cruft has been present since the entry's inception.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan.md` §Blocked obligations, in the `V9l` bullet, strike the parenthetical "(Pi reloads-and-rebinds the extension for the swapped-in session, so the drained `LoomRegistry` the branch depends on does not survive)" so the sentence ends at "…at the loom 1.0 Pi-SDK pin." The surrounding normative clause — the block state, the `host-prerequisites.md#degraded-state-host-prerequisites` anchor, the deciding-resolution owner (`version-bump-step2.md#bump-checklist-instance-survival` / `V18c`'s Deps edge), and the pre-designed fallback — is retained unchanged; the spec anchors remain the home for the contradiction's mechanism. Do not edit the spec for this fix.

## Consequence

**Severity:** cosmetic

The parenthetical is non-actionable design narrative; leaving it costs nothing functional but adds a second, un-gated surface restating the spec's contradiction mechanism, which can silently diverge from `host-prerequisites.md` / `version-bump-step2.md` if either is later revised. No implementer's decision changes either way.

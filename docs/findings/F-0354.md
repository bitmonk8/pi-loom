## Problem

`V4e` packages all eight closed pre-evaluation failure causes (`ERR-1`…`ERR-7` plus `ERR-16`) under a single `Ships when` gate. Seven of these (`ERR-1`…`ERR-6`, `ERR-16`) are self-contained load-time routing assertions, while `ERR-7` is a different complexity class — a watcher-time reload-integration scenario injected through `V9b`'s `ReloadFailureInjector.injectReloadFailure` seam that must exercise `V9b`'s registry-swap and re-parse arms together with `V10d`'s settings-re-merge arm, and is the sole reason `V4e` carries `Deps` on `V9b` and `V10d`. Because one gate covers all eight causes, progress on the seven routing checks is unobservable until the `ERR-7` reload-injection harness is also stood up.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commit:** `c6a664e` (2026-06-10, "pi-loom plan: build/update plan for spec.md + review")
**History:** `V4e` has bundled the full pre-evaluation failure set (`ERR-1`…`ERR-7` + `ERR-16`) under the "all eight pre-eval causes" `Ships when` since the leaf was first authored in `c6a664e`. Later commits (`49e3837`, `66acde6`, `b2d5a1f`, `d64dce5`, `fdce2a2`, all 2026-06-11) did not introduce the bundling; they progressively deepened `ERR-7`'s integration burden — pinning the `ReloadFailureInjector` seam, its single owner, and the registry-swap / re-parse / settings-re-merge arms — which sharpened the heterogeneity between `ERR-7` and the seven routing checks but left the single-leaf structure unchanged.

## Solution approach

Split the `V4e` / `V4e-T` leaf-pair, separating the seven self-contained routing checks (`ERR-1`…`ERR-6`, `ERR-16`) from the `ERR-7` watcher-time reload-integration scenario into its own implementation/tests leaf-pair. Retain `V4e`'s existing ID for the routing leaf and rescope its `Tests`, `Deps`, and `Ships when` to the causes it still owns (the `V9b`/`V10d` deps move to the new leaf); allocate the new leaf ID per the plan's `H/M/V` leaf-ID scheme. Propagate the split through `coverage-matrix.md`'s `ERR-1 … ERR-7 | V4e` row, `H5b`'s `Deps`, and a `docs/plan.md` §V4 navigation bullet for the new leaf.

## Solution constraints

- Keep `ERR-16`'s routing assertion and the `CIO-1` matrix membership on the routing (`V4e`) leaf, where the slash-load consult lives; the `CIO-1` row annotation is owned by F-0319 — coordinate, do not pre-empt it.
- The new `ERR-7` leaf ID must be reachable from `H5b`'s `Deps` and backtick-delimited in its `coverage-matrix.md` cell, else `H5d`'s closing-gate check silently drops it from the gate's set.

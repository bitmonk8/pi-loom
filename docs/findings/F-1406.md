## Problem

`H5a`'s **Adds** makes the leaf the owner of the closing gate's warn-only live-corpus mode: a mode that runs the live-corpus reconciliation surfaces but, instead of throwing or setting a non-zero `npm test` exit status, returns an in-process findings collection — one structured entry per gap, each tagged with its gap kind — so the build stays green. None of `H5a`'s four `Tests.` bullets and no clause of its `Ships when` exercises that return path; all four bullets drive the hard-fail seeded-fixture path (gate fails on a seeded violation, passes on the no-violation fixture). The warn-only contract is asserted only downstream at `H5b`, whose `Deps.` span essentially the whole coverage-producing plan. A warn-only mode implemented wrong at `H5a` — throwing instead of returning, omitting the gap-kind tag, or miscounting entries — passes every `H5a` gate and ships green, deferring detection to the maximally-deep release-gate canary immediately before `H6a`'s hard flip.

## Issue introduction

**Verdict:** present-since-inception (of the warn-only mode)
**Introducing commit:** `ea6b1da` — "pi-loom plan: resolve \"Live-corpus gate activation has no documented rollback\"" (2026-06-11)
**History:** `git log -G "warn-only" --follow` on `docs/plan_topics/H5a-closing-gate-automation.md` shows the warn-only live-corpus mode first entered `H5a`'s **Adds** in `ea6b1da`; that commit added the mode to the Adds prose but added no `Tests.` bullet for it (the Tests block retained only the hard-fail seeded-fixture bullets). Later commits touching the warn-only prose — `7b88876` (F-0922, refined the return contract to an "in-process findings collection the gate machinery returns") and `cd27b78` (F-1241, added the `H5f` per-facet surface to the enumeration) — strengthened the Adds contract but still added no asserting bullet to `H5a`. The verification was instead authored into `H5b`. The in-leaf test gap has therefore existed continuously since the mode was introduced.

## Solution approach

Add a `Tests.` bullet to `docs/plan_topics/H5a-closing-gate-automation.md` that exercises the warn-only live-corpus mode over the seeded violation fixtures this leaf already owns (the seeded spec REQ-ID with no coverage-matrix row, and the seeded coverage-matrix mapping whose numbered REQ-ID has no citing test — the two gap kinds available at this leaf's dependency level), asserting that the mode returns an in-process findings collection with one tagged entry per seeded gap while `npm test` stays green (no throw, zero exit). Extend `H5a`'s `Ships when` to name this warn-only green-return outcome alongside the existing hard-fail seeded-fixture outcomes.

## Solution constraints

- Out of scope: `H5b`'s own warn-only assertion remains the live-surface verifier — do not modify `H5b`, and do not exercise the `H5e`/`H5f` gap kinds, which arrive with those leaves.
- The spec is read-only for this fix.

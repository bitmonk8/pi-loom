## Problem

The H5a closing gate's failure-mode enumeration (its `Adds.`) and the convention it cites — `conventions.md` *REQ-ID discipline* — both list "a retired/live ID clash" / "a retired-ID clashing with a live-ID" as a CI failure, but neither location defines what the clash is. Every sibling failure mode in that enumeration is pinned to a precise predicate the gate can evaluate (the *per-prefix numbering hole*, the *executable spec REQ-ID* set, the *retired interior ID* carve-out), leaving the clash arm the one undefined arm with at least three incompatible readings. The defect is compounded because H5a's `Tests.` and `Ships when.` exercise no clash fixture, so the arm is named in the gate's contract but is neither defined nor red-able. An implementer therefore has no spec to build the arm against and could build divergent behaviour or omit the arm entirely.

A current grep of `conventions.md` and `H5a-closing-gate-automation.md` confirms no definition for the clash predicate exists, while the sibling per-prefix-numbering-hole and retired-interior-ID predicates are both defined inline. The defect is original to both passages rather than a regression introduced by a later edit, so the durable fix is to define the term at its canonical home (the convention), not to revert either commit.

## Solution approach

Define "retired/live ID clash" at its canonical home — `conventions.md` *REQ-ID discipline*, adjacent to the existing undefined phrase "a retired-ID clashing with a live-ID" and the *per-prefix numbering hole* / *executable spec REQ-ID* predicates that page already defines. Pin the predicate to both-set membership: a `PREFIX-N` that appears simultaneously as a live REQ-ID for a prefix and in that prefix's `## Retired REQ-IDs` retirement table — the both-sets complement of the per-prefix numbering hole (neither-set), surfacing the GOV-8 *Deletion* no-reuse violation as a CI failure. Seed a clash fixture (a `PREFIX-N` present in both a fixture page's live and retired tables) with a no-violation counterpart in H5a's `Tests.`/`Ships when.` so the arm is exercised like every other H5a gate arm. H5a's `Adds.` may keep the short token unchanged, since it cites *REQ-ID discipline*.

## Solution constraints

- The clash arm's live-corpus footing follows the same H6a-deferred posture as the other H5a live-corpus reconciliation arms — exercised against seeded fixtures at H5a, binding against the live corpus only at the H6a-activated loom 1.0 release gate.

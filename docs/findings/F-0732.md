## Problem

The GOV-15 disambiguation sentence — *"Observable (c)'s equivalence relation, not the exclusion list, governs variation that is embedded as a placeholder sub-field inside a content string."* — uses the definite phrase *"the exclusion list"* in a paragraph that names two distinct lists: the variable-placeholder normalisation list introduced one sentence earlier, and the independent-observables exclusion list (wall-clock timing, token counts, log-line volume) named two sentences later. The sentence is grammatical against either antecedent, and the two readings yield different scopes for observable (c) — one in which embedded placeholder sub-fields fall under (c)'s normalised byte-identicality, one in which they are unhandled by either mechanism. The same ambiguity is mirrored in the orientation aggregator's Source-language stability NFR bullet (`overview-and-orientation.md`, anchor `source-language-stability`), where the surrounding context is thinner.

## Issue introduction

**Verdict:** present-since-inception (of the disambiguation sentence)
**Introducing commit:** `1406d57` (2026-06-01) — *"pi-loom spec: resolve 'loom-system-note equivalence undefined for variable sub-fields'"*
**History:** The sentence *"Observable (c)'s equivalence relation, not the exclusion list, governs variation that is embedded as a placeholder sub-field inside a content string."* was introduced in `1406d57`, the same commit that added the variable-placeholder normalisation list (per-invocation IDs, `<ms>` wall-clock sub-fields, category-8 tails) to GOV-15 and to the `spec.md` NFR bullet, in mirrored form. The pre-existing paragraph already named *"Wall-clock timing, token counts, and log-line volume are explicitly excluded from this expectation"* — the *independent-observables exclusion list*. `1406d57` therefore added a second list (the normalisation list) and, in the same commit, a sentence referring to *"the exclusion list"* with the definite article — without renaming, numbering, or qualifying either of the now-two candidate antecedents. The ambiguity is intrinsic to that edit; no later commit has altered the sentence. Subsequent commits (`494f7fc`, `fe8aa5c`, `5a772e3`) re-edited the surrounding paragraph for unrelated concerns, the topic split (`81bf52a`, 2026-06-11) relocated GOV-15 to `governance/source-language-stability.md`, and `7995d3c` (2026-06-05) extracted GOV-15 from a retired-prefix-table file — all carried the defective sentence forward verbatim. There is no later commit that "broke" a previously-clear sentence; the defect is co-extensive with the sentence's existence.

## Solution approach

Rename the bare phrase *"the exclusion list"* to *"the independent-observables exclusion list"* in the GOV-15 disambiguation sentence, binding it to the named role the following sentence already establishes. Apply the identical rename at both edit sites: the owner paragraph (`governance/source-language-stability.md`, anchor `gov-15`) and the orientation-aggregator NFR bullet (`overview-and-orientation.md`, anchor `source-language-stability`).

## Solution constraints

- Both edit sites must land in the same commit per the GOV-30 lock-step rule binding the orientation aggregator to the GOV-15 owner paragraph.

## Finding

The FNV-1a reference-vector table in `binder/determinism-cancellation-failure.md` §Determinism is preceded by prose that names the hash input using the corpus-canonical term "slash name" — "the bare command name, without the leading `/`… on a slash name `s` matching `^[a-z0-9][a-z0-9_-]*$` the input bytes are the ASCII code points of `s` in order". The table itself then introduces a fresh, non-canonical label "Loom name" as the column header for the same value:

```
| Loom name | Seed |
| --- | --- |
| `code-review` | `0x7ba86b63` |
```

"Slash name" is the term DISC-4 (`discovery/discovery-sources.md` §Filename validity / §Slash-name collision rules) treats as canonical for the value being shown — the filename stem matching `^[a-z0-9][a-z0-9_-]*$` — and it is the term used throughout this section's own surrounding prose. "Loom name" appears nowhere else in the spec corpus as a designator for this concept; the closest neighbouring use, in `schemas.md`, applies "loom name" to a schema *field* identifier (contrasted with "wire name"), which is a different concept entirely. The single-row mismatch leaves an implementer eye-balling the table to bridge two terms for one quantity without a glossary entry tying them together.

## Issue introduction

**Verdict:** single-introducing-commit
**Introducing commits:** `a7c882a` — *pi-loom spec: resolve "FNV-1a binder seed: byte encoding unspecified, no normative test vector"* (2026-05-05)
**History:** The FNV-1a reference-vector table did not exist before commit `a7c882a`, which added the surrounding "byte sequence hashed is the UTF-8 encoding…" prose and the three-row vector table to the then-monolithic `spec_topics/binder.md` in response to an earlier review finding about missing normative test vectors. That same commit's diff is where the column header "Loom name" was first authored; the surrounding prose introduced in the very same hunk already uses "slash name" (`…on a slash name s matching ^[a-z0-9][a-z0-9_-]*$ …`), so the inconsistency is present from the moment the table appears. Commit `f5e89f4` (2026-06-04) later split `binder.md` into the subdirectory layout, relocating the table verbatim from `spec_topics/binder.md` §Determinism to `spec_topics/binder/determinism-cancellation-failure.md` §Determinism — `git blame` of lines 7–11 attributes the current header to `f5e89f4` only because the split rewrote the file, but `git log -S "Loom name"` confirms the string itself entered the corpus at `a7c882a` and has been carried unchanged since. No intermediate commit touched the header.

## Solution Space

**Shape:** single

### Recommendation

Rename the column header from `Loom name` to `Slash name`, leaving the three data rows and every other line of §Determinism untouched:

```
| Slash name | Seed |
| --- | --- |
| `code-review` | `0x7ba86b63` |
| `hello` | `0x4f9f2cab` |
| `a` | `0xe40c292c` |
```

This is a one-cell edit: the table is referenced only positionally from elsewhere in the spec and from `plan_topics/coverage-matrix.md`'s `cka-42` row (which cites the section, not the column heading), so no cross-link, anchor, or `<a id>` changes are needed. No other occurrence of "Loom name" in the determinism table exists in the corpus to harmonise.

## Consequence

**Severity:** cosmetic

A reader resolves the term mismatch trivially in context. No implementation diverges, no test changes, no rendered surface is affected. The cost is a brief snag while reading the determinism section and a residual risk that "loom name" leaks into downstream documentation or diagnostic prose as if it were a coined term, when it is not.

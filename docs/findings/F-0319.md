## Problem

`conventions.md` release-time residue-inspection item 7 (*Split-row joint-completeness*) derives a row's facet partition by an un-annotated branch that takes the closing-leaf set "minus any co-witness leaf", but an un-annotated cell in `coverage-matrix.md`'s *Numbered REQ-IDs* table carries no co-witness marker — the co-witness/facet distinction lives only in the *CIO closure rationale* prose paragraph, which item 7 does not read. The `CIO-1 | V16a, V4e, V11f` cell therefore defaults to treating all three leaves as facets, directly contradicting the closure rationale, which makes `V16a` the sole facet with `V4e`/`V11f` as droppable co-witnesses (item 7 hardcodes the answer only for `CIO-5 | V16a, H7a`). The derivation is not mechanically reproducible: two reviewers running item 7 over the same row partition it differently, raising false residue defects against co-witnesses or masking a genuinely-uncovered facet.

## Issue introduction

**Verdict:** multi-commit-interaction

**Introducing commits:** `cda161b` — "pi-loom plan: resolve \"split REQ-IDs pass closing gate on one facet citing test\"" (2026-06-17, proximate); `8ef7055` — "pi-loom plan: resolve \"V16a over-claims all CIO bullets provable at stateless seam\"" (2026-06-11, latent precondition)

**History:** The `CIO-1` row has existed since the initial plan build (`c6a664e`, 2026-06-10) and was refined in `2565ddd` (2026-06-10). `8ef7055` (2026-06-11) introduced the co-witness *concept* into `coverage-matrix.md`: `git log -S 'co-witness' -- docs/plan_topics/coverage-matrix.md` shows that commit added the *CIO closure rationale* paragraph describing `V16a` as the facet closer and `V4e`/`V11f`/`H7a` as co-witnesses — but only in narrative prose, never as inline cell annotations. The mechanical defect was then introduced by `cda161b` (2026-06-17), which authored item 7's *Split-row joint-completeness* sweep (`git log -S 'minus any co-witness leaf' -- docs/plan_topics/conventions.md` and `git log -S 'Split-row joint-completeness'` both resolve to `cda161b`): its "otherwise the partition is the cell's closing-leaf set minus any co-witness leaf" branch assumes a cell-level co-witness marker that `8ef7055` had only ever placed in prose, and item 7 hardcoded the `CIO-5` example while leaving `CIO-1` underivable. The gap is the interaction of the two — the prose-only co-witness record (`8ef7055`) and the mechanical rule that presumes it is readable from the cell (`cda161b`).

## Solution approach

Make the split-row facet partition fully derivable from each matrix cell alone, with no recourse to the *CIO closure rationale* prose. Rewrite `conventions.md` item 7 (*Split-row joint-completeness*) so co-witness leaves are identified by an explicit cell-level marker — distinct from the parenthetical facet-annotation form so the two read as opposite classifications — rather than by the unreadable "minus any co-witness leaf" default. Mark the co-witness leaves inline in the `coverage-matrix.md` *Numbered REQ-IDs* table so each multi-leaf cell names which leaves are dropped: `V4e` and `V11f` under `CIO-1`, and `H7a` under `CIO-5`.

## Solution constraints

- The co-witness marker must survive `H5d`'s leaf-ID tokeniser (backtick-delimited leaf IDs only, parenthetical prose dropped) without altering the leaf-ID set `H5d` and the `H5a` per-REQ-ID citing-test scan read.

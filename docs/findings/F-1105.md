## Problem

The `H1a` `engines.node` literal-read Tests bullet ends with a clause saying a Node-floor move during the version-bump procedure ([`version-bump-step2b.md` step 3]) "keeps it aligned with `capability-probe.md` §(a)". The phrasing reads as if the bump procedure mechanically maintains the inline literal, but that literal is a hand-transcribed copy resident in the H1a test code, and `version-bump-step2b.md` step 3 enumerates the artefacts a contributor edits (`package.json#engines.node`, `capability-probe.md` §(a), the orientation sentence, and the `SDK_SURFACE_INVENTORY` `pi-engines-node` row) without naming the H1a inline literal. The behaviour is safe — the H1a test reds on drift — but the prose obscures that the inline literal is a manual co-update obligation surfaced reactively by a red test, not maintained by the procedure.

**Verdict:** present-since-inception
**Introducing commit:** `f1a276a` (2026-06-16) — "pi-loom plan: resolve \"engines.node literal-read test names no in-code comparison operand\""
**History:** The transcribed-literal comparison-operand mechanism and the "keeps it aligned with `capability-probe.md` §(a)" phrasing entered together in `f1a276a` (`git log -S "keeps it aligned with"` over `docs/plan_topics/H1a-scaffold-and-toolchain.md` first reports the phrase at that commit). A later commit `1752a4e` (2026-06-20) — "fix F-0470 — drop stale-prone Node-floor literal snapshot in H1a Tests" — revised the same sentence to remove the embedded `(currently >=22.19.0)` literal snapshot but preserved the "keeps it aligned" clause verbatim, so the clarity defect has stood since the mechanism was authored.

## Solution approach

Rewrite the "keeps it aligned" clause in the `H1a` `engines.node` literal-read Tests bullet to state that the inline transcribed literal is hand-maintained: a Node-floor move that updates `package.json#engines.node` obligates the contributor to also update this inline literal to the new `capability-probe.md` §(a) value, and `H1a`'s own test reds if the manifest moves without that co-update. Make the red test the described alignment mechanism rather than the bump procedure.

## Solution constraints

- The spec is read-only for this fix; limit edits to the `H1a` `engines.node` Tests bullet and do not add the inline literal to `version-bump-step2b.md` step 3's artefact list.
- Do not assert that `version-bump-step2b.md` step 3 currently names the H1a inline literal; step 3 enumerates `package.json#engines.node`, `capability-probe.md` §(a), the orientation sentence, and the `SDK_SURFACE_INVENTORY` `pi-engines-node` row only.

## Problem

`V14a` carries three conceptually distinct verification surfaces in one paired leaf: the parse-time argument checks and the `CodeToolError` API (both citing `tool-calls.md` and closing coverage-matrix row `cka-13`), plus the five accepted-path `execute()` envelope-lowering mechanics (citing `host-interfaces-core.md` §*Tool execution from loom code* and the sole closure of coverage-matrix row `cka-46`). The envelope surface is a self-contained obligation set with its own spec page and its own coverage row, independent of the `cka-13` surface. The single `Ships when` gate folds seven-plus independently-failable assertions into one claim, so a leaf author who lands the parse checks and the `CodeToolError` API but mis-implements one envelope mechanic still passes the majority of the leaf's bullets, hiding a partially-correct envelope implementation at review time.

## Solution approach

Split the five `execute()` envelope-lowering mechanics out of `V14a` into a new paired V14 leaf whose **Spec** is `host-interfaces-core.md` §*Tool execution from loom code*, moving the envelope-lowering Tests bullet into it. In `V14a`, strike the envelope clause from **Ships when** and the `#tool-execution-from-loom-code` citation from the **Spec** field. Retarget the `cka-46` row's closing leaf in `coverage-matrix.md` from `V14a` to the new leaf, and extend `H5b`'s `V14a`–`V14f` Deps range to cover the new leaf. Link the new leaf into `plan.md` §V14.

## Solution constraints

- Keep the parse-time argument checks, the `CodeToolError` enum, the return-type table, the `.loom`-callable failure surface, and the `ERR-13` live-carrier witness in `V14a` — only the five envelope-lowering mechanics move.
- Leave the `cka-13` row unchanged; only the `cka-46` row retargets.
- Allocate the new leaf ID as the next free letter in the V14 phase, authored as a paired bare + `-T` leaf with `V14a` in its Deps; do not reuse or invent IDs.

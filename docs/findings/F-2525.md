## Problem

The `model` row's body cell of the field-contract table in `frontmatter/frontmatter-fields-a.md` describes the absent-`model:` default as the loom inheriting "the caller's model", while every other authoritative statement of the same default names the Pi session model: the row's own default column ("Pi session's model at invocation time"), the `model` prose bullet (`ctx.model`), and the `ctx.model` pin in `host-interfaces-core.md`. "caller" is an overloaded term in this corpus (slash caller vs invoke parent), so the table cell read in isolation can suggest an `invoke` child adopts the parent loom's pinned model — contradicting the actual mechanism, where an invoke child reads `ctx.model` (the parent's session-view value) and looms never write their pinned model back into `ctx.model`. The table-cell referent wording is the only live defect; the snapshot-stability concern is already settled by the surrounding prose.

## Issue introduction

**Verdict:** single-commit
**Introducing commit:** `fb54586` (2026-05-04, "pi-loom spec: resolve \"Frontmatter field required/optional/default contract is unspecified\"")
**History:** `git log -S"inherits the caller's model"` over the spec corpus identifies `fb54586` as the commit that first authored the field-contract table, introducing the `model` row as `| ... | Pi session's model at invocation time | The loom inherits the caller's model and pins it for the loom's lifetime ... |`. The body/default-column term mismatch ("caller's model" body vs "Pi session's model" header) was present from that authoring commit. A later commit, `616f046` (2026-06-04, resolve "Frontmatter `model` — loom's lifetime vs per-invocation inheritance"), rewrote the row and prose to fix the per-invocation snapshot-stability ambiguity (loom's lifetime → "duration of that one invocation"; the heading's secondary concern) but carried the "caller's model" body wording through unchanged, so the referent mismatch persisted into the current text.

## Solution approach

Rewrite the `model` row's body cell of the field-contract table, replacing "the caller's model" with the canonical "Pi session's active model (`ctx.model`)" so the cell matches the row's own default column, the `model` prose bullet, and the `ctx.model` pin in `host-interfaces-core.md`. Keep the existing per-invocation / re-inheritance tail unchanged. No behavioural change.

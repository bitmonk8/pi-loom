## Finding

`host-interfaces-services.md`'s scope, established by the PIC-10 … PIC-20 sequence
that fills the body of the file, is the runtime's host-facing **DI seams**:
`Checkpoint`, `SchemaValidator`, `Clock` / `FakeClock`, `FileSystem` /
`FakeFileSystem`, `FileWatcher` / `FakeFileWatcher`, `TokenEstimator`, `IdSource`.
Each PIC entry pins one constructor-injected interface, its production adapter,
its test fake, and the per-runtime lifetime rule.

After the last PIC entry (`PIC-20 IdSource`) the file appends three paragraphs
whose subject matter is not DI-seam shape:

1. **`**Discovery API.**`** — describes how the extension owns enumeration,
   that it subscribes to `resources_discover` only as a re-discovery trigger,
   returns `{}`, and that cross-format collision detection runs from
   `session_start`. This is a **registration / discovery-flow** statement.
2. **`> **loom 1.0 seam — Pi-owned subagents collision source set.**`** — pins
   the `SlashCommandSource` collision-candidate set (`"prompt"` / `"extension"`
   / `"skill"`) as a single named runtime constant, anchored
   `#loom-1-0-seam-pi-owned-subagents-collision-source-set` (with legacy alias
   `#v1-seam-pi-owned-subagents-collision-source-set`). It is the normative
   home of the *collision source set* and is referenced as such from
   `registration-steps.md` step 3, `host-prerequisites.md` (c-ii), and
   `future-considerations/surface-extensions.md`.
3. **`**No Pi-owned discovery path enumerates `.loom` or `.warp`.**`** — the
   `.loom` / `.warp` namespace-exclusivity guarantee, derived from Pi's
   three-arm `SlashCommandSource` union, the absence of a `loomPaths` slot
   on `ResourcesDiscoverResult`, and the closed `pi.*` manifest-namespace set.
   This is a *discovery-side* invariant tied to the
   `discovery/discovery-sources.md#file-extension-namespace` paragraph that
   already documents the parallel manifest-namespace check.

None of the three is a DI seam or a property of one, so a reader scanning
`host-interfaces-services.md` for seam contracts finds three off-topic
paragraphs at the end, and a reader looking for the discovery-API contract or
the namespace-exclusivity guarantee has to know to grep
`host-interfaces-services.md` rather than the obvious targets
(`registration-steps.md`, `discovery/discovery-sources.md`). The "collision
source set" callout is additionally cross-referenced from three other files by
a `host-interfaces-services.md#…` URL even though those references' actual
subject — slash-command registration / collision — lives in
`registration-steps.md`.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `b987cb4` (2026-05-04 — *resources_discover is a
contribution event, not an inbound path feed*) introduced the **Discovery
API** paragraph; `db0dd3c` (2026-05-06 — *warp slash-invocation prevention
unverified negative claim*) introduced the **No Pi-owned discovery path
enumerates `.loom` or `.warp`** paragraph; `7a6cf96` (2026-05-07 — *V1
seam-preservation MUSTs hidden inside the deferred-features narrative page*)
introduced the **V1 seam — Pi-owned subagents collision source set**
blockquote. All three landed in the then-monolithic
`spec_topics/pi-integration-contract.md`, where they sat in the trailing
*Discovery API* / *Pi-version-bump* region of a 576 KB file. The file split
commit `f5e89f4` (2026-06-04 — *enforce 100KB hard / 30KB recommended size
cap on spec set*) routed all three paragraphs verbatim into the new
`host-interfaces-services.md` child page on the mechanical "trailing block of
the parent file goes to the file that absorbed the last preceding section"
heuristic the splitter used, even though none of them is about a DI seam.
**History:** The misplacement is the interaction between three
independently-motivated additions and the size-cap split: each paragraph was
appended without a placement audit (each commit message confirms it
addresses a different review finding entirely), and the splitter then
cemented the trailing position without re-classifying by subject.

## Solution Space

**Shape:** single

### Recommendation

Move each of the three paragraphs to the file whose subject it actually
matches, preserve the three existing HTML anchors at the new sites so that
the four already-present cross-references continue to resolve without a
sweep, and leave a one-line forwarding pointer in
`host-interfaces-services.md` for the cross-references that have not yet been
migrated.

Target placement:

- **`**Discovery API.**`** — append to `registration-steps.md` as the
  registration-flow framing paragraph immediately before step 1 (or as a
  framing note at the top, above step 1), since it summarises the very
  registration flow that file's numbered steps then specify. The sentence
  "Cross-format collision detection consults `pi.getCommands()` on
  `session_start` …" already names registration-step 3 by reference.
- **`> **loom 1.0 seam — Pi-owned subagents collision source set.**`** —
  move to `registration-steps.md` immediately after step 3 (the step that
  consults `pi.getCommands()` and applies the collision-source-set
  membership test). Preserve both the `loom-1-0-seam-pi-owned-subagents-collision-source-set`
  anchor and the `v1-seam-pi-owned-subagents-collision-source-set`
  legacy-alias anchor on the moved blockquote.
- **`**No Pi-owned discovery path enumerates `.loom` or `.warp`.**`** —
  move to `discovery/discovery-sources.md` adjacent to the existing
  *File-extension namespace* paragraph it already overlaps with, since the
  namespace-exclusivity guarantee is the load-bearing companion to the
  manifest-namespace check that paragraph documents.

Implementer-relevant edge cases:

- Anchor preservation is mandatory: `surface-extensions.md`,
  `host-prerequisites.md` (c-ii), and `registration-steps.md` step 3 all
  link to `host-interfaces-services.md#v1-seam-pi-owned-subagents-collision-source-set`.
  Either (a) re-point those four references to the new file in the same
  edit, or (b) leave a stub anchor at the old site that redirects readers
  to the new home. Option (a) is cleaner; option (b) is safer if any
  external review artefact (e.g. the consolidated spec-review document)
  also carries the old URL.
- The `host-prerequisites.md` (c-ii) and `registration-steps.md` step 3
  references are circular: both currently link "into" the seam from
  `registration-steps.md` and from `host-prerequisites.md`, then the seam
  links back to `discovery/discovery-sources.md` and
  `future-considerations.md`. After the move the cross-link from
  `registration-steps.md` step 3 collapses into a same-file anchor; from
  `host-prerequisites.md` (c-ii) it becomes a `./registration-steps.md#…`
  link.
- The `surface-extensions.md` *Carrier* line names the target as "Pi
  Integration Contract — loom 1.0 seam: Pi-owned subagents collision
  source set"; the textual label survives the move (the carrier is still
  part of the Pi Integration Contract subtree), only the path / anchor
  fragment changes.
- Do not silently rename the heading; the `Original heading` and the
  `loom 1.0 seam — Pi-owned subagents collision source set` wording are
  load-bearing for the forward-compatibility narrative in
  `future-considerations/surface-extensions.md`.

## Consequence

**Severity:** advisory

Readers and reviewers navigating the spec from its natural subject anchors
(discovery, registration, namespace exclusivity) do not find these paragraphs
where they belong. Cross-references that already point into
`host-interfaces-services.md` for slash-command-registration concepts are
mildly misleading about where the concept lives. No implementer is blocked,
and no two implementers diverge on behaviour, but the discovery and
registration narratives are weaker by the absence of the relocated content.

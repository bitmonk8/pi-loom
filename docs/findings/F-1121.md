## Problem

The family-(4) CJS-reach prohibition in `audit-recognised-shapes.md` extends from a finite syntactic enumeration into an open-ended data-flow obligation: "any call to a function whose source binding transitively resolves to the `createRequire` named export ... MUST NOT exist anywhere in the audited source tree, and the audit MUST enforce that prohibition." "Transitively resolves to" has no bounded definition and is not decidable from a per-module AST walk — it admits arbitrary chains through helper modules, factory returns, conditional re-exports, and reflection. Three sibling audit rules of the same mechanically-undecidable shape are framed as "acknowledged soft surface", but this clause is an absolute MUST with no caveat. That makes its satisfiability depend on tooling power, contradicting the closing "behavioural contract is invariant under tooling choice" sentence in the same paragraph.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** e5cd514 — pi-loom spec: resolve "T35 SDK capability inventory closed-set negative-direction audit" (2026-05-09, Thomas Andersen)
**History:** Commit `e5cd514` introduced the *Recognised import/access shapes* paragraph in the monolithic `pi-integration-contract.md`, including both the bounded syntactic enumeration of CJS-reach shapes *and* the unbounded *"transitively resolves to the `createRequire` named export … MUST NOT exist anywhere in the audited source tree, and the audit MUST enforce"* clause. The same commit already used the *"acknowledged soft surface"* framing (four instances in the diff, e.g. for *Substitution limitation* and *Stale-citation limitation*) but did not pair it with the new transitive clause. Later commits (`f5e89f4` 2026-06-04 split the page into sub-pages without changing this wording; `81b0e67` 2026-06-22 tightened "no such call MUST exist" to "any such call MUST NOT exist"; `23d28a1` 2026-06-22 swept cross-refs after a basename realignment; `81b0e67` and `ce48e09` 2026-06-22 added the `.resolve()` carve-out and relocated the ESM `type: module` obligation) refined the clause but never retrofitted the soft-surface acknowledgment.

## Solution approach

Narrow the family-(4) CJS-reach clause's "transitively resolves to the `createRequire` named export" sentence in `audit-recognised-shapes.md` to enumerate the closed set of syntactic shapes the audit MUST detect, scoped to bindings resolvable within a single module's syntactic scope. Add a *Transitive-resolution limitation (acknowledged soft surface)* note mirroring the `Substitution limitation` and `Stale-citation limitation` sub-bullets in `audit-failures.md` and the `Generated-output-in-src limitation` in `audit-resolution.md`, acknowledging that `createRequire` provenance reaching across module boundaries, through factory returns, or through reflective lookup is not mechanically detectable, that the mitigation is the contributor-side version-bump-checklist review obligation, and that widening detection is deferred past loom 1.0.

## Solution constraints

- Preserve the existing `.resolve(<specifier>)` carve-out unchanged — it narrows the rule rather than authorising a marker, and the soft-surface note MUST NOT be read as opening a marker path on family (4).
- Family (4) remains non-exemptible via the same-line `// allow-pi-surface:` marker per *Exemption mechanism*; the acknowledgment changes only the boundary of what the audit MUST detect, not the exemptibility of what it does detect.

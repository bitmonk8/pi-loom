## Problem

The `V14a` impl leaf (`docs/plan_topics/V14a-tool-calls.md`) and its paired `V14a-T` tests leaf carry eight distinct, independently-testable concerns — parse-time argument codes, the closed `CodeToolError` enum, four off-surface outcome routing paths, both return-type rows, the `.loom`-callable failure surface, the host-denial MUST, the ceiling-#4 depth-6 routing row, and the swallowing-handler / three-channel late-settlement suppression assertion — drawn from four separate spec documents (`tool-calls.md`, `conversation-drive.md`, `cancellation.md`, `ceilings-3-and-4.md`) behind a single compound **Ships when**. A vertical-slice leaf is defined as "the smallest feature that can ship and be tested independently" (`conventions.md` §3); this leaf is well past that boundary. No reviewer can verify it in one pass, and the per-phase TDD ritual reds/greens all eight concerns at once, so a concern can be under-implemented without an isolated signal flagging it.

**History:** `V14a` was over-scoped in its first commit (c6a664e): it already bundled the parse-time argument codes, the closed `CodeToolError` enum, three off-surface outcomes, the return-type table, the `.loom`-callable failure, and the host-denial MUST across two spec documents (`tool-calls.md`, `conversation-drive.md`). Three later per-finding plan fixes accreted onto the same leaf rather than splitting it: f5e57dd positively asserted both return-type rows, 3d1ffd7 added the ceiling-#4 depth-6 row (third spec document, `ceilings-3-and-4.md`), and bd2eaf4 split the conflated `internal-error` bullet into four off-surface outcome paths and added the swallowing-handler-attachment assertion (fourth spec document, `cancellation.md`) — producing the present eight-concern / four-document leaf.

## Solution approach

Split the `V14a` impl leaf and its `V14a-T` tests partner into concern-bounded paired `-T`/impl sub-leaves under `docs/plan_topics/`, each owning one coherent concern group with its own **Ships when**, and relink the V14 slice listing in `docs/plan.md`. Perform the decomposition first so the new sub-leaf ID set is fixed before any reference pass. Then retarget every cross-reference that currently pins `V14a` to the owning sub-leaf IDs — the `coverage-matrix.md` TOOL and denial-surface rows, `H5b` Deps, `H7a` Deps / provenance / permitted-code list, `V17a` / `V17a-T`, `V5e` / `V5e-T`, `V13f`, `V14b` / `V14b-T`, `V18d`, and `V4c` / `V4c-T`.

## Solution constraints

- The spec is read-only for this fix; no spec edits.
- New sub-leaf IDs follow the `V<N><letter>` scheme — next free letter in the V14 slice, no reuse of retired IDs, no invented IDs — per the project's Plan rules.
- The four-site cancellation-routing references in `V17a`, `V13f`, and `V5e` must continue to name exactly one V14-side owner after the split.
- `H5b` Deps and the coverage-matrix closing-leaf columns must stay in lockstep — add each coverage-producing sub-leaf to `H5b` Deps per the transitive-completeness rule, and leave no retired `V14a` ID referenced in either.

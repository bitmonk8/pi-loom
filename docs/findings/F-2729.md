## Problem

The manual real-host smoke gate (`real-host-smoke-gate.md`) reuses `H7a`'s committed multi-feature fixture `.loom` and scores a live run against pass criteria (b)–(d) that presuppose the fixture exercises a fixed set of observable behaviours: a binder pass whose output validates against the `ok | needs_info | ambiguous` envelope schema; a spawned subagent-mode `AgentSession` with committed turns and an injected mid-stream cancel point; and the four fidelity axes (streamed-token order vs `ctx.waitForIdle()`, single-turn prompt-mode append, a `pi.on` cancel-forward subscription, and cancellation propagation). `H7a`'s leaf imposes none of these as a fixture-content requirement — it carries no reference to subagents at all, and `V9i` (subagent isolation) is absent from its `Deps`, so the fixture has no obligation or dependency basis to spawn the subagent-mode `AgentSession` criterion (c) cancels inside. If the committed fixture omits any of these behaviours, the corresponding smoke criteria have nothing to observe and pass vacuously, silently voiding loom's only real-host divergence-detection mechanism on the production seam.

## Issue introduction

**Verdict:** present-since-inception (later aggravated)
**Introducing commits:** `fd22a3d` (2026-06-17) created `real-host-smoke-gate.md` with criteria (a)–(e) already presupposing a binder envelope pass (b), the four fidelity axes (d), and the permitted-code list (e); `c768005` (2026-06-25, F-1363) widened the presupposition by adding the criterion-(c) subagent-mode `AgentSession` mid-stream-cancellation witness when it relocated the `V9i` ERR-8 live-surface confirmation to the smoke gate.
**History:** `git log --follow`/`--diff-filter=A` shows the smoke-gate artefact was born presupposing fixture content; `git show c768005` confirms F-1363 edited only `V9i`, `V4c`, `coverage-matrix.md`, and `real-host-smoke-gate.md`, never touching `H7a`. `git log -S "V9i"` and `grep` over `H7a-integration-acceptance.md` return zero hits — `H7a` has never carried a subagent or `V9i` reference, so no commit ever propagated either presupposition into a fixture-content requirement on `H7a`.

## Solution approach

In `docs/plan_topics/H7a-integration-acceptance.md`, require the committed multi-feature fixture `.loom` to exercise the observable behaviours the real-host smoke criteria (b)–(d) score, so the same fixture is scoreable against both the in-process double and a live host. Add `V9i` to `H7a`'s `Deps` to give the fixture a dependency basis for the subagent-mode spawn criterion (c) drives the mid-stream cancellation inside (today `Deps` lists `V17a` for cancellation but no subagent leaf).

- The spec and the session-double contract are read-only for this fix.

## Problem

The inventory-closure audit's marker-scope contract is entirely line-keyed — *Marker scope is per source line* binds authorisation to "line N", *Multi-line surface placement* pins an *originating line* per shape for references that span multiple physical lines, the *Malformed-marker discriminator* clause (e) and the *Stale sub-kind (s1)* are defined against physical lines, and the wire format emits an integer `line` field — yet no rule names the set of code points that terminate a physical line. The *Source-file encoding* paragraph pins UTF-8 but encoding does not fix line boundaries, and candidate walkers diverge: ECMAScript's `LineTerminator` production recognises LF, CR, CRLF, LS, and PS; TypeScript's line-start primitive recognises only LF/CR/CRLF; naive splitters recognise less. Two conformant audits picking different models flip observable outputs on the same source tree — a marker counts as on or off the offending line, a multi-line surface picks a different originating line, and the family-(5) `(s1) no-surface-on-line` predicate fires or not — defeating the *Wire serialisation* byte-identical-output guarantee. The per-`<symptom>` fixture floor pinned by the *Failure-surface contract* sits directly on this divergence.

**History:** `git log -S "physical line"` and `git log -S "Multi-line surface placement"` over `docs/` each return exactly `e5cd514` as the introducing commit (a later `f5e89f4` merely split the monolithic file into `audit-resolution.md` / `audit-failures.md` without touching the wording, and `81bf52a` moved the directory under `spec_topics/`). `git log -S "line-terminator"` / `"CRLF"` / `"U+2028"` over `docs/spec_topics/pi-integration-contract/` returns no commits — the terminator set has never been pinned in PIC. The sibling `lexical.md` rule that normalises CRLF / bare CR to LF before lexing exists for the `.loom` source language, not for the audit's TypeScript-source walker, so it does not transitively fix the audit's gap.

## Solution approach

Add a *Source-file line terminators* sentence to the *Source-file encoding* paragraph in `audit-resolution.md` (anchor `audit-scope`) that normatively pins the line-terminator set the audit's "physical line" / "same-line" / `line N` rules resolve against — the ECMAScript `LineTerminator` set (LF U+000A, CR U+000D, LS U+2028, PS U+2029, with the CRLF sequence counted as a single boundary) — together with the byte sequence that constitutes one line boundary. Pin the `line` field's counting basis to the same set so the wire `path:line` ordinal, *Marker scope*'s `line N`, and *Multi-line surface placement*'s *originating line* denote the same line number for a given reference.

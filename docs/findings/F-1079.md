## Problem

The `Adds.` placeholder note in `leaf-template.md` summarises Class-1 binding as a mechanism that "binds only if it carries a cited REQ-ID or is a cross-leaf seam that some other leaf binds against." That inline summary collapses the three Class-1 binding paths defined in `conventions.md` *Leaf format* §Adds — a cited `PREFIX-N` REQ-ID, a cited `loom/...` diagnostics-registry code, or a named normative step on a code-keyed obligation page — down to the REQ-ID arm alone. A leaf author who stops at the template's note would mis-classify a registry-code-bound or code-keyed-obligation mechanism as non-binding, the opposite of what `conventions.md` states.

## Issue introduction

**Verdict:** present-since-the-note's-introduction (later widened)
**Introducing commits:** `c6a664e` (2026-06-10, "pi-loom plan: build/update plan for spec.md + review") — added the `Adds.` binding note to `leaf-template.md` already phrased as "binds only if it carries a cited REQ-ID …", dropping the non-REQ-ID Class-1 arm(s) then defined in `conventions.md`. `b737beb` (2026-06-10, "resolve 'Adds. binding clause (i) cannot bind code-keyed obligations'") — expanded `conventions.md` Class-1 to add the code-keyed-obligation-area arm, widening the template↔conventions gap without reconciling the template.
**History:** `git log --follow` / `git log -S` over `leaf-template.md` shows the `Adds.` note's single-arm "cited REQ-ID" summary has been present since `c6a664e` first authored the note; it has never matched `conventions.md`'s Class-1 enumeration. Two later commits (`dd28dbb`, `9b8964f`, both refining the *Class-2 seam* clause of the same note) touched the surrounding sentence but left the Class-1 summary untouched. The corpus is git-tracked; the verdict is determinate.

## Solution approach

Rewrite the `Adds.` placeholder note in `leaf-template.md` so its Class-1 binding summary agrees with `conventions.md` *Leaf format* §Adds. Either enumerate all three Class-1 paths inline alongside the existing Class-2 cross-leaf-seam clause, or replace the inline Class-1 enumeration with a deferral to `conventions.md` for the three binding paths.

## Solution constraints

- `conventions.md` is read-only for this fix — it is the authoritative source and already states the three-arm rule correctly.

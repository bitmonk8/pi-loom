## Problem

Step 0 (c)'s `AgentSession.prototype.abort` bullet in `capability-probe.md`
misattributes the class's provenance. It claims `AgentSession` is re-exported into
`@earendil-works/pi-coding-agent` from `@earendil-works/pi-agent-core` ("where the
underlying class lives"), and that the loom runtime depends on that cross-package
re-export. At the loom 1.0 Pi-SDK pin the class is declared directly in
`@earendil-works/pi-coding-agent` at `dist/core/agent-session.d.ts`; the token
`AgentSession` does not appear in `@earendil-works/pi-agent-core` at all. The named
cross-package re-export does not exist, so the "Pi minor drops the re-export"
fragility scenario points at a non-existent dependency. The downstream behavioural
claim ŌĆö that a missing class or `abort` member surfaces as
`loom/load/host-incompatible` rather than a runtime `TypeError` ŌĆö remains correct.

## Issue introduction

**Verdict:** single-commit

**Introducing commits:** `68ff2d1` ŌĆö `pi-loom spec: resolve "Pi API surfaces asserted without .d.ts citations"` (2026-05-07)

**History:** Before `68ff2d1`, the `AgentSession.prototype.abort` bullet named only the probe expression and the namespace ("imported `AgentSession` class on the `@mariozechner/pi-coding-agent` namespace") and made no claim about where the class is declared. `68ff2d1` rewrote the bullet to add the "import path is load-bearing" rationale, introducing both the `dist/core/index.d.ts` re-export citation and the parenthetical "(where the underlying class lives)" pointing at `@mariozechner/pi-agent-core` ŌĆö the defect this finding reports. The later scope rename to `@earendil-works/*` (`5e73072`) and the SDK-pin bumps (`f985a17`, etc.) carried the wrong attribution through verbatim without re-checking it; none of them introduced the defect, they only preserved it. `git log -G "underlying class lives"` and `git log -S "where the underlying class lives"` both isolate the introducing commit to `68ff2d1`.

## Solution approach

Rewrite the provenance sentence and the fragility clause inside the
`AgentSession.prototype.abort` bullet of Step 0 (c). Record that the class is declared
in `@earendil-works/pi-coding-agent` at `dist/core/agent-session.d.ts`, consistent with
`host-interfaces-core.md` ┬¦`AgentSession (member surface loom touches)`, and resolved
through the package-root specifier rather than a deep import. Reframe the fragility
clause so that a Pi minor dropping the package-root re-export, or relocating the class
into a different `@earendil-works/*` package, surfaces at this probe as
`loom/load/host-incompatible`. Leave the missing-or-renamed-`abort`-member sentence
and the probed expression unchanged.

## Finding

The `cka-33` coverage-matrix row (swallowing-handler / total three-side-channel suppression) is a multi-leaf row whose facet partition is the five closing leaves `V17a`, `V14f`, `V13f`, `V15h`, `V9o`. The `coverage-matrix.md` *Multi-leaf-row per-facet citing tests* convention, and the `H5f` per-facet citing-test gate that enforces it, require each facet leaf to carry a **facet-naming citing test** — a test that cites *both* the row's `cka-<n>` token (here `cka-33`) **and** that facet's own closing-leaf-ID inline in the test source.

Every facet leaf's swallowing-handler Tests bullet, however, cites the obligation only by spec-section paraphrase: `[cancellation.md — *Race semantics — swallowing-handler attachment on every abandonable Promise*]`. None of the five bullets instructs the implementer to write `cka-33` or the facet's own leaf-ID into the test. `V17a`'s "Swallowing-handler side-channel suppression" bullet and the "Swallowing-handler attachment at this site" bullets in `V14f`, `V13f`, `V15h`, and `V9o` all share this shape.

An implementer authoring tests verbatim from these bullets produces test files that cite neither `cka-33` nor the facet leaf-ID. `H5f`'s best-effort inline-citation existence scan therefore finds no facet-naming citing test for any of the five facets and reddens the loom 1.0 release gate (the footing flip activated by `H6a`), even though the suppression behaviour is fully implemented and asserted.

**History:** The per-site facet leaves were carved out of the bundled cancellation/tool/query/invoke leaves across 2026-06-16/17 (eaa5000, 5a1915d, 5f4d271, 6d7661d), each authored with a swallowing-handler Tests bullet citing only the `cancellation.md` spec-section paraphrase. Commit 26a8794 (2026-06-18) then added the `H5f` gate and the *Multi-leaf-row per-facet citing tests* convention requiring each facet test to cite both `cka-33` and its facet leaf-ID, but touched only `H5f-per-facet-citing-test-gate.md`, `conventions.md`, and `coverage-matrix.md` — it never back-propagated the new citation requirement into the facet leaves' Tests bullets. The defect is the interaction: the requirement and the bullets that must satisfy it diverged.

## Solution Space

**Shape:** single

### Recommendation

In each of the five facet leaves, revise the swallowing-handler Tests bullet so it instructs the test to cite both the row token `cka-33` and that facet's own closing-leaf-ID inline, alongside the existing `cancellation.md` section reference:

- `docs/plan_topics/V9o-subagent-swallowing-handler.md` — the "Swallowing-handler attachment at this site" bullet: instruct the test to cite `cka-33` and `V9o` inline.
- `docs/plan_topics/V13f-query-cancellation-routing.md` — same bullet: cite `cka-33` and `V13f` inline.
- `docs/plan_topics/V14f-tool-calls-swallowing-handler.md` — same bullet: cite `cka-33` and `V14f` inline.
- `docs/plan_topics/V15h-invoke-ceiling-swallowing.md` — same bullet: cite `cka-33` and `V15h` inline.
- `docs/plan_topics/V17a-cancellation-core.md` — the "Swallowing-handler side-channel suppression" Tests bullet: cite `cka-33` and `V17a` inline (this leaf is itself the substrate-suppression facet of the `cka-33` row).

The inline citation is the literal `cka-33` token and the facet's literal leaf-ID (e.g. `V9o`) appearing in the test source; the existing `cancellation.md` section reference stays. `H5f`, `coverage-matrix.md`, and `conventions.md` are read-only for this fix — the gate and convention already specify the requirement correctly; only the leaf Tests bullets must be brought into line.

The plan instructions and the release gate they feed are in direct conflict.

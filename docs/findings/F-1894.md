## Problem

The `cka-33` coverage-matrix row is a multi-leaf row whose facet partition is the five closing leaves `V17a`, `V14f`, `V13f`, `V15h`, `V9o`. The *Multi-leaf-row per-facet citing tests* convention on `coverage-matrix.md`, enforced by the `H5f` per-facet citing-test gate, requires each facet leaf to carry a test citing both the row's `cka-33` token and that facet's own leaf-ID inline in the test source. Every facet leaf's swallowing-handler Tests bullet instead cites the obligation only by spec-section paraphrase (`cancellation.md — Race semantics — swallowing-handler attachment on every abandonable Promise`), instructing neither token. An implementer authoring tests faithfully from these bullets produces sources that cite neither `cka-33` nor the facet leaf-ID, so `H5f`'s scan finds no facet-naming citing test and reddens the loom 1.0 release gate at `H6a` activation despite the suppression behaviour being implemented and asserted.

Commit 26a8794 (2026-06-18) then added the `H5f` gate and the *Multi-leaf-row per-facet citing tests* convention requiring each facet test to cite both `cka-33` and its facet leaf-ID, but touched only `H5f-per-facet-citing-test-gate.md`, `conventions.md`, and `coverage-matrix.md` — it never back-propagated the new citation requirement into the facet leaves' Tests bullets. The defect is the interaction: the requirement and the bullets that must satisfy it diverged.

## Solution approach

In each of the five facet leaves — `V9o`, `V13f`, `V14f`, `V15h`, and `V17a` — rewrite the swallowing-handler Tests bullet so it instructs the test to cite both the row token `cka-33` and that facet's own leaf-ID (`V9o`, `V13f`, `V14f`, `V15h`, `V17a` respectively) inline in the test source. Retain the existing `cancellation.md` section reference alongside the added citation. For four leaves this is the "Swallowing-handler attachment at this site" bullet; for `V17a` it is the "Swallowing-handler side-channel suppression" bullet (that leaf is itself the substrate-suppression facet of the row).

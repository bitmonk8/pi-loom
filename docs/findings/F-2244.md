## Problem

In `future-considerations/surface-extensions.md`, the *Typed-query support for providers without named-tool forcing* bullet (under "Surface extensions (loom 1.0 leaves a seam)") carries an inline cross-reference whose link text names "Provider compatibility for typed queries" but whose URL targets the Pi Integration Contract hub page (`../pi-integration-contract.md`), which has no such section. That section lives on `pi-integration-contract/conversation-drive.md` (the `**Provider compatibility for typed queries.**` paragraph). A reader following the link lands on the hub and must hunt for the named section. The same bullet's `*Anchored at:*` line already points at the correct sub-page, making the inline link internally inconsistent within the bullet.

## Issue introduction
**Verdict:** single-commit regression
**Introducing commit:** `f5e89f4` — "docs: enforce 100KB hard / 30KB recommended size cap on spec set" (2026-06-04)
**History:** Before `f5e89f4`, the Pi Integration Contract was a single page `docs/spec_topics/pi-integration-contract.md` and this bullet lived in `docs/spec_topics/future-considerations.md` (same directory). The same-directory link `./pi-integration-contract.md` therefore resolved to the page that carried the "Provider compatibility for typed queries" section — correct at section granularity, though it already lacked a fragment anchor. `f5e89f4` split the PIC monolith into the `pi-integration-contract/` subdirectory — creating `conversation-drive.md`, the new owner of that section — and simultaneously moved this bullet into `future-considerations/surface-extensions.md`. The link target was mechanically reprefixed `./` → `../pi-integration-contract.md` but was not repointed to the new sub-page, so it has landed on the hub ever since. `git log -G` confirms the `../pi-integration-contract.md` form first appears in `f5e89f4`; `git log -S` confirms the section content moved to `conversation-drive.md` in the same commit.

## Solution approach

Repoint the inline link in the typed-query-support bullet from `../pi-integration-contract.md` to `pi-integration-contract/conversation-drive.md`. Because the `**Provider compatibility for typed queries.**` paragraph on `conversation-drive.md` has no dedicated anchor, add anchor `id="provider-compatibility-for-typed-queries"` on that paragraph and target it with a fragment so the link lands on the named section.

## Solution constraints

- The bullet's `*Anchored at:*` line must keep targeting `#v1-seam-typed-query-supported-provider-set`; the seam-blockquote reference and the provider-compatibility-prose reference are distinct, intentional targets and must not be collapsed into one.

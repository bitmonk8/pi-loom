## Problem

The **Discovery roots** block (anchor `id="discovery-roots"` in `discovery-sources.md`) defines a discovery root as "the directory each source contributes to the discovery walk", then says a settings `loomPaths` file entry "contributes its parent directory". A reader can take that parent directory as a discovery-walk input and scan it for sibling `*.loom` files the author never named. The intended behaviour — settled only on `package-and-settings.md`'s `loomPaths` entry schema — is that a file entry registers exactly the one named `.loom` file, while the parent directory exists solely as a path-restriction anchor for callee containment on `invoke` / `.loom` `tools:`. The block overloads "discovery root" across both roles and carries no cross-reference to the schema that disambiguates.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** 833d8d5 — spec: resolve five invoke edge cases (Decision #4) (2026-05-05, Thomas Andersen)
**History:** The Discovery-roots concept did not exist before Decision #4; commit 833d8d5 introduced it (in the then-monolithic `spec.md`) specifically to anchor the new `invoke` / `tools:` path-restriction rule, and the conflation was present at birth — the same block both defined a root as "the directory each source contributes to the discovery walk" and used a file entry's parent directory solely as a path-restriction anchor. Later commits only relocated and relabelled the text (4aef0e2 renamed `looms`→`loomPaths`; f5e89f4 split the spec into topic files, moving the block verbatim into `discovery-sources.md`) without resolving the overload.

## Solution approach

In the Discovery-roots block, clarify that a discovery root anchors the path-restriction rule (callee containment for `invoke` / `.loom` `tools:`) while each source's discovered-loom set is governed separately, and that a `loomPaths` file entry's parent directory is a path-restriction anchor only — its discovered-loom set is the single named `.loom` file, not a directory scan. Add a forward-link from the block to `package-and-settings.md#loompaths-entry-schema` (the **File entries** / **Directory entries** rows).

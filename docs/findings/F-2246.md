## Problem

The `Post-cancel resolution` bullet under *Outcome enumeration (normative)* in `tool-calls.md` restates the full swallowing-handler attachment-timing MUST (attach to the `execute()` Promise at construction time, before the first microtask boundary, with the `.catch(() => {})` justification) as its own local normative requirement. That same constraint is canonically owned by `cancellation.md` under *Race semantics — swallowing-handler attachment on every abandonable Promise*, and the `tool-calls.md` bullet already closes with a `See [Cancellation — Race semantics]` pointer deferring normativity there. The duplication — not the prescription — is the defect: a future edit to the canonical rule that is not mirrored into this restatement would leave implementers reading divergent normative timings.

## Issue introduction
**Verdict:** single-commit
**Introducing commits:** `2ebe288` ("pi-loom spec: resolve \"Post-cancel late Promise settlement: unhandledRejection discard\"", 2026-05-07)
**History:** Before `2ebe288`, the `Post-cancel resolution` bullet in `tool-calls.md` was a terse summary closing with `See [Cancellation — Race semantics] for the normative discard rule`. That commit landed the swallowing-handler attachment rule (construction-site, before-first-microtask-boundary, `.catch(() => {})` justification) into `cancellation.md` as the canonical owner and, in the same commit, expanded the `tool-calls.md` bullet to restate the same normative MUST inline. The duplication therefore entered atomically with the rule itself; there was no prior single-owner state to revert to. The fix removes the duplicated restatement only — the canonical text introduced by `2ebe288` in `cancellation.md` is retained.

## Solution approach

Delete the swallowing-handler-MUST sentence (the construction-time / before-first-microtask-boundary / `.catch(() => {})` mechanism prose) from the `Post-cancel resolution` bullet in `tool-calls.md`, leaving the observable post-cancel outcome (later settlement discarded — no rebind `CNCL-1`, no second `Err` `CNCL-2`, no second `RuntimeEvent` `CNCL-3`) and the existing `See [Cancellation — Race semantics](./cancellation.md)` cross-reference that already defers the attachment-timing mechanism to its single owner.

## Solution constraints

- Out of scope: `cancellation.md`'s *swallowing-handler attachment on every abandonable Promise* paragraph (the canonical owner — leave its text intact) and `host-interfaces-core.md`'s parallel restatement (a separate finding's territory, not this bullet's edit).

## Finding

The `always-log set` glossary entry (`docs/spec_topics/glossary.md`) closes with `See: [Pi Integration Contract — Runtime event channel](./pi-integration-contract.md)`. The link text names the *Runtime event channel* sub-page, but the URL resolves to the Pi Integration Contract **hub** (`./pi-integration-contract.md`) — a short index page that lists sub-pages and does not contain the always-log set definition. The canonical definition (the set's group-A/group-B partition, the enumerated members, and the four justified exclusions) lives on `./pi-integration-contract/runtime-event-channel.md`, which exists as a standalone file.

The mismatch is local and demonstrable: the sibling `occurrence` vs. `origin` entry in the same glossary points at the same target using the correct sub-page URL `./pi-integration-contract/runtime-event-channel.md`. An implementer following the `always-log set` link lands on the hub index and must take one extra navigation hop to reach the page the link text already named.

## Issue introduction
**Verdict:** single-commit
**Introducing commit:** `f5e89f4` — "docs: enforce 100KB hard / 30KB recommended size cap on spec set" (2026-06-04, Thomas Andersen)
**History:** `git log --diff-filter=A` shows `f5e89f4` created `spec_topics/pi-integration-contract/runtime-event-channel.md` by extracting the Runtime event channel content out of the then-monolithic `pi-integration-contract.md` to satisfy the size cap. Comparing the glossary at `f5e89f4^` and `f5e89f4` (`git show …:docs/spec_topics/glossary.md`) confirms the `always-log set` entry's `See:` URL was `./pi-integration-contract.md` both before and after the split: prior to the split that hub URL was the correct target (the content lived there), and the split moved the content to the new sub-page without repointing the link. No glossary entry referenced the sub-page URL before the split (`grep -c` of the sub-page path at `f5e89f4^` returns 0); the correct sub-page URL on the sibling `occurrence`/`origin` entry was added in a later commit, leaving this entry behind.

## Solution Space

**Shape:** single

### Recommendation

In the `always-log set` entry of `docs/spec_topics/glossary.md`, change the `See:` URL from `./pi-integration-contract.md` to `./pi-integration-contract/runtime-event-channel.md`, leaving the link text unchanged. This matches the sibling `occurrence`/`origin` entry, which already points at the same sub-page.

## Consequence

**Severity:** cosmetic

A reader following the `always-log set` cross-reference lands on the PIC hub index rather than the canonical definition page named in the link text, costing one extra navigation hop. No runtime behaviour, wire format, or normative obligation is affected; the canonical content is reachable.

## Problem

Step 0(a) routes a syntactically invalid `process.versions.node` to `kind: "probe-failed"` via a non-throw value test (`semver.valid(...) → null`), but the *Self-failure* section defines `probe-failed` solely as a trapped throw whose `details.cause` is the coerced underlying-error string, and the `loom/load/host-incompatible` row in `code-registry-load.md` mirrors that throw-only framing. The non-throw route therefore has no defined `details.cause`, so conforming implementations diverge on the emitted `loom/load/host-incompatible` payload (one omits `details.cause`, another synthesises one).

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** 126ff8a — pi-loom spec: Step 0 capability probe rewrite + V1.0 deferrals + Choice B (2026-05-06, Thomas Andersen); eb33bf9 — pi-loom spec: resolve "Post-probe SDK-shape drift has no stated routing or operator attribution" (2026-05-07, Thomas Andersen)
**History:** The bundled defects have distinct (non-causal) origins, so the union spans two commits. The Step 0 capability-probe rewrite 126ff8a introduced obligations 1, 2, and 3 simultaneously: it added the `PiExtensionAPI` adapter-layer wording in the *On failure* bullet, the invalid-SemVer-`semver.valid`-routes-to-`probe-failed` clause alongside a throw-only *Self-failure* `details.cause` contract, and the "MUST stay disjoint" sentence in Step 0(c). Obligation 4's inline deferral entered one day later in eb33bf9, which added the *Post-probe SDK-shape drift* section (including the `details.kind`-discriminator-deferred-past-loom-1.0 statement). The 2026-06-04 commit f5e89f4 only relocated the content from `docs/spec/` into `docs/spec_topics/` (the size-cap collapse) and did not introduce any of the four defects.

## Solution approach

Clarify in Step 0(a) that the invalid-SemVer route is a non-throw value pre-check distinct from the trapped-throw *Self-failure* path, and state its `details.cause` disposition for that route. Rewrite the *Self-failure* `details.cause` contract and the `loom/load/host-incompatible` `probe-failed` description in `code-registry-load.md` so `details.cause` is documented as present-on-throw rather than universally the caught throw's underlying string.

## Solution constraints

- The `loom/load/host-incompatible` `details.kind` enumeration MUST NOT grow; the invalid-SemVer route stays on the existing `probe-failed` kind.

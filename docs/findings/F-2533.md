## Problem

The "No concurrent user sessions in the same host process" non-goal bullet in `model-changes-and-non-goals.md` buries its single load-bearing sentence — loom 1.0 presupposes one active user session per extension instance and defines no new disposition for an unanticipated second session — under roughly 600 words of detection-gap analysis (typecheck vs surface-inventory vs editorial-review behaviour, the `session_shutdown` closed-reason validator, the `pi.openParallelSession` hypothetical). The walkthrough constrains no implementer and is recoverable from the cited bump-procedure pages.

## Solution approach

Rewrite the "No concurrent user sessions in the same host process" bullet down to its normative core: the non-goal statement, the single load-bearing invariant (every site loom 1.0 scopes to one active user session remains so scoped, and the runtime defines no new disposition for a second session), and the existing `*Recorded at:*` line. Move the detection-gap walkthrough out of the normative bullet into a non-normative provenance note or ADR, or delete it.

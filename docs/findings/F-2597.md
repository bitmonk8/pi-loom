## Problem

DISC-5 (`#disc-5`) binds `pi.looms` glob resolution in normative prose by naming the `minimatch` engine and Pi's private build artefact (`dist/core/package-manager.js`, symbol `applyPatterns`). The observable obligation that actually matters тАФ the glob dialect plus the `!`/`+`/`-` override ordering loom must apply тАФ is already stated alongside, so the named-library and private-path pin adds no observable requirement. It also goes stale silently if Pi changes its backing matcher, entangling loom's correct obligation (track Pi's glob dialect) with a now-wrong library identity.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `1cb54c7` тАФ pi-loom spec: resolve "Glob !/+/- precedence and matcher engine unspecified for pi.looms / loomPaths" (2026-06-06, Thomas Andersen); `1ea0455` тАФ pi-loom spec: resolve "DI-seam vs reference-implementation placement boundary" (2026-06-04, Thomas Andersen); `1064946` тАФ pi-loom plan: resolve "YAML frontmatter parsing mechanism is never declared as a dependency" (2026-06-10, Thomas Andersen)
**History:** The normative `minimatch` reliance in DISC-5, together with the `dist/core/package-manager.js` / `applyPatterns` private-symbol pin, entered the corpus in commit `1cb54c7` (2026-06-06) тАФ `git log -S "applyPatterns"` and `git log -S "minimatch"` against `package-and-settings.md` each return exactly that one commit. The dependency inventory bullet in `implementation-notes.md` predates that reliance (created by `1ea0455`, 2026-06-04, for `semver`) and was subsequently extended for sibling packages (`chokidar`, then `yaml` via `1064946`, 2026-06-10) in response to parallel "X is never declared as a dependency" findings, but was never extended for `minimatch` тАФ `git log -S "minimatch" -- docs/spec_topics/implementation-notes.md` returns nothing. The "undeclared dependency" gap thus arose from the interaction between the reliance-introducing commit and the separately-evolving inventory that closed the sibling cases but never the `minimatch` one.

## Solution approach

Demote the `minimatch` identity and the `dist/core/package-manager.js` / `applyPatterns` references in DISC-5 (`#disc-5`) from a normative engine pin to a non-normative realisation note, leading instead with the observable dialect contract: loom MUST resolve `pi.looms` glob patterns under the same glob dialect and `!`/`+`/`-` override ordering Pi applies to its own resource arrays. Preserve the already-pinned observable specifics (fixed order plain тЖТ `!` тЖТ `+` тЖТ `-`, `+`/`-` as exact-path operands, `nocase` off, insertion-order preservation on same-class overlaps).

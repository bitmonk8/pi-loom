## Problem

`conventions.md` §REQ-ID discipline and the `H5e` leaf describe the un-anchored-MUST gate's un-rowed-page sub-recogniser as a mechanical arm that reddens on an un-rowed `spec_topics/**` page unless it is a GOV-24 table-of-contents hub stub, which it excludes; `H5e` hardens this into a seeded fixture asserting the hub-stub case stays green. No mechanical discriminator supports this exclusion: the only marker the plan supplies — "no row and no REQ-ID anchors" — is shared by every un-rowed page, since GOV-6 only forces a prefix-table row where a prefix appears in a REQ-ID, so a hub stub and an un-rowed content page carrying an un-anchored obligation are indistinguishable from the stated marker. Meanwhile residue-inspection item 5 concedes un-rowed-page classification is "which the row-keyed recogniser does not classify" and routes it to the manual sweep, directly contradicting the mechanical-arm claim. As written the gate cannot own un-rowed-page disposition, and the seeded `H5e` fixture cannot be satisfied as specified.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `a593446` (2026-06-17 17:15:19 +0200) "pi-loom plan: resolve \"un-anchored-MUST recogniser un-rowed-page branch\""; `5f33832` "pi-loom plan: resolve \"H5a bundles seven gate arms behind a single Ships-when\""
**History:** The un-rowed-page branch did not exist before `a593446`. That commit added, in a single edit to `conventions.md`, both the §REQ-ID-discipline prose declaring un-rowed pages "in scope … unless it is a GOV-24 … hub stub … which is excluded" and residue-inspection item 5's countervailing concession that un-rowed pages are "which the row-keyed recogniser does not classify" and are swept manually — introducing the latent tension and the marker-less hub-stub exclusion together. `5f33832` then split `H5e` out of `H5a` and hardened the prose into a mechanical arm with a seeded fixture asserting the hub-stub exclusion as a green case, turning the latent under-specification into a fixture that cannot be implemented from the stated discriminator. `git log -S 'the row-keyed recogniser does not classify'` returns only `a593446`; the `H5e` un-rowed sub-recogniser and fixture text first appear in `5f33832`. The durable fix addresses the interaction across both `conventions.md` and `H5e`, not either commit in isolation.

## Solution approach

In `conventions.md` §REQ-ID discipline, replace the marker-less "no row and no REQ-ID anchors" hub-stub characterisation with a mechanical discriminator grounded in the prefix table — e.g. an un-rowed page is excluded exactly when the table holds a trailing-slash row for the subtree that page indexes. Reword residue-inspection item 5 so it is residue-only and no longer claims to own un-rowed-page detection, leaving a single owner of that disposition. Update the `H5e` un-rowed-page arm and pin its hub-stub fixture case to whatever discriminator §REQ-ID discipline now states, so the seeded green case is realisable.

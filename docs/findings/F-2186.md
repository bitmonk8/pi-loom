## Finding

The H1 heading of `docs/spec_topics/future-considerations/model-changes-and-non-goals.md` reads `# Model changes and non goals`, spelling the compound noun as two unhyphenated words. Every other occurrence of the term in the same file is hyphenated — the section heading `## loom 1.0 non-goals`, the `<a id="loom-1-0-non-goals">` / `<a id="v1-non-goals">` anchors, and the body prose ("The loom 1.0 non-goals are…"). The filename itself uses the hyphenated form (`model-changes-and-non-goals.md`), and the hyphenated spelling is the universal corpus convention (e.g. `overview-and-orientation.md#scope`, `plan.md`'s out-of-scope preamble, `future-considerations.md`).

The H1 is the sole site that drops the hyphen. The inconsistency is purely orthographic — it does not affect anchors, links, or any normative content — but it is a visible blemish on the page title of a topic page.

## Issue introduction
**Verdict:** single-commit
**Introducing commits:** f5e89f4 ("docs: enforce 100KB hard / 30KB recommended size cap on spec set", 2026-06-04)
**History:** `git log -L 1,1:` on the file shows the H1 line has exactly one revision — its creation. The file was created in f5e89f4 when the spec set was split under a size cap, and the heading was written `# Model changes and non goals` (unhyphenated) at that moment. `git log -S'Model changes and non goals' --all` returns only f5e89f4, confirming the unhyphenated string never existed in any earlier parent file. The defect has been present since the file's inception; it was not introduced by a later edit drifting away from a once-correct heading.

## Solution Space

**Shape:** single

### Recommendation

Change the H1 to `# Model changes and non-goals`, matching the filename, the page's own `## loom 1.0 non-goals` heading and anchors, and corpus-wide usage. No anchors or inbound links target the H1 text, so the edit is confined to line 1 with no follow-on repointing.

## Consequence

**Severity:** cosmetic

A reader sees an inconsistently-spelled page title; the hyphenated anchors, links, and body prose are unaffected, so no implementer behaviour or cross-reference resolution diverges.

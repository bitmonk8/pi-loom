## Problem

Rule 4 of *Conversation drive — subagent mode* prescribes a loom-constructed `ResourceLoader` adapter whose payload is deliberately degenerate: `getExtensions()` / `getSkills()` / `getPrompts()` / `getThemes()` / `getAgentsFiles()` return empties, `extendResources` and `reload` are no-ops, and only `getSystemPrompt()` carries load-bearing content. The subagent spawn therefore depends on `createAgentSession` and every downstream Pi consumer of `ResourceLoader` tolerating this all-empty / no-op loader — not rejecting empty resource sets, not throwing on a no-op `reload`, and not silently falling back to a parent-`cwd` discovery scan — a behavioural property the type declaration does not encode. The *Editorial-review checklist for unpinned host presuppositions* item (o) re-validates only the `ResourceLoader` member *shape*, not this consumption posture; no bump-time hook covers it, unlike the (o)/(aj) split for `SessionManager.inMemory` and the (n)/(am) split for `tools`. A future Pi minor that preserved every member signature but tightened empty-loader tolerance would pass item (o) and break every subagent spawn at runtime with no bump-time signal.

## Issue introduction

**Verdict:** present-since-inception

**Introducing commits:**

- `6b3e9ba` (2026-05-07) "pi-loom spec: resolve \"Subagent state-isolation matrix asserts non-inheritance without naming the Pi mechanism for two of its six rows\"" — first appearance of rule 4 with the explicit enumeration of the empty/no-op members (`getExtensions`, `getSkills`, `getPrompts`, `getThemes`, `getAgentsFiles` returning empties; `extendResources`, `reload` as no-ops). The behavioural-tolerance presupposition is implicit in this text from its first commit and was never surfaced as a named presupposition with an audit hook.
- `5d55c12` (2026-06-05) "pi-loom spec: resolve \"Subagent-spawn satellite types ResourceLoader and SessionManager.inMemory not pinned\"" — added the *Spawn-block satellite type pins* paragraph and routed `ResourceLoader` to bump-checklist item (o), but only at the *member shape* level; the behavioural-tolerance facet was not split out.
- `51b6e75` (2026-06-05) "pi-loom spec: resolve \"createAgentSession spawn options elide required fields\"" — promoted the prose from four rules to six and made the empty/no-op member set explicit in the code block, leaving the implicit-tolerance assumption load-bearing without flagging it.

**History:** Rule 4 enumerated the degenerate `ResourceLoader` adapter at its first introduction (`6b3e9ba`), and every subsequent edit on the spawn block (`5d55c12` pinning the satellite types; `51b6e75` codifying the code-block shape; `077afb5` introducing item (am) for the `tools`-allowlist behavioural sub-properties; `9ced9f1` introducing item (aj) for `SessionManager.inMemory` no-persistence behaviour) added bump-time audit hooks for sibling presuppositions while leaving the `ResourceLoader` behavioural-tolerance facet implicit. The defect is the absence of a parallel split for `ResourceLoader`, not a regression from any single commit.

## Solution approach

Add a named behavioural-tolerance consumption-posture paragraph with a stable anchor on `subagent.md`, adjacent to rule 4 and the *Spawn-block satellite type pins* paragraph (`#subagent-spawn-satellite-types`), stating that the rule-4 adapter relies on `createAgentSession` and every downstream Pi consumer of `ResourceLoader` to tolerate the degenerate payload — the empty collection getters and no-op `extendResources` / `reload` — driving the spawned `AgentSession` to a normally-initialised state without parent-`cwd` discovery fallback and without surfacing the empties as a load-time error. Add a complementary bump-checklist item under *Editorial-review checklist for unpinned host presuppositions* in `version-bump-step2.md` that forward-links the new anchor and audits the candidate Pi minor for empty-tolerance, no-discovery-fallback, and no-op-callback behaviour, modelled on the sibling (aj) and (am) complement-style items.

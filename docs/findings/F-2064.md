## Problem

GOV-23 (anchor `gov-23` on `docs/spec_topics/governance/anchor-scheme-and-retired.md`) opens with a link `[`spec.md`](../../spec.md)` and pins anchor-scheme-stability MUSTs over the Session Model `sm-N-…` anchor cluster. That URL resolves to `docs/spec.md` — the hub/contents file, whose body is a three-item table of contents with no Session Model section and no `sm-N` anchors. The anchors GOV-23 governs actually live on `docs/spec_topics/session-model-and-appendix.md` (the umbrella `#session-model` anchor and `sm-1-…` … `sm-8-…`, including the `sm-3-…` / `sm-7-…` sub-letter anchors). A maintainer following GOV-23's link to apply its MUSTs lands on the hub and finds no anchors to govern, leaving the rule's binding target unresolvable from its own text; sibling rules GOV-15/GOV-30/GOV-31 point the link URL at the real owning topic page while keeping `spec.md` in the link text.

## Issue introduction
**Verdict:** single-commit
**Introducing commit:** `f5e89f4` — "docs: enforce 100KB hard / 30KB recommended size cap on spec set" (2026-06-04)
**History:** Before `f5e89f4`, the Session Model section and the `sm-1 … sm-8` anchors lived directly on `docs/spec.md` (confirmed: `git show f5e89f4^:docs/spec.md` carries `## Session Model`, `<a id="session-model">`, and `<a id="sm-1-single-active-session-binding">`), and GOV-23 (then in `docs/spec_topics/governance.md`) linked `` [`spec.md`](../spec.md) `` — a correct target at that time. `f5e89f4` split the oversized spec files into the index+children layout, relocating the Session Model section and its anchors off the `spec.md` hub onto `session-model-and-appendix.md`. The commit's map-driven cross-reference rewrite ("1571 cross-references repointed") did not repoint GOV-23's `spec.md` link to the new owner page; the commit message itself records "2 dangling refs remain (target anchors absent in corpus, need author intent)". The defect is thus a relocation the cross-reference rewrite missed at the split, not a flaw present since the rule's inception.

## Solution approach

Rewrite GOV-23's opening link in `docs/spec_topics/governance/anchor-scheme-and-retired.md` to point at the page that owns the anchors: change the URL from `../../spec.md` to `../session-model-and-appendix.md#session-model`, keeping `spec.md` in the link text per the resolving convention GOV-15/GOV-30/GOV-31 already follow.

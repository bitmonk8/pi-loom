## Problem

The step-5 inbound-reference sweep in `version-bump-triggers.md` greps the fixed superset `src/`/`test/`/`docs/` and justifies that choice only against future subdirectories and `docs/` siblings. It never asserts that all runtime/source/test/doc references must live under those three top-level roots. A new top-level root holding such a reference is therefore silently elided by the sweep on the next bump, with no enumeration delta reported.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** 6798428 — resolve "Step 3 engines.node test describes mutually exclusive right-hand operands" (2026-06-05, Thomas Andersen); c0259ca — resolve "Version-bump procedure: four obligations have no verifiable acceptance criterion" (2026-06-07, Thomas Andersen); e785f77 — resolve "Version-bump procedure six clarity/scope gaps (T061)" (2026-06-07, Thomas Andersen); 2ecf650 — resolve "Re-validating provider-owned behaviour has no enumerated data source or owner" (2026-06-05, Thomas Andersen)
**History:** Each of the four preconditions is an omission baked into its section when that section was authored, not a regression a later commit introduced into previously-correct text. The three-way `engines.node` equality was reworked at `6798428` without ever adding an absent-field arm; the `src/`/`test/`/`docs/` grep superset and its self-justification were authored under the acceptance-criteria pass at `c0259ca` without stating a top-level-root invariant; the step-2(b) reviewer-discretion tie-breaks (`e785f77`) presupposed a bump-commit review gate that no companion commit ever established; and the "maintained corpus" re-validation language (`2ecf650`) named a maintenance owner but never an initial-population owner or existence precondition. Subsequent targeted edits to each file touched adjacent concerns but never closed these gaps.

## Solution approach

Clarify the *Enumeration mechanism — grep target* clause of step 5 in `version-bump-triggers.md` so the grep-root set is closed under new top-level roots. Either state the invariant that all loom runtime/source/test/doc references live under `src/`, `test/`, or `docs/` (so introducing a new top-level root holding such a reference must extend the grep-root set in the same edit), or redefine the grep root as repo-root-relative minus a named excluded set so a new top-level root is covered by construction. Apply the chosen treatment to both the contributor-side recipe and the optional post-1.0 audit-pass description.

## Finding

`V18b` is titled "Inventory-closure audit gate" and the spec names the gate **inventory-closure audit** (`inventory-audit-intro.md` §"Inventory-closure audit", anchor `#sdk-cap-inventory-closure-audit`). The bump checklist step that consumes it (`version-bump-step2b.md`) likewise calls it the "inventory-closure audit". The spec reserves the phrase **surface-set closure** for the *property* the gate enforces — "every Pi-side surface reference … resolves to an `SDK_SURFACE_INVENTORY` entry" (`inventory-audit-intro.md` §8).

The opening sentence of `V18b`'s **Adds** field splices the property name into the gate name: "The build-time **surface-set-closure audit** over the audited source tree". This is the only occurrence of "surface-set-closure audit" anywhere in the plan corpus; every other reference — the leaf's own title, `plan.md`, and the spec — uses "inventory-closure audit". A reader cross-referencing the gate by name finds a label that appears in no other leaf and matches no spec anchor.

The fix is confined to that one Adds sentence; the property phrase "surface-set closure" is correct wherever the leaf names the property rather than the gate.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** c6a664e ("pi-loom plan: build/update plan for spec.md + review", 2026-06-10)
**History:** `docs/plan_topics/V18b-inventory-audit.md` was created in c6a664e, and its Adds field opened with "The build-time surface-set-closure audit over the audited source tree" in that first revision (`git show c6a664e:…` confirms the literal phrase present at creation). The later commit 402bdbe expanded the Adds (land-green sweep / taxonomy) but did not introduce or remove the conflated gate name; `git log -S "surface-set-closure audit"` reports a count change only at c6a664e. The defect therefore entered with the leaf itself, not via a later edit.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V18b-inventory-audit.md`, in the **Adds** field's opening sentence, replace "surface-set-closure audit" with "inventory-closure audit", matching the leaf title and the spec-canonical gate name at `inventory-audit-intro.md` §"Inventory-closure audit" (anchor `#sdk-cap-inventory-closure-audit`). Where the leaf names the *property* the gate enforces rather than the gate itself, keep "surface-set closure" (the spec's property name, `inventory-audit-intro.md` §8). No other field changes; the title, Spec citations, Tests, and Ships-when already use the canonical name or generic shorthand.

## Consequence

**Severity:** cosmetic

A reader resolving the gate by name from `V18b` Adds encounters a term ("surface-set-closure audit") that occurs nowhere else and matches no spec anchor, costing a cross-reference hop to confirm it denotes the same gate as the leaf title. No implementer divergence results — the title and spec carry the canonical name — so the defect is wording only.

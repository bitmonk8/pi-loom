## Problem

Step 6's trailing anti-regression clause — "do not silently regress a provider for which pi-ai has added seed support" — is the step's substantive obligation yet carries no normative modal, leaving its conformance weight undefined. It sits directly beside a "MUST consult …" obligation, so two reasonable implementers can read the same clause as either binding or merely advisory. The anti-regression requirement is therefore unverifiable.

## Solution approach

Rewrite step 6 ("Re-validate the provider seed-field table")'s trailing clause with an explicit normative modal so its conformance weight is unambiguous and consistent with the adjacent "MUST consult …" obligation. Select the modal level (MUST NOT vs SHOULD NOT) the anti-regression obligation is intended to carry.

## Problem

`V17c`/`V17c-T` assert that a cancellation checkpoint fires immediately before each of five concrete interpreter sites — each loop iteration, each `@`-query dispatch, each tool call, each `invoke`, and the slash-command argument-binder's LLM call — plus a loop-iteration macrotask-yield property, yet their `Deps` pull in none of the site-introducing leaves (`V3c`, `V13c`, `V14a`, `V15a`, `V9j`). Both leaves therefore become pickable before any of those sites exists, so the `V17c-T` suite reds out for a non-behavioural reason (the site constructs do not parse/run), violating the §Per-phase TDD ritual "fail red for the intended reason" gate. The obvious repair — adding the five site leaves to `V17c.Deps` — closes a `V17c ↔ V4f` dependency cycle for the `@`-query/tool-call/`invoke` sites, because `V4f` declares `V17c` in its `Deps` and `V13c`/`V14a`/`V15a` each declare `V4f`.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `5b95143`
**History:** The `V17c` leaf was created by commit `5b95143` (2026-06-18, "pi-loom plan: resolve \"V17a bundles forwarding contract and checkpoint-site wiring\""), which peeled the checkpoint-granularity surface off `V17a`. `git show 5b95143:docs/plan_topics/V17c-checkpoint-granularity.md` shows the `Deps` line `` `V17c-T`, `V17a`, `V8a`, `H4a` `` present verbatim in the file's first commit, and `git log -L` on that line confirms it has never been edited since. The split moved a surface that asserts the real interpreter's five concrete site kinds into a new leaf without expanding its `Deps` to the site-introducing leaves (`V3c`/`V13c`/`V14a`/`V15a`/`V9j`), so the omission is present from the leaf's inception rather than introduced by a later edit. The subsequent commit `79f756b` (2026-06-24, "add cka-47 coverage row for V17c §Granularity") added the coverage-matrix row and token citations only; it did not touch the `Deps`. The `V4f → V17c` dependency that makes the naive dep-addition fix cyclic for the `@`-query/tool-call/`invoke` sites is an existing edge that coexists with this split.

## Solution approach

Add the two cycle-free site leaves — `V3c` (the loop site plus its macrotask-yield arm) and `V9j` (the binder LLM-call site, satisfied transitively via `V11f`) — to `V17c.Deps` and `V17c-T.Deps`. For the three cyclic sites (`V13c` `@`-query, `V14a` tool-call, `V15a` `invoke`), distribute their per-site checkpoint-presence assertions onto those feature leaves rather than depending on them from `V17c`, since each already lands after `V17c` via the `V4f` chain — adding a checkpoint-before-site `Tests` bullet to each and the `V8a` seam to the `Deps` that lack it. Add a V17 interleave note to `docs/plan.md` mirroring the existing slice notes, expand the `cka-47` closing-leaf cell to the multi-leaf facet set, and keep `H5b.Deps` transitively complete over the newly coverage-producing leaves.

## Solution constraints

- The plan dependency DAG must stay acyclic — do not add `V13c`/`V14a`/`V15a` to `V17c.Deps` (that closes the `V17c ↔ V4f` cycle); leave `V4f`'s existing `V17c` dependency intact.
- The single-program cross-site closed-set assertion ("exactly five sites, at no other node kinds") is owned by the sibling closed-set finding — coordinate on how that negative is bounded; do not unilaterally re-author it here.
- The spec is read-only for this fix.

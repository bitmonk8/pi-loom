## Finding

`V16a`'s `Adds.` carries an explicit carve-out: its cross-ceiling arbitration function is a Class-2 named cross-leaf seam that "is imported and *consulted* directly at each call site, never constructor-injected," and the carve-out warns that the No-globals/statics/singletons rule "must not be misread as a directive to wire this stateless pure function as an injected interface or a module-level singleton." The authoritative cross-cutting rule it points at — `conventions.md` §"No globals, statics, singletons" — carries no such exception. Its headline reads absolutely: "All collaborators passed by constructor."

The carve-out is correct: a stateless pure function holds no cross-invocation mutable state, so it is not a global/static/singleton, and the release-time residue inspection's own Clean-when condition (checklist item 1) already qualifies the rule to "every *stateful* collaborator is constructor-injected." But that qualification lives only in the residue-inspection prose; the rule's headline does not state it, and the one place the exemption is spelled out for the seam shape is `V16a`'s `Adds.` — which is descriptive (Class-3 shape guidance) and therefore non-binding even for `V16a` itself.

The result is a placement defect: the convention that ought to own the exemption does not state it, and the only statement of it sits in a single leaf's non-binding prose, invisible to a reviewer applying the residue-inspection's item-1 sweep to `V16a`'s directly-imported arbitration function and invisible to any future analogous stateless pure cross-leaf seam.

## Issue introduction

**Verdict:** traced-to-commit
**Introducing commit:** `6826ca0` ("pi-loom plan: resolve \"V16a arbitration seam reuses spec's reserved seam noun\"", 2026-06-17)
**History:** `conventions.md`'s No-globals rule has carried the absolute "collaborators passed by constructor" wording since before the `docs/` restructure (`31ff060`) and has never carried a stateless-pure-seam carve-out (`git log -S "stateless pure" -- docs/plan_topics/conventions.md` is empty). `V16a` was created in `c6a664e` (2026-06-10); the "never constructor-injected" / "must not be misread" carve-out prose was authored into its `Adds.` in `6826ca0` (2026-06-17) while resolving an earlier seam-noun finding. The placement defect entered at `6826ca0`: the carve-out was added to the leaf without a parallel edit to the convention that owns the rule it qualifies.

## Solution Space

**Shape:** single

### Recommendation

Add a carve-out clause to `conventions.md` §"No globals, statics, singletons" stating that a Class-2 named cross-leaf seam that is a stateless pure function — one that holds no cross-invocation mutable state — is consulted directly at its call site, is not a "collaborator" requiring constructor injection, and is not a global/static/singleton; the "all collaborators passed by constructor" requirement governs stateful collaborators. Tie the clause to the existing Class-2 seam definition in the `Adds.` field bullet and to the residue-inspection item-1 Clean-when condition ("every *stateful* collaborator is constructor-injected"), so the rule headline and the Clean-when condition agree.

Suggested literal insertion into the No-globals rule body (exact wording at the fixer's discretion): "A **Class-2 named cross-leaf seam** that is a stateless pure function — holding no cross-invocation mutable state — is consulted directly at its call site and is neither a constructor-injected collaborator nor a global/static/singleton; the constructor-injection requirement governs stateful collaborators only (mirroring the release-time residue inspection's item-1 *Clean when* condition)."

The spec is read-only for this fix: the spec's PIC-10…PIC-16 host DI seams are construction-time–injected and are a different category; do not touch any `spec_topics/**` page.

Once the convention states the exemption, `V16a`'s inline carve-out may be reduced to a back-reference to the convention rather than restating it (option-dependent; not required to resolve the finding).

## Consequence

**Severity:** advisory

A reviewer running the release-time residue inspection's item-1 singleton sweep over `src/**` has no convention-level basis to exempt `V16a`'s directly-imported arbitration function, and a future leaf introducing a similar stateless pure cross-leaf seam reads only the absolute "all collaborators passed by constructor" headline with no documented exception. Implementers can still ship a working seam (the Clean-when condition already permits stateless forms), so this is a guidance gap rather than a blocker, but it invites a reviewer to flag a clean construct or an implementer to over-apply constructor injection.

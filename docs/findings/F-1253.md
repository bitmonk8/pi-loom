## Problem

`leaf-template.md` is the copy-source for authoring every new leaf, and its `**Spec.**` field states the closure obligation as a single-level rule: if a listed page cites a normative rule on another page, that other page is also listed (excluding narrative cross-links). It never re-applies the obligation to the newly-listed page's own cross-links. The authoritative rule in `conventions.md` §Leaf format is stronger — closure "applies transitively to a fixed point" — so the template and the convention describe two different obligations for the same read-restriction-bearing field. An author who copies the template and reads only it closes the field one hop and stops, silently omitting any normative page reachable only transitively, with no mechanical gate to catch it.

## Issue introduction

**Verdict:** present-since-inception (of the current template scaffold)
**Introducing commit:** `657ee76` (2026-05-25) "pi-loom plan: reset to scaffold + template; clear spec-debt register; remove forensic spec-review docs"
**History:** `conventions.md` has carried "applies transitively to a fixed point" since `31ff060` (2026-05-08), predating the template scaffold. When `657ee76` (re)authored `leaf-template.md`'s `**Spec.**` field, it stated only the single-level form ("that other page is also listed") — so the template was divergent from the pre-existing convention from the moment it was written. A later commit `7cc0c2a` (2026-06-17) "resolve 'Spec-field closure rule uses undefined cross-link terms'" edited this exact template sentence to add the narrative-cross-link carve-out in both files but left the transitivity gap unaddressed — a missed-correction window, not the introduction.

## Solution approach

Rewrite the `**Spec.**` field's closure sentence in `leaf-template.md` so it states transitive closure to a fixed point, matching `conventions.md` §Leaf format, and have it cross-reference that section. Keep the existing narrative-cross-link exclusion intact.

## Solution constraints

- `conventions.md` is the authoritative source and is read-only for this fix; align the template to it, not the reverse.

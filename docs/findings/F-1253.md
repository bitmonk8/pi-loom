## Finding

`leaf-template.md` is the copy-source for authoring every new leaf, and it states the
`Spec.`-field closure obligation as a single-level rule: "The field MUST be closed under
normative cross-link: if a listed page cites a normative rule on another page, that other
page is also listed — unless the cross-link is *narrative* …". It never states that the
obligation re-applies to the newly-added page's own cross-links, i.e. that closure must
continue transitively to a fixed point.

The authoritative rule in `conventions.md` §Leaf format is stronger: "closure runs from each
listed topic outbound, applies transitively to a fixed point, and excludes narrative
cross-links". The template and the convention therefore describe two different obligations
for the same field. A leaf author who copies the template and reads only it (the template
explicitly invites copying and the `Spec.` field explicitly says "An implementer MAY restrict
their reading to these pages") performs one pass of cross-link expansion and stops, omitting
any second- or third-hop normative page reachable only transitively.

Because the `Spec.` field is read-restriction-bearing — an implementer may legitimately
confine reading to the listed pages — an under-closed field means a transitively-reachable
normative rule can be silently missed at implementation time. No mechanical gate in the
plan corpus enforces transitive closure of the `Spec.` field; the closure is authored and
reviewed by hand against whichever phrasing the author consulted.

## Issue introduction

**Verdict:** present-since-inception (of the current template scaffold)
**Introducing commit:** `657ee76` (2026-05-25) "pi-loom plan: reset to scaffold + template; clear spec-debt register; remove forensic spec-review docs"
**History:** `conventions.md` has carried "applies transitively to a fixed point" since `31ff060` (2026-05-08), predating the template scaffold. When `657ee76` (re)authored `leaf-template.md`'s `**Spec.**` field, it stated only the single-level form ("that other page is also listed") — so the template was divergent from the pre-existing convention from the moment it was written. A later commit `7cc0c2a` (2026-06-17) "resolve 'Spec-field closure rule uses undefined cross-link terms'" edited this exact template sentence to add the narrative-cross-link carve-out in both files but left the transitivity gap unaddressed — a missed-correction window, not the introduction.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/leaf-template.md`, amend the `**Spec.**` field's closure sentence so it
states transitive closure to a fixed point, matching `conventions.md` §Leaf format. Change the
clause "if a listed page cites a normative rule on another page, that other page is also listed"
so it also requires re-applying the rule to each newly-listed page's own normative cross-links
until no new page is added — for example by replacing it with: "if a listed page cites a
normative rule on another page, that other page is also listed; the rule then re-applies to
that page's own normative cross-links, transitively to a fixed point (see
[`conventions.md`](./conventions.md) §Leaf format)". Keep the existing narrative-cross-link
exclusion intact. `conventions.md` is the authoritative source and is read-only for this fix.

## Consequence

**Severity:** correctness

A leaf author working from the template alone closes the `Spec.` field one hop and stops,
omitting transitively-reachable normative pages. Two reasonable authors then diverge (one
transitively closes, one does not), and an implementer who honours the "MAY restrict reading
to these pages" licence on an under-closed field can miss a normative rule reachable only at
the second hop — producing a leaf that does not match the spec, with no mechanical gate to
catch it.

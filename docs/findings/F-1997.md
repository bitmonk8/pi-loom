## Finding

The phrase "inside the YAML block scalar" scopes the escape rule to one specific YAML scalar type (block scalars `|` / `>`), where `\` is a literal character and passes through unchanged. The spec does not state whether `system:` must be a block scalar, nor what happens when `system:` is authored as a YAML double-quoted scalar. In YAML double-quoted scalars, `\` is the YAML escape introducer; `\$` is not a recognized YAML 1.2 escape sequence, so its behavior (error, or literal `\$`) is implementation-defined by the YAML parser layer — before the loom parser ever sees the string.

## Problem

The `V17b` and `V17b-T` **Tests.** bullets close their edge-case clause with "the `session_shutdown` teardown-iteration path is governed by its own sub-step-2 swallow rule and is out of scope for this steady-state trap." The phrase "its own sub-step-2 swallow rule" carries no anchor, and the rule it defers to lives in `patch-skew-degradation.md` (the per-entry `try`/`catch` in the `session_shutdown` teardown sequence at sub-step 2) — a file not cited anywhere in either leaf. The leaf already uses a resolvable citation form for the adjacent sub-step elsewhere in the plan (`V9g`, `V9g-T`, `coverage-matrix.md` all link `patch-skew-degradation.md` §`session_shutdown` sub-step 3), so the un-anchored sub-step-2 reference is an inconsistency against an established in-plan convention.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** 0bc84dd — pi-loom plan: resolve "Forwarding-listener throw-trap contract has no asserting test in V17a" (2026-06-11, Thomas Andersen)
**History:** The un-anchored phrase ("governed by its own sub-step-2 swallow rule … out of scope for this steady-state trap") entered the corpus in 0bc84dd, where a plan-review *suggested fix* containing that exact wording was materialised verbatim into the V17a cancellation leaf's Tests bullet. The over-bundling-split commit cb8737a (2026-06-12) then peeled the throw-trap into the dedicated `V17b`/`V17b-T` leaves, carrying the same un-anchored wording forward; the opaque reference was present from the first authoring and was copied, not newly created, by the split.

## Solution approach

In the **Tests.** bullet edge-case clause of both `V17b` and `V17b-T`, replace the un-anchored "is governed by its own sub-step-2 swallow rule" with a resolvable cross-reference to `patch-skew-degradation.md` §`session_shutdown` sub-step 2, mirroring the existing sub-step-3 citation form used in `V9g`/`V9g-T`. The clause's existing "out of scope for this steady-state trap" framing stays; only the reference is being made resolvable.

## Solution constraints

- The spec is read-only for this fix; the sub-step-3 citation form links to the file without a fragment anchor, so make the reference resolvable from the plan side without editing `patch-skew-degradation.md`.

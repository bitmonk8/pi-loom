## Problem

Step 5's build-time audit gate (the *Enumeration mechanism — grep target* paragraph) fails `npm test` on a result-delta against a documented inbound-site enumeration and applies a documented-enumeration exclusion list, but both the expected-enumeration set and the exclusion set exist only as English prose in this step and in step 1's carve-out. The spec names no machine-readable artefact that holds them, no key by which the audit matches each grep hit against an entry, and no mapping from the prose listings to that artefact — the exclusion entry for the `docs/spec.md` SM-2 restatement is identified only by prose location. Two reasonable implementers therefore diverge into opposite failure modes: a line-keyed encoding goes perpetually red on unrelated edits to a cited file, while a path-only encoding silently authorises new inbound references added to a file already on the list. The symmetric `(quit|reload)` complement sweep inherits whatever artefact shape step 5 picks but is never told that it does.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** c0259ca — pi-loom spec: resolve "Version-bump procedure: four obligations have no verifiable acceptance criterion" (2026-06-07, Thomas Andersen)
**History:** Prior to `c0259ca` the build-time audit was opt-in
(`The implementation MAY additionally implement a build-time audit pass
over the same three grep targets so a result-delta against the
documented enumeration fails npm test rather than relying on
contributor-side discipline; PIC does not require the audit`) and the
exclusion list was not normatively named. `c0259ca` promoted the
audit from `MAY` to `MUST`, added the explicit `npm test` red, and
introduced the "audit MUST apply the same documented-enumeration
exclusion list the contributor-side recipe applies" sentence — but
without defining where the documented enumeration and exclusion list
live as an artefact, what they store, or how the audit matches a grep
hit against them. The gap is wholly produced by that single
`MAY → MUST` escalation: the previous MAY framing tolerated the
under-specification because PIC owned no artefact contract; once the
gate became mandatory the missing artefact contract became
load-bearing.

## Solution approach

In the *Enumeration mechanism — grep target* paragraph, immediately after the build-time audit gate sentence, clarify the machine-readable artefact the audit consumes: its storage site, the key by which the audit matches each grep hit against an entry, and how the prose baseline enumeration and exclusion listing reduce to that artefact. Pin a content-anchored match key rather than a raw line number, so the SM-2 exclusion is matched by content/anchor and survives line drift in `docs/spec.md`. Extend the same artefact and match key to the symmetric `(quit|reload)` complement sweep so that gate carries an explicit contract rather than a silence.

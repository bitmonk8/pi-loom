## Problem

The `query/` subtree owns the registered `QRY` REQ-ID prefix (per the GOV-1 prefix table) but carries zero coined `QRY-N` anchors, while holding several individually-citable RFC-2119 normative obligations: the empty-template short-circuit `MUST NOT` in `query-forms.md` *Degenerate rendered templates*; the paired token-count nullability MUSTs in `query-failure-and-repair.md` *loom 1.0 seam — pre-flight token-count nullability*; the template-verbatim `MUST` in *Follow-up turn templates (normative)*; and the fenced-body-verbatim `MUST` in `query-tool-loop.md` *Initial forced respond turn template (normative)*. Because none carry a `QRY-N` defining anchor, GOV-9's `#prefix-n` cross-link form is unsatisfiable for them and citers must fall back to fragile section anchors or the whole-subtree `cka-12` bucket. GOV-22 catalogues this as progressively-draining residue; `QRY` is the largest active-prefix residue in the corpus, carrying zero anchors at all.

## Issue introduction

**Verdict:** present-since-prefix-coinage (multi-commit residue under GOV-22's progressive-drain regime)
**Introducing commits:** `c1e5d64` (2026-05-05, *spec: REQ-ID infrastructure + Phase 12a heading promotion on five named pages (Decision #12, partial)*) registered the `query.md` → `QRY` row in the prefix table without coining any `QRY-N` anchor; every subsequent commit that edited the QRY-prefixed pages — notably `124b31c` (2026-05-05, *Coercion follow-up user turn lacks a normative template per methodology*) which introduced the `Renderers MUST emit` obligation, `02d7ca5` (2026-05-08, *T02 surface-extension seam obligations not consistently marked*) which introduced the pre-flight-token nullability MUSTs, `708a42f` (2026-05-06, *coercion overloaded …*) which introduced the empty-template `MUST NOT trigger respond-repair` obligation, `bbe2602` (2026-06-05, *Initial forced respond turn — instruction wording status undeclared*) which introduced the forced-respond `MUST emit fenced body` obligation, and `1d773d6` (2026-06-04, *GOV-3/GOV-6 subdirectory scan …*) which rebased the prefix row from `query.md` to `query/` for the split — left the prefix uncoined.
**History:** GOV-22 itself was coined in `4f588e0` (2026-06-03, *Non-narrative pages with zero coined REQ-IDs non-referenceable*); from that commit onward the un-coined `QRY` set has been the named "standing spec defects, draining progressively" the rule contemplates. No commit in the history has ever coined `QRY-1` (`git log -G 'QRY-[0-9]' -- docs/` returns only review/plan-store commits, never a spec-corpus addition). The defect is therefore best characterised as a *multi-commit-interaction* residue: each commit that added a QRY-prefixed MUST is individually defensible under GOV-22's per-commit trigger (because each added a *new* defining obligation site and so was bound to coin an anchor for *that* site alone, not the whole set), but the cumulative result is a prefix with no anchors at all.

## Solution approach

Coin `QRY-N` dual-form anchors (per GOV-1 *Dual-form layout*, numeric tails allocated per GOV-3 under the already-registered `QRY` prefix) at the `query/` subtree's defining obligation sites: the empty-template `MUST NOT` in `query-forms.md` *Degenerate rendered templates*, the token-count nullability MUSTs in `query-failure-and-repair.md` *loom 1.0 seam — pre-flight token-count nullability*, the template-verbatim `MUST` in *Follow-up turn templates (normative)*, and the fenced-body-verbatim `MUST` in `query-tool-loop.md` *Initial forced respond turn template (normative)*. Migrate each newly-anchored obligation from the `cka-12` code-keyed row to the *Numbered REQ-IDs* table in `coverage-matrix.md` in the same change. Repoint any corpus cross-link that referenced an affected section anchor to the more-specific `#qry-n` where one now exists.

## Solution constraints

- Out of scope: the dedent / newline-trim normative table and the schema-inference algorithm — they are not *defining obligation sites* under GOV-22 (no co-located RFC-2119 modal token at the obligation's defining site).

## Problem

The four per-site swallowing-handler leaves — `V9o`, `V13f`, `V14f`, `V15h` and their `-T` partners — carry a byte-identical "Swallowing-handler attachment at this site" Tests bullet asserting that the abandonable Promise attaches its swallowing handler at the construction site before the first microtask boundary, but the only test vector the bullet and the impl-leaf Ships-when describe lands a late settlement via the `Checkpoint` seam after cancellation has surfaced. A late-settlement-after-cancellation vector only proves a handler is attached by the time the late settlement lands; it does not witness construction-site attachment before the first microtask boundary, so a build that attaches the handler lazily (the exact failure mode the spec MUST guards against) passes all three side channels green yet violates the spec. The bullet's clause "a build that bypasses the substrate reddens this leaf's tests" is unfalsifiable because the suppression assertions are outcome-only — a build that bypasses the `Checkpoint`-seam substrate but reproduces identical three-channel suppression is not detected. In `V15h`/`V15h-T` the bare phrase "the substrate" is ambiguous between the two substrates named in the leaf's Adds.

## Issue introduction

**Verdict:** introduced by a single commit
**Introducing commits:** `75a9bcd` (2026-06-11, "pi-loom plan: resolve \"V17a substrate-level suppression test claims four-site coverage\"")
**History:** The over-claiming bullet — the construction-site / first-microtask attachment assertion, the single late-settlement-only test vector, and the "a build that bypasses the substrate reddens this leaf's tests" clause — entered the corpus in `75a9bcd`, the plan-review fix that distributed `V17a`'s four-site swallowing-handler coverage onto the then-host leaves `V14a`/`V13c`/`V15a`/`V9i` (`git log -S 'bypasses the substrate'` / `-S 'before the first microtask boundary'` both bottom out at `75a9bcd`; no earlier commit contains either string). The wording was subsequently propagated verbatim into the dedicated per-site leaves when they were carved out — `V13f`/`V14f`/`V9o` via `6d7661d` (2026-06-17) and `V15h` via `323d361` (2026-06-18) — with no change to the asserted property. The defect is thus a single-origin wording choice, not an interaction between independent edits; the later split commits only relocated it.

## Solution approach

In the "Swallowing-handler attachment at this site" Tests bullet of `V9o`, `V13f`, `V14f`, `V15h` and their `-T` partners — and the matching impl-leaf Ships-when — add an early-rejection test vector alongside the existing late-settlement vector that rejects the abandonable Promise within the first microtask boundary and asserts no `unhandledRejection` fires, so a lazily-attaching build reddens. Drop the unfalsifiable clause "a build that bypasses the substrate reddens this leaf's tests" from the Tests bullet, the impl-leaf Ships-when, and the `V15h` Adds in all eight leaves. In `V15h`/`V15h-T`, replace the bare phrase "the substrate" with its explicit referent, the `Checkpoint`-seam swallowing-handler substrate.

## Solution constraints

- The early-rejection vector must land the rejection before the first microtask boundary with no intervening `await` or other attach point — an intervening `await` creates its own attach point and defeats the witness.
- Out of scope: `docs/spec_topics/cancellation.md` is read-only for this fix, which edits only the eight named plan leaves.

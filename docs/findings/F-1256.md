## Problem

`V11c` and `V11c-T` share a final `Tests` bullet that bundles two facts ŌĆö the relaxed-copy / `maxLength:500` budget and the bypass path skipping the LLM call ŌĆö but carries no verification predicate (no `BNDR-N` REQ-ID, no `Convention:` citation), unlike the `BNDR-1`/`BNDR-2`/`BNDR-3` bullets above it. The bypassŌåÆskip-binder-call behaviour is a normative spec property (`binder-bypass-and-envelope.md` ┬¦"Binder bypass": the no-params and single-string cases skip the binder call entirely), yet it appears nowhere in the plan except this un-asserted line: `V11c`'s `Adds` names "the binder bypass path" without stating that the bypass skips the inference call. Meanwhile `V11c`'s `Ships when` claims `npm test` validates the bypass path, so the leaf can be marked done with no observable assertion that the bypass actually skips the call.

## Solution approach

Add the bypass-path skip semantics to `V11c`'s `Adds` so the descriptive fact has an authoritative home, citing `binder-bypass-and-envelope.md` ┬¦"Binder bypass" as the source of the no-params / single-string bypass behaviour. In the `Tests` block of both `V11c` and `V11c-T`, rewrite the final non-asserting bullet so it asserts the observable bypass consequence (a single-string-param or no-params loom binds without issuing a binder inference call), or ŌĆö if no bypass assertion is intended for this leaf ŌĆö delete the bullet and remove "and the bypass path" from `V11c`'s `Ships when`. Do not restate the relaxed-copy / `maxLength:500` fact, which `Adds` and `BNDR-1`/`BNDR-2`/`BNDR-3` already cover.

## Solution constraints

- Apply the `Tests`-block change symmetrically to both `V11c` and `V11c-T` (paired-leaf convention; `conventions.md`).
- The spec page `binder-bypass-and-envelope.md` is read-only for this fix.

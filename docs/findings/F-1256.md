## Finding

`V11c` and `V11c-T` share a final `Tests` bullet that reads: *"The envelope is a relaxed copy (`maxLength:500` as a model budget, not a user cap); the bypass path skips the LLM call."* This bullet carries no verification predicate — no `BNDR-N` REQ-ID and no `Convention:` citation — unlike the three bullets above it (`BNDR-1`/`BNDR-2`/`BNDR-3`). It is a descriptive statement of implementation behaviour, not an assertion of what a test checks, so it does not belong in a `Tests` block whose every other entry names what is asserted.

Of the two facts the bullet bundles, the relaxed-copy / `maxLength:500` part is already stated in `V11c`'s `Adds` ("the relaxed copy with `maxLength:500`"), but the bypass-path LLM-skip behaviour is stated nowhere except this bullet. `V11c`'s `Adds` names "the binder bypass path" without saying the bypass *skips the binder inference call*; `V11c-T`'s `Adds` is the standard "Failing tests for the paired `V11c` implementation leaf" boilerplate. The skip semantics — a real spec behaviour (`binder-bypass-and-envelope.md` §"Binder bypass": the no-params and single-string cases "skip the binder call entirely") — therefore has no descriptive home in the plan and is only visible as an un-asserted line item.

This is compounded by `V11c`'s `Ships when` ("`npm test` validates the three-arm envelope and the bypass path"), which claims the bypass path is validated while the only bullet touching it asserts nothing.

The defect entered with the leaves themselves; no later edit introduced or relocated it.

## Solution Space

**Shape:** single

### Recommendation

State the bypass-path skip semantics in `V11c`'s `Adds` so the descriptive fact has an authoritative home: amend the existing `Adds` sentence so it records that the bypass path skips the binder inference call (e.g. extend "The binder bypass path … consumed by the inference call." to note that on the bypass cases the runtime sets the param value directly and issues no binder/LLM call). Cite `binder-bypass-and-envelope.md` §"Binder bypass" as the source of the no-params / single-string bypass behaviour.

In the `Tests` block of both `V11c` and `V11c-T` (paired-leaf convention — apply the same change in each), drop the descriptive prose from the final bullet. Since `V11c`'s `Ships when` already claims the bypass path is validated, replace the non-asserting line with a bullet that asserts the observable bypass consequence — that a single-string-param (or no-params) loom binds without any binder inference call being issued — rather than restating the relaxed-copy fact already covered by `BNDR-1`/`BNDR-2`/`BNDR-3` and `Adds`. If no bypass assertion is intended for this leaf, instead delete the redundant bullet and remove "and the bypass path" from `Ships when` so the leaf does not claim coverage it does not provide.

The `maxLength:500` relaxed-copy clause is already in `Adds`; do not duplicate it back into `Tests`.

## Consequence

**Severity:** advisory

The bypass→skip-binder/LLM-call behaviour, a normative spec property, has no descriptive statement in either leaf and survives only as a non-asserting `Tests` line. An implementer working from the leaves sees the property only as an un-verified aside, while `V11c`'s `Ships when` claims the bypass path is validated; the leaf can be marked done without an observable assertion that the bypass actually skips the inference call.

## Finding

The `BNDR-2` bullet in both `V11c-bypass-envelope.md` and `V11c-T-bypass-envelope.md` reads:
`` `BNDR-2`: `ambiguous.candidates` stays in the schema (`array<string>|null`) though it is not surfaced in 1.0. ``
The operative assertion this bullet owns is schema retention — that `ambiguous.candidates` remains in the envelope schema with type `array<string>|null`. The trailing clause "though it is not surfaced in 1.0" describes a separate property (loom 1.0 must not render `candidates` on the user-facing system note) that this bullet does not test.

That non-surfacing property is a real loom 1.0 normative requirement (spec `binder-bypass-and-envelope.md` BNDR-2: "The loom 1.0 runtime MUST NOT surface it on the user-facing system note"), but it is owned and tested elsewhere: `V11e` (system-note rendering) rule 5 asserts "the `ambiguous` failure row renders only the model's `message` and never surfaces `candidates` in loom 1.0", and that same rule-5 bullet explicitly back-references that "the envelope-schema retention of `candidates` is asserted by `V11c`'s `BNDR-2`." The division of responsibility is already clean: `V11c` BNDR-2 = schema retention; `V11e` rule 5 = non-surfacing.

The trailing clause is therefore a redundant version-scope aside in the BNDR-2 bullet. Removing it loses no coverage (non-surfacing stays asserted by `V11e` rule 5) and leaves BNDR-2 as a clean schema-retention assertion matching its `V11e` back-reference.

## Solution Space

**Shape:** single

### Recommendation

In both `docs/plan_topics/V11c-bypass-envelope.md` and `docs/plan_topics/V11c-T-bypass-envelope.md`, strike "though it is not surfaced in 1.0" from the `BNDR-2` Tests bullet, leaving:
`` `BNDR-2`: `ambiguous.candidates` stays in the schema (`array<string>|null`). ``
The loom 1.0 non-surfacing requirement remains owned and tested by `V11e`'s system-note rule-5 bullet (which already names `V11c` BNDR-2 as the owner of schema retention), so no coverage is lost.

## Consequence

**Severity:** cosmetic

The schema-retention test BNDR-2 specifies is unaffected and no implementer diverges; the aside is accurate, just out of place. Worst case, a reader misjudges the scope of the BNDR-2 assertion or treats the field as conditional, but the non-surfacing requirement is independently owned and tested by `V11e` rule 5.

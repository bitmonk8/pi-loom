## Problem

In the `node_modules/` root (item 3) paragraph of `docs/spec_topics/discovery/package-and-settings.md`, the trailing pnpm sentence states version-dependent pnpm facts as present-tense, version-independent prose: the *isolated* `node-linker` default, the `node_modules/.pnpm/…` virtual-store path, and the *hoisted* opt-out mode. Each of these is a configurable, version-dependent pnpm detail, so the sentence reads as a normative claim about pnpm's internal layout. A future pnpm release that changes the default linker or moves the virtual store would make the spec text wrong without changing what the extension does — the implementer-relevant rule is solely the `lstat`/`isSymbolicLink()` immediate-child filter, which is independent of pnpm's internals.

**History:** The `node_modules/` (root #3) entry was a one-line bullet until commit 27e914a, which closed the prior review item "node_modules walk skips pnpm-isolated entries" by appending the symlink-filter rule plus the pnpm-specific explanation in a single sentence. The robust mechanism (`lstat` no-follow → `isSymbolicLink()` filter) and the version-dependent pnpm prose ("default isolated node-linker", "`node_modules/.pnpm/…`", "hoisted node-linker mode") landed together in that commit; no later edit on this file has touched the paragraph since.

## Solution approach

Rewrite the trailing pnpm sentence in the root-#3 `node_modules/` paragraph so the version-dependent pnpm details (default linker mode, virtual-store path, hoisted opt-out) are framed as a non-normative example rather than a normative assertion. Leave the `lstat`/`isSymbolicLink()` filter mechanism as the sole normative content, and state the recourse generically (install loom packages via `pi install`, or configure the package manager to materialise real directories under `node_modules/`).

## Solution constraints

- The `lstat`/`isSymbolicLink()` immediate-child filter (per [PIC-13](../pi-integration-contract/host-interfaces-services.md#pic-13)) stays normative — the demotion applies only to the pnpm-specific prose, not to the classification mechanism the test gate enforces.

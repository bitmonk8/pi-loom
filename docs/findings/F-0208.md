## Problem

The closing sentence of `conventions.md` §"Cross-cutting rules (every leaf)" asserts an unconditional guarantee — "no rule-subject behaviour falls outside the gated tree" — that actually depends on `extensions/index.ts` remaining a pure re-export shim. That file sits outside `src/**`, and every mechanical gate (the `H2a` no-globals test, the `no-broad-catch` and `no-restricted-syntax` lint rules, the `H3a` ambient-primitive scan, and the `H5a` / `H5c` allow-list reconciliation) reads `src/**` and no other tree, so none inspects the shim. No `H4a` test and no item in the loom 1.0 release-time residue inspection reads it either. The claim is presented as a mechanically-closed guarantee but rests on unobserved contributor discipline; an edit adding a broad catch, ambient read, global, or `Promise` combinator to the shim would silently falsify it.

## Issue introduction

**Verdict:** single-commit-introduction
**Introducing commits:** `2f18db6` (`pi-loom plan: resolve "reconcile src/** production-code root with extensions/index.ts entry point"`, 2026-06-16)
**History:** The plan corpus is git-tracked. `git log -S` over `conventions.md` shows the entire scope paragraph — the `thin delegating entry shim` description and the closing "no rule-subject behaviour falls outside the gated tree" guarantee — was added (not modified) in commit `2f18db6`. `git log -S "thin delegating entry shim"` over `H4a-factory-shell-and-harness.md` returns the same single commit. The overclaim was therefore born with the reconciliation fix itself: that fix introduced the `src/**`-scope reconciliation and the entry-shim concept together, asserting the completeness guarantee on a prose-only property without backing it with a check or disclosing it as discipline. The durable fix addresses what that reconciliation commit established, not a later regression.

## Solution approach

Add a checklist item to the loom 1.0 release-time residue inspection in `conventions.md` (anchor `release-time-residue-inspection`) that reads `extensions/index.ts` and confirms it remains a pure re-export carrying no broad `catch`, no global / static / singleton, no ambient-primitive read, and no `Promise` combinator of its own. Rewrite the final sentence of the §"Cross-cutting rules (every leaf)" scope paragraph so it cites that new residue-inspection item as the surface that verifies entry-shim purity, rather than asserting unconditional mechanical coverage the `src/**`-scoped gates do not provide. Cross-reference the new item from the `H4a` leaf so the leaf that authors the shim points at the surface that audits it.

## Solution constraints

- The new residue-inspection item must name `extensions/index.ts` explicitly as its read target, since that file falls outside the inspection's default `src/**` scope and would otherwise not be covered.
- The spec (including `host-prerequisites.md`) is read-only for this fix; resolve it entirely within `conventions.md` and the `H4a` leaf.

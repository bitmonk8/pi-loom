## Problem

`V2e`'s **Adds** claims the leaf delivers the inbound/outbound wire-name translation boundary, explicitly naming outbound wire-name JSON as part of its deliverable, and the spec (`runtime-value-model.md` §Wire-name translation) backs a two-directional boundary with distinct *Inbound* and *Outbound* translation. But the single Tests bullet in `V2e` (mirrored in `V2e-T`) asserts only the inbound direction; no assertion owned by `V2e`/`V2e-T` exercises outbound translation. An outbound-rename regression — emitting the loom-side field name instead of the declared `field as "Wire"` wire name, or emitting a tagged enum object instead of the bare wire string — would turn no test red within this leaf despite the **Adds** claim. Because `V2e` closes a code-keyed area (`cka-2`, RVM) rather than a numbered REQ-ID, no coverage-matrix row flags the missing direction; the leaf's own Tests list is the only gate.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `c6a664e` ("pi-loom plan: build/update plan for spec.md + review", 2026-06-10) authored the original `V2c` leaf with an Adds claim covering the "inbound/outbound wire-name translation boundary … outbound wire-name JSON" but a single Tests bullet asserting only the inbound direction. `d4f6e5c` ("split V2c value model from wire-name translation boundary", 2026-06-17) split that boundary out into the new `V2e`/`V2e-T` leaves, copying the Adds claim and the inbound-only Tests bullet verbatim.
**History:** `git log -S` confirms the inbound-only bullet and the inbound/outbound Adds claim first co-existed in `c6a664e` on the pre-split `V2c` leaf; the asymmetry was never corrected. The `d4f6e5c` split carried the gap through unchanged rather than introducing it — the defect is original to the plan's first authored form of this boundary.

## Solution approach

Add an outbound-direction Tests assertion to the `V2e`/`V2e-T` pair, authored red-first in `V2e-T` with the matching bullet added to `V2e`, citing `runtime-value-model.md` §Wire-name translation (*Outbound*). The assertion exercises that a loom-side value carrying a `field as "Wire"` rename serialises outbound to JSON under its wire name (not the loom-side name) and that an enum value emits its bare wire string. Extend `V2e`'s **Ships when** so it names the outbound vector alongside the existing inbound one.

## Solution constraints

- `V2e`/`V2e-T` own the outbound assertion; if it is instead routed to a downstream construction-site leaf (tool input, query response payload, or `invoke` arguments), that leaf must be named explicitly in `V2e`'s **Adds** and must carry the assertion — do not leave the outbound claim in `V2e`'s **Adds** with the assertion owned nowhere.

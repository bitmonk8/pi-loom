## Problem

The *Retired REQ-IDs* table in `docs/spec_topics/governance/anchor-scheme-and-retired.md` (anchor `id="retired-req-ids"`) closes with a normative value-space statement: the `Retired in` column carries either a 7-character abbreviated commit SHA or a release tag. The `GOV-12` and `GOV-14` rows instead carry the literal token `pending`, which is neither. Both retirements are substantively complete (`GOV-12` split into `GOV-30`/`GOV-31`; `GOV-14` demoted to an editorial note), so the record contradicts its own stated column contract and omits the commit anchor it promises for two finished retirements.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** fe8aa5c — pi-loom spec: resolve "GOV-12 bundles ≥3 independently-testable sub-obligations" (2026-06-05, Thomas Andersen); 5a772e3 — pi-loom spec: resolve "GOV-14 two SHOULD-NOTs admit no mechanical witness" (2026-06-05, Thomas Andersen)
**History:** A prior review pass (commit `3f00256`, 2026-06-05) had explicitly closed off placeholder values in the *Retired in* column by replacing every `<retirement commit>` / `<demotion commit>` token with a concrete abbreviated SHA and by deleting the parenthetical that had previously authorised such placeholders. Later the same day, `fe8aa5c` appended the GOV-12 retirement row with `` `pending` `` in `Retired in`, and `5a772e3` appended the GOV-14 row using the same token; neither commit re-opened the column-definition paragraph (still pinning SHA-or-tag), so the contradiction was introduced by the interaction of the column-grammar tightening in `3f00256` and the two retirement-row appends in `fe8aa5c` / `5a772e3`.

## Solution approach

Rewrite the `pending` cell in the `GOV-12` row and the `pending` cell in the `GOV-14` row of the *Retired REQ-IDs* table with a value conformant to the column's stated value space — the abbreviated commit SHA of the commit that performed that row's retirement, or a loom 1.x release tag. Match the SHA form the other retired rows already use.

## Solution constraints

- Out of scope: adding a standing gate against future placeholder reintroduction; backfill only the two named cells.

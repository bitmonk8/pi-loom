## Problem

In `V9c` and `V9c-T`, the error-detection Tests bullet that maps a `stopReason: "error"` driven turn to a `transport` `QueryError` attributes the `provider` value to "V9j's provider-error-mapping surface." That attribution is wrong: `V9j` owns the pi-ai `complete()` provider-error classifier, which is never invoked on the prompt-mode path â€” that path reads the driven turn's trailing `assistant` message off the `ReadonlySessionManager` surface after `waitForIdle()` resolves. The spec already pins the source: `conversation-drive.md` populates `provider` from the loom's resolved `model:` regardless of detecting surface, and `queryerror-variants.md` defines `TransportError.provider` as the resolved `Model<Api>.api` value. An implementer following `V9c-T` to source `provider` would look to a classifier with no prompt-mode entrypoint and either thread a non-existent read or invent a divergent source.

## Issue introduction

**Verdict:** introduced-by-specific-edit
**Introducing commits:** `10cbf96` ("pi-loom plan: resolve \"V9c prompt-mode transport-failure detection asserted by no test\"", 2026-06-16)
**History:** `git log -S"sourced from V9j's provider-error-mapping surface"` reports the clause entering both `docs/plan_topics/V9c-conversation-drive.md` and `docs/plan_topics/V9c-T-conversation-drive.md` in `10cbf96` and nowhere earlier. That commit was itself a plan-fix resolving a prior finding that prompt-mode transport-failure detection was asserted by no test; the fix added the `stopReason: "error"` error-detection Tests bullet and, in authoring it, attributed `provider` to `V9j`'s classifier. The bullet did not exist before `10cbf96`. The `V9j` entry in `V9c`'s `Deps` line predates this commit (present since the initial plan build `c6a664e`) and is independent of the mis-attribution.

## Solution approach

Rewrite the trailing `provider`-source clause in the `stopReason: "error"` error-detection Tests bullet of both `V9c` and `V9c-T` so it cites the loom's resolved `model:` `.api` value (`Model<Api>.api`) â€” the frontmatter `model:`, or the inherited `ctx.model` when `model:` is absent (resolution owned by `V6a`) â€” rather than `V9j`'s classifier, matching `conversation-drive.md`'s resolved-`model:` rule and `queryerror-variants.md`'s `TransportError.provider` derivation. Apply the identical correction to both leaves so `V9c` and `V9c-T` stay symmetric.

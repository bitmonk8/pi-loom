## Problem

The `tool-call round slot accounting` glossary entry carries normative ceiling-#2 accounting law: the per-round slot-increment bookkeeping, its role as ceiling #2's boundary-check input, the over-cap `tool_loop_exhausted` return behaviour, and the forced-respond-turn exempt-routing exclusion. That routing and predicate content belongs on the canonical query/ceiling pages, not the glossary, whose charter declares each entry a descriptive one-paragraph reminder with the canonical page as tie-break authority. A duplicated routing rule that later changes on only one side leaves the glossary asserting stale law.

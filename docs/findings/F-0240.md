## Finding

Six leaves carry a Tests (or paired tests) bullet of the form
`[return.md — final-value contract](../spec_topics/return.md) (RET code-keyed area): …`.
`return.md` has no "final-value contract" section — it contains only `# Return Statement`, which defines the `return` *statement* (short-circuit exit, type-check, bare-`return`-in-`void`, unreachable-code). The normative *final value* definition the bullets actually assert against is **Final value (language definition)** in [`functions.md`](../spec_topics/functions.md), anchored at `#final-value-language-definition`. That anchor already existed when these leaves were authored, so the citation has pointed at the wrong file since the bullets were written.

The fail/cancel half of each bullet ("…and is absent on fail/cancel") is also mis-attributed: the success/fail/cancelled trichotomy and what the `Err` envelope carries per outcome are owned by [`errors-and-results/error-model.md`](../spec_topics/errors-and-results/error-model.md) at `#terminal-outcomes`, which `functions.md` itself defers to. `return.md` owns neither half.

The original framing names only `V3d` and `V3d-T`; the same wrong citation in fact appears in six leaves: the `V3d` pair, the `V9i` (subagent) pair, and the `V15a` (invocation) pair. The bullet text spells out the asserted behaviour inline, so the misdirection is a provenance/clarity defect rather than a missing rule — but an implementer who follows the link to locate the normative source lands on a page that does not contain it.

The defect is thus an inception error in the `V3d` pair replicated to its callers by a later refinement.

## Solution Space

**Shape:** single

### Recommendation

Retarget the wrong citation in each of the six leaves' Tests (or paired tests) bullet from `return.md` to the rules' actual owners, leaving the inline assertion prose unchanged.

In `docs/plan_topics/V3d-functions-and-return.md` and `docs/plan_topics/V3d-T-functions-and-return.md`, the bullet:

```
- [return.md — final-value contract](../spec_topics/return.md) (RET code-keyed area): the function body's produced value is its final value at the function-result seam on success and is absent on fail/cancel.
```

becomes a bullet citing `[functions.md — final value (language definition)](../spec_topics/functions.md#final-value-language-definition)` for the success/produced-value half and `[error-model.md — terminal outcomes](../spec_topics/errors-and-results/error-model.md#terminal-outcomes)` for the "absent on fail/cancel" half, preserving the existing assertion sentence.

In `docs/plan_topics/V9i-subagent-isolation.md`, `docs/plan_topics/V9i-T-subagent-isolation.md`, `docs/plan_topics/V15a-invocation-core.md`, and `docs/plan_topics/V15a-T-invocation-core.md`, apply the same link retarget to their `[return.md — final-value contract](../spec_topics/return.md)` bullet; the caller-facing wording ("propagates to the `invoke`/subagent caller …") stays as-is.

The relative path from `docs/plan_topics/` to the anchors is `../spec_topics/functions.md#final-value-language-definition` and `../spec_topics/errors-and-results/error-model.md#terminal-outcomes`. The `V3d` **Adds** field's descriptive phrase "the final-value contract" refers to the same rule and may stay; if reworded for accuracy it should read "the final-value language definition".

The spec is read-only for this fix: do not add a "final-value" section to `return.md`. The definition is single-owned by `functions.md#final-value-language-definition`, and the fail/cancel trichotomy by `error-model.md#terminal-outcomes`; the fix moves the plan's pointer to those owners rather than relocating the definition.

## Consequence

**Severity:** advisory

An implementer following the citation to locate the normative final-value rule lands on `return.md`, which has no such section, and must hunt for the real owner (`functions.md#final-value-language-definition`) or risk re-deriving the contract from the inline bullet alone. The assertion text is spelled out inline, so a working leaf is still producible; the cost is misdirection and a degraded provenance trail across six leaves.

## Problem

The un-anchored-MUST recogniser in `H5a` (and the identical predicate in `conventions.md` §REQ-ID discipline) classifies a `spec_topics/**` page as non-narrative — and thus in scope for the `MUST`/`MUST NOT` token scan — "iff its row in the REQ-ID prefix table carries a cell that is not the byte-exact `(no IDs — narrative)` literal." This two-valued test presupposes the page has a row and has no defined branch for a `spec_topics/**` page with no prefix-table row at all. No governance rule makes the prefix tables exhaustive over `spec_topics/**` (GOV-6 only requires every prefix appearing in a REQ-ID to be in the table), so un-rowed pages exist; a normative MUST/MUST-NOT on one rides to the loom 1.0 release gate unverified by both the mechanical recogniser and the §release-time residue inspection item 5 sweep, which shares the same row-keyed predicate.

## Issue introduction

**Verdict:** introduced-by-commit
**Introducing commits:** `3df06a7` — "pi-loom plan: resolve \"un-anchored-MUST recogniser narrative classification source\"" (2026-06-11)
**History:** The un-anchored-MUST recogniser was first introduced in `0603eb4` ("un-anchored normative MUSTs invisible to closing gate"), where its page scope was described loosely as "non-narrative `spec_topics/**` pages" with no operational classification. `git log -S` and `git show 3df06a7` confirm that commit `3df06a7` — resolving a prior review finding about the recogniser's narrative-classification source — replaced that loose wording in both `H5a` (Adds) and `conventions.md` (§REQ-ID discipline) with the row-keyed predicate "non-narrative iff its row in the prefix table carries a cell that is not the byte-exact `(no IDs — narrative)` literal." That refinement is what tied classification to "its row" and thereby imported the row-existence assumption; the two-valued predicate has no branch for a page with no row. The defect entered as a side effect of tightening an unrelated classification source, not at the recogniser's inception.

## Solution approach

Add an explicit third branch to the recogniser so that a non-hub-stub `spec_topics/**` page absent from the prefix table reddens the gate, stated in both `H5a` Adds and `conventions.md` §REQ-ID discipline; `H6a`'s live-corpus footing inherits this with no separate edit. Align `conventions.md` residue-inspection item 5's page scope so an un-rowed page is swept (or surfaced as a residue defect) rather than implicitly excluded. Scope the new branch to exclude GOV-24 hub stubs (subtree TOC files that deliberately carry no row and no anchors). The spec-side root-cause fix — a page-completeness invariant alongside GOV-6 — is out of scope here.

- The plan-diff fixer must not edit `docs/spec.md` or `docs/spec_topics/**`; those are spec-owned and read-only for this fix, so the spec-side page-completeness invariant is deferred to the spec owner.

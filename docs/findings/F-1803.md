## Problem

In `docs/spec_topics/related-work.md`, the *Direct influences* section's `mech` bullet links `[backlot](../backlot)` and names the inline path `backlot/docs/MECH_SPEC.md`. Resolved relative to this document's location (`docs/spec_topics/`), `../backlot` points at `docs/backlot`, which does not exist — the sibling `backlot` checkout lives above the repository root. The Markdown link is therefore dead and the inline `backlot/docs/MECH_SPEC.md` code-span names a real file at a path not resolvable from this document. The defect is confined to this non-normative appendix; the only observable effect is broken reader navigation.

## Issue introduction

**Verdict:** introduced-by-relocation (single introducing commit)
**Introducing commits:** `fecb504` ("Split spec.md and plan.md into per-topic / per-phase files")
**History:** The `[backlot](../backlot)` link was authored in `53f5831` (2026-05-03) inside the monolithic repo-root `spec.md`; from the repo root, `../backlot` correctly resolved to the sibling checkout, so the link was sound at inception. `fecb504` split `spec.md` into `spec_topics/related-work.md` — one directory deep — without deepening the relative path, breaking resolution. `31ff060` (2026-05-08) later moved the topic tree under `docs/`, widening the gap to two levels, but the link was already dead as of `fecb504`. The inline `backlot/docs/MECH_SPEC.md` code-span entered with the same `53f5831` authoring and was likewise never repointed.

## Solution approach

Rewrite the *Direct influences* `mech` bullet's `backlot` reference so it no longer carries the dead `../backlot` relative link or the unresolvable inline `backlot/docs/MECH_SPEC.md` intra-tree path. Name the `mech` engine and its `backlot` project as plain prose; if a canonical public URL for `backlot`/`MECH_SPEC.md` exists, an absolute URL may be used instead.

## Problem

In `docs/spec_topics/related-work.md`, the *Direct influences* section's `mech` bullet links `[backlot](../backlot)`. Resolved relative to `docs/spec_topics/`, `../backlot` points at `docs/backlot`, which does not exist; the real sibling `backlot` project lives above the repository root, so the link is off by two directory levels and dead as written. The accompanying code-span `backlot/docs/MECH_SPEC.md` likewise names a real file (present at `../../../backlot/docs/MECH_SPEC.md`) at a path not resolvable from this document. The appendix is non-load-bearing, so the defect is confined to reader navigation within it.

## Issue introduction
**Verdict:** single-commit
**Introducing commits:** fecb504 — Split spec.md and plan.md into per-topic / per-phase files (2026-05-04, Thomas Andersen)
**History:** The `[backlot](../backlot)` link was authored in `53f5831` (2026-05-03) inside the monolithic repo-root `spec.md`; from the repo root `../backlot` correctly resolved to the sibling checkout, so the link was sound at inception. `fecb504` split `spec.md` into `spec_topics/related-work.md` (one directory deep) without deepening the relative path, breaking resolution. A later relocation `31ff060` (2026-05-08) moved the topic tree under `docs/`, widening the gap to two levels, but the link was already broken at `fecb504`.

## Solution approach

Repoint the `mech` bullet's Markdown link target from `../backlot` to `../../../backlot`, and correct the parenthetical code-span path `backlot/docs/MECH_SPEC.md` to `../../../backlot/docs/MECH_SPEC.md`, so both resolve to the sibling `backlot` checkout from `docs/spec_topics/related-work.md`. If the sibling-checkout layout is not a guaranteed expectation, substituting an absolute/public URL for the link target is an acceptable alternative.

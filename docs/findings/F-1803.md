## Finding

In `docs/spec_topics/related-work.md`, the *Direct influences* section's `mech` bullet links `[backlot](../backlot)` and names the inline path `backlot/docs/MECH_SPEC.md`. Resolved relative to the document's location (`docs/spec_topics/`), `../backlot` points at `docs/backlot`, which does not exist; the actual sibling `backlot` checkout lives above the repository root (`../../../backlot` from this file). The Markdown link is therefore dead as written, and the inline `backlot/docs/MECH_SPEC.md` code-span names a real file at a path not resolvable from this document.

The defect is confined to a non-normative appendix â€” the file itself states "nothing in the rest of the spec depends on it" â€” so the only observable effect is broken reader navigation within the appendix. The original framing ("non-existent sibling directory") is imprecise: the `backlot` project does exist as a sibling of the repo root; the link is simply off by directory levels after the spec was relocated. The substance of the finding is unaffected.

## Issue introduction

**Verdict:** introduced-by-relocation (single introducing commit)
**Introducing commits:** `fecb504` ("Split spec.md and plan.md into per-topic / per-phase files")
**History:** The `[backlot](../backlot)` link was authored in `53f5831` (2026-05-03) inside the monolithic repo-root `spec.md`; from the repo root, `../backlot` correctly resolved to the sibling checkout, so the link was sound at inception. `fecb504` split `spec.md` into `spec_topics/related-work.md` â€” one directory deep â€” without deepening the relative path, breaking resolution. `31ff060` (2026-05-08) later moved the topic tree under `docs/`, widening the gap to two levels, but the link was already dead as of `fecb504`. The inline `backlot/docs/MECH_SPEC.md` code-span entered with the same `53f5831` authoring and was likewise never repointed.

## Solution Space

### Recommendation

In `docs/spec_topics/related-work.md`, edit the *Direct influences* `mech` bullet to remove the broken intra-tree reference to the sibling `backlot` checkout. Drop both the Markdown hyperlink `[backlot](../backlot)` and the inline code-span path `backlot/docs/MECH_SPEC.md`, replacing the phrase `the sibling [backlot](../backlot) project (`backlot/docs/MECH_SPEC.md`)` with plain prose that names the `mech` engine in the `backlot` project â€” no Markdown link and no intra-tree path. If a canonical public URL for `backlot`/`MECH_SPEC.md` exists, use that absolute URL as the link instead; otherwise leave it as plain text. Remove both the hyperlink and the inline code-span path, not just one.

## Consequence

**Severity:** cosmetic

A reader following the `mech`/`backlot` reference in the Related Work appendix hits a broken link. The appendix is non-normative and nothing in the spec corpus or plan depends on it, so no implementer behaviour diverges.

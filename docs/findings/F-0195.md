## Finding

The *No silent test skipping* convention in `docs/plan_topics/conventions.md` reads: "Vitest's `assert.fail` / `expect.unreachable` when prerequisites are missing — never silent `return` early. A skipped test is a lie: it reports success when nothing was verified." It enumerates exactly one prohibited escape hatch — the early `return` — and stops there.

The governing CLAUDE.md *No silent test skipping* directive bans two forms: "not early `return` or skip macros." The convention bullet reproduces the first half and drops the second. Vitest's skip family (`test.skip()` / `describe.skip()` / `it.skip()`, and the adjacent `.todo` / `.skipIf` / `.runIf` modifiers) is the more common way a test silently elides verification, yet it is never named at the rule site.

This rule carries no mechanical gate — `H2a-cross-cutting-gates.md` lists *No silent test skipping* among the conventions but installs no lint or architectural test for it, so it is contributor discipline verified only at the per-phase TDD-ritual self-review step. An implementer consulting `conventions.md` as the authoritative rule statement could reasonably conclude that `test.skip()` is permitted because it is "not a silent `return` early," directly contradicting CLAUDE.md.

A later revision (3edff0f) replaced `panic` with Vitest's `expect.unreachable` to fix the assertion-API naming but preserved the same single-form phrasing, so the skip-macro omission has been present since inception and was never corrected.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/conventions.md`, extend the *No silent test skipping* bullet so it names the Vitest skip macros alongside the existing early-`return` prohibition, bringing it into alignment with CLAUDE.md's "early `return` or skip macros." Replace the sentence

`Vitest's `assert.fail` / `expect.unreachable` when prerequisites are missing — never silent `return` early.`

with text that prohibits both forms, e.g.

`Vitest's `assert.fail` / `expect.unreachable` when prerequisites are missing — never a silent `return` early and never a skip macro (`test.skip` / `describe.skip` / `it.skip`, or the `.todo` / `.skipIf` / `.runIf` modifiers).`

The edit is confined to this one bullet; `H2a-cross-cutting-gates.md` references the rule by name only and needs no change. The fix is internal to the plan conventions file — the spec is read-only for this finding.

## Consequence

**Severity:** advisory

The rule is contributor discipline with no mechanical gate, so an implementer who treats `conventions.md` as the authoritative statement can use `test.skip()` / `it.skip()` / `describe.skip()` believing they comply, leaving an obligation unverified while the suite reports green. Working leaves remain producible, so the gap is a guidance defect rather than a blocker.

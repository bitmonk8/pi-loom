## Finding

The §Release gate paragraph in `docs/plan.md` says `H6a` "sequences immediately after the canary (its **Deps** name `H5b`)". `H6a`'s declared **Deps** are `H5b, H7a`. The narrative names only the `H5b` edge and omits the `H7a` edge, so the stated prerequisite set is incomplete.

The `H7a` edge is load-bearing: `H6a`'s **Release-gate acceptance (manual real-host smoke)** item drives `H7a`'s committed multi-feature fixture `.loom` and checks emitted codes against the committed permitted-code list checked in alongside it. `H6a`'s own leaf body states this explicitly ("That `H7a` edge is pinned directly here … because `H7a` closes no new spec REQ-ID and so is not a member of `H5b`'s coverage-producing set"). The leaf's **Deps** are therefore correct and authoritative; only the plan-level narrative summary disagrees with them.

## Finding

`V9i` and `V9i-T` carry an `ERR-8` "delegated live-carrier witness" Tests bullet that asserts *a mid-stream cancellation driven inside the real subagent `AgentSession` does not mutate the subagent session's committed turns*, and `V9i`'s **Ships when** hardens this into a `npm test` gate (*"`npm test` … asserts the `ERR-8` live-surface outcome — a mid-stream cancellation inside the real subagent `AgentSession` leaves its committed turns unmutated"*). The oracle presupposes the test can read the private session's committed turns at mid-stream cancellation.

Two pinned spec facts contradict that presupposition. `error-model.md` `ERR-12` states subagent mode *"has no external observer of its conversation outside the subagent itself"*, and `ERR-8` makes the visibility of a partial fragment *"determined entirely by Pi's session semantics"* — the loom's obligation is non-mutation, not observation. Separately, `host-interfaces-core.md` §`AgentSession` (member surface loom touches) enumerates the load-bearing member set as exactly `abort()` / `dispose()` / `sendUserMessage()` / `subscribe()`, declares omitted members MUST-NOT-call (a GOV-18 arm (a) constraint), and states explicitly that the `AgentSession.messages` getter is **not** loom-load-bearing. There is therefore no loom-permitted seam to read the spawned session's committed turns; the runtime extracts text only from the terminal `agent_end` event payload, which a mid-stream cancellation never delivers. Compounding this, `PIC-9` disposes the private session in the per-invocation `finally`, so the session is gone before the parent observes the result.

As written, an implementer cannot satisfy the `npm test` Ships-when clause without either inventing an inspection handle the member surface forbids (reading `AgentSession.messages`) or reaching into a session that is disposed in `finally`. The non-mutation guarantee `ERR-8` actually demands is structurally delivered by the member-surface restriction (the runtime calls no mutating member) and is already witnessed offline by `V4c`'s `H4a`-double `ERR-12` coverage; the live-surface bullet over-states what the automated suite can observe.

## Solution Space

**Shape:** multiple

### Option A — Relocate the live-surface ERR-8 confirmation to the manual real-host smoke gate

**Approach.** Treat the live subagent-surface ERR-8 confirmation as a real-host smoke-gate concern rather than an automated `npm test` assertion. The automated ERR-12 non-mutation coverage remains on `V4c`'s `H4a` double; the live carrier becomes a manual real-host check.

**Plan edits.** In `V9i-subagent-isolation.md`, strike the `ERR-8` live-surface clause from **Ships when** (leaving the `dispose()`-in-`finally` idempotency gate). Reword the `ERR-8` Tests bullet in `V9i` and `V9i-T` to mark it as a manual real-host smoke-gate assertion, citing `real-host-smoke-gate.md` (and `H6a`) as the owning gate rather than `npm test`.

**Spec edits.** None.

**Pros.** Honest about `ERR-12` (no external observer) and the `AgentSession` MUST-NOT-call member surface; invents no inspection seam; the automated non-mutation guarantee is already delivered structurally and by `V4c`'s `H4a` double.

**Cons.** No automated regression of the live subagent surface; relies on the manual gate being run.

**Risks.** The manual smoke gate can be skipped pre-merge.

### Option B — Restate the witness against a loom-controlled observable surface

**Approach.** `ERR-8`'s normative obligation binds the loom *runtime* (MUST NOT truncate/rewrite/replace/remove committed surfaces), not the inspection of Pi's private session. Restate the `V9i`/`V9i-T` witness so its oracle observes the loom runtime issuing no mutation operation against the subagent session on the cancel-then-dispose path, observed at a loom-controlled seam, instead of reading the private session's committed turns.

**Plan edits.** Reword the `ERR-8` Tests bullet in `V9i` and `V9i-T` so the asserted outcome is the runtime's non-mutation behaviour at a loom-controlled seam; update `V9i` **Ships when** to the same surface. Name the seam that drives the mid-stream cancellation under `npm test` (couples with the streaming-source gap in the related finding below).

**Spec edits.** None.

**Pros.** Keeps automated coverage; aligns the oracle with `ERR-8`'s actual runtime obligation; no MUST-NOT-call violation.

**Cons.** Requires a loom-controlled cancellation/observation seam to exist offline; the witness no longer inspects committed turns directly (and is near-vacuous given the member surface already forbids mutating calls).

**Risks.** If no offline seam is available, the bullet remains unimplementable — a decision-dependency on the streaming-source finding.

### Recommendation

Take Option A: drop the `npm test` `ERR-8` live-surface clause from `V9i`'s **Ships when** and re-home the live carrier on the manual real-host smoke gate (`real-host-smoke-gate.md` / `H6a`). The non-mutation guarantee is already structurally enforced by the `AgentSession` member-surface restriction and witnessed offline by `V4c`'s `H4a` double, so the automated value of the live bullet is marginal while its observability claim is unsatisfiable under `npm test`. Whichever option is taken, reconcile the `ERR-12` delegated-wording in `V4c` and `V4c-T` (and the coverage-matrix `ERR-8 … ERR-12` co-witness annotation) so they no longer assert inspection of the private session's committed turns.

## Consequence

**Severity:** correctness

Two reasonable implementers diverge: one routes the assertion through `V4c`'s `H4a`-double ERR-12 coverage or a manual gate, the other reaches for `AgentSession.messages` and violates the MUST-NOT-call member-surface constraint. The `npm test` Ships-when clause names an outcome no loom-permitted seam can observe against the real, soon-disposed private session, so the leaf's done-condition cannot be honoured as written.

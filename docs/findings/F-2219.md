## Problem

The `sendSystemNote` best-effort fallback chain in `runtime-event-channel.md` states the behaviour of each recovery arm in running prose but carries no structured pass/fail acceptance-criteria block. The chain has two independent step-1 skip predicates (`display: false`, `content === ""`), a step-1-then-step-2 ordering with its own throw-tolerance, a `Clock.wallNow()`-throw `hint` path, the PIC-54 terminal-`console.error` swallow, and a "never aborts the slash-command handler" invariant. Peer PIC contracts (`session-shutdown-semantics.md`, `unknown-reason-rule.md`, and others in the cluster) pin comparably branchy behaviour with an explicit `## Acceptance criteria` section, so a conformance author here must reconstruct each fixture from prose, raising the risk that an arm goes unverified.

**Verdict:** present-since-inception
**Introducing commits:** 29c29b2 — pi-loom spec: resolve "pi.sendMessage is synchronous (void); spec treats it as returning a Promise" (2026-05-06, Thomas Andersen); f5e89f4 — docs: enforce 100KB hard / 30KB recommended size cap on spec set (2026-06-04, Thomas Andersen)
**History:** The best-effort fallback chain (the `ctx.ui.notify` step, the `display: false` skip, and the step-2 `loom/runtime/system-note-delivery-failed` diagnostic) first entered the corpus in 29c29b2 inside the pre-split `pi-integration-contract.md`, and migrated verbatim into `runtime-event-channel.md` when f5e89f4 split the oversized spec page. A pickaxe for `Acceptance criteria` across all history of this file returns nothing: the structured block has never existed for this section. Later commits (e.g. cfb3943 adding PIC-54) elaborated the chain's throw handling but never added acceptance criteria, so the omission is present since the chain was first written.

## Solution approach

Add a `## Acceptance criteria` block for the `sendSystemNote` fallback chain in `runtime-event-channel.md`, matching the structured form sibling PIC contracts use (`session-shutdown-semantics.md`, `unknown-reason-rule.md`). Make each behavioural arm falsifiable: the step-1 `display: false` and `content === ""` skips, the step-1-then-step-2 ordering, the continue-after-step-1-also-throws case, the no-retry / no-`Clock.wallNow()`-re-invocation rule, and the PIC-54 terminal-swallow invariant. Forward-link `#pic-54` for the terminal-`console.error` swallow rule rather than restating it.

## Problem

The `patch-skew-degradation-contract` page prescribes concrete test mechanics inside a normative contract page: the `(P-c-2)` build-time sub-fixture's `tsc`-as-harness split, and the spy/seam wiring on the injected `LoomRegistry`, `chokidar`-watcher, `Clock`, and signal-listener seams used to witness the full five-sub-step teardown. This over-constrains test infrastructure that the same page states "the implementation owns". An implementer reading the contract cannot tell which mechanics are binding obligations versus illustrative harness detail.

## Solution approach

## Problem

The final sentence of the **Deps.** field description in `docs/plan_topics/conventions.md` (§Leaf format) summarises the `H5b` obligation as "A new leaf may additionally need adding to [`H5b`]'s `Deps.`". The "may additionally need adding" phrasing is read-twice ambiguous: a reader can take it deontically (adding is optional / at the implementer's discretion) rather than epistemically (the leaf qualifies depending on the trigger). The authoritative clause it points to — the *Transitive-completeness plan-maintenance* clause in §REQ-ID discipline — is unambiguous: a qualifying leaf **MUST** be added to `H5b`'s `Deps.`. The summary thus softens a conditional MUST into a discretionary-sounding "may need", inviting an implementer who stops at the field description to treat the `H5b` update as optional.

## Issue introduction

**Verdict:** single-commit introduction
**Introducing commits:** `73b695d`
**History:** The "may additionally need adding to [`H5b`]'s `Deps.`" phrasing was introduced in `73b695d` (2026-06-20, "pi-loom spec-plan: fix F-0588 — Deps-field trigger summary defers to authoritative clause") in `docs/plan_topics/conventions.md`; `git log -S 'may additionally need adding'` reports that commit as the sole introducer. That commit's diff replaced a prior, more declarative summary — "A new leaf whose `Tests.` bullets carry an executable REQ-ID, a citing test that *closes* a coverage-matrix-mapped numbered REQ-ID, or an un-anchored normative MUST **is additionally added to** [`H5b`]'s `Deps.` per the *Transitive-completeness plan-maintenance* clause … which owns the exact triggers." — with the current looser sentence. The F-0588 fix intentionally stopped the summary from restating the exact triggers (the prior restatement listed "an executable REQ-ID" among them, mismatching the authoritative clause), correctly deferring to the owning clause; in doing so it swapped the declarative "is additionally added to" for the discretionary-sounding "may additionally need adding to", which is the ambiguity this finding records.

## Solution approach

Rewrite the final sentence of the **Deps.** field description in `conventions.md` so the `H5b` obligation reads as a conditional MUST — a leaf meeting the *Transitive-completeness plan-maintenance* triggers MUST additionally be added to `H5b`'s `Deps.` — while still deferring to that clause in §REQ-ID discipline for the exact triggers.

## Solution constraints

- Keep the deferral intact: the *Transitive-completeness plan-maintenance* clause in §REQ-ID discipline remains the single source of the exact triggers — do not re-enumerate the trigger conditions in the Deps-field summary.

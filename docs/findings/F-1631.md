## Finding

The `loom/load/cross-format-collision` row's *Trigger* column in
`docs/spec_topics/diagnostics/code-registry-load.md` closes with two clauses:

> The code name retains the historical `cross-format` token as a stable
> diagnostics-contract identifier per [Diagnostic shape — DIAG-3](diagnostic-shape.md#diag-3);
> its scope is the full same-priority slash-name collision rule, not only the
> cross-format arm.

The first clause is naming rationale — it explains *why* the code is still
spelled `cross-format` even though the diagnostic now covers same-format
collisions too. An implementer does not need that justification to emit or
route the code correctly; it is decision-log narration explaining a past
naming choice, embedded in a normative Trigger description.

The second clause ("its scope is the full same-priority slash-name collision
rule, not only the cross-format arm") *is* normative: it tells the implementer
the diagnostic's firing condition is broader than the code name suggests, which
is genuinely needed to avoid under-emitting on the same-format arm. That clause
must stay; only the rationale half is cruft.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `839db71` (2026-05-04, "Diagnostic codes are declared as namespaces but never assigned to specific errors") and `d124b8c` (2026-05-05, "spec: same-priority slash-name collisions are uniform load-time errors (Decision #10)").
**History:** `839db71` first introduced the `loom/load/cross-format-collision` row with a Trigger column that read "A `.loom` and a Pi prompt template (`.md`) or subagent share the same slash name." — code name and trigger were aligned. The next day, `d124b8c` broadened the rule to also cover same-format (`.loom` vs `.loom`) collisions at the same priority without renaming the code (its commit body explicitly notes "loom/load/cross-format-collision row broadened to cover both shapes; no new code introduced"). At that point the code name became a misnomer for the new same-format arm. A later partial fix, `7e440c9` (2026-06-06, "resolve `cross-format-collision misnomer / DISC-4 heading scope`"), renamed the DISC-4 heading to "Slash-name collision rules" and appended the DIAG-3 self-disclosure sentence to the registry row, but did not lift the disclosure to a visible callout or pin a loom 2.0 rename target. The defect's residual surface — disclosure prominence and missing rename intent — has held in equivalent form since that commit.

## Solution Space

**Shape:** single

### Recommendation

Delete the rationale clause from the *Trigger* cell:

> ~~The code name retains the historical `cross-format` token as a stable diagnostics-contract identifier per [Diagnostic shape — DIAG-3](diagnostic-shape.md#diag-3);~~

Retain the scope-clarification that follows it, re-capitalised as its own
sentence:

> Its scope is the full same-priority slash-name collision rule, not only the
> cross-format arm.

The on-wire contract that fixes the `cross-format` token is owned by DIAG-3 on
`diagnostic-shape.md`; the Trigger cell does not need to restate why the token
is stable.

## Consequence

**Severity:** cosmetic

A reader of the Trigger column encounters historical naming justification
inside a normative cell; nothing observable diverges and the firing condition is
unaffected. Removing the rationale tightens the cell without changing the
contract.

## Finding

Five sentences across `runtime-event-channel.md` and `glossary.md` describe spec-corpus editorial state or speculative forward work rather than constraints an implementer can act on. Each is locally redundant against authoritative content that lives elsewhere; none carries a normative obligation that would be lost on deletion.

1. **`runtime-event-channel.md` — `Aggregation, latency histograms, per-loom token reports, retention policies, and a consumer-facing read API for the event stream are deferred to Future Considerations.`** This deferral is already owned by `future-considerations/surface-extensions.md`'s "Richer runtime-event telemetry" bullet, which enumerates the same five items and links back to PIC. The inline sentence neither adds a new deferral nor pins a normative constraint; the second sentence in the same paragraph (`loom 1.0 ships the always-log set and the payload shape only; the channel is intentionally write-only at the spec level …`) is the only normative content in the paragraph and stays.
2. **`glossary.md` — `GOV-N (governance rule)` entry, parenthetical `(currently running through GOV-23, with some IDs retired)`.** The live ceiling is GOV-31 (the prefix table in `governance/req-id-prefix-table-active-b.md` registers GOV-25..GOV-31); the parenthetical is therefore incorrect on its own terms, and it directly contradicts the entry's next clause (`cite governance.md for the authoritative range rather than a fixed upper bound`). A reader following the parenthetical away from the entry's own forward-link arrives at a number that is eight IDs out of date.
3. **`glossary.md` — `V1 / V1.0 / V1.x / V2` entry, sentence `Conversion to the new-scheme spelling is incremental; the spec corpus carries mixed spellings during the transitional period with every untouched callsite governed by the alias mapping.`** This describes the in-progress editorial state of the spec corpus, not a rule binding an implementer. The normative content — the alias mapping and its closure heuristic — is the surrounding text and is owned by GOV-20 on `governance/release-version-naming.md`, which already restates the incremental-conversion permission verbatim. Deleting the sentence leaves no normative gap.
4. **`glossary.md` — `schema slug` entry, clause `a future grep gate uses this entry as its source of truth.`** This justifies the preceding synonym-avoidance rule by referencing a planned (not-yet-built) spec-tooling artefact. The synonym-avoidance rule itself stands on its own ("are spec-prose drift to be avoided"); the tooling-justification clause adds no constraint readable today.
5. **`glossary.md` — `Pi tool vs. .loom callable` entry, sentence `The rename does NOT affect any code identifiers, file extensions, diagnostic codes, frontmatter field names, or wire formats — it is purely terminology.`** This annotates a past editorial rename event; a reader encountering the entry fresh has no rename to be reassured about. The terminology obligation (MUST use `.loom callable` over named synonyms) is in the preceding sentence and stands without the annotation.

All five share one shape: stranded narrative about the spec's own past or planned editorial work, deletable in isolation. Item (2) is the only one with a small reader-misdirection cost (an out-of-date number); the rest are pure organisational drag.

**History:** Each sentence was authored by a different commit, each resolving a different upstream finding, and each was a defensible local edit at the time (the deferral sentence pinned what loom 1.0 does not ship; the GOV-23 parenthetical answered a "what's the live range?" reader question with the then-current number; the V1 sentence and the rename annotation both pre-emptively reassured readers about in-flight editorial transitions; the grep-gate clause justified an aggressive synonym ban by promising mechanical enforcement). The defect is emergent across that sequence — none of the five commits, read alone, introduced "stale narrative"; the corpus accumulated the residue as one editorial transition completed, the next finding resolved, and the GOV ceiling moved past the parenthetical's hardcoded number, without any of the originating commits revisiting their prose. The relevant `git log -S` runs confirm each sentence was added once and never re-edited.

## Solution Space

**Shape:** single

### Recommendation

Delete the five sentences in place. None requires a relocation — sub-defect (1)'s deferral is already inventoried at `future-considerations/surface-extensions.md` under "Richer runtime-event telemetry", and the other four are pure removals whose normative neighbours stand unchanged. Concretely:

1. In `runtime-event-channel.md`, remove the sentence `Aggregation, latency histograms, per-loom token reports, retention policies, and a consumer-facing read API for the event stream are deferred to [Future Considerations](../future-considerations.md).` from the paragraph before the `<a id="pic-1">` anchor. Keep the following sentence (`loom 1.0 ships the always-log set and the payload shape only; the channel is intentionally write-only at the spec level …`) unchanged.
2. In `glossary.md`'s `GOV-N (governance rule)` entry, remove the parenthetical `(currently running through GOV-23, with some IDs retired — see [Retired REQ-IDs](./governance/anchor-scheme-and-retired.md#retired-req-ids))`. The next clause already tells the reader to consult `governance.md` for the authoritative range.
3. In `glossary.md`'s `V1 / V1.0 / V1.x / V2` entry, remove the sentence `Conversion to the new-scheme spelling is incremental; the spec corpus carries mixed spellings during the transitional period with every untouched callsite governed by the alias mapping.` The same permission is recorded normatively at GOV-20.
4. In `glossary.md`'s `schema slug` entry, remove the clause `; a future grep gate uses this entry as its source of truth` (the preceding statement that the listed synonyms `are spec-prose drift to be avoided` remains intact).
5. In `glossary.md`'s `Pi tool vs. .loom callable` entry, remove the sentence `The rename does NOT affect any code identifiers, file extensions, diagnostic codes, frontmatter field names, or wire formats — it is purely terminology.`

Edge cases the implementer must watch:

- The conditional future SHOULD on `runtime-event-channel.md` (the `excludeFromContext` opt-out sentence on the `custom-message-context-entry-presupposition` paragraph) is named in this finding's original heading but is not in scope here — it is owned by the sibling finding listed below and must be left alone in this edit so its resolution does not bifurcate.
- The cross-reference from `version-bump-step2.md` checklist item (r) into the `custom-message-context-entry-presupposition` anchor is unaffected; do not retarget it.
- The five edits are independent — order does not matter, and partial application leaves the spec still valid — but the commit message should list all five so a future reviewer reading `git log -S` for any one of them can find the consolidation.

## Consequence

**Severity:** advisory

A fresh reader of `glossary.md` learns from the `GOV-N` entry that the live governance range tops out at GOV-23 when in fact it tops out at GOV-31; the same entry then tells them to consult `governance.md` instead, leaving them to discover the contradiction on their own. The other four sentences add no incorrect information but lengthen pages whose value is precisely their terseness; in aggregate they erode the convention that normative pages carry only constraints an implementer must follow.

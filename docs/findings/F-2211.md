## Problem

GOV-20's *Closure heuristic* default justification reads *"frozen-baseline is the stronger commitment, and design-scope claims subsume frozen-baseline claims."* The bare verb **subsume** admits two incompatible readings: an authority reading (design-scope is the broader/dominant concept, hence the *stronger* commitment) that contradicts the sentence's own conclusion, and a set-containment reading (`loom 1.0 ⊇ loom 1.0.0`, so the frozen-baseline is the narrower, stronger commitment) that is consistent with it. The justification clause carries no `⊇` cue of its own, so the self-contradictory authority reading is grammatically available. The operative directive (`Default on ambiguity: loom 1.0.0`) is unambiguous as to outcome; only its rationale is muddied.

**Introducing commits:** `4a7afbf` (2026-05-28, "pi-loom spec: introduce release-version naming + legacy token aliases (GOV-19/20/21)")
**History:** `git log -S "subsume frozen-baseline"` over the spec body (`docs/spec_topics/governance.md` and its post-split successor `docs/spec_topics/governance/release-version-naming.md`) returns exactly two commits. `4a7afbf` introduced the GOV-20 closure heuristic with the justification sentence verbatim — *"Default on ambiguity: `loom 1.0.0` — frozen-baseline is the stronger commitment, and design-scope claims subsume frozen-baseline claims"* — i.e. the ambiguous "subsume" wording was present at the rule's first authoring. `f5e89f4` (2026-06-04, "docs: enforce 100KB hard / 30KB recommended size cap on spec set") only carried the text across the `governance.md` → `governance/release-version-naming.md` file split; it did not alter the sentence. The intervening `5e5f2f1` ("resolve Frozen-baseline reclassification at closure callsites") touched the closure-callsite handling but preserved this justification clause. The defect therefore dates to GOV-20's inception, not to any later edit.

## Solution approach

Rewrite GOV-20's *Closure heuristic* default-justification sentence to pin the set-containment sense and drop the overloaded bare "subsume", spelling out the containment relation with the explicit `loom 1.0 ⊇ loom 1.0.0` cue already established in GOV-19's preamble. Make the "stronger commitment" conclusion follow from the frozen-baseline being the narrower, more specific reading.

## Solution constraints

- Keep the operative directive `Default on ambiguity: loom 1.0.0` (the default outcome) unchanged; only the rationale clause is in scope.

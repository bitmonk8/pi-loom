## Problem

`V13g` and `V13g-T` state the discard-observability event's firing predicate as the bare `Err`/`Ok` split — "fires … when — and only when — it settles to `Err`" — in their `Tests` bullet (byte-identical in both leaves) and in `V13g`'s `Ships when`. The channel `V13g` inherits from `runtime-event-channel.md` defines an always-log set (group A) and excludes four `QueryError` kinds — `validation`, `context_overflow`, `cancelled`, `invoke_callee` — even at a discard site. Because the per-phase TDD ritual derives the failing tests from the `-T` `Tests` bullet, an author following it verbatim asserts an event for a discarded excluded-kind `Err`, contradicting the spec the same leaf cites; over-emitted `loom-system-note` payloads then enter the LLM context via `convertToLlm`.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `4a04570` (V13g/V13g-T carved out of V13a); reaffirmed without correction by `650adcf`
**History:** `git log --follow` on `V13g-query-discard-observability.md` shows the leaf first appeared in `4a04570` (2026-06-17) with the firing predicate "the runtime discard-observability event fires on an explicit `Err` discard" — already unqualified by the always-log set. `git show 650adcf` (2026-06-18, "resolve explicit Err discard trigger condition is ambiguous") rewrote the bullet to the current "when — and only when — it settles to `Err`, and a discarded `Ok` … emits no event" form: it added the `Ok`/`Err` distinction but preserved (and emphasised) the over-broad `Err` predicate, so the gap against the always-log set has been present since the leaf's creation. No commit ever narrowed the discard predicate to the group-A membership the cited spec requires.

## Solution approach

Narrow the firing predicate in the `Tests` bullet of both `docs/plan_topics/V13g-query-discard-observability.md` and `docs/plan_topics/V13g-T-query-discard-observability.md` — editing the byte-identical bullet in lockstep — so the discard-observability event fires only for a discarded `Err` whose `kind` is in the always-log set (group A), and name the four excluded kinds (`validation`, `context_overflow`, `cancelled`, `invoke_callee`) as emitting nothing at a discard site. Add a `Tests` assertion witnessing a discarded excluded-kind `Err` (e.g. `cancelled`) producing zero events. Apply the same group-A-qualified condition to `V13g`'s `Ships when`, and align `V13g`'s `Adds` to name the preserved `Err` as an always-log-set `Err`.

## Solution constraints

- The always-log set partition and the four-kind exclusion are owned by `V9d` / `runtime-event-channel.md`; cite that ownership rather than restating the full group-A enumeration inside `V13g`.
- The spec is read-only for this fix, and `V9d` is out of scope.

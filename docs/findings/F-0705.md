## Finding

The glossary's `GOV-N (governance rule)` entry in `docs/spec_topics/glossary.md` parenthetically states that the live range is "currently running through `GOV-23`, with some IDs retired." The authoritative live set on `docs/spec_topics/governance.md` (and its `governance/` sub-pages) now runs `GOV-1 … GOV-31` with `GOV-2`, `GOV-4`, `GOV-10`–`GOV-14`, and `GOV-21` retired — eight retired IDs out of a high-water mark of thirty-one. The glossary's "through `GOV-23`" is therefore eight numbers low.

The entry is self-undermining: in the same parenthetical it warns "cite `governance.md` for the authoritative range rather than a fixed upper bound," then proceeds to commit exactly that fault by embedding a concrete `GOV-23` literal. A reader who treats the glossary as the canonical secondary source will under-cite the live set; a reviewer aware of the discrepancy still has to reconcile two non-aligned numbers in adjacent sentences.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:**

- `96cfc08` (2026-06-04) — "pi-loom spec: resolve 'Glossary GOV-N entry under-describes the live GOV range and scope'" rewrote the `GOV-N` entry and introduced the parenthetical "currently running through `GOV-23`" literal. The companion `cite governance.md … rather than a fixed upper bound` sentence was added in the same edit, so the entry was internally inconsistent from the moment the literal landed.
- `1d773d6` (2026-06-04, same day) — "GOV-3/GOV-6 subdirectory scan scope and prefix-table hub-stub bindings" introduced `GOV-24` on `governance.md`; from this commit forward the glossary literal was numerically stale.
- `fe8aa5c` (2026-06-05) — "GOV-12 bundles ≥3 independently-testable sub-obligations" introduced `GOV-30`/`GOV-31`, widening the gap to its current eight-rule magnitude.

**History:** `git log --follow docs/spec_topics/glossary.md` shows `96cfc08` as the sole commit touching the `GOV-N` entry's range literal; `git log -S 'GOV-24'`, `git log -S 'GOV-30'`, and `git log -S 'GOV-31'` over `docs/` identify the three coinage commits above (`1d773d6`, `8e3ccb4` for `GOV-25`/`GOV-27`/`GOV-28` via `dc2e720`/`8e3ccb4`, and `fe8aa5c`). No subsequent commit edited the glossary literal to keep step. The defect is the **interaction**: the glossary embedded a concrete upper bound while the governance page kept coining new IDs, and the embedded-literal pattern itself is what the original commit explicitly counselled against.

## Solution Space

**Shape:** single

### Recommendation

Drop the concrete `GOV-23` literal from the parenthetical and let the existing "cite `governance.md` for the authoritative range" directive stand alone. Replace the inner clause:

> currently running through `GOV-23`, with some IDs retired — see [Retired REQ-IDs](./governance/anchor-scheme-and-retired.md#retired-req-ids)

with:

> with some IDs retired — see [Retired REQ-IDs](./governance/anchor-scheme-and-retired.md#retired-req-ids)

so the full parenthetical reads "The live range is the set registered on `governance.md` (with some IDs retired — see Retired REQ-IDs); cite `governance.md` for the authoritative range rather than a fixed upper bound."

This is preferable to bumping the literal to `GOV-31`: the glossary entry already names exactly why the literal form is wrong, and any future GOV coinage will re-stale the same number. Removing the literal also closes the self-contradiction in one edit. Implementer edge case: do not delete the surrounding "with some IDs retired" half-sentence or its anchor link — that pointer is the entry's remaining substantive content beyond the deferral; only the "currently running through `GOV-23`," fragment is dropped.

## Consequence

**Severity:** advisory

A reader using the glossary as a reference under-counts the live GOV set by eight rules and may miss obligations introduced from `GOV-24` onward (e.g. `GOV-24` un-rowed-page / hub-stub semantics, `GOV-30` aggregator lock-step, `GOV-31` literal-vs-count parity), which are load-bearing for plan gates such as `H5e` and `H5d`. Nothing in the runtime breaks, but the glossary contradicts the canonical source on a count, weakening the entry's own "defer to `governance.md`" directive.

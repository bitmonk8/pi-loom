## Problem

The `H5e` un-rowed-page recogniser reddens CI on any `spec_topics/**` page absent from the REQ-ID prefix table, "unless it is a GOV-24 table-of-contents hub stub — a subtree TOC file that deliberately carries no row and no REQ-ID anchors — which is excluded" (`conventions.md` §REQ-ID discipline, restated in the `H5e` leaf and residue-inspection item 5). The only mechanical properties the exclusion names — "no row" and "no REQ-ID anchors" — are exactly the properties every in-scope un-rowed page also has, so a hub stub and a reddening residue page are indistinguishable by them; the remaining discriminators ("table-of-contents", "subtree TOC file", "deliberately") are semantic, not token/path-readable, and GOV-24 supplies no in-file marker, filename rule, or anchor pattern either. A faithful implementer must therefore invent a discriminator, and two implementers will pick different ones. At the `H6a` live-corpus gate the divergence is observable and expensive: an over-broad rule false-fails a legitimate hub stub (blocking release), while an over-narrow rule false-cleans a genuine un-rowed obligation page (letting its content escape the gate the recogniser exists to enforce).

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** a593446 — pi-loom plan: resolve "un-anchored-MUST recogniser un-rowed-page branch" (2026-06-17, Thomas Andersen)
**History:** Commit a593446 added the un-rowed-page recogniser branch together with its GOV-24 hub-stub carve-out, inserting the "in scope … unless it is a GOV-24 table-of-contents hub stub — a subtree TOC file that deliberately carries no row and no REQ-ID anchors — which is excluded" wording into `conventions.md` §REQ-ID discipline, the parallel H5a gate text, and residue-inspection item 5 in the same commit. That commit supplied the exclusion without any mechanical signal distinguishing a hub stub from an in-scope un-rowed page, so the defect has been present since the branch was authored. The `H5e` leaf later carried the wording verbatim when H5a's arms were split into separate leaves (f3bb326 / 5f33832, 2026-06-18).

## Solution approach

Give the `H5e` un-rowed-page recogniser a mechanical discriminator for the GOV-24 hub-stub exclusion — a canonical in-file marker a token/path scan can read — replacing the semantic "subtree TOC file that deliberately carries no row and no REQ-ID anchors" clause. Rewrite that exclusion clause in `conventions.md` §REQ-ID discipline so the exclusion turns on the marker, mirror the same discriminator in the `H5e` leaf's Adds/Tests/Ships-when, and update residue-inspection item 5's hub-stub parenthetical to match. Make the seeded GOV-24 hub-stub fixture carry the discriminator so the green-path test exercises the real recogniser rather than an implicit one.

- The spec is read-only for this fix: coining the marker as a GOV-24 authoring obligation and adding it to live hub-stub pages under `docs/spec_topics/**` is owned by the spec corpus, not this plan finding (`fix_surface` is plan).

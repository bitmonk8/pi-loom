## Finding

`V6a` Adds and `V11a` Adds both assert that the loom's `model:` resolution and `V11a`'s binder-model resolution "cannot decide the same 'reference matches no available model' condition differently" / "cannot diverge on the 'reference matches no available model' condition." That is an outcome-equivalence guarantee: it claims the two paths always reach the same verdict for the same input.

The verification, however, is explicitly scoped to instance identity. `V9b`'s "model-reference-matcher production wiring" Tests bullet and Ships-when, the `V11a` "single-matcher cross-resolution reconciliation" Tests bullet, and the paired `V9b-T`/`V11a-T` bullets all state the property is "observed by single-source-of-construction (instance identity), not equivalence-of-outcome against a shared fake." Instance identity proves both paths hold the same matcher object; it does not prove both route the unresolved-reference decision exclusively through that object.

A faithful implementer could pre-normalise the reference on one path (e.g. lower-case or strip the provider token before the call) or post-filter the matcher's verdict on the other, producing divergent outcomes for the same reference while the instance-identity assertion stays green. The plan therefore advertises a non-divergence guarantee stronger than what its tests enforce.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** 5953b18, ccedadf (entrenched by 3424d54)
**History:** The strong non-divergence claim entered `V6a` Adds in `5953b18` (2026-06-11, "resolve 'V6a model-reference-matcher seam has no declared producer'"), phrased as the two paths "cannot decide the same 'reference matches no available model' condition differently." The deliberately-narrow verification method — "single-source-of-construction (instance identity), not equivalence-of-outcome against a shared fake" — was added to `V9b` Tests/Ships-when later, in `ccedadf` (2026-06-16, "resolve 'V9b model-reference-matcher wiring untested'"). `3424d54` (2026-06-17, "resolve 'V9b matcher witness asserts a reconciliation only V11a can exercise'") moved the cross-resolution witness onto `V11a` and co-located both the "cannot diverge" prose and the instance-identity qualifier in `V11a` (and propagated the qualifier into `V11a-T`/`V9b-T`). The defect is the gap between the strong prose of `5953b18` and the explicitly-narrow verification of `ccedadf`; neither commit is wrong in isolation, but their juxtaposition asserts an outcome guarantee the verification declines to test.

## Solution Space

**Shape:** multiple

### Option A — Condition the prose on delegation, matching what instance identity delivers

**Approach.** Rewrite the non-divergence claims so the guarantee is stated as conditional on each path delegating the unresolved-reference decision wholly to the shared matcher instance, rather than as an unconditional outcome guarantee.

**Plan edits.**
- `V6a` Adds: replace "so this `model:` resolution and `V11a`'s binder-model resolution cannot decide the same 'reference matches no available model' condition differently" with a delegation-conditioned form, e.g. "…cannot decide the same 'reference matches no available model' condition differently, provided each path delegates that decision wholly to the shared matcher instance — no reference normalization before the call and no verdict filtering after it."
- `V11a` Adds: apply the same proviso to "so the two paths cannot diverge on the 'reference matches no available model' condition."
- `V11a` "single-matcher cross-resolution reconciliation" Tests bullet: apply the same proviso to its trailing "so the two paths cannot diverge…" clause.

**Spec edits.** None.

**Pros.** Matches the deliberately-chosen instance-identity verification; no test additions; smallest edit surface.

**Cons.** Leaves the delegation obligation as prose the implementer must honour rather than as a tested assertion.

**Risks.** The proviso can be overlooked if the implementer reads only the assertion.

### Option B — Add an outcome-equivalence assertion alongside instance identity

**Approach.** Keep the strong non-divergence prose and add a test that drives both resolution paths over a shared set of model references and asserts identical verdicts, in addition to the existing instance-identity assertion.

**Plan edits.**
- `V11a` / `V11a-T` (and `V9b` / `V9b-T` if the assertion is sited there): add an outcome-equivalence Tests bullet exercising both paths over a shared reference set (including the ambiguous-bare-`modelId` and zero-match cases) and asserting equal outcomes; leave `V6a`/`V11a` prose unchanged.

**Spec edits.** None.

**Pros.** Makes the "cannot diverge" claim literally verified.

**Cons.** Reverses the deliberate decision recorded in the verification prose ("not equivalence-of-outcome against a shared fake"); larger test surface; requires defining the shared reference set.

**Risks.** A fixed reference set still does not prove universality, so the strong prose can remain marginally ahead of the test.

### Recommendation

Adopt **Option A**. The instance-identity verification was a deliberate authoring choice — the Tests bullets explicitly state "not equivalence-of-outcome against a shared fake" — so the durable fix is to make the prose match that chosen scope rather than reverse the decision. Insert the delegation proviso in `V6a` Adds and in `V11a` (Adds + the "single-matcher cross-resolution reconciliation" Tests bullet). The proviso should name both failure modes it rules out — pre-call reference normalization and post-call verdict filtering — since either one breaks non-divergence while leaving the instance-identity assertion green.

## Consequence

**Severity:** advisory

If shipped unfixed, the plan asserts a non-divergence property it does not test for: an implementer can satisfy every green gate while building two paths that disagree on the "matches no available model" verdict (one pre-normalises the reference, the other post-filters), so the advertised guarantee silently does not hold in the delivered code.

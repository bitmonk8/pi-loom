## Problem

`V15g-invoke-provenance.md` and its paired `V15g-T` each carry a Tests bullet that cites `SLSH-5` inline — the per-`invoke`-hop producer seam recording the call-site provenance that `V12b` consumes to render the chain-attribution suffix. Per `conventions.md` §Leaf format **Tests**, an inline REQ-ID citation in a Tests bullet is a claim that the leaf implements at least part of that REQ-ID. But the `coverage-matrix.md` *Numbered REQ-IDs* row for SLSH-5 reads `SLSH-3, SLSH-4, SLSH-5 | V12b` — it names only the renderer `V12b`. So `V15g`/`-T`'s SLSH-5 citation resolves to no coverage cell, and the many-to-many mapping the convention promises is unrealised: the producer-seam half of SLSH-5 is implemented and tested at `V15g` yet recorded as closed by nobody.

## Issue introduction

**Verdict:** single-commit introduction
**Introducing commits:** `65363c9` (relocated unchanged into `5f4d271`)
**History:** The SLSH-5 coverage-matrix row `SLSH-3, SLSH-4, SLSH-5 | V12b` was authored naming only the renderer `V12b` in `c6a664e` (2026-06-10, "pi-loom plan: build/update plan for spec.md + review"). The producer-seam SLSH-5 citation entered in `65363c9` (2026-06-12, "pi-loom plan: resolve \"SLSH-5 chain attribution provenance ownership\""), which assigned the invocation-record provenance ownership to `V15a`, added the `SLSH-5 invocation-record provenance seam` Tests bullet (citing SLSH-5) to `V15a`/`V15a-T`, and touched `V12b`/`V12b-T` — but left the SLSH-5 coverage row at `V12b` only, so the producer-seam citation and the matrix row diverged from that commit onward. `5f4d271` (2026-06-17, "pi-loom plan: resolve \"V15a bundles ~14 independent test obligations into one impl+T pair\"") split the producer seam out of `V15a` into the new `V15g`/`V15g-T` leaves, relocating the citation unchanged; the matrix row was again not updated, so `V15g` inherited the dangling cross-reference. (Established via `git log -S 'SLSH-5' --follow` on the leaf files and `git log -S 'SLSH-3, SLSH-4, SLSH-5'` on `coverage-matrix.md`.)

## Solution approach

In `coverage-matrix.md`, add `V15g` to the SLSH-5 row's *Closing leaf(s)* cell alongside `V12b`, so `V15g`/`-T`'s existing SLSH-5 Tests citation resolves to a coverage cell and the §Leaf format **Tests** many-to-many mapping holds. Both leaves are genuine partial implementers of SLSH-5 — `V15g` produces the provenance record, `V12b` renders it. No `V15g`/`-T` leaf-file edit is required; those Tests bullets are already correct once the matrix lists `V15g`.

## Solution constraints

- The `V15g` entry must be marked a `co-witness` in its parenthetical annotation, with the leaf ID backtick-delimited per the *Closing-leaf column authoring* rule, so the `H5f` facet partition for the row stays `{V12b}` and no new per-facet citing-test obligation is minted.

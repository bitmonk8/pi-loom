## Finding

The §Leaf format section of `conventions.md` closes its **Deps.** field entry with an indented note (`conventions.md:51`):

> In the **coverage matrix**, each spec REQ-ID maps to the **implementation** leaf (`<id>`) that closes it (its green tests are the closure evidence); the paired `<id>-T` tests task is not listed separately.

This restates the coverage-matrix authoring rule that already lives authoritatively in `coverage-matrix.md`'s preamble (`coverage-matrix.md:3`):

> Every executable spec REQ-ID is mapped here to the **implementation** leaf (`<id>`) that closes it (its green tests are the closure evidence; the paired `<id>-T` tests task is not listed separately).

The two statements are near-identical, differing only in lead-in phrasing and the placement of the `<id>-T` clause inside vs outside the parenthesis. The note concerns how the coverage matrix maps REQ-IDs to closing leaves — it does not govern the **Deps.** field it is attached to, so it is mislocated, and being a second copy of a rule owned elsewhere it can silently drift out of sync with `coverage-matrix.md` as either copy is edited.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `f9c5354` (`pi-loom plan: add scope section + two-task TDD pairing convention`, 2026-06-09) added the `conventions.md` §Leaf format copy; `c6a664e` (`pi-loom plan: build/update plan for spec.md + review`, 2026-06-10) added the near-identical `coverage-matrix.md` preamble copy. `f9c5354` is an ancestor of `c6a664e`; both are in `HEAD` history.
**History:** `git log -S "tests task is not listed separately"` isolates each copy to a single introducing commit per file. The `conventions.md` sentence was authored first as part of the two-task TDD pairing convention; the next day's plan-build commit independently restated it in the `coverage-matrix.md` preamble. The duplication came into existence at `c6a664e`, when the second copy landed; neither commit was a deliberate cross-reference.

## Solution Space

**Shape:** single

### Recommendation

In `conventions.md` §Leaf format, replace the indented note under the **Deps.** bullet (the sentence beginning "In the **coverage matrix**, each spec REQ-ID maps to…") with a cross-reference that defers to `coverage-matrix.md` for the authoritative rule, and leave the `coverage-matrix.md` preamble copy as the single source of truth. A suitable replacement is a one-line pointer such as: "In the **coverage matrix**, each spec REQ-ID maps to its closing implementation leaf (`<id>`), not the paired `<id>-T` tests task — see [`coverage-matrix.md`](./coverage-matrix.md) for the authoritative authoring rule." The `coverage-matrix.md` preamble (`coverage-matrix.md:3`) is read-only for this fix.

## Consequence

**Severity:** cosmetic

If left unfixed the two copies can diverge as either file is edited, leaving readers with two slightly different statements of the same REQ-ID-to-leaf mapping rule and no indication of which is authoritative. No implementer is blocked and no gate misfires today; the cost is maintenance ambiguity.

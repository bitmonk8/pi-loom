## Finding

`H4a`'s 3rd `Tests` bullet states the session-double fidelity-contract self-check: the in-process double must reproduce, relative to the pinned SDK, (i) streamed-token order before `ctx.waitForIdle()` resolves, (ii) single-turn prompt-mode append, (iii) the `pi.on` cancel-forward subscription, and (iv) cancellation propagation. The bullet then closes with a scope-exclusion sentence: "The manual real-host smoke run that gates a double-vs-real divergence is **not** part of this self-check and never runs under `npm test`; it is defined as a release-gate / version-bump governance protocol in [`real-host-smoke-gate.md`]."

That closing sentence is not an assertion the harness self-check makes — it is a boundary / cross-reference note clarifying what the bullet does **not** cover. Sitting inside the bullet's assertion text, it can be read as a test obligation the self-check must encode (e.g. an attempt to "assert that the smoke run never runs under `npm test`", which is vacuous and unverifiable). The bullet should carry only the four assertions the self-check makes; the cross-reference to the manual smoke protocol belongs outside the assertion text.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/H4a-factory-shell-and-harness.md`, strike the trailing scope-exclusion sentence from the 3rd `Tests` bullet — "The manual real-host smoke run that gates a double-vs-real divergence is **not** part of this self-check and never runs under `npm test`; it is defined as a release-gate / version-bump governance protocol in [`real-host-smoke-gate.md`](./real-host-smoke-gate.md)." — leaving the bullet to end at the four-axis self-check assertion. Re-state the same content as a standalone note following the Tests bullet list (a `Note:`-style sentence, outside any bullet's assertion text), so the cross-reference to the manual smoke protocol is preserved without reading as a self-check obligation. The protocol it points at is already described in `H4a`'s Adds paragraph and `real-host-smoke-gate.md`; no new content is needed.

## Consequence

**Severity:** cosmetic

A reader scanning `H4a`'s Tests for the obligations the self-check must witness encounters a non-assertion (a "does not cover / never runs under `npm test`" boundary note) interleaved with the four real assertions. The executable test is unaffected; the cost is a reader briefly mistaking a scope-exclusion note for a required boundary check.

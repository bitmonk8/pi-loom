## Finding

"MUST be treated as read-only by convention" couples a normative MUST with an explicit statement that the constraint is unenforced ("by convention"). The array returned by `pi.getCommands()` is a snapshot; mutating it has no observable effect on Pi's internal state and no observable effect on the collision-check outcome. No test can distinguish a compliant implementation from one that mutates the array. The MUST is therefore unverifiable as written.

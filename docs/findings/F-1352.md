## Problem

BNDR-10 — the runtime MUST treat `bind_context: session` on a `mode: subagent` loom as `bind_context: none` for binder-input construction (truncation walk skipped, no *Recent session context* block) — is a live numbered REQ-ID with no row in the Numbered REQ-IDs table of `coverage-matrix.md` and no citing leaf. `V11i` implements the general `bind_context: session` truncation walk but not the subagent-mode skip. At the loom 1.0 release gate the `H5a` closing-gate automation reddens CI for an executable numbered REQ-ID with no closing leaf, and until then the obligation ships unverified.

## Issue introduction

**Verdict:** introduced-by-later-spec-edit (two distinct commits; never propagated to the plan)
**Introducing commits:** `bc77cf6` (`pi-loom spec-plan: fix F-1130 — coin BNDR-10 …`, 2026-06-24) for BNDR-10; `1b6d2f1` (`pi-loom spec-plan: fix F-1282 — coin BNDR-11 …`, 2026-06-24) for BNDR-11.
**History:** `git log -S bndr-10` / `-S bndr-11` over `docs/spec_topics/binder/binder-model-and-context.md` traces each REQ-ID to a single spec-coining commit (`bc77cf6` and `1b6d2f1` respectively). `git log -G 'BNDR-7, BNDR-8, BNDR-9'` and `git log -S 'BNDR-10' / -S 'BNDR-11'` over `docs/plan_topics/coverage-matrix.md` show the BNDR coverage row was authored once at the plan build (`c6a664e`, `pi-loom plan: build/update plan for spec.md + review`, 2026-06-10) and BNDR-10 / BNDR-11 were never added. `git merge-base --is-ancestor` confirms `c6a664e` predates both coining commits: the coverage matrix was built when only BNDR-1…9 existed, and each later spec edit that coined a new BNDR REQ-ID landed without a paired coverage-matrix row or citing leaf. The defect is not present-since-inception of the matrix; it is two independent spec-after-plan additions whose plan-side propagation was omitted.

## Solution approach

Add a `BNDR-10` row to the Numbered REQ-IDs table in `coverage-matrix.md` closing at leaf `V11i`. Add a *Tests* bullet to `V11i` and to its paired `V11i-T` that cites `BNDR-10` inline and asserts that, for a `mode: subagent` loom declaring `bind_context: session`, the truncation walk is skipped and no *Recent session context* block is emitted (binder input constructed as for `bind_context: none`).

## Solution constraints

- Backtick-delimit the `V11i` token in the new closing-leaf cell per the *Closing-leaf column authoring* rule in `coverage-matrix.md`; the `H5d` transitive-completeness tokeniser silently drops an un-delimited leaf ID from the cell.
- Out of scope: BNDR-11, which closes independently at a different leaf.

- same-cluster (resolves independently of BNDR-11, at a different leaf).

## Problem

Items `(aa)` (`complete()` retry-timing), `(ab)` (`complete()` `options.signal` cancellation), and `(af)` (`complete()` `onResponse` delivery) each direct the bump contributor to "re-run the per-provider … fixtures against each loom 1.0-supported provider (`anthropic-messages`, `openai-completions`, `mistral`, `amazon-bedrock`)" but nowhere state what the fixture run exercises — whether each per-provider run is a corpus replay or a live HTTP call. The wording naturally reads as exercising the real provider, silently presupposing the contributor holds working credentials, a paid account, and network reach to all four upstream APIs. Sibling items `(i)` and `(u)` on the same page already settle this the right way, each carrying an explicit "fixtures match … against a maintained corpus … rather than against live provider responses" clause that `(aa)`, `(ab)`, and `(af)` did not adopt. The result is a contributor-discipline gate whose `pass` / `not-performed` record carries no signal about which execution model was used, and no defined disposition for a contributor lacking any one provider's credentials.

## Solution approach

Clarify the execution model of items `(aa)`, `(ab)`, and `(af)` in `version-bump-step2.md` (anchors `#bump-checklist-complete-retry-timing`, `#bump-checklist-complete-signal-honouring`, `#bump-checklist-complete-onresponse-delivery`) by adding to each a corpus-not-live statement parallel to the one siblings `(i)` (`#bump-checklist-provider-overflow-wording`) and `(u)` (`#bump-checklist-complete-forced-tool`) already carry: the fixtures exercise the loom-side path against a maintained captured-response corpus in the repo's `npm test` suite rather than against live provider calls, and re-capturing each provider's responses on an API-version change is the responsibility of the contributor performing the bump. Swap in each item's own observable — inter-attempt retry/backoff timing for `(aa)`, `options.signal` cancellation for `(ab)`, `onResponse` `ProviderResponse { status, headers }` invocations for `(af)`. Retain `(af)`'s `@earendil-works/pi-ai` package qualifier in its re-capture trigger so the trigger does not widen to a `pi-coding-agent` API-version change.

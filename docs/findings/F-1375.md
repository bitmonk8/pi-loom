## Finding

`V18d`'s **Adds.** field closes with a `**Blast radius.**` paragraph: "A divergent pin's blast radius is every runtime leaf — it is real-host-only and is **not** witnessed by the double-backed acceptance gate above, which is why the pre-merge real-host smoke is the gate that closes the window." This is a risk/operational-impact characterisation of a divergent Pi-SDK pin escaping to `main` — NFR/rationale content, not a statement of what the leaf ships.

Per `conventions.md` §Leaf format, **Adds.** is the descriptive "what the leaf introduces" field; its binding obligations live in **Tests.** and **Ships when**, and Adds prose alone does not bind the implementer. Risk/NFR rationale has no home in Adds. The blast-radius assessment is squarely a concern of the manual real-host smoke gate, which already documents the same window: `real-host-smoke-gate.md` states that a confirmed behavioural-divergence finding "blocks the bump from merging — the prior Pi-SDK pin is restored before merge (see `V18d`'s revert path …)" and that the smoke is the pre-merge gate catching divergence "before it reaches `main`." The blast-radius rationale is the *why* behind that gate and should sit with it.

This mirrors `V16a`'s pattern of citing the cross-cutting home for a cross-cutting concept rather than restating policy/rationale inline in a delivery leaf.

## Issue introduction

**Verdict:** introduced-by-specific-commit
**Introducing commits:** `15eb44c` — "pi-loom plan: resolve \"real-host divergence pre-merge gate (posture i)\"" (2026-06-11)
**History:** `git log -S "Blast radius" --follow` on `docs/plan_topics/V18d-version-bump-acceptance.md` returns exactly one commit, `15eb44c`. `git show 15eb44c` confirms the `**Blast radius.**` sentence was added to **Adds.** in the same edit that introduced the manual-real-host-smoke pre-merge divergence-gate prose: the solution that established the pre-merge gate placed its risk rationale inline in Adds rather than in `real-host-smoke-gate.md`. The defect was therefore born with that resolution, not present since the leaf's inception.

## Solution Space

**Shape:** single

### Recommendation

Strike the `**Blast radius.**` sentence from `V18d`'s **Adds.** field — the clause "A divergent pin's blast radius is every runtime leaf — it is real-host-only and is **not** witnessed by the double-backed acceptance gate above, which is why the pre-merge real-host smoke is the gate that closes the window." Relocate that rationale into `real-host-smoke-gate.md`, where the divergent-pin pre-merge window and the merge-blocker / revert path are already described, so the blast-radius *why* sits with the gate it justifies.

The adjacent ownership pointer "The static surface-inventory / `engines.node` / peer-dep / reason-snapshot gates are owned by `V18c`." is a delivery-boundary cross-reference, not risk content, and may remain in Adds.

`real-host-smoke-gate.md` is the edit target for the relocated rationale; the spec topics under `spec_topics/**` are read-only for this fix.

## Consequence

**Severity:** cosmetic

Risk/NFR rationale sits in a field `conventions.md` reserves for deliverable description, so the blast-radius reasoning is scattered away from the gate it justifies. No implementer is misled into divergent behaviour — Adds is descriptive-by-default and the gating obligations live in Tests / Ships when — so the impact is organisational dispersion of the rationale, not a behavioural or coverage defect.

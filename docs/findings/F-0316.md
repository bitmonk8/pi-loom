## Problem

The normative tool-execution lowering MUSTs in `spec_topics/pi-integration-contract/host-interfaces-core.md` §*Tool execution from loom code* (anchor `#tool-execution-from-loom-code`) — the `type === "text"` filter and single-`"\n"` join, the `Ok("")` empty-result case with no diagnostic, the `isError: true` → `Err(CodeToolError{cause:"execution"})` mapping with 4096-byte code-point-boundary truncation, the fixed `"tool reported an error with no text content"` message, and the `execute()`-throw coercion — are owned by no plan leaf and anchored by no coverage-matrix row. `V14a`'s `Spec.` cites only `tool-calls.md`, and its `Tests.` assert the closed `CodeToolError` enum, the two accepted-path return-type rows, and the `.loom`-callable failure surface — none exercise these lowering mechanics. These MUSTs are the GOV-22 un-anchored-obligation residue that `conventions.md` *REQ-ID discipline* makes the third closing-gate surface, so the `H5a` closing gate reddens against them once `H6a` flips the matrix to live-corpus footing.

The spec's tool-execution lowering MUSTs and the `#tool-execution-from-loom-code` anchor were already present in `host-interfaces-core.md` by f5e89f4 (2026-06-04, the spec size-cap split), so the content existed before the plan build. `coverage-matrix.md` has never referenced that surface (`git log -S "tool-execution-from-loom-code" -- coverage-matrix.md` is empty), so the gap is present from the leaf's inception, not introduced by a later edit.

## Solution approach

Add a rule-driven row to the *Code-keyed obligation areas (no numbered REQ-IDs)* table in `coverage-matrix.md` keyed on `host-interfaces-core.md §Tool execution from loom code`, naming `V14a` as the closing leaf and mirroring the sibling `pi-integration-contract/**` un-anchored-MUST rows (e.g. the `conversation-drive.md §untyped-query-ok-extraction → V9c` row, with the `(un-anchored; GOV-22 residue)` marker). Add `host-interfaces-core.md` (anchor `#tool-execution-from-loom-code`) to `V14a-tool-calls.md`'s `Spec.` alongside `tool-calls.md`, add `Tests.` in `V14a-T-tool-calls.md` asserting the five accepted-path lowering mechanics — text-filter + `\n` join, `Ok("")` with no diagnostic, `isError: true` → `Err(CodeToolError{cause:"execution"})` with 4096-byte code-point-boundary truncation, the fixed `"tool reported an error with no text content"` no-text message, and `execute()`-throw coercion under the same truncation rule — and extend `V14a`'s `Ships when.` to require those assertions green.

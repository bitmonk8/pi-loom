## Problem

The diagnostics cluster hub `docs/spec_topics/diagnostics.md` carries only an introductory paragraph and a Contents table; its sole scope statement is the embedded `LoadExtensionsResult.errors` disclaimer. The cluster's substantive boundaries are pinned only on sub-pages — the `loom/typecheck/*` out-of-scope note and the Re-scan deduplication / no-suppress-duplicates rules in `diagnostic-shape.md` — so a reader scanning the hub for the cluster's scope edges cannot find them. One boundary, LSP wire-protocol transport of diagnostics, is never declared in or out of scope anywhere; `diagnostic-shape.md` names LSP only as a consumer of `details.diagnostics`.

## Issue introduction
**Verdict:** present-since-inception
**Introducing commits:** `31ff060` (initial appearance of `docs/spec_topics/diagnostics.md` in version control); the gap was carried forward unchanged by `f5e89f4` ("docs: enforce 100KB hard / 30KB recommended size cap on spec set"), which split the monolithic file into the current hub + sub-pages without adding a non-goals/scope section.
**History:** `git log --diff-filter=A` shows the file first entered the tree at `31ff060`. `git show 31ff060:docs/spec_topics/diagnostics.md` is a single combined file containing the channel, shape, and registry content with no non-goals, out-of-scope, or `## Scope` section (`grep -in 'non-goal\|out of scope\|## scope'` over that revision returns nothing). The hub/sub-page split that produced today's slim hub was introduced by `f5e89f4` (first commit touching the `Contents` table, per `git log -S 'Contents'`), and the resulting hub likewise carries no such section (same grep over the working tree returns nothing). The defect is therefore an omission that has existed continuously since the file's inception; no later commit introduced it.

## Solution approach

Add a `## Non-goals` section to `docs/spec_topics/diagnostics.md` that surfaces the cluster's scope edges at the hub: forward-cross-reference the `id="loom-typecheck-namespace"` out-of-scope note, the Re-scan deduplication paragraph (runtime does not clear or supersede prior `loom-system-note` entries), and its renderer no-suppress-duplicates rule, all in `diagnostic-shape.md`. Author a fresh normative exclusion for LSP wire-protocol transport of diagnostics — the one boundary not stated anywhere today.

## Solution constraints

- The sub-page exclusions (`loom/typecheck/*`, Re-scan deduplication, no-suppress-duplicates) remain owned by `diagnostic-shape.md`; the hub section references that prose and MUST NOT re-state them as fresh normative pins.

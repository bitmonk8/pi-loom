## Problem

`.pi/project-config.md` (§Spec rules, "Stable inline labels" bullet) snapshots the live GOV-16 inline-label prefixes as only `HC3` and `NOCEIL`, both described as "(both bound to `hard-ceilings.md`)". The authoritative GOV-16 per-page inline-label prefix table on `docs/spec_topics/governance/stable-inline-labels.md` registers three live prefixes — `HC3` (`hard-ceilings/ceilings-3-and-4.md`), `NOCEIL` (`hard-ceilings/ceiling-invariants-and-audit.md`), and `SNK` (`slash-invocation.md`) — so the snapshot omits `SNK` and mis-states the `HC3`/`NOCEIL` page bindings. Unlike the REQ-ID snapshot immediately above it, this snapshot carries no non-authoritative disclaimer, so a fixer or tool reading it treats `SNK` as unregistered and surfaces a spurious prefix-add for an already-governed prefix.

## Issue introduction
**Verdict:** single-commit
**Introducing commit:** `27b2576` (2026-06-05, "pi-loom spec: resolve \"SLSH-4 covers 11 independently normative template rows under one REQ-ID\"")
**History:** The `.pi/project-config.md` inline-label snapshot was created in `899b8fa` (2026-05-22) listing `HC3`, `NOCEIL`, which was accurate at that time — only those two prefixes were registered. `27b2576` later added the `SNK` row to the GOV-16 per-page inline-label prefix table on `stable-inline-labels.md` (and the `SNK-a … SNK-k` anchors on `slash-invocation.md`) but did not update the snapshot in the same commit, so the snapshot drifted out of sync with the live table. The defect is the un-propagated registration, not a flaw present since the snapshot's inception.

## Solution approach

In `.pi/project-config.md`'s "Stable inline labels" bullet, add the missing `SNK` (`slash-invocation.md`) prefix and correct the `HC3`/`NOCEIL` bindings to their GOV-16 sub-pages (`hard-ceilings/ceilings-3-and-4.md`, `hard-ceilings/ceiling-invariants-and-audit.md`), dropping the inaccurate "(both bound to `hard-ceilings.md`)" parenthetical. Add a non-authoritative hedge naming `governance/stable-inline-labels.md` as owner of the live inline-label set, matching the disclaimer the REQ-ID snapshot already carries.

## Finding

The Echo policy format bullet for defaulted fields reads, in full:

> Defaulted fields tagged `(default)`: `focus_areas=[] (default)`. The tag fires only when the field took its declared default (default-supplied, not binder-supplied); per [Defaulting](#defaulting)'s fill-if-absent rule a binder-supplied value for a defaulted field is rendered untagged.

The rule text specifies *when* the tag fires but never states *where* it appears relative to the formatted value: that it is appended after the value, that exactly one U+0020 space separates them, and that the literal sequence is `(default)` with the surrounding parentheses included. All of this is implied solely by the inline example `focus_areas=[] (default)`. The BNDR-6 reference-rendering table — which is the normative ground truth for every other formatter behaviour in this file ("conforming implementations MUST reproduce these exactly") — contains no row exercising a defaulted field, so the example next to the bullet is the only artefact pinning placement, and it is not labelled normative.

The gap is small but it is the only formatter rule in the section whose placement is left to be inferred from an unlabelled example. Every neighbouring bullet either gives a complete syntactic specification (the unquoted-vs-quoted predicate, the array-truncation grammar `[a, b, c, …+N more]`, the object-hint shape `{first-field-value, …}`) or backs an example with a BNDR-6 row.

## Solution Space

**Shape:** single

### Recommendation

Replace the bullet's leading sentence with a self-contained syntactic specification, then keep the example as illustration:

> Defaulted fields are rendered as `<formatted-value> (default)` — the field's formatted value (produced by the rules above for its declared type), then one U+0020 space, then the seven-character literal `(default)` (open paren, the lowercase word `default`, close paren). Example: `focus_areas=[] (default)`. The tag fires only when the field took its declared default (default-supplied, not binder-supplied); per [Defaulting](#defaulting)'s fill-if-absent rule a binder-supplied value for a defaulted field is rendered untagged.

Optionally — and consistent with how every other formatter rule in this file is pinned — append one row to the BNDR-6 table exercising a defaulted field so the placement has a normative reference rendering. A natural choice is `focus_areas=[] (default)` (array-typed defaulted field, ties the BNDR-6m `[]` row to the new tagging rule); the next free sub-letter is `bndr-6y`. This is a strict superset of the rule-text fix and does not require renumbering existing sub-letters per the table's "later row append takes the next free sub-letter" clause.

Edge cases the implementer must watch:

- The shared 120-code-point line cap from System-note rendering rule 2 applies *after* per-value rendering, so the trailing ` (default)` (9 code points including the leading space) participates in the cap and can itself be truncated by the `…` rule. The spec already says the cap applies post-interpolation; no extra wording is needed, but a test exercising a defaulted field near the cap is worth having.
- Quoting interacts cleanly: a quoted string value renders as `field="value with space" (default)` — the space-then-`(default)` is *outside* the closing quote, never inside it.

## Consequence

**Severity:** advisory

Two implementers reading only the rule text could render `focus_areas=[](default)` (no separator), `focus_areas=[] (default)` (one space), or `(default) focus_areas=[]` (prepended); the inline example pushes them toward the middle form but is not labelled normative. In practice the example is co-located with the bullet so divergence is unlikely, but the gap leaves a conformance test without a normative target to cite.

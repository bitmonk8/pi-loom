## Problem

The *Panic message string (normative)* paragraph of `errors-and-results/error-model.md` reproduces all six `loom/runtime/*` panic message templates in an inline `| Code | Message template |` table, duplicating the authoritative per-code rows in `diagnostics/code-registry-runtime.md`. Both copies read as normative to an implementer, since the prose states the templates are normative and that conformance tests MAY assert on the exact string. The only thing preventing a live contradiction is an embedded "the registry is authoritative if the two ever drift" tiebreak sentence — a maintenance hazard rather than a true single-owner contract. Any future edit to one table that misses the other silently diverges.

## Issue introduction
**Verdict:** single-commit
**Introducing commit:** `0c230a1` — "pi-loom spec: resolve \"Panic message content per source unspecified\"" (2026-05-04 17:47:19 +0200)
**History:** The authoritative panic-message templates first landed in the diagnostics registry in commit `839db71` ("Diagnostic codes ... assigned to specific errors", 2026-05-04 17:43:22). About four minutes later, commit `0c230a1` resolved the "panic message content per source unspecified" review item by adding the inline five-row (later six-row) shorthand table to the then-`spec_topics/errors-and-results.md`, together with the "the registry is authoritative if the two ever drift" sentence — i.e. the duplication and its self-acknowledged drift risk were introduced in the same edit (`git log -S "shorthand for the registry rows"` and `git log -S "the registry is authoritative if the two ever drift"` both resolve to this lineage). The later size-cap split commit `f5e89f4` (2026-06-04) relocated the prose into `errors-and-results/error-model.md` and preserved the duplicate verbatim; it did not introduce the defect.

## Solution approach

Delete the inline `| Code | Message template |` table from the *Panic message string (normative)* paragraph in `error-model.md` and rewrite the surrounding clause to reference the diagnostics code registry as the sole authoritative source, forward-linking to `../diagnostics/code-registry-runtime.md#loomruntime-runtime-panics-runtime-defect-surface-and-delivery-failures`. Drop the now-redundant "the registry is authoritative if the two ever drift" tiebreak clause.

## Solution constraints

- Out of scope: the templates-are-normative statement, the conformance-test-MAY-assert allowance, the placeholder-rendering pointer to `placeholder-rendering-a.md`, the one-message-string-per-panic rule, and the per-surface routing bullets in the same paragraph — none depend on the table and stay unchanged.

## Finding

Rule 2 (Length cap) of *System-note rendering* in `binder/defaulting-system-note-echo.md` closes with a host-specific implementation note: *"Note: `string.length` in JavaScript returns UTF-16 code units and over-counts every astral code point as 2; count scalars via `Array.from(str).length` or a `for…of` iterator."* The behavioural contract the rule must pin is purely observable — *measure the cap in Unicode scalar values (code points), never UTF-16 code units* — and rule 2 already states that contract independently. Naming concrete JavaScript idioms (`Array.from(str).length`, `for…of`, the `string.length` over-count) is implementation guidance, not a constraint on observable output.

The note is embedded mid-body in a normative MUST rule rather than fenced as a non-normative aside. The same page already demonstrates the correct shape one section down: BNDR-5 carries its host-mechanics guidance under an explicitly labelled *"Derivation recipe (non-normative)."* heading, keeping the normative byte-output contract clean. Rule 2's inline `Note:` breaks that established convention and mixes informative how-to text into the normative line-discipline.

The defect is mild: the `Note:` prefix already signals informativeness, and `runtime-value-model.md` (JavaScript engine assumptions) pins loom 1.0 to a Node-only host, so the JS idioms are not misleading. It is an organisational/consistency blemish, not an ambiguity or a behaviour divergence.

## Solution Space

**Shape:** single

### Recommendation

In rule 2, keep the normative requirement in the rule body — the rendered note's 120-code-point cap and truncation are measured in Unicode scalar values (code points), never UTF-16 code units. Move the host-specific idioms (`string.length` over-counts every astral code point as 2; count scalars via `Array.from(str).length` or a `for…of` iterator) out of the rule body into a fenced non-normative aside immediately after the rule, mirroring BNDR-5's *"Derivation recipe (non-normative)."* formatting on the same page. The aside text and the normative contract are unchanged in substance; only the placement and the explicit non-normative label change.

## Consequence

**Severity:** cosmetic

No observer is affected: the cap's observable contract (count Unicode scalar values, never UTF-16 code units) is unambiguous with or without the idiom note, and the host is JS-exclusive. The cost is purely organisational — informative host mechanics sit inside a normative rule, inconsistent with the page's own BNDR-5 fencing convention.

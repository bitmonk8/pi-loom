## Finding

The **Unknown-key policy** paragraph in `frontmatter-fields-a.md` closes with a design-rationale sentence: *"The seam is what allows the deferred `binder_temperature` and user-overridable binder-prompt extensions in [Future Considerations](../future-considerations.md) to land additively without a frontmatter-schema bump."* The normative behaviour — unknown top-level keys are ignored at the value level, surface as a single per-key warning (`loom/load/unknown-frontmatter-field` or `loom/load/deferred-frontmatter-field`), and the loom continues to load — is already fully and independently stated by the sentences that precede it, including the immediately prior sentence that justifies the warning-not-error posture in terms an implementer can act on. The trailing sentence adds only forward-looking rationale that names specific deferred features; it imposes no constraint on any implementer.

The sentence is additionally inaccurate: `binder_temperature` does not appear anywhere in `future-considerations.md` (the deferred field was re-spelled to satisfy the binder-model root-word convention, but this rationale callsite retained the old spelling). A rationale aside that cites a token absent from the very page it points at is pure liability — it can only mislead a reader who follows the reference.

This is the same defect pattern as the sibling seam blockquote in `binder-model-and-context.md`: a deferred-feature rationale embedded inline beside normative text. Both originate from the same authoring commit that introduced surface-extension seam annotations.

## Issue introduction
**Verdict:** single introducing commit
**Introducing commit:** `09c42c0` — *pi-loom spec: resolve "Surface-extension Seam annotations carry V1 obligations from non-goals page"* (2026-05-05)
**History:** The corpus is git-tracked. `git log -S 'The seam is what allows the deferred'` and `git log -S 'binder_temperature'` resolve the rationale sentence to a single authoring event. Commit `09c42c0` added the entire **Unknown-key policy** paragraph to the then-monolithic `frontmatter.md`, including this trailing seam-rationale sentence verbatim, as part of relocating surface-extension seam obligations off the non-goals page. The paragraph (and sentence) was later carried unchanged when `f5e89f4` split `frontmatter.md` into `frontmatter-fields-a.md` under the spec size-cap work; that split is a pure relocation, not the introduction. The defect did not predate `09c42c0` — the Unknown-key policy paragraph did not exist before it. The `binder_temperature` → `bind_temperature` rename (`5476cf9`) corrected other callsites but left this rationale sentence's spelling stale, which is consistent with the sentence being incidental rationale rather than a tracked normative surface.

## Solution Space

**Shape:** single

### Recommendation

Delete the final sentence of the Unknown-key policy paragraph (*"The seam is what allows the deferred `binder_temperature` and user-overridable binder-prompt extensions … without a frontmatter-schema bump."*). The remaining paragraph retains the complete normative rule (ignore-at-value-level, single per-key warning with the two diagnostic codes, continue-to-load) and the warning-not-error justification. No cross-reference elsewhere depends on this sentence; forward-compatibility of the frontmatter vocabulary is already conveyed by the preceding "future minor versions extend it additively" clause.

## Consequence

**Severity:** cosmetic

No observable behaviour is affected: the normative unknown-key rule is complete without this sentence. The only impact is reader-facing — a stale, deferred-feature rationale (citing a `binder_temperature` token no longer present in the referenced page) that adds maintenance drift to a normative paragraph.

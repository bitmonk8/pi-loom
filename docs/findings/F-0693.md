## Problem

`match`'s grammar is split across two files with no reciprocal cross-reference at the relevant production or table. The normative six-form Pattern table lives in `errors-and-results/error-model.md`; the `MatchArm ::= Pattern "=>" ArmBody` and `ArmBody` productions live in `grammar.md` under `## match arm body`. The `MatchArm` production cites a `Pattern` non-terminal the file does not define, and its only pointer to the table is an aside under the unrelated `LetStmt` production; the Pattern table in turn carries no pointer back to the grammar appendix. A reader of either file alone cannot navigate to the other half of the grammar.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `53f5831` (2026-05-03, "spec: rewrite query primitive, system prompts, and schema syntax") landed the Pattern grammar table in the then-monolithic `spec.md`; `f841b12` (2026-05-04, "spec: resolve Cluster 2 (literal sublanguage + grammar appendix)") created the grammar appendix and decided to leave the pattern table in the errors-and-results material while introducing `MatchArm ::= Pattern "=>" ArmBody` in the new appendix without an inline back-pointer at the `MatchArm` production.
**History:** The pattern table predates the grammar appendix by one day. The placement defect did not exist when `53f5831` landed — there was no second file to cross-reference. It crystallised in `f841b12`, which authored the appendix with the express scope of owning "surface-syntax forms no single topic page owns end-to-end," authored `MatchArm`/`ArmBody` there, attached a `Pattern` cross-reference only to the unrelated `LetStmt` production, and chose not to relocate or back-reference the pattern table from the appendix. The split-into-topics commit `fecb504` (2026-05-04) is mechanical and did not contribute to the defect. The 100 KB-cap rename pass `f5e89f4` is a verbatim search-and-replace and is also non-contributory.

## Solution approach

Add a forward-link at the `MatchArm` production in `grammar.md` that resolves the `Pattern` non-terminal to the Pattern grammar table in `error-model.md`. Add a reciprocal forward-link under the Pattern grammar table in `error-model.md` pointing to the `## match arm body` section in the grammar appendix. Add a stable anchor at the `**Pattern grammar (loom 1.0).**` heading in `error-model.md` so the new `MatchArm`-adjacent link references the table by anchor rather than by section title.

## Solution constraints

- Keep the existing `LetStmt` aside in `grammar.md` that cross-references the Pattern grammar; do not collapse it into the new `MatchArm`-adjacent pointer — the two pointers serve different productions.

## Problem

The `No concurrent user sessions in the same host process.` bullet under loom 1.0 non-goals (`#loom-1-0-non-goals`) in `future-considerations/model-changes-and-non-goals.md` is a roughly 900-word paragraph in which only the opening single-active-session presupposition and the closing *Recorded at:* citations carry non-goal content. The remainder is procedural detection-mechanism analysis — surface-inventory presence-check limits, the `npm run typecheck` exhaustive-matching carve-out, the hypothetical `pi.openParallelSession` wholly-new-member case, and point-release closed-set validator behaviour — that is owned elsewhere: by `pi-integration-contract/version-bump-step2.md` (surface-inventory scope and the version-bump checklist), `pi-integration-contract/capability-probe.md` (Step 0 probe scope), and the `session_shutdown` unknown-reason rule. The bullet therefore duplicates each owning page's semantics inline, so a maintainer editing an authoritative page must hunt this paragraph down to avoid divergence, and a reader cannot quickly identify the loom 1.0 disposition.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `0e12140` (2026-05-08, *pi-loom spec: resolve "Non-goals are not consolidated into a single section"*); `068cb27` (2026-05-17, *pi-loom spec: resolve "T22b — Multi-session contingency response is unspecified in Future Considerations"*)
**History:** `git log -G "concurrent user sessions in the same host process" --reverse -- docs/spec_topics/future-considerations.md docs/spec_topics/future-considerations/model-changes-and-non-goals.md` shows the bullet first appearing at `0e12140` already in its long form — the typecheck/`openParallelSession`/closed-set/wholly-new-member analysis was authored as part of the initial consolidation into the non-goals section, not later. `git show 068cb27 -- docs/spec_topics/future-considerations.md` then shows the "Disposition for an unanticipated second session that nonetheless reaches the extension…" enumeration of every site loom 1.0 scopes to a single active user session being appended to resolve a downstream review finding (T22b). The intervening `a6d82db` did a V1→loom 1.0 rename pass and `81bf52a` moved the file under `future-considerations/`; neither altered the bullet's structure. Both content commits were responses to spec-review findings demanding the non-goal "answer" how a future Pi minor would be caught, and each answered by adding more contingency prose inline rather than relocating it. The defect is therefore the *pattern* of resolving "detection-mechanism unspecified" review findings by inlining the mechanism here, rather than any single commit.

## Solution approach

Narrow the bullet to its load-bearing single-active-session statement plus the existing *Recorded at:* citations, forwarding to the pages that own each detection mechanism instead of restating them inline. Relocate the surface-inventory presence-check scope material to `pi-integration-contract/version-bump-step2.md` at the step 2(a) surface-inventory assertion description, and place the no-concurrent-session-probe disposition at `capability-probe.md`'s Step 0 scope (`#entry-capability-probe`). Preserve the T22b "Disposition for an unanticipated second session" enumeration of single-active-session sites — fold it into SM-1 (`#sm-1-single-active-session-binding`) or keep the "no new disposition" sentence in the narrowed bullet.

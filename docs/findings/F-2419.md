## Problem

The *Category-(1)/(3) inventory join key* rule in `audit-resolution.md` matches a `pi.<member>` or `ctx.<member>` access against the rightmost dot-separated segment of an entry's `path` field with no restriction on the entry's carrier category, so a `pi.<member>` access can false-resolve against a category-(3) `ctx.*` entry (and vice versa) whenever a member name is shared across carriers (e.g. a `shutdown` declared on both surfaces), leaving resolution non-deterministic as specified. The same omission corrupts the cross-category `duplicate-rightmost-segment` uniqueness rule in the *Category-(2) inventory join key* paragraph: a legitimate category-(1) `pi.shutdown` and category-(3) `ctx.shutdown` share the rightmost segment and collide, firing a false `literal-read/duplicate-rightmost-segment` failure that forbids a correct inventory. Separately, the category-(3) resolution prose in `audit-target-categories.md` cross-references the leftmost-segment anchor `#audit-category-2-inventory-join-key` (with a stale "(generalised to category (3) per this paragraph)" clause) instead of the rightmost-segment `#audit-category-1-3-inventory-join-key` that actually governs category (3); the minimum-entry-shape paragraph's hyperlink on the same page mis-points identically.

## Solution approach

Narrow the *Category-(1)/(3) inventory join key* rule in `audit-resolution.md` so a category-(1) `pi.<member>` access resolves by rightmost-segment match only against category-(1) entry kinds and a category-(3) `ctx.<member>` access only against category-(3) entry kinds, keyed off the entry `kind` discriminator the minimum-entry-shape obligation already pins. Narrow the cross-category `duplicate-rightmost-segment` uniqueness rule in the *Category-(2) inventory join key* paragraph so two entries collide only when they share both the rightmost segment and the carrier category. In `audit-target-categories.md`, repoint the category-(3) resolution cross-reference and the minimum-entry-shape paragraph's hyperlink from `#audit-category-2-inventory-join-key` to `#audit-category-1-3-inventory-join-key`, and drop the stale "(generalised to category (3) per this paragraph)" clause.

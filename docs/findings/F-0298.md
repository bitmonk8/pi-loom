## Finding

`V17a` (Cancellation core) carries eleven Tests bullets drawn from three spec topics (`cancellation.md`, `host-interfaces-core.md`, `patch-skew-degradation.md`) and folds several independently-buildable, independently-verifiable mechanisms under one Ships-when gate. The bundled modules are: (1) the `loomAbort` controller plus signal-forwarding wiring and downward-only propagation; (2) the CNCL late-settlement discard invariants (CNCL-1/2/3/5/6); (3) checkpoint-granularity (the presence arm over the five sites, the best-effort negative arm, and the loop-iteration macrotask-yield); and (4) swallowing-handler substrate suppression at the `Checkpoint` seam.

The plan's own atomicity rule (`conventions.md` §Vertical slices: "each leaf inside a slice is the smallest feature that can ship and be tested independently") is the standard these modules fail against. The slice already demonstrates the corrective pattern: `V17b` was peeled out of `V17a` as a "distinctly complex, independently-verifiable sub-mechanism." The same reasoning applies to the swallowing-handler substrate suppression and the checkpoint-granularity arms, which share only the `loomAbort` controller and `Checkpoint` seam with the rest of the leaf and are otherwise orthogonal.

The four modules cannot be sequenced, parallelised, or partially landed: a single Ships-when gate requires all of them green at once, so a defect in any one arm blocks the entire leaf and produces a large, hard-to-review diff.

## Solution Space

**Shape:** multiple

This finding carries two coupled obligations — decompose the leaf, then repair every reference to it. The split defines new leaf IDs that the retargeting depends on, so the obligations must land in order; doing both in one edit produces an oversized diff. One option per obligation follows.

### Option A — Decompose V17a's bundled modules into separate leaves

- **Approach:** Peel `V17a`'s independent verification surfaces into separate leaves, following the precedent already set by the `V17b` throw-trap peel. The candidate modules are (1) `loomAbort` controller + signal forwarding + downward-only propagation; (2) the CNCL late-settlement discard invariants (CNCL-1/2/3/5/6); (3) checkpoint-granularity (presence arm + negative arm + loop-iteration macrotask-yield); (4) swallowing-handler substrate suppression. New IDs continue the slice's `V17<letter>` form; use the `<new>` placeholder until the fixer allocates real IDs (no-invented-IDs rule).
- **Plan edits:** Move the corresponding Tests bullets and matching Ships-when clauses out of `V17a-cancellation-core.md` / `V17a-T-cancellation-core.md` into the new leaf files; retain the remaining core in `V17a`. Each new leaf cites only the spec topics its bullets reference (the swallow leaf cites `cancellation.md`; the wiring leaf cites `cancellation.md` + `host-interfaces-core.md`; the checkpoint leaf cites `cancellation.md` + `host-interfaces-services.md`). Add the new leaf links to the V17 slice in `docs/plan.md`.
- **Spec edits:** None.
- **Pros:** restores leaf atomicity; each sub-mechanism becomes independently buildable and testable; smaller per-leaf diffs.
- **Cons:** adds new leaf files plus their `-T` partners to maintain.
- **Risks:** CNCL-4 spans wiring (forwarding paths) and abort-reason identity, so its bullet must land with the wiring leaf to avoid splitting one REQ-ID across leaves without a corresponding coverage-matrix many-to-many entry.

### Option B — Retarget dependents and coverage rows onto the split

- **Approach:** After the new leaf IDs exist, re-point every Deps edge and coverage-matrix row that currently names `V17a` / `V17a-T` to the specific sub-leaf each consumer actually depends on.
- **Plan edits:** Update Deps in `V17b`/`V17b-T`, `V9c`/`V9c-T`, `V9g`/`V9g-T`, `V9i`/`V9i-T`, `V11j`/`V11j-T`, `V4c`/`V4c-T`, `V18d`, `H7a`, and the `H5b` Deps union — each naming the sub-leaf providing what it consumes (controller/wiring for `V9c`/`V9g`/`V17b`; `Checkpoint` seam for `V11j`; the swallow site for `V9i`; the cancellation seam for `V4c`). Update `coverage-matrix.md` rows for CNCL-1/2/3/5/6 and CNCL-4 to name the owning sub-leaf(s); CNCL-4's row must continue to name both its `V17a`-side owner and `V9g`.
- **Spec edits:** None.
- **Pros:** keeps the dep DAG and coverage matrix accurate after the split; prevents stale/dangling Deps.
- **Cons:** broad touch across roughly ten leaves plus the coverage matrix.
- **Risks:** a missed consumer leaves a dangling Deps reference to a non-existent or wrong-scope leaf; broad-union dependents (`H5b`, `H7a`, `V18d`) may need to name multiple sub-leaves.

### Recommendation

Resolve Option A before Option B. Option A is scope-bounding: it fixes the new leaf set and IDs that Option B's retargeting consumes, so landing A first means B's propagation pass runs against a stable baseline rather than co-evolving IDs. Following the `V17b` precedent, prefer peeling the distinctly-complex, independently-verifiable sub-mechanisms (the swallowing-handler substrate suppression and the checkpoint-granularity arms) so the retained `V17a` core stays coherent. Throughout, keep CNCL-4's forwarding-path facet and reason-identity facet with the wiring leaf, and keep its `V9g` co-owner entry intact in `coverage-matrix.md`.

## Consequence

**Severity:** advisory

If `V17a` ships as one leaf, an implementer must land controller wiring, the CNCL discard invariants, checkpoint-granularity, and `Checkpoint`-seam substrate suppression in a single task gated by one Ships-when condition. The orthogonal sub-mechanisms cannot be sequenced or parallelised, a defect in any one arm blocks the whole leaf, and the resulting diff is large enough to be hard to review. A working leaf is still producible, so the gap is guidance-level rather than blocking.

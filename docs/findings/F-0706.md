## Problem

The hub paragraph at the top of `docs/spec_topics/governance.md` (line 5) enumerates the retired GOV-rule set inline (`GOV-2, GOV-10, GOV-11, GOV-12, GOV-13, GOV-14 retired`) and asserts a live range "GOV-3 through GOV-9". This contradicts the canonical Retired REQ-IDs table at `governance/anchor-scheme-and-retired.md#retired-req-ids`, which also lists `GOV-4` and `GOV-21` as retired: the inline enumeration omits both, and the inclusive "GOV-3 through GOV-9" range positively asserts that the retired `GOV-4` is live. The stale inline list precedes its own "see Retired REQ-IDs" pointer, so a reader meets the contradicting enumeration before the canonical table.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `1d773d6` (2026-06-04, retired GOV-4); `8e3ccb4` (2026-06-05, retired GOV-21)
**History:**

`git log -L 5,5:docs/spec_topics/governance.md` shows the hub paragraph has been touched five times since its current shape was set in `64cdc60` (2026-05-26), and every edit since has only added new live IDs (`GOV-15`, `GOV-16`, `GOV-30`, `GOV-31`) or moved an ID between live and retired within the closed set `{GOV-2, GOV-10, GOV-11, GOV-12, GOV-13, GOV-14}` — most recently `5a772e3` (2026-06-05) which moved GOV-14 to retired.

`git show 1d773d6 --stat` reveals the retiring commit for GOV-4 edited `governance/anchor-scheme-and-retired.md`, `governance/req-id-prefix-table-active-a.md`, `governance/req-id-prefix-table-active-b.md`, `governance/stable-inline-labels.md`, and `hard-ceilings.md` — but did **not** touch `governance.md` or `session-model-and-appendix.md`. `git show 8e3ccb4 --stat` for GOV-21 likewise edited only `anchor-scheme-and-retired.md` and `release-version-naming.md`. Neither retirement commit propagated its effect into the hub paragraph's inline enumeration or the session-model appendix bullet that mirrors it.

The drift is therefore the joint residue of two independent retirement commits, each of which correctly updated the canonical table but left the editorial restatement on the hub page (and its appendix mirror) untouched. The current `session-model-and-appendix.md` was created at `81bf52a` (2026-06-11) by mechanical move from the pre-existing `spec/session-model-and-appendix.md`, inheriting the already-stale enumeration.

## Solution approach

Rewrite the hub-paragraph parenthetical on `governance.md` line 5 so it stops restating the retired GOV set inline and instead defers to the canonical [Retired REQ-IDs](./governance/anchor-scheme-and-retired.md#retired-req-ids) table. If the inline enumeration is editorially load-bearing and must be retained, correct it to match the canonical table instead — replace the "GOV-3 through GOV-9" range with explicit live IDs so the retired `GOV-4` is no longer asserted live, and add `GOV-4` and `GOV-21` to the retired set.

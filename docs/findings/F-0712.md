## Problem

`host-prerequisites.md` is the dedicated collection page for loom's "treats it as a presupposition" host-behavioural framing, and it indexes only the seven presuppositions it owns. A further ~twelve presuppositions using the same framing live only on the topic pages that define the SDK surfaces they constrain â€” `host-interfaces-core.md`, `host-interfaces-services.md`, `registration-steps.md`, `conversation-drive.md`, `subagent.md`, `provider-error-mapping.md`, and `runtime-event-channel.md` â€” with no inbound pointer from `host-prerequisites.md`. An auditor enumerating loom's unpinned host-behavioural presuppositions from that page therefore sees only the seven it owns and silently undercounts the rest, which are reachable only by walking out from `version-bump-step2.md`'s editorial-review checklist items or reading the topic pages directly.

## Issue introduction

**Verdict:** present-since-inception (for the cluster) / multi-commit-accretion (for the specific gap)
**Introducing commits:** the establishment of `host-prerequisites.md`'s "treats it as a presupposition" framing predates the spec's current size-cap split (`f5e89f4` of 2026-06-04 mechanically split `pi-integration-contract.md` into the topic-page tree the finding observes), but the scattered presuppositions accumulated across multiple later resolve commits â€” notably `aebec4e` (2026-06-04, added `#messages-chronological-order-presupposition`), `cb3a2f2` (2026-06-04, widened it to surface (2) and added the subagent surface), `ce4e476` (2026-06-06, added `#messages-leading-user-message-presupposition`), `e89e42e` (2026-05-17, established `#pi-slash-handler-promise-lifecycle-presupposition` on the pre-split file), `7f07a96` (2026-06-05, added presupposition (v)), `fdc5115` (2026-06-06, added presupposition (vi)), and similar commits that authored `#cwd-project-root-presupposition`, `#provider-overflow-wording-presupposition`, `#complete-*`, `#dispose-mid-abort-presupposition`, `#register-tool-post-startup-presupposition`, `#getcommands-completeness-presupposition`, and `#custom-message-context-entry-presupposition` on their respective topic pages.
**History:** Each presupposition was authored on the topic page that defines the SDK surface it constrains (the natural local home) and was reflexively wired into `version-bump-step2.md`'s checklist as the per-bump detection gate. `host-prerequisites.md`'s own presupposition cluster (degraded-state (b)â€“(d), model-registry-population, settings-write-back, operator-always-present) grew on the same timeline (`79fa3b9`, `66da202`, `bfef26e`, `6caacfe`, `60fba43` â€” all 2026-06-04/05/07), and the "in the same sense as the â€¦ prerequisite above" chain was maintained inside that cluster but never extended to back-reference the topic-page-resident siblings. `803106d` (2026-06-07, "Pi behavioural presuppositions lack authoritative behavioural pointers") added *Authoritative source* / known-fragile-evidence sentences to most of these presuppositions, confirming that the corpus-wide pattern was recognised, but did not address the inbound-pointer gap. The defect is therefore an accretion gap: each commit added a presupposition correctly on its owning page and correctly to the version-bump checklist, but the dedicated collection page on `host-prerequisites.md` was never updated to index the topic-page-resident set.

## Solution approach

Add an index paragraph to `host-prerequisites.md` after `#operator-always-present-presupposition` that links each topic-page-resident behavioural presupposition by anchor, grouped by owning topic page, and names `version-bump-step2.md`'s editorial-review checklist as the per-bump detection gate. Add a forward-link from `version-bump-step2.md`'s closing maintenance sentence requiring the `host-prerequisites.md` index be updated whenever a new topic-page presupposition is authored.

## Solution constraints

- Out of scope: relocating or restating the presupposition declarations on their owning topic pages (`host-interfaces-core.md`, `host-interfaces-services.md`, `registration-steps.md`, `conversation-drive.md`, `subagent.md`, `provider-error-mapping.md`, `runtime-event-channel.md`) â€” they remain the normative home and the index links to them only.

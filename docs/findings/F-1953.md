## Finding

`V18d`'s **Adds** states the revert path for two distinct red-signal arms. The first arm is explicit — "when the double-backed runtime-evidence run is red the restore is pre-merge (the bump MUST NOT merge at the candidate pin)". The second arm folds back to the first by reference: "when the bump's **manual real-host smoke** … surfaces a confirmed behavioural-divergence finding the restore is **likewise pre-merge** (the smoke runs before merge, so the divergent pin never lands on `main`)."

The word "likewise" carries two readings the sentence does not disambiguate: (A) a *timing* reference — the restore happens before merge, exactly as the runtime-evidence arm does; (B) an *obligation-strength* reference — the restore is equally mandatory. Both readings terminate at the same observable outcome (a pre-merge restore), and the trailing parenthetical "(the smoke runs before merge…)" supplies the timing fact, so no implementer is actually steered to a divergent action. The cost is purely that the operative rule — restore-before-merge — is reached by inference across the "likewise" anaphor rather than stated outright, which is a precision regression against the explicit phrasing of the first arm in the same sentence.

Its diff shows the pre-existing two-arm Adds wording distinguished a `pre-merge` restore (runtime-evidence-red arm) from a `post-merge revert commit` (real-host-smoke arm). `15eb44c` collapsed both arms onto a unified pre-merge posture and, in doing so, rewrote the second arm to reuse the first via the connector "likewise pre-merge". The ambiguity is an artefact of that posture-unification rewrite; it did not exist in the earlier two-arm phrasing (which named each timing explicitly).

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V18d-version-bump-acceptance.md` **Adds**, replace the clause `the restore is likewise pre-merge` with `the restore is also required before merge`. This states the restore-timing rule of the real-host-smoke arm directly rather than by anaphora to the runtime-evidence arm. Retain the existing trailing parenthetical `(the smoke runs before merge, so the divergent pin never lands on \`main\`)`. No other field of `V18d` or `V18d-T` carries the phrase (confirmed by grep over `docs/`), so the edit is confined to this one clause.

## Consequence

**Severity:** cosmetic

Both readings of "likewise" yield the same pre-merge restore, and the trailing parenthetical already states the timing, so no implementer diverges. The defect is a wording-precision regression only — the second arm states its restore-timing rule by anaphora where the first arm states it explicitly.

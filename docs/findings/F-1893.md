## Problem

The *Transient toasts (auxiliary)* paragraph at `diagnostics/diagnostic-shape.md#transient-toasts` carries normative MUST/MUST-NOT obligations — three failure modes routed through `ctx.ui.notify(message, "error")`, the `ctx.ui.notify`-throw swallow MUST, and three per-mode no-drain-state-tag post-failure pins — that by construction carry no `loom/...` registry code and no numbered REQ-ID. The *Code-keyed obligation areas (no numbered REQ-IDs)* table in `coverage-matrix.md` has no row whose subject is this surface, and no closing leaf asserts these obligations. Because `diagnostic-shape.md` is a non-narrative page, its un-anchored MUSTs are in scope for the `H5e` un-anchored-normative-MUST scan, which reddens on this unmapped paragraph (`H5b` flags it warn-only pre-activation).

## Solution approach

Add one row to the *Code-keyed obligation areas (no numbered REQ-IDs)* table in `docs/plan_topics/coverage-matrix.md`, keyed by the next free `cka-<n>` token, whose subject is `diagnostics/diagnostic-shape.md` §*Transient toasts (auxiliary)* (anchor `#transient-toasts`) and whose *Closing leaf(s)* cell names `V10d`. Extend `V10d`/`V10d-T` to assert the surface's obligations — the three `ctx.ui.notify` failure modes, the notify-throw swallow MUST, and the three per-mode no-drain-state-tag dispositions — with Tests bullets that cite the new `cka-<n>` token and `V10d` inline per `H5f`. If folding all the obligations into `V10d` would over-grow it, allocate a dedicated `<new>` leaf and point the new row's closing-leaf cell at it instead.

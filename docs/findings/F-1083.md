## Problem

`V9g` (session-shutdown teardown) and its paired `V9g-T` are built on timing behaviour the spec pins to the injected `Clock` seam: `SHUTDOWN_AWAIT_CAP_MS` bounds the sub-step-3 `Promise.allSettled` await, `loom/runtime/reload-teardown-timeout` fires at that cap, and `patch-skew-degradation.md` cancels pending debounce handles via `Clock.clearTimeout`. The `Clock` seam is owned by `V8d`, and the PIC-12 architectural test bans direct ambient timing outside the `WallClock` adapter, so the only conformant way to read the cap or call `clearTimeout` is through that seam. Neither `V9g` nor `V9g-T` lists `V8d` in its `Deps.`, so the pickability predicate permits them to be picked before `V8d` lands — at which point the `FakeClock`-driven timeout-at-cap test has no seam to drive. Sibling leaves consuming the same seam (`V9e`, `V10d`) declare `V8d` explicitly; `V9g` is the outlier.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** c6a664e ("pi-loom plan: build/update plan for spec.md + review", 2026-06-10)
**History:** `git log` over `docs/plan_topics/V9g-session-shutdown.md` shows the leaf was created in `c6a664e` already carrying both the `Clock`-backed timing content (`SHUTDOWN_AWAIT_CAP_MS`, `reload-teardown-timeout`, the sub-step-4 `Clock.clearTimeout`) and a `Deps.` line that omitted `V8d` (`git log -S "SHUTDOWN_AWAIT_CAP_MS"` confirms the timing text is original to that commit; `git log -L` on the `Deps.` line shows it was born as `V9g-T, V9e, V9h, V17a`). A later edit, `a6e2ecb` (2026-06-17), revised the same `Deps.` line to add `V9a` but still did not add `V8d`. The omission has therefore been present since the leaf first existed; it was not introduced by any later edit.

## Solution approach

Add `V8d` to the `Deps.` field of both `docs/plan_topics/V9g-session-shutdown.md` and `docs/plan_topics/V9g-T-session-shutdown.md`. In `V9g`'s `Adds.`, attribute the `SHUTDOWN_AWAIT_CAP_MS` cap, the `reload-teardown-timeout` emission at the cap, and the sub-step-4 `Clock.clearTimeout` call to the injected `Clock` seam that `V8d` owns, and note that the `V9g-T` timeout-at-cap test is driven via the `V8d` `FakeClock`.

## Solution constraints

- The spec is read-only for this fix — `patch-skew-degradation.md` already pins the cap to the `Clock` seam, so no spec edit is required.

## Finding

GOV-29 (*cross-page canonical-arm uniqueness*) closes with a worked example naming the collision it disambiguates: *"The known instance … is `v1-non-goals` (present on both `docs/spec.md` and `docs/spec_topics/future-considerations.md`)."* The same rule's text also states the disambiguating tail *"`loom-1-0-non-goals-aggregator` for the orientation copy that aggregates the authoritative `loom-1-0-non-goals` on `future-considerations.md`."* Neither named file hosts the relevant anchor.

The actual layout in the corpus is: the canonical `loom-1-0-non-goals` arm (paired with the `v1-non-goals` alias) lives on `docs/spec_topics/future-considerations/model-changes-and-non-goals.md`, and the disambiguated aggregator arm `loom-1-0-non-goals-aggregator` (also paired with `v1-non-goals`) lives on `docs/spec_topics/language-and-architecture.md`. `docs/spec.md` carries no `v1-non-goals` or `loom-1-0-non-goals` anchor at all (it is a three-link hub), and `docs/spec_topics/future-considerations.md` is an index page whose non-goals content is a forward-link, not an anchor host.

A maintainer applying GOV-29 — checking that the two colliding sites carry the bare canonical arm at the GOV-30 normative-owner and the tail-marked arm at the aggregator — would open the two files GOV-29 names and find neither anchor present. The worked example points at the wrong sites and references a non-existent `future-considerations.md` anchor home.

## Issue introduction
**Verdict:** introduced-by-later-commit (stale-after-refactor)
**Introducing commit:** `f5e89f4` — *docs: enforce 100KB hard / 30KB recommended size cap on spec set*
**History:** The worked example was authored in `44c7c19` (2026-05-29, *extend GOV-21 to cover all version-segment positions + cross-page uniqueness*) as new sub-clause 5 of the then-`GOV-21` rule. At that revision the example was accurate: `docs/spec.md` carried `<a id="v1-non-goals">` (verified at `44c7c19:docs/spec.md` line 97) and `docs/spec_topics/future-considerations.md` carried it (line 96). The 100KB-cap split `f5e89f4` (2026-06-04) then relocated both anchors — the `spec.md` aggregator copy moved to the newly-created `language-and-architecture.md` (verified: `language-and-architecture.md` did not exist before `f5e89f4`), and the `future-considerations.md` canonical copy moved to the new child page `future-considerations/model-changes-and-non-goals.md`. That commit mechanically repointed cross-references but carried GOV-29's worked-example *prose* over verbatim (still naming `docs/spec.md` and `docs/spec_topics/future-considerations.md`), and the subsequent governance renumber to GOV-25–GOV-29 preserved the stale text. The defect is therefore a single-commit stale-after-refactor introduced by `f5e89f4`, not present since the example's inception.

## Solution Space

**Shape:** single

### Recommendation

Correct GOV-29's worked-example sentence to name the files that actually host the collision: the canonical owner `docs/spec_topics/future-considerations/model-changes-and-non-goals.md` and the aggregator copy `docs/spec_topics/language-and-architecture.md`. Likewise update the `loom-1-0-non-goals-aggregator` tail illustration so the "authoritative `loom-1-0-non-goals`" is attributed to `future-considerations/model-changes-and-non-goals.md` rather than the bare `future-considerations.md` index page. Keep the rule's normative content unchanged; only the file references in the illustrative clause change.

Edge case for the fixer: the GOV-30 normative-owner of the non-goals aggregation is `future-considerations/model-changes-and-non-goals.md` (which holds the bare `loom-1-0-non-goals` arm), and `language-and-architecture.md` is the aggregator copy (holding the `-aggregator`-tailed arm) — keep that owner/aggregator direction consistent with GOV-29's own "normative-owner retains the bare arm, aggregator takes the tail" rule when rewording.

## Consequence

**Severity:** advisory

A maintainer auditing or extending the dual-anchor collision set per GOV-29 is directed to two files that do not host the `v1-non-goals` collision and to a `future-considerations.md` anchor home that does not exist; they must rediscover the real sites by grep before the rule can be applied. The normative uniqueness obligation itself remains clear, so no implementation diverges — only the illustrative example is stale.

## Problem

PIC-22's parallel-spawn conformance witness in `subagent.md` is under-specified on two independent points. The `N (for any fixed N ≥ 3)` lower bound carries no justification, so a reader cannot tell whether `N = 2` is excluded because two parallel calls are observationally insufficient or merely because a stronger witness was preferred — the threshold actually discriminates a fixed-`2` admission cap from a serialise-one-at-a-time scheduler, but the page never says so, leaving the choice indistinguishable from a typo. Separately, the witness phrase "entered each session's `sendUserMessage`" is observationally ambiguous between the call being *issued* (the `sendUserMessage` spy was invoked and its promise constructed — externally observable on a fake `AgentSession`) and control flow *entering the method body* (an internal SDK property a black-box test cannot observe). Two test authors can therefore write non-portable assertions while both believing they conform to PIC-22.

Commit 7a3b907 later split the paragraph into a standalone `<a id="pic-22"></a>` block when the page was sharded into `subagent.md`, preserving both unjustified phrasings; the wording has been unchanged since.

## Solution approach

Clarify the PIC-22 witness sentence under `<a id="pic-22"></a>` in `subagent.md` on both points. Add a justification clause that names the scheduler shape the larger threshold is meant to falsify — a fixed-`2` admission window, which a binary `N = 2` witness would leave undetected — keeping the lower bound at `3`. Rewrite the "entered each session's `sendUserMessage`" clause to the call-issued observable (a black-box test reads "`sendUserMessage` was called on each session" off the fake `AgentSession` spy). Propagate the same call-issued wording to the two plan leaves that restate the witness — `docs/plan_topics/V9i-subagent-isolation.md` and `docs/plan_topics/V9i-T-subagent-isolation.md` — leaving their `N = 3` pin unchanged.

## Problem

PIC-30 in `drain-state-contract.md` is framed in implementation terms — it forbids a third boolean drain-state field, an arm-specific gate on `LoomRegistry.readDrainState`, or any other suppression mechanism. None of these are inspectable by a conformance test, which can observe only slash-dispatch behaviour. The prohibition does have a concrete observable consequence: after the all-three-throw corner case leaves the registry at `(drained: false, tag: undefined)`, a subsequent slash invocation routes through arm (a) of `readDrainState` and dispatches the loom, where a suppression gate would instead short-circuit. No fixture witnesses this: the `## Acceptance criteria` block (a)–(m) in `session-only-degraded-state.md` covers every other branch, and fixture (b) explicitly defers multi-site failure to the all-three-throw corner case — which is never instantiated as a fixture.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** 52b9101 ("pi-loom spec: resolve T24 — Fork-reason watcher closure leaves the extension in an unspecified, silently degraded state", 2026-05-11)
**History:** Commit 52b9101 introduced both the all-three-throw corner case (the first appearance of the `all-three-throw` token in the corpus) and the per-branch acceptance-criteria fixture list (the "Positive — clean session-only reason" set) in the same edit, inside the then-monolithic `pi-integration-contract.md`. The corner case was documented as a loom-1.0 acceptance with observable consequences but received no fixture at inception, while every sibling branch did. Later edits extended the fixture set without closing the gap: f5e89f4 (2026-06-04, the spec-set size-cap split that created `session-only-degraded-state.md`) added fixture (b)'s "covered only by the all-three-throw corner case" out-of-scope marker — naming the corner case as the coverage vehicle — and 2775382 (2026-06-05) added the read-failure fixtures (l)/(m) citing the corner case as their shared acceptance footing. Across all three commits the named coverage vehicle was never instantiated.

## Solution approach

Add an acceptance fixture witnessing the all-three-throw corner case to the `## Acceptance criteria` block of `session-only-degraded-state.md` (the page owning the corner-case prose and the (a)–(m) fixture set), and add a forward-link from `drain-state-contract.md`'s PIC-30 (`#pic-30`) to it so the prohibition carries a witnessing criterion. The fixture drives the all-three-throw tuple (`(drained: false, tag: undefined)`) and asserts the behaviourally observable outcomes that distinguish a conformant runtime from one carrying a suppression seam: a slash invocation routes through arm (a) of `readDrainState` and dispatches the loom, and a follow-up session-only `session_shutdown` emits an additional `loom/host/session-shutdown-runtime-degraded` row.

## Solution constraints

- Out of scope: witnessing PIC-30 by internal-state inspection — the fixture MUST NOT assert the non-existence of an internal drain-state field, gate, or suppression seam, since those are not test-observable.

## Problem

The Baseline acquisition block (`#bump-baseline-acquisition` in
`version-bump-step2.md`) presupposes the pre-bump pinned version is still
published and reachable from the registry at bump time, but never states this
as a precondition and specifies no fallback "from" source for when it is not.
The block also forecloses the obvious fallback by declaring the extracted tree
"not retained as an in-repo snapshot" and MUST-NOT-commit.

## Issue introduction
**Verdict:** single-commit
**Introducing commits:** ccc6e38 — "pi-loom spec: resolve \"Item (e)/(r) dist/*.js baseline source unspecified\"" (2026-06-07, Thomas Andersen)
**History:** `git log -S 'npm pack @earendil-works' -- docs/spec_topics/pi-integration-contract/version-bump-step2.md` returns only `ccc6e38`. Before that commit the diff baseline was described only abstractly as the loom 1.0 Pi-SDK pin's `dist/*.js` snapshot, with no concrete acquisition mechanism. `ccc6e38` created the **Baseline acquisition** block, introducing in one edit both the `npm pack @...@<pinned-version>` command-as-MUST and the ephemeral / "not retained as an in-repo snapshot" / MUST-NOT-commit rule, while resolving a prior review finding about the unspecified baseline source. Both defects — the command-verb-as-MUST and the unstated fetchability precondition with no fallback — entered together at the block's inception.

## Solution approach

Clarify the `#bump-baseline-acquisition` block to either state the
registry-fetchability precondition or specify a retained fallback baseline.
Either add an explicit precondition that the pre-bump pinned version (read at
`#pi-sdk-pin`) must be fetchable at bump time, with a defined contributor
action when it is not; or define a retained fallback baseline of the pinned
minor's `dist/*.js` tree and amend the block's "not retained as an in-repo
snapshot / MUST NOT be committed" sentence accordingly.

## Solution constraints

- If a retained fallback baseline is chosen, the existing "not retained as an
  in-repo snapshot / MUST NOT be committed" sentence MUST be reconciled rather
  than left standing alongside it.
- Out of scope: the `npm pack`-command-verb-as-MUST over-prescription on the
  same block (owned by the sibling split finding); this finding adds only the
  precondition/fallback.

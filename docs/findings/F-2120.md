## Problem

In the *Hang handling* clause of `conversation-drive.md`, the cancellation short-circuit rule closes with "…even when Pi tore down cleanly and `waitForIdle()` resolved without any error written to the session." The phrase "Pi tore down cleanly" has no specified referent: in context it means the cancelled in-flight user run completed teardown cleanly, but it reads equally as Pi itself (the extension host or session) undergoing a clean teardown such as `/reload` or `session_shutdown` — a distinct event handled elsewhere in the contract. The defect is confined to the explanatory "even when…" clause; the normative `loomAbort.signal.aborted` ⇒ synthesise `cancelled` rule it motivates is itself unambiguous.

## Issue introduction
**Verdict:** single-commit
**Introducing commits:** d54d798 ("pi-loom spec: resolve \"waitForIdle resolution, error, and hang semantics not contracted\"", 2026-05-06)
**History:** The corpus is git-tracked. `git log -S 'tore down cleanly'` / `-G 'even when Pi tore down cleanly'` and `git show d54d798` confirm that d54d798 first authored the prompt-mode Hang-handling cancellation short-circuit clause, introducing the phrasing "even when Pi tore down cleanly and `waitForIdle()` resolved without any error written to the session"; the parent tree contained no occurrence of the phrase in this content. The clause did not exist before this commit (its title — "resolve waitForIdle … hang semantics not contracted" — marks the first time hang semantics were contracted), so the ambiguity is present since the clause's inception. Later commits (e.g. f5e89f4's 100KB size-cap split that relocated the content into `conversation-drive.md`, and the PIC-50/51 transport-mapping edits) carried the wording forward unchanged rather than introducing or altering it.

## Solution approach

Rewrite the explanatory "even when Pi tore down cleanly…" clause in the *Hang handling* paragraph so its subject is named as the cancelled user run's clean teardown rather than the unqualified "Pi tore down cleanly", removing the host/session-teardown reading while preserving the clause's role as motivation for the precedence rule.

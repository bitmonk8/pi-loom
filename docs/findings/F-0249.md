## Finding

`V4e`'s `ERR-16` arm is "the slash-load `params` arm of ceiling #4," and `V4e` Adds describes it as the live load-time wrapping site for that ceiling's breach. Ceiling #4 is JSON document depth; the loom-owned depth walk that detects a depth-6 breach and computes the per-boundary routing decision is owned by `V5e`. `V5e` is explicit that the slash-load `params` row is "decision-only here" and that "the actual wrapping of a depth-6 breach into each carrier is asserted at the site owner: ... the slash-load cross-route at `V4e`." So `V4e` consumes the depth-detection output / routing decision that `V5e` produces.

`V4e` Deps are `V4e-T, V9a, V6a, V11f, V10a, V16a, V9b, V10d`, and `V4e-T` Deps mirror that list minus the self-reference. Neither names `V5e` (nor `V5d`, which `V5e` builds on). None of the declared deps transitively reach `V5e` — `V16a`'s deps are `V16a-T, V9d`, and `V16a` is a pure cross-ceiling arbitration seam that computes only the surfacing-order / `masked` decision; it runs no depth check and is not a substitute depth-detection source.

Because the prerequisite is undeclared and unreachable, a literal dependency graph can schedule `V4e` before `V5e` exists, leaving `ERR-16` with no depth-detection source. The leaf does not state whether `ERR-16`'s depth breach is meant to come from `V5e`'s live walk or from a synthetic signal injected at `V4e`'s load-time site, so two implementers would resolve the gap differently.

## Solution Space

**Shape:** multiple

### Option A — Declare the V5e dependency

- **Approach:** Add `V5e` to `V4e` and `V4e-T` Deps so the depth-walk producer precedes `V4e` in build order; `ERR-16` then drives `V5e`'s live depth-walk emission and routing decision.
- **Plan edits:** `V4e` Deps and `V4e-T` Deps each gain `V5e`. Adding `V5e` transitively pulls `V5d` (its own dep), so `V5d` need not be listed separately. Name the consumed `V5e` depth-walk output in `V4e` Adds so the seam binds per the conventions Class-2 consumer-names-in-Adds rule (`V4e` already names the `V16a` seam this way).
- **Spec edits:** None.
- **Pros:** Removes the ordering hazard directly; `ERR-16` exercises the real ceiling-#4 emission; matches `V5e`'s statement that the slash-load wrapping is "asserted at the site owner ... `V4e`."
- **Cons:** Lengthens `V4e`'s prerequisite chain by the `V5e`→`V5d` edge.
- **Risks:** `V4e` and `V4e-T` Deps must stay in lockstep.

### Option B — Drive ERR-16 from a synthetic depth-breach signal

- **Approach:** Keep `V4e` free of a live `V5e` edge; declare that `ERR-16`'s depth-6 breach is injected synthetically at `V4e`'s load-time site, mirroring `V5e`'s isolation harness, `V16a`'s synthesised-candidate strategy, and `V4e`'s existing `ERR-7` idiom ("without standing up a live `V10d`/`V9b` watcher").
- **Plan edits:** `V4e` `ERR-16` Tests bullet states the depth breach is synthesised rather than produced by a live `V5e` walk; `V4e` Adds notes the depth-walk emission contract it mirrors is owned by `V5e` (read-only reference, no Deps edge).
- **Spec edits:** None.
- **Pros:** Keeps `V4e` pickable independent of `V5e`; consistent with the plan's pervasive isolation pattern.
- **Cons:** `V4e` still depends on `V5e`'s emitted-surface contract by reference; with no build-order link, a contract drift is not caught at `V4e`.
- **Risks:** Drift between `V5e`'s `maxDepth` emission and `V4e`'s synthetic fixture surfaces only at `H7a` integration.

### Recommendation

Take Option A. The finding is an ordering hazard, and adding `V5e` to both `V4e` and `V4e-T` Deps closes it directly while matching `V5e`'s own statement that the slash-load cross-route is asserted at `V4e`. Edge cases for the implementer: add `V5e` only — it transitively pulls `V5d`, so do not add a redundant `V5d` entry; keep the `V4e` and `V4e-T` Deps lists identical apart from the `V4e-T` self-reference. If the team instead prefers isolation consistency, Option B is acceptable but must reference `V5e`'s emission contract so the synthetic fixture has a named source of truth.

## Consequence

**Severity:** correctness

A literal build of the dependency graph can pick up `V4e` before `V5e` is authored, leaving `ERR-16`'s depth-ceiling breach with no producer to drive it. Two reasonable implementers diverge: one declares a `V5e` edge and drives the live walk; another synthesises the breach at `V4e`'s load-time site — producing materially different `V4e` Deps and test wiring for the same arm.

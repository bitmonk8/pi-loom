## Finding

`docs/spec_topics/imports.md` states the `Resolver`-seam design twice in immediate succession. The **"Resolver interface."** paragraph (the one beginning "Import-path resolution goes through a single named seam … a `Resolver` interface with the shape `resolve(spec: string, fromFile: string): string`") already carries the full statement: one named seam, exactly one loom 1.0.0 implementation (relative-path, `.warp`-required), non-relative specs (`@scope/pkg`, `/looms/...`) failing as unresolvable, the seam enabling the deferred package-style / project-rooted import extensions, and a closing non-normative note that the single-seam factoring "is a maintainability convention, not an observable conformance point."

Two bullets later the `> **loom 1.0 seam — \`Resolver\` interface.**` blockquote re-states every one of those points in near-identical wording, adding nothing normative. The blockquote's only unique payload is the two anchor tags it hosts — `<a id="loom-1-0-seam-resolver-interface">` and `<a id="v1-seam-resolver-interface">` — and `docs/spec_topics/future-considerations/surface-extensions.md` deep-links the latter (`../imports.md#v1-seam-resolver-interface`). Deleting the blockquote outright would therefore break that inbound link unless the anchors are relocated.

The fix is to delete the blockquote and move both `<a id="…">` tags onto the existing "Resolver interface." paragraph so the inbound deep-link continues to resolve and the seam content is stated once.

## Issue introduction

**Verdict:** introduced-by-commit
**Introducing commits:** `9c90227` (2026-06-06, "pi-loom spec: resolve \".warp import resolution has no diagnostic code or resolver failure contract\"")
**History:** `git log -S "MUST signal an unresolvable spec by throwing" -- docs/spec_topics/imports.md` returns exactly one commit, `9c90227`, which added the entire IMP-1 paragraph in resolution of an earlier spec-review finding about a missing diagnostic code and a missing resolver failure contract. The defect — pinning `by throwing` as the MUST mechanism — was present in IMP-1's first form and survived a later edit (`9aa73b1`, 2026-06-? "Filesystem case-sensitivity for .warp basenames and invoke/tools containment") which only widened the closed unresolvable-case set; the leading mechanism clause was never revisited. The companion "loom 1.0 seam — `Resolver` interface" boxed note had its observable-only disclaimer ("not an observable conformance point") added one commit earlier in `8362824` (2026-06-04, "pi-loom spec: resolve \"Demote over-prescriptive and non-testable MUSTs on unobservable implementation structure\""), so the MUST-on-throw was introduced two days *after* the page had already adopted the very principle it violates — a missed application of an active convention rather than an interaction effect.

## Solution Space

**Shape:** single

### Recommendation

Delete the `> **loom 1.0 seam — \`Resolver\` interface.**` blockquote from `imports.md`. Relocate both anchor tags it carried — `<a id="loom-1-0-seam-resolver-interface"></a>` and `<a id="v1-seam-resolver-interface"></a>` — onto the retained "Resolver interface." paragraph (placing them at the paragraph's start is sufficient; anchor position within a block does not affect fragment resolution). Both must move, not just one: `surface-extensions.md` links `#v1-seam-resolver-interface`, and the `loom-1-0-seam-*` form is the spelling that matches the visible heading text used elsewhere. After the move, verify no other corpus file links either fragment expecting the blockquote to remain a distinct element (current grep shows only the one inbound link, to `#v1-seam-resolver-interface`).

## Consequence

**Severity:** cosmetic

No observer is affected: the seam is explicitly "not an observable conformance point" and both copies agree today. The cost is editorial — a maintainer who later updates one statement risks leaving the other stale, and a reader encounters the same content twice in a few lines.

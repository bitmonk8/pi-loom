## Finding

The version-bump re-audit checklist in `version-bump-step2.md` contains several SHOULD-level editorial audits that direct the contributor to read or diff a Pi-internal JavaScript body at a named `dist/core/*.js` path against the loom 1.0 baseline:

- **(r)** `convertToLlm` ‚Äî "by diffing the `convertToLlm` transformer in `dist/core/messages.js`".
- **(aj)** `SessionManager.inMemory` transcript-non-retention ‚Äî "read the `SessionManager.inMemory` in-memory-session branch's JS body in `dist/core/session-manager.js`".
- **(am)** `createAgentSession` tools-allowlist suppression ‚Äî "read the `createAgentSession` `tools`-field handling branch's JS body in `dist/core/sdk.js`".

These three items name the candidate-minor `dist/core/*.js` path as the audit locus and stop there. The structurally parallel audits in the same checklist treat the cited path as *illustrative, known-fragile evidence* and supply an explicit fallback when the path no longer textually resolves:

- Item **(e)** carries a *"Textual-locatability with no equivalent site"* fail condition that fires only when the cited `dist/*.js` evidence is un-locatable **AND** no equivalent reachable site can be identified ‚Äî explicitly because Pi may "minify, re-bundle, rename the `_`-prefixed symbol, or restructure the `dist/` tree".
- Item **(q)** states the cited bundled symbols and `dist/*.js` paths are "known-fragile evidence" and requires the auditor to "confirm the observable behaviours by an equivalent reachable site rather than recording `pass` on textual locatability alone".

Items (r)/(aj)/(am) carry no equivalent clause. They make the candidate-minor JS path the normative audit target rather than an illustrative locus of a behavioural confirmation, leaving the auditor without a defined action when Pi re-bundles, minifies, or restructures the `dist/` tree at a candidate minor ‚Äî exactly the fragility the parallel items anticipate. The behavioural property each item is actually confirming (`CustomMessage`‚Üícontext conversion, in-memory transcript discard, `tools`-allowlist suppression) is path-independent; the path is only where the loom 1.0 pin happens to locate it.

## Solution Space

**Shape:** single

### Recommendation

Add the known-fragile / equivalent-reachable-site carve-out already used by items (e) and (q) to items (r), (aj), and (am), so each cited candidate-minor `dist/core/*.js` path reads as an illustrative locus of a behavioural confirmation rather than the normative audit target. Concretely, append to each of the three items a sentence in the (q) mould: the cited `dist/core/*.js` path and bundled symbol are known-fragile evidence (Pi may minify, re-bundle, rename `_`-prefixed internals, or restructure the `dist/` tree); where the cited name/path is no longer textually locatable, the contributor confirms the observable behaviour ‚Äî (r) the `CustomMessage`‚Üí`{ role: "user" }` context conversion; (aj) in-memory transcript discard on loom return; (am) the three `tools`-allowlist suppression sub-properties ‚Äî by an equivalent reachable site rather than recording `pass` (or `fail`) on textual locatability alone, with the conservative-posture rule applying when no equivalent site can be positively identified. Keep each item's existing baseline-diff instruction; the carve-out only governs the case where the loom 1.0-cited path has moved.

## Consequence

**Severity:** advisory

A contributor running the re-audit against a candidate Pi minor that has re-bundled or restructured `dist/core/` has no instruction for items (r)/(aj)/(am) once the named path stops resolving textually: literally followed, the items would either stall or record a spurious `fail` even where the audited behaviour still holds at an equivalent reachable site. The two sibling items already model the intended posture, so an attentive implementer can infer it; the gap is missing guidance, not an undecidable contradiction.

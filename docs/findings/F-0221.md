## Problem

`H6a`'s **Tests** section carries a fourth bullet, **Release-gate acceptance (manual real-host smoke)**, alongside the three automated live-corpus gate assertions. That bullet is an explicitly *manual* checklist item that never runs under `npm test`, so it is not a TDD assertion and does not belong in **Tests**, whose other bullets are all `npm test`-observable gate checks. `H6a`'s **Ships when** does not state the manual-gate criterion directly — it back-references the Tests bullet — so the release-gating completion criterion is split across two sections.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** e7f14dd — pi-loom plan: resolve "Real-host fidelity of the session double has no reproducible detection point" (2026-06-11, Thomas Andersen)
**History:** Before e7f14dd, `H6a`'s Tests section held only the three automated live-corpus gate assertions and Ships when carried no manual-smoke content. Commit e7f14dd added the "Release-gate acceptance (manual real-host smoke)" item as a fourth Tests bullet and rewrote Ships when to back-reference it ("the manual real-host smoke checklist item above"), introducing the manual completion criterion as a Tests entry rather than a Ships-when criterion.

## Solution approach

In `docs/plan_topics/H6a-live-corpus-activation.md`, move the manual completion criterion into **Ships when** so the release-completion condition is self-contained there, and strike the **Release-gate acceptance (manual real-host smoke)** bullet from **Tests**. State the relocated criterion directly in **Ships when** rather than having it point back to a Tests bullet. After the move, **Tests** retains only the three `npm test`-observable live-corpus gate assertions (green run, removed-mapping red, removed-citation red).

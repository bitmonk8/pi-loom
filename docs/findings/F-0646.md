## Problem

The diagnostic code `loom/load/cross-format-collision` names only its cross-format arm, but its registered scope is the full DISC-4 same-priority slash-name collision rule — covering both the same-format arm (two `.loom` files at the same priority) and the cross-format arm. DIAG-3 (`diagnostic-shape.md#diag-3`) forecloses renaming the code within loom 1.x, so the name/scope divergence is intentional. The only text explaining the gap is an inline gloss in the registry row's Trigger cell; it does not appear at the **Code namespaces** / *Naming convention* block in `diagnostic-shape.md` where new contributors first meet the `cross-format` token, nor at DISC-4 (`discovery-sources.md#disc-4`). A reader or contributor searching for `cross-format` outside the registry row cannot discover that the code's name and scope deliberately diverge.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** `d124b8c` (2026-05-05, "spec: same-priority slash-name collisions are uniform load-time errors (Decision #10)")
**History:** The code was authored at the original cross-format-only scope in commit `c5c4e79` (2026-05-04, "pi-loom spec: resolve 'Cross-format slash collision rule cannot achieve neither is registered'"), where its Trigger genuinely was cross-format only ("A `.loom` candidate's slash name collides with an already-registered Pi prompt template, subagent, or other extension's command"). The name and the scope agreed at that point. `d124b8c` then broadened the diagnostic to cover same-priority same-format collisions ("two packages, two settings entries, two `--loom` components") while explicitly noting in the commit message "no new code introduced" — at this commit the registered code name `cross-format-collision` ceased to name its own scope, and DISC-4 lost the `across formats` qualifier in `discovery-sources.md`. A later commit `7e440c9` (2026-06-06, "pi-loom spec: resolve 'cross-format-collision misnomer / DISC-4 heading scope'") attempted to close the gap by adding the DIAG-3 / loom-2.0-rename gloss inline to the Trigger cell, but the gloss reaches only readers who open the row; the present finding is the residual gap that mitigation left behind.

## Solution approach

Add a stable-anchored note in `diagnostic-shape.md`'s **Code namespaces** / *Naming convention* block recording that `loom/load/cross-format-collision` is the registered identifier for the full DISC-4 same-priority slash-name collision rule (both same-format and cross-format arms), with the `cross-format` token retained as a stable identifier per DIAG-3. Add a forward-link to that anchor from the `loom/load/cross-format-collision` row in `code-registry-load.md` and from DISC-4 (`#disc-4`) in `discovery-sources.md`.

## Solution constraints

- The registered code identifier `loom/load/cross-format-collision` MUST NOT change (DIAG-3, `diagnostic-shape.md#diag-3`).

## Problem

`cancellation.md` uses *steady-state* as the normative partition discriminator that separates
the *Forwarding-listener throw* trap rule (which routes a listener throw to
`loom/runtime/internal-error`) from the `session_shutdown` teardown swallow rule, and the same
bare word is reused as a term of art across the Pi-integration-contract cluster. The term is
load-bearing — it decides whether a forwarding-listener throw panics or is swallowed — yet it
has no definition anywhere in the corpus: there is no glossary entry, and no page states what it
denotes. A reader who arrives at the *Forwarding-listener throw* rule first has no anchor telling
them which regime a given listener firing belongs to; the boundary is only inferable by reading
the contrasted `session_shutdown` clauses.

## Issue introduction
**Verdict:** single-commit
**Introducing commits:** f20ef73 — pi-loom spec: resolve "Per-call AbortController/AbortSignal defect routing has gaps" (2026-05-07, Thomas Andersen)
**History:** The *Forwarding-listener throw* paragraph — including both bare uses of *steady-state* ("a steady-state forwarding listener" and "covers steady-state forwarding only") — entered `cancellation.md` verbatim in commit f20ef73, which added the forwarding-listener defect-routing rule. The term was used as the partition label against the `session_shutdown` teardown path from this first introduction, with no accompanying definition; the file's earlier history (the 2026-05-04 cancellation commits) did not contain the word. The term later spread to the Pi-integration-contract cluster, but the cited defect — an undefined partition discriminator in `cancellation.md` — was present at the paragraph's inception.

## Solution approach

Coin `steady-state` as a spec term: add a normative definition on `cancellation.md` (the page
that owns the steady-state-vs-teardown partition) distinguishing the ordinary per-invocation
forwarding/checkpoint regime from the `session_shutdown` teardown-iteration regime, and have the
*Forwarding-listener throw* and *Propagation* uses refer to it rather than relying on context.
Add a `steady-state` glossary entry following the existing entry shape, with a `See:` link to
that canonical definition. Reconcile the narrower local uses (e.g. the drain-state field-tuple in
`drain-state-contract.md`) against the coined regime-level meaning.

## Solution constraints

- The coined definition MUST subsume the existing narrower local uses of `steady-state` (e.g. the `drain-state-contract.md` field-tuple) rather than introduce a second, conflicting sense.

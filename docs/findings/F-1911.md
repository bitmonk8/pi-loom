## Finding

`H6a`'s **Ships when** field defines the manual real-host smoke pass criterion by contrast: the live run's emitted `loom-system-note` codes must fall within the committed permitted-code list, "not exact reproduction of `H7a`'s double-recorded goldens." The phrase "double-recorded goldens" is never defined anywhere in the plan. `H7a` names three committed artefacts — the **committed golden transcript**, the **committed golden diagnostics list**, and the **committed permitted-code list** — but never the term "double-recorded goldens."

A reader must infer that "double-recorded goldens" denotes the two recorded golden artefacts (the golden transcript and the golden diagnostics list), distinct from the permitted-code list the same sentence already names. The inference is recoverable — `real-host-smoke-gate.md` makes the same contrast and expands it to "exact turn order/count or exact emitted-code set," matching the transcript and diagnostics list — but the term itself is a coined label that resolves to no artefact `H7a` defines, forcing the reader to reconstruct the mapping.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** 3911733 — pi-loom plan: resolve "Live-host smoke pass criterion assumes a non-deterministic LLM reproduces a transcript" (2026-06-11, Thomas Andersen)
**History:** Commit 3911733 added the manual real-host smoke acceptance criterion to H6a, coining "not exact reproduction of `H7a`'s double-recorded goldens" to express that a non-deterministic live model is not held to the deterministic double's recorded outputs. The same edit (and the parallel `real-host-smoke-gate.md` text) introduced the term, which H7a never defines; later edits to H6a (e.g. ee9acd0, 9d63d2a) revised neighbouring clauses but never reconciled the term against H7a's artefact names.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/H6a-live-corpus-activation.md`'s **Ships when** field, replace the trailing clause `not exact reproduction of `H7a`'s double-recorded goldens` so that it names the two artefacts H7a actually defines — its committed golden transcript and committed golden diagnostics list — e.g. `not exact reproduction of `H7a`'s committed golden transcript and golden diagnostics list`. This keeps the contrast intact (the live run scores against the permitted-code list, not against the deterministic goldens) while citing only artefact names H7a establishes.

## Consequence

**Severity:** advisory

The H6a auditor reading the Ships-when criterion must map "double-recorded goldens" onto H7a's actual artefact names to know what the live run is *not* required to reproduce; the mapping is inferrable from context, so the gate remains usable, but the undefined label is a needless reconstruction step in the leaf that gates the loom 1.0 release.

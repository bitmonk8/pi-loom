## Problem

GOV-22 requires every *defining obligation site* on a non-narrative spec page — every line carrying an RFC-2119 normative modal the page owns — to carry a co-located `PREFIX-N` REQ-ID in dual-form layout, and GOV-9 requires that ID be the cross-link target other pages reach the obligation through. Eight pages carry clusters of currently-undrained defining obligation sites whose prose is normative MUST / MUST NOT yet whose source lines have no co-located REQ-ID anchor and therefore no `#prefix-n` fragment to cite: `binder/determinism-cancellation-failure.md`, `hard-ceilings/ceiling-invariants-and-audit.md`, `hard-ceilings/ceilings-3-and-4.md`, `pi-integration-contract/drain-state-contract.md`, `pi-integration-contract/active-invocation-registry.md`, `pi-integration-contract/diagnostic-emission-isolation.md`, `implementation-notes.md`, and `lexical.md`. Roughly 25 atomic obligations are affected; each is enumerated as GOV-22 residue in a `cka-<n>` row of coverage-matrix.md but has never been coined spec-side, leaving GOV-9's cross-link contract unsatisfiable for them.

**History:** Each obligation in the cluster predates GOV-22 by weeks-to-months: the prose was authored as plain "X MUST Y" sentences under the prior editorial regime, when ID-coinage was not a precondition for normative prose. GOV-22 (4f588e0) was the rule that retroactively defined every such site as a *defining obligation site* requiring a co-located REQ-ID; coverage-matrix.md's `cka-<n>` *Code-keyed obligation areas* table (introduced later still, in the plan-side closing-gate work that became H5a / H5e) was the indirection that *enumerated* the residue without coining it, deliberately deferring coinage on the GOV-22 progressive-drain footing. The current state is therefore not a single-commit defect but the durable residue GOV-22's progressive-drain clause assumes will exist until each affected page is next edited. The eight pages in this cluster have not been touched under the GOV-22 trigger predicate since the rule landed, so the residue has accumulated without draining.

## Solution approach

Coin one `PREFIX-N` REQ-ID per atomic obligation in GOV-1 dual-form layout (`<a id="prefix-n"></a> **PREFIX-N.**`), allocating numeric tails per GOV-3 under each page's already-registered prefix — `BNDR` for `binder/determinism-cancellation-failure.md`, `CIO` for the two `hard-ceilings/` pages, `PIC` for the three `pi-integration-contract/` pages, `IMPL` for `implementation-notes.md`, and `LEX` for `lexical.md`. Where the prose currently bundles independently-testable rules in one sentence, split it into separate prose units so each obligation carries its own ID; the `lexical.md` PascalCase / lowercase-first casing pair is enforced via two distinct diagnostic codes and should split. Update the affected `cka-<n>` rows in coverage-matrix.md to cite the new IDs and drop the `(un-anchored; GOV-22 residue)` trailer where the row is fully ID-covered.

## Solution constraints

- Out of scope: the back-reference cross-references on `ceilings-3-and-4.md` (lines 44/64) point to PIC-1-owned obligations and need no ID here; cross-references outside the corpus are out of scope per GOV-22's per-commit-trigger semantics.
- Use only the prefix already registered for each page; promoting any obligation to `CEIL-` or any other new prefix requires a separate GOV-7 prefix-table decision and is out of scope (coining `CIO-N` keeps coverage-matrix.md's "`CEIL` anchors no live REQ-ID" closure rationale true).

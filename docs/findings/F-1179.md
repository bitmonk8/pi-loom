## Finding

GOV-22 (governance/req-id-prefix-table-active-b.md) requires every *defining obligation site* on a non-narrative spec page — every line carrying an RFC-2119 normative modal that the page owns — to carry a co-located `PREFIX-N` REQ-ID in dual-form layout, and GOV-9 requires that ID to be the cross-link target other pages reach the obligation through. The contract drains the pre-existing residue progressively, but the eight pages below carry clusters of *currently-undrained* defining obligation sites whose prose is normative MUST / MUST NOT yet whose source lines have no co-located REQ-ID anchor and therefore no `#prefix-n` URL fragment to cite — leaving GOV-9's cross-link contract permanently unsatisfiable for them until they are coined.

The clusters:

- `binder/determinism-cancellation-failure.md` — the FNV-1a hash-reproduce MUST ("Conforming implementations MUST reproduce these values exactly", line 13), the failure-mode template-verbatim MUST ("Renderers MUST emit the surrounding template text verbatim", line 42), and the immediate-re-issue MUST on the per-invocation retry budget (line 57). The page carries zero `BNDR-` anchors despite being a topic file under the `binder/` subtree (prefix `BNDR`); the residue is mapped under `cka-42` / `cka-43` in coverage-matrix.md but the spec-side IDs have never been minted.
- `hard-ceilings/ceiling-invariants-and-audit.md` — the *Routing obligation for future ceilings* MUST (line 37) and the *Five-site list co-edit obligation* MUST on ceiling #4's per-boundary table (line 47). Both sit on a page in the `hard-ceilings/` subtree (prefixes `CEIL`, `CIO`) yet neither carries a `CEIL-N` or `CIO-N` anchor.
- `hard-ceilings/ceilings-3-and-4.md` — the `masked: []` canonical-absence MUST NOT ("implementations MUST NOT emit `masked: []`", line 60), plus the dual delegations on lines 44/64 stating that PIC-1 owns ancillary obligations (the latter back-reference content is owned elsewhere and is out of scope of this finding).
- `pi-integration-contract/drain-state-contract.md` — the closed-arm dispatch / rename-sweep / read-failure fail-safe MUSTs identified in coverage-matrix row `cka-22` (line 14 et seq.) and the no-suppression *idempotent handler clause* (line 3). Page prefix `PIC`; no `PIC-N` is co-located with these obligations.
- `pi-integration-contract/active-invocation-registry.md` — the *Dispatch-site setup wrap* MUST and the `sendSystemNote`-fallback-chain MUST NOT in the teardown handler (lines 11 and 21), plus the insertion-order / `IdSource.newInvocationId()` MUSTs mapped under `cka-26`. Page prefix `PIC`; no anchor.
- `pi-integration-contract/diagnostic-emission-isolation.md` — the teardown-time `console.error` isolation MUST cluster mapped under `cka-29`. Page prefix `PIC`; no anchor.
- `implementation-notes.md` — the multi-error parser MUST (line 10) and the `Clock`-discipline MUST NOT against `Date.now` / `performance.now` / global `setTimeout` (line 30). Page prefix `IMPL`; no `IMPL-N` co-located.
- `lexical.md` — the PascalCase-required and lowercase-first-required casing rules (lines 15–16) are stated declaratively ("is required for: …") and a partner sentence ("**first letter's case is enforced** by the parser", line 13) names them collectively. Page prefix `LEX`; no `LEX-N` anchor co-located, despite the obligation being enforced parser-side via two named diagnostic codes.

The aggregate is roughly 25 atomic obligations across the eight pages. Each is `cka-<n>`-mapped in coverage-matrix.md (one row per page, multiple MUSTs bundled per row), so the implementation-side closing surface exists — but the spec-side `PREFIX-N` anchors GOV-9 / GOV-22 demand do not, and the H5e un-anchored-MUST text-scan gate ([`H5e`](../../docs/plan_topics/H5e-un-anchored-must-gate.md)) flips from seeded-fixture footing to live-corpus footing at H6a, at which point an un-coined obligation reddens `npm test`.

## Issue introduction

**Verdict:** present-since-inception (multi-commit accretion)
**Introducing commits:** `8eea523` (May 2026, `spec: fill implementation-blocking gaps` — original `lexical.md` casing prose); `b3bc4ce` (May 2026, `spec: pin failure-observability surface` — original "Renderers MUST emit the surrounding template text verbatim" prose, since relocated to `binder/determinism-cancellation-failure.md`); `a7c882a` (May 2026, `spec: resolve "FNV-1a binder seed…"` — original "Conforming implementations MUST reproduce" prose); `1c7f550` and later `hard-ceilings.md` edits (June 2026) — original *Routing obligation for future ceilings* and *Five-site list co-edit* MUSTs; `fecb504` (the topic-split commit) for the wholesale move into per-topic files; `4f588e0` (June 2026, `spec: resolve "Non-narrative pages with zero coined REQ-IDs non-referenceable"`) — introduction of GOV-22 itself, which is the rule against which all the prior un-anchored MUSTs first became "residue".
**History:** Each obligation in the cluster predates GOV-22 by weeks-to-months: the prose was authored as plain "X MUST Y" sentences under the prior editorial regime, when ID-coinage was not a precondition for normative prose. GOV-22 (4f588e0) was the rule that retroactively defined every such site as a *defining obligation site* requiring a co-located REQ-ID; coverage-matrix.md's `cka-<n>` *Code-keyed obligation areas* table (introduced later still, in the plan-side closing-gate work that became H5a / H5e) was the indirection that *enumerated* the residue without coining it, deliberately deferring coinage on the GOV-22 progressive-drain footing. The current state is therefore not a single-commit defect but the durable residue GOV-22's progressive-drain clause assumes will exist until each affected page is next edited. The eight pages in this cluster have not been touched under the GOV-22 trigger predicate since the rule landed, so the residue has accumulated without draining.

## Solution Space

**Shape:** single

### Recommendation

Coin one `PREFIX-N` REQ-ID per atomic obligation, in the dual-form layout GOV-1 *Required HTML-anchor contexts* and *Dual-form layout* mandate (`<a id="prefix-n"></a> **PREFIX-N.**`), allocating numeric tails per GOV-3 (next free integer under the page's prefix, never collapsing to fill holes), using the prefixes already registered for each page:

- `binder/determinism-cancellation-failure.md` → next free `BNDR-N` per page (the `binder/` subtree owns `BNDR`); coin one ID for the hash-reproduce MUST, one for the template-verbatim MUST, and one for the immediate-re-issue MUST on the per-invocation retry budget.
- `hard-ceilings/ceiling-invariants-and-audit.md` → next free `CIO-N` (the *Routing obligation for future ceilings* and *Five-site list co-edit* MUSTs are cross-ceiling rules and sit naturally under `CIO`; `CEIL` is registered but currently anchors no live ID, so leaving it dormant is consistent with the existing closure rationale in coverage-matrix.md).
- `hard-ceilings/ceilings-3-and-4.md` → next free `CIO-N` for the `masked: []` canonical-absence MUST NOT (the cross-references on lines 44 and 64 are back-references to obligations owned on the PIC-1 page and need no ID here).
- `pi-integration-contract/drain-state-contract.md` → next free `PIC-N` per the `pi-integration-contract/` subtree; one ID per atomic obligation in the cluster (idempotent-handler closed-arm enumeration, `readDrainState` closed three-arm dispatch, no-fourth-arm prohibition, read-failure fail-safe).
- `pi-integration-contract/active-invocation-registry.md` → next free `PIC-N`; one ID per atomic obligation (*Dispatch-site setup wrap* `try`/`catch` routing, `sendSystemNote`-fallback-chain MUST NOT in teardown, insertion-order iteration, `invocationId`-from-`IdSource` source-of-truth).
- `pi-integration-contract/diagnostic-emission-isolation.md` → next free `PIC-N` per atomic obligation in the `cka-29` cluster.
- `implementation-notes.md` → next free `IMPL-N`; one for the multi-error parser MUST, one for the `Clock`-discipline MUST NOT.
- `lexical.md` → next free `LEX-N` for the PascalCase-required and lowercase-first-required rules (one ID per rule, since they are independently testable via `loom/parse/schema-case-mismatch` and `loom/parse/binding-case-mismatch`).

When an obligation is genuinely a single rule with multiple clauses, prefer one ID with the clauses as sub-points of one paragraph; when the prose currently bundles independently-testable rules in one sentence (as several of these do — the `lexical.md` casing pair is the clearest example, but the `Clock` MUST NOT and the drain-state read-failure fail-safe MUSTs are similar), split the sentence into separate prose units and give each its own ID. The split is governed by GOV-1's atomicity discipline (one obligation per anchor); the goal is that a test failing one item can cite exactly one REQ-ID.

After coinage, update each affected `cka-<n>` row in coverage-matrix.md to cite the new `PREFIX-N` IDs in its row prose, dropping the "(un-anchored; GOV-22 residue)" trailer when the row's whole content is ID-covered (per coverage-matrix.md's existing convention for fully-coined rows). Coining and the coverage-matrix.md update should land in the same commit to satisfy GOV-1's lock-step pinning of dual-form anchor citation.

Edge cases the splitter must watch:

- Some of these obligations are referenced by other spec pages today via section-level links because no ID existed to cite. After coinage, those cross-references should be repointed to `#prefix-n` in the same edit per GOV-9, but only where the cross-reference is internal to the corpus — outside-the-spec links are out of scope (per GOV-22's per-commit-trigger semantics; the residue clause covers them).
- The `lexical.md` casing pair is enforced via two named diagnostic codes already registered in the diagnostics registry. The codes are not REQ-IDs and do not satisfy GOV-22 (which is REQ-ID-specific); the rule's *enforcement mechanism* is the code, but the *defining obligation site* still needs a `LEX-N` anchor on the spec page.
- `hard-ceilings/ceiling-invariants-and-audit.md` and `ceilings-3-and-4.md` carry several existing `CIO-N` IDs (CIO-1…CIO-6). The new IDs continue that sequence per GOV-3 (next free integer, no hole-filling).
- The cross-ceiling closure rationale in coverage-matrix.md (line 68) currently states "`CEIL` is a registered prefix that currently anchors no live REQ-ID". Coining new `CIO-N` IDs (not `CEIL-N`) keeps that statement true and avoids a separate edit there; the splitter should not promote any of these to `CEIL-` without a separate GOV-7 *Narrative-to-normative promotion* decision.

## Consequence

**Severity:** correctness

GOV-9's universal cross-link contract is unsatisfiable for ~25 obligations: any spec page that needs to depend on one of them must either restate it locally (defeating the single-canonical-home discipline) or link to a section heading rather than the obligation itself (defeating the `#prefix-n` precision GOV-9 mandates). Editorial drift on the unanchored prose is undetectable via REQ-ID-keyed lookup. The H5e gate's live-corpus footing — flipped at H6a — reddens `npm test` against every un-enumerated un-anchored MUST in `spec_topics/**`, so the obligations either need to be coined or kept enumerated as GOV-22 residue in coverage-matrix.md indefinitely; the current state is the latter, but each new edit that touches one of these sites triggers GOV-22's per-commit coinage MUST, so the residue drains commit-by-commit anyway. Coining them in one focused pass is strictly cheaper than draining them incrementally under PR pressure.

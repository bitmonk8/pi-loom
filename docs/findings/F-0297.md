## Problem

The spec reserves "seam" for a construction-time–injected host interface (the PIC-10…PIC-16 DI seams in `host-interfaces-services.md`, modelled by leaves V8a–V8e). `V16a`'s **Adds** reuses the same noun for a categorically different thing: a "cross-ceiling arbitration seam," the stateless pure function `arbitrate(candidate) → { surfaced, masked? }` that takes no construction-time collaborators and is *consulted* (not injected) at each call site; the five consumer leaves V4e, V5e, V11f, V13c, and V15b carry the same coinage. A reader holding the spec's "seam = injected interface" sense can infer the arbitration function must be wired through the constructor-injection skeleton, adding needless DI ceremony and risking a module-level singleton the No-globals/statics/singletons rule bans. The collision is between `conventions.md`'s broad Class-2 "cross-leaf seam" sense (which V16a's usage does fit) and the narrow spec/V8* DI-seam sense — not a claim that "seam" is categorically wrong.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** e2b7e81 — pi-loom plan: resolve "isolated cross-ceiling unit interface/authority undefined" (2026-06-10, Thomas Andersen)
**History:** Before e2b7e81, V16a's **Adds** described "the fixed cross-ceiling evaluation order" and named no interface — the cross-ceiling unit's interface/authority was undefined (the file carried no "seam" token at all). Resolving that prior finding, e2b7e81 coined the `arbitrate(candidate)` interface, named it a "cross-ceiling arbitration seam," and propagated the "consults … the seam" prose into the downstream consumer leaves. The "seam" overload entered the plan at that commit; the spec's `seam = DI-injected interface` usage (PIC-10…PIC-16) predates it.

## Solution approach

Disambiguate V16a's "cross-ceiling arbitration seam" against the spec's narrow DI-seam sense within the plan. Either rename the noun to a non-`seam` term (e.g. "cross-ceiling arbitration function"/"module") at V16a, V16a-T, and the consumer leaves, or add an inline gloss at V16a's first use of "seam" stating the arbitration function is consulted/imported directly and never constructor-injected — a Class-2 cross-leaf seam per `conventions.md`, distinct from the spec's construction-time–injected PIC-10…PIC-16 host seams. Whichever is chosen, the disambiguation must name the contrasting spec DI-seam sense so the No-globals/DI rule is not misapplied to wire the pure function as an injected singleton.

## Solution constraints

- Out of scope: the spec is read-only for this fix (`fix_surface` is plan-only); the `glossary.md` spec entry the enriched Recommendation suggested is not in scope — resolve the overload within the plan.
- If the term is renamed, apply it at every occurrence (V16a, V16a-T, and consumer leaves V4e/V5e/V11f/V13c/V15b); a partial rename reintroduces terminology variance.

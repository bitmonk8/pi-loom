## Problem

`V4g`'s **Adds** field labels the method-qualified phrase a seam — "injected through `V9b`'s `ReloadFailureInjector.injectReloadFailure` seam". The declaring leaf `V9b` and `V4g`'s own **Tests** bullet instead name the injection point the **watcher-time reload failure-injection seam** and cite `ReloadFailureInjector.injectReloadFailure` only as the method. A reader comparing the Adds field against the Tests bullets and `V9b` sees the same injection point labelled two ways with no equivalence statement, against the plan's one-name-per-seam convention.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** 324813d — pi-loom plan: resolve "split ERR-7 reload scenario out of V4e routing checks" (2026-06-18, Thomas Andersen)
**History:** The **watcher-time reload failure-injection seam** name predates this leaf (coined in `d64dce5`, refined when the `ReloadFailureInjector` interface was introduced in `fdce2a2`). `V4g`/`V4g-T` were created from scratch in `324813d` when the `ERR-7` reload scenario was split out of `V4e`; that commit authored `V4g`'s Adds with the method-qualified "`ReloadFailureInjector.injectReloadFailure` seam" phrasing while the Tests bullets used the pre-existing canonical bold seam name — the divergence was present at the leaf's inception.

## Solution approach

In `V4g`'s **Adds** field, rename the injection point to the canonical **watcher-time reload failure-injection seam** that `V9b` declares, and cite `ReloadFailureInjector.injectReloadFailure` as the injection method rather than calling it a seam.

## Solution constraints

- Out of scope: `V9b` and `V4g-T` are read-only for this fix — both already use the canonical seam name and method/seam distinction correctly.

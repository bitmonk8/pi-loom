## Finding

The single test-only injection point for the `ERR-7` watcher-time reload failure has one canonical name in the corpus: `V9b` (its declaring leaf) coins it as the **watcher-time reload failure-injection seam** and names `ReloadFailureInjector.injectReloadFailure` as the *method* all three arms (registry-swap, `.loom`/`.warp` re-parse, settings re-merge) inject through. `V4g-T` and `V4g`'s own Tests bullet follow that pattern correctly — they inject *via* `ReloadFailureInjector.injectReloadFailure` *at* the **watcher-time reload failure-injection seam**, distinguishing the method from the seam.

`V4g`'s **Adds** field breaks the pattern: it calls the method-qualified phrase itself a seam — "injected through `V9b`'s `ReloadFailureInjector.injectReloadFailure` seam". A reader comparing the Adds field against the Tests bullets (and against `V9b`) sees the same injection point labelled two ways with no equivalence statement, against a plan convention where each seam carries one name across its `-T`/impl pair (e.g. `FileWatcher` seam, `FileSystem` seam, model-reference-matcher injection seam).

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** 324813d — pi-loom plan: resolve "split ERR-7 reload scenario out of V4e routing checks" (2026-06-18, Thomas Andersen)
**History:** The **watcher-time reload failure-injection seam** name predates this leaf (coined in `d64dce5`, refined when the `ReloadFailureInjector` interface was introduced in `fdce2a2`). `V4g`/`V4g-T` were created from scratch in `324813d` when the `ERR-7` reload scenario was split out of `V4e`; that commit authored `V4g`'s Adds with the method-qualified "`ReloadFailureInjector.injectReloadFailure` seam" phrasing while the Tests bullets used the pre-existing canonical bold seam name — the divergence was present at the leaf's inception.

## Solution Space

**Shape:** single

### Recommendation

In `V4g`'s **Adds** field, stop calling the method-qualified phrase a seam and use the canonical seam name `V9b` declares, naming `ReloadFailureInjector.injectReloadFailure` only as the injection method. Concretely, replace "injected through [`V9b`](./V9b-registration-reload-wiring.md)'s `ReloadFailureInjector.injectReloadFailure` seam" with text that injects the `ERR-7` cause through the **watcher-time reload failure-injection seam** `V9b` declares, citing `ReloadFailureInjector.injectReloadFailure` as the method (for example: "injected through the **watcher-time reload failure-injection seam** [`V9b`](./V9b-registration-reload-wiring.md) declares — via its `ReloadFailureInjector.injectReloadFailure` method"). `V9b` (the declaring leaf) and `V4g-T` already use the canonical name and the method/seam distinction correctly; both are read-only for this fix.

## Consequence

**Severity:** advisory

A reader scanning the `V4g` pair encounters the seam referred to two ways — "`ReloadFailureInjector.injectReloadFailure` seam" in Adds versus the canonical "watcher-time reload failure-injection seam" in the test bullets and `V9b` — and must infer they denote the same injection point. The concrete method name keeps implementation unambiguous, so no implementer would diverge, but the inconsistency is a guidance gap against the plan's one-name-per-seam convention.

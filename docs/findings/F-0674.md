## Problem

`diagnostic-shape.md`'s **Location-less diagnostics** paragraph closes the location-less set to eight codes (the four `loom/load/*` codes and the four `loom/host/session-shutdown-*` teardown codes) and makes that closure normative, but omits `loom/runtime/reload-teardown-timeout`. That code represents a handler-scoped timeout across N in-flight invocations with no single source span, so the structured `Diagnostic` envelope demands `file`/`range` values the runtime has none of. PIC's `diagnostic-emission-isolation.md` *Wire format* clause already grants the `file`/`range` omission to all five `console.error` teardown codes including this one, directly contradicting the closure rule. A consumer that validates strictly against the closed location-less set as it stands rejects every conformant `loom/runtime/reload-teardown-timeout` emission.

## Issue introduction

**Verdict:** present-since-inception

**Introducing commits:** `b7d383a` (2026-06-06, "pi-loom spec: resolve \"Diagnostic envelope unsatisfiable for location-less codes\"")

**History:** The **Location-less diagnostics** paragraph and the closed eight-code set entered `docs/spec_topics/diagnostics/diagnostic-shape.md` in `b7d383a` as a single new block; `git show b7d383a -- docs/spec_topics/diagnostics/diagnostic-shape.md` lists exactly the four `loom/load/*` codes and four `loom/host/session-shutdown-*` codes carried today, with no `loom/runtime/*` member at any later commit (`git log --all -S 'location-less' -- docs/spec_topics/diagnostics/diagnostic-shape.md` shows `b7d383a` as the sole content-modifying commit on that token; subsequent commits to the file did not touch the set). The "five carved-out exceptions" framing that names `loom/runtime/reload-teardown-timeout` alongside the four `loom/host/session-shutdown-*` codes predates the location-less set (it has been in `diagnostic-shape.md`'s opening paragraph since the teardown contract landed in `5e8b499`, 2026-05-06), so the closure paragraph was authored against an already-present asymmetry between the five-code `console.error` carve-out set and the four-code `loom/host/session-shutdown-*` subset it chose to enumerate, and `loom/runtime/reload-teardown-timeout` was omitted at the location-less set's introduction rather than dropped later.

## Solution approach

Add `loom/runtime/reload-teardown-timeout` to the closed location-less set in `diagnostic-shape.md`'s **Location-less diagnostics** paragraph, as the one `loom/runtime/*` member alongside the four `loom/load/*` and four `loom/host/session-shutdown-*` members already enumerated. The set stays closed; only its membership grows.

- Out of scope: the opening paragraph's "five carved-out exceptions" namespace framing — a same-cluster sibling finding rewords it independently.

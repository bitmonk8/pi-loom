## Problem

The *Per-family record-shape table* (`audit-per-family-record-shape-table` in `audit-failures.md`) pins a violation record as four fields and states the conformance MUST that a record carrying a value "outside the per-family range" violates the rule. For `path`, `line`, and `symbol` the per-family cells give a mechanically decidable shape, but for `proposed-resolution` the cells give only a semantic-type description (free-text remediation suggestions), so "outside the per-family range" is not a decidable test — a conformance suite cannot determine whether a given `proposed-resolution` string is in-range. The *Category-(2) inventory join key* (`audit-category-2-inventory-join-key` in `audit-resolution.md`) compounds this: its lex-min sentence requires the `proposed-resolution` field to report the lexicographically-smallest `path` entry, but the field's content format is never pinned, so there is no parseable target a test can assert against. The defect is confined to `proposed-resolution`; the `audit/<class>/<family>/<symptom>` discriminator token that CI routes on remains fully pinned and testable.

The untestable `proposed-resolution` "range" MUST and the unverifiable lex-min selection were both present in the construct's first commit. They carried through unchanged; the later `f5e89f4` size-cap split and the `spec_topics/` collapse only relocated the prose across files without touching the wording.

## Solution approach

Narrow the conformance MUST at `audit-per-family-record-shape-table` so its "outside the per-family range … violates this rule" predicate scopes only to `path`, `line`, and `symbol`, and clarify `proposed-resolution` as advisory contributor-facing remediation guidance whose free-text content is not a conformance-checkable obligation. Demote the lex-min sentence in `audit-category-2-inventory-join-key` so the tie-break is a deterministic, run-stable selection the implementation pins, retaining only same-implementation reproducibility (two runs over the same tree emit identical output) as the testable property.

## Solution constraints

- Out of scope: redefining the `path`/`line`/`symbol` per-family cell shapes in the *Per-family record-shape table* (sibling finding owns the testable shape of those three fields).

## Problem

`V1a`'s **Adds** declares "the closed statement-termination / newline-continuation trigger set" in scope, and the paired test bullet in both `V1a` and `V1a-T` claims coverage "per the closed trigger table". But the only listed vectors are the two violation codes `loom/parse/single-line-if` and `loom/parse/stray-backslash`, which exercise none of the four positive continuation triggers (unmatched open bracket, trailing operator, trailing comma, leading operator) nor the blank-line-spanning rule defined in `grammar.md` §Newline continuation. `V1a`'s **Ships when** gate greens after verifying only the two negative-space codes, so a tokeniser that mishandles continuation still passes while the continuation engine ships untested. The coverage claim is broader than the listed evidence.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `c6a664e` (2026-06-10, "pi-loom plan: build/update plan for spec.md + review")
**History:** The plan corpus is git-tracked. `git log --follow` shows `V1a-lexer-core.md` and `V1a-T-lexer-core.md` were created in `c6a664e`. `git log -S "closed trigger table"` and `git log -S "newline-continuation trigger set"` both return only `c6a664e`, and `git log -L 13,13:docs/plan_topics/V1a-lexer-core.md` shows the test bullet was added at file creation against `/dev/null`. The later commit `fa18160` rewrote only the **Adds** byte-source clause and left the trigger-set claim and the under-covering test bullet untouched. The overclaim/under-test mismatch has existed since the leaf was authored.

## Solution approach

Add positive continuation-acceptance vectors to `V1a-T`'s **Tests** covering each row of `grammar.md` §Newline continuation — the four continuation triggers plus the blank-line-spanning rule — each asserting the spanning lines tokenise as a single continued statement. Mirror the same vectors in `V1a`'s **Tests**. Update `V1a`'s **Ships when** gate so its observable condition references the continuation-acceptance vectors alongside the two listed violation codes.

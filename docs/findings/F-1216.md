## Finding

Seven distinct paragraphs across the PIC shard-08 pages mandate a specific code mechanism where the externally observable contract is purely behavioural. The cluster is one defect (prescription drifts into implementation when only outcome is testable), instantiated at seven sites that touch four spec files:

- **`diagnostic-emission-isolation.md` — *Hoist obligation*.** Pins a "hoist to a named local string binding (e.g. `const detailsEventReason: string = ...`) **before** entering the wrapped serialisation-and-emission sequence" and dictates that the catch arm "MUST interpolate the pre-hoisted local rather than re-reading from the partially-constructed structured object or from `entry.shutdownReason` directly". The observable invariant is narrower: the catch arm MUST emit a string `details.event.reason` value (one of the four captured-value cases) even when payload construction was interrupted, and MUST NOT re-read `event.reason`. Any implementation strategy that delivers that — a hoisted local, a closure-captured copy, an immutable post-snapshot value object, a defensive read with explicit fallback — satisfies the contract.
- **`session-only-degraded-state.md` — *Seam minimality (normative)*.** Forbids any "handler-scoped state seam beyond `pinnedConstantReadOk` and `capturedEventReason` — no `branchTakenWasClosedSet` flag, no per-sub-step success roll-up, and no separate `!pinnedConstantReadOk` disjunct or `"<unreadable>"` sentinel-string equality check to back arms (2)/(3)". The two named variables are an implementation realisation of two information requirements: (i) whether the snapshot read succeeded, and (ii) what value (closed-set member, coerced unknown, or sentinel) the handler captured. The observable invariants are the two predicate outcomes (narrow diagnostic-emission, broad tag-transition) plus the prohibition on a fresh `event.reason` re-read at the discriminator site; how those are computed is implementation-owned.
- **`unknown-reason-rule.md` — `SDK_SURFACE_INVENTORY === undefined` guard.** Mandates an "explicit `SDK_SURFACE_INVENTORY === undefined` check **before** invoking the iteration primitive" and calls out "a naive `SDK_SURFACE_INVENTORY.find(…)?.literals` pattern" as non-conformant because it routes the circular-init / live-binding case to the throw arm instead of `"missing-entry"`. The observable rule is the discriminator value, not the JavaScript expression that produces it: an unresolved/undefined inventory MUST route to `"missing-entry"`, not to `"throw:TypeError: ..."`. Any implementation that achieves that routing — pre-check, try/catch with a sentinel-typed throw, optional-chaining plus a follow-up emptiness branch — is conformant.
- **`unknown-reason-rule.md` — *Pin the coercion to `String(event.reason)`*.** Pins the JS-level construct. The observable invariant is that the captured string MUST be (a) defined for symbols, `undefined`, numbers, booleans, ordinary objects, and `null` without itself throwing in the common case, and (b) MUST fall back to `"<unreadable>"` if the coercion itself throws (a getter / `Symbol.toPrimitive` / `toString` that throws). The bug the prose names (template-literal coercion re-introducing the symbol-throw failure mode) is one specific *non-conformant* alternative, not the only conformant one.
- **`patch-skew-degradation.md` sub-step 3 — `SHUTDOWN_AWAIT_CAP_MS = 2000`.** Pins both the JavaScript identifier and the literal value. The behavioural invariant is "a bounded wait of approximately 2 s on subagent disposal, after which the runtime emits `loom/runtime/reload-teardown-timeout` and proceeds to sub-step 4"; the identifier and the exact ms value are leaf concerns. Note that `V18c`'s build-time literal-read assertion depends on this constant existing as a named, byte-readable literal in the pinned-constants block — so any demotion MUST either keep an implementation-owned named constant in the same block (and rephrase the spec as "a named constant whose value is approximately 2000 ms"), or retire that arm of `V18c`'s assertion and accept the loss of that build-time gate.
- **`patch-skew-degradation.md` sub-step 3 — the `await Promise.allSettled(activeInvocations.map(inv => inv.disposeBarrier))` expression.** Pins the exact JS expression. The observable invariant is "wait for every in-flight entry's `disposeBarrier` to settle, in insertion order, without unwinding on any one rejection". Any equivalent shape — a hand-rolled await loop with rejection swallow, a `for await ... of` over a settled iterable, a typed wrapper — satisfies it.
- **`tool-registration-lifetime.md` — `Map<schemaSlug, registeredToolName>` registration cache.** Names the JavaScript collection type. The behavioural contract is dedup by canonical schema bytes: a new lowered schema whose canonical-form bytes match an already-cached entry's bytes reuses the registration name; a slug-equal but byte-different schema fires `loom/runtime/registration-cache-collision` and registers under the disambiguated name. Whether the cache is realised as `Map`, a plain object, a `WeakMap` keyed off a slug-interned symbol, or a typed wrapper is leaf concern.

In every instance the prescription was added defensively, to head off a specific known failure mode; the failure modes (template-literal symbol throw, naive optional-chain hiding the undefined-inventory case, etc.) are real and the resulting behavioural invariants must be preserved. The defect is not that the rules exist but that they restate themselves at the mechanism layer rather than the outcome layer, narrowing implementer freedom without any observable test gaining coverage.

## Solution Space

**Shape:** single

### Recommendation

Restate each prescription as the behavioural invariant it was added to guarantee, and demote the named mechanism to a non-normative *example realisation* (or delete the example entirely where the invariant is self-evident).

**Per-site restatements:**

- **Hoist obligation** — "The catch arm of the construction-site wrap MUST emit a string `details.event.reason` value drawn from the same captured-value enumeration the structured-emission path would have used (or the `"<unreadable>"` sentinel for the per-invocation note's catch arm on a construction-site throw), and MUST NOT re-read `event.reason` or `entry.shutdownReason` while the structured object is partially constructed." Drop the `const detailsEventReason: string = ...` example, or move it to a non-normative parenthesis.
- **Seam minimality (normative)** — Replace with the two information-requirement invariants: "(i) The diagnostic-emission predicate's snapshot-read-success conjunct MUST be derived from the same handler-entry observation the `loom/host/session-shutdown-pinned-constant-unreadable` catch-arm site uses, not from any re-derivation later in the handler; (ii) the tag-transition predicate's `event.reason ∉ {"quit","reload"}` branch MUST be evaluated against the value the handler captured at entry, not a fresh `event.reason` property re-read at the discriminator site. The runtime MUST NOT introduce an additional per-sub-step success roll-up or a `branchTakenWasClosedSet`-style flag whose absence would change either predicate's outcome." The two named seam variables become a non-normative example of a sufficient realisation.
- **`SDK_SURFACE_INVENTORY === undefined` guard** — Replace arm (1) with the routing invariant: "When `SDK_SURFACE_INVENTORY` is unresolved at handler entry (circular module init, live-binding gap, or any other state in which the source-of-truth pinned-constants block has not finished initialising), the discriminator MUST be `"missing-entry"`. An implementation that lets this case throw out of an iteration primitive and routes via the throw-arm discriminator is non-conformant — operator deduplication on `(code, details.failure)` would split the same root cause across two buckets depending on the implementation's iteration choice."
- **Pin the coercion to `String(event.reason)`** — Replace with the coercion contract: "The coercion that fills `details.observed` and `capturedEventReason` MUST tolerate `undefined`, numbers, booleans, ordinary objects, `null`, and symbols without itself throwing in the common case; MUST fall back to `"<unreadable>"` when the underlying `toString`/`Symbol.toPrimitive`/getter throws; and MUST NOT be implemented as a template-literal interpolation `` `${event.reason}` `` (which re-introduces a symbol-throw failure mode the coercion contract is chosen to avoid). `String(event.reason)` is one conformant realisation."
- **`SHUTDOWN_AWAIT_CAP_MS = 2000`** — Replace with: "Sub-step 3's await is bounded by a named runtime constant whose value is 2000 ms (±implementation tolerance for monotonic-clock granularity); the constant MUST live in the same source-of-truth pinned-constants block as the Step-0 capability-probe constants so the build-time literal-read assertion can reach it. The identifier `SHUTDOWN_AWAIT_CAP_MS` is the loom 1.0 implementation's name and is the symbol `V18c`'s assertion and the bump-procedure cross-references key on; renaming it requires a co-edit across those sites in the same commit." This keeps `V18c`'s build-time gate intact while shedding the "exact JS literal" framing.
- **`Promise.allSettled(...map(...))` expression** — Replace with: "Sub-step 3 MUST wait for every in-flight `ActiveInvocationRegistry` entry's `disposeBarrier` to settle in insertion order, without unwinding on any individual rejection, before transitioning to sub-step 4 (or before the `SHUTDOWN_AWAIT_CAP_MS` deadline fires, whichever is earlier)." Reserve `Promise.allSettled(...)` for a non-normative example.
- **`Map<schemaSlug, registeredToolName>` cache** — Replace with: "The runtime maintains an extension-scoped dedup cache keyed by the schema slug of the lowered tool `parameters` (produced by the recipe in [Schema Subset — Canonical schema hash](../schema-subset.md#canonical-schema-hash)), mapping to the content-addressed registered name. The cache MUST be drop-and-restart on `/reload` (Pi's `ctx.reload()` re-executes the factory; no persistent state survives). `Map<schemaSlug, registeredToolName>` is one conformant realisation."

**Edge cases the implementer must watch:**

- The `SHUTDOWN_AWAIT_CAP_MS` demotion is the one site with a real implementer-visible knock-on: `V18c`'s build-time literal-read assertion currently pins the integer literal `2000`. The recommended phrasing keeps the named constant in the same block (so the assertion still finds a symbol to read), but explicitly allows the value to drift within a tolerance. If the spec instead chose to allow the value itself to be implementation-owned, `V18c`'s assertion arm against this constant MUST be retired in the same edit, and the cross-references in `host-interfaces-services.md` / `subagent.md` / `session-shutdown-semantics.md` / `version-bump-step2.md` / `version-bump-triggers.md` MUST stop naming the literal. The recommended phrasing keeps the build-time gate intact, so prefer it.
- The seam-minimality restatement MUST preserve the prohibition on a fresh `event.reason` re-read at the discriminator site — that prohibition is load-bearing against the unknown-reason path (a fresh access would either re-throw or read a value already classified as unknown). The new wording above retains it as an explicit MUST NOT.
- The `Map` demotion MUST NOT loosen the byte-equality check on cache hit: the existing `loom/runtime/registration-cache-collision` rule (slug equality without byte-equality is non-conformant) stays unchanged. Only the collection type is demoted.
- The `=== undefined` demotion MUST preserve the discriminator-value invariant verbatim — the failure mode the original prose names (split operator dedup buckets across implementations) is a real consequence of getting the discriminator wrong, and the restated invariant must make that consequence explicit.

## Consequence

**Severity:** advisory

Two implementers following these rules verbatim produce code with identical observable behaviour, so user-visible outcomes are unchanged. The cost is narrowed implementation freedom: a contributor who would pick a different but equivalent realisation (e.g. an immutable post-snapshot record in place of two named seam variables, or a hand-rolled bounded settle-all in place of `Promise.allSettled`) cannot do so without flagging a spec violation in code review. The cluster also dilutes the prescriptive-vs-behavioural boundary the rest of PIC works hard to preserve, training future readers to treat further mechanism-pinning as normal.

## Problem

The implementation-task **Implement** step of the per-phase TDD ritual in `docs/plan_topics/conventions.md` bans speculative APIs as "unused exports, public hooks no test in this leaf or its **declared downstream consumers** exercises." The phrase "declared downstream consumers" occurs exactly once in the entire plan corpus and is never defined; the §Leaf format section declares no field, relation, or token by that name. At least three readings select different leaf sets (reverse-`Deps.` leaves, the strictly narrower Class-2 seam consumers, or runtime importers). Because the clause is a cross-cutting self-review rule every implementation task applies, the ambiguity is exercised against every leaf.

## Issue introduction

**Verdict:** introduced-by-prior-fix (single commit)
**Introducing commit:** `aa2acf17d03e52ce422a77d16e1371bf5c367fc0` — "pi-loom plan: resolve \"Step 2 of TDD ritual minimum code invites test-gaming and speculative APIs is undefined\"" (2026-05-05)
**History:** `git log -S "declared downstream consumers" --all --reverse` identifies `aa2acf1` as the commit that first introduced the literal phrase into the plan corpus; `git show aa2acf1` confirms it added the parenthetical to `plan_topics/conventions.md` ("…public hooks no test in this leaf or its declared downstream consumers exercises) that anticipate later leaves"). That commit was itself the resolution of an earlier plan-review finding ("speculative APIs is undefined") — the fix replaced the undefined adjective "No speculative APIs" with a parenthetical that introduced the new undefined term "declared downstream consumers"; the intended reverse-`Deps.` meaning was stated only in the resolution finding's own prose and never landed in the `conventions.md` definition. A later edit `f9c5354` (2026-06-09, two-task TDD pairing convention) restructured the line and dropped the "that anticipate later leaves" tail but preserved the undefined term; `31ff060` (2026-05-08) moved the file into `docs/`. The defect is original to the `aa2acf1` clause, not to plan inception, and has carried its undefined meaning through every later edit. (Note: this corrects the existing finding `F-0060`, which attributes introduction to `f9c5354`/`present-since-inception`; `git log -S` over the plan corpus shows `f9c5354` did not contain the phrase before that commit.)

## Solution approach

In the Implementation-task **Implement** sub-step of `docs/plan_topics/conventions.md`, rewrite the speculative-API ban to re-point the consumer set from the undefined "declared downstream consumers" to the reverse-`Deps.` relation already defined in §Leaf format (leaves that list this leaf in their `Deps.` field). Reading (a), reverse-`Deps.`, is the correct target: it is mechanically determinable, names the only consumer relation the plan declares, and matches the surface-then-runtime intent recorded in the introducing commit's resolution note.

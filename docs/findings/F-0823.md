## Problem

Item (a) of the version-bump editorial-review checklist (`version-bump-step2.md`, anchor `bump-checklist-instance-survival`) carries a one-shot, binary, dispositive obligation: the contributor MUST record an authoritative determination of whether loom's package-loaded `extensions/index.ts` is governed by Pi's documented teardown-and-rebind lifecycle or provably differs from it. The step-2 preamble defines only a single per-item recording shape (`pass` / `fail` / `N/A` with a one-line rationale), which is shaped for the recurring per-bump audit outcome and gives no mapping from the binary determination onto that slot. An implementer is therefore left to guess among incompatible recording disciplines. Because V9l is blocked until item (a) "authoritatively resolves" the contradiction, while the V18c release-gate checks only that a per-item outcome was recorded in the mandated shape, neither gate has a recordable shape to test the determination against.

## Issue introduction

**Verdict:** defect-introducing commit identified
**Introducing commits:** `7447d1e` (2026-06-10) — *pi-loom plan: resolve "Session-only degraded-state presupposition (a) contradicts Pi's documented lifecycle"*
**History:** Commit `7447d1e` extended item (a) with the "Open at the loom 1.0 pin, not merely a future-minor re-audit" block and the `MUST record an authoritative determination …` obligation, in the same edit that anchored `bump-checklist-instance-survival` and added `V9l`'s blocked-obligation linkage to it. The step-2 preamble's `pass / fail / N/A with one-line rationale` recording schema predated this commit and was not extended in the same edit to carry a binary determination distinct from a per-bump audit outcome. The defect is therefore a single-commit introduction: a new obligation landed on item (a) without a corresponding extension of the recording schema that the preamble fixes. No later commit on the file rewords either side of the mismatch.

## Solution approach

Define a recording shape for item (a)'s one-shot authoritative determination that is distinct from the per-bump `pass` / `fail` / `N/A` audit outcome, stating it in the step-2 preamble recording-schema sentence and referencing it from item (a)'s body at anchor `bump-checklist-instance-survival`. Extend the V9l unblocking-gate and the V18c release-gate prose so each can test against the determination's recorded value. Cross-reference the new shape from the `bump-step-2-outcome-recording-conventions` block alongside item (e)'s existing item-specific recording carve-out.

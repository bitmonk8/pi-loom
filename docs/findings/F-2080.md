## Problem

In the *Green-path emission contract* (anchor `audit-non-empty-scan-canary`), the canary record's `proposed-resolution` field permits an implementation-chosen `key=value` pair set in place of the canonical `walked=<N> recognised=<M>`, which defeats the same sentence's stated purpose — letting a CI parser inspect stdout to confirm both counters on a green build. A parser not bespoke to the concrete adoption cannot match renamed keys, and no other clause pins the chosen names to a parser-discoverable invariant. This is the sole CI-parser-facing field left implementation-defined in an otherwise tightly-pinned wire format. The "in that order" qualifier compounds the looseness: a name-addressed pair set is order-independent, so "in that order" is meaningful only to a positional parser that ignores key names.

## Issue introduction
**Verdict:** present-since-inception (single introducing commit)
**Introducing commits:** `e5cd514` — `pi-loom spec: resolve "T35 SDK capability inventory closed-set negative-direction audit"` (2026-05-09)
**History:** The corpus is git-tracked. `git log -S "walked=<N> recognised=<M>"` and `git log -S "Green-path emission contract"` both bottom out at `e5cd514`, which added the entire *Non-empty-scan canary* / *Green-path emission contract* block. Inspecting that commit's diff shows the `proposed-resolution` counter clause was authored from the start with both the renamable-keys alternative ("or an equivalent leaf-pinned plain-ASCII key=value pair set …") and the CI-parser-confirmation rationale ("so a CI parser may inspect the audit's stdout to confirm both counters were observed") side by side — the tension was baked in when the canary contract was first written. Later commits (`9e53613`, `a6d82db`, and the `81bf52a` page-collapse) only reworded the alternative ("leaf-pinned" → "implementation-chosen", dropped `plan_topics` references, `V1` → `loom 1.0`); none altered the substance of the loosening. No earlier ancestor of the canary content exists.

## Solution approach

Narrow the `proposed-resolution` encoding in the *Green-path emission contract* to the canonical `walked=<N> recognised=<M>` form as the sole permitted encoding, on the same footing as the rest of the *Wire serialisation* contract, deleting the implementation-chosen-alternative latitude from both the contract sentence and the trailing PIC-pins summary sentence. Pin the literal key names `walked` and `recognised` so a CI parser confirms both counters by exact key-name match. Resolve the "in that order" contradiction, since fixed key names make emission order immaterial to a name-addressed parser.

## Problem

GOV-22's *defining obligation site* definition excludes "back-references to obligations another page owns" from the per-commit coinage trigger, but this exclusion is semantic and undecidable from the line text alone. A `MUST` re-stating a rule owned elsewhere is indistinguishable on the line from a co-located restatement, which GOV-9 explicitly permits ("MUST either state the rule locally or reference it by a markdown link…"). Two reviewers reading the same edit will disagree about whether a given MUST is owned locally or is an excluded back-reference, and the disagreement lands on opposite sides of the coining obligation. Because GOV-22 is reviewer-and-committer enforced with no CI gate, the exclusion is the obligation's boundary and the text supplies no rubric for the borderline case.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `4f588e0` (Wed 3 Jun 2026, "pi-loom spec: resolve \"Non-narrative pages with zero coined REQ-IDs non-referenceable\"")
**History:** GOV-22 was authored in `4f588e0` with the disputed clause already in place — the *defining obligation site* definition's closing `…and excluding back-references to obligations another page owns` appeared in the original add to `docs/spec_topics/governance.md`. The file split commit `f5e89f4` (Thu 4 Jun 2026) relocated the paragraph verbatim into `docs/spec_topics/governance/req-id-prefix-table-active-b.md` without touching the exclusion text. No commit between authorship and the current `main` has edited the predicate. The defect is therefore an authoring oversight at GOV-22's inception rather than a regression introduced by a later edit.

## Solution approach

Clarify the "back-references to obligations another page owns" exclusion in GOV-22 (`#gov-22`) so a reviewer can decide the borderline case, framing the predicate as reviewer judgement on the same footing as GOV-8 *Pure rewording*. Make the boundary between an obligation the page owns and an excluded restatement of another page's rule operable against the source line plus its surrounding paragraph, and reconcile it with GOV-9's express permission to restate a depended-upon rule locally.

## Solution constraints

- Must not make GOV-22 contradict GOV-9's express permission to restate a depended-upon rule locally.
- Out of scope: introducing a CI gate or mechanical predicate — GOV-22's enforcement is reviewer-and-committer with no CI gate reserved, consistent with its sibling governance MUSTs.

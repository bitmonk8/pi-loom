## Finding

GOV-22 (`docs/spec_topics/governance/req-id-prefix-table-active-b.md`) defines the per-commit coinage trigger over the set of *defining obligation sites*. The trigger's exclusion list mixes two kinds of predicate:

> excluding tokens inside fenced code blocks, inline code spans, and HTML comments (the GOV-3 exclusion list) and **excluding back-references to obligations another page owns**.

The first three exclusions (fenced code blocks, inline code spans, HTML comments) are structural — a line-level scanner can decide them from the post-diff text alone. The fourth — "back-references to obligations another page owns" — is semantic: it asks the committer/reviewer to recognise that a `MUST`/`MUST NOT` token, in normative prose, on a non-narrative page, is *re-stating* an obligation whose owner lives elsewhere. There is no surface signal on the line that partitions a new obligation from a back-reference: a back-reference rendered as a link with anchor `#prefix-n` looks the same as a co-located restatement; a co-located restatement is in fact what GOV-9 explicitly permits ("MUST either state the rule locally or reference it by a markdown link…"). Two diligent committers reading the same edit will disagree about whether a given MUST is "owned here" or "owned elsewhere and re-stated for local readability", and the disagreement lands on opposite sides of the coining MUST.

The undecidability matters even though GOV-22 commits to reviewer-and-committer enforcement ("no CI gate is reserved"): the exclusion is the *boundary* of the obligation, not a CI-only concern. A reviewer asked to enforce GOV-22 on a PR has no rubric for the borderline case — restating a rule locally per GOV-9 is by construction a `MUST` on a non-narrative page that another page owns. Under a strict reading of the exclusion this is exempt; under a loose reading every local restatement requires its own coined REQ-ID, generating spurious anchors and noise. The spec leaves both readings open.

## Solution Space

**Shape:** multiple

### Option A — Replace the semantic exclusion with a structural-proxy disjunction

**Approach.** Strike "and excluding back-references to obligations another page owns" and replace with a disjunction of surface-decidable signals. A `MUST`/`MUST NOT` token is excluded iff it sits in any of: (a) a link-target slot (the bracketed text or URL of a markdown link), (b) an inline code span (already covered), (c) the *same paragraph* as an existing `#prefix-n` anchor whose target prefix is owned by a *different* page (recognisable by resolving the anchor against the GOV-3 prefix table), or (d) the same paragraph as an existing `per PREFIX-N` / `under PREFIX-N` / `cited at PREFIX-N` lexical marker that names a non-owning prefix. Any `MUST` token that fails all four tests is in scope of the coinage trigger.

**Spec edits.** Rewrite GOV-22's *defining obligation site* definition in `req-id-prefix-table-active-b.md` to enumerate the four structural signals; add a one-sentence note that any genuinely-back-referencing MUST whose surface form is not in the proxy set is in scope and must either acquire a coined REQ-ID or be reworded to carry one of the structural markers.

**Pros.** Boundary is decidable by a single-pass scanner over the post-diff text without semantic reading. Two reviewers will agree.

**Cons.** Tightens coinage scope: a normative restatement on the same page that happens to omit all four markers (e.g. a recapitulation paragraph) now MUSTs a fresh REQ-ID even when the substantive obligation belongs elsewhere. Authors will need to insert a marker or accept the new ID. The set of proxies must be exhaustive enough to cover real cross-page restatement patterns; under-coverage forces spurious coining.

**Risks.** Interaction with GOV-9, which expressly permits local restatement: under Option A a local restatement that does not include a structural marker now must coin, which is a stricter regime than GOV-9 contemplates. Mitigate by requiring authors to add one of the markers (`per PREFIX-N`, link, etc.) when restating — that pattern is already idiomatic in the corpus.

### Option B — Keep the semantic predicate, add a rubric and worked examples

**Approach.** Retain the semantic exclusion (a back-reference to an obligation another page owns) but pin a reviewer rubric: a `MUST` token is a back-reference iff every normative claim the sentence makes is sourced — verbatim or by paraphrase — from a `PREFIX-N`-anchored paragraph on another page that the surrounding paragraph cites by `#prefix-n` link or `per PREFIX-N` token. Add three worked examples (one in-scope new obligation, one out-of-scope cross-link restatement, one borderline restatement resolved by adding a `per PREFIX-N` citation), and reaffirm that the predicate is reviewer-judgement on the same footing as GOV-8's *Pure rewording* boundary.

**Spec edits.** Append a parenthetical "(reviewer judgement: a MUST is a back-reference iff every normative claim it makes is sourced from a `PREFIX-N`-anchored paragraph on another page cited in the surrounding paragraph)" after the disputed clause, and add a short *Worked examples* block at the GOV-22 paragraph's tail covering the three cases above.

**Pros.** No tightening of GOV-22's scope, no conflict with GOV-9's local-restatement permission, no churn for existing co-located restatements. Honest about the predicate's nature (it joins GOV-8 *Pure rewording*, GOV-9 cross-link trigger, GOV-30 lock-step, and GOV-31 in the reviewer-judgement family).

**Cons.** Stays undecidable — two reviewers can still disagree on a borderline case the worked examples do not anticipate. Adds prose weight to an already-dense paragraph.

**Risks.** The worked examples may themselves drift from the corpus over time; revisit at any GOV-22 amendment.

### Recommendation

Take **Option B**. GOV-22's enforcement model is already reviewer-and-committer ("no CI gate is reserved"), and the rule sits in a cluster of governance MUSTs (GOV-8 *Pure rewording*, GOV-30 lock-step, GOV-31 single-source rewording) that the spec elsewhere acknowledges are reviewer-judgement; introducing a structural predicate here would single GOV-22 out for a stricter regime than its siblings and would conflict with GOV-9's express permission to restate rules locally. The rubric-plus-examples approach matches how the corpus already disposes of the GOV-8 *Pure rewording* and GOV-30 lock-step boundaries, and the reviewer-friction it leaves behind is the same friction those rules already accept. Implement Option A only if a subsequent ratchet establishes a corpus-wide policy that all reviewer-judgement governance MUSTs must be replaced by structural proxies — at which point GOV-8, GOV-9, GOV-30, and GOV-31 should be edited together.

Edge cases the implementer must watch: the worked examples must show the *borderline-restatement-resolved-by-adding-`per PREFIX-N`* case explicitly (this is the recurring pattern that motivates the predicate); the rubric must be careful not to imply that the presence of any `#prefix-n` link on the page is sufficient to discharge a MUST as a back-reference — the link must sit in the surrounding paragraph and the cited prefix must be owned by a non-current page.

## Consequence

**Severity:** advisory

The rule is reviewer-enforced (no CI gate), so the spec does not break in the mechanical sense, but the boundary of the obligation is ambiguous: a reviewer cannot tell from the GOV-22 text whether a co-located restatement permitted under GOV-9 also triggers coining under GOV-22. The likely failure mode is silent inconsistency across PRs — some restatements acquire fresh REQ-IDs, others do not — eroding the prefix table's signal value over time and leaving GOV-22's residue-draining intent half-met.

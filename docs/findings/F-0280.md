## Problem

`V9k` closes the *Extension-bootstrap SDK failures* granularity rule with five Tests bullets that are each a distinct control-flow circuit with no shared assertion surface: a `registerMessageRenderer` degrade-and-proceed, a `registerCommand` per-loom partial drop, a `registerFlag` whole-extension abort, a `pi.on(...)` whole-extension abort, and a `session_start`-time `getCommands()` read-failure pending-list drop (which, unlike the four write-side surfaces, must NOT write drain state). The five surfaces span three behaviour classes plus a read-side no-write obligation, forcing a reviewer to hold all five circuits in one pass. The single compound Ships-when does not localise which surface a regression touches.

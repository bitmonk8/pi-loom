## Problem

In `docs/spec_topics/expressions.md` §"Grammar disambiguation", the *Struct-expression in scrutinee position* bullet's prose and example describe two different rules and a parser implementer cannot tell which is normative. The prose ("these positions require parentheses around any constructor") reads as wrapping the whole scrutinee, while the example `if (Author { ... } == author) { ... }` is also consistent with wrapping only the constructor. Bare scrutinee conditions are legal in general (the `IfStmt ::= "if" Expr StmtBlock` production in `grammar.md` admits a bare `Expr`, and `control-flow.md`'s own example uses `if author.experience_years < 2 { ... }`), so the constructor rule is an added requirement whose firing level — whole condition vs. each constructor — is undefined. Two conforming implementations will accept and reject different source files with no diagnostic to disambiguate.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `fae85d3` ("Tighten spec for implementation-plan readiness", 2026-05-04)
**History:** The "Grammar disambiguation" section, including the *Struct-expression in scrutinee position* bullet with its current prose ("These positions therefore require parentheses around any constructor") and example (`if (Author { … } == author) { … }`), was added in `fae85d3` when the spec was tightened for implementation-plan readiness. The prose/example mismatch entered the corpus with the rule and was preserved verbatim through the `fecb504` split that moved `spec.md` into per-topic shards under `spec_topics/`. No subsequent commit has touched this bullet (`git log -S "parentheses around any constructor"` returns only the introducing commit and the split).

## Solution approach

Rewrite the *Struct-expression in scrutinee position* bullet so the parenthesisation rule applies locally to each constructor in scrutinee position rather than to the whole scrutinee, and align the example so its parens directly enclose the constructor (e.g. `if (Author { ... }) == author { ... }`). Clarify that an outer paren around the whole condition is neither required nor sufficient to satisfy the rule for a constructor it merely contains, and that bare scrutinee conditions remain admissible. Address the nested-constructor, `${...}`-interpolation, and call-argument edge cases where the brace boundary is already disambiguated and the rule does not re-fire.

## Solution constraints

- Out of scope: the `IfStmt ::= "if" Expr StmtBlock` production in `grammar.md` — the scrutinee rule is an added requirement layered on the bare-`Expr` grammar, not a grammar change.

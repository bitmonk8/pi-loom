## Finding

The six failure-mode template rows in [`binder/determinism-cancellation-failure.md` — *Failure-mode templates (normative)*](../../docs/spec_topics/binder/determinism-cancellation-failure.md) share a uniform user-visible shape — `loom /<name>: argument binding <verb-phrase>` — in five rows: `argument binding needs more info`, `argument binding failed — could not parse arguments`, `argument binding produced invalid args`, `argument binding cancelled`, and the `ambiguous arguments` row (which alone drops the gerund for parallelism with `needs_info`). The transport-failure row breaks the pattern by switching to a noun phrase: `loom /<name>: argument binder unavailable (<provider>: <message>)`. Because the surrounding template text is **MUST-emit-verbatim** (the same paragraph says "Renderers MUST emit the surrounding template text verbatim" and "Wording changes are spec-versioned breaking changes"), conforming implementations are obligated to ship the odd-one-out wording. The drift is observer-visible in any UI that surfaces these notes side-by-side.

Separately, the page H1 — `# Determinism cancellation failure` — is an unpunctuated three-noun compound covering three independent sub-topics that each have their own `##` section on the page (Determinism, Cancellation, Failure modes). Without a separator, the heading reads as one compound concept ("determinism-cancellation-failure" as a single notion) rather than as an enumeration. The filename slug (`determinism-cancellation-failure.md`) drove the auto-generated H1 when the monolithic spec was split into per-topic files; the slug is acceptable as a path, but the rendered heading needs human-grade punctuation. Twelve cross-references in `docs/plan_topics/` and other spec pages point at this file by path, not by title, so retitling is safe.

The two defects are independent: the row-wording fix touches one table cell; the H1 retitle touches the page header. They should be resolved sequentially (smaller first), not bundled, to keep each fix's diff narrow.

**History:** `git log -S "argument binder unavailable" --reverse` resolves to `83802db` as first introduction; that commit is also the only commit before `f5e89f4` to touch the string. `git log --diff-filter=A -- docs/spec_topics/binder/determinism-cancellation-failure.md` resolves to `f5e89f4` (file creation at the spec-split). The two sub-defects originated in different commits with different intents (a table authored by hand vs an H1 derived mechanically from a path), so the finding is a true multi-commit interaction rather than a single regression.

## Solution Space

**Shape:** multiple

### Option A — Rename the transport-failure row to the gerund form

**Approach.** In the *Failure-mode templates (normative)* table, change the transport-failure row's system-note column from `loom /<name>: argument binder unavailable (<provider>: <message>)` to `loom /<name>: argument binding unavailable (<provider>: <message>)`. Leave the `Cause` column ("Binder model transport failure (after 1 retry)") unchanged.

**Spec edits.**
- `docs/spec_topics/binder/determinism-cancellation-failure.md` — single cell in the six-row table.

**Pros.**
- Restores the uniform `argument binding <…>` prefix across all six rows; greppable.
- Pure wording edit; no anchors, no cross-refs, no schema, no taxonomy touched.

**Cons.**
- Counts as a spec-versioned breaking change under the table's own MUST ("Wording changes are spec-versioned breaking changes"); implementations on the prior wording must re-render.

**Risks.**
- Test fixtures in `V11f` / `V11f-T` that assert the verbatim template string must be updated in the same change-set; missing them produces a false test failure rather than a silent drift.

### Option B — Retitle the page H1 to a punctuated three-topic enumeration

**Approach.** Change the H1 from `# Determinism cancellation failure` to `# Binder determinism, cancellation, and failure modes`. The filename, anchors (`#failure-class-taxonomy`, `#failure-mode-templates-normative`, `#per-invocation-retry-budget`), and `## ` sub-headings stay unchanged. Twelve `../spec_topics/binder/determinism-cancellation-failure.md` cross-references in `docs/plan_topics/` and `docs/spec_topics/` reference the file by path, not by title, so they need no update.

**Spec edits.**
- `docs/spec_topics/binder/determinism-cancellation-failure.md` — H1 only.

**Pros.**
- Heading reads as an enumeration of the three `##` sections it contains, matching the page's actual scope.
- Adds "Binder" to the title so the page is discoverable from a file tree without context.
- Zero cross-reference fan-out (paths and anchors are untouched).

**Cons.**
- Longer heading; mild departure from the auto-slugged style of sibling files (`binder-bypass-and-envelope.md` → `# Binder bypass and envelope`, `binder-model-and-context.md` → `# Binder model and context`). Sibling pages are themselves two-topic compounds and arguably warrant the same treatment under a separate finding; this one fix does not address them.

**Risks.**
- None observable; the filename and all anchors are unchanged, so no link or tooling breaks.

### Recommendation

Apply both options, **Option A first**, then Option B as a separate fix. Option A is the smaller, more bounded edit (one table cell + two test-fixture strings) and is the one that has an observable downstream consequence (the gerund/noun split is shipped through the MUST-verbatim renderer). Option B is purely editorial and lands on the same page without touching anchors or cross-refs; sequencing it second keeps each fix's diff small and each test pass independently loopable.

Edge cases the implementer must watch:
- Option A: update the `V11f` / `V11f-T` template assertion list and any test snapshot containing `argument binder unavailable`. Search the entire corpus (`grep -rn "argument binder unavailable" docs/ tests/ src/`) — the string occurs in `docs/spec.md` legacy review references and possibly in plan-coverage prose.
- Option B: do not rename the filename or anchors; the H1 change is the only edit. Verify with `docs/_tools/check_links.py` after the edit.

## Consequence

**Severity:** advisory

Users see one binder failure note that reads in a different register from the other five, and operators grepping notes for the consistent `argument binding ` prefix miss the transport-failure case. The H1 makes the page harder to discover and orient against from a file tree or generated TOC. Neither defect blocks implementation — the templates are still MUST-verbatim and unambiguous — but both leak into observer-visible surfaces.

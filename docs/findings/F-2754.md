## Problem

The `FN-5` Tests bullet on `V3d` (and verbatim on its `V3d-T` partner) asserts the function body's produced value "is absent on fail/cancel" at the function-result seam without naming what "absent" looks like to a programmatic caller. Read on its own, "absent" admits mutually-incompatible observables: a JS `undefined` returned through the value channel, a rejected seam promise, or a distinct sentinel. The spec pins exactly one — `functions.md#final-value-language-definition` states that on failure (`?` propagation, panic, exhausted runtime ceiling) and on cancellation NO final value flows and the caller observes only the corresponding terminal `Err` envelope, with per-outcome contents owned by `errors-and-results/error-model.md#terminal-outcomes`. A leaf asserting promise-rejection or `undefined` would ship green against behaviour the spec does not specify, leaving the actual `Err`-envelope contract unverified.

**Verdict:** present-since-inception

**History:** `git log -S "absent on fail/cancel" -- docs/plan_topics/` shows the loose "absent on fail/cancel" phrasing entered with the original plan build (`c6a664e`) on the predecessor `FN`-citation bullet. `7a8565a` ("resolve 'V3d-T over-asserts final-value propagation'") reworded the bullet to the current "the function body's produced value is its final value at the function-result seam on success and is absent on fail/cancel" and updated `Ships when`, but preserved the under-specified "absent" term unchanged; it neither introduced nor repaired the ambiguity. `5f4d271` propagated the same phrasing into `V15a`. No commit ever named the concrete observable.

## Solution approach

Clarify the `FN-5` Tests bullet on `V3d` to name the concrete observable: on fail/cancel the function-result seam yields the corresponding terminal `Err` envelope and no value flows through the success channel. Cite both `functions.md#final-value-language-definition` and `errors-and-results/error-model.md#terminal-outcomes`. Mirror the named observable onto the identical `V3d-T` `FN-5` bullet and bring the `V3d` `Ships when` "absent on fail/cancel" clause into line.

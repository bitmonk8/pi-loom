## Problem

`V2b`'s **Adds** lists "a runtime AJV safety-net for statically-unresolvable operands" as a `V2b` deliverable, and its **Ships when** restates that it "leaves statically-unresolvable operands to the runtime AJV safety-net." Neither line names the seam performing that validation, and `V2b`'s **Deps** (`V2b-T`, `V2a`, `V5d`) carry no edge to a runtime validator — `V5d` is the parse-time schema-subset reject gate, not a runtime validator. `V2b` is the static `T₁ ⊑ T₂` compatibility engine; it does not itself invoke AJV at runtime, which is performed by the `V8c` `SchemaValidator` seam at the value-binding boundary sites that own it (`V6b`, `V11g`, `V15f`, `V14a`). As written the backstop's existence and reachability are asserted rather than traced to any seam or boundary leaf.

## Issue introduction

**Verdict:** present-since-inception (with subsequent oscillation at the same locus)
**Introducing commit:** `c6a664e` (2026-06-10, "pi-loom plan: build/update plan for spec.md + review") — `V2b`'s Adds was authored from the start with "a runtime AJV safety-net for statically-unresolvable operands," naming no seam, and Deps `V2b-T, V2a, V5d` carried no validator edge.
**History:** The defect locus has been edited twice since, without durable resolution. `210ed9b` (2026-06-10, "resolve 'V2b ship-gate references the runtime AJV validator seam (V8a) outside its declared dependency closure'") reworded the line to "a deferral marker … consumed by the downstream runtime-validator sites (`V6b`, `V11d`)" to drop an out-of-closure `V8a` seam reference. `a0f6c6b` (2026-06-17, "resolve 'V2b Adds claims deferral marker consumed by V6b/V11g but neither binds the seam'") then reverted to the original bare "runtime AJV safety-net" phrasing after the consumer-site framing was flagged for not binding the seam. The current finding re-flags that reverted phrasing — the underlying ownership question (`V2b` does not itself invoke the validator) was never settled, so the wording has cycled between two incompatible framings.

## Solution approach

Rewrite `V2b`'s **Adds** trailing clause and its **Ships when** clause so neither claims `V2b` delivers or performs runtime AJV validation; instead defer statically-unresolvable operands to the `V8c` `SchemaValidator` (AJV) seam, validated at the boundary leaves that own that validation — `V6b` (`params`), `V11g` (post-merge binder args), `V15f` (`invoke`), and `V14a` (tool-call returns). Limit `V2b`'s own claim to the static `⊑` relation and its per-site diagnostics, stating that `V2b` itself does not invoke the validator.

## Solution constraints

- Do not add a `Deps` edge from `V2b` to `V8c`; the `V8c` reference is a non-binding cross-reference, since adding the edge would over-couple `V2b`'s pickability to a seam it never invokes.
- Do not reintroduce either prior failing framing: a `V2b`-owned "deferral marker" consumed by `V6b`/`V11g`, or the bare "runtime AJV safety-net" deliverable.
- The spec is read-only for this fix.

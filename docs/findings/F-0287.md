## Finding

The `SLSH-2` streaming Tests bullet in both `V12a` and `V12a-T` ends with a trailing sentence that is editorial-process / host-risk rationale, not a test assertion:

> The real-Pi behaviour this models — Pi's TUI streaming tokens into the transcript before the terminal `agent_end` settles `ctx.waitForIdle()` — is an unpinned host presupposition audited by editorial review per [bump-checklist item (ak)](../spec_topics/pi-integration-contract/version-bump-step2.md#bump-checklist-user-visible-streaming-ordering).

The binding obligation of the bullet is already fully stated by the preceding clauses: streamed tokens are observable before `ctx.waitForIdle()` resolves, the test is driven through the Pi session double, and the asserted ordering is the one the session-double fidelity contract requires the double to model. The trailing sentence adds only why the modelled real-Pi ordering is unverified between version bumps and who audits it (editorial review under bump-checklist item (ak)) — process/risk context, not something the test asserts. Embedded in a normative Tests bullet it reads as if the editorial audit were part of the test's scope.

The identical sentence appears in both the implementation leaf (`V12a`) and its paired tests leaf (`V12a-T`), so it is a maintenance surface that must be kept in sync. It belongs in a scope/risk note (the `(ak)` cross-reference is worth retaining as scope context), not inside the assertion.

## Issue introduction

**Verdict:** introduced-by-later-edit
**Introducing commits:** 85fc906 ("pi-loom plan: resolve \"streamed-token-before-waitForIdle() ordering has no version-bump checklist item\"", 2026-06-10)
**History:** The SLSH-2 streaming bullet predates this commit. `git log -S 'unpinned host presupposition audited by editorial review'` over `V12a-slash-dispatch.md` / `V12a-T-slash-dispatch.md` returns a single commit, 85fc906, and its diff appends the trailing sentence to the pre-existing bullet in both leaves. That commit resolved a different finding (the streamed-token ordering lacking a version-bump checklist item) and correctly added the `(ak)` cross-reference, but inlined the host-presupposition / editorial-review rationale into the normative test bullet rather than placing it in a scope note — that placement is the defect.

## Solution Space

**Shape:** single

### Recommendation

Strike the trailing sentence from the SLSH-2 streaming Tests bullet in both leaves. The literal text to remove (including the leading space that joins it to the prior sentence) is:

> ` The real-Pi behaviour this models — Pi's TUI streaming tokens into the transcript before the terminal `` `agent_end` `` settles `` `ctx.waitForIdle()` `` — is an unpinned host presupposition audited by editorial review per [bump-checklist item (ak)](../spec_topics/pi-integration-contract/version-bump-step2.md#bump-checklist-user-visible-streaming-ordering).`

In `docs/plan_topics/V12a-slash-dispatch.md`, relocate the substance to the `Adds` field as a scope/risk note that retains the `(ak)` cross-reference — e.g. record that the modelled real-Pi streaming-before-`waitForIdle()` ordering is an unpinned host presupposition audited by editorial review per bump-checklist item (ak), with the same link target `../spec_topics/pi-integration-contract/version-bump-step2.md#bump-checklist-user-visible-streaming-ordering`.

In `docs/plan_topics/V12a-T-slash-dispatch.md`, delete the sentence; its `Adds` is only a back-reference to the paired implementation leaf, so the relocated note in `V12a` is the single home for the scope context.

Both leaves carry the sentence verbatim today, so apply the strike to both in the same pass to keep them in sync.

## Consequence

**Severity:** cosmetic

No runtime behaviour or gate is affected; the binding assertion is already stated by the preceding clauses. The cost is organisational: an implementer scanning the SLSH-2 Tests bullet may mistake the editorial-review audit for a test obligation, and the identical sentence in two leaves is a drift surface.

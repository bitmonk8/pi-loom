## Finding

`future-considerations/model-changes-and-non-goals.md` § *loom 1.0 non-goals* lists eight non-goal bullets ("No per-loom sandbox or capability model", "No formal source-language migration mechanism …", "No non-Node JavaScript host support", "No concurrent user sessions in the same host process", "No reliance on a Pi extension-host stdio-capture facet", "No parallel-`invoke` surface", "No parallel fan-out across prompt-mode `.loom` callees", "No admission cap on in-flight loom invocations"). Only one of the eight — `pi-stdio-capture-facet` — carries an inline `<a id>`. The remaining seven sit on a bare unordered-list item with no anchorable identifier; the only stable target a cross-reference can cite is the section-level pair `#loom-1-0-non-goals` / `#v1-non-goals`.

Every cross-reference from another spec page that wants to invoke a *specific* non-goal therefore has to bold-paraphrase the bullet name and link the paragraph to the section anchor. That pattern is in active use: `host-prerequisites.md` cites *"No concurrent user sessions in the same host process"* four times (`#v1-non-goals` on each of the degraded-state-prereqs, binder-model-registry, settings write-back, and operator-always-present paragraphs); `session-model-and-appendix.md` SM-1 cites it again at `#loom-1-0-non-goals`; `tool-registration-lifetime.md` and `version-bump-step2.md` both cite the *same* non-goal *plus* "No per-loom sandbox or capability model"; `diagnostic-emission-isolation.md` and `session-only-degraded-state.md` cite the stdio-capture bullet (the one bullet that *does* have a per-item anchor, which is how those two sites successfully fragment-link). `language-and-architecture.md` opens with an aggregator paragraph that enumerates all eight items by bold paraphrase. No site can fragment-link to the seven anchorless bullets except through the section heading, and the bold paraphrase is not byte-checked: a future GOV-7 *Rename* on a bullet's wording silently breaks every paraphrasing citation with no anchor-equivalence to catch the drift.

This is the same defect class the spec has already remediated for the bullet that does carry an anchor (`pi-stdio-capture-facet`), and it is structurally identical to the un-anchored MUST patterns governance addresses elsewhere (e.g. the *Recovery-path prohibitions lack per-obligation REQ-IDs* finding on `session-only-degraded-state.md`, which raises the same gap on a different cluster).

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:**

- `52b9101` (2026-05-11) — *pi-loom spec: resolve "T24 — Fork-reason watcher closure leaves the extension in an unspecified, silently degraded state".* Inserted the seventh bullet ("No reliance on a Pi extension-host stdio-capture facet") *with* an inline `<a id="pi-stdio-capture-facet"></a>` because the patch needed `diagnostic-emission-isolation.md` and `session-only-degraded-state.md` to fragment-link to it from a *single* bullet. The other six bullets were left unanchored.
- `8e3ccb4` (2026-06-05) — *pi-loom spec: resolve "GOV-21 bundles five independently testable sub-clauses".* Added GOV-25 – GOV-29, including GOV-27 (*dual-anchored intensional definition*) and GOV-29 (*cross-page canonical-arm uniqueness*), which formalised the dual-anchor convention that *new* per-item anchors must follow but did not retroactively author them on the six pre-existing bullets.
- `f5e89f4` (2026-06-04) — *docs: enforce 100KB hard / 30KB recommended size cap on spec set.* Moved the section into its current owner-file `future-considerations/model-changes-and-non-goals.md` without altering the anchor inventory.

**History:** The defect is an asymmetric outcome of the bullet-by-bullet authoring pattern: only the bullet whose introducing commit *needed* an inbound fragment-link grew a per-item anchor; the other six bullets' introducing commit (`0e12140`) had no cross-shard inbound link to motivate one, and every later commit that added a paraphrase-citation to those bullets (`host-prerequisites.md`, `tool-registration-lifetime.md`, `version-bump-step2.md`, `session-model-and-appendix.md` SM-1, `language-and-architecture.md` aggregator) accumulated the inbound-citation evidence without back-filling the anchors. The single-commit-revert framing does not apply: reverting `0e12140` would delete the section entirely; the durable fix is to back-fill the six missing anchors and bring them under the GOV-25 – GOV-29 convention `8e3ccb4` later established.

## Solution Space

**Shape:** single

### Recommendation

Assign one stable canonical `<a id="loom-1-0-no-…">` anchor immediately before each of the seven currently-anchorless non-goal bullets, using the GOV-29-compliant slug form (canonical-arm only; no `v1-…` alias arm is required because the bullets are *new* anchor sites with zero inbound `#v1-…` cross-references — GOV-27's dual-anchor convention is forward-looking and explicitly *does not* require retro-fitting an alias arm on a site that never had one). Suggested slugs, following the bullet's leading noun-phrase:

- `loom-1-0-no-per-loom-sandbox`
- `loom-1-0-no-source-language-migration`
- `loom-1-0-no-non-node-host`
- `loom-1-0-no-concurrent-user-sessions`
- (existing) `pi-stdio-capture-facet` — leave unchanged
- `loom-1-0-no-parallel-invoke`
- `loom-1-0-no-parallel-fan-out`
- `loom-1-0-no-admission-cap`

Then repoint every paraphrase-style citation enumerated under **Spec Documents** above to the new per-bullet anchors. Per GOV-25 (*canonical-arm citation*), new cross-references in the spec corpus MUST cite the `loom-1-0-…` canonical arm; per GOV-29 (*cross-page canonical-arm uniqueness*) each canonical-arm slug must be unique across `docs/spec.md` + `docs/spec_topics/*.md`, which the slugs above satisfy because no other site currently authors any of those strings (verifiable mechanically with `grep -rn 'id="loom-1-0-no-' docs/spec.md docs/spec_topics/`).

Edge cases the implementer must watch:

1. *Aggregator lock-step.* `language-and-architecture.md`'s opening non-goals enumeration is itself a GOV-30 *multi-source-page* aggregator and a GOV-31 *integer-count* aggregator (it pins "eight items"). The seven new anchors will become the canonical fragment targets for that paragraph's bold-paraphrase enumeration; ensure each of its eight bullet-paraphrases is repointed to the corresponding per-bullet anchor in the same edit, otherwise GOV-30's lock-step convention drifts.
2. *Bullet whose body cross-references another bullet.* The "No concurrent user sessions" bullet's *Disposition for an unanticipated second session* sentence already names `pi-stdio-capture-facet` indirectly; if a future edit moves cross-bullet references inside the section to fragment-links, the same anchors are reused — no further work.
3. *No alias-retirement implication.* GOV-28 (*alias retirement discharge*) is not triggered: there are no `v1-…` alias arms on these six bullets to retire. The pre-existing section-level dual pair (`#loom-1-0-non-goals` / `#v1-non-goals`) and the `pi-stdio-capture-facet` single anchor are unaffected.
4. *Plan-side citation.* `docs/plan.md`'s Scope paragraph cites `#v1-non-goals` for the *whole section*; do not repoint it to a per-bullet anchor — its referent is the entire enumeration, not any single bullet.

## Consequence

**Severity:** advisory

Maintenance and traceability degrade: any GOV-7 *Rename* on a non-goal bullet (e.g. a future cleanup of the legacy "V1" wording, or a rephrasing that drops the "in the same host process" qualifier) silently invalidates every bold-paraphrase citation across at least eight other pages, with no anchor-mismatch to flag the drift. Implementers reading those citing paragraphs land on the eight-item section and have to scan the bullet names to find the right one. No runtime obligation is at stake; both implementers and reviewers can still produce a working system.

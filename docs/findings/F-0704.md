## Problem

The `future-considerations/model-changes-and-non-goals.md` § *loom 1.0 non-goals* section lists eight non-goal bullets, but only one (`pi-stdio-capture-facet`) carries an inline per-item `<a id>`. The other seven sit on bare list items whose only stable cross-reference target is the section-level pair `#loom-1-0-non-goals` / `#v1-non-goals`. Multiple other pages already cite specific bullets by bold-paraphrasing the bullet name and linking the section anchor — `host-prerequisites.md` (four "No concurrent user sessions" citations), `session-model-and-appendix.md` SM-1, `tool-registration-lifetime.md`, `version-bump-step2.md`, and the `language-and-architecture.md` aggregator paragraph. Those paraphrases are not byte-checked, so a future rename of any bullet's wording silently breaks every paraphrasing citation with no anchor-equivalence to catch the drift.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:**
- `0e12140` (2026-05-08) — *pi-loom spec: resolve "Non-goals are not consolidated into a single section".* Authored the section heading with the dual `<a id="loom-1-0-non-goals"></a><a id="v1-non-goals"></a>` pair and introduced the first six non-goal bullets (per-loom sandbox, source-language migration, non-Node host, concurrent user sessions, parallel-`invoke`, parallel fan-out, admission cap) with no per-item anchors.
- `52b9101` (2026-05-11) — *pi-loom spec: resolve "T24 — Fork-reason watcher closure leaves the extension in an unspecified, silently degraded state".* Inserted the seventh bullet ("No reliance on a Pi extension-host stdio-capture facet") *with* an inline `<a id="pi-stdio-capture-facet"></a>` because the patch needed `diagnostic-emission-isolation.md` and `session-only-degraded-state.md` to fragment-link to it from a *single* bullet. The other six bullets were left unanchored.
- `8e3ccb4` (2026-06-05) — *pi-loom spec: resolve "GOV-21 bundles five independently testable sub-clauses".* Added GOV-25 – GOV-29, including GOV-27 (*dual-anchored intensional definition*) and GOV-29 (*cross-page canonical-arm uniqueness*), which formalised the dual-anchor convention that *new* per-item anchors must follow but did not retroactively author them on the six pre-existing bullets.
- `f5e89f4` (2026-06-04) — *docs: enforce 100KB hard / 30KB recommended size cap on spec set.* Moved the section into its current owner-file `future-considerations/model-changes-and-non-goals.md` without altering the anchor inventory.

**History:** The defect is an asymmetric outcome of the bullet-by-bullet authoring pattern: only the bullet whose introducing commit *needed* an inbound fragment-link grew a per-item anchor; the other six bullets' introducing commit (`0e12140`) had no cross-shard inbound link to motivate one, and every later commit that added a paraphrase-citation to those bullets (`host-prerequisites.md`, `tool-registration-lifetime.md`, `version-bump-step2.md`, `session-model-and-appendix.md` SM-1, `language-and-architecture.md` aggregator) accumulated the inbound-citation evidence without back-filling the anchors. The single-commit-revert framing does not apply: reverting `0e12140` would delete the section entirely; the durable fix is to back-fill the six missing anchors and bring them under the GOV-25 – GOV-29 convention `8e3ccb4` later established.

## Solution approach

Add a stable per-item anchor to each of the seven currently-anchorless `loom 1.0 non-goals` bullets in `future-considerations/model-changes-and-non-goals.md`, using the GOV-29 canonical-arm slug form; leave the existing `pi-stdio-capture-facet` anchor unchanged. Repoint the paraphrase-style citations in `host-prerequisites.md`, `session-model-and-appendix.md` SM-1, `tool-registration-lifetime.md`, `version-bump-step2.md`, and the `language-and-architecture.md` aggregator paragraph to the new per-bullet anchors.

## Solution constraints

- New per-bullet anchor slugs MUST be unique across `docs/spec.md` + `docs/spec_topics/*.md` (GOV-29 cross-page canonical-arm uniqueness).
- Out of scope: `docs/plan.md`'s Scope-paragraph `#v1-non-goals` citation — its referent is the whole section, not any single bullet; leave it pointing at the section anchor.

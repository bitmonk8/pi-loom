## Problem

The `## Binder model` section of `binder/binder-model-and-context.md` marks its load-time model-identifier resolution algorithm with the anchor `binder-model-parse-rule` and closes the rule with "loom coins no binder-specific **parsing** convention beyond this exact-match rule." The algorithm so named resolves a free-form configured string against the model registry by exact-match lookup over `Model<Api>.id` / `Model<Api>.provider` — registry lookup, not syntactic analysis of Loom source. The word "parse" is reserved corpus-wide for source-language analysis (the `loom/parse/*` diagnostic family, the Lexical Structure page, the Grammar Appendix), so naming a registry-resolution rule a "parse rule" invites a reader to look for grammar/lexer semantics that do not exist. The misnomer reaches at least seven inbound citation sites plus an in-page self-reference.

## Issue introduction
**Verdict:** introduced-by-commit
**Introducing commit:** `54c4e39` ("pi-loom spec: resolve \"Model<Api>/ModelRegistry pin + binder-model parse rule\" cluster", 2026-06-03)
**History:** `git log -S 'binder-model-parse-rule'` and `git log -S 'id="binder-model-parse-rule"'` place the anchor's first appearance at `54c4e39`; `git grep -l 'binder-model-parse-rule\|binder-specific parsing convention' 54c4e39^` returns nothing, confirming neither the anchor nor the "parsing convention" wording existed before that commit. Inspecting `git show 54c4e39`, the commit coined the anchor and the "loom coins no binder-specific parsing convention" sentence fresh, applying the loaded word "parse" to the resolution algorithm at the moment of coinage; the pre-commit `binder.md` used "parse" only in the source-code sense (`loom/parse/*`, JSON parsing). A later commit `f5e89f4` ("enforce 100KB hard / 30KB recommended size cap on spec set") split `binder.md` into `binder/binder-model-and-context.md`, carrying the anchor and prose verbatim into the current location. The defect is therefore attributable to a single introducing commit, not an inception-time or multi-commit-interaction artefact.

## Solution approach

Rename the anchor `binder-model-parse-rule` to a resolution-named token (e.g. `binder-model-resolution-rule`) and rewrite the in-prose closing clause so it reads "resolution convention" rather than "parsing convention". Repoint every inbound `#binder-model-parse-rule` citation to the new anchor and update the "binder-model parse rule" link text to match the new term across the citing pages and the linking plan/test leaves.

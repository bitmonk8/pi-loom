## Finding

The final sentence reads: "If a future host-interop surface ever exposes a live `Result` or enum value to JS (rather than a JSON-serialised projection), this section MUST be revisited before that surface ships." This uses MUST as an editorial maintenance obligation to spec authors, embedded inside a block explicitly labeled non-normative. A reader implementing loom 1.0 should neither act on this MUST (it governs a future feature) nor can they act on it (it is an instruction to spec maintainers, not implementers). The normative-keyword usage in a non-normative block is also confusing.

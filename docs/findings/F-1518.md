## Problem

The version-bump step-2 editorial-review checklist defines a **Baseline acquisition** block (anchor `bump-baseline-acquisition`) that materialises the loom 1.0 Pi-SDK pin's `dist/*.js` tree — the canonical "from" side every `dist/*.js` diff on the page reads against. It is a shared preparatory step serving four checklist items: (e), (r) (`bump-checklist-convert-to-llm-context-entry`), (aj) (`bump-checklist-inmemory-transcript-discard`), and (am) (`bump-checklist-tools-allowlist-suppression`). Yet it is physically nested inside item (e)'s body, so a contributor working the later items must jump backwards into the middle of an unrelated, very large checklist item to find a precondition for all of them. Separately, the block's own inline consumer enumeration names only "items (e), (r), and (aj)", omitting item (am), which also diffs against the snapshot.

## Issue introduction

**Verdict:** present-since-inception (of the block)
**Introducing commit:** `ccc6e38` (2026-06-07) — *pi-loom spec: resolve "Item (e)/(r) dist/*.js baseline source unspecified"*
**History:** `git log -S 'bump-baseline-acquisition' -- docs/spec_topics/pi-integration-contract/version-bump-step2.md` and `git show ccc6e38` confirm the Baseline acquisition block did not exist before `ccc6e38`; that commit created it and placed it nested inside item (e)'s body, already cross-referenced from the remote item (r). The shared-prerequisite-buried-under-one-consumer placement was therefore present at the block's inception. Two later commits widened the consuming set but did not introduce the placement defect: `9ced9f1` (2026-06-08, subagent transcript-discard) added item (aj)'s reference, and `077afb5` (2026-06-20, F-0806) added item (am)'s reference — these compounded the back-tracking surface from two consumers to four but left the structural siting unchanged.

## Solution approach

Move the Baseline acquisition block out of item (e)'s body to a stand-alone preparatory sub-step ahead of the checklist, before item (a) (`bump-checklist-instance-survival`), so contributors meet it before any individual item requires it. Leave item (e)'s own `dist/*.js` diff prose in place under item (e); only the shared materialisation paragraph moves. Rewrite the block's inline consumer enumeration to also name item (am).

## Solution constraints

- Preserve the `bump-baseline-acquisition` anchor on the relocated block — the (e)/(r)/(aj)/(am) cross-references depend on it.

## Problem

The §"Canonical schema hash" recipe in `schema-subset.md` is pinned as part of the on-disk and on-wire contract, yet its five-step input→canonical-bytes→slug chain is specified entirely in prose with no end-to-end reference vector. Several points the recipe pins are ambiguous from prose alone — e.g. `const: -0` and `const: 1e21` rendering, code-point key-sort over surrogate-pair keys, RFC 8259 minimal-escape of control characters, the bytes emitted for an empty `properties` / `enum` / `required`, and canonical key-sort versus source-order emission. Two conforming implementers who both believe they followed the prose can produce different slugs for the same lowered fragment with no mechanical detector. Because the slug keys the `__inline_<slug>` dedup, the `__loom_respond_<slug>` tool name, the AJV validator cache, and the PIC `pi.registerTool` registration cache, the divergence is silent until two runtimes try to share a cached artefact, fixture, or replayable payload.

The commit's diff against `docs/spec_topics/schema-subset.md` shows the prior recipe bullet was edited in place to drop `default` from the numeric-literal enumeration (and to drop the trailing structural-keywords sentence), and a fresh `<!-- … -->` block was inserted immediately after that bullet to record the rationale for the deletions. The comment has been carried unchanged since, surviving the `81bf52a` directory move into `docs/spec_topics/`. The defect therefore entered as part of a corrective revision that intended to add rationale but used the wrong inline-comment syntax for a rendered spec.

## Solution approach

Add a normative reference-vector subsection to §"Canonical schema hash", anchored with the `<a id="subs-N">` sub-letter scheme the neighbouring SUBS clauses use, giving each vector end-to-end as input lowered fragment → exact canonical-form UTF-8 bytes → 16-hex slug, and covering the byte-level ambiguities named in Problem. Add forward-links to the new subsection from `#synthesised-names` and from `#schema-slug-collision-posture`.

## Solution constraints

- Out of scope: amending recipe steps 1–5 of §"Canonical schema hash" themselves — the recipe is the pinned on-disk/on-wire contract; the new vectors document the existing recipe exactly and must not redefine it.

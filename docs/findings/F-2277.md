## Problem

The `query-terminating` glossary entry (anchor `query-terminating` in `docs/spec_topics/glossary.md`) carries full normative per-ceiling routing law: which ceilings panic versus return `Err`, the unhandled-`Err`/discard conditions under which an `Err`-class breach reaches the *fail* arm, and ceiling #4's per-arm classification. The glossary charter declares each entry descriptive — a one-paragraph reminder with the canonical page winning — so this routing law is reserved for the canonical owner page. Duplicating it in the glossary erodes the descriptive/normative boundary and creates a stale-law hazard if either side later diverges.

## Issue introduction
**Verdict:** multi-commit-interaction
**Introducing commits:** `b4fbfac` (glossary charter), `8168ea8`, `9aeb318`, `94e292f`, `c45de81`, `22619b0`, `0e2eb05`, `51ae8f1`, `ace24a0`
**History:** The descriptive-only charter ("Each entry is descriptive — a one-paragraph reminder … the canonical page wins") was established at the glossary's inception in `b4fbfac` ("resolve 'no central glossary'"). The four entries then accreted normative content across a sequence of later "resolve …" commits, each of which placed a routing or predicate rule into the glossary entry rather than confining it to the canonical page: `8168ea8` ("T30 ceiling #4 model-driven row trichotomy carve-out") created the `query-terminating` and `in-loop` anchored entries already carrying ceiling-#4 routing detail, with `ace24a0` and `51ae8f1` extending them; `9aeb318` / `94e292f` / `c45de81` built up the `no-params` / `single-string` bypass trigger, warning-code, and ordering detail; `22619b0` and `0e2eb05` added the slot-accounting bookkeeping and CIO-4 cross-references. The scope violation is therefore the cumulative product of the inception-set charter interacting with many subsequent content-placement decisions, not a single defective commit.

## Solution approach

Narrow the `query-terminating` entry in `docs/spec_topics/glossary.md` to a definitional stub that names the concept and points at its canonical owner. Delete the per-ceiling routing-class resolution, the unhandled-`Err`/discard rules, and ceiling #4's per-arm classification, which are already owned by `ceilings-3-and-4.md` (`#ceiling-4-table`, the ceiling-#1 panic-uniqueness invariant, CIO-4). If any routing sentence proves absent from the canonical page, relocate it there before deleting it from the glossary. Retain the existing `See:` forward-links to `overview-and-orientation.md#hard-runtime-ceilings` and `ceilings-3-and-4.md#ceiling-4-table`.

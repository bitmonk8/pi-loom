## Problem

PIC-29 makes the tuple-to-arm mapping normative across all three arm-(b) tuples, but `(false, "shutting-down")` has no witnessed routing assertion. The arm-(b) acceptance section in `drain-state-contract.md` authored a witnessed routing fixture only for `(true, undefined)`, and item (j)'s quit/reload variant in `session-only-degraded-state.md` asserts only the post-handler tag value while explicitly forbidding a slash-arm assertion. A runtime mis-routing `(false, "shutting-down")` into a half-torn-down dispatch therefore passes the suite latent.

## Issue introduction

**Verdict:** later-commit-introduced (one introducing commit per vector)

**Introducing commits:**
- Drain-state arm-(b) vector: `8008b91` ("pi-loom spec-plan: fix F-1217 — add acceptance-criteria/fixture blocks for four Shard-08 MUSTs", 2026-06-25).
- Forced-respond no-transcript vector: `7f75947` ("pi-loom spec: resolve 'Typed-query forced-respond tool choice delivery channel'", 2026-06-05).

**History:** `git log -S` over `drain-state-contract.md` shows the *Acceptance criteria — `(drained: true, tag: undefined)` → arm (b)* section first appeared in `8008b91`; its item (b) enumerates all three arm-(b) tuples (`(false, "shutting-down")`, `(true, "shutting-down")`, `(true, undefined)`) but the commit authored a witnessed routing fixture only for `(true, undefined)`, leaving `(false, "shutting-down")` uncovered from the section's inception (the PIC-29 six-tuple decomposition itself predates it, having entered `drain-state-contract.md` at `f5e89f4`, 2026-06-04, when the contract was split into its own file). `git log -S 'attaches no turn to the driven session'` over `conversation-drive.md` and `git log -S 'not rendered as a transcript turn'` over `implementation-notes.md` both resolve to `7f75947`, which introduced the off-session delivery channel and its no-transcript property; no later commit added an acceptance criterion for it, so that vector is present-since-introduction.

## Solution approach

Add a fixture obligation asserting `LoomRegistry.readDrainState` returning `(false, "shutting-down")` routes through arm (b) (the `"loom /<name>: extension shutting down"` system note), witnessed at the `readDrainState` return mapping rather than via a live slash dispatch. Author it either as a new bullet in `drain-state-contract.md`'s arm-(b) acceptance section or as a `readDrainState`-return-mapping arm assertion extending item (j) in `session-only-degraded-state.md`. Add a forward-link to PIC-29's tuple-to-arm map so each of the three arm-(b) tuples is independently witnessed.

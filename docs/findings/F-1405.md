## Problem

`V9b-T`'s "model-reference-matcher production wiring" test asserts that the load pass injects loom's exact-match resolver into the model-reference-matcher injection seam `V6a` defines, and that `V6a`'s load-time `loom/load/model-unresolved` resolution binds that injected instance. Exercising that assertion requires both `V6a`'s seam interface and its resolution path to exist. But `V9b-T.Deps` is `V9a`, `V10a`, `V8e`, none of which pulls in `V6a` directly or transitively — only the paired impl leaf `V9b` declares `V6a`. Under the pickability predicate `V9b-T` therefore becomes pickable before `V6a` lands, so the matcher-wiring test cannot compile and cannot fail red for the intended reason, and `V9b-T`'s `Ships when` cannot be honestly satisfied.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** 44a96f5
**History:** `V9b-T` was created as a separate tests leaf in commit 44a96f5 (2026-06-16, "resolve V9b filename slug contradicts retitled scope"). Its first revision already carried the "model-reference-matcher production wiring" `Tests.` bullet referencing `V6a`'s injection seam and the `loom/load/model-unresolved` resolution path, while its `Deps` were `V9a`, `V10a`, `V8e` — `V6a` omitted from inception. At that same commit the paired impl leaf `V9b` carried `V6a` in its `Deps`, so the `-T`/impl asymmetry existed from the moment `V9b-T` came into being. Later commits that re-specified the matcher (2424003 "fix F-1311", e084853 "fix F-1359") rewrote the bullet's wording but never added `V6a` to `V9b-T.Deps`.

## Solution approach

Add `V6a` to `V9b-T`'s `Deps` field so the tests leaf sequences after `V6a`, mirroring the edge already present on the paired `V9b`. The edge ensures `V9b-T` becomes pickable only once `V6a`'s seam interface and `loom/load/model-unresolved` resolution path exist, so the matcher-wiring test compiles and fails red for the intended reason.

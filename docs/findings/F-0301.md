## Finding

`V9a`'s Adds declares the single physical home of `SHUTDOWN_AWAIT_CAP_MS = 2000` and explicitly delegates the literal-read assertion to `V18c` ("it lives alongside the probe constants here so the block has one explicit physical owner and `V18c`'s build-time literal-read assertion can read it"). `V18c`'s Adds accepts that delegation: it names "the cancellation-runtime constant `SHUTDOWN_AWAIT_CAP_MS` — … read from `V9a`'s single source-of-truth pinned-constants block by the build-time literal-read assertion per [`patch-skew-degradation.md` §`session_shutdown` sub-step 3]".

Neither `V18c`'s Tests nor `V18c-T`'s Tests enumerate that assertion. The enumerated Tests bullets in both leaves cover the SDK surface inventory (step 2(a)/2(b)), the `engines.node` three-way floor gate, the `peerDependencies` pin, the reason-snapshot brand-string gate, the provider seed-field `Api`-coverage gate, and the strict-capability probe gate — but no bullet asserts `SHUTDOWN_AWAIT_CAP_MS === 2000` against its spec source. `V9a`'s own Tests assert only that `FACTORY_PROBABLE_CAPABILITIES` enumerates exactly the five probe identifiers; they do not assert the cap value either.

The result is a gate that is described in prose but backed by no enumerated test. `V18c-T`'s Ships-when is "the tests above exist, compile, and fail red," and `V18c`'s Ships-when names the green test list — neither includes a literal-value assertion. An implementer working from the Tests bullets would not write one, so a future edit setting the constant to a wrong value (for example `5000`) would compile and ship green; the literal-read assertion the Adds promises never fires.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** `622d0d3` (2026-06-11, "pi-loom plan: resolve 'SHUTDOWN_AWAIT_CAP_MS physical home contradicts pinned-constants-block placement'")
**History:** Before `622d0d3`, `V18c`'s Adds did not mention `SHUTDOWN_AWAIT_CAP_MS`; the literal-read assertion claim did not exist, so there was no coverage obligation. `622d0d3` rewrote one Adds line to add "the cancellation-runtime constant `SHUTDOWN_AWAIT_CAP_MS` — … read … by the build-time literal-read assertion" without adding a corresponding Tests bullet to `V18c` or `V18c-T`. `git log -S 'SHUTDOWN_AWAIT_CAP_MS'` over both `V18c-version-bump-checklist.md` and `V18c-T-version-bump-checklist.md` returns only `622d0d3`, and that commit's stat shows it never touched the `V18c-T` tests file. The gap — an Adds-declared gate with no enumerated test — entered with that single commit.

## Solution Space

**Shape:** single

### Recommendation

Add a Tests bullet to `docs/plan_topics/V18c-T-version-bump-checklist.md` enumerating a build-time assertion that the loom-exported `SHUTDOWN_AWAIT_CAP_MS` equals the spec-pinned value `2000`, sourced from [`patch-skew-degradation.md` §`session_shutdown` sub-step 3](../spec_topics/pi-integration-contract/patch-skew-degradation.md) (where the constant is pinned as `SHUTDOWN_AWAIT_CAP_MS = 2000`). Mirror the bullet in `docs/plan_topics/V18c-version-bump-checklist.md`'s Tests, matching the form of the existing build-time literal-read gates already enumerated there. Add the new assertion to `V18c`'s Ships-when green-test list (the `npm test` clause) so the gate is part of `V18c`'s done condition.

Keep the constant's physical home and the delegation in `V9a`'s Adds unchanged: `V9a` delegates the literal-read assertion to `V18c`, so the test belongs in the `V18c` / `V18c-T` pair, not in `V9a`. The spec is read-only for this fix — `patch-skew-degradation.md` is the value source only; no spec edit is required.

## Consequence

**Severity:** correctness

`V18c`'s Adds promises a build-time literal-read assertion for `SHUTDOWN_AWAIT_CAP_MS`, but no enumerated test backs it, and both Ships-when conditions track only the enumerated tests. An implementer following the Tests/Ships-when fields produces a `V18c` that never checks the cap value, so a wrong literal (a drift from the spec's `2000`) ships green and the version-bump gate the Adds describes silently fails to fire.

## Problem

The `#session-only-reason-degraded-state` opening sentence is self-contradictory: its parenthetical states unconditionally that Pi swaps the user session without tearing the extension runtime down, while the main clause says session-only reasons "do not always tear down the *extension* runtime" (conditional). A reader cannot tell whether a session-only reason ever tears the extension runtime down. The conditional reading also admits a case the watcher-stays-live degraded-state contract on this page contradicts.

## Solution approach

Rewrite the `#session-only-reason-degraded-state` opening clause so the extension-runtime-survival statement is unconditional, aligned with its own parenthetical and the teardown-vs-session-swap reason partition grounded in `host-prerequisites.md#degraded-state-host-prerequisites` (session-only reasons leave the extension runtime live). Since the defect is the wording, replace "does not always tear down the *extension* runtime — only the user session" with unconditional phrasing such as "does not tear down the *extension* runtime — only the user session".

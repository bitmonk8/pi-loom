## Problem

The `session_shutdown` teardown handler pins an at-most-once-per-extension-instance-lifetime bound on `loom/host/session-shutdown-runtime-degraded`, then accepts two paths that breach it — the all-three-throw corner case (`#idempotent-handler-clause` in `drain-state-contract.md`) and the read-failure fallback (PIC-31, `#read-failure-fallback`). At each accepting site the breach is discharged with the same present-tense assertion that "operator deduplication on `(code, details.event.reason)` collapses the resulting duplicate rows." Operator tooling is outside loom's control surface; on a host with no dedup tooling the duplicate rows persist and reach the operator, so the acceptance rests on a downstream behaviour the spec cannot enforce. The phrasing overstates the bound as preserved end-to-end when it is in fact a best-effort loom-side target these corner cases knowingly breach.

rationale was written — the same paragraphs introduced in `52b9101` describe
re-emission on "follow-up session-only `session_shutdown` events," which can
differ in `event.reason` from the original; the dedup key was never reconciled
with that reason-variance.

## Solution approach

Rewrite the breach-acceptance rationale at the all-three-throw paragraph and the PIC-31 *Read-failure fallback* clause in `drain-state-contract.md`, and the *Idempotent re-entry and the all-three-throw corner case* paragraph plus acceptance (m) in `session-only-degraded-state.md`, so conformance rests on the loom-side emission guarantee — the additional `loom/host/session-shutdown-runtime-degraded` rows on these enumerated paths are conformant regardless of any downstream collapse. Demote the `(code, details.event.reason)` dedup-key mention to a structural note about how the rows are shaped to be deduplicable, consistent with the existing *Co-emission rules* "bucket them separately by construction" framing, and drop the present-tense assertion that dedup collapses them.

## Solution constraints

- Must not weaken the existing MUST-NOT-suppress prohibitions (PIC-30, PIC-31, and the all-three-throw no-additional-seam pin) — this is a rationale reframe, not a behaviour change.
- Out of scope: the chokidar version pinning owned by F-2140.

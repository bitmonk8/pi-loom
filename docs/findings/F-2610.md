## Problem

In `query/query-failure-and-repair.md`, `## Options surface` is a standalone H2 whose entire body is a single non-normative seam blockquote describing the runtime-internal options record as an open struct so deferred per-call-timeout and per-query `model`/`tools`/`system` override extensions can land additively. The heading reads as a present feature of the query primitive, but the section delivers no current loom 1.0 capability — it is purely a forward-compatibility note for deferred work already enumerated in `future-considerations/surface-extensions.md`. The same file treats its other seam (the `ContextOverflowError` pre-flight-token-count seam, anchor `v1-seam-pre-flight-token-nullability`) as a `> **loom 1.0 seam — …**` blockquote nested inside a normative section, so the standalone-heading framing is inconsistent and overstates the section's normative weight.

## Issue introduction

**Verdict:** single-commit introduction
**Introducing commit:** `09c42c0` (2026-05-05) — "pi-loom spec: resolve \"Surface-extension Seam annotations carry V1 obligations from non-goals page\""
**History:** The `## Options surface` H2 was authored net-new into `spec_topics/query.md` by `09c42c0` while resolving an earlier review finding about surface-extension seam annotations; that commit created parallel `## Options surface` standalone sections across the query, tool-calls, and invocation pages, each body being a single open-struct seam paragraph. The paragraph was later reshaped into the `> **loom 1.0 seam — per-call timeout / per-query overrides.**` blockquote form (`52219c4`, 2026-05-06) and relocated into `query/query-failure-and-repair.md` by the topic-file split (`f5e89f4`, 2026-06-04). The standalone-heading-over-a-pure-deferral structure persisted unchanged across every reshaping — the defect was present from the section's first authoring, not introduced by the later reshapes.

## Solution approach

Demote the standalone `## Options surface` heading and fold its seam note into the existing `## Failure modes` section as a `> **loom 1.0 seam — …**` blockquote, matching the sibling pre-flight-token-count seam in the same file. Preserve a stable anchor at the seam's new location so the `surface-extensions.md` cross-references that target the Query Options surface retain a resolvable target. Keep the open-struct / tolerate-unknown-fields contract and the forward-reference to Future Considerations intact.

## Solution constraints

- Out of scope: the `surface-extensions.md` cross-reference link-repair pass (the "Query — Options surface" links point at `../query.md` with no fragment; the fold does not require touching them).

## Finding

The *Depth Enforcement* section of `docs/spec_topics/schema-subset.md` closes its per-boundary surfaces table with: "The full per-boundary routing-class table for ceiling #4 … **lives in** [`spec.md` — Hard ceilings](./overview-and-orientation.md#hard-runtime-ceilings); this section's table mirrors it…". That pointer resolves to the `#hard-runtime-ceilings` orientation aggregator in `overview-and-orientation.md`, which carries only the four one-line ceiling summaries. Ceiling #4's one-liner there does **not** contain the full table — it instead forwards onward ("Per-boundary destination/surface table: [Hard Runtime Ceilings — Per-boundary destination/surface table (ceiling #4)](./hard-ceilings/ceilings-3-and-4.md#ceiling-4-table)").

The canonical full table actually lives at `docs/spec_topics/hard-ceilings/ceilings-3-and-4.md#ceiling-4-table`, under the heading "Per-boundary destination/surface table (ceiling #4)". So a reader who follows `schema-subset.md`'s pointer expecting the authoritative table lands on an aggregator one-liner and must take a second, undeclared hop to reach it. The prose claim "lives in [`spec.md` — Hard ceilings]" is simply false for the full table: the aggregator is not its home.

The `spec.md` label itself is consistent with the corpus convention (GOV-23 / the split-page denotation where `spec.md` stands for the published split pages); the defect is purely that the URL+anchor names the aggregator rather than the table's true owner page. The fix is to repoint the link target, exactly as the aggregator's own ceiling-#4 entry already does.

## Issue introduction
**Verdict:** single-commit
**Introducing commits:** `f5e89f4` ("docs: enforce 100KB hard / 30KB recommended size cap on spec set", 2026-06-04)
**History:** The pointer was introduced in `1750848` ("Ceiling #4 routing class collapses four boundaries whose destinations differ", 2026-05-07) as `[`spec.md` — Hard ceilings](../spec.md#hard-runtime-ceilings)`, and was **correct then**: at that commit the full per-boundary table was inline in `spec.md` directly under the `#hard-runtime-ceilings` anchor (confirmed via `git show 1750848:spec.md`). Commit `f5e89f4` split the oversized spec, moving the `#hard-runtime-ceilings` content into the new `overview-and-orientation.md` aggregator (one-liners only) and relocating the full table into the new `hard-ceilings/ceilings-3-and-4.md#ceiling-4-table`. In that same commit, `schema-subset.md`'s pointer was mechanically rewritten from `../spec.md#hard-runtime-ceilings` to `../spec/overview-and-orientation.md#hard-runtime-ceilings` (later normalised to `./overview-and-orientation.md#…`), preserving the anchor while the table it claimed to reach was simultaneously moved elsewhere — so the repoint carried the now-stale "lives in" claim forward. `spec.md`'s own ceiling-#4 aggregator entry was correctly repointed to `…/ceilings-3-and-4.md#ceiling-4-table` in the same commit; `schema-subset.md`'s pointer was not. (`git log --follow -G` over `schema-subset.md`; `git show f5e89f4^:…` vs `git show f5e89f4:…` on the pointer line.)

## Solution Space

**Shape:** single

### Recommendation

In `schema-subset.md`'s *Depth Enforcement* trailing paragraph, change the "lives in" link target from `./overview-and-orientation.md#hard-runtime-ceilings` to `./hard-ceilings/ceilings-3-and-4.md#ceiling-4-table` — the actual owner of the full per-boundary routing-class table. Keep the existing `[`spec.md` — Hard ceilings]` link **text** convention if desired, or update the visible text to "Hard Runtime Ceilings — Per-boundary destination/surface table (ceiling #4)" to match how `overview-and-orientation.md` and `ceilings-3-and-4.md` already name that target; the binding requirement is that the URL resolve to `#ceiling-4-table`. Verify the relative path resolves from `docs/spec_topics/schema-subset.md` (i.e. `./hard-ceilings/ceilings-3-and-4.md#ceiling-4-table`).

## Consequence

**Severity:** advisory

A reader or implementer following `schema-subset.md`'s explicit "the full table lives in …" pointer lands on the orientation aggregator's one-liner instead of the table, and must discover the second hop to `ceilings-3-and-4.md#ceiling-4-table` unaided. No runtime behaviour diverges (the two tables presently agree); the cost is a misdirected citation and a mis-stated table home.

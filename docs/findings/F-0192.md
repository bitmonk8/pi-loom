## Problem

The `§Leaf format` **Spec** field bullet in `docs/plan_topics/conventions.md` states a MUST — "The **Spec** field MUST be closed under normative cross-link … and excludes narrative cross-links per [`governance.md` GOV-3]" — but uses *normative cross-link* and *narrative cross-link* without defining either. The GOV-3 reference does not close the gap: GOV-3 defines a narrative *page* (a page whose REQ-ID-prefix-table row carries the byte-exact `(no IDs — narrative)` cell), not a cross-link, so a leaf author must infer unaided that a narrative cross-link is one whose target page is narrative. The same file already states the precise byte-exact narrative-page test in `§REQ-ID discipline` and the `§Tests` bullet, but the **Spec** bullet does not link its terms to it, leaving one operational concept spelled out under one naming and undefined under another. `docs/plan_topics/leaf-template.md`'s `**Spec.**` note has the mirror gap: it defines the normative side but never names or defines the narrative exclusion.

## Issue introduction

**Verdict:** single-commit (regression by condensation)
**Introducing commit:** `7fb7939` — "pi-loom spec: resolve \"Authoring conventions paragraph misplaced in informative introduction\"" (2026-05-06, Thomas Andersen)
**History:** From the scaffold reset (`657ee76`) onward, the §Leaf format **Spec** bullet carried an explicit inline definition: "Narrative cross-links — those whose row in the [REQ-ID prefix table] carries the `(no IDs — narrative)` cell, which per `governance.md` GOV-3 is the single source of truth for narrative-page status — do not trigger the closure obligation." Commit `7fb7939` rewrote the bullet, condensing that sentence to "excludes narrative cross-links per GOV-3" and dropping the inline definition; the undefined-term reading dates from that condensation. A later commit (`8ac3d60`, 2026-05-27) removed the now-retired GOV-11 wrapper the same bullet had acquired but did not restore the definition. `leaf-template.md`'s narrative-side omission predates this, present since `657ee76`.

## Solution approach

Restore an inline definition of *normative cross-link* and *narrative cross-link* in the `§Leaf format` **Spec** bullet of `docs/plan_topics/conventions.md`, reusing the byte-exact narrative-page test the same file already states in `§REQ-ID discipline` / `§Tests` rather than coining a divergent one. Tie the definition to the same GOV-3 anchor the `§Tests` bullet cites so the two share one source. Add the matching narrative-exclusion clause to `docs/plan_topics/leaf-template.md`'s `**Spec.**` note, which currently defines only the normative side.

## Solution constraints

- Out of scope: the `spec_topics/governance/**` pages — the `(no IDs — narrative)` mechanism already lives in GOV-3 and the spec corpus is read-only for this fix.

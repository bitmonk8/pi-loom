## Finding

The single BNDR-5 concept — rendering a `number` value in shortest, scientific-notation-free decimal form — is named three different ways within the `V2d` leaf and its paired tests leaf. `V2d`'s **Adds** field calls it "shortest round-tripping fixed-point"; both `V2d` and `V2d-T`'s **Tests** `BNDR-5` bullets call it "shortest round-trip fixed-point" (dropping the `-ping` and the round-trip-as-adjective sense); and `V2d`'s **Ships when** line calls it "shortest-fixed-point renderings", omitting the round-trip qualifier entirely.

The spec's authoritative term, [`BNDR-5`](../spec_topics/binder/defaulting-system-note-echo.md#bndr-5), is "the shortest round-tripping decimal that reparses to the same IEEE-754 double". The round-trip property (reparses to the same double) is the load-bearing half of the contract; the "shortest-fixed-point" phrasing in **Ships when** silently drops it, so the three field phrasings do not even agree on which property the acceptance gate is checking.

This is wording only — both leaves cite the spec REQ-ID, so an implementer reading `BNDR-5` gets the full contract regardless. The variance is a readability and consistency defect, not a behavioural one.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** c6a664e (2026-06-10) "pi-loom plan: build/update plan for spec.md + review"
**History:** All three phrasings were present in the first commit that authored these leaves. `git show c6a664e:docs/plan_topics/V2d-number-rendering.md` already carries "shortest round-tripping fixed-point" in **Adds**, "shortest round-trip fixed-point" in the **Tests** `BNDR-5` bullet, and "shortest-fixed-point renderings" in **Ships when**; `V2d-T`'s **Tests** bullet carried "shortest round-trip fixed-point" at the same commit. Subsequent commits (31f4a86, cb8737a) expanded the threshold/vector wording and renumbered downstream leaf references but never reconciled the three names. The variance is original drafting noise, not a regression introduced by a later edit.

## Solution Space

**Shape:** single

### Recommendation

Use the spec-aligned term "shortest round-tripping fixed-point" (which preserves the load-bearing round-trip property from `BNDR-5`) consistently across both leaves:

- In `docs/plan_topics/V2d-number-rendering.md`, the **Tests** `BNDR-5` bullet currently reads "number echo is shortest round-trip fixed-point"; change "shortest round-trip fixed-point" to "shortest round-tripping fixed-point" to match the **Adds** field.
- In `docs/plan_topics/V2d-number-rendering.md`, the **Ships when** line currently reads "the BNDR-5 shortest-fixed-point renderings"; change "shortest-fixed-point renderings" to "shortest round-tripping fixed-point renderings" so the gate names the same property the **Adds**/**Tests** fields do.
- In `docs/plan_topics/V2d-T-number-rendering.md`, the **Tests** `BNDR-5` bullet currently reads "number echo is shortest round-trip fixed-point"; apply the same change to "shortest round-tripping fixed-point".

This is a pure wording reconciliation; the byte-output contract (no scientific notation, both `String(n)` switch ranges expanded, fractional-digit and `-0 → 0` pins) is unchanged and continues to be owned by `BNDR-5` in the spec, which is read-only for this fix.

## Consequence

**Severity:** cosmetic

A reader cross-referencing `V2d`'s **Ships when** against its **Adds** and **Tests** sees three names for one concept and may wonder whether "shortest-fixed-point" (no round-trip) is a distinct, weaker acceptance bar than the **Tests** bullet's "shortest round-trip fixed-point". No implementer is blocked — the cited `BNDR-5` spec rule is the single source of truth — but the internal disagreement adds friction and a momentary ambiguity about what the gate asserts.

## Finding

`binder/binder-bypass-and-envelope.md` is 152 lines. Lines 3–80 cover the two topics the filename advertises (`## Binder bypass`, `## Binder envelope`). Lines 81–152 (~47%) are a distinct normative surface: `### Binder system prompt` and `#### System-prompt structure (normative)`, with eight numbered structural items and the type/default/parameter-line rendering tables. None of that is about bypass or the envelope schema; it is the contract for the system prompt the runtime constructs when the binder *does* run.

The other three binder sub-pages each carry one cohesive topic named in their filename — this is the only multi-topic page, and its name conceals the third topic. A reader needing the system-prompt obligations (e.g. `V11d`, coverage-matrix row `cka-45`) has no filename hint; cross-references reach it only via the deep anchor `#system-prompt-structure-normative`. Discoverability defect with no behavioural consequence.

**Verdict:** present-since-inception (of the current file)
**Introducing commits:** `f5e89f4` ("docs: enforce 100KB hard / 30KB recommended size cap on spec set", 2026-06-04)
**History:** The current file was added in `f5e89f4` as part of the size-cap split that broke monolithic `binder.md` into four child pages; the split's filename grouping placed bypass, envelope, AND the system-prompt sections together in one child, while the three others carry one topic each. `git log -L 81,152:spec_topics/binder/binder-bypass-and-envelope.md` confirms the only later touch is `2fb5783` (in-place clarification). Inception defect of the split's grouping decision, not later drift.

## Solution Space

**Shape:** multiple

### Option A — Rename the file to reflect all three topics

`git mv` to `binder-bypass-envelope-and-system-prompt.md`, preserve all anchors, run the map-driven rewrite over the 41 inbound citations. Pros: smallest edit, anchors stable. Cons: perpetuates the multi-topic pattern; long compound name.

### Option B — Split the system-prompt sections into a dedicated `binder-system-prompt.md`

Move lines 81–152 into new `binder/binder-system-prompt.md` (re-promote H3→H2, H4→H3), preserve `<a id>` anchors, repoint inbound citations that target moved anchors (V11b/V11b-T, V11d/V11d-T, `cka-45`, the binder-model-and-context cross-refs) to the new file; leave bypass/envelope citations on the old file. Pros: restores one-topic-per-file; V11d's surface gets a named file. Cons: larger mechanical edit (per-anchor repoint decision); one more page.

### Recommendation

Take Option B. It aligns the file with the one-topic-per-file pattern the three sibling binder pages follow and gives the system-prompt contract a filename that names it. Repoint cost is bounded (six plan leaves, one coverage-matrix row, two cross-references); `check_links.py` catches misses. Edge cases: re-promote moved headings; preserve every `<a id>` (chiefly `system-prompt-structure-normative`); do not rewrite citations to anchors that remain on the bypass/envelope page.

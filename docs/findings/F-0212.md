## Finding

`H7a` asserts the committed golden transcript on every later run (turn order, count, and per-turn content), so the gate's value depends entirely on the golden being correct at first commit. The leaf carefully establishes derivation provenance for most of the golden: each golden turn and each `loom/...` code that maps onto a behaviour already gated by a Deps leaf is sourced from that leaf's already-established expected output. But the one part no per-leaf gate covers — the *composed turn ordering* across slices — is established only by "human-reviewed against `docs/spec.md` and the cited spec topics at first commit."

That human-review step has no recorded criterion. Nothing captures which spec section each composed-ordering decision was reviewed against, so the review is not reproducible: a later maintainer (or an auditor at the `H6a` release gate) cannot re-derive or check the first-commit judgement, and a mis-derived composed ordering silently enshrines wrong behaviour as the regression target — the assertion thereafter merely confirms the golden matches itself.

The derivation prose makes the review an obligation but stops short of making it auditable. Recording, per composed-ordering turn the per-leaf provenance does not already cover, the spec section(s) it was reviewed against (committed alongside the fixture and golden) closes the gap without changing what the gate asserts.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/H7a-integration-acceptance.md`, amend the golden-derivation sentence in **Adds** so the first-commit composed-ordering review produces a recorded, checked-in provenance artefact. Where the prose currently reads "...the composed turn ordering no single per-leaf gate covers is human-reviewed against [`docs/spec.md`](../spec.md) and the cited spec topics at first commit", extend it to require that the review records, for each composed-ordering turn the per-leaf provenance does not already account for, the spec section(s) (`docs/spec.md` / cited spec topic) it was reviewed against, committed alongside the fixture `.loom` and the golden transcript.

This is the same per-Deps-slice provenance discipline the leaf already applies to per-turn content and the permitted-code list, extended to the composed-ordering review so the first-commit judgement is reproducible and auditable by the `H6a` release gate. The review reference points at the spec; the spec itself is not edited (read-only for this fix), and the change is confined to `H7a` — `H4a`'s smoke criterion and `H6a`'s release-gate audit consume the goldens unchanged.

Edge case for the implementer: a composed-ordering turn whose ordering is fully implied by an existing per-leaf gate's expected output needs no separate spec citation (the per-leaf provenance already covers it); the recorded criterion is required only for ordering decisions the per-leaf gates do not constrain.

## Consequence

**Severity:** advisory

The per-turn content already carries per-leaf provenance; only the cross-slice composed-ordering review lacks a recorded criterion, so the gate still functions and an implementer can still produce a working golden. The exposure is reproducibility/auditability: if the unrecorded first-commit ordering review is wrong, the wrong ordering becomes the permanent regression target and the assertion can never catch it, and the `H6a` release-gate auditor has nothing to check the first-commit judgement against.

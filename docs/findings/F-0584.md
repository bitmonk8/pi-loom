## Problem

`docs/plan.md` §Vertical slices uses a `> **Interleave note.**` blockquote to narrate dep-order inversions (the V9 and V11 sections each carry one), but several structurally-equivalent backward edges encoded only in leaf `Deps` fields carry no such note: V10 (the canonical order is `V10c → V10a → V10b → V10d`, not the listed letter order), `V11e → V11h`, `V12a`/`V12a-T → V13c`, the V14 series → `V15a`, and `V15b → V16a`. The slice preamble globally states that numbering is not topological and `Deps` are authoritative, so execution does not break. The defect is that the interleave-note set — the human-facing index of inversions — is selectively incomplete, so a reader who treats a note as the signal of an inversion will miss these edges until opening each leaf's `Deps`.

## Issue introduction

**Verdict:** single-commit (the interleave-note remedy was established for V9/V11 only; the backward edges pre-existed and were left undocumented)
**Introducing commits:** `b9b427d` ("resolve 'slice-ordering narrative overpromises DAG order'", 2026-06-10); `c6a664e` (plan build, 2026-06-10); `e965634` ("resolve 'V10c settings keys read but application not asserted'", 2026-06-11)
**History:** The V12→V13 and V14/V15/V16 backward edges, and the V11e→V11h edge, entered with the plan build `c6a664e` (`git show b9b427d:` confirms `V14a`'s Deps already named `V15a`, `V15b`'s named `V16a`, and `V12a`'s named `V13c` at `b9b427d`). The interleave-note infrastructure was added in `b9b427d`, which strengthened the §Vertical-slices preamble and added `> **Interleave note.**` blockquotes — but only to the V9 and V11 sections (`git show b9b427d -- docs/plan.md` shows the two added notes and no V10/V12/V14/V15/V16 edit). The `V10c` edge on `V10a`/`V10b` was added later in `e965634`, creating the V10 inversion without a corresponding note. The parity gaps have stood unchanged since (the later `e2b7e81` only added `V15c` to `V15b`'s Deps, not the `V16a` edge).

## Solution approach

Add `> **Interleave note.**` blockquotes mirroring the existing V9/V11 notes to the V10, V12, V14, and V15 slice sub-sections of `docs/plan.md` §Vertical slices, and surface the `V11e → V11h` intra-slice inversion in the V11 section. Each note names the backward edge(s) and closes with the standard "Sequence by **Deps**, not slice number." pointer: V10 builds `V10c → V10a → V10b → V10d`; `V11e` depends on `V11h`; `V12a`/`V12a-T` depend on `V13c`; the V14 series depends on `V15a` (via `V14a`); `V15b` depends on `V16a`.

## Solution constraints

- Leave every leaf `Deps` field unchanged — the edges are correct; this is a navigational-index gap only.
- Do not renumber the leaves: slice/leaf IDs are referenced across `Deps`, the coverage matrix, and cross-references, so renumbering churns stable IDs.

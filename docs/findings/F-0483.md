## Finding

`H5b-warn-only-canary.md` appends an italicised, multi-sentence parenthetical block after its `Deps.` line. That block is a decision-log: it explains why the dependency set is "the complete coverage-producing set," walks through which leaf categories (`H5a`, `H5e`, `H5f`, `H1a`, the MVP/vertical implementation leaves) are in the set and why, restates that keeping the set "transitively complete … is the standing plan-maintenance obligation stated in *REQ-ID discipline*," and explains that `H6a` "inherits this set by depending on this leaf rather than restating it."

Every one of those claims is already owned, authoritatively, by `conventions.md` §REQ-ID discipline — specifically its "**Transitive-completeness plan-maintenance**" subsection, which defines the coverage-producing trigger, mandates that any qualifying new leaf be added to `H5b`'s `Deps.`, and states that `H6a` inherits the set transitively through `H5b`. The `H5b` parenthetical adds no normative content beyond what that subsection already governs; the normative content of the `Deps.` field is the leaf-ID list itself. The parenthetical also reproduces the prospective "can introduce" phrasing that the authoritative rule has since recast, so it is a second copy that can drift out of step with its owner.

## Solution Space

**Shape:** single

### Recommendation

Delete the italicised parenthetical block that follows `H5b`'s `Deps.` line in `docs/plan_topics/H5b-warn-only-canary.md`. The `Deps.` leaf-ID list is the normative content and remains intact; the rationale for the set's composition and the transitive-completeness maintenance obligation are owned by `conventions.md` §REQ-ID discipline ("Transitive-completeness plan-maintenance"), which `H5b`'s existing `Convention.` field already points to. No cross-reference into the deleted block exists elsewhere, so no other leaf needs updating.

## Consequence

**Severity:** cosmetic

No implementer is misled and execution is unaffected — the `Deps.` list stands on its own as the normative content. The defect is duplication: the parenthetical is a second, non-authoritative copy of the conventions.md transitive-completeness rule, so an edit to that rule (or the already-flagged "can introduce" vs "carries" wording fix) must be mirrored here or the two drift apart.

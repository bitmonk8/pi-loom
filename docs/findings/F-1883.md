## Problem

The *Sequential by default* rule in `docs/plan_topics/conventions.md` now governs two distinct same-line exemption-comment forms: `// allow:` for the four banned Promise combinators, and `// allow-sync:` for the blocking-runtime ban (callees whose name ends in the literal suffix `Sync`). The test-code carve-out, however, is phrased in the singular — "Test code (`**/*.test.ts`) is unrestricted but MUST NOT carry the exemption comment" — and does not say whether the prohibition covers `// allow:` only, `// allow-sync:` only, or both. Sitting between the two paragraphs, the sentence makes the narrower (`// allow:`-only) reading plausible, so the `H2a` implementer cannot tell which forms its lint must reject in `**/*.test.ts`. Only the both-forms reading satisfies the rule's stated grep-cleanliness rationale.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** c33d309 (2026-06-17, "pi-loom plan: resolve 'Blocking-runtime ban concedes enforcement to manual review'"), cf47e6f (2026-06-18, "pi-loom plan: resolve 'H2a *Sync ban excludes V8* seam-adapter modules…'") — interacting with 861e935 (2026-05-05, "pi-loom plan: resolve 'Promise.all/race exemption rule has no operational definition'")
**History:** `git log -S` over `docs/plan_topics/conventions.md` shows the singular sentence "Test code (`**/*.test.ts`) is unrestricted but MUST NOT carry the exemption comment" was authored at 861e935 (2026-05-05), when the rule defined exactly one exemption-comment form (`// allow:`); the singular was correct then (`git show 861e935` contains zero `allow-sync` occurrences). c70ebce (2026-06-12) subsumed the "Never block the async runtime" directive into the rule as manual-review-only with no comment form. The second exemption-comment form `// allow-sync:` first entered `conventions.md` at c33d309 (2026-06-17) and was refined at cf47e6f (2026-06-18) — `git log -S "allow-sync"` returns only those two commits. Neither reconciled the pre-existing singular sentence, so the ambiguity is the latent 861e935 wording made wrong by the later second-form additions.

## Solution approach

Rewrite the test-code carve-out sentence in the *Sequential by default* rule so it names both exemption-comment forms — `// allow:` and `// allow-sync:` — as forbidden in `**/*.test.ts`, replacing the singular "the exemption comment". Retain the grep-cleanliness rationale parenthetical.

## Solution constraints

- Out of scope: `H2a`'s Ships-when gate and Tests bullets — they remain accurate and need no change to resolve this ambiguity.

## Problem

The single normative sentence pinning `ValidationError.attempts = 0` across the three concrete emission sites under `respond_repair.methodology: none` — initial AJV failure, depth-walk failure, and the `#err-17` forced-respond branch — lives in the `none` bullet of the `respond_repair` field contract on `frontmatter/frontmatter-fields-b-and-templates.md`, not on the schema's owning page. On `errors-and-results/queryerror-variants.md` the `ValidationError.attempts` field carries only a comment, and the page pins `attempts: 0` only in the narrow ERR-17 wrong-tool footnote, which neither covers the initial-AJV and depth-walk paths nor generalises. An author reading the `ValidationError` schema to learn what `attempts` holds for the most common configuration must cross-jump to a frontmatter field page on a different topic. Field-shape contracts belong on the schema's owning page.

## Issue introduction

**Verdict:** single-commit introduction

**Introducing commit:** `0b05c10` (2026-06-07) — *pi-loom spec: resolve "respond_repair methodology:none attempts behaviour and ValidationError.attempts"*

**History:** The `none` bullet at lines 57–60 of `frontmatter/frontmatter-fields-b-and-templates.md` was created in the file's initial split (`f5e89f4`, 2026-06-04) with only the short form *"Equivalent to `attempts: 0`. No template applies (no turn is emitted)."*, which did not address `ValidationError.attempts`. The cross-page rule pinning `attempts = 0` across initial AJV, depth-walk, and ERR-17 was added in a single commit (`0b05c10`) that resolved an earlier review finding (T087) about `methodology: none + attempts: N>0` and `ValidationError.attempts` being unspecified. That commit's resolution note (visible in the same commit's `docs/spec-review.md` removal) records the deliberate choice to make the `none` bullet self-contained rather than splitting the contract. `git log -S 'attempts: 0' -- docs/spec_topics/errors-and-results/queryerror-variants.md` returns only `34517a0` (2026-06-05), which added the ERR-17 footnote — a strictly narrower statement than the matrix that later landed on the frontmatter page. The placement defect is intrinsic to commit `0b05c10`'s choice; no later commit has touched this line.

## Solution approach

Move the canonical `attempts = 0` statement from the `respond_repair` `none` bullet on `frontmatter/frontmatter-fields-b-and-templates.md` to `errors-and-results/queryerror-variants.md`, co-located with the `ValidationError.attempts` field in `### Query-time variants`. The relocated statement names all three `schema_validation`-arm emission sites that surface `ValidationError` under `methodology: none` — initial AJV failure, depth-walk failure, and the `#err-17` forced-respond branch. Rewrite the frontmatter `none` bullet's matrix sentence as a forward-link to the relocated rule.

## Solution constraints

- Do not delete the ERR-17 forced-respond `attempts: 0` statement on `errors-and-results/queryerror-variants.md`; GOV-28 anchor stability binds the `#err-17` anchor and its content.
- Out of scope: the `empty_template` arm of `ValidationError`; the relocated statement covers the `schema_validation` arm only.

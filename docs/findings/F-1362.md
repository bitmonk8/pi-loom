## Problem

The `cka-12` (QRY) coverage-matrix row's facet leaves ‚Äî `V13a`, `V13b`, `V13c`, `V13d`, `V13g`, and `V13h` ‚Äî carry no `Tests` bullet that cites both the `cka-12` token and that leaf's own leaf-ID inline. `V13c`/`V13d`/`V13h` cite "(QRY code-keyed area)" prose, `V13a`/`V13g` cite only `loom/parse/*` diagnostic codes, and `V13b`'s query bullets carry no token. The `H5f` per-facet citing-test gate requires each facet leaf of a multi-leaf coverage-matrix row to carry such a facet-naming citing test, so none of these leaves conforms.

## Issue introduction

**Verdict:** introduced-by-later-edit
**Introducing commit:** `26a8794` (Thu Jun 18 2026, Thomas Andersen) ‚Äî "pi-loom plan: resolve 'split coverage-matrix rows can ship a facet untested'"
**History:** The facet leaves' `Tests` bullets have carried prose area labels ("(QRY code-keyed area)", "(IMPL area)", "cancellation.md ‚Äî *Race semantics‚Ä¶*") since the initial plan build (`c6a664e`); the `cka-<n>` tokens were minted later (`7259777`, "code-keyed obligation areas lack a canonical citable token"). Neither created a defect, because no rule yet required a facet `Tests` bullet to cite the `cka-<n>` token plus its own leaf-ID inline. Commit `26a8794` introduced **both** the `coverage-matrix.md` *Multi-leaf-row per-facet citing tests* convention and the `H5f` gate arm that enforce exactly that inline citation; its diff touched only `H5f-per-facet-citing-test-gate.md`, `conventions.md`, `coverage-matrix.md`, `H5a`, `H5b`, and `H6a` ‚Äî it retrofitted none of the `cka-12` / `cka-33` / `cka-44` facet leaves' `Tests` bullets. The facet bullets became non-conforming at that commit.

## Solution approach

In each of `V13a`, `V13b`, `V13c`, `V13d`, `V13g`, and `V13h` (and their paired `-T` leaves), make a QRY `Tests` bullet cite both the `cka-12` token and that leaf's own leaf-ID inline, per the `coverage-matrix.md` *Multi-leaf-row per-facet citing tests* convention. For `V13b`, edit only its query-area (QRY) bullets.

## Problem

The `query/query-forms.md` § "Dedent and newline-trim — normative behaviour" opening sentence names two normative authorities at once: CPython 3.x `textwrap.dedent` and the eight-vector table that "illustrates" it. The two cannot be equivalent — the table covers only ASCII space/tab indentation with LF line endings, while `textwrap.dedent` has externally observable behaviour on inputs the section never addresses (CRLF and bare-CR line endings, and non-ASCII leading whitespace such as U+00A0 or U+3000). An implementer who pins behaviour to the table and one who pins to a `textwrap.dedent` port produce divergent rendered prompt text on those inputs without either contradicting the spec, and because rendered text is what the model sees this is a wire-observable conformance gap.

## Issue introduction

**Verdict:** introduced-by-commit
**Introducing commits:** `66e419d` (2026-05-04 — "pi-loom spec: resolve \"Dedent normalisation has no normative test vectors\"")
**History:** The prior wording (commit `53f5831`, 2026-05-03, "spec: rewrite query primitive, system prompts, and schema syntax") named a single authority — "(Python's `textwrap.dedent` algorithm)" — as an offhand parenthetical and supplied no table. Commit `66e419d` added the table, the three behaviour bullets, and the dual-authority sentence ("The normative reference is the behaviour of CPython 3.x `textwrap.dedent` *as illustrated by the table below*; an implementer who cannot read the CPython source can still pass the conformance tests from the spec alone") in a single hunk against `spec_topics/query.md`. That commit was a direct response to a prior review finding that the dedent rule lacked normative test vectors; in fixing the missing-vectors problem it introduced the dual-authority problem this finding now names. The file split (`f5e89f4`, 2026-06-04) and the addition of vector 8 (`d083d70`, 2026-06-06) carried the dual-authority sentence forward verbatim without revisiting it.

## Solution approach

Rewrite the section's opening sentence so the spec text — the behaviour bullets, the vector table, and the obligations below — is the sole normative authority, and demote the CPython `textwrap.dedent` reference to a non-normative pointer to one working implementation of the same algorithm. Clarify the behaviour bullets to close the currently-undefined inputs: pin newline-trim and dedent to operate on LF (`\n`)-terminated lines (a literal `\r` byte is ordinary content that neither participates in line splitting nor satisfies the whitespace-only-line predicate), and narrow the leading-whitespace alphabet to ASCII space (U+0020) and tab (U+0009), with any other code point treated as ordinary content for both the common-prefix walk and the whitespace-only-line predicate. Add normative table vectors with matching commentary exercising the LF-only line-ending rule and a non-ASCII-whitespace leading character so the new obligations have falsifiable fixtures.

## Solution constraints

- When pinning the leading-whitespace alphabet, forward-link the canonical ASCII-whitespace definition at `defaulting-system-note-echo.md#system-note-rendering` rule 1 rather than restating it, keeping one whitespace definition across the spec.

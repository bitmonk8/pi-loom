## Problem

The paired leaves `V2d` (`docs/plan_topics/V2d-number-rendering.md`) and its tests leaf `V2d-T` (`docs/plan_topics/V2d-T-number-rendering.md`) state their `BNDR-5` number-rendering coverage differently. `V2d`'s `BNDR-5` Tests bullet and Ships-when require that the small-magnitude switch be witnessed by an interior-range value ("an in-range value such as `5e-8` is likewise expanded — not only the named magnitudes"); `V2d-T`'s `BNDR-5` Tests bullet stops at the `±1e21` and `|value| < 1e-7` switches and names no interior-range value. Because the tests leaf lands first and is authoritative under the TDD ritual, a test author working from `V2d-T` alone covers only the boundary magnitudes, and the interior-range assertion `V2d` requires is never written even though `V2d` later passes its gate.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** 31f4a86 — pi-loom plan: resolve "BNDR-5 number-renderer impl leaf cites only example vectors, not the |value| < 1e-7 threshold" (2026-06-11, Thomas Andersen)
**History:** `V2d` and `V2d-T` were created together in `c6a664e` with matching `BNDR-5` Tests bullets that named only the `1e21`/`1e-8` fixed-point expansions. Commit `31f4a86` resolved a prior finding by rewriting `V2d`'s `BNDR-5` Tests bullet and Ships-when to add the small-magnitude-switch framing and the interior-range obligation ("an in-range value such as `5e-8` is likewise expanded — not only the named magnitudes"). That commit edited `V2d` only; `V2d-T` was left untouched, so the two paired leaves diverged in a single edit and have stayed diverged since.

## Solution approach

Extend `V2d-T`'s `BNDR-5` Tests bullet to require that an interior-range value (such as `5e-8`) expands to full fixed-point — not only the named boundary magnitudes `1e21`/`1e-8` — mirroring the clause already in `V2d`'s `BNDR-5` Tests bullet so the paired leaves state the same obligation.

## Solution constraints

- Out of scope: the `V2d` leaf is the source-of-truth for the `BNDR-5` interior-range obligation and is not edited by this fix.
- If finding 78 pins a specific required vector in `V2d`'s `BNDR-5` clause, the clause carried into `V2d-T` must use that same pinned vector rather than the illustrative `5e-8`.

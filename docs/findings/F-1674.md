## Problem

The *Patch-skew degradation contract* opening paragraph (`#patch-skew-degradation-contract`) claims the loom 1.0 Pi-SDK pin (`#pi-sdk-pin`) permits patch-only Pi updates without re-running the Pi version bump procedure (`#pi-version-bump-procedure`), then specifies graceful degradation for a single surface — `SessionShutdownEvent['reason']`. The contract is silent about the rest of the unpinned host-presupposition catalogue in `host-prerequisites.md` (ten-plus presuppositions whose re-audit routes only to editorial review gated on a *minor* bump). The patch-skew allowance therefore tacitly extends to every catalogued presupposition without naming the assumption, so a Pi patch that violates any of them reaches users with no loom-side detection until the next minor bump's editorial review runs. Two reasonable implementers will read the silence differently — one as "only `SessionShutdownEvent['reason']` can drift on a patch," the other as "every minor-bump-gated presupposition is unprotected against patches."

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:**
- `75a9626` (2026-05-09) `pi-loom spec: resolve "SessionShutdownEvent.reason closed set has no build-time pin against the SDK type"` — introduced the *Patch-skew degradation contract* paragraph (and the "permits patch-only Pi updates without re-running [the bump procedure]" sentence) scoped to one surface only.
- A later sequence of presupposition-adding commits then accumulated unpinned-presupposition paragraphs against `host-prerequisites.md`, each routing its re-audit to editorial-review checklist items gated on a *minor* bump only: `79fa3b9` (2026-06-04, `model-registry-population-presupposition`), `6caacfe` (2026-06-05, `settings-write-back-preservation-presupposition`), `803106d` (2026-06-07, *Pi behavioural presuppositions* batch), `db69f2b` (2026-06-22, `loom-flag-namespace-presupposition`), `bef37f3` (2026-06-25, `registry-persistence-presupposition`), `fa69d8f` (2026-06-25, relocating `getcommands-completeness` and `register-tool-post-startup` into the host-prerequisites inventory), `0ffc66e` (2026-06-25, relocating both `.messages` presuppositions), `605a3a7` (2026-06-25, relocating `pi-slash-handler-promise-lifecycle`).

**History:** the defect is the *interaction* between two corpora that evolved independently. The patch-skew contract was authored in May for one surface only — at the time it landed, the host-prerequisites file did not yet enumerate the ten-plus presupposition paragraphs the contract now implicitly covers. The June presupposition-adding commits then doubled the host-prerequisites inventory without anyone going back to widen the patch-skew window's scope; each one added a checklist item under "re-audit on each Pi minor bump" without touching the patch-skew page. No single commit authored the silent assumption — it was created by additions to one file leaving a peer file's scope claim stale. The durable fix addresses that scope interaction at the patch-skew opening paragraph rather than reverting any of the per-presupposition adds.

## Solution approach

Add an assumption clause to the *Patch-skew degradation contract* opening paragraph (`#patch-skew-degradation-contract`) stating that loom 1.0 assumes a Pi patch release preserves every unpinned host presupposition catalogued in `host-prerequisites.md`, including the bundler-output "known-fragile evidence" sites those presuppositions cite. Name the failure mode: a Pi patch that violates any catalogued presupposition reaches users with no loom-side build-time gate, and detection routes through the next minor bump's editorial-review checklist. Cite the host-prerequisites presupposition inventory by anchor rather than enumerating presupposition names inline, so a later-added presupposition is covered without re-editing this clause.

## Solution constraints

- Out of scope: the per-presupposition paragraphs in `host-prerequisites.md` and their bump-procedure editorial-review routing in `version-bump-intro.md` — the fix adds patch-window scope at the patch-skew page, not new patch-bump gating on individual presuppositions (F-1535 owns the editorial-review-checklist surface and resolves independently).

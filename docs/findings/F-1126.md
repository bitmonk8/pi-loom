## Problem

Two MUSTs in `audit-wire-and-canary.md` delegate their contentful value wholesale to the implementation with no enforceable shape constraint. The *Wall-clock budget* clause (anchor `audit-infrastructure-failure-handling`) requires a budget but pins no value, so a conformant implementation may pin 1 ms (every real run trips the timeout) or 86 400 s (the fail-closed property is neutralised because the outer `npm test` timeout fires first and removes the routable infrastructure-failure record); the parallel `npm test`-side timeout the *Pre-emission termination carve-out* names is unbounded the same way. The *Green-path emission contract* clause (anchor `audit-non-empty-scan-canary`) requires the canary's `proposed-resolution` field to carry `walked=<N> recognised=<M>` "or an equivalent implementation-chosen" pair set, so no portable CI parser can write against the only contentful green-path field — name-based parsing breaks across implementations, and the positional guarantee is buried inside the substitution license.

**History:**

- `e5cd514` (2026-05-09, *"pi-loom spec: resolve 'T35 SDK capability inventory closed-set negative-direction audit'"*) first added both the *Wall-clock budget* clause and the *Green-path emission contract* clause, in `docs/spec_topics/pi-integration-contract.md` (the monolithic PIC page that predated the split). At inception the wall-clock budget was delegated to *"the H1 leaf pins the budget value, e.g. 60 s, in [`h1-scaffold.md`](../plan_topics/h1-scaffold.md)"* and the canary counter-key wording was delegated to *"an equivalent leaf-pinned plain-ASCII key=value pair set whose pair names the H1 leaf publishes per [`h1-scaffold.md`]"*. In both cases the value/name was still implementation-defined in the strict sense, but it was anchored to a single named, version-controlled plan artefact, so a reviewer could read the pinned value and the spec corpus pointed at a single source of truth.
- `9e53613` (2026-05-26) rewrote both delegations as part of a sweep that removed H1 / plan-leaf references from the spec corpus: *"the H1 leaf pins the budget value, e.g. 60 s, in `h1-scaffold.md`"* became *"the implementation pins the budget value, e.g. 60 s"*; *"an equivalent leaf-pinned plain-ASCII key=value pair set whose pair names the H1 leaf publishes per `h1-scaffold.md` identify the same two counters"* became *"an equivalent implementation-chosen plain-ASCII key=value pair set whose pair names identify the same two counters"*. The parallel `npm test`-side-timeout clause was rewritten in the same way (*"with the H1 leaf pinning a `npm test`-side timeout"* → *"with the implementation pinning a `npm test`-side timeout"*). This is the commit that turned delegation-to-a-named-artefact into delegation-to-any-conformant-implementation, producing the unbounded budget MUST and the un-portable counter-name allowance the finding describes. The sweep was scoped to *moving* the pin off the H1 leaf, not to *bounding* it or to closing the substitution surface; adding numeric bounds and pinning canonical key names were not part of its mandate, so both gaps are oversights of the corpus-direction reframing rather than deliberate loosenings.
- `f5e89f4` (2026-06-04, *"docs: enforce 100KB hard / 30KB recommended size cap on spec set"*) and `81bf52a` (2026-06-11, *"docs: collapse spec/ section pages into spec_topics/"*) split the monolithic PIC page into the subdirectory layout, moving both clauses into the current `audit-wire-and-canary.md`. Neither commit changed the clause text.

The defect's modern shape is owned by `9e53613`; the inception form (`e5cd514`) carried a milder version of the same gap on both halves — still no numeric bound and still a substitution allowance on the key names, but both were anchored to a single plan artefact rather than to "any conformant implementation".

## Solution approach

In the *Green-path emission contract* clause (anchor `audit-non-empty-scan-canary`), narrow the `proposed-resolution` pair-set to pin `walked=<N> recognised=<M>` as the sole required form, dropping the implementation-chosen-substitution allowance, and rewrite the closing PIC-pins sentence of the same paragraph in lockstep so the page does not contradict itself on whether substitution is permitted. In the *Wall-clock budget* and *Pre-emission termination carve-out* clauses (anchor `audit-infrastructure-failure-handling`), pin a concrete numeric audit budget and a concrete `npm test`-side timeout in place of the present implementation-delegated values.

## Solution constraints

- The inner audit wall-clock budget MUST be strictly less than the outer `npm test`-side timeout, so the audit's routable infrastructure-failure record fires before the outer `npm test` kill removes the routing surface.

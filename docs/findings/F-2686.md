## Problem

The step-2 editorial-review checklist requires recording a `pass` / `fail` / `N/A` outcome (with `not-performed` for a skipped SHOULD-level audit) for each presupposition-audit item, but `N/A` carries a defined legitimacy condition only for item (e), whose conservative-posture rule forces `fail` rather than `N/A` when the auditor merely cannot demonstrate the property. For the generic SHOULD-level items (f)–(aq) and item (as), nothing states when `N/A` is legitimate versus when a performed-but-undemonstrable audit must be recorded `fail`; a contributor who is uncertain can write `N/A`, silently discharging a real candidate-minor regression and defeating the editorial gate that is loom 1.0's only enforcement for these unpinned presuppositions. Item (ar) is the opposite hole: it is a mandatory per-bump reconciliation with no presupposition that can "fail to apply", so `N/A` is meaningless for it, yet it inherits the generic `pass` / `fail` / `N/A` shape from the preamble.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `52b9101` (generic-`N/A` arm); `1a59ccf` (item-(ar) facet)
**History:** The editorial-review checklist was introduced in `52b9101` (2026-05-11, "T24 — Fork-reason watcher closure …"), whose added preamble already read `MUST record the per-item audit outcome (pass / fail / N/A with one-line rationale)` with four presuppositions and **no** generic `N/A` legitimacy condition. Subsequent commits expanded the SHOULD-level item range to (f)–(as) under that same shape (`aebec4e`, `29f4cba`, `fdd8e76`, and others), and item (e) alone later acquired a specific `N/A` definition plus the conservative-posture rule (`c5f97f6`, "Item (e) recovery over-prescribes mutex; N/A and recording conventions buried") — the generic items never did. The item-(ar) facet entered with `1a59ccf` (F-2264, "demote build-time inbound-reference audit gate to advisory/post-1.0, wire 1.0 enforcement via editorial-checklist item (ar)"), which added item (ar) as a MUST-each-bump reconciliation but recorded it under the inherited `pass / fail / N/A` shape, carrying the undefined `N/A` onto an obligation that has no presupposition to disapply.

## Solution approach

Add a generic `N/A` legitimacy condition covering items (f)–(aq) and (as), whose natural home is the existing `Outcome recording conventions` subsection (`#bump-step-2-outcome-recording-conventions`): mirror item (e)'s form so `N/A` is permitted only when the audited presupposition demonstrably does not apply to the candidate minor for a named reason, a performed-but-undemonstrable audit is recorded `fail` (generalising item (e)'s conservative-posture rule), and a skipped SHOULD-level audit remains `not-performed`. Separately, narrow item (ar)'s recorded-outcome shape at `#bump-checklist-inbound-reference-reconciliation` to drop `N/A` (record `pass` / `fail` only), since it is a mandatory per-bump reconciliation with no presupposition that can fail to apply.

## Solution constraints

- The generic rule MUST NOT subsume or override item (e)'s bespoke `N/A` definition and conservative-posture rule, nor item (a)'s authoritative-determination recording shape; both remain carve-outs.

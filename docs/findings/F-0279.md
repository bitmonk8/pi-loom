## Finding

The swallowing-handler test bullet in both `V9i` and `V9i-T` closes with the rationale clause "…so a build that bypasses the substrate reddens this leaf's tests." The noun "substrate" has no antecedent inside either leaf. A reader working from the leaf alone has two plausible referents: the `H4b` response-programming surface (declared in `Deps`, the harness the leaf's tests are built on) and the `V8a` `Checkpoint` seam (named one clause earlier in the same sentence as the site where the late settlement is landed). The two referents point at opposite layers — the test harness versus the production cancellation seam — so the clause's failure-mode meaning is ambiguous.

The intended referent is the `V8a` `Checkpoint` seam: `cancellation.md` defines the term ("The `Checkpoint` seam … is the deterministic-test substrate for this discard" and "The `Checkpoint` seam provides the deterministic-test substrate for landing the late settlement at a chosen point"), and the sibling `V13f` leaf glosses it inline as "the `Checkpoint`-seam swallowing-handler substrate" in its `Adds`. Neither `V9i` nor `V9i-T` carries that gloss, so the term arrives undefined at its point of use.

The undefined-term condition has existed in the V9i pair only since that commit.

## Solution Space

**Shape:** multiple

### Option A — Strike the rationale clause

**Approach.** Remove the trailing "— so a build that bypasses the substrate reddens this leaf's tests" clause from the swallowing-handler bullet in both `V9i` and `V9i-T`, leaving the fully-specified assertion (construction-site attachment, late settlement via the `Checkpoint` seam `V8a`, three-channel suppression).

**Plan edits.** Delete the clause from the `Tests` swallowing-handler bullet in `docs/plan_topics/V9i-subagent-isolation.md` and `docs/plan_topics/V9i-T-subagent-isolation.md`.

**Spec edits.** None.

**Pros.** Removes the ambiguous term entirely; no risk of the gloss drifting from the spec's wording.

**Cons.** Loses the "why this test exists / what a regression looks like" framing that the clause supplied.

**Risks.** None of consequence; the assertion content is untouched.

### Option B — Name the seam inline

**Approach.** Replace the undefined "the substrate" with an explicit reference to the `Checkpoint`-seam swallowing-handler mechanism (`V8a`), matching the gloss the sibling `V13f` leaf already uses, so the rationale clause survives but resolves unambiguously.

**Plan edits.** In the `Tests` swallowing-handler bullet of both `docs/plan_topics/V9i-subagent-isolation.md` and `docs/plan_topics/V9i-T-subagent-isolation.md`, rewrite "bypasses the substrate" to name the `Checkpoint`-seam swallowing-handler substrate (`V8a`) — e.g. "a build that bypasses the `Checkpoint`-seam swallowing-handler substrate (`V8a`) reddens this leaf's tests."

**Spec edits.** None.

**Pros.** Preserves the failure-mode rationale; aligns V9i/V9i-T phrasing with `V13f`'s already-glossed usage.

**Cons.** Slightly longer bullet; the inline gloss must stay consistent with `cancellation.md`'s definition.

**Risks.** None of consequence.

### Recommendation

Take Option B: name the `Checkpoint`-seam swallowing-handler substrate (`V8a`) inline, because "substrate" is already the spec's term for the `Checkpoint` seam and the sibling `V13f` leaf glosses it the same way — adopting that phrasing resolves the ambiguity while keeping the rationale and keeping the V9i pair consistent with its sibling. Apply the identical edit to both `V9i` and `V9i-T` so the paired implementation and test leaves stay in sync. The spec (`cancellation.md`) is read-only for this fix.

## Consequence

**Severity:** cosmetic

The test obligation itself is fully specified by the rest of the bullet (attach the swallowing handler at the Promise-construction site, land a late settlement via the `Checkpoint` seam `V8a`, assert three-channel suppression). Only the trailing rationale flourish is ambiguous, so no implementer is misdirected on what to build; the cost is a reader momentarily mis-resolving "substrate" to the `H4b` harness instead of the `V8a` seam.

## Finding

"loom does **not** run an automated cadence (no Renovate, no scheduled CI re-install, no nightly cron) that proactively re-imports patches in loom 1.0, so the build-time gate is best-effort: the window's duration depends on contributor cadence and is not bounded by spec, and it detects the skew only as fast as a loom contributor next runs `npm run typecheck` against the candidate patch." This is a build/contributor-process constraint (no automated dependency update tooling, contributor cadence obligation) embedded inside a runtime behavioral contract about patch-skew handling.

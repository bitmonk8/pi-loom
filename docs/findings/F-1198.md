## Problem

The `provider-error-mapping.md` shard carries five independent classifier defects sharing a page but not an edit. (1) The `#stop-reason-classification` catch-all bullet calls a content-filter terminator a "non-terminal stop reason", which is self-contradictory — a content-filter stop is terminal. (2) The output-boundary bullet asserts pi-ai surfaces truncation as `stopReason: "length"`, but no fixture or editorial-checklist item pins that vocabulary token, so a pi-ai rename would silently dead-letter every output overflow into the catch-all `TransportError` branch with nothing going red. (3) The `#provider-seed-field-mapping` `Api`-coverage assertion enumerates pi-ai's `KnownApi` union and asserts every value is a row key, but the four-row table keys on short names (`mistral`, `amazon-bedrock`) that are not union members and omits five real `KnownApi` values, with no default-omit rule and no rule for the open `(string & {})` arm — so the assertion reds on day one. (4) The `mistral` and `amazon-bedrock` overflow rows read `body matching`, contradicting the `#classifier-input-surface` rule that every body-wording match is against `AssistantMessage.errorMessage` — pi-ai exposes no parsed body for them to match. (5) The `#transport-error-retryable` capability-gap clause points "above" to *Provider compatibility for typed queries*, which lives in `conversation-drive.md` after the shard split, so the pointer dangles cross-file.

## Solution approach

Rewrite the `#stop-reason-classification` catch-all bullet so its lead clause describes the set complement (any `stopReason` other than `"length"` and other than the `end_turn` / `stop` / `tool_use` turn-boundary set) rather than calling a content-filter terminator "non-terminal" — this also resolves F-0750. Convert the `mistral` and `amazon-bedrock` overflow rows from `body matching` to `errorMessage matching`, the only surface the classifier reads per `#classifier-input-surface`. Re-key the seed-field table to pi-ai's actual `KnownApi` tokens (`mistral-conversations` for `mistral`, `bedrock-converse-stream` for `amazon-bedrock`, plus rows for the five currently-missing members), add a default-omit rule covering unenumerated members and the open `(string & {})` arm, rewrite the coverage-assertion sentence to match, and express the lookup on `Model<Api>.api` per `binder-inference.md`. Pin the `stopReason: "length"` vocabulary by adding an editorial-review checklist item in `version-bump-step2.md` (mirroring existing items (m) / (af) / (al)) with a forward-reference from the output-boundary bullet. Fix the dangling "above" pointer in `#transport-error-retryable` to a cross-file link to `conversation-drive.md#provider-compatibility-for-typed-queries`.

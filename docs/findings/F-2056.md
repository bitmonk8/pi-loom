## Finding

The glossary entry that defines the term carries a normative sentence: *"Authors and implementers MUST use the canonical term `.loom callable` (or, in running prose, `` `.loom` callable ``) for a `.loom` entry in `tools:`."* This designates two distinct forms: the **canonical** form `.loom callable` (a single code span wrapping the whole phrase) and an **acceptable prose variant** `` `.loom` callable `` (a code span on `.loom` only, followed by the bare word `callable`).

The entry's own heading is **Pi tool** vs. **`` `.loom` callable ``** — i.e. it uses the prose-variant form, not the canonical form the sentence beneath it pins as primary. An entry whose job is to coin and canonicalise a term should headline with that term's canonical spelling; here the definitional heading instead uses the secondary, prose-only variant. The MUST does permit the variant in running prose, so this is not a normative violation — it is an internal-consistency mismatch between the heading and the canonicalisation rule it introduces.

The fix is a single edit: change the heading to **Pi tool** vs. **`.loom callable`** (single code span), matching the MUST-canonical form. The cross-reference in the sibling `callable set` entry (*"Cf. *Pi tool* vs *`.loom` callable* below"*) is non-normative descriptive prose and is not in scope for this finding.

No earlier commit could exhibit the defect because no canonical-term rule existed before then.

## Solution Space

**Shape:** single

### Recommendation

Change the entry heading from **Pi tool** vs. **`` `.loom` callable ``** to **Pi tool** vs. **`.loom callable`**, wrapping the whole phrase `.loom callable` in a single code span so the heading matches the MUST-canonical form stated in the entry body. Leave the body sentence, the synonym list, and the `See:` links unchanged. The page-local entry ordering is alphabetised loosely and is not keyed on the code-span boundary, so the heading change does not perturb entry placement or any inbound anchor.

## Consequence

The heading uses a spelling the same entry classifies as the secondary prose variant rather than its canonical form. No implementer behaviour, wire format, or anchor is affected; the inconsistency is purely editorial and slightly undercuts the entry's own canonicalisation directive.

## Finding

In `host-interfaces-core.md`, the `*Per-mode override semantics.*` sub-block — its lead sentence ("The runtime synthesises the `ctx` it hands to `tool.execute(...)` by starting from the live host's `ExtensionContext` and overriding exactly three members"), the prompt-mode/subagent-mode override table for `signal` / `sessionManager` / `abort`, the two paragraphs itemising every forwarded-unchanged member in each mode, and the closing both-modes `abort()` paragraph — sits inside the `**ExtensionCommandContext (per-handler subtype loom touches)**` section.

Its subject is not `ExtensionCommandContext`. The block describes how the runtime synthesises the `ctx` passed to `tool.execute(...)`, which the spec types and names elsewhere as a plain `ExtensionContext` ("the synthesised tool-execution `ctx`"). The `ExtensionCommandContext` section's own scope is narrow: per the spec's statement immediately above the block, `waitForIdle` is the only `ExtensionCommandContext`-specific member loom touches in loom 1.0, and that member is explicitly *not* present on the synthesised tool-execution `ctx`.

The misfiling also breaks an inbound pointer. The `**Tool execution from loom code**` section's `ctx` bullet resolves the synthesised context as "the synthesised `ExtensionContext` defined under [`ExtensionContext` (member surface loom touches)](#extensioncontext-interface) above … per the per-mode-override table". That citation directs a reader to the `ExtensionContext` section for the override table, but the table actually lives one section further down, under `ExtensionCommandContext`.

## Issue introduction
**Verdict:** single-commit
**Introducing commits:** 68ff2d1 — pi-loom spec: resolve "Pi API surfaces asserted without .d.ts citations" (2026-05-07, Thomas Andersen)
**History:** The `*Per-mode override semantics.*` block was authored in 0e080df (2026-05-06) correctly inside the `ExtensionContext (member surface loom touches)` section, directly between that section's signature block and `Tool execution from loom code`. Commit 68ff2d1 then inserted a new `ExtensionCommandContext (per-handler subtype loom touches)` section heading and body ahead of the pre-existing override block, leaving the block stranded on the wrong side of the new heading. The page later split out of `pi-integration-contract.md` into `host-interfaces-core.md` (81bf52a) but carried the misplacement through unchanged.

## Solution Space

**Shape:** single

### Recommendation

Move the entire `*Per-mode override semantics.*` block — lead sentence, override table, both forwarded-unchanged paragraphs, and the closing both-modes `abort()` paragraph — back into the `**ExtensionContext (member surface loom touches)**` section, positioned after the "Members loom does not touch (e.g. `reload()`, …)" paragraph and before the `**ExtensionCommandContext (per-handler subtype loom touches)**` heading. This restores the block to the location the `Tool execution from loom code` `ctx` bullet already cites ("under [`ExtensionContext` (member surface loom touches)] above … per the per-mode-override table"), so no cross-reference needs to change.

Edge cases for the implementer: keep the block below the `ExtensionContext` TypeScript signature code-fence and its `*Pi-owned snapshot.*` notes (do not merge it into the code block); and leave the `waitForIdle`-absence note in the `ExtensionCommandContext` section, since that note is genuinely `ExtensionCommandContext`-scoped.

## Consequence

**Severity:** advisory

An implementer reading the `ExtensionCommandContext` section encounters override semantics for a different type (`ExtensionContext`), and an implementer following the `Tool execution from loom code` cross-reference to the `ExtensionContext` section does not find the per-mode-override table where it is pointed. The `signal`/`sessionManager`/`abort` override rules are load-bearing for tool-execution and cancellation in both modes, but the content itself is correct and findable, so no implementer would derive divergent behaviour — only a navigation/orientation gap.

## Finding

The catch-all row uses a `<message>` placeholder without specifying its source. Named rows derive `<message>` from specific, documented fields of known `QueryError` variants. For an unlisted future variant, the spec does not say whether `<message>` should come from a `message` property on the `QueryError` object (if present), from `String(error)`, from an empty string, or from some other derivation. A renderer implementing this catch-all has two or more plausible interpretations with different observable outputs.

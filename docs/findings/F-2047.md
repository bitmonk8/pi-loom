## Problem

The `callable set` glossary entry in `docs/spec_topics/glossary.md` positively coins `ambient tool set` as a load-bearing term — the host Pi session's currently-active tools, which a callable set deliberately does *not* inherit — and bars its near-synonyms, yet gives the term no entry of its own. The concept is reused beyond that entry: `ambient tool set` appears in `invocation.md`, the scoped `ambient Pi tool set` in `pi-integration-contract/subagent.md`, and `ambient tools` in `tool-registration-lifetime.md` and `frontmatter/frontmatter-fields-a.md`. This clears the glossary's own "reused on more than one page" admission threshold, but a reader searching the glossary for the concept the callable set deliberately excludes has no direct target — it is discoverable only by reading the `callable set` entry in full.

## Issue introduction
**Verdict:** single-commit

**Introducing commits:** `fec0870` ("pi-loom spec: resolve \"callable-set prose synonyms erode no-inheritance invariant\"")

**History:** `git log -S "single distinct concept that does use the bare phrase"` resolves to a single commit, `fec0870`. That commit ran the coordinated terminology pass that coined `ambient tool set`: it added the disambiguating sentence to the `callable set` glossary entry and converted the bare `tool set` occurrences in `spec.md` and the integration-contract pages to `ambient tool set`. The same commit edited `glossary.md` (+4 lines on the `callable set` entry) but did not add a sibling `ambient tool set` entry, so the term was positively coined and propagated across pages in the very change that left it unregistered. The predecessor finding the commit resolved had itself flagged re-tagging the Pi-session tools as `ambient tool set`; the glossary registration was simply never carried out alongside the coinage.

## Solution approach

Add an alphabetised `ambient tool set` entry to `docs/spec_topics/glossary.md` defining the term as the Pi host session's currently-active tool set that a loom's `callable set` deliberately does not inherit. Register both the bare `ambient tool set` and the scoped `ambient Pi tool set` spellings as the permitted names for the concept, and bar its use as a synonym for `callable set`. Cross-reference the canonical owners with a `See:` line to [Parameters and Frontmatter](./frontmatter.md) and [Pi Integration Contract — Tool-registration lifetime and visibility](./pi-integration-contract.md).

## Solution constraints

- The new entry points at the no-inheritance MUST; it must not duplicate or restate it. The MUST stays owned by the `callable set` entry and the canonical pages, so the new entry is a discovery surface, not a second normative home.

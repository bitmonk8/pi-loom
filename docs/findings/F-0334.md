## Problem

`H4a`'s session-double **fidelity contract** enumerates four pinned-`@earendil-works/pi-coding-agent` behavioural axes the in-process double must reproduce — (i) streamed-token order relative to `ctx.waitForIdle()` resolution, (ii) single-turn prompt-mode append semantics, (iii) the `pi.on` cancel-forward subscription, and (iv) cancellation propagation — but the leaf points at no spec page that defines them. These are behavioural contracts not pinned by the typed `.d.ts` surface, so an implementer cannot recover their semantics from SDK type declarations. Without spec pointers the implementer builds the double and its self-check from intuition, and two reasonable implementers can produce doubles that disagree on commit-ordering or cancel-forward timing while each self-check passes. Because every downstream V-leaf harness inherits this double, a wrong model propagates corpus-wide with no mechanical gate (only the manual real-host smoke).

That introducing commit enumerated all four axes but added no spec-page citation, and every subsequent edit that reworked the block (`07403da`, `e7f14dd`, `85fc906`, `3911733`, `ad23856`, `4b3c2c0`) preserved that gap (`git log -G "host-interfaces-core" --follow` over the file returns nothing). The missing-citation defect is therefore co-extensive with the contract itself — the spec sources were never present.

## Solution approach

Add read-only cross-references to the authoritative spec pages where `H4a`'s fidelity-contract block names the four axes — in the Adds sentence that enumerates them and/or the third Tests self-check bullet — mirroring the existing `host-prerequisites.md §"Extension entry point"` cross-reference already in Adds, so the self-check is stated to assert the double against the spec model rather than an invented one. Attach each axis to its source: axis (i) to `conversation-drive.md#user-visible-streaming-ordering-presupposition` and the `host-interfaces-core.md#extensioncommandcontext-interface` `waitForIdle()` block; axis (ii) to `conversation-drive.md#driven-turn-commit-ordering-presupposition`; axis (iii) to `conversation-drive.md#pic-18` and `cancellation.md` "Forwarding into `loomAbort`"; axis (iv) to `cancellation.md#cncl-4`. Frame the citations as the authoritative behaviour model the double reproduces (loom-side consumption postures / presuppositions), not as a mechanically verifiable host contract.

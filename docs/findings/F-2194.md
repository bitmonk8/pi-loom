## Problem

Placeholder-rendering §6 ("Underlying-error coercion", `id="underlying-error-coercion"`) bundles two rules whose scopes are entangled: the coercion step reaches broadly into structured payload fields (`details.error`, `details.cause`, a `hint`/`message` carrying a caught throw, `CodeToolError.message`), then states "The truncation rule below then operates on this guaranteed string", which a reasonable implementer reads as applying §6's first-line truncation to those structured fields too. This contradicts normative registry rows that require intentionally-multiline structured payloads: `loom/runtime/internal-error` mandates `hint` carry the full multiline `error.stack`, and `loom/runtime/registration-cache-collision` (full canonical-form bytes) and `loom/runtime/reload-teardown-timeout` (numeric elapsed-ms) carry untruncated `hint` payloads. With no per-field rule, two implementers diverge — one first-line-truncates every coerced structured field (mangling the stack and blobs), the other truncates only the rendered Message-column placeholders — producing byte-divergent diagnostics for the same caught throw.

## Issue introduction
**Verdict:** multi-commit-interaction
**Introducing commits:** f8f4813 — pi-loom plan: resolve "Panic routing has no defined path for unexpected interpreter exceptions" (2026-05-05, Thomas Andersen); bf85092 — pi-loom spec: resolve "non-Error-throw coercion + session_start pi.getCommands() failure contract" (2026-06-06, Thomas Andersen)
**History:** f8f4813 introduced the `loom/runtime/internal-error` row whose `hint` carries the full multiline `error.stack` "for operator triage". bf85092 later added §6's broad underlying-error-coercion clause — extending coercion to structured fields including `hint` and `details.error`/`details.cause`, immediately followed by "The truncation rule below then operates on this guaranteed string." The contradiction arises from the interaction: bf85092's truncation-scope wording does not exempt the structured fields that f8f4813's row requires to stay multiline, and neither commit reconciled the two.

## Solution approach

Split §6's bundled paragraph so the coercion step and the first-line truncation rule carry distinct scopes. Keep coercion broad over the caught-throw placeholders and the structured fields; rewrite the "The truncation rule below then operates on this guaranteed string" sentence to scope first-line truncation to the rendered underlying-error placeholders (`<error.message>`, `<original content first line>`, `<dispose error first line>`) and the §8 freeform-tail placeholders (`<error>`, `<message>`, `<observed>`). Clarify that coerced structured payload fields carry the full untruncated string, naming `loom/runtime/internal-error`'s `hint = error.stack` as the canonical multiline case.

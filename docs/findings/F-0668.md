## Finding

The opening fenced loom example in `docs/spec_topics/descriptions.md` closes with the line

```loom
/// (See Errors and Results for the canonical QueryError declaration)
```

placed after the closing `}` of `enum Severity` and not above any declaration. The same page's own `Placement` rule (and `grammar.md#-placement`, which it cites as normative) lists the legal anchors for `///` — `schema`, `enum`, schema field, enum variant, `fn` — and states that a `///` above any other production is `loom/parse/doc-comment-misplaced`. End-of-block with no following declaration falls in that "any other production" set, so the example demonstrates code that the very next paragraph defines as a parse error.

The line's content is also redundant: the paragraph immediately under the fence already explains that `QueryError` is declared in `errors-and-results/queryerror-variants.md` and is not restated here. The example needs the line neither for instructional value nor for cross-reference; it is residue from a prior revision in which a real `schema QueryError = …` declaration sat below the `///`.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** `6750c81` ("spec: consolidate QueryError variants; rename to InvokeInfraError / CodeToolError / ModelToolError", 2026-05-05)
**History:** Before `6750c81` the example contained `/// Top-level error returned by every query` followed by a `schema QueryError = …` declaration — a legal placement under the page's own rules. That commit (implementing spec-review decision #1: move all QueryError variants to a single canonical home in `errors-and-results.md`) deleted the `schema` declaration but kept and rewrote the leading `///` line as `/// (See Errors and Results for the canonical QueryError declaration)`. The deletion of the anchor below it is what made the `///` orphan; the surrounding prose update in the same commit was meant to carry the cross-reference, making the `///` line itself redundant once the schema disappeared. No later commit touched this fenced block.

## Solution Space

**Shape:** single

### Recommendation

Delete the trailing `/// (See Errors and Results for the canonical QueryError declaration)` line from the opening fenced loom example. The blank line above it and the closing ` ``` ` should remain; the paragraph that follows the fence already names the canonical home of `QueryError` in `errors-and-results/queryerror-variants.md`, so no replacement text is required.

Edge cases: the line must be removed, not rewritten as a `//` comment — a `//` line at the end of the example would be benign syntactically but adds noise the prose already covers. Do not introduce a placeholder declaration solely to host the `///`; the example's purpose is to show `///` attached to real declarations above (the `schema ReviewRequest` block and `enum Severity` variants), and a stub would add no instructional value.

## Consequence

**Severity:** cosmetic

A learner skimming the example before reading the Rules block sees `///` used in a position the rules then forbid, undermining the page's pedagogical value and the credibility of its own placement rule. No implementation behaviour is affected.

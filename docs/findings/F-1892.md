## Problem

PIC-55 (`registration-steps.md#pic-55`) is a runtime obligation for watcher post-error recovery: either a transient continues-delivering toast, or — when one or more watched roots stop delivering — a torn-down watcher that emits exactly one persistent `loom/runtime/watcher-terminated` `loom-system-note` prompting `/reload`, keeps `LoomRegistry` live and dispatchable, and writes no drain-state tag. No plan leaf implements either arm; the only watcher-touching leaf `V8e` is scoped to the `FileWatcher.watch` seam (PIC-14/16) and disclaims the recovery posture. PIC-55 also has no `coverage-matrix.md` row, and the registered `watcher-terminated` diagnostic has no asserting test, so at the `H6a` release-gate `H5a` hard-fails both as an unmapped executable REQ-ID and as a registered code with no asserting test.

**Introducing commits:** `0b195eb` (2026-06-25, "pi-loom spec-plan: fix F-1299+F-1308 — coin PIC-55 watcher post-error posture + loom/runtime/watcher-terminated")
**History:** `git log -S "pic-55"` and `git log -S "watcher-terminated"` over `docs/spec_topics/` both return `0b195eb` as the sole introducing commit; `git log -S "PIC-55" -- docs/plan_topics/` returns nothing, confirming PIC-55 has never appeared in any plan leaf or in `coverage-matrix.md`. `git show --stat 0b195eb` shows the commit touched only five spec-side files (`registration-steps.md`, `code-registry-runtime.md`, `diagnostic-shape.md`, `host-interfaces-services.md`, `spec-plan-review.md`) and no `docs/plan_topics/` file. The plan-side coverage gap was therefore introduced atomically by `0b195eb` coining the PIC-55 obligation and the `watcher-terminated` code spec-side without adding the corresponding plan leaf and coverage-matrix row; the gap has been continuously present since that commit.

## Solution approach

Add a new paired vertical-slice leaf (tests `-T` plus implementation, per the plan-rules pairing for vertical features) that closes PIC-55 and asserts the `loom/runtime/watcher-terminated` code, citing `registration-steps.md#pic-55`, the `watcher-terminated` row in `code-registry-runtime.md`, and PIC-14 `watch` in `host-interfaces-services.md`, and depending on `V8e`, `V7b`, and `V7d`. Register the PIC-55 row in the Numbered REQ-IDs table of `coverage-matrix.md` pointing at the new leaf's ID, link the leaf into the Vertical slices section of `plan.md`, and add the new leaf to `H5b`'s `Deps.`

## Solution constraints

- Out of scope: the `FileWatcher.watch` seam shape (PIC-14/16) owned by `V8e` — the new leaf depends on `V8e` but does not redefine that seam.
- The continues-delivering transient-toast arm overlaps a separate transient-toasts coverage finding's surface; assert that arm in only one leaf to avoid double-coverage.

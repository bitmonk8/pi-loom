## Problem

The leaf pair `docs/plan_topics/V5d-subset-lowering.md` / `docs/plan_topics/V5d-T-subset-lowering.md` carries the path slug `subset-lowering`, but V5d's title and Adds scope the leaf to the schema-subset reject gate only; V5d's Adds explicitly assigns the lowering pass and canonical-hash recipe to `V5f`. The result is two files whose slugs both advertise "subset lowering" (`V5d-subset-lowering` and `V5f-subset-lowering-hash`) while only V5f owns the lowering pass, so the slug points a directory- or TOC-navigating reader at the wrong leaf. The leaf IDs `V5d` / `V5d-T` and every `Deps.` citation reference the bare ID and are correct — the defect is confined to the file-path slug and the two inbound hyperlinks that target it.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** cb8737a — pi-loom plan: resolve "Systemic leaf over-bundling across the plan corpus" (2026-06-12, plan author)
**History:** At inception (c6a664e) V5d was a single leaf titled "Schema-subset gate, lowering, and canonical hash" that genuinely owned the reject gate, the lowering pass, and the canonical hash — so the `subset-lowering` slug was accurate then. Commit cb8737a de-bundled the leaf: it created `V5f-subset-lowering-hash.md` to own the lowering pass + canonical hash and rewrote V5d's title/Adds to "Schema-subset reject gate", but left V5d's and V5d-T's filename slug `subset-lowering` unchanged, stranding it on the narrowed scope. `git show cb8737a -- docs/plan_topics/V5d-subset-lowering.md` confirms the title/Adds rewrite without a filename change in that commit.

## Solution approach

Rename the V5d leaf pair so the slug reflects its narrowed reject-gate scope: `docs/plan_topics/V5d-subset-lowering.md` → `docs/plan_topics/V5d-reject-gate.md` and `docs/plan_topics/V5d-T-subset-lowering.md` → `docs/plan_topics/V5d-T-reject-gate.md`. Retarget the two inbound hyperlinks that reference the old path: the V5d link in the `docs/plan.md` V5 TOC entry, and the `[`V5d`]` link in the `docs/plan_topics/V5f-subset-lowering-hash.md` Adds. Leave the leaf IDs `V5d` / `V5d-T` unchanged.

## Solution constraints

- Leave the leaf IDs `V5d` / `V5d-T` unchanged: every `Deps.` citation across the corpus and the `coverage-matrix.md` row reference the bare ID, not the filename.

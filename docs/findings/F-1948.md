## Finding

`V18b` Adds bundles two qualitatively different deliverables under one Ships-when. The first is the build-time surface-set-closure audit gate itself — the walker over `src/**/*.ts`, the category-(1)/(2)/(3) recogniser detection, the non-exemptible family-(4) out-of-scope discriminator, the fail-closed non-empty canary, and the `audit/<class>/<family>/<symptom>` record format — all of which `V18b-T` exercises against seeded fixtures. The second is the land-green `src/` sweep the spec assigns to the audit-introducing commit: deleting, promoting to `SDK_SURFACE_INVENTORY`, exempting with `// allow-pi-surface:`, or rewriting every existing Pi-side reference the new audit would surface as a violation.

The sweep is a potentially wide refactoring pass across the live source tree, and it has no tests of its own — `V18b-T`'s four Tests bullets all assert gate behaviour (off-inventory reference reddens, exemption/typebox allow-lists pass, canary fail-closed, family-(4) non-exemptible), none of them assert anything about the swept production tree. The sweep also cannot be performed or verified until the gate exists to surface which references need fixing. Folding an untestable, broad refactor into the same leaf as the gate implementation produces an oversized diff spanning two abstraction layers (gate tooling vs. production-tree cleanup) with a single done-condition.

## Solution Space

**Shape:** single

### Recommendation

Factor the land-green `src/` sweep out of `V18b` into a new sibling leaf, leaving `V18b`/`V18b-T` owning the audit-gate implementation verified against a fixture corpus, and the new leaf owning the live-`src/` activation plus the sweep.

- In `V18b` Adds, strike the final sentence beginning "This leaf also owns the land-green `src/` sweep the spec assigns to the audit-introducing commit…" and scope `V18b` to coining the audit tooling (walker, category-(1)/(2)/(3) recognisers, family-(4) discriminator, canary, `audit/<class>/<family>/<symptom>` record format) verified against a seeded fixture corpus, not the live `src/` tree. `V18b-T` already tests exactly this; leave its bullets as the gate-behaviour assertions.
- Add a new leaf (`<new>`, paired `<new>-T`) Deps on `V18b` that activates the audit over the live audited source tree and, in the same commit, performs the land-green sweep (delete / promote to `SDK_SURFACE_INVENTORY` / `// allow-pi-surface:` exempt / rewrite into a recognised shape) for every reference the now-live audit surfaces, with Ships-when "`npm test` runs the closure audit green on `main`". Move `V18b`'s current Ships-when green-on-`main`-against-live-`src/` criterion to this new leaf.
- Link the new leaf into `docs/plan.md` §Vertical slices under the V18 slice.

Edge case the implementer must watch — the spec is **read-only** for this fix, and its land-green invariant constrains where the boundary falls: `inventory-audit-intro.md` §Inventory-closure audit requires the commit that *introduces the audit over `main`* to also carry the sweep, and forbids a "land red, fix incrementally" gate ("an implementation whose first `npm test` invocation against `main` is red on this audit violates this paragraph"). The split therefore must keep live-`src/` activation and the sweep together in the new leaf's single commit; `V18b` must verify the gate against fixtures only and must not wire the audit to run over the live `src/` tree, or `V18b` alone would redden `main`. Retarget `V18c`/`V18c-T` Deps (and, if `V18b`'s ID changes rather than staying the gate leaf, the `cka-20` closing-leaf cell) onto the leaf that delivers the green live-`src/` state, since the version-bump gates' `npm test` depends on it.

## Consequence

**Severity:** advisory

The leaf is still pickable and an implementer can produce a working result, so this is not a blocker; but the bundled wide refactor lands as a single oversized, partly-untested diff that is hard to review and whose sweep cannot be checked independently of the gate. The sweep's correctness rests entirely on TDD discipline and self-review rather than on any authored assertion.

## Problem

`V18b` Adds bundles two qualitatively different deliverables under one Ships-when: the build-time surface-set-closure audit-gate implementation (walker, category-(1)/(2)/(3) recognisers, family-(4) discriminator, fail-closed canary, `audit/<class>/<family>/<symptom>` record format) and the land-green `src/` sweep that deletes / promotes / exempts / rewrites every existing Pi-side reference the new audit would surface. `V18b-T`'s four Tests bullets all assert gate behaviour against seeded fixtures; none assert anything about the swept production tree, so the sweep ships untested. The sweep also cannot be performed or verified until the gate exists to surface which references need fixing. The result is an oversized diff spanning two abstraction layers with a single done-condition.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** 402bdbe — pi-loom plan: resolve "V18b Adds omits broader-inventory population, taxonomy, and land-green sweep" (2026-06-11, Thomas Andersen)
**History:** V18b's first authored Adds (c6a664e, 2026-06-10) described only the audit-gate implementation; it carried no land-green sweep. Commit 402bdbe (2026-06-11) appended the land-green `src/` sweep clause to close a prior review finding that the sweep obligation was omitted, folding that wide refactoring obligation into the existing gate-implementation leaf and creating the present bundling.

## Solution approach

Split the land-green `src/` sweep out of `V18b`. Narrow `V18b`/`V18b-T` to owning the audit-gate tooling verified against the seeded fixture corpus, deleting the sweep clause from `V18b` Adds. Add a new V18-slice leaf (paired with its `-T` tests task) that Deps on `V18b`, activates the audit over the live audited source tree, and performs the sweep in the same commit. Move `V18b`'s green-on-`main` Ships-when criterion onto the new leaf, and link the new leaf into `docs/plan.md` §Vertical slices under the V18 slice.

## Solution constraints

- The spec is read-only for this fix; `inventory-audit-intro.md` §Inventory-closure audit requires live-`src/` activation and the sweep to land in the same commit and forbids a land-red gate, so `V18b` must verify the gate against fixtures only and must not wire the audit over the live `src/` tree.
- Retarget `V18c`/`V18c-T` Deps onto whichever leaf delivers the green live-`src/` state, since the version-bump gates' `npm test` depends on it.

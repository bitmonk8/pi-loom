## Problem

V11f carries a normative production rule for the depth-walk fast-fail sub-case of the AJV-on-`args` failure class (a depth-6 breach at the `params` boundary routed ceiling #4 → ceiling #3 via CIO-1): the AJV-on-`args` row's `<ajv-summary>` must be synthesised from the depth-walk's canonical `ValidationIssue` (`schema_keyword:"maxDepth"`, message `"JSON document depth exceeds 5"`, single-issue form, no `; ` separator), because AJV does not run at this site and its `errors` array is empty. That rule lives only inside V11f's depth-walk fast-fail **Tests** bullet (duplicated in V11f-T). V11f's **Adds** field describes the output format but omits this synthesis constraint. The leaf-format convention assigns normative production rules to **Adds**, so a rule that determines what code the implementer writes is misplaced in the test-obligation section.

The constraint entered the corpus inside that Tests bullet and was never placed in V11f's Adds — the diff adds only the test bullet. The placement defect is a side effect of closing the coverage gap in the test section alone.

## Solution approach

Add the depth-walk fast-fail synthesis production rule to V11f's **Adds** field: for the AJV-on-`args` sub-case routed ceiling #4 → ceiling #3 via CIO-1, the `<ajv-summary>` is synthesised from the depth-walk's canonical `ValidationIssue` (`schema_keyword:"maxDepth"`, message `"JSON document depth exceeds 5"`, single-issue form, no `; ` separator) and AJV is not invoked at this site. Cite the existing landmark, [`determinism-cancellation-failure.md` §Failure-class taxonomy](../spec_topics/binder/determinism-cancellation-failure.md#failure-class-taxonomy), rather than re-deriving the spec rule. The existing depth-walk fast-fail Tests bullet may remain as written.

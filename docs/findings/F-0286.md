## Finding

`V11e` carries two unrelated obligations drawn from two different spec documents. Its **Spec** field cites both [`binder/defaulting-system-note-echo.md`](../spec_topics/binder/defaulting-system-note-echo.md) (the five system-note line-discipline rendering rules) and [`binder/determinism-cancellation-failure.md`](../spec_topics/binder/determinism-cancellation-failure.md) §Determinism (the `temperature: 0` contract and the FNV-1a seed derivation). Its **Adds**, **Tests**, and **Ships when** all span both: five rendering-rule assertions plus a single FNV-1a-seed/`temperature:0` assertion under one compound done-condition.

The two surfaces have distinct verification purposes — rendering is about output formatting correctness, while the determinism contract is about reproducible binder inputs (seed-value derivation and call config). They share no test fixtures and no implementation surface. Under the plan's own leaf-sizing rule (`conventions.md` §3: "Each leaf inside a slice is the smallest feature that can ship and be tested independently"), the determinism contract is a separable feature that is presently filed under a leaf named for system-note rendering. The coverage matrix already reflects the split internally — it carries two distinct rows (rendering rules, and the `temperature: 0` + FNV-1a determinism MUSTs), both pointing at `V11e`.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `c6a664e` (`pi-loom plan: build/update plan for spec.md + review`, 2026-06-10)
**History:** The leaf was created in `c6a664e` already titled "Binder system-note rendering and determinism" and already bundling both concerns — both spec topics in **Spec**, both clauses in **Adds**, and a rendering bullet plus a determinism bullet in **Tests**. A later commit `475d018` (2026-06-11) only expanded the rendering surface from two to five rules; it did not touch the determinism bundling. A dedicated de-bundling pass, `cb8737a` (`pi-loom plan: resolve "Systemic leaf over-bundling across the plan corpus"`, 2026-06-12), modified this file but changed only its **Deps** (`V11d` → `V11h`), leaving the rendering+determinism bundling intact. The cross-spec bundling has therefore been present since the leaf was first authored and survived the corpus-wide over-bundling sweep.

## Solution Space

**Shape:** multiple

### Option A — Split the determinism contract into a new leaf

**Approach.** Reduce `V11e` to the five system-note rendering rules only; move the binder determinism contract (`temperature: 0` + FNV-1a seed derivation) into a new leaf and its `-T` partner.

**Plan edits.** In `V11e-system-note-determinism.md`: strike the `determinism-cancellation-failure.md` entry from **Spec**; strike the determinism clause from **Adds** (`and the binder determinism contract (\`temperature:0\`, the FNV-1a seed derivation, fixed messages)`); strike the `determinism-cancellation-failure.md — binder determinism` **Tests** bullet; strike `and the deterministic FNV-1a seed` from **Ships when**. Create a `<new>` leaf (and `<new>-T`) citing `determinism-cancellation-failure.md` §Determinism, carrying the FNV-1a-seed-derivation and `temperature: 0` obligation. In `docs/plan.md` V11 list, rename the `V11e` entry to drop "and determinism" and add the new leaf. Retarget the coverage-matrix row `…§Determinism — \`temperature: 0\` + FNV-1a seed-derivation MUSTs` from `V11e` to the new leaf. Add the new leaf to `H5b`'s **Deps** coverage-producing set per how-to-use step 2.

**Spec edits.** None.

**Pros.** Each leaf is single-concern; coverage rows map 1:1 to a single-purpose leaf.

**Cons.** Spawns a leaf + `-T` pair for one seed assertion; the largest plan churn (new files, `plan.md` list, `H5b` **Deps**).

**Risks.** Omitting the `H5b` **Deps** update for the new coverage-closing leaf; the new leaf is near-minimal, trading one bundling smell for a single-assertion leaf.

### Option B — Fold the determinism contract into `V11f`

**Approach.** Reduce `V11e` to the five rendering rules; move the determinism contract into `V11f`, which already cites `determinism-cancellation-failure.md` (it owns that document's §Failure modes). `V11f` then owns §Determinism and §Failure modes of one spec document, so the *cross-spec* bundling the finding names disappears.

**Plan edits.** In `V11e-system-note-determinism.md`: the same strikes as Option A (the `determinism-cancellation-failure.md` **Spec** entry, the **Adds** determinism clause, the determinism **Tests** bullet, the **Ships when** determinism clause). In `V11f-binder-retry-taxonomy.md`: add the binder determinism contract (`temperature: 0` + FNV-1a seed derivation, including the reference seed vectors) to **Adds**, add a **Tests** bullet asserting the FNV-1a seed reference vectors and `temperature: 0`, and extend **Ships when** to name it; mirror the test into `V11f-T`. In `docs/plan.md` V11 list, rename the `V11e` entry to drop "and determinism". Retarget the coverage-matrix `…§Determinism — \`temperature: 0\` + FNV-1a seed-derivation MUSTs` row from `V11e` to `V11f`.

**Spec edits.** None.

**Pros.** No new leaf; determinism joins a leaf already citing the same spec document, removing the cross-spec bundling; minimal churn; `H5b` coverage-set membership is unchanged because `V11f` is already in it.

**Cons.** `V11f`'s scope grows; determinism (call config) and the retry taxonomy are distinct sub-concerns, though they share one spec document.

**Risks.** `V11f-T` must gain the seed test; the inbound links to `V11e` (from `H7a` and the coverage matrix's rendering row) still resolve because they reference rendering, which stays in `V11e`.

### Recommendation

Take Option B. It eliminates the cross-spec bundling the finding targets — `V11e` becomes single-document (`defaulting-system-note-echo.md`) and determinism rejoins `V11f`, which already owns `determinism-cancellation-failure.md` — without spawning a new leaf for a single assertion or disturbing the `H5b` coverage-producing set. Edge case the implementer must watch: `V9j` independently asserts that the binder *call* carries `temperature: 0` and the mapped seed *field*, while the determinism portion owns the FNV-1a seed-*value* derivation (reference vectors) and the `temperature: 0` contract — keep that derivation-vs-transport boundary intact and do not duplicate the seed-field-mapping assertion in `V11f`.

## Consequence

**Severity:** cosmetic

If shipped unfixed the leaf still implements and tests cleanly, so no implementer is blocked or misled. The cost is purely organisational: one leaf carries two unrelated test surfaces under a single compound ships-when, and the binder determinism contract is filed under a leaf named for system-note rendering rather than alongside the rest of the binder-call contract surface, mildly degrading discoverability and contradicting the one-concern-per-leaf editorial guideline.

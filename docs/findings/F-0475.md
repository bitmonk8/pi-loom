## Problem

`H4b`'s functional-effect self-check (the second **Tests** bullet) asserts that scripted injection-point categories (c), (d), and (f) each yield a specific *observable*, but `H4b` cites no spec contract defining those observable shapes. Category (c) names malformed-envelope and transport-class failures, (d) names a `tool_loop.max_rounds` round-exhaustion, and (f) names a completed-invoke-child outcome тАФ each references only a bare V-leaf vector name (`V11f`, `V13c`, `V4f`), which are downstream plan leaves rather than spec contracts, and none is on `H4b`'s dependency path (`Deps` is `H4a` only). Category (g) alone cites its spec contract (`subagent.md` via `V9i`). An implementer therefore has no spec anchor for those observable shapes and must invent them, risking divergence from the contracts downstream consumers rely on.

## Problem

`H4b`'s **Ships when** gate ("the self-check passes — including the determinism gate and the per-category functional-effect assertions (a)–(e)") is one completion criterion standing in for six independently-failable sub-checks: the determinism gate plus the five functional-effect assertions `(a)`–`(e)` the second **Tests** bullet enumerates. None of the six carries a stable, citable identifier — the bare letters `(a)`–`(e)` are reused for unrelated obligations elsewhere in the corpus, so a letter is meaningful only when qualified by "H4b". The plan already establishes an inline-label convention for exactly this addressability need (the `HC3-a` family, per `conventions.md` §REQ-ID discipline), but `H4b`'s horizontal self-check assertions have no equivalent anchor, leaving no token to cite a specific sub-check across documents.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `fd6169b` (2026-06-12, "resolve 'H4a bundles three independent units into one leaf'") created `H4b` with the bundled, un-labelled Ships-when criterion already in place (originally "(a)–(g)"); `1063255` (2026-06-20, "fix F-0476 — split H4b (a)-(e) from (f)/(g) into new H4c leaf") narrowed the bundle to "(a)–(e)" when (f)/(g) moved to `H4c`, preserving the pattern.
**History:** `git log --follow` over `H4b-response-programming-surface.md` shows the file first appeared at `fd6169b`; `git show fd6169b:` confirms the Ships-when line already bundled the per-category assertions into a single criterion with no sub-IDs. `git log -S 'assertions (a)–(e)'` attributes the current "(a)–(e)" wording solely to the `1063255` split. The defect is therefore a structural property of the leaf since its inception, not introduced by a later targeted edit; the F-0476 split merely re-scoped the same un-addressable bundle.

## Solution approach

Give each independently-failable sub-check in `H4b` a stable, citable label in the leaf body — the five functional-effect assertions `(a)`–`(e)` plus the determinism gate — and have the **Ships when** criterion and the second **Tests** bullet reference those labels instead of the bare letters. Mirror the existing `HC3-a`-style inline-label precedent cited in `conventions.md` §REQ-ID discipline.

## Solution constraints

- The spec is read-only for this fix; the labels live entirely in the `H4b` leaf body.

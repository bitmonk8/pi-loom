## Finding

The PIC-12 `Clock` seam `now()` bullet in `host-interfaces-services.md` states the member is "Used for deadline math by the `SHUTDOWN_AWAIT_CAP_MS` shutdown-await (per the **Extension entry point** `session_shutdown` step above)". That parenthetical is the only pointer this bullet gives to the constant, and it does not resolve from here: there is no "Extension entry point" section in `host-interfaces-services.md` (the file begins at PIC-10), so the word "above" points at nothing in the reader's current file. The constant's value and its sole load-bearing definition (`SHUTDOWN_AWAIT_CAP_MS = 2000`, the bounded `Promise.allSettled` await in sub-step 3) live in a different topic page, `session-shutdown-semantics.md`, with no hyperlink from the PIC-12 bullet to that site.

The other deadline-math caller named in the same bullet — the `looms.scanPackagesTimeoutMs` package-walk bound — is given as a resolving Markdown link to `discovery/package-and-settings.md#disc-6`. The `SHUTDOWN_AWAIT_CAP_MS` caller is the only one of the two carried as bare prose, so the inconsistency is internal to a single bullet: an implementer wiring `Clock.now()` deadline math can follow one pointer and not the other.

## Issue introduction
**Verdict:** single-commit (with a later aggravating refactor)
**Introducing commit:** `9183961` — *pi-loom spec: resolve "RuntimeEvent.occurred_at source clock contradicts the Clock.now() monotonic pin"* (2026-06-04)
**History:** The spec corpus is git-tracked. `git log -S 'SHUTDOWN_AWAIT_CAP_MS'` shows the constant was coined earlier in the session-shutdown semantics by `7f37dd7` (*resolve "Session-swap behaviour for in-flight loom invocations is under-specified"*). The PIC-12 `now()` bullet acquired its reference to that constant at `9183961`: that commit split the seam into `now()` / `wallNow()`, and in doing so rewrote the `now()` bullet from "Used by the **Runtime event channel** above to stamp `RuntimeEvent.occurred_at`…" to "Used for deadline math by the `SHUTDOWN_AWAIT_CAP_MS` shutdown-await (per the **Extension entry point** `session_shutdown` step above)…" (`git show 9183961`). The pointer was prose-only — never a resolving link and never carrying the value — from the moment this bullet text was authored. At that time the bullet lived in the monolithic `pi-integration-contract.md`, where an "Extension entry point" section did sit "above" it in the same file, so "above" was at least intra-file. The later size-cap split `f5e89f4` (*docs: enforce 100KB hard / 30KB recommended size cap on spec set*, 2026-06-04) carved the monolith into per-topic children, moving the `now()` bullet into `host-interfaces-services.md` and the `session_shutdown` definition into `session-shutdown-semantics.md`; that refactor turned the already-prose-only pointer into a cross-file dangling reference and made "above" actively wrong. The originating defect is `9183961`; `f5e89f4` aggravated it.

## Solution Space

**Shape:** single

### Recommendation

In the PIC-12 `now()` bullet of `host-interfaces-services.md`, replace the prose parenthetical "(per the **Extension entry point** `session_shutdown` step above)" with a resolving Markdown link to the constant's definition site — the `session_shutdown` sub-step 3 bounded-await in `session-shutdown-semantics.md` — mirroring the link form the same bullet already uses for the `looms.scanPackagesTimeoutMs` / `#disc-6` caller. This removes the misleading "above" (the target is in a different file) and lets the Clock-seam implementer reach the value (`2000`) and its settle-bounded semantics in one hop.

Edge cases for the implementer:
- Do **not** restate `2000` inline in the PIC-12 bullet. The constant has a single source-of-truth definition; duplicating the literal here introduces a drift surface (the value is also build-time literal-read-asserted at its declaration site). A link, not a copy, is the correct repair.
- Confirm the chosen anchor actually resolves in `session-shutdown-semantics.md` (sub-step 3 is currently un-anchored — GOV-22 residue per the coverage matrix `cka-31` row); if no stable anchor exists, link to the page and add or cite the nearest existing anchor rather than leaving a bare prose pointer.

## Consequence

**Severity:** advisory

An implementer reading the PIC-12 `Clock` seam to wire `now()` deadline math cannot resolve `SHUTDOWN_AWAIT_CAP_MS` from this page: the "step above" pointer dead-ends in-file and there is no link to the definition. A diligent reader recovers the value (`2000`) by searching the corpus; a less thorough one risks inventing a timeout, which would shift the observable `loom/runtime/reload-teardown-timeout` boundary the registry contract depends on. No working system is precluded, but the navigation gap costs search effort and invites divergence.

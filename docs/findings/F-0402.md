## Problem

`V15h` bundles two orthogonal failure modes into one leaf: the ceiling-#4 depth-surface witnesses that wrap a depth-6 `invoke(...)` `params` argument and a depth-6 `invoke<T>` return value into `InvokeInfraError` (resting on `V5e`), and the `invoke` child execution Promise's swallowing-handler attachment plus three-channel late-settlement suppression at the `Checkpoint` seam (resting on `V8a`, with the `H4b` harness). These concerns have disjoint dependency sets and disjoint spec anchors (`hard-ceilings/ceilings-3-and-4.md` vs `cancellation.md`), sharing only the `invoke` call site. The parallel tool-call site already keeps them separate as `V14e` (depth) and `V14f` (swallowing), making `V15h` the only one of the four swallowing-handler routing sites (`V14f`, `V13f`, `V15h`, `V9o`) that also carries an unrelated depth-surface obligation.

There is no earlier revision in which the two concerns were separate; the bundling entered the corpus the moment the leaf was created.

## Solution approach

Split `V15h` so the ceiling-#4 depth-surface half moves to a new vertical leaf (with its paired `-T` test leaf), leaving `V15h` as the `invoke`-child swallowing-handler leaf only — mirroring the `V14e` / `V14f` split already at the tool-call site. Remove the depth obligation and the `V5e` dependency from `V15h`, and give the new leaf the `V5e` dependency that the depth half rests on. Add the new leaf to the `V15 — Invocation and imports` list in `docs/plan.md` and extend the `V15a`–`V15h` Deps range in `docs/plan_topics/H5b-warn-only-canary.md` to include it.

## Solution constraints

- Only the depth obligation moves: the residual `V15h` stays the swallowing-handler owner, so the four-site routing-set references (`V13f`, `V14f`, `V9o`, `V17a`) and the swallowing-handler row in `coverage-matrix.md` stay pointed at `V15h` and are out of scope for this fix.

## Problem

`V4f` delegates the live-surface re-assertion of completed-callee finality (`ERR-13`) to `V15a`, `V13c`, and `V14a` as named delegated-live-carrier witnesses, and each of those implementation leaves carries an `ERR-13` witness bullet in its **Tests** section. But that witness is absent from the paired `V15a-T`, `V13c-T`, and `V14a-T` tests-tasks and from all three implementation leaves' **Ships when** clauses. Under the per-phase TDD ritual (`conventions.md` §"MVP and vertical features — tests-task → implementation-task pairing"), a Tests bullet that lives only on the implementation leaf — with no paired `-T` red-first failing test and no `Ships when` criterion — is verified by nothing, so `V4f`'s delegated obligation ships unenforced.

## Issue introduction

**Verdict:** introduced-by-commit (asymmetry never subsequently corrected for the `ERR-13` carriers)
**Introducing commit:** `7bd65a7` (`pi-loom plan: resolve "V4c/V4f defer ERR-8 to ERR-13 live-surface confirmation to downstream owners that never re-assert it"`, 2026-06-18)
**History:** `git log -S 'ERR-13\` (delegated live-carrier witness' -- docs/plan_topics/` returns the single commit `7bd65a7`, whose `--stat` shows it added the `ERR-13` witness bullet to `V13c`, `V14a`, and `V15a` (and the `ERR-8` bullet to `V9i`) but touched none of the `-T` leaves and added no `Ships when` clause (`git show 7bd65a7` confirms the diff appends only the impl-leaf bullet and a `V4f` Deps edge). The sibling `ERR-8` carrier was later completed in two follow-up edits — `f892019` (`fix F-1243 — add ERR-8 red-first witness to V9i-T + Ships-when gate`) and `c768005` (`fix F-1363 — relocate V9i ERR-8 real-session witness to smoke gate`) — bringing `V9i`/`V9i-T` into the paired shape. `git log -S 'ERR-13' -- docs/plan_topics/V15a-T-invocation-core.md docs/plan_topics/V13c-T-query-tool-loop.md docs/plan_topics/V14a-T-tool-calls.md` returns no commits: the `ERR-13` carriers never received the analogous `-T` + `Ships when` follow-up, so the defect is the un-finished half of the `7bd65a7` change rather than a later regression.

## Solution approach

Restore the tests-task ↔ implementation-task pairing for the three `ERR-13` delegated-live-carrier witnesses, following the established `V9i`/`V9i-T` (`ERR-8`) treatment: add an `ERR-13` witness **Tests** bullet to `V15a-T`, `V13c-T`, and `V14a-T` mirroring the bullet already on each paired implementation leaf, add `V4f` to each `-T` leaf's `Deps.`, and add an `ERR-13` completed-callee-finality clause to each implementation leaf's `Ships when`. Treat the `ERR-13` witness as `npm test`-assertable (a red-first `-T` assertion and an `npm test` `Ships when` clause), not real-host-only — these live surfaces are loom-runtime-internal, so do not carry over the `ERR-8` manual-smoke-gate caveat; give a site real-host-only treatment only if its live-surface re-assertion turns out to require a real host.

## Solution constraints

- Out of scope: removing the `ERR-13` witness bullets from the `V15a`/`V13c`/`V14a` implementation leaves — `V4f` names those leaves as its delegated live-carriers, so striking the bullets would orphan `V4f`'s delegation.

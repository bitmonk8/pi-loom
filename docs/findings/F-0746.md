## Finding

The Narrowing paragraph states that, during a patch-skew window in which a Pi
patch has dropped a `SessionShutdownEvent['reason']` literal, "the snapshot's
`literals` array is transiently superset to the SDK union." The construction
"superset to X" is not English; the natural parse — "is [a] superset [, relative]
to the SDK union" — reads the SDK union as the larger set containing the
snapshot, which is the **opposite** of the intended relation. The snapshot is
the strict superset (it still pins the removed literal); the SDK union is the
subset. Context elsewhere in the same paragraph confirms the intent: the next
sentence pins "the SDK-union side becomes the strict subset," and the (P-c-1)
fixture description, two paragraphs later, correctly says "`literals` array is
a strict superset of the SDK union."

The same malformed phrasing recurs in the Mixed sub-rule's narrowing-half
clause: "the snapshot's `literals` array is transiently superset and the stale
entry is harmless until the snapshot edit ships." That second occurrence is the
same defect (the trailing "to the SDK union" is elided but the construction is
identically broken) and any fix to the Narrowing paragraph must repair the
Mixed sub-rule's clause in the same edit, or the corpus will end up with two
divergent phrasings of the same relation.

## Solution Space

**Shape:** single

### Recommendation

Replace both occurrences of the malformed construction in
`docs/spec_topics/pi-integration-contract/patch-skew-degradation.md` in one
edit, restoring the correct set-relation direction:

1. In the **Narrowing** paragraph, change
   "the snapshot's `literals` array is transiently superset to the SDK union"
   to
   "the snapshot's `literals` array is transiently a superset of the SDK union".
2. In the **Mixed** sub-rule's narrowing-half clause, change
   "the snapshot's `literals` array is transiently superset and the stale entry is harmless until the snapshot edit ships"
   to
   "the snapshot's `literals` array is transiently a superset of the SDK union and the stale entry is harmless until the snapshot edit ships".

Edge cases for the fixer to watch:

- Use the exact wording "a superset of the SDK union" in both sites so the
  paragraph reads consistently with the existing "`literals` array is a strict
  superset of the SDK union" phrasing in the (P-c-1) fixture description, which
  already gets the relation right and is the local reference style.
- Do **not** touch the immediately-following sentence "the SDK-union side
  becomes the strict subset and the right-to-left direction breaks" — it is
  already correct and load-bearing for the build-time-gate description.
- No anchors, no pinned substrings, and no cross-references in other shards
  quote these two clauses (grep confirms), so the edit is local to the file
  and does not propagate.

## Consequence

**Severity:** cosmetic

The asymmetric-skew property is fully recoverable from the surrounding prose
(the immediately following sentence and the (P-c-1) fixture description both
state the relation correctly), so an attentive implementer reaches the right
behaviour and no observer-visible outcome diverges. The defect is a localised
prose error that degrades only the readability and self-evidence of the
Narrowing and Mixed paragraphs.

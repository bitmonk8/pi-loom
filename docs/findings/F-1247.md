## Finding

`docs/plan.md` §How to use this plan, step 2 instructs maintainers to add a newly-authored leaf to `H5b`'s **Deps** when "the new leaf's `Tests.` bullets carry an executable REQ-ID, a citing test that *closes* a coverage-matrix-mapped numbered REQ-ID, or an un-anchored normative MUST," and asserts that `conventions.md` §REQ-ID discipline (*Transitive-completeness plan-maintenance*) "states that obligation once and governs its exact triggers."

The authoritative rule it points to defines exactly two triggers, not three. `conventions.md` §REQ-ID discipline *Transitive-completeness plan-maintenance* fires when a new leaf carries (1) "the citing test that *closes* a [`coverage-matrix.md`]-mapped numbered REQ-ID" — and it explicitly excludes re-citations: "the trigger turns on the coverage-matrix closing-leaf mapping, not on whether the citation is the REQ-ID's first textual occurrence, so a leaf that merely re-cites an already-closed REQ-ID does not qualify" — or (2) "an un-anchored normative MUST."

Step 2's first disjunct, "carry an executable REQ-ID," has no counterpart in the authoritative rule and is strictly broader: any leaf whose tests cite an executable REQ-ID — including a re-citing leaf that `conventions.md` deliberately excludes — satisfies it. The summary therefore presents two incompatible predicates for the same obligation in a single sentence that simultaneously claims `conventions.md` "governs its exact triggers."

The later edit `ef9d7a7` (F-0432, 2026-06-20) touched the same sentence — changing the verb from "can introduce" to "carry" — but preserved the broad first disjunct and so did not fix the defect. ## Solution Space

**Shape:** single

### Recommendation

In `docs/plan.md` §How to use this plan, step 2, strike the first disjunct so the trigger condition mirrors the two predicates `conventions.md` defines. Replace the current sentence:

> If the new leaf's `Tests.` bullets carry an executable REQ-ID, a citing test that *closes* a coverage-matrix-mapped numbered REQ-ID, or an un-anchored normative MUST, add it to [`H5b`](./plan_topics/H5b-warn-only-canary.md)'s **Deps** per [`conventions.md`](./plan_topics/conventions.md) §REQ-ID discipline (*Transitive-completeness plan-maintenance*), which states that obligation once and governs its exact triggers.

with text that carries only the two authoritative triggers and the re-citation exclusion, e.g.:

> If the new leaf's `Tests.` bullets carry the citing test that *closes* a coverage-matrix-mapped numbered REQ-ID (a leaf that merely re-cites an already-closed REQ-ID does not qualify), or an un-anchored normative MUST, add it to [`H5b`](./plan_topics/H5b-warn-only-canary.md)'s **Deps** per [`conventions.md`](./plan_topics/conventions.md) §REQ-ID discipline (*Transitive-completeness plan-maintenance*), which states that obligation once and governs its exact triggers.

`conventions.md` is the authoritative source for the exact trigger and stays read-only for this fix; the spec is not touched. The literal closing-leaf-mapping predicate and the re-citation exclusion above are taken verbatim from `conventions.md` so the summary and the authoritative rule agree.

## Consequence

**Severity:** correctness

A maintainer following step 2's broad first disjunct would add re-citing leaves to `H5b`'s `Deps` that `conventions.md` excludes, while a maintainer trusting the "governs its exact triggers" pointer follows the narrower `conventions.md` rule — two reasonable maintainers diverge on `H5b`'s `Deps` membership. The `H5d` transitive-completeness gate is per-cell-at-least-one, so an over-inclusive `Deps` does not redden it, but the contradiction defeats step 2's purpose as a faithful one-line summary of the obligation.

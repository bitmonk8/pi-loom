## Problem

`docs/plan.md` §How to use this plan, step 2 lists three disjuncts for when a new leaf must be added to `H5b`'s **Deps** — "carry an executable REQ-ID, a citing test that *closes* a coverage-matrix-mapped numbered REQ-ID, or an un-anchored normative MUST" — while asserting that `conventions.md` §REQ-ID discipline (*Transitive-completeness plan-maintenance*) "governs its exact triggers." The authoritative rule defines only two triggers: the citing test that closes a coverage-matrix-mapped numbered REQ-ID (explicitly excluding re-citations of an already-closed REQ-ID) and an un-anchored normative MUST. Step 2's first disjunct, "carry an executable REQ-ID," has no counterpart and is strictly broader, capturing re-citing leaves that `conventions.md` deliberately excludes. The sentence therefore presents two incompatible predicates for the same obligation while claiming `conventions.md` governs the exact triggers.

## Issue introduction

**Verdict:** introduced by a specific commit (`ab8e297`), with multi-commit context
**Introducing commit:** `ab8e297` — "pi-loom plan: resolve 'Transitive-completeness obligation invisible at leaf authoring'" (2026-06-11)
**History:** `ab8e297` first authored step 2's three-disjunct trigger restatement in `docs/plan.md`, including the over-broad first disjunct ("can introduce an executable REQ-ID"). Six minutes earlier the same day, `d7173b3` ("pi-loom plan: resolve 'Transitive-completeness rule trigger broader than H5b coverage set'") had narrowed the authoritative trigger in `docs/plan_topics/conventions.md` by adding the exclusion "a leaf that merely re-cites an already-closed REQ-ID does not qualify." `ab8e297`'s plan.md restatement did not mirror that exclusion, crystallising the divergence between step 2 and conventions.md the moment it landed. The later edit `ef9d7a7` (F-0432, 2026-06-20) touched the same sentence — changing the verb from "can introduce" to "carry" — but preserved the broad first disjunct and so did not fix the defect. (Established via `git log -S`/`git show` over `docs/plan.md` and `docs/plan_topics/conventions.md`.)

## Solution approach

In `docs/plan.md` §How to use this plan, step 2, strike the first disjunct ("carry an executable REQ-ID") so the trigger condition lists only the two predicates `conventions.md` §REQ-ID discipline (*Transitive-completeness plan-maintenance*) defines: the citing test that closes a coverage-matrix-mapped numbered REQ-ID, and an un-anchored normative MUST. Carry through the re-citation exclusion so a leaf that merely re-cites an already-closed REQ-ID does not qualify, matching the authoritative rule the sentence points to.

## Solution constraints

- `conventions.md` is the authoritative source for the exact trigger and stays read-only for this fix; the spec is not touched.

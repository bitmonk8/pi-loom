## Problem

The *Wire serialisation* clause of `audit-wire-and-canary.md` pins the audit record format to the byte but never pins the *form* of the emitted `path` field value — whether it is repo-root-relative or absolute, and whether directory separators are normalised to POSIX `/` or left host-native. The clause sends contradictory signals: the deterministic walk order is described as "lexicographic on POSIX-form path" (implying a POSIX-normalised basis), while the segment-(1) right-most-`:` split is justified by accommodating "Windows hosts whose `path` field may itself contain a `:` in the drive prefix" (which only holds for a native absolute Windows path). Because the form is unpinned, two conforming implementations on different hosts or checkout roots emit different `path` tokens for the same source line. That defeats the wire contract's purpose of letting a cross-host CI parser route, deduplicate, and locate records, across every record class that carries `path`.

## Issue introduction
**Verdict:** present-since-inception
**Introducing commits:** `e5cd514` (2026-05-09, "pi-loom spec: resolve \"T35 SDK capability inventory closed-set negative-direction audit\"")
**History:** `git log -S 'drive prefix'`, `git log -S 'lexicographic on POSIX-form path'`, and `git log -G 'POSIX-form path'` over the spec corpus all converge on `e5cd514` as the commit that first authored the *Wire serialisation* clause, including both the "lexicographic on POSIX-form path" iteration-order example and the right-most-`:` segment split with its Windows drive-prefix accommodation. The clause has carried both signals — and the unpinned emitted-`path` form between them — since that authoring. The only later commits touching the file (`f5e89f4`, the 100KB/30KB size-cap split; `81bf52a`, the `spec/` → `spec_topics/` collapse) relocated the text without pinning the form. The defect is original to the clause, not introduced by a later edit.

## Solution approach

Clarify the *Wire serialisation* clause in `audit-wire-and-canary.md` to pin the emitted `path` field as repo-root-relative with POSIX (`/`) separators, normalised before emission, relative to the `src/**/*.ts` walk base defined at `audit-resolution.md`'s `id="audit-scope"`. Reconcile the right-most-`:` split's drive-prefix justification in the same clause, since a repo-root-relative POSIX path carries no drive-prefix colon. Make the pinned form normative rather than illustrative.

## Finding

The glossary entry for `` `.warp` file (library module) `` in `docs/spec_topics/glossary.md` opens with "A library file containing only top-level `schema`, `enum`, and `fn` declarations…", enumerating three permitted top-level forms. The normative pages enumerate five: `imports.md` says "Top-level may contain only `import`, `export`, `schema`, `enum`, and `fn` declarations" (and pins `loom/parse/warp-top-level-statement` to that list), and `overview-and-orientation.md` repeats the same set parenthetically as "currently five: `import`, `export`, `fn`, `schema`, `enum`".

The glossary entry's surrounding clause does mention `import` ("importable from `.loom` and `.warp` files via `import`"), but the "containing only …" clause is the entry's structural statement of what may appear at the top level — and as written it omits both `import` and `export`. An author reading the glossary first would write a `.warp` whose `import` and `export` lines they expect to fail to parse, then be confused by the normative pages.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `d54b426` "pi-loom spec: resolve \"Glossary cluster: six independent defects in spec_topics/glossary.md\"" (2026-05-07)
**History:** The `` `.warp` file (library module) `` glossary entry was added in `d54b426` already in its current 3-form shape ("containing only top-level `schema`, `enum`, and `fn` declarations"). The 5-form normative list in `imports.md` had been in place since the prior commit `b0b9a14` (2026-05-05, "Plan V17b permits enum in .warp files; spec .warp allowlist omits it") which raised the imports.md allowlist from 4 forms to 5 by adding `enum`. The glossary entry was authored two days after the normative page was already at 5 forms, so the under-enumeration is a defect present from the entry's inception rather than introduced by a later drift. No subsequent commit touches the entry's permitted-forms clause.

## Solution Space

**Shape:** single

### Recommendation

Reword the glossary entry's opening clause to defer to the normative page rather than re-enumerating the list. Replace "A library file containing only top-level `schema`, `enum`, and `fn` declarations, importable from `.loom` and `.warp` files via `import`, …" with wording of the form "A library file whose top level is restricted to the declaration forms enumerated in [Imports — Permitted top-level forms](./imports.md#permitted-top-level-forms), importable from `.loom` and `.warp` files via `import`, …". Deferring (rather than restating "the five forms `import`, `export`, `schema`, `enum`, and `fn`") keeps the glossary out of the maintenance path the next time the allowlist changes; `overview-and-orientation.md` already uses this same defer-by-anchor pattern (its parenthetical "*orientation aggregator per …*") and is the model to mirror. The rest of the entry (pairing with `.loom`, byte-exact lowercase extension matching, See-links) stays unchanged.

## Consequence

**Severity:** advisory

A glossary reader could misjudge which top-level forms a `.warp` accepts and propose a smaller allowlist than the normative pages prescribe. The defect cannot reach an implementation because `V15c-imports.md`'s `Tests` are written against the imports.md list, not the glossary; the cost is reader friction and one round of "which page wins" reconciliation per affected reader.

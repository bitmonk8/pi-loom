## Problem

The authoritative contract for the audit's `proposed-resolution` field is split across two pages that frame the field differently. The lex-min selection sentence in `audit-resolution.md` (`#audit-category-2-inventory-join-key`) describes the field as a determinism tie-break — the entry with the lexicographically-smallest `path` — while the *Per-family record-shape table* in `audit-failures.md` (`#audit-per-family-record-shape-table`) describes it as a per-family suggestion value (inventory-entry / recognised-shape / deletion-or-rewrite suggestion). The table's "carries four fixed fields … value outside the per-family range" intro is not reconciled with the lex-min sentence, so no single statement authoritatively governs the field.

## Solution approach

Designate one statement as the authoritative owner of the `proposed-resolution` field's value contract and rewrite the other to cross-reference it, so the determinism-tie-break framing and the per-family-suggestion framing describe a single field consistently. Add a forward-link between the lex-min sentence in the *Category-(2) inventory join key* (`#audit-category-2-inventory-join-key`) and the *Per-family record-shape table* (`#audit-per-family-record-shape-table`) in whichever direction the chosen owner dictates.

## Problem

Version-bump output (c) in `version-bump-triggers.md` — the merge-blocking runtime-evidence acceptance gate — requires "a *representative* integrated `.loom`" exercised through the loom's end-to-end harness, driving six named surfaces (typed query, tool-call loop, `invoke(...)`, schema validation, the binder, and cancellation) in one run. The six-surface enumeration is the operative requirement, but the qualifier "representative" adds a selection judgement the spec never defines: representative *of what*, and selected *how*. Reading the spec alone, an implementer cannot tell whether the gate accepts any `.loom` exercising the six surfaces, one mirroring real user scenarios, or a dedicated committed integration fixture. The qualifier is therefore either redundant or ambiguous.

## Issue introduction
**Verdict:** single-commit
**Introducing commits:** `81ab342` (2026-06-10, "pi-loom plan: resolve \"version-bump runtime-evidence acceptance gate and revert path\"")
**History:** `git log -S "representative integrated" -- docs/spec_topics/pi-integration-contract/version-bump-triggers.md` returns a single commit, `81ab342`; `git show 81ab342^:docs/spec_topics/pi-integration-contract/version-bump-triggers.md` contains zero occurrences of "representative" and no output (c) at all (the parent enumerated only outputs (a) and (b)). Output (c) and its "a representative integrated `.loom`" phrasing were therefore added wholesale in one edit, with the undefined qualifier present from the inception of the clause rather than drifting in later. The bare phrase had been coined earlier on the plan side (first appearing corpus-wide in `c6a664e`), but the spec-side defect — the qualifier landing in normative spec text without an accompanying definition — entered at `81ab342`.

## Solution approach

Rewrite output (c)'s fixture-selection clause in `version-bump-triggers.md` so the spec self-sufficiently names which `.loom` the gate runs. Either delete the qualifier "representative" and rest the obligation on the explicit six-surface enumeration as the sole selection criterion, or state that the gate drives a single committed integration fixture exercising those six surfaces in one run. The operative selector must not remain an unqualified judgement word ("representative", "typical", "real-world").

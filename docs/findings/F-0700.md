## Problem

The `system:` *Path grammar* block's `.Ident` reachability clause requires each subsequent step to "name a reachable field of an *object* schema in the lowered params schema" — a wire-keyed namespace — but the head `Ident` byte-match rule and the stated resolution target ("the validated params object") both use loom-side `params` names. For any object-typed param carrying an `as "WireName"` rename the two namespaces contradict on the same path, so under the literal text neither the loom-side nor the wire spelling is well-defined and `loom/parse/system-interp-bad-field` fires inconsistently. This also clashes with the global invariant in `runtime-value-model.md` that loom code never sees wire names, and with the object-navigation rules in `expressions.md`.

## Issue introduction

**Verdict:** single-commit

**Introducing commits:** `428fcb7` (2026-05-04, "pi-loom spec: resolve \"system: interpolation: path grammar, stringification, and parse errors unspecified\"")

**History:** `git log -S 'in the lowered params schema' -- docs/ spec_topics/` resolves to exactly two commits — `428fcb7` (the original authoring of the `system:` *Path grammar* block in `spec_topics/frontmatter.md`) and `f5e89f4` (the later 100KB/30KB shard split that moved the prose into `docs/spec_topics/frontmatter/frontmatter-fields-b-and-templates.md` unchanged); the intermediate `31ff060` ("Move spec, plan, and review docs into docs/") rehomed the file without touching the sentence. The defect is therefore the single phrase "in the lowered params schema" added in `428fcb7` inside an otherwise loom-side-keyed paragraph (the head-`Ident` byte-match rule and the "validated params object" resolution target in the same paragraph were both authored loom-side in the same commit). No earlier commit carried any version of the *Path grammar* block — `git log --follow` over the file pre-`428fcb7` returns no hits for the sentence or its components.

## Solution approach

Rewrite the `.Ident` reachability sentence in `frontmatter-fields-b-and-templates.md`'s *Path grammar* paragraph to gate each subsequent `.Ident` on the loom-side `params:` declared object schema, dropping the "in the lowered params schema" clause. Clarify that `${param.field}` paths use loom-side names throughout, never the `as "WireName"` rename target, consistent with the `runtime-value-model.md` invariant and the `expressions.md` object-indexing rules. Align the *Cause* cell of the `loom/parse/system-interp-bad-field` row in `code-registry-parse.md` to the same loom-side reachability wording.

## Problem

The `cka-31` Tests bullet — appearing identically in `V9g` and `V9g-T` under `patch-skew-degradation.md` §`session_shutdown` sub-step 3 — restates the V9g `Adds` mechanics ("awaits every in-flight entry's `disposeBarrier` to settle via `Promise.allSettled`, bounded by `SHUTDOWN_AWAIT_CAP_MS`") with the assertion frame dropped. It names the internal call and control flow rather than an observable outcome a test runner can check, unlike every sibling bullet in the same Tests list, which each carry an assertion frame. The white-box phrasing leaves a TDD author free to assert the internal mechanism (that `Promise.allSettled` is the call used) rather than the externally observable behaviour the `cka-31` code-keyed obligation governs.

## Issue introduction

**Verdict:** defect-introduced-by-specific-commit
**Introducing commits:** `659aa21` (2026-06-11 — "pi-loom plan: resolve \"Promise.allSettled aggregate-await ownership credited to V9e but call site in V9g\"")
**History:** The `cka-31` Tests bullet was first added to both `V9g` and `V9g-T` at `659aa21` already in its impl-describing form ("sub-step 3 awaits every in-flight entry's `disposeBarrier` to settle via `Promise.allSettled`, bounded by `SHUTDOWN_AWAIT_CAP_MS`"). Commit `7259777` (2026-06-18) only re-labelled the citation parenthetical from "(PIC code-keyed area)" to "(code-keyed-area token `cka-31`)" and did not touch the assertion frame. The defect has therefore been present since the bullet's inception.

## Solution approach

Rewrite the `cka-31` Tests bullet in both `V9g` and `V9g-T` so it asserts an observable outcome — that every in-flight entry's `disposeBarrier` is observed to settle before sub-step 3 returns, and that sub-step 3 returns no earlier than the last entry settles or `SHUTDOWN_AWAIT_CAP_MS` elapses — rather than the `Promise.allSettled` call. Keep the citation parenthetical (`code-keyed-area token cka-31`) and the cap constant; leave the `Promise.allSettled` / await-cap mechanics where they already live, in V9g's `Adds`.

## Solution constraints

- The paired `-T`/impl Tests bullets in `V9g` and `V9g-T` must remain byte-identical after the rewrite.
- Do not assert cap-fire behaviour that contradicts the sibling `loom/runtime/reload-teardown-timeout` "fires at the cap" bullet.

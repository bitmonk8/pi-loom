## Finding

`V15h` (and its paired `V15h-T`) is the `invoke`-child entry in the four-site swallowing-handler routing set that `V17a` (cancellation core) delegates to its owning leaves (`V14f`, `V13f`, `V15h`, `V9o`). Both leaves' Tests assert that a late settlement landed at the `Checkpoint` seam (`V8a`) **after the checkpoint has surfaced `cause: "cancelled"`** is suppressed along all three side channels. The `cause: "cancelled"` surfacing and the swallowing-handler substrate at the `Checkpoint` seam are `V17a` artifacts — `V17a` owns the seam-level three-channel suppression that these per-site routing leaves witness against. The leaf's behaviour therefore directly consumes `V17a`.

Yet `V15h.Deps = V15h-T, V15a, V8a` and `V15h-T.Deps = V15a, V8a` — neither lists `V17a`. The dependency is satisfied only incidentally through the chain `V15h → V15a → V4f → V17a` (`V4f.Deps` lists `V17a` directly; the same chain also reaches `V17a` via `V4f → V17c → V17a`). Among the four routing siblings, only `V9o` declares `V17a` in its Deps (`V9o-T, V9i, V17a, V8a, H4c`); `V13f` and `V14f` share the same omission as `V15h`, so the routing set is asymmetric about an edge all four leaves semantically depend on.

Because pickability is evaluated solely against each entry's declared `Deps` completion tags (How-to-use step 3 / conventions §Leaf format **Deps**), the declared Deps of `V15h`/`V15h-T` understate the leaf's real prerequisite. The DAG is acyclic and currently sequences correctly, but the binding edge is latent in the `V15a → V4f` link rather than declared where the consumption occurs.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `5f4d271` (2026-06-17)
**History:** `V15h` was created in `5f4d271` ("pi-loom plan: resolve \"V15a bundles ~14 independent test obligations into one impl+T pair\"") as a delegated witness peeled off `V15a`, with `Deps. V15h-T, V15a, V5e, V8a` — `V17a` was never listed, even though the same commit's Adds and Tests already routed the `invoke`-child Promise through `V17a`'s `Checkpoint`-seam swallowing substrate and asserted `cause: "cancelled"` suppression. `323d361` (2026-06-18, "resolve \"V15h bundles two orthogonal failure modes\"") later narrowed the leaf and removed `V5e`, leaving `Deps. V15h-T, V15a, V8a` — still without `V17a`. `V15h-T` has likewise never listed `V17a`. The omission has therefore been present since the leaf's first commit; no later edit introduced it.

## Solution Space

**Shape:** single

### Recommendation

Add `V17a` to the **Deps** field of both `V15h` and `V15h-T`, matching the sibling routing leaf `V9o`, which already declares `V17a` directly. No re-sequencing results: `V17a` is already complete before `V15h` is pickable through the existing `V15a → V4f → V17a` chain, so the edit only makes the existing semantic prerequisite explicit and robust against future restructuring of that chain.

- In `docs/plan_topics/V15h-invoke-ceiling-swallowing.md`, the **Deps** line currently reads `V15h-T`, `V15a`, `V8a`; add `V17a` to it.
- In `docs/plan_topics/V15h-T-invoke-ceiling-swallowing.md`, the **Deps** line currently reads `V15a`, `V8a`; add `V17a` to it.

Cite the leaf ID exactly as `V17a` (backtick-delimited, matching the surrounding entries). The sibling leaves `V13f` and `V14f` share the same omission, but resolving them is out of scope for this finding.

## Consequence

**Severity:** advisory

The dependency is presently satisfied transitively, so no implementer is currently misled into building `V15h` before `V17a`'s substrate exists, and the DAG sequences correctly today. The defect is latent: the declared Deps do not capture the semantic prerequisite the leaf's own tests exercise, so any later restructuring of the `V15a → V4f → V17a` chain could silently make `V15h`/`V15h-T` pickable before `V17a`'s swallowing substrate lands, at which point their `cause: "cancelled"` suppression assertions would have no substrate to witness.

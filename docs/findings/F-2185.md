## Problem

The **"No concurrent user sessions in the same host process"** bullet under `## loom 1.0 non-goals` in `model-changes-and-non-goals.md` is a single ~720-word paragraph that restates one detection-gap scenario three times with slightly different framing, including an explicit self-reference ("is the case already carved out earlier in this bullet"), and re-derives the same closed-set-validator behaviour twice. Its actual normative content is small and arrives only at the end: loom 1.0 presupposes a single active user session, an unanticipated second session is itself out of loom 1.0 scope, and the runtime defines no new disposition for it. The repetition buries the lone disposition an implementer must honour and inflates a fringe non-goal bullet to the weight of a core normative section.

## Issue introduction
**Verdict:** present-since-inception
**Introducing commits:** `0e12140` — *pi-loom spec: resolve "Non-goals are not consolidated into a single section"* (2026-05-08)
**History:** The corpus is git-tracked. The `git log -S 'openParallelSession'` and `git log -S 'already carved out earlier in this bullet'` traces both bottom out at `0e12140`, the commit that consolidated the loom 1.0 non-goals (into the then-single `future-considerations.md`, later split into `future-considerations/model-changes-and-non-goals.md` at `f5e89f4`). At `0e12140` the bullet was already 601 words and already carried the self-referential restatement language, so the multi-fold-restatement defect was born with the bullet rather than accreting later; subsequent edits grew it to its current ~720 words but did not introduce the defect.

## Solution approach

Rewrite the bullet under `## loom 1.0 non-goals` so it states the normative scope (loom 1.0 presupposes a single active user session per extension instance and exposes no workaround), the normative disposition (an unanticipated second session is itself out of loom 1.0 scope and the runtime defines no new disposition — no refuse-to-load, no bind-to-first-session, no per-session registry keying, no host-incompatibility diagnostic — with every already-single-session-scoped site remaining so scoped), and a single detection-gap summary, then delete the repeated scenario re-framings, the self-referential "already carved out earlier" sentences, and the duplicate closed-set-validator walk-through.

## Problem

The `cka-33` facet leaves `V17a`, `V14f`, `V13f`, `V15h`, and `V9o` each cite the `cancellation.md` *Race semantics — swallowing-handler attachment on every abandonable Promise* prose in their `Tests` bullet without naming the `cka-33` token or that leaf's own leaf-ID inline, so none carries the facet-naming citing test `H5f` requires. `H5f` requires every facet leaf of a multi-leaf coverage-matrix row to carry a citing test naming both the row's `cka-<n>` token and that facet's closing-leaf-ID inline. That gate becomes binding at the loom 1.0 release gate activated by `H6a`, where it then reddens once per affected facet until each facet's citing test is conformant.

## Issue introduction

**Verdict:** introduced-by-later-edit
**Introducing commit:** `26a8794` (Thu Jun 18 2026, Thomas Andersen) — "pi-loom plan: resolve 'split coverage-matrix rows can ship a facet untested'"
**History:** The facet leaves' `Tests` bullets have carried prose area labels ("(QRY code-keyed area)", "(IMPL area)", "cancellation.md — *Race semantics…*") since the initial plan build (`c6a664e`); the `cka-<n>` tokens were minted later (`7259777`, "code-keyed obligation areas lack a canonical citable token"). Commit `26a8794` introduced **both** the `coverage-matrix.md` *Multi-leaf-row per-facet citing tests* convention and the `H5f` gate arm that enforce exactly that inline citation; its diff touched only `H5f-per-facet-citing-test-gate.md`, `conventions.md`, `coverage-matrix.md`, `H5a`, `H5b`, and `H6a` — it retrofitted none of the `cka-12` / `cka-33` / `cka-44` facet leaves' `Tests` bullets. The facet bullets became non-conforming at that commit.

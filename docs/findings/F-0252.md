## Problem

`V5d`'s Adds claims the schema-subset reject gate "accepts the permitted subset and rejects everything else" — a universal, reject-by-default (allowlist) property — but the paired Tests bullet only witnesses rejection "for each rejected JSON-Schema keyword", i.e. the enumerated unsupported set. The same per-keyword framing recurs in `V5d`'s Ships-when and in `V5d-T`'s Tests bullet. The plan never states whether the gate is an allowlist (reject anything outside the permitted subset) or a denylist (reject only the enumerated keywords), and the two diverge on any construct that is neither in the permitted subset nor explicitly enumerated. A denylist build would pass every stated test while violating the "rejects everything else" claim. `schema-subset.md` defines "a fixed, loom-defined subset", so the allowlist reading is the spec-faithful one — the plan simply fails to pin it.

**History:** The plan corpus is git-tracked. `V5d-subset-lowering.md` was created in c6a664e with an Adds that enumerated specific rejections ("reject pattern/format/min*/max*") and a Tests bullet already framed per-keyword ("fires the subset-violation codes for each rejected keyword and accepts the permitted subset") — so the allowlist/denylist ambiguity was latent from creation. `git log -S "rejects everything else"` and `git show cb8737a` show the universal allowlist-implying clause ("that accepts the permitted subset and rejects everything else") was introduced in cb8737a while the Tests/Ships-when bullets retained the per-keyword framing, sharpening the latent ambiguity into a visible contradiction the per-keyword test cannot witness.

## Solution approach

In `V5d`'s Adds, pin the gate as allowlist / reject-by-default — accepting only the permitted subset and rejecting any construct outside it with `loom/parse/unsupported-feature` — matching `schema-subset.md`'s "fixed, loom-defined subset" framing. In the Tests bullet of both `V5d-subset-lowering.md` and `V5d-T-subset-lowering.md`, add a witness that a construct not present in the enumerated reject list is still rejected, distinguishing allowlist from denylist. Resequence `V5d`'s Ships-when so its assertion mirrors the revised Tests.

## Solution constraints

- The spec `schema-subset.md` is read-only for this fix; it already defines the fixed subset and requires no edit.

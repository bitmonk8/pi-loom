## Problem

The retired `cka-12` row in the *Code-keyed obligation areas* table of `coverage-matrix.md` carries the closing-leaf cell `*(numbered above)*`, which is neither a backtick-delimited leaf-ID nor the literal `<new>` placeholder the *Closing-leaf column authoring* rule recognises as an empty cell. The `H5d` transitive-completeness gate tokenises the cell to the empty set and reddens CI vacuously on that row. Because `H5d` reads ever-present plan files, the spurious failure fires the moment the gate's transitive-completeness arm runs against the real matrix, blocking merges. The row owns no remaining code-keyed obligation: the `query/` subtree is fully covered by the numbered `QRY-1 â€¦ QRY-21` rows in the *Numbered REQ-IDs* table.

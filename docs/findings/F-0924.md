## Finding

The release-gate automation artifact has one canonical name, **closing gate**: it appears in `H5a`'s heading ("REQ-ID / diagnostic-code closing-gate automation"), in `H5a`'s Ships-when ("runs the closing gate green"), in every sibling-arm heading (`H5c`/`H5d`/`H5e`/`H5f`), and in `H6a`, `coverage-matrix.md`, and `conventions.md`. The Adds fields of the H5 family, however, introduce the same artifact as **CI gate**: `H5a` Adds opens "The CI gate that reconciles…", and `H5c`, `H5e`, and `H5f` Adds each open "The CI gate arm that reconciles…".

Two surface names are used for one artifact with no stated equivalence. Each affected Adds does cross-reference "the unified closing-gate machinery `H5a` establishes", which keeps the two terms tied in practice, but the leading noun phrase that names the artifact is the non-canonical variant.

Note this is distinct from the separate "CI finding" reporting-contract concern: the phrase "emits each gap as a CI finding" in `H5a`/`H5b` is about the warn-only reporting channel, not the artifact name, and is out of scope here.

## Solution Space

**Shape:** single

### Recommendation

In the four Adds fields, replace the non-canonical artifact name with the canonical "closing gate":

- `docs/plan_topics/H5a-closing-gate-automation.md` Adds — change the opening `The CI gate that reconciles` to `The closing gate that reconciles`.
- `docs/plan_topics/H5c-broad-catch-allow-list-gate.md` Adds — change the opening `The CI gate arm that reconciles` to `The closing gate arm that reconciles`.
- `docs/plan_topics/H5e-un-anchored-must-gate.md` Adds — change the opening `The CI gate arm that reconciles` to `The closing gate arm that reconciles`.
- `docs/plan_topics/H5f-per-facet-citing-test-gate.md` Adds — change the opening `The CI gate arm that reconciles` to `The closing gate arm that reconciles`.

Leave the `emits each gap as a CI finding` wording in `H5a` (and the parallel phrase in `H5b`) untouched — that is the warn-only reporting term, addressed by the separate "CI finding" reporting-contract finding.

## Consequence

**Severity:** cosmetic

Both names denote the same artifact and each Adds ties itself to the "unified closing-gate machinery", so no implementer produces a divergent gate. The only effect is reader friction — a reader could momentarily read "CI gate" and "closing gate" as two artifacts.

## Problem

`V15e`'s eviction drops the static-resolution per-pass parse-cache entry for the changed file and for every transitive `.warp` importer, but nothing pins the three path identities that must compare equal for a drop to hit: the watcher-delivered changed-file path (`V8e`/`V9b`), the parse-cache key (`V15a`), and the import-edge-graph node identity (`V15c`). The only path-normalisation pin in the spec — `invocation.md` §Resolution's `realpath`-then-forward-slash form — is scoped to discovery-root containment and does not propagate to the cache key, graph node, or watcher path. If the watcher delivers a non-canonical form (symlink, differing case on a case-insensitive filesystem, `.`/`..` segments, or relative-vs-absolute mismatch), the eviction key-match misses, the stale parsed entry survives the `LoomRegistry` swap, and the next load pass reuses it. The failure is silent: no diagnostic, no panic.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `d38ffdb` (`spec: hot-reload via in-process LoomRegistry swap; no ctx.reload() for content edits (Decision #8)`, 2026-05-05) — the commit that first specified the per-pass-cache eviction on reload in `implementation-notes.md` §"Static-resolution load pass". It described dropping "the per-pass cache entry for the changed file and every transitive `.warp` importer" but never pinned the path form of the changed-file path, the cache key, or the import-edge-graph node, so the three-way identity assumption has been latent since the eviction was first written. `a3dd2f0` (`pi-loom plan: resolve "hot-reload cache-eviction sequenced before parse cache and .warp import edges"`, 2026-06-16) later created the `V15e` / `V15e-T` leaves to carry this eviction; they inherited the gap unchanged rather than introducing it.
**History:** `git log -S 'per-pass cache entry for the changed file'` and `git log -G 'transitive .warp importer'` show the eviction text entering at `d38ffdb` and never acquiring a path-normalisation pin across the three identities; `git log --diff-filter=A --follow` on `V15e-hot-reload-cache-eviction.md` shows the leaf created whole at `a3dd2f0` with no such pin.

## Solution approach

Pin a single canonical path identity for the eviction in `docs/spec_topics/implementation-notes.md` §"Static-resolution load pass", reusing the `realpath`-then-forward-slash form already defined in `invocation.md` §Resolution and linking to that anchor rather than restating the form. State that the watcher-delivered changed-file path, the per-pass parse-cache key, and the `.warp` import-edge-graph node identity are all compared in that one canonical form. Add a Test bullet and the matching Ships-when clause to `docs/plan_topics/V15e-hot-reload-cache-eviction.md` asserting eviction still hits when the watcher delivers the changed file in a non-canonical form that differs from the form the cache was keyed under, and mirror the obligation into `docs/plan_topics/V15e-T-hot-reload-cache-eviction.md`.

## Solution constraints

- Out of scope: `V15a`, `V15c`, `V8e`, and `V9b` are read-only here — they supply the cache, the import-edge graph, and the watcher path; only the spec and the `V15e`/`V15e-T` leaves change.

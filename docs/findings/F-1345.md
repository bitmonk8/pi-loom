## Problem

`registration-steps.md` carries four independent un-anchored normative obligation areas, none of which bears a `PIC-N` REQ-ID anchor: (a) the registry-swap atomicity contract emitting `loom/runtime/registry-swap-failed` (step 5); (b) the structural-change watcher-note pair at `#structural-changes-no-unregister` (the `MUST NOT` emit on an empty window and the `MUST` emit on a same-window rename); (c) the `pi.getCommands()` snapshot read-only-by-convention `MUST` (step 2); and (d) the factory-time renderer-registration pair. Because none carries a co-located `PIC-N` anchor, GOV-9's `#prefix-n` cross-link contract is unsatisfiable for these sites.

## Solution approach

Add a `PIC-N` REQ-ID anchor in GOV-1 dual form at each of the four defining obligation sites, allocating the numeric tail under the page's registered `PIC` prefix per GOV-3 and querying the current `PIC` high-water mark at edit time. Coin a separate `PIC-N` for each predicate of the watcher-note pair (the empty-window `MUST NOT` and the same-window-rename `MUST`). Update `cka-23`'s facet sub-labels to reference the coined `PIC-N` IDs.

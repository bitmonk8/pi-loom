## Problem

`H4a`'s **Adds** defines an in-memory fixture-supply mechanism that supplies fixture `.loom` content to the extension while claiming no ambient `src/**` read and no `FileSystem` seam (`V8b`) dependency, keeping the `H3a` ambient-access scan clean — but never names the route by which that in-memory content reaches discovery. The `H3a` seam set contains no "discovery source" seam, and production discovery (`V10a`) reads files only through the `FileSystem` seam, so the mechanism asserts fixture content reaches discovery while bypassing the only seam through which discovery reads files. `H7a` drives a multi-feature fixture through this harness against the integrated pipeline with no stated loading route at all. Two implementers diverge: one supplies the fixture via an in-memory `FileSystem`-seam fake (contradicting the leaf's own "no `FileSystem` seam dependency" claim), the other invents a harness-only discovery-source override.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** 2d52ad1 — pi-loom plan: resolve "M's happy-path discovery read has no defined source under the ambient-access ban" (2026-06-11, Thomas Andersen)
**History:** The "in-memory fixture-supply mechanism" wording entered `H4a` (and the matching consumer note in `M`) in commit 2d52ad1, which resolved an earlier finding that `M`'s happy-path discovery had no defined source under the ambient-access ban. That fix named the mechanism and asserted it bypasses both ambient reads and the `FileSystem` seam, but never specified the injection point by which harness content reaches discovery; the present gap was introduced together with the mechanism in that single commit. `git log -S 'in-memory fixture-supply'` confirms 2d52ad1 is the phrase's first and only introducing commit.

## Solution approach

Add to `H4a`'s **Adds** a named injection affordance — a harness-supplied discovery source the factory accepts in harness mode, distinct from the `H3a` seam set — and state that harness-driven discovery reads fixture `.loom` content from it. State in `H7a`'s **Adds** that its integrated-run fixture reaches the pipeline via this same `H4a` injection point. `M` / `M-T` may cite the named injection point without rewording their existing happy-path discovery text.

## Solution constraints

- The injection point must not be an in-memory implementation of the `FileSystem` seam (`V8b`); the leaf's "no `FileSystem` seam dependency / clean `H3a` ambient-access scan" invariant must remain intact.
- The spec is read-only for this fix.

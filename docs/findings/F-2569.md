## Problem

PIC-13's **Case-canonicalisation guarantee** binds the production `PiFileSystem` adapter, but the property is unobservable by automated test on a case-sensitive CI host and PIC-13 states no editorial-review or known-fragile verification fallback for the production half. The seam-owning plan leaf currently exercises only `.code` mapping. A leaf-case canonicalisation bug can therefore ship undetected and break the byte-for-byte discovery-root containment comparison the guarantee exists to protect.

**Verdict:** multi-commit-interaction
**Introducing commits:** e9b0597 — pi-loom spec: resolve "invocationId has no injectable id-source seam" (2026-06-05); a765f6ca — fix F-0977: pin PIC-13 realpath case-canonicalisation guarantee for DISC-3 dedup (2026-06-22); 4556623 — fix F-1178: PIC-12 timing-primitive ban gets observable build-time conformance criterion (2026-06-25)
**History:** PIC-20 was introduced whole by e9b0597 (the only commit touching the `any other ambient UUID source` clause, per `git log -S`), already lacking any conformance criterion. a765f6ca added the PIC-13 `realpath` case-canonicalisation guarantee binding the production adapter with no stated test path. The PIC-20/PIC-12 asymmetry then crystallised on 2026-06-25 when 4556623 retrofitted the build-time-check conformance criterion onto PIC-12 (and PIC-13's `homedir` bullet by reference) but not PIC-20 — even though PIC-20 was edited the same day by 0c6c347 (fix F-1396, toolCallId minting) — leaving PIC-20 the only ambient-primitive ban in the section without a criterion.

## Solution approach

Clarify PIC-13's **Case-canonicalisation guarantee** (anchor `pic-13`) to state how the production `PiFileSystem` adapter half is verified — by editorial review / known-fragile evidence, since it is unobservable by automated test on a case-sensitive CI host — following the verification posture already used for host-presupposition guarantees. Add a corresponding item to the Pi version-bump procedure checklist (`version-bump-*`). Pin an automated conformance test for the `FakeFileSystem` half via constructor-injected case-variant symlink-table aliases that must resolve to one byte-identical canonical path, so the testable half runs on every CI run regardless of host case-sensitivity.

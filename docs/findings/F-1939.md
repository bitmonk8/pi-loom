## Problem

`V15b` owns the INV-4 invoke-depth bound and declares its per-chain counter counts cross-file `.warp fn` frames, and both `V15b` and `V15b-T` author an INV-4 boundary-chain test that requires one cross-file `.warp fn` frame to reach the 32-frame boundary. No leaf reachable from `V15b`'s Deps (`V15a`, `V15c`, `V16a`) builds the `.warp fn` call-execution path: nothing evaluates a `.warp fn` call, pushes a per-chain countable frame, or classifies the call as cross-file vs intra-file by the callee's residence. The leaves that touch `fn` semantics — `V3a` (expression evaluator) and `V3d` (functions and return) — describe declaration and return-value semantics but name no call-frame hook, and `V15b` depends on neither. The INV-4 cross-file `.warp fn` arm therefore cannot be constructed, and its boundary-chain test bullet has no runtime substrate to exercise for that frame class.

## Issue introduction

**Verdict:** present-since-inception

**Introducing commits:** `c6a664e` (2026-06-10 08:16:57 +0200) "pi-loom plan: build/update plan for spec.md + review"

**History:** The cross-file `.warp fn` frame-counting obligation is in `V15b`'s Adds at the initial plan build (`c6a664e`), with Deps `V15b-T`, `V15a`, `V16a` and no leaf owning a `.warp fn` call-execution / frame-counting path. A later commit `084f171` (2026-06-10 18:51:17 +0200, "resolve 'V15b counts .warp fn frames introduced by V15c but declares no dependency on it'") added `V15c` to the Deps and the explicit cross-file `.warp fn` test language, but `V15c` only binds imported symbols — it added no call-execution producer or residence-classification owner — so the deeper gap this finding describes has been present since inception.

## Solution approach

In `docs/plan_topics/V15b-invoke-depth-cycle.md` Adds, state that `V15b` instruments the `.warp fn` call site to increment the shared per-chain counter and classifies a fn's residence by its declaration `.warp` file per `invocation.md` §INV-4. Add to both `V15b.Deps` and `V15b-T.Deps` the leaves that own `.warp fn` call evaluation — `V3a` and `V3d` — plus `V15i` so residence can resolve through re-exports and aliased imports.

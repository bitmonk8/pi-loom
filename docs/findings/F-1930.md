## Problem

`cka-27` is the canonical code-keyed-area token that `coverage-matrix.md` assigns to the `session-only-degraded-state.md` obligation set, with `V9l` as its sole closing leaf. All four Tests bullets in `V9l` and its mirror `V9l-T` reference that obligation only by spec-section paraphrase and never name `cka-27`, diverging from the plan-wide convention where each code-keyed-area Tests bullet cites its token inline (`V9g` cites `cka-31`, `V14b` cites `cka-13`). Because the token is absent from the leaf prose, the implementer has no instruction to embed `cka-27` in the test source, a reviewer cannot `grep cka-27` to confirm `V9l` coverage, and a cka-token index misses `V9l`. The gap is traceability/consistency only — no closing gate reddens, since `cka-27` is a single-leaf row outside the `H5f` per-facet arm.

**Verdict:** multi-commit-interaction
**Introducing commits:** 55ed974 — pi-loom plan: resolve "degraded-state obligations green over open spec contradiction" (2026-06-12, Thomas Andersen); 7259777 — pi-loom plan: resolve "code-keyed obligation areas lack a canonical citable token" (2026-06-18, Thomas Andersen)
**History:** `V9l` and `V9l-T` were created on 2026-06-12 (55ed974) citing the obligation only by spec-section paraphrase, before any `cka-<n>` token existed. The canonical code-keyed-area token scheme was added on 2026-06-18 (7259777), which introduced the `cka-27` row in `coverage-matrix.md` and migrated the sibling `V9g` and `V14b` Tests bullets to cite their tokens inline, but did not touch `V9l`/`V9l-T` — leaving them as the residue of an incomplete migration. A pickaxe search (`git log -S 'cka-27'`) over both leaf files confirms the token was never present in either.

## Solution approach

In `docs/plan_topics/V9l-session-only-degraded-branch.md` and its `-T` partner `docs/plan_topics/V9l-T-session-only-degraded-branch.md`, add an inline `cka-27` citation to each of the four Tests bullets alongside its existing `session-only-degraded-state.md §…` reference, matching the sibling form `(code-keyed-area token `cka-27`)` used by `V9g` for `cka-31` and `V14b` for `cka-13`. Keep both leaf files' bullets identical, since `V9l-T` mirrors `V9l`.

## Solution constraints

- The `coverage-matrix.md` `cka-27` row already names `V9l` as the closing leaf and is out of scope.

## Problem

The canonical schema hash recipe in `schema-subset.md` § *Canonical schema hash* step 2 does not define how numeric literals in `const` / `enum` positions are serialised; it forwards to `binder/defaulting-system-note-echo.md` § *Echo policy* and borrows BNDR-4, BNDR-5, and the BNDR-6 reference-renderings table as the normative algorithm. This makes a byte-level cryptographic contract — the cache keys for `__inline_<slug>`, `__loom_respond_<slug>`, the AJV compiled-validator cache, and every replayable provider payload — depend on a page whose primary topic is binder echo formatting and whose evolution pressure differs. An echo-formatter-driven edit to BNDR-4/5/6 would silently change canonical-hash bytes and invalidate every cached lowered-schema artefact, while the recipe's "the loom 1.0 spec pins it exactly" claim sits on a page with a different stability budget.

## Issue introduction

**Verdict:** single-commit-introduction
**Introducing commits:** `c5bb5e1` (2026-06-03, "pi-loom spec: resolve \"Canonical schema hash numeric serialization underspecified\"")
**History:** Prior to `c5bb5e1`, step 2 of the canonical-form bullet read only `"numeric literals in their JSON Schema integer/number form (no trailing zeros, no exponent unless necessary to represent the value)"` — under-specified, but locally owned. `c5bb5e1` resolved the under-specification by *citing* the binder echo page's integer/number rules rather than promoting them to a shared neutral location, introducing the cross-page co-ownership this finding flags. Two subsequent edits refined the citation without changing its shape: `111e533` (2026-06-07, "Canonical hash recipe cites keywords the lowered subset never emits") removed `default` from the cited positions and added the inline HTML-comment rationale, and `7321f01` (later) added the array-element-order bullet alongside. Both retained the borrow-from-binder-echo structure.

## Solution approach

Relocate the integer/number value-rendering rule — BNDR-4, BNDR-5, and the BNDR-6 reference-renderings table — out of the binder echo page's ownership into a shared spec-topic page that both the canonical schema hash and the binder argument echo consume as a neutral primitive. Replace `schema-subset.md` § *Canonical schema hash* step 2's inline forward to `binder/defaulting-system-note-echo.md#echo-policy` with a forward-link to the new home. Replace the corresponding integer/number Format-rules bullets in `defaulting-system-note-echo.md` § *Echo policy* with a forward-link to it, leaving the echo-specific rules (120-code-point cap, `(default)` suffix, array `…+N more`) in place. Register the new page per the project's page-registration governance.

## Solution constraints

- BNDR-4, BNDR-5, and BNDR-6 REQ-IDs are preserved verbatim; REQ-ID renumbering or retirement requires GOV machinery and is out of scope for this move.
- The page-level anchors `#bndr-4`, `#bndr-5`, `#bndr-6` and the sub-letter anchors `bndr-6a`…`bndr-6x` must stay reachable (via GOV-26 alias redirects) because `V2d-T`, the BNDR-coverage rows, and existing fixtures cite them individually.

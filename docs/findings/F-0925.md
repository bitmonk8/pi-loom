## Finding

`M-T`'s second `SLSH-2` Tests bullet asserts that running the fixture loom through the harness "produces exactly one appended turn **and no diagnostic**." The zero-diagnostic half of that assertion is not self-evident: it holds only because a loom terminating with `Ok(v)` emits no `loom-system-note` keyed on the success outcome. That guarantee is the *success-side null-policy*, owned by `spec_topics/pi-integration-contract/runtime-event-channel.md#success-side-null-policy`, which establishes it by construction from the always-log set (all members are `QueryError` kinds or runtime panics, neither of which a successful happy-path turn produces).

`M-T`'s Spec list cites only `overview.md`, `slash-invocation.md`, `discovery/discovery-sources.md`, and `frontmatter/frontmatter-fields-a.md`. The owning anchor is reachable only transitively — `slash-invocation.md` (cited) links to the null-policy from its prompt-mode bullet — so the load-bearing premise behind "no diagnostic" is never surfaced at the leaf. `M` shares the same happy-path framing and the same Spec list, with the same gap.

An implementer authoring the test from `M-T`'s citations alone cannot see *why* zero diagnostics is the expected count, and has no anchor to confirm the always-log set emits nothing on a successful turn. If that premise were ever false, the assertion would be wrong for a reason invisible to anyone reading the leaf.

## Solution Space

**Shape:** single

### Recommendation

Add `[`../spec_topics/pi-integration-contract/runtime-event-channel.md#success-side-null-policy`]` to the **Spec.** field of both `M-T-minimal-slash-command.md` and `M-minimal-slash-command.md`, alongside the existing four citations. In `M-T`'s second `SLSH-2` Tests bullet, attribute the no-diagnostic expectation to that policy — e.g. extend "and no diagnostic" to "and no diagnostic (the success-side null-policy emits no `loom-system-note` on an `Ok(v)` happy-path turn)." This makes the premise behind the zero-diagnostic count explicit and gives the implementer the owning anchor to confirm it.

## Consequence

**Severity:** advisory

The assertion is correct, so no implementer is blocked, but the premise it rests on is undiscoverable from the leaf's own citations. An implementer cannot independently confirm the expected diagnostic count or reason about it if the always-log set changes; the no-diagnostic expectation reads as an unexplained constant.

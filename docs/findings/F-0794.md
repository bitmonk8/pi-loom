## Problem

The §*Recovery path and prohibitions* section of `docs/spec_topics/pi-integration-contract/session-only-degraded-state.md` carries four independently-satisfiable MUST NOT obligations (un-drain the registry; re-subscribe/re-establish the closed watchers; poll after watcher closure; any other discovery/re-scan/self-recovery path) as an unannotated bullet list. None carries a REQ-ID anchor, and the section's only anchor is the GitHub auto-slug `#recovery-path-and-prohibitions`. The four obligations land in `coverage-matrix.md` as a single GOV-22-residue token (`cka-27`) bundled with three other unrelated obligations. A sibling page or test that touches only one prohibition has no per-obligation fragment-link target, and `cka-27` gives no way to assert one bullet closed without claiming all four — so GOV-9's `#prefix-n` cross-link contract is unreachable for these obligations.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** `52b9101` (2026-05-11) — "pi-loom spec: resolve \"T24 — Fork-reason watcher closure leaves the extension in an unspecified, silently degraded state\""
**History:** `git log -S "MUST NOT un-drain the registry"` and `git log -S "MUST NOT introduce any other discovery"` over the spec corpus each return two commits — `52b9101` (which authored the four prohibitions as an unanchored bullet list inside the original `pi-integration-contract.md` mega-file) and `f5e89f4` (the 2026-06-04 mechanical page-split commit that lifted the §Recovery-path prohibitions section into the new `session-only-degraded-state.md` file without changing the bullets). The four prohibitions were therefore never anchored — the un-anchored shape entered the corpus with the rule itself, not via later regression. The `cka-27` coverage-matrix row that tags the bullet block as `(un-anchored; GOV-22 residue)` was added later in `bad3b99` (2026-06-11, "V9h pi-integration-contract pages have no coverage-matrix rows"); that commit acknowledged the residue rather than introducing it.

## Solution approach

Coin one REQ-ID anchor per MUST NOT bullet in the §*Recovery path and prohibitions* section, allocating next-free numeric tails under the page's already-registered `PIC` prefix per GOV-3, rendered in GOV-1 dual-form layout. Thread the new per-obligation IDs into the `cka-27` row of `coverage-matrix.md` so individual-bullet closure can be asserted.

## Solution constraints

- Allocate the new IDs under the existing `pi-integration-contract/` `PIC` prefix row; do not register a new prefix (e.g. `DEGR`), which would require a GOV-7 *Add* row.

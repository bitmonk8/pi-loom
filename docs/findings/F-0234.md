## Problem

`V2d`'s `BNDR-5` **Tests** bullet and its **Ships when** clause require coverage of an interior small-magnitude value via the hedged phrasing "an in-range value such as `5e-8`" / "including an in-range value such as `5e-8`". The hedge does not say whether `5e-8` is the required vector the test must assert or merely one illustrative member of the `|value| < 1e-7` range. Two reasonable test authors therefore diverge: one asserts a concrete interior vector, the other treats the clause as optional flavour text and asserts only the named boundary magnitudes `1e21` / `1e-8` — silently dropping the interior coverage the clause was added to guarantee. The criterion does not pin down its own pass condition.

## Issue introduction

**Verdict:** introduced-by-later-edit (single commit)
**Introducing commit:** `31f4a86` — "pi-loom plan: resolve 'BNDR-5 number-renderer impl leaf cites only example vectors, not the |value| < 1e-7 threshold'" (2026-06-11)
**History:** `git log --follow` over `docs/plan_topics/V2d-number-rendering.md` shows the file first authored at the initial plan build (`c6a664e`, 2026-06-10), then edited by `31f4a86`, then `cb8737a` (the over-bundling resolution). `git log -S '5e-8' -- docs/plan_topics/V2d-number-rendering.md` returns `31f4a86` as the sole introducing commit. The diff confirms that before `31f4a86` the `BNDR-5` bullet named only the `1e21` and `1e-8` fixed-point expansions; `31f4a86` — resolving a prior review finding that the leaf cited only example vectors and not the `|value| < 1e-7` threshold — rewrote the bullet and Ships when to name the `|value| < 1e-7` switch and added the "an in-range value such as `5e-8`" clause. The hedged "such as" phrasing was introduced by that fix, not present at inception; this is a fix-introduced clarity regression in a single commit.

## Solution approach

In `V2d`'s `BNDR-5` **Tests** bullet and **Ships when** clause, replace the hedged "an in-range value such as `5e-8`" with an unambiguous required interior assertion. Since the defect is the wording, carry a definite obligation: either name `5e-8 → 0.00000005` as a mandatory interior vector (it sits strictly inside the small-magnitude switch, `|5e-8| < 1e-7`), or state the requirement as the range predicate "at least one value strictly interior to the small-magnitude switch (`1e-8 < |value| < 1e-7`)". Either form must read as a required assertion, not an example.

## Finding

In `docs/plan_topics/coverage-matrix.md`, the *Numbered REQ-IDs* table row `ERR-8 … ERR-12` closes with `` `V4c`, `V9i` (live subagent-surface ERR-8 co-witness) ``. The `V9i` parenthetical places the descriptive adjectives ("live subagent-surface ERR-8") *before* the `co-witness` role token, leaving the token in trailing position.

Every other co-witness annotation in the same table is role-first — the `co-witness` token leads, followed by an em-dash and the description:

- `ERR-13` → `` `V14a` (co-witness — delegated completed-callee-finality live-carrier) `` (and the same for `V13c`, `V15a`).
- `CIO-1` → `` `V4e` (co-witness — slash-load-before-runtime temporal-placement load-time consult) `` (and `V11f`).
- `CIO-5` → `` `H7a` (co-witness — cross-site never-interleaves integration witness) ``.
- `cka-44` → `` `V7d` (co-witness — PIC-21 multi-error-batch back-reference closed on its owning leaf) ``.

The ERR-8 row is the sole co-witness annotation that does not follow the role-first grammar. The literal `co-witness` token is present, so the [`H5f`](docs/plan_topics/H5f-per-facet-citing-test-gate.md) facet-partition derivation and the `conventions.md` §release-time-residue-inspection item-7 split-row sweep — both of which key only on the *presence* of the `co-witness` token in the immediately-following parenthetical to drop `V9i` from the facet partition — are unaffected. The defect is purely the inconsistent annotation grammar.

## Issue introduction

**Verdict:** introduced-by-later-edit (single introducing commit; incomplete normalization sweep)
**Introducing commit:** `685e45b` (`pi-loom plan: resolve "co-witness leaves indistinguishable from facets in item 7 split-row sweep"`, 2026-06-18 12:04).
**History:** `git log -S 'live subagent-surface ERR-8 co-witness'` shows the ERR-8 `V9i` annotation was first authored in `7bd65a7` (`pi-loom plan: resolve "V4c/V4f defer ERR-8 to ERR-13 live-surface confirmation…"`, 2026-06-18 11:11), which added `` `V9i` (live subagent-surface ERR-8 co-witness) `` in the then-current token-last form — not a deviation at the time, since no role-first convention yet existed. About an hour later `685e45b` established the role-first `(co-witness — <description>)` grammar: its diff converts `ERR-13` from the token-last `(live completed-callee-finality co-witnesses)` form to role-first, and likewise normalizes `CIO-1`, `CIO-5`, and `cka-44`. That sweep did not retrofit the `ERR-8` row, leaving it as the lone surviving token-last annotation. The deviation is therefore an incomplete normalization in `685e45b`, not original to the ERR-8 line's authoring.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/coverage-matrix.md`, in the `ERR-8 … ERR-12` row's closing-leaf cell, replace the `V9i` parenthetical `(live subagent-surface ERR-8 co-witness)` with the role-first form `(co-witness — live subagent-surface ERR-8)`, so the cell reads `` `V4c`, `V9i` (co-witness — live subagent-surface ERR-8) ``.

The annotation prose stays backtick-free — the bare `ERR-8` inside the parenthetical carries no backticks, per the *Closing-leaf column authoring* rule (only leaf-ID spans may be backtick-delimited; H5d tokenises backtick spans only). Leave the `` `V4c` `` and `` `V9i` `` leaf-ID tokens unchanged.

## Consequence

**Severity:** cosmetic

No implementer or gate is affected: the `co-witness` token is present, so H5f and the item-7 split-row sweep correctly exclude `V9i` from the ERR-8 facet partition and require no per-facet citing test for it. The only impact is a within-table grammar inconsistency that makes the annotation set harder to read and maintain consistently.

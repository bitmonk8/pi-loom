## Problem

The seven DI injection points in `host-interfaces-services.md` (PIC-10 through PIC-20) are the same construct — an internal DI seam with a production adapter and a test fake — but their `**PIC-N. ...**` headings name it with two different words. PIC-10, PIC-16, and PIC-20 say "seam"; PIC-11 (`SchemaValidator`), PIC-12 (`Clock` / `FakeClock`), PIC-13 (`FileSystem`), and PIC-14 (`FileWatcher`) say "interface". The body prose of all seven entries consistently uses "seam" ("modelled on the other DI seams in this section", "the seam is internal"), so the heading word disagrees with the body word for four of the seven entries.

## Issue introduction
**Verdict:** multi-commit-interaction
**Introducing commits:** `bc50e69` (2026-05-05, Clock seam), `8a6d15f` (2026-05-06, FileSystem/FileWatcher/SchemaValidator), `033b8e3` (2026-05-05, Checkpoint seam), `e9b0597` (2026-06-05, IdSource seam), `1f37b15` (2026-06-05, TokenEstimator seam)
**History:** The seven entries were authored across five independent fix commits, never in one pass. Each commit message refers to its construct as a "seam," but the heading word landed inconsistently: the entries authored alongside an inlined TypeScript `interface { ... }` declaration block (`SchemaValidator`, `Clock`/`FakeClock`, `FileSystem`, `FileWatcher` — commits `8a6d15f` and `bc50e69`) took the heading word "interface" from the TS construct, while the entries authored without that framing (`Checkpoint` in `033b8e3`, then `IdSource` in `e9b0597` and `TokenEstimator` in `1f37b15`) used "seam." No single commit introduced the divergence; it accreted as later DI-seam additions chose "seam" without anyone reconciling the earlier "interface"-worded headings. The pre-split monolithic `spec.md` did not yet contain these entries, so the split itself (`f5e89f4`) only carried the already-mixed state forward.

## Solution approach

Rename the construct-label word in the PIC-11, PIC-12, PIC-13, and PIC-14 headings from "interface" to "seam", matching the body prose and the PIC-10 / PIC-16 / PIC-20 headings that already say "seam".

## Solution constraints

- Out of scope: the `<a id="…-interface">` anchor identifiers — inbound cross-references and anchor-scheme-stability governance depend on them; only the heading prose changes.
- Out of scope: body occurrences of "interface" that name the TypeScript `interface { ... }` reference declaration; only the heading construct-label word changes.

## Problem

The `CIO` / `HC3` / `NOCEIL` / `CEIL` closure-rationale note in [`coverage-matrix.md`](docs/plan_topics/coverage-matrix.md) asserts that `HC3-a … HC3-e` close in `V11f`, but no convention documents *how* a GOV-16 stable-inline-label family closes on its authoring leaf. Inline-label families are excluded from the executable-REQ-ID closing gate by `conventions.md` §REQ-ID discipline and have no coverage-matrix row, and the §Leaf format *Tests* rule speaks only of "one bullet per **REQ-ID**" — silent on inline labels. An author opening `V11f` (or closing the `NOCEIL-1 … NOCEIL-4` family across `V6a` / `V4d` / `V4b` / `V15b`) has no rule for whether to add a coverage-matrix row, a citing *Tests* bullet, both, or nothing. The gap recurs for any future GOV-16 family.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** c6a664e — pi-loom plan: build/update plan for spec.md + review (2026-06-10, Thomas Andersen)
**History:** The inline-label closure claim ("the `HC3`/`NOCEIL` inline labels … closed by `V16a` and `V11f`") and `V11f`'s `HC3-a … HC3-e` Tests bullets both entered the corpus in the initial plan build c6a664e; later commits (2565ddd, 8ef7055, aa1b666) only refined the rationale prose. The `conventions.md` §Leaf format *Tests* rule has always been worded "one bullet per REQ-ID" and never covered the inline-label class, so the guidance gap has been present since the inline-label closure content was first authored.

## Solution approach

Add a closure-semantics statement for GOV-16 stable-inline-label families to `conventions.md` §REQ-ID discipline, recording that such families are not executable REQ-IDs, carry no coverage-matrix mapping obligation, are not subject to the closing-gate surfaces, and close at the leaf(s) named in the `coverage-matrix.md` closure rationale via that leaf's *Tests* bullets under the same code-citation discipline applied to REQ-IDs. Clarify the §Leaf format *Tests* rule so it admits inline labels as citable *Tests*-bullet subjects rather than reading as REQ-ID-exclusive.

## Solution constraints

- The closure-semantics rule belongs in `conventions.md`; do not duplicate it into the `coverage-matrix.md` note, whose inline-label table structure is owned by T27.
- The spec is read-only for this fix; GOV-16 itself is referenced, not edited.

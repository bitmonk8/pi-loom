## Finding

GOV-15's closure clause states that "adding a further observable to either list is a substantive edit to GOV-15 governed by the present rule's edit conventions, not by GOV-8's *Pure rewording* arm." The phrase *the present rule's edit conventions* points back at GOV-15 itself, but GOV-15 nowhere defines any such conventions: it carves a `Ceiling-set carve-out`, a `Diagnostic-registry carve-out`, an `Attribution test`, and the four `Operational definitions` of the ceiling-set verbs, none of which govern edits *to GOV-15's own list of observables or exclusions*. The only positive content the sentence carries is the negation (it is *not* GOV-8's *Pure rewording* arm).

An editor confronted with the natural next case — "we have a fourth observable that should be added to the (a)–(c) list" — therefore has no procedure to follow. GOV-8 supplies the substantive-edit machinery for every other normative rule (Split / Merge / Deletion-plus-add), and GOV-15's own *Operational definitions* paragraph already cites GOV-8's lifecycle as the default for substantive ceiling-page edits before carving its in-place exception. The "present rule's edit conventions" phrase appears to intend a parallel in-place carve-out for GOV-15's two closed lists, but never names one, and the result is a dangling self-reference that neither permits in-place editing nor licenses any specific substitute path.

## Issue introduction

**Verdict:** single-commit-introduction
**Introducing commit:** `2ff03c2` (2026-05-10) — *pi-loom spec: resolve "Source-language stability bullet leaves equivalence undefined at the aggregator and silently contradicts the open-set ceiling clause"*
**History:** The phrase "*governed by the present rule's edit conventions, not by GOV-8's Pure rewording arm*" was added verbatim in `2ff03c2` when the closure-of-the-two-lists clause was first written into GOV-15. Before that commit, GOV-15 carried no closure clause and the question did not arise. Subsequent commits (`8d730c5`, `7995d3c`, the V→loom 1 rename family, `c8043ef`, `1406d57`) moved GOV-15 between files and refined adjacent text but left the self-referential phrase untouched. The defect was therefore present from the moment the closure clause was introduced and has propagated unchanged into the current `docs/spec_topics/governance/source-language-stability.md`.

## Solution Space

**Shape:** single

### Recommendation

Drop the phantom self-referent and route substantive edits to GOV-15's two closed lists through GOV-8's existing substantive-edit machinery. Concretely, replace the trailing clause

> *…is a substantive edit to GOV-15 governed by the present rule's edit conventions, not by GOV-8's Pure rewording arm.*

with an explicit GOV-8 hand-off, e.g.

> *…is a substantive edit to GOV-15 and MUST be modelled as a GOV-8 Split or Deletion-plus-add against the GOV-15 anchor (the in-place edit licensed for ceiling-page Relax/Tighten by the Ceiling-set carve-out below does not extend to GOV-15's own observable/exclusion lists).*

Edge cases for the implementer:

- The parallel `Operational definitions` paragraph already cites GOV-8's lifecycle as the substantive-edit default before carving its in-place exception for *Relax*/*Tighten*; the new GOV-15 wording should mirror that pattern so the two paragraphs use identical language for "GOV-8 substantive path is the default; in-place carve-outs are explicit."
- The "References elsewhere in the corpus to 'four ceilings'…" sentence that follows is governed by GOV-30 lock-step and is independent of this fix; leave it untouched.
- GOV-8 itself (per `req-id-prefix-table-active-b.md`) treats Split / Deletion as causing the original ID to retire and N fresh IDs to be appended. An editor adding a fourth observable to GOV-15 would therefore retire `GOV-15` and coin a fresh `GOV-N`; the existing cross-references to `gov-15` across the corpus (diagnostics, hard-ceilings, glossary, governance hub, session-model-and-appendix, overview-and-orientation, release-version-naming, etc.) would need a GOV-30 lock-step pass in the same commit. This is a feature of GOV-8, not a new obligation, but worth calling out in the replacement wording so a future editor is not surprised.

## Consequence

**Severity:** advisory

A future editor extending GOV-15's (a)–(c) observable list or the wall-clock/token-count/log-line-volume exclusion list has no documented procedure: the cited conventions do not exist, and the only stated constraint is that GOV-8's *Pure rewording* arm does not apply. Two reasonable editors will diverge — one performing an in-place edit under the (silent) assumption that GOV-15 is self-carve-out, the other performing a GOV-8 Split / Deletion-plus-add. The rule's intent is recoverable from context, so this does not block loom 1.0, but it leaves the next governance edit unspecified.

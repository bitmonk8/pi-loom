## Problem

Eight normative MUSTs introduced across four neighbouring PIC shard files pin precise runtime behaviour but supply no pass/fail criterion, no fixture shape, and no observable a conformance test could assert against. The fixture-less obligations are: the `willRetry` retry-handling clause and the spawn-then-immediate-cancel ordering clause at `subagent.md#subagent-untyped-query-ok-extraction`, plus `#pic-9`'s `dispose()`-throws result-preservation paths; `tool-registration-lifetime.md`'s PIC-8 retry cap (`#pic-8`) and PIC-17 subagent-mode `setActiveTools` exclusion (`#pic-17`); `unknown-reason-rule.md`'s control-character escaping rule and its empty-registry + unknown-reason five-sub-step conjunction (`#unknown-reason-rule`); and `session-shutdown-semantics.md`'s `invoke`-parent observation rule. Each MUST is visible to the un-anchored-normative-MUST gate but absent from any conformance fixture, so two conformant implementations can ship green suites that disagree on observable behaviour at each of these eight axes.

## Issue introduction

**Verdict:** present-since-inception (at each shard's introduction).

**Introducing commits and history:**
- `subagent.md` `willRetry` clause — introduced by commit `024215d` (*"pi-loom spec: resolve 'untyped-query Ok(string) extraction mechanism (prompt + subagent)'"*, 2026-06-01) into `docs/spec_topics/pi-integration-contract.md`, then carried forward into `subagent.md` by the `81bf52a` shard-split (*"docs: collapse spec/ section pages into spec_topics/"*). No subsequent edit added a Tests-side criterion. `git log --all --follow -G 'agent_end\` event whose \`willRetry'` on `subagent.md` returns only header-rewriting edits (`cb3a2f2`, `aa334c2`) that did not touch the criterion-absence.
- `subagent.md` spawn-then-immediate-cancel clause — introduced alongside the cancellation-listener wiring by commit `5c8e630` (*"pi-loom spec: resolve 'Subagent cancellation wiring depends on a non-existent createAgentSession({ signal }) option'"*); `git log --all -G 'spawn-then-immediate-cancel path remains correct'` confirms the clause has been unaltered through every subsequent edit that touched the file.
- `tool-registration-lifetime.md` PIC-8 retry-cap clause — introduced by commit `d4ac81b` (*"pi-loom spec: resolve 'PIC sections beyond Probe-wide invariants — missing REQ-ID anchors'"*, 2026-06-04) at the same edit that added the PIC-8 anchor. `git log --all -G 'no exponential backoff, no third try'` shows the literal has never been re-edited.
- `tool-registration-lifetime.md` PIC-17 subagent-mode exclusion clause — present at the same `d4ac81b` PIC-anchor introduction; unchanged since.
- `unknown-reason-rule.md` control-character escaping clause and fast-path prohibition — both present at the file's split (`81bf52a`), then refined by `aa2f9d7` (*"String(event.reason) coercion-throw unreadable fallback"*) and `a7d2a80` (*"patch-skew teardown / literals-shape-invalid fixture enumeration"*). `a7d2a80` is notable: it added the explicit per-sub-case fixture requirement for `"literals-shape-invalid"` but did not extend the same per-fixture discipline to the adjacent control-character escaping rule, leaving the asymmetry this finding flags.
- `session-shutdown-semantics.md` `invoke`-parent observation rule — introduced by commit `7f37dd7` (*"pi-loom spec: resolve 'Session-swap behaviour for in-flight loom invocations is under-specified'"*); never re-edited.

**Pattern.** Each obligation entered the corpus as a behavioural pin without a paired Tests-side criterion, and the per-finding fixes that subsequently touched neighbouring text (e.g. `a7d2a80`'s `literals-shape-invalid` fixture enumeration, `aa2f9d7`'s coercion-throw fallback) tightened the same files without re-examining whether other MUSTs in the same paragraph carried fixtures. This is the spec-corpus analogue of `present-since-inception` — at each shard the obligation has shipped fixture-less since first appearance, and the standing testability lens is what surfaces them as a cluster now.

## Solution approach

Clarify each of the eight MUSTs in the four cited files (`subagent.md`, `tool-registration-lifetime.md`, `unknown-reason-rule.md`, `session-shutdown-semantics.md`) by adding an inline conformance-criterion sentence to the same paragraph as each obligation, naming the observable a conformance test asserts against. The existing `literals-shape-invalid` per-sub-case fixture clause in `unknown-reason-rule.md` is the in-corpus wording template. Add a matching Tests bullet per obligation to the paired plan leaves `V9i`/`V9i-T`, `V9f`/`V9f-T`, `V9g`/`V9g-T`, `V9h`/`V9h-T`, and `V9l`/`V9l-T`, each citing the freshly-added criterion by section anchor.

## Solution constraints

- Out of scope: changing the normative behaviour of the eight MUSTs themselves — that surface is owned by F-1028; this finding adds acceptance criteria only.
- MUST NOT add new sections or new REQ-ID/HTML anchors; criteria attach inline to the existing MUST paragraphs.

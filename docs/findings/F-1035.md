## Finding

Eight normative obligations introduced across four neighbouring PIC files state behaviour but supply no pass/fail criterion, no fixture shape, and no observable a conformance test could assert against:

1. **`subagent.md` тАФ `willRetry` retry handling** (`#subagent-untyped-query-ok-extraction`). "An `agent_end` event whose `willRetry` field is `true` тАж does not resolve the query: the runtime ignores such an event, keeps the subscription live, and resolves only from the terminal `agent_end` event whose `willRetry` is `false`." No criterion describes the multi-event fixture, the non-resolve assertion, or that the resolved `Ok(string)` is extracted from the terminal event's `messages`, not the intermediate one.
2. **`subagent.md` тАФ spawn-then-immediate-cancel ordering.** "If `loomAbort.signal.aborted` is already true at the moment `createAgentSession(...)` resolves, the runtime invokes `session.abort()` synchronously *before* the listener is registered and *before* issuing the first user turn." The MUST is about call ordering at the spawn site; no test specifies the spy that would catch `sendUserMessage` being called at all on that path, nor that `abort()` precedes any listener registration.
3. **`session-shutdown-semantics.md` тАФ `invoke`-parent observation rule.** Parent `?` / `match` arms covering a child's `Err(QueryError { kind: "cancelled" })` MUST NOT run inside the sub-step-3 bounded-await window. No criterion describes how to observe that absence тАФ a parent arm whose side effect (e.g. a distinguishable `loom-system-note`) is asserted not to fire before both `disposeBarrier`s settle.
4. **`subagent.md` PIC-9 тАФ `dispose()`-throws result preservation.** Three separate paths (Ok-return, Err-return, panic teardown) MUST all preserve the original result when `dispose()` throws, with a `loom/runtime/subagent-dispose-failure` advisory emitted. The current `V9i` Tests bullet collapses these into "teardown never masks the original `Err`/`Ok`"; no criterion enumerates the three paths as independent fixtures, and the panic-path arm is uncovered.
5. **`tool-registration-lifetime.md` PIC-8 тАФ retry MUST NOT exceed one.** "The retry MUST NOT chain back into `pi.setActiveTools` beyond the single re-attempt above (no exponential backoff, no third try)." No criterion pins the observable: a persistent-failure fixture must witness exactly two calls, with a third being a conformance failure.
6. **`tool-registration-lifetime.md` PIC-17 тАФ subagent mode never calls `setActiveTools`.** "Subagent-mode invocations are unaffected тАФ the subagent's tools are scoped to the spawned `AgentSession` and released by `dispose()`, so this rule applies only to the prompt-mode and prompt тЖТ prompt cross-mode `invoke` snapshot/restore paths." This is a negative invariant about the host `ExtensionAPI`; an incorrect implementation silently corrupts the user session's active set. No criterion specifies asserting `pi.setActiveTools` is never invoked between `createAgentSession` and `dispose()` for a subagent-mode loom.
7. **`unknown-reason-rule.md` тАФ control-character escaping in the `"throw:<String(error)>"` discriminator.** The rule pins precise transformations: `\n` / `\r` / `\t` as their two-character literal escapes and every other code point in U+0000тАУU+001F and U+007FтАУU+009F as `\\uXXXX`. The sibling `"literals-shape-invalid"` rule explicitly mandates one fixture per sub-case; no parallel fixture requirement is stated for the four named escape forms.
8. **`unknown-reason-rule.md` + `session-only-degraded-state.md` тАФ empty-registry + unknown-reason five-sub-step conjunction.** The unknown-reason path explicitly forbids the no-active-invocations fast-path: "the registry drain, the bounded `Promise.allSettled` await, the watcher closure, and the listener detachment all run unconditionally." No criterion asserts the *conjunction* (empty registry + unknown `event.reason` тЖТ every one of the five sub-steps ran), which is the only path that can witness the fast-path was *not* taken.

The pattern is uniform: a precise normative behaviour is pinned in prose, but the plan leaves that close the surrounding REQ-IDs / code-keyed areas (`V9c`, `V9f`, `V9g`, `V9h`, `V9i`, `V9l`) carry no Tests bullet that exercises the specific obligation, leaving each MUST visible to the un-anchored-normative-MUST gate but absent from any conformance fixture.

## Issue introduction

**Verdict:** present-since-inception (at each shard's introduction).

**Introducing commits and history:**
- `subagent.md` `willRetry` clause тАФ introduced by commit `024215d` (*"pi-loom spec: resolve 'untyped-query Ok(string) extraction mechanism (prompt + subagent)'"*, 2026-06-01) into `docs/spec_topics/pi-integration-contract.md`, then carried forward into `subagent.md` by the `81bf52a` shard-split (*"docs: collapse spec/ section pages into spec_topics/"*). No subsequent edit added a Tests-side criterion. `git log --all --follow -G 'agent_end\` event whose \`willRetry'` on `subagent.md` returns only header-rewriting edits (`cb3a2f2`, `aa334c2`) that did not touch the criterion-absence.
- `subagent.md` spawn-then-immediate-cancel clause тАФ introduced alongside the cancellation-listener wiring by commit `5c8e630` (*"pi-loom spec: resolve 'Subagent cancellation wiring depends on a non-existent createAgentSession({ signal }) option'"*); `git log --all -G 'spawn-then-immediate-cancel path remains correct'` confirms the clause has been unaltered through every subsequent edit that touched the file.
- `tool-registration-lifetime.md` PIC-8 retry-cap clause тАФ introduced by commit `d4ac81b` (*"pi-loom spec: resolve 'PIC sections beyond Probe-wide invariants тАФ missing REQ-ID anchors'"*, 2026-06-04) at the same edit that added the PIC-8 anchor. `git log --all -G 'no exponential backoff, no third try'` shows the literal has never been re-edited.
- `tool-registration-lifetime.md` PIC-17 subagent-mode exclusion clause тАФ present at the same `d4ac81b` PIC-anchor introduction; unchanged since.
- `unknown-reason-rule.md` control-character escaping clause and fast-path prohibition тАФ both present at the file's split (`81bf52a`), then refined by `aa2f9d7` (*"String(event.reason) coercion-throw unreadable fallback"*) and `a7d2a80` (*"patch-skew teardown / literals-shape-invalid fixture enumeration"*). `a7d2a80` is notable: it added the explicit per-sub-case fixture requirement for `"literals-shape-invalid"` but did not extend the same per-fixture discipline to the adjacent control-character escaping rule, leaving the asymmetry this finding flags.
- `session-shutdown-semantics.md` `invoke`-parent observation rule тАФ introduced by commit `7f37dd7` (*"pi-loom spec: resolve 'Session-swap behaviour for in-flight loom invocations is under-specified'"*); never re-edited.

**Pattern.** Each obligation entered the corpus as a behavioural pin without a paired Tests-side criterion, and the per-finding fixes that subsequently touched neighbouring text (e.g. `a7d2a80`'s `literals-shape-invalid` fixture enumeration, `aa2f9d7`'s coercion-throw fallback) tightened the same files without re-examining whether other MUSTs in the same paragraph carried fixtures. This is the spec-corpus analogue of `present-since-inception` тАФ at each shard the obligation has shipped fixture-less since first appearance, and the standing testability lens is what surfaces them as a cluster now.

## Solution Space

**Shape:** multiple

This finding is bimodal at the meta level: the eight obligations split into two distinct edit shapes, and the right resolution path differs by shape. Each option addresses one obligation class; the recommendation orders them.

### Option A тАФ Spec-side: pin each MUST's observable as inline acceptance text

For each of the eight MUSTs, add a single sentence (or short bullet) to the same paragraph that pins the observable a conformance fixture must assert. Use the existing `literals-shape-invalid` four-sub-case fixture clause in `unknown-reason-rule.md` as the template: it pins fixture-per-sub-case, the observable to assert (`details.failure === "literals-shape-invalid"`), and a negative observable (no `loom/host/session-shutdown-reason-unknown` fires).

**Approach.** Add per-MUST acceptance criteria inline in the four files, using these per-obligation observables:

| # | File / clause | Observable to pin |
|---|---|---|
| 1 | `subagent.md` `willRetry` | Multi-event fixture (`willRetry: true` then `willRetry: false`); assert the query promise does not resolve after the first event and resolves with `Ok(string)` extracted from the terminal event's `messages`. |
| 2 | `subagent.md` spawn-then-immediate-cancel | Fixture where `loomAbort.abort()` runs before the spawn-mock resolves; assert `session.abort()` is invoked before any `session.sendUserMessage` call and before any listener registration on `loomAbort.signal`. |
| 3 | `session-shutdown-semantics.md` `invoke`-parent observation rule | Parent loom with a child `invoke?` and a distinguishable `loom-system-note` side-effect on the `Err(QueryError{kind:"cancelled"})` arm; on `session_shutdown`, assert that note is not emitted before both `disposeBarrier`s settle. |
| 4 | `subagent.md` PIC-9 dispose-throws | Three independent fixtures: Ok-return + throwing `dispose()`; Err-return + throwing `dispose()`; panic + throwing `dispose()`. Each asserts the parent observes the original outcome unchanged and exactly one `loom/runtime/subagent-dispose-failure` row. |
| 5 | `tool-registration-lifetime.md` PIC-8 retry cap | Persistent-failure fixture (`pi.setActiveTools` throws on every call); assert exactly two invocations before steps (b)тАУ(d) run; a third call is a conformance failure. |
| 6 | `tool-registration-lifetime.md` PIC-17 subagent-mode exclusion | Subagent-mode invocation with a non-empty callable set; assert `pi.setActiveTools` is never invoked between `createAgentSession` and `dispose()` on the host `ExtensionAPI`. |
| 7 | `unknown-reason-rule.md` control-character escape rule | Conformance suite fixture set covering at least `\n` тЖТ `\\n`, `\r` тЖТ `\\r`, U+0001 тЖТ `\\u0001`, U+007F тЖТ `\\u007f` in the `details.failure` payload (mirror the `literals-shape-invalid` per-sub-case fixture enumeration verbatim). |
| 8 | `unknown-reason-rule.md` + `session-only-degraded-state.md` empty-registry+unknown-reason conjunction | Fixture with empty `ActiveInvocationRegistry` and `event.reason` outside the closed set; assert via spies that the registry drain, `Promise.allSettled`, both watcher closures, the `Clock.clearTimeout`, and the `ctx.signal.removeEventListener` each ran exactly once. |

**Spec edits.** Inline single-sentence "A conformance test MUST тАж" or "Conformance fixture тАж" additions in the four cited files. No new sections, no new anchors. The control-character clause reuses the `literals-shape-invalid` clause's wording mechanically.

**Pros.** Single source of truth (criterion lives next to the MUST it tests); future edits to the MUST and the criterion stay synchronised; the GOV-22 un-anchored-MUST gate continues to see each obligation; matches the precedent already set by the `literals-shape-invalid` sub-case clause.

**Cons.** Eight inline edits across four files; mild repetition where two obligations share a fixture skeleton.

**Risks.** None material тАФ the precedent edit (`a7d2a80`) demonstrates the pattern lands cleanly.

### Option B тАФ Plan-side: extend Tests bullets on the five affected leaves

Leave the spec text unchanged; add one Tests bullet per obligation to the appropriate plan leaf (and its `-T` paired tests task), so the citing-test discipline forces the fixture into existence at leaf-completion.

**Approach.** Mapping:

- `V9i` / `V9i-T` тАФ obligations 1 (willRetry), 2 (spawn-then-immediate-cancel), 4 (PIC-9 dispose-throws three paths).
- `V9f` / `V9f-T` тАФ obligations 5 (PIC-8 retry cap), 6 (PIC-17 subagent-mode never-calls-`setActiveTools`).
- `V9g` / `V9g-T` тАФ obligation 3 (`invoke`-parent observation rule).
- `V9h` / `V9h-T` тАФ obligation 7 (control-character escapes).
- `V9l` / `V9l-T` тАФ obligation 8 (empty-registry+unknown-reason conjunction). Already blocked on `V18c` independently; the new bullet inherits that block.

Each new Tests bullet cites the obligation inline (e.g. `PIC-9: dispose-throws on Ok-return path тАж`) and mirrors the existing `V9i` PIC-9 bullet's style.

**Spec edits.** None.

**Pros.** Zero spec churn; closes via the existing per-leaf TDD ritual; testability lens stops surfacing the cluster once the bullets land.

**Cons.** Criterion lives one hop away from the obligation (in `plan_topics/`, not `spec_topics/`); a future spec edit to one of the eight MUSTs has no mechanical signal that the matching plan bullet must be re-examined.

**Risks.** Drift between spec wording and plan-leaf assertion over time; the existing GOV-22 un-anchored-normative-MUST gate cannot detect a spec/leaf semantic mismatch (only existence-of-citing-leaf).

### Recommendation

**Take Option A first, then Option B as a follow-on.** Each shape resolves one half of the testability gap and does so additively тАФ Option A closes the spec-side criterion gap once, in the file where the MUST lives, and Option B lifts each criterion into a concrete plan-leaf Tests bullet that gates implementation.

**Order of resolution:**

1. **Option A** тАФ author the eight inline acceptance-criterion edits in the four spec files. This is the scope-bounding edit: it pins the observable per MUST and gives Option B's bullets something authoritative to cite. Resolve obligation 7 (control-character escapes) by reusing the `literals-shape-invalid` clause wording verbatim; the other seven each take one or two sentences. Land as eight separate per-finding fixes so the spec-diff-fix-loop sees one obligation at a time.
2. **Option B** тАФ add one Tests bullet per obligation to the five named leaves (`V9c`/`V9f`/`V9g`/`V9h`/`V9i`/`V9l` and their `-T` pairs). Each bullet cites the freshly-added acceptance-criterion sentence by section anchor. The `V9l` bullet inherits that leaf's existing block on `V18c`; nothing additional is needed there.

**Edge cases for implementers.**

- PIC-9 dispose-throws is genuinely three independent fixtures (Ok / Err / panic teardown), not one parameterised test тАФ the current single-bullet treatment in `V9i` is what enabled the gap.
- PIC-17's subagent-suppression criterion (obligation 6) is a negative invariant; the spy must cover the entire span `createAgentSession` тЖТ `dispose()`, not just the body of `sendUserMessage`, because an incorrect implementation can call `pi.setActiveTools` from a cancellation listener or a `finally` block.
- The control-character fixture set (obligation 7) must include both the named-escape arm (`\n` / `\r` / `\t`) and the `\\uXXXX` arm with at least one C0 and one C1 codepoint to witness both ranges in U+0000тАУU+001F and U+007FтАУU+009F.
- The empty-registry+unknown-reason conjunction (obligation 8) requires spies on five distinct call sites; merely asserting "no fast-path" is insufficient, because the fast-path's observable absence overlaps with the conformant unconditional-run.

## Consequence

**Severity:** advisory

Two conformant implementers can ship green test suites that disagree on observable behaviour at each of these eight MUSTs: the `willRetry` pre-resolution can be missing, the subagent-mode path can silently call `pi.setActiveTools` on the host API and corrupt the user session, the PIC-8 retry can chain into exponential backoff, the empty-registry unknown-reason path can silently take the fast-path. The gap is uniformly testability тАФ the behaviour is pinned, but no fixture is mandated тАФ so the spec ships shippable but the conformance suite cannot tell a buggy implementation from a correct one along these eight axes.

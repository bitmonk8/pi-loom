## Problem

`real-host-smoke-gate.md`'s pass/fail criteria (a)–(e) include two non-surface observables: (c) requires injecting cancellation at a designated point during the live run and observing propagation, and (d) requires confirming the four session-double fidelity-contract axes hold (streamed-token order relative to `ctx.waitForIdle()` resolution, single-turn prompt-mode append, the `pi.on` cancel-forward subscription, and cancellation propagation). These are internal runtime observables, not values a contributor reads off a rendered transcript. The note names no component that drives the fixture `.loom` against a live Pi host, no mechanism that injects the cancel during that run, and no instrumentation that surfaces axis (d)'s internal observables. The only harness that observes those axes — `H4a`'s end-to-end harness — drives the in-process double and disclaims a live Pi session, so the smoke's scoring surface is undefined precisely where it diverges from `H7a`'s in-process gate.

## Issue introduction

**Verdict:** introduced by a specific plan-fix edit (single introducing commit)
**Introducing commit:** `3911733` — "pi-loom plan: resolve \"Live-host smoke pass criterion assumes a non-deterministic LLM reproduces a transcript\"" (2026-06-11)
**History:** The live-host smoke concept first appeared in `328ba4d` (2026-06-10, "resolve \"real-host verification gap\"") with a pass criterion that matched `H7a`'s exact golden transcript and code set — a surface-level, fully-observable criterion. Commit `3911733` rewrote that criterion to score on model-output-invariant observables, introducing the (a)–(e) structure and, with it, (c) "cancellation propagation is observed at the injected cancel point" and (d) the four fidelity-contract axes. That rewrite added internal observables without naming the live-host driver, the cancel-injection mechanism, or the instrumentation that surfaces them, and left adjacent the statement that the harness "deliberately avoids a live Pi session" — so the unnamed-surface gap was present at introduction. Commit `fd22a3d` (2026-06-17) relocated the criteria verbatim from `H4a` into the new `real-host-smoke-gate.md`, carrying the gap unchanged.

## Solution approach

In `real-host-smoke-gate.md`, name the smoke's live driver (the contributor running the fixture against their own Pi install, consistent with the existing Live-host precondition) and define how the cancel is injected during that manual run. Re-express criteria (c) and (d) as externally-observable effects a contributor can score from the live session's rendered transcript and emitted diagnostics, giving each axis-(d) sub-property a concrete observable proxy. Any sub-property with no observable proxy must be explicitly dropped from the smoke or escalated rather than left implicit.

## Solution constraints

- Out of scope: `H4a`'s end-to-end harness, which deliberately avoids a live Pi session — the smoke stays a contributor-manual run and introduces no automated live-driving harness.
- The spec is read-only for this fix (`host-prerequisites.md`, `version-bump-*` pages and other spec topics are not editable here).

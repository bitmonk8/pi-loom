## Problem

The patch-skew degradation contract in `patch-skew-degradation.md` is correctly normative about *what* the runtime and build-time gates must do, but its fixture-obligation paragraphs (P-a, P-b-1, P-b-2, P-c-1, P-c-2) cross from behavioural outcome into test-construction mechanism inside MUST-level prose. (P-c-2) mandates a `tsc`-driven harness "not a vitest case"; P-a, P-b-1, and P-c-1 mandate witnessing the five-sub-step teardown "via spies on the injected `LoomRegistry`, `chokidar`-watcher, `Clock`, and signal-listener seams" and via named per-entry / `disposeBarrier` / `Clock.setTimeout` spies. These mechanism prescriptions contradict the paragraph's own closing disclaimer, which cedes "exact assertion shape" to the implementation, and couple the behavioural contract to one mocking strategy a future injection-seam refactor would needlessly invalidate.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** `a7d2a80` (2026-06-07) — *pi-loom spec: resolve "patch-skew teardown / literals-shape-invalid fixture enumeration"*
**History:** `git log -S 'spies on the injected' -- docs/` shows the phrase first appears in `a7d2a80`; that commit's diff against `patch-skew-degradation.md` adds the entire passage in one shot, including the seam list ("`LoomRegistry`, `chokidar`-watcher, `Clock`, and signal-listener seams"), the per-entry registry-iteration spy, the `disposeBarrier`/`Clock.setTimeout` spy disjunction, and the fresh-`LoomRegistry`-per-case requirement. The commit's stated intent was to give (P-a)/(P-b-1)/(P-c-1) a per-sub-step observable enumeration so a conformance suite witnesses every sub-step; in doing so it cross-referenced the closed `details.call` label set (the observable layer) but also reached past it to pin the verification primitive (the mechanism layer). Notably the same commit's other half — the `literals-shape-invalid` four-sub-case enumeration in `unknown-reason-rule.md` — used the conforming "leaf owns fixture-shape" formula, so the over-prescription is localised to the patch-skew edit rather than reflecting a corpus-wide drift.

## Solution approach

Demote the construction-mechanism clauses in the patch-skew fixture-obligation paragraphs (P-a, P-b-1, P-c-1, P-c-2) from MUST-level prose to non-normative guidance, so the paragraph stops contradicting its own normative-by-reference disclaimer. Keep the behavioural assertions normative — the eight `details.call` call-site coverage on the teardown, the per-sub-step witness obligations, and the build-time diagnostic carrying the verbatim brand string `loom/typecheck/session-shutdown-reason-snapshot` — while converting the named spy-seam observation mechanism and the `tsc`-harness clause to illustrative asides.

## Solution constraints

- Out of scope: relocating or redefining the closed `details.call` label set — owned by F-2162; the teardown-witness assertions only reference it.
- Do not weaken the existing behavioural MUSTs (eight-call-site coverage, the brand string, the "no other failure-mode diagnostic fires" clauses, the fresh-`LoomRegistry`-per-case precondition); only the observation-mechanism clauses are demoted.

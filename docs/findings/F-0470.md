## Problem

In `H1a`'s **Tests** section, the literal-read assertion bullet anchored to `capability-probe.md §(a)` carries a descriptive prose parenthetical `(currently >=22.19.0)` that restates the current value of a Node floor whose canonical owner is already named in the same sentence. This is a second, ungated copy of the floor: after a Node-floor bump the version-bump procedure updates `package.json#engines.node` and the test's transcribed in-code literal, but nothing forces this prose snapshot to move, so it silently drifts to a stale value. The snapshot contradicts the same leaf's anti-restatement discipline, which sources the adjacent `engines.node` **Adds** value and the `typescript >=5.4.0` floor from their anchors rather than restating them.

## Issue introduction

**Verdict:** single-commit introduction
**Introducing commits:** `f1a276a` — "pi-loom plan: resolve \"engines.node literal-read test names no in-code comparison operand\""
**History:** `git log -S'22.19.0'` and `git log -G'22\.19\.0'` over `docs/plan_topics/H1a-scaffold-and-toolchain.md` both return `f1a276a` as the sole commit touching the `22.19.0` token; it never appears earlier in the file's history. That commit was itself the fix for a prior review finding (the `engines.node` test naming no in-code comparison operand): it added the transcribed-literal design to the bullet and, as a side effect, introduced the `(currently >=22.19.0)` prose snapshot alongside it. The defect entered as a by-product of that earlier fix and has been present unchanged since.

## Solution approach

In `docs/plan_topics/H1a-scaffold-and-toolchain.md`, in the **Tests** bullet anchored to `capability-probe.md §(a)`, delete the descriptive parenthetical `(currently >=22.19.0)` so the clause no longer restates the floor value verbatim. The bullet should still name `capability-probe.md §(a)` as the canonical owner and describe the comparison operand as the Node-floor literal transcribed inline into the `H1a` test.

## Solution constraints

- Do not alter the test's transcribed in-code Node-floor literal or the version-bump-alignment sentence — only the redundant prose snapshot in the leaf is in scope.

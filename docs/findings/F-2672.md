## Problem

The `"throw:<String(error)>"` value of the `details.failure` discriminator is pinned as a byte-exact contract (the *Acceptance criteria* section declares the exact discriminator string normative, and PIC-48 lists the template as anchor-stable contract surface). Its control-character escape grammar renders "every other Unicode code point in U+0000–U+001F … and U+007F–U+009F as the six-character escape `\uXXXX`" but never pins the case of the four hex digits. For a control code such as U+007F, both `\u007f` and `\u007F` satisfy the rule yet are different byte sequences, so two conforming implementations produce different `details.failure` strings for the same thrown `error` — breaking the byte-exact claim, making any exact-output conformance fixture non-portable, and splitting operator deduplication on `(code, details.failure)` across two buckets. The same unspecified form is restated in acceptance criterion (b) and in the `loom/host/session-shutdown-pinned-constant-unreadable` row of `diagnostics/code-registry-host.md`, both of which must stay aligned with the owning rule.

**History:** The `"throw:<String(error)>"` discriminator and its control-character escape grammar — including the `\uXXXX` six-character escape with no hex-digit-case pin — were first authored in 75a9626 in the then-monolithic `pi-integration-contract.md`; the defect has been present since that grammar was introduced. The 100KB-cap split (f5e89f4) relocated the text verbatim into `unknown-reason-rule.md`, and 8008b91 (2026-06-25) propagated the same unspecified `\uXXXX` form into acceptance criterion (b) when it added the fixture blocks; neither commit introduced the ambiguity, they carried/duplicated it.

## Solution approach

Pin the four hex digits of the `\uXXXX` escape to lowercase (`0-9`, `a-f`), zero-padded to the full four-digit width (e.g. U+007F → `\u007f`, not `\u7f`), in the *Unknown-reason rule* discriminator-grammar clause. Restate the same lowercase pin in acceptance criterion (b) so the fixture asserts the byte-exact output, and update the restated escape grammar in the `loom/host/session-shutdown-pinned-constant-unreadable` row of `diagnostics/code-registry-host.md` to match.

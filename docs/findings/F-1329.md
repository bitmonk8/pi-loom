## Problem

GOV-16's *Per-page inline-label prefix table* registers the `NOCEIL` prefix against `hard-ceilings.md`, but every `NOCEIL-1` … `NOCEIL-4` anchor site physically lives on the sub-page `hard-ceilings/ceiling-invariants-and-audit.md`. `hard-ceilings.md` is a thin index page carrying no `NOCEIL-N` anchor, yet its *Identifier classes used on this page* paragraph repeats the claim that the prefix is registered there with markers "below". The mis-attribution misroutes the three GOV-16 obligations keyed on the `Page` column — cross-link resolution, per-page progressive normalisation, and `## Retired inline labels` placement — to a file holding none of the anchors.

## Issue introduction

**Verdict:** single-commit

**Introducing commit:** `f5e89f4` — *"docs: enforce 100KB hard / 30KB recommended size cap on spec set"* (2026-06-04).

**History:** GOV-16 was introduced at `c4ad105` (2026-05-08) on the then-monolithic `governance.md`, registering `NOCEIL` on `hard-ceilings.md` — accurate at that time, since `NOCEIL-1` … `NOCEIL-4` and the four-axis audit closure lived directly on `hard-ceilings.md`. `f5e89f4` split the over-size `hard-ceilings.md` into an index page plus two children (`ceilings-3-and-4.md` and `ceiling-invariants-and-audit.md`), moved the four `NOCEIL-N` markers and the *Audit methodology* block into `ceiling-invariants-and-audit.md`, and repointed cross-references — but did not touch the `Page` cell of GOV-16's `NOCEIL` row, which was relocated wholesale into the new `governance/stable-inline-labels.md` extract still naming `hard-ceilings.md`. Subsequent commits on `stable-inline-labels.md` (e.g. `27b2576`, adding `SNK`) preserved the stale row unchanged.

## Solution approach

Rewrite the `NOCEIL` row's `Page` cell in GOV-16's *Per-page inline-label prefix table* to the full repo-relative path `hard-ceilings/ceiling-invariants-and-audit.md`. Clarify GOV-16's introductory paragraph so the `NOCEIL` registration names that sub-page as the anchor site rather than `hard-ceilings.md`. In `hard-ceilings.md`'s *Identifier classes used on this page* paragraph, rewrite the "registered on this page … markers below" phrasing for `NOCEIL` into a forward-reference to the sub-page.

## Solution constraints

- Out of scope: the `HC3` prefix registration in the same table and intro sentence — owned by F-1328.
- Do not move or retitle the `NOCEIL-1` … `NOCEIL-4` anchor sites; only the registration metadata is wrong, and moving the anchors would break the existing inbound `…#noceil-n` cross-links.

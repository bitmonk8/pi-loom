## Problem

The `loom/runtime/watcher-terminated` row in `code-registry-runtime.md` carries a normative invariant in its Trigger cell — when the file watcher terminates and the adapter leaves it torn down, no `LoomRegistry` drain-state tag is written — but states it purely in implementation terms (a prohibition on an internal write) with the observable consequence given only as rationale. PIC-55 states the same rule as a MUST NOT. No spec text turns the consequence into a falsifiable pass/fail check, leaving the invariant asserted but unverified at the spec level. The rule is in fact observable two ways: directly via `LoomRegistry.readDrainState()`'s returned `tag` field, and indirectly because a wrongly-written tag would satisfy the `session_shutdown` handler-entry short-circuit predicate and suppress a later `loom/host/session-shutdown-runtime-degraded` emission.

## Issue introduction
**Verdict:** single-commit
**Introducing commits:** `0b195eb` (2026-06-25, "pi-loom spec-plan: fix F-1299+F-1308 — coin PIC-55 watcher post-error posture + loom/runtime/watcher-terminated")
**History:** `git log -S "watcher-terminated" -- docs/spec_topics/diagnostics/code-registry-runtime.md` and `git log -G "drain-state tag is written, preserving" -- docs/spec_topics/diagnostics/code-registry-runtime.md` both return `0b195eb` as the sole commit touching the row; the coining commit also authored the matching PIC-55 paragraph in `registration-steps.md`. The row and its no-tag-write rule were written in implementation-terms-only form, with the short-circuit consequence as rationale, in that single commit. `git log -S "Given the chokidar watcher has terminated" -- docs/` returns nothing, confirming no later commit ever added an acceptance criterion. The testability gap is therefore an authoring omission present in the row since its introduction, not a regression of a later edit.

## Solution approach

Add a falsifiable acceptance criterion to the `loom/runtime/watcher-terminated` row in `code-registry-runtime.md` (or a normative note it references) for the no-tag-write invariant. Pin the direct observable on `LoomRegistry.readDrainState()` (`drain-state-contract.md#loom-registry-drain-state-contract`) returning the steady-state tuple `(drained: false, tag: undefined)` after the event, corroborated by `loom/host/session-shutdown-runtime-degraded` being emitted on a subsequent session-only `session_shutdown` (`event.reason ∈ {"new","resume","fork"}`). Scope the corroboration to a first-entry `session_shutdown` with a successful pinned-constant read so the all-three-throw and `readDrainState`-throw re-entry corner cases (`#read-failure-fallback`), which legitimately re-emit `runtime-degraded`, are not conflated with the falsifier.

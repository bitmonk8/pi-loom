## Problem

`H1a`'s **Adds** declares `minimatch` in loom's runtime `dependencies` "as the glob matcher engine for `pi.looms` resolution — the `!`/`+`/`-` override ordering and exact-path operand treatment", attributing all of that behaviour to the library. The DISC-5 contract the same clause cites (`discovery/package-and-settings.md` §DISC-5) draws a sharper line: minimatch supplies only the glob dialect used to test plain-include and `!`-exclude patterns, while the four-class `!`/`+`/`-` override ordering and the exact-path `+`/`-` operand treatment (Pi-style `matchesAnyExactPattern`, not glob matching) are loom's own layer. The clause therefore conflates three distinct concerns and contradicts its own cited source.

## Issue introduction

**Verdict:** present-since-introduction
**Introducing commit:** `8279cb15cff893b288c3d02afcab893426a66797` (2026-06-11) — "pi-loom plan: resolve \"minimatch runtime dependency required by V10b is never provisioned\""
**History:** `git log -S` over the H1a leaf shows the clause "the `!`/`+`/`-` override ordering and exact-path operand treatment" first appeared in `8279cb1`, the same commit that introduced the `minimatch` dependency to H1a's **Adds** to close an unrelated provisioning gap. The mischaracterisation has existed unchanged since that single authoring; no later commit touched it. The defect was introduced incidentally by the prose written to justify the new dependency, not by a subsequent edit or a multi-commit interaction.

## Solution approach

Rewrite the `minimatch` clause in `H1a`'s **Adds** so the library is credited only as the glob-dialect engine loom uses to match plain-include and `!`-exclude patterns, and so the `!`/`+`/`-` override ordering and the exact-path `+`/`-` operand treatment are attributed to loom's own layer pinned in `discovery/package-and-settings.md` §DISC-5. Keep the §DISC-5 anchor and the `V10b` consumption pointer. Optionally mirror the same attribution in `V10b`'s **Adds** shorthand in the same pass; the H1a clause is the load-bearing edit.

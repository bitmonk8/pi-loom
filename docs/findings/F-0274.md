## Finding

`V9c`'s `PIC-18` Tests bullet carries an inline clause — "forwarding into the `loomAbort` controller owned by V17a" — that the paired `V9c-T` Tests bullet does not. The two PIC-18 bullets are otherwise byte-identical: both assert the prompt-mode `pi.on` subscription is process-global, has no per-session marker, and is used only for cancel-forwarding, never for completion.

Under the project's TDD pairing convention (`conventions.md` §"MVP and vertical features — tests-task → implementation-task pairing"), `V9c-T` is the tests-task driver: the test author writes one assertion per REQ-ID from the `-T` leaf's Tests bullets, and the paired implementation leaf's green tests are the coverage-matrix closure evidence. Any detail that lives only in the implementation leaf's mirror bullet — here, the destination of the cancel-forward — never reaches the test author.

The behaviour itself is not at risk: both leaves list `V17a` in **Deps**, both carry the cancellation short-circuit Tests bullet referencing `loomAbort.signal.aborted`, and `V9c`'s **Adds** still names "forwarding Pi's `ctx.signal` into the `loomAbort` controller owned by V17a". The divergence is therefore a textual asymmetry between the two PIC-18 bullets, not an unimplemented or untested obligation.

## Issue introduction

**Verdict:** multi-commit-interaction

**History:** The PIC-18 bullets were identical in both leaves at corpus creation (c6a664e). Commit ecedd5f resolved a V9c-coupling finding by adding `V17a` to `V9c`'s **Deps** and, in the same edit, embellishing `V9c`'s **Adds** and its PIC-18 Tests bullet with the inline "forwarding into the `loomAbort` controller owned by V17a" clause — touching `V9c` only. The next morning, commit 3462575 resolved the companion "V9c-T omits V17a for PIC-18 cancel-forwarding" finding, but closed only the **Deps** gap (`V17a` added to `V9c-T`'s Deps) and left the PIC-18 bullet text untouched. The two fixes together closed the dependency gap on both leaves while leaving the PIC-18 bullet text divergent — the asymmetry is the residue of the second fix not mirroring the first fix's bullet-text edit.

## Solution Space

**Shape:** multiple

### Option A — Strike the inline clause from `V9c`

**Approach.** Remove the V17a-destination clause from `V9c`'s PIC-18 Tests bullet so it reads identically to `V9c-T`'s.
**Plan edits.** In `docs/plan_topics/V9c-conversation-drive.md`, the PIC-18 Tests bullet, replace the trailing phrase `used only for cancel-forwarding — forwarding into the `loomAbort` controller owned by V17a — never for completion.` with `used only for cancel-forwarding, never for completion.` (matching `V9c-T`'s PIC-18 bullet verbatim). Leave `V9c`'s **Adds** clause and `V17a` **Deps** entry intact.
**Spec edits.** None.
**Pros.** Smallest diff; keeps the test driver's PIC-18 bullet authoritative; the V17a destination remains captured in Adds, Deps, and the cancellation short-circuit bullet.
**Cons.** The destination detail no longer appears inline in the implementation leaf's PIC-18 bullet (still recoverable from Adds).
**Risks.** None of consequence; behaviour and Deps unchanged.

### Option B — Add the clause to `V9c-T`

**Approach.** Mirror the V17a-destination clause into `V9c-T`'s PIC-18 Tests bullet so the test author sees it.
**Plan edits.** In `docs/plan_topics/V9c-T-conversation-drive.md`, the PIC-18 Tests bullet, insert the clause `— forwarding into the `loomAbort` controller owned by V17a —` so the bullet reads identically to `V9c`'s current PIC-18 bullet.
**Spec edits.** None.
**Pros.** Makes the V17a forwarding destination an explicit, test-author-visible part of the PIC-18 assertion.
**Cons.** The destination assertion overlaps with the cancellation short-circuit Tests bullet (already present in `V9c-T`), so PIC-18 and the short-circuit bullet both gesture at `loomAbort`.
**Risks.** None of consequence.

### Recommendation

Take Option A: strike the inline clause from `V9c`'s PIC-18 Tests bullet so it matches `V9c-T` verbatim. `V9c-T` is the tests-task driver and its PIC-18 bullet defines the assertion the test author writes; the V17a-forwarding destination is already carried by `V9c`'s **Adds**, by both leaves' `V17a` **Deps**, and by the cancellation short-circuit Tests bullet present in both leaves, so removing the duplicate destination phrase from PIC-18 loses no obligation. The spec is read-only for this fix. Watch that `V9c`'s **Adds** clause and `V17a` **Deps** entry are preserved — only the PIC-18 Tests-bullet phrasing changes.

## Consequence

**Severity:** cosmetic

A reviewer or test author comparing the two paired leaves sees a PIC-18 bullet that reads differently on the implementation side than on its test driver, which invites doubt about whether a distinct assertion was dropped. No implementer diverges in behaviour: the V17a-forwarding destination is already carried by `V9c`'s Adds, by both leaves' `V17a` **Deps**, and by the cancellation short-circuit Tests bullet present in both leaves.

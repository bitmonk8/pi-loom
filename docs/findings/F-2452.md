## Problem

The registered diagnostic `loom/load/cross-format-collision` fires for the entire same-priority slash-name collision rule — both the cross-format arm (`.loom` vs a Pi-owned `.md` prompt / `.md` skill / another extension's command) and the same-format arm (two `.loom` files at the same priority) — but its `cross-format` token announces only the first arm. The mislabel propagates into `active-invocation-registry.md`, which cites the rule twice as "[Discovery — Slash-name collisions across formats]" for references that resolve the same-format `.loom`-vs-`.loom` arm, while the page's other references to the same target read "[Discovery — Slash-name collision rules]". The code itself cannot be renamed in loom 1.0: DIAG-3 makes every registered code a stable public-contract identifier, and a rename is a breaking change deferred to loom 2.0 outside the GOV-15 add/remove carve-out. The actionable 1.0 work is editorial.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** `d124b8c` (2026-05-05, "spec: same-priority slash-name collisions are uniform load-time errors (Decision #10)")
**History:** `git log -S 'cross-format-collision'` shows the code entered as a cross-format-only diagnostic — through `c5c4e79` (2026-05-04, "resolve 'Cross-format slash collision rule cannot achieve neither is registered'") its trigger read "A `.loom` candidate's slash name collides with an already-registered Pi prompt template … the loom is dropped", and the name was accurate. `d124b8c` (Decision #10) expanded the same code to "Two or more candidates at the same priority derive the same slash name. Covers both the same-format case … and the cross-format case", folding the loom-vs-loom arm into the existing code while retaining the `cross-format` token. The same commit renamed the discovery.md section heading to "Slash-name collisions at the same priority" but left the `cross-format` code name and the older "across formats" cross-reference label in place — the moment the name/scope mismatch and the stale link text were introduced.

## Solution approach

Rewrite the inaccurate "collisions across formats" cross-reference link text in `active-invocation-registry.md` to match the DISC-4 bold lead-in heading in discovery-sources.md, aligning it with the page's other references to the same target. Surface the dual-arm scope clarification earlier in the `loom/load/cross-format-collision` Trigger cell in code-registry-load.md so the full same-priority scope is read ahead of the cross-format detail. Record the intended loom 2.0 rename to `loom/load/slash-name-collision` in future-considerations.md so the deferral is tracked.

## Solution constraints

- Out of scope: the code token, its severity (`E`), the DISC-4 behaviour, and the rendered message template `slash name '<name>' collides at the same priority: <paths>` — all are stable public contract under DIAG-3.
- The `discovery.md` cross-reference target is an aggregator page with no per-section anchor; do not invent an anchor fragment (no-invented-ids).

## Problem

The DISC-4 "Slash-name collision rules" paragraph in `discovery-sources.md` opens with a universally-quantified rule — "none of the colliding entries register" — that ranges over both loom-vs-loom and loom-vs-Pi-owned (cross-format) collisions, then closes the same paragraph with a carve-out stating that for the cross-format slice only the colliding loom(s) drop and the Pi-owned entry survives. A clause-by-clause reading therefore yields two incompatible behaviours for the cross-format slice: drop-all (universal opening authoritative) versus drop-loom-only (carve-out authoritative). An implementer who reads the universal opening as the rule statement will attempt to unregister the Pi-owned prompt template / skill / sibling-extension command on collision, which Pi exposes no API to do.

## Issue introduction

**Verdict:** present-since-introduction-of-current-rule
**Introducing commit:** `d124b8c` — "spec: same-priority slash-name collisions are uniform load-time errors (Decision #10)" (Tue May 5 2026)
**History:** The pre-`d124b8c` paragraph was titled *Slash-name collisions across formats* and described only the cross-format arm (loom drops, Pi-owned survives) — no universal "none of the colliding entries register" claim existed, so no contradiction was reachable. `d124b8c` generalised the section to *Slash-name collisions at the same priority* covering both same-format and cross-format arms; in the same edit it introduced the bold universal opening "**none of the colliding entries register**" *and* the trailing cross-format carve-out "the Pi-owned entry survives — only the colliding loom(s) drop." The contradiction has therefore been present since the rule's current shape was introduced — five later edits to the same paragraph (`7e440c9`, `d488eaa`, `f2a82ad`, `4a10cde`, `f5e89f4`) refined the diagnostic code reference, the hyphen-normalisation clause, the `de-register` wording, and the file split, but none touched the opening/carve-out tension. `git log -S "none of the colliding entries register" -- "*.md"` confirms `d124b8c` as the first appearance of the universal opening.

## Solution approach

Restructure the DISC-4 lead paragraph into two explicitly-scoped collision arms — loom-vs-loom (same priority), where every colliding loom drops; and loom-vs-Pi-owned (cross-format), where only the colliding loom(s) drop and the Pi-owned entry stays registered (the loom extension cannot unregister Pi-owned commands; forward-link `registration-steps.md#structural-changes-no-unregister`). Remove the universally-quantified opening and the trailing carve-out together so neither sentence is readable as authoritative in isolation. Move the "three packages each shipping `lint.loom`" example into the loom-vs-loom arm so it stays anchored to the same-format slice. Keep the existing follow-up material as elaboration that both arms anchor into.

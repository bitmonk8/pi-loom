## Problem

The step-3 three-way `engines.node` equality in `version-bump-steps-3-4.md` defines only the case where the upstream floor moves — operand (iii) changes value while operands (i) and (ii) stay pinned. It leaves undefined what happens when the candidate package omits `engines.node` entirely: operand (iii) is then absent, and the "copy the new floor" remedy has no floor to copy. Two implementers can converge on different behaviours for the absent-field case at the gate.

## Solution approach

Add an explicit arm to `version-bump-steps-3-4.md` step 3 for operand (iii) being absent: the test fails red, and resolution routes to editorial review rather than a mechanical literal re-copy. State within that arm the disposition of the loom `package.json#engines.node` literal and the `pi-engines-node` inventory row pending the review — whether left unchanged or set to a defined sentinel — so the absent-field path resolves deterministically.

## Problem

The **Conversation drive — subagent mode** paragraph in `subagent.md` fully authors the success arm of a subagent untyped query at `#subagent-untyped-query-ok-extraction` (await `session.subscribe` for the terminal `agent_end`, ignore `willRetry: true` events, read `messages: AgentMessage[]`, assemble `Ok(string)` by the final-assistant-text rule) but names no failure arm — neither a detection surface nor a `QueryError` mapping for a subagent driven turn that fails for transport/provider reasons. The prompt-mode counterpart at `conversation-drive.md#prompt-mode-error-detection` pins both halves; the subagent paragraph pins only the success half, breaking the "same rule on both sides" symmetry the success arm advertises. The prompt-mode probe does not transfer verbatim because subagent mode differs on every relevant axis: `AgentSession.sendUserMessage` returns `Promise<void>` (so a rejection is observable at the call site), the driver awaits a session-local `subscribe` rather than `ctx.waitForIdle()`, and a failed turn appears in the terminal `agent_end` event's `messages` array rather than on `ctx.sessionManager`. The missing clause is therefore normative content the spec lacks, not behaviour implied by the prompt-mode rule; implementers reading the section in isolation would disagree on observable behaviour (`Ok("")`, an uncaught rejection, or an ad-hoc `QueryError` kind).

## Issue introduction

**Verdict:** multi-commit interaction (regression by omission)
**Introducing commits:**
- `024215d` (2026-06-01) `pi-loom spec: resolve "untyped-query Ok(string) extraction mechanism (prompt + subagent)"` — added the success-arm extraction symmetrically on both sides.
- `d37c563` (2026-06-05) `pi-loom spec: resolve "Prompt-mode errorMessage probe unreachable from specified surface"` — replaced the prompt-mode `AgentSession.errorMessage` probe with the trailing-assistant-message `stopReason` / `errorMessage` probe and anchored it as `#prompt-mode-error-detection`. The same edit deliberately scoped the subagent forced-respond-turn discussion (`complete()` lives off-session), but did **not** restate the new probe symmetrically on the subagent untyped arm.

**History:** Before `024215d` the success arm itself was unpinned on both sides — the symmetry defect did not exist because nothing was pinned. `024215d` made the success arm symmetric but did not touch the failure arm (it was authored as a focused success-extraction fix, mirroring T23/T24 of the then-current spec review, which scoped only the success path). `d37c563` then meaningfully rewrote the prompt-mode failure-detection surface (the old `AgentSession.errorMessage` probe was unreachable through `ExtensionContext`; the new probe reads the trailing assistant message via `ctx.sessionManager`). That rewrite extended the prompt-side authority of the failure arm but did not propagate the symmetric clause back into `subagent.md` — partly because the new prompt-mode formula does not transfer verbatim (the subagent driver awaits a session-local `subscribe`, not `waitForIdle()`, and the failed turn appears in the terminal `agent_end` event's `messages` array on the spawned `AgentSession`, not on `ctx.sessionManager`). The defect is therefore the interaction of the success-arm symmetry committed by `024215d` with the prompt-mode failure-arm rewrite committed by `d37c563`: each commit was locally coherent, but together they left the subagent failure arm visibly absent next to a fully-authored subagent success arm and a fully-authored prompt-mode failure arm.

## Solution approach

Add a failure-arm clause in `subagent.md` mirroring `conversation-drive.md#prompt-mode-error-detection`, running before the success-arm extraction at `#subagent-untyped-query-ok-extraction`, that covers the three subagent-mode failure channels: the cancellation short-circuit (consuming the `loomAbort.signal.aborted` precedence owned by `../cancellation.md`), the rejection of the awaited `session.sendUserMessage(text)` `Promise<void>` at the call site, and the terminal-`agent_end` trailing-assistant-message probe (`stopReason: "error"` mapping to `QueryError { kind: "transport" }`, with the fixed `"provider transport failure"` string when `errorMessage` is absent). Cross-reference `#prompt-mode-error-detection` so the symmetry is auditable from either side, and add a forward-link from the `AgentSession` member surface at `host-interfaces-core.md#agentsession-interface` to the new clause. In `version-bump-step2.md`, audit whether the failed-turn `agent_end` presupposition warrants a SHOULD-level checklist item paralleling items (h)/(n), or state explicitly that an existing item covers it.

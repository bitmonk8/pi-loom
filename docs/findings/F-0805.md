## Problem

The **Conversation drive — subagent mode** paragraph in `subagent.md` fully authors the *success* arm of an untyped subagent query at `#subagent-untyped-query-ok-extraction`, but pins no *failure* arm: no detection surface, no `QueryError` mapping for transport / provider / retry-exhausted failures, and no cancellation-precedence rule. The prompt-mode counterpart at `conversation-drive.md#prompt-mode-error-detection` is symmetric on both halves. As a result a subagent query whose terminal `agent_end` carries an empty or error-flagged `messages` array resolves to `Ok("")` — indistinguishable from a clean turn with no assistant text — and a `sendUserMessage` rejection has no specified mapping at all. PIC-9's lifecycle bullet already presupposes the failed-query → `Err` mapping that the spec never defines, so two reasonable implementers diverge on observable behaviour.

## Issue introduction

**Verdict:** multi-commit interaction (regression by omission)
**Introducing commits:**
- `024215d` (2026-06-01) `pi-loom spec: resolve "untyped-query Ok(string) extraction mechanism (prompt + subagent)"` — added the success-arm extraction symmetrically on both sides.
- `d37c563` (2026-06-05) `pi-loom spec: resolve "Prompt-mode errorMessage probe unreachable from specified surface"` — replaced the prompt-mode `AgentSession.errorMessage` probe with the trailing-assistant-message `stopReason` / `errorMessage` probe and anchored it as `#prompt-mode-error-detection`. The same edit deliberately scoped the subagent forced-respond-turn discussion (`complete()` lives off-session), but did **not** restate the new probe symmetrically on the subagent untyped arm.

**History:** Before `024215d` the success arm itself was unpinned on both sides — the symmetry defect did not exist because nothing was pinned. `024215d` made the success arm symmetric but did not touch the failure arm (it was authored as a focused success-extraction fix, mirroring T23/T24 of the then-current spec review, which scoped only the success path). `d37c563` then meaningfully rewrote the prompt-mode failure-detection surface (the old `AgentSession.errorMessage` probe was unreachable through `ExtensionContext`; the new probe reads the trailing assistant message via `ctx.sessionManager`). That rewrite extended the prompt-side authority of the failure arm but did not propagate the symmetric clause back into `subagent.md` — partly because the new prompt-mode formula does not transfer verbatim (the subagent driver awaits a session-local `subscribe`, not `waitForIdle()`, and the failed turn appears in the terminal `agent_end` event's `messages` array on the spawned `AgentSession`, not on `ctx.sessionManager`). The defect is therefore the interaction of the success-arm symmetry committed by `024215d` with the prompt-mode failure-arm rewrite committed by `d37c563`: each commit was locally coherent, but together they left the subagent failure arm visibly absent next to a fully-authored subagent success arm and a fully-authored prompt-mode failure arm.

## Solution approach

Add a failure-detection clause to the **Conversation drive — subagent mode** paragraph in `subagent.md`, placed upstream of the `#subagent-untyped-query-ok-extraction` success-arm extraction so the success arm is reached only when no failure channel fires, mirroring the symmetry of `conversation-drive.md#prompt-mode-error-detection`. Cover the three subagent failure channels: a cancellation short-circuit on `loomAbort.signal.aborted` mapping to `Err(QueryError { kind: "cancelled" })` per `cancellation.md`; a rejection of the awaited `session.sendUserMessage(text)` `Promise<void>`; and a `stopReason: "error"` probe of the trailing `assistant` message in the terminal `agent_end` event's `messages: AgentMessage[]` array, mapping to `Err(QueryError { kind: "transport", ... })` symmetric with the prompt-mode probe. Pin a failed-turn `agent_end` delivery presupposition on the same footing as the success arm's ordering presupposition, add a forward-link from `host-interfaces-core.md#agentsession-interface` to the new clause, and add (or record existing coverage of) the corresponding SHOULD-level editorial-review item in `version-bump-step2.md`.

## Solution constraints

- Out of scope: the rule-1 built-in tool name→factory mapping (owned by F-0801) and the all-empty `ResourceLoader` adapter discussion (owned by F-1030) in the same paragraph.
- A late rejection of the discarded `AgentSession.abort()` promise MUST NOT be promoted to a `QueryError` (governed by Cancellation's swallowing-handler rule); the new failure clause does not consume that surface.

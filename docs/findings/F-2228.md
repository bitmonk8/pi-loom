## Problem

The `## Accepted loom 1.0 residual gaps` section of `session-only-degraded-state.md` opens with explicit decision-log framing ("documented so a future edit that reduces any contributing cost can be assessed against this acknowledged baseline") and its three numbered entries are written as decision-log records — each describing a failure scenario, then justifying the absence of compensating behaviour with the recurring "Accepted (rather than &lt;alternative&gt;) because the assessed probability … is low" construction. This is design-decision rationale and probability-assessment commentary living inline in a page whose surrounding sections are tight normative obligations, matching the anti-decision-log convention applied elsewhere in the corpus. The complication: gaps 1 and 2 are pure rationale, but gap 3 (anchor `#substep-2-stamp-throw-residual-gap`) interleaves genuinely normative pins — the `"<unreadable>"` sentinel substitution for `details.event.reason` (with `details.event.loom` unaffected) and the `cancelled-by-session-shutdown` ↔ `reload-teardown-timeout` co-emission carve-out — both cited from elsewhere, so a blunt relocation would orphan live normative content.

## Solution approach

Rewrite the `## Accepted loom 1.0 residual gaps` section in place — keeping the enumeration so reviewers still recognise these as known-accepted — and delete the decision-log framing sentence and the "Accepted (rather than …) because the assessed probability is low" justification clauses from gaps 1, 2, and 3, leaving only the observable residual symptom and the operator's recovery/observability signal for each. Retain gap 3's `<unreadable>` sentinel-substitution sentence (with `details.event.loom` unaffected) and `cancelled-by-session-shutdown` ↔ `reload-teardown-timeout` co-emission sentence in place, along with the `#substep-2-stamp-throw-residual-gap` anchor. Drop gap 1's forward-link prose to the Pi stdio-capture facet where it survives only as rationale.

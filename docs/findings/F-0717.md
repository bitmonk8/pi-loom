## Problem

`host-prerequisites.md` §`pi-sdk-pin` (item 1) states that loom's minor-lock-step line is "inherited from" `@earendil-works/pi-coding-agent`'s `dependencies` block, which it describes as pinning the three siblings (`pi-agent-core`, `pi-ai`, `pi-tui`) at the same `~X.Y.Z` (tilde) line. Upstream actually pins those siblings with caret (`^X.Y.Z`); the two ranges coincide only because Pi is in `0.x`, and the inheritance claim becomes false once any package crosses `1.0.0`, at which point caret admits the entire major. Loom's own `peerDependencies` correctly pin all four `@earendil-works/pi-*` entries at the tilde range, so the operative minor-lock-step guarantee for loom consumers is loom's own tilde pin, not derivation from upstream.

## Issue introduction

**Verdict:** specific-commit
**Introducing commit:** `c307a62` ("pi-loom spec: resolve \"^X.Y.Z prose label minor-version line only coincides with npm caret semantics while Pi is in 0.x\"", 2026-05-07)
**History:** Before `c307a62`, the same paragraph correctly described upstream as pinning the three siblings at `^X.Y.Z` (matching `@earendil-works/pi-coding-agent`'s `dependencies` block, which has used caret throughout). `c307a62` flipped loom's own `peerDependencies` from caret to tilde — the correct edit, motivated by npm caret no longer coinciding with "lock to minor" once Pi crosses `1.0.0` — and in the same diff rewrote the upstream-inheritance description from `^X.Y.Z` to `~X.Y.Z`, projecting the loom-side change onto the upstream description rather than re-inspecting upstream. The subsequent pin moves (`f985a17` → `~0.74.1`, `ae529db` → `~0.75.5`, `5e73072` scope rename to `@earendil-works`) and the May-2026 spec split (`81bf52a`) carried the defective `~X.Y.Z` text forward verbatim; none of them touched the inheritance claim.

## Solution approach

Rewrite the inheritance sentence in §`pi-sdk-pin` (item 1) to state upstream's actual pin shape — `@earendil-works/pi-coding-agent`'s `dependencies` block pins the three siblings with caret (`^X.Y.Z`), not tilde — and re-anchor the minor-lock-step guarantee on loom's own `peerDependencies` tilde range (the literal already asserted by the H1a four-peer shared-line architectural test) rather than on derivation from upstream. Frame the upstream caret as a present-day coincidence that holds only while Pi is in `0.x`, so the prose stays accurate after a future Pi `1.x` release widens upstream's caret semantics.

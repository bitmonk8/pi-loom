## Finding

V9l's **Ships when** field is a two-arm conditional. The first arm is the actionable completion gate: "Once clause (a) is resolved *branch reachable*, `npm test` exercises the degraded path with its code … tag the commit `V9l-complete`." The second arm is a trailing parenthetical — "(If clause (a) instead resolves *branch unreachable*, the pre-designed fallback in [`plan.md` §Blocked obligations](../plan.md#blocked-obligations) governs what ships.)" — that does not state an observable pass/fail criterion for this leaf. It instead names a different disposition (the leaf collapses to full teardown + `/reload`, with degraded-branch obligations reworked spec-side) that takes effect only if the open clause-(a) contradiction resolves the other way.

A **Ships when** field is the leaf's observable completion gate. The `branch unreachable` arm is not something an implementer can observe or test against on this leaf — it is a contingency that would re-scope the leaf entirely. That contingency is already documented authoritatively in `plan.md` §Blocked obligations (the V9l bullet's "Pre-designed fallback" sentence), which the leaf's **Deps** field already points to as the blocking-resolution home. Carrying the second arm inline duplicates that pointer's content into the completion gate and muddies which condition actually closes the leaf.

The leaf is blocked on the clause-(a) resolution regardless (its `Deps` edge on `V18c` gates pickability), so this does not break execution — the cost is a completion gate that reads as conditional on an unresolved decision rather than as a single observable criterion.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V9l-session-only-degraded-branch.md`, **Ships when** field, strike the trailing parenthetical contingency arm:

> (If clause (a) instead resolves *branch unreachable*, the pre-designed fallback in [`plan.md` §Blocked obligations](../plan.md#blocked-obligations) governs what ships.)

Leave the `branch reachable` arm intact, ending the field at "… and the predicate split; tag the commit `V9l-complete`." No information is lost: the leaf's **Deps** field already links to `plan.md` §Blocked obligations, and that section's V9l bullet already carries the authoritative `branch unreachable` fallback (full teardown + `/reload`, with `session-only-degraded-state.md` / `SM-4`/`SM-5`/`SM-6`/`SM-3b` reworked spec-side). The fix is confined to the V9l leaf; do not edit `plan.md` §Blocked obligations or the spec.

## Consequence

**Severity:** advisory

The completion gate reads as conditional on an unresolved open decision rather than as a single observable pass/fail criterion, and duplicates the contingency already recorded in `plan.md` §Blocked obligations. Pickability and the actionable `branch reachable` arm are unaffected, so no implementer produces a wrong leaf; the cost is reader friction and a two-source maintenance burden if the fallback wording changes.

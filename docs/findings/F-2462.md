## Problem

The Trigger column of the `loom/runtime/internal-error` row in
`code-registry-runtime.md` writes the slash-command system note as
`"loom /<name> aborted with internal error: <message>"`, contradicting the
systematic panic framing the same row claims to share ("Routed alongside
panics"). `error-model.md`'s "Panic message string (normative)" fixes exactly
one message string per panic, with the `"loom /<name> aborted: "` prefix
*wrapping* the row's Message-template value rather than replacing it — yielding
`loom /<name> aborted: internal error: <error.message>`. The row's prose
replaces that prefix and is ambiguous about its `<message>` token, so two
implementers emit observably different bytes for the same DIAG-4-versioned
delivery and their conformance tests cannot both pass.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** f8f4813 — pi-loom plan: resolve "Panic routing has no defined path for unexpected interpreter exceptions" (2026-05-05, Thomas Andersen)
**History:** The `loom/runtime/internal-error` row was added in f8f4813, born with both its Message template `internal error: <error.message>` and the divergent Trigger-prose slash-note string `"loom /<name> aborted with internal error: <message>"` in the same diff. The systematic panic-message convention it contradicts (`"exactly one message string per panic"`, with the `aborted: ` prefix wrapping the message) already existed one day earlier (0c230a1, 2026-05-04) and was reinforced in error-model.md the following day (5c8d782, 2026-05-06), so the row diverged from an established convention at the moment it was written.

## Solution approach

Reconcile the Trigger column's slash-note string with `error-model.md`'s
"Panic message string (normative)" framing in `#runtime-panics`: either rewrite
the inline `"loom /<name> aborted with internal error: <message>"` to the
systematic `"loom /<name> aborted: <message>"` form with `<message>` being the
row's Message-template value, or delete the literal and forward-link
`error-model.md`'s "Panic message string (normative)" as the owner of the
slash-surface framing. Leave the Message-template column unchanged as the
DIAG-4 normative source.

## Problem

The *Enumeration mechanism — grep target* clause of step 5 in `version-bump-triggers.md` pins the inbound-reference sweep, and its build-time audit gate, to a closed literal set of three grep roots (`src/`, `test/`, `docs/`). The clause's justification covers new subdirectories under those roots but presupposes — without stating it as a precondition — that the entire Pi-touching runtime footprint will forever live under exactly those three roots. If a future layout change adds a sibling top-level code directory (e.g. `lib/`, `cli/`, `runtime/`) carrying a `SessionShutdownEvent['reason']` reference, both the contributor recipe and the audit gate silently skip it — the exact silent-miss the sweep exists to prevent. The asymmetry is sharp: the same paragraph already mandates a forward-compatibility extension for a new *reference shape*, but a new *grep root* triggers nothing.

## Issue introduction
**Verdict:** single introducing commit
**Introducing commits:** `52b9101` (2026-05-11, "pi-loom spec: resolve \"T24 — Fork-reason watcher closure leaves the extension in an unspecified, silently degraded state\"")
**History:** `git log -S 'Enumeration mechanism — grep target'` and `git log -G 'grep across the layout-invariant superset'` both bottom out at `52b9101`, which authored the entire inbound-reference sweep whole-cloth — the three-root grep target (`src/`, `test/`, `docs/`), the contributor-side `rg` recipe, and the "MUST extend this second-stage anchor set" forward-compatibility obligation for reference shapes. The same commit's diff already contains the anchor-set extension obligation but no parallel grep-root extension obligation, and `git log -G 'new top-level code directory'` returns no commit, confirming the grep-root extension obligation was never added in any later edit. The asymmetry has therefore been present since the feature entered the corpus; it is a gap baked into the original authoring rather than a regression introduced by a later change.

## Solution approach

In the *Enumeration mechanism — grep target* clause of step 5, add a forward-compatibility obligation for the grep-root set that mirrors the existing second-stage anchor-set extension obligation: a future edit that introduces a new top-level runtime code directory outside `src/`/`test/`/`docs/` carrying an inbound `SessionShutdownEvent['reason']` reference MUST extend the sweep's grep-root set in the same edit, binding both the contributor-side recipe and the build-time audit gate so their root sets cannot diverge. State the current three-root set explicitly as the loom 1.0 closed-root presupposition so the extension trigger is unambiguous. Scope the trigger to top-level trees carrying runtime references; pure-documentation siblings under `docs/` are already subsumed.

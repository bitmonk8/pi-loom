## Problem

Checklist item `(ah)` (anchor `id="bump-checklist-pi-on-process-global"`) opens with the SHOULD-level framing shared by every (f)–(ak) entry, then switches register to a bare imperative — "Falsification of either half independently invalidates a distinct PIC-18 argument, so the contributor MUST audit both." No other entry in the (f)–(ak) block embeds a MUST. The bare MUST admits two incompatible readings: a conditional one (the MUST is governed by the enclosing SHOULD, so both halves must be covered only when the auditor performs the (ah) audit) and an absolute one (the MUST elevates (ah) into the (a)–(e) MUST-audited tier). The two readings yield different conformance verdicts on the same auditor behaviour and reroute the V18c merge-gate disposition between non-blocking and blocking; the intended reading is the conditional one, but the surface wording does not say so.

Subsequent commits to the file (`60fba43` adding `(ai)`, `9ced9f1` adding `(aj)`, `85fc906` adding `(ak)`, `81bf52a` collapsing `spec/` into `spec_topics/`) did not alter the `(ah)` body. The modal conflict is therefore not the residue of a later append disturbing earlier wording; it was present on first landing.

## Solution approach

Rewrite the embedded MUST sentence in item `(ah)` so the obligation is explicitly bound to the SHOULD-gated antecedent — the both-halves coverage requirement applies only when the auditor performs the (ah) audit — and so the SHOULD-tier classification is unambiguous against the preamble's recording scope. Skipping the audit entirely must remain a conformant outcome.

## Solution constraints

- Leave the item's SHOULD-tier escalation clause ("This obligation is SHOULD-level only at loom 1.0 …") unchanged.
- Use the step-2 preamble's `pass` / `fail` / `N/A` / `not-performed` recording vocabulary verbatim; do not paraphrase it.

## Problem

`audit-wire-and-canary.md` levies RFC 2119 normative modals (MAY/MUST)
directly on CI parsers — a downstream consumer layer this audit cluster
neither specifies nor verifies. Three sites do this: the *Green-path
emission contract*'s "CI parsers MAY assert that exactly one canary
record appears per invocation", the *Infra-aborted-run carve-out*'s "CI
parsers asserting on the canary obligation MUST exclude infra-aborted
runs…", and the *Wire serialisation* clause's "sole machine-parseable
surface a CI parser is permitted to consume in loom 1.0". Each is a
derivable consequence of a producer obligation the same file already pins
on the audit independently, so phrasing them as consumer obligations
imports an un-owned surface into an implementation spec. Nothing in the
cluster marks which modals bind the audit producer (the cluster's
subject) from those addressed to the downstream consumer.

## Issue introduction
**Verdict:** present-since-inception
**Introducing commits:** e5cd514 — pi-loom spec: resolve "T35 SDK capability inventory closed-set negative-direction audit" (2026-05-09, spec author); none earlier
**History:** The canary/wire contract and its CI-parser MUST/MAY modals entered the corpus together in e5cd514, the commit that first authored the inventory-closure audit cluster (`git log -S` on each CI-parser clause and `git log -G 'Non-empty-scan canary'` both bottom out at e5cd514; no CI-parser text exists before it). Later commits only relocated the content — f5e89f4 (size-cap split) and 81bf52a (collapse into `spec_topics/`) — without introducing or altering the modals. The defect is therefore present in the location's first commit.

## Solution approach

Demote the three CI-parser modals in `audit-wire-and-canary.md` to
informative consumer guidance, leaving the producer obligations they
derive from as the normative content. The *Green-path emission
contract*'s "MAY assert" sentence, the *Infra-aborted-run carve-out*'s
"MUST exclude infra-aborted runs" sentence, and the *Wire serialisation*
clause's consumer half each become non-normative, while the producer-side
MUSTs already in the file (exactly-one-canary-record, stdout-only canary
routing, and the producer carve-out that suspends the canary obligation
on infra-aborted runs) continue to back them.

## Solution constraints

- Preserve every producer obligation: each demoted consumer sentence must
  still have a corresponding producer MUST elsewhere in the file; the
  rewording must not weaken or remove any producer rule.
- Out of scope: the cluster-level non-goals declaration naming CI-parser
  implementation out of scope is owned by F-2044 — do not author it here.

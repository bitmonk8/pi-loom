## Finding

H7a's third `Tests.` bullet carries the `Convention:` prefix that marks it as a test obligation, but its body is almost entirely artifact description and risk disclosure: it states that the committed permitted-code list is a manually-maintained, best-effort union of each Deps slice's emittable codes, that it is deliberately broader than (a superset of) the golden diagnostics list, that its completeness is not mechanically verified so a stale union can omit a benign code, and that it is the reviewed reference set H4a's real-host smoke criterion (e) and the H6a release-gate evidence record check live-emitted codes against. None of that is a thing the H7a test run asserts — it describes properties of an artifact this leaf delivers, which is `Adds.` material.

The bullet's only operative, gated statement is "only the `golden ⊆ permitted` containment is gated", and that assertion is already authored as the fourth Tests bullet (the containment check that reads both checked-in artifacts and reddens `npm test`). The descriptive and limitation prose is also already present nearly verbatim in the leaf's `Adds.` permitted-code-list paragraph. The result is that an implementer scanning the Tests section for assertions to implement encounters a non-assertive description bullet wedged between two real assertions, duplicating Adds.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/H7a-integration-acceptance.md`, relocate the third Tests bullet's artifact-description and limitation content into the existing permitted-code-list paragraph in `Adds.`, then remove the third Tests bullet:

- The consumer-binding sentence — that the permitted-code list "is the single committed, reviewed reference set the [`H4a`](./H4a-factory-shell-and-harness.md) manual real-host smoke run's pass criterion (e) and the [`H6a`](./H6a-live-corpus-activation.md) release-gate evidence record check the live run's emitted codes against, replacing a per-run by-hand reconstruction of the union" — is the one piece not already in Adds; fold it into the Adds permitted-code-list paragraph.
- The remaining content of the third bullet (manually-maintained best-effort union, superset-of-golden framing, completeness-not-mechanically-verified limitation) already appears in the Adds permitted-code-list paragraph and is redundant; drop it.
- The bullet's only operative claim, `golden ⊆ permitted` is gated, is already asserted by the fourth Tests bullet (the containment check), so the third bullet can be removed in full without losing a gated assertion.

After the edit, the Tests section retains the golden-transcript assertion, the golden-diagnostics assertion, the `golden ⊆ permitted` containment assertion, and the ceiling-arbitration assertion; the permitted-code list's nature, limitation, and consumers live in Adds.

## Consequence

**Severity:** cosmetic

A reader scanning H7a's Tests section for assertions hits a bullet that asserts nothing — it is artifact description that already lives in Adds, and its sole gated claim is restated by the next bullet. No implementer diverges and nothing ships wrong; the cost is reduced scannability and a duplicated description that can drift out of sync with Adds.

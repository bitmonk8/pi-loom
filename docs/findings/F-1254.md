## Problem

`V13b`'s third *Tests* bullet closes the SUBS lowering-algorithm step 4 (per-query `$defs` pruning) using the obligation-closing `(SUBS code-keyed area)` citation format, but the `cka-10` (SUBS) coverage row in `coverage-matrix.md` names only `V5d`, `V5e`, `V5f` ŌĆö `V13b` sits in the `cka-12` (QRY) row. A SUBS obligation-closing citation therefore lives on a leaf the SUBS coverage row does not name. `V5f`'s *Adds* claims "per-query `$defs` pruning" as part of the lowering pass, yet `V5f`'s *Tests* cover only step 5, so step 4's only obligation-closing assertion sits on `V13b` outside the SUBS facet set. Two implementers can diverge on which leaf authoritatively closes the per-query-pruning SUBS obligation.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** ba7da00 ŌĆö pi-loom plan: resolve "V5d lowering-pass outputs bound by no Tests bullet" (2026-06-10, Thomas Andersen)
**History:** The `cka-10` (SUBS) coverage row has listed `V5d`, `V5e`, `V5f` and never `V13b` since the matrix's origin, and `V13b` has always sat in the `cka-12` (QRY) row. Commit ba7da00 ŌĆö resolving an unrelated finding about `V5d`'s unbound lowering-pass outputs ŌĆö added the SUBS-step-4 per-query `$defs` pruning obligation-closing *Tests* bullet (with the `(SUBS code-keyed area)` annotation) to `V13b` and added `V5d` to `V13b`'s Deps, but did not add `V13b` to the `cka-10` row, introducing the mismatch between `V13b`'s SUBS obligation-closing citation and its absence from the SUBS facet set.

## Solution approach

Move the SUBS lowering-algorithm step-4 (per-query `$defs` pruning) obligation-closing citation onto `V5f`, which owns the lowering pass and already names "per-query `$defs` pruning" in its *Adds*. Add a `V5f` *Tests* bullet citing `schema-subset.md` Lowering Algorithm step 4 annotated `(SUBS code-keyed area)` as the single obligation-closing citation for that step. Reword `V13b`'s third *Tests* bullet into typed-query integration evidence that binds `V5f`'s per-query `$defs` pruning seam by `V5f`'s leaf ID and drops the `(SUBS code-keyed area)` annotation.

## Solution constraints

- Out of scope: the `coverage-matrix.md` `cka-10` (SUBS) row ŌĆö its facet set stays `V5d`, `V5e`, `V5f`; do not add `V13b` to it.

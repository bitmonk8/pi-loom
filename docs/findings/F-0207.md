## Finding

`H3a`'s **Adds** declares each host-seam interface's full member signatures and states they are "the single source of each interface's shape," sourced from the seam-contract pages [`host-interfaces-core.md`](../spec_topics/pi-integration-contract/host-interfaces-core.md) and [`host-interfaces-services.md`](../spec_topics/pi-integration-contract/host-interfaces-services.md), with the `V8a`–`V8e` leaves implementing against those signatures rather than redeclaring members. `H3a`'s **Tests**, however, assert only two things: that two runtime roots yield isolated seam graphs, and that no `src/**` module directly references a banned ambient primitive outside an exempt site. Neither test checks that the members `H3a` declares correspond to the members the seam-contract pages name.

Consequently a member that `H3a` omits, mis-names, or mistypes relative to its contract page is not caught at `H3a` time. The interface is loom-internal and the contract pages are prose, so TypeScript cannot flag a member that is simply absent from both the `H3a` interface and the code that consumes it. The divergence ships green until a `V8*` seam-adapter leaf (much later in the build order) implements the seam — and even then it surfaces only if that leaf's author independently re-reads the contract page and notices the gap. The conformance of `H3a`'s declared shapes to the spec is therefore contributor discipline plus the per-phase TDD self-review and the version-bump re-validation, not the mechanical gate the "single source of shape" framing implies.

Note a spec-side constraint that bounds any fix: the contract pages' interface member sets are deliberately **non-binding** under `GOV-18` arm (a) — `V8b` adds a `readBytes` member beyond `PIC-13`'s enumerated set on exactly that basis. An exact member-set conformance check would therefore contradict the spec; at most a fix may assert coverage of the members a contract page names while leaving room for permitted loom additions.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** 592219d — pi-loom plan: resolve "H3a seam interfaces cite no source for member shapes" (2026-06-11, Thomas Andersen)
**History:** Before 592219d, `H3a`'s Adds said only "Seam interfaces only; their normative behaviour is implemented by the `V8*` leaves" and made no claim about being the source of interface shape, so there was no source↔declaration correspondence to gate. Commit 592219d resolved a prior finding by adding the "single source of each interface's shape, sourced from the seam-contract pages" claim to Adds, but left the Tests block (isolation + ambient scan) unchanged — so the new correspondence claim shipped with no asserting gate.

## Solution Space

**Shape:** multiple

### Option A — Disclose that the source↔declaration correspondence is review discipline, not a gate

**Approach.** In `H3a`'s **Adds**, qualify the "single source of each interface's shape" claim so it states that the correspondence between the `H3a`-declared signatures and the seam-contract pages is carried by the per-phase TDD self-review and the version-bump re-validation, not by a mechanical assertion — mirroring how `H3a`'s ambient-scan residue already attributes its blind spots to the *Per-phase TDD ritual* self-review.

**Plan edits.** Adjust the Adds sentence beginning "These `H3a`-declared signatures are the single source of each interface's shape…" to disclose the non-mechanical nature of the source↔declaration correspondence.

**Spec edits.** None.

**Pros.** Spec-correct (does not imply a check that `GOV-18` arm (a) forbids); zero new tooling; consistent with the entry-shim disclosure pattern (finding 45) and the existing ambient-scan residue language.

**Cons.** Leaves the correspondence as discipline; a future authoring slip in an `H3a` interface still ships green.

**Risks.** Low. Confined to wording in one leaf.

### Option B — Add a contract-page member-coverage assertion to H3a

**Approach.** Add an `H3a` **Tests** bullet asserting each declared seam interface exposes at least the member set its contract page names (coverage, not exact-match, to respect `GOV-18` arm (a)).

**Plan edits.** New Tests bullet in `H3a`; possibly reuse `H2a` architectural-test machinery.

**Spec edits.** None — but the check needs a machine-readable enumeration of contract-page members, which the prose spec does not provide.

**Pros.** Catches an omitted/mis-named member at `H3a` time rather than at a later `V8*` leaf.

**Cons.** Requires duplicating the prose member lists into a fixture, creating an ongoing maintenance and drift surface; coverage-only semantics are awkward to define against prose contract pages.

**Risks.** The duplicated enumeration itself drifts from the contract pages, reintroducing the very divergence it aims to catch, now in two places.

### Recommendation

Take **Option A**. `GOV-18` arm (a) makes the contract pages' member sets non-binding, so an exact-conformance gate is not spec-correct and a coverage gate forces a duplicated, drift-prone enumeration of prose member lists. Disclosing that the source↔declaration correspondence is review discipline aligns `H3a` with the analogous entry-shim handling (finding 45) and with its own ambient-scan residue language. Edge case for the implementer: scope the disclosure to the source↔declaration correspondence only; do not weaken the genuinely-gated seam-isolation and ambient-scan assertions in the same Tests block.

## Consequence

**Severity:** advisory

A seam member that `H3a` omits or mistypes relative to its contract page is not detected at `H3a` time and is not mechanically detected at all (the spec is prose; TypeScript cannot see an absent member). It surfaces, if ever, only when a `V8*` adapter author re-reads the contract page. The "single source of shape" framing overstates the safety net: the source↔declaration correspondence rests on author discipline and version-bump re-validation, not a gate.

## Problem

The Echo policy's *Object values* bullet defines first-field rendering for discriminated-union values, but its trailing conditional — "the discriminator field is included in that order if it appears there" — reads as if the discriminator's presence among the variant's declared fields were optional. That leaves the absent-discriminator case unspecified, inviting an implementer to invent a fallback rendering for `{first-field-value, …}` when the discriminator is missing. The schema layer already forecloses that arm: `schemas.md` §Discriminated unions requires the discriminator to be present in every variant, and the parse diagnostics reject nested and missing discriminators, so the discriminator is always one of the variant's declared top-level fields. The conditional therefore guards a case that cannot occur on a conforming variant.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** `edc2e1b` ("pi-loom spec: resolve \"echo policy: special characters and first field undefined\"", 2026-05-06)
**History:** Commit `edc2e1b` introduced the *Object values* bullet's first-field definition and added the sentence *"For a value whose static type is a discriminated union, the variant's declared fields are used in the variant's own source order; the discriminator field is included in that order if it appears there."* The "if it appears there" conditional has been present verbatim since that commit. Subsequent edits to the bullet — `800b3f9` (composite first-field recursion), `71e8313` (inline-object first-field rule), `e44c234` (illustrative example), and the corpus-wide split into `binder/defaulting-system-note-echo.md` at `f5e89f4` — left the conditional unchanged. No commit ever revisited the wording against the `schemas.md` always-present + top-level invariants that make the conditional dead.

## Solution approach

Rewrite the discriminator clause in `defaulting-system-note-echo.md`'s Echo policy *Object values* bullet so it states the discriminator always appears in the variant's own source order, rather than hedging on "if it appears there". Add a forward-link to `schemas.md` §Discriminated unions to ground the always-present invariant. BNDR-6h already pins the intended rendering, so no table rows change.

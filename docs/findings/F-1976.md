## Finding

"deduped" admits two readings: (a) if a canonical-extension sibling for the same `realpath` has already emitted a diagnostic, this warning is suppressed (one-winner dedup), or (b) if multiple non-canonical files resolve to the same `realpath`, only one warning is emitted for that path (count-reducing dedup). In the case of two non-canonical files mapping to the same `realpath` with no canonical sibling, reading (a) emits two warnings and reading (b) emits one. - **Suggested clarification:** Replace "deduped" with a precise rule, e.g.: "One warning is emitted per unique `realpath`;

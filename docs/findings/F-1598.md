## Problem

The *Failure-surface contract* section of `audit-failures.md` (anchor `id="audit-failure-surface-contract"`) opens with a `MUST` record-shape sentence enumerating the `proposed-resolution` field's permitted shapes as exactly two — an `SDK_SURFACE_INVENTORY` entry or an `// allow-pi-surface:` exemption marker — which is correct only for families (1)–(3). The *Per-family record-shape table* later in the same section contradicts this: family (4) requires a recognised-shape rewrite suggestion (and explicitly forbids an exemption-marker suggestion), and family (5) requires a deletion-or-rewrite suggestion. The two normative statements disagree on the permitted value set of `proposed-resolution`, so an implementer building the violation-record format from the intro sentence cannot represent the family-(4) or family-(5) resolutions the table simultaneously mandates. Two reasonable implementers diverge on the field's wire-format contract.

## Issue introduction

**Verdict:** single-commit introduction
**Introducing commit:** `e5cd514` (2026-05-09, *pi-loom spec: resolve "T35 SDK capability inventory closed-set negative-direction audit"*)
**History:** `git log --reverse -G "MUST NOT suppress the family \(4\)"` on the spec corpus identifies `e5cd514` as the sole add of the *Exemption mechanism* paragraph, the *Failure-surface contract* family enumeration, the *Malformed-marker discriminator* clause list, and the dual-emission co-commit obligation. The defect is present at first introduction — the sentence with the ambiguous *"that discriminator"*, the malformed-marker clause list with no (h) covering this case, and the dual-emission rule scoped to family (1)/(2)/(3) only all entered together in that single commit. Subsequent edits to the same paragraphs (the `@mariozechner → @earendil-works` rename in `5e73072`, the corpus-direction sweep in `9e53613`, and the size-cap collapse / file split in `f5e89f4` and `81bf52a`) preserve the wording verbatim and do not address the gap.

## Solution approach

Rewrite the `id="audit-failure-surface-contract"` section's record-shape sentence so its `proposed-resolution` clause no longer enumerates a two-value field, deferring per-family detail to the *Per-family record-shape table* as the single authoritative source for each family's resolution value. Retain the four-field record shape (`path`, `line`, `symbol`, `proposed-resolution`).

## Problem

The `cka-10` row in `coverage-matrix.md`'s *Code-keyed obligation areas* table maps the SUBS obligation to leaves `V5d`, `V5e`, `V5f`, and the `co-witness` `V15j`. Under the matrix's *Multi-leaf-row per-facet citing tests* convention the co-witness drops out, leaving facets `{V5d, V5e, V5f}`, each of which MUST carry a facet-naming citing test that cites both the `cka-10` token and that facet's leaf-ID inline. None of the three facet leaves or their paired tests tasks carries such a citation — `V5f` / `V5f-T` cite only "(SUBS code-keyed area)", and `V5d` / `V5d-T` and `V5e` / `V5e-T` cite no `cka-10` token at all. The `H5f` per-facet citing-test gate arm therefore reddens for all three facets on the live-corpus footing.

## Issue introduction

**Verdict:** single-commit
**Introducing commit:** `26a87941019d2e4e46044a87463f77453933cb64` ("pi-loom plan: resolve \"split coverage-matrix rows can ship a facet untested\"", 2026-06-18)
**History:** The `cka-10` row listed `V5d`, `V5e`, `V5f` from `cb8737a4d19856053efa4ac7acc6902409908904` (2026-06-12), and `V5f`'s "(SUBS code-keyed area)" annotation dates to the same commit — compliant at the time, since no per-facet citing-test obligation existed then. Commit `26a8794` introduced both the *Multi-leaf-row per-facet citing tests* convention on `coverage-matrix.md` and the `H5f` gate arm, creating the per-facet obligation over the already-listed `cka-10` facets without bringing `V5d` / `V5e` / `V5f` into compliance. A later edit (`8664ca36291447c278a7c6e58970e46ffab96a00`, F-1373) added `V15j` as a `cka-10` co-witness with its own token citation but did not touch the three facets, so the gap persisted.

## Solution approach

In each facet's leaf and its paired tests leaf — `V5d` / `V5d-T`, `V5e` / `V5e-T`, `V5f` / `V5f-T` — designate one existing `Tests.` bullet as the `cka-10` facet-naming citing test, citing both the `cka-10` token and that facet's own leaf-ID inline, following the `cka-33` / `V9o` form already established on `coverage-matrix.md`. On `V5f` / `V5f-T` this replaces the existing "(SUBS code-keyed area)" annotation on the per-schema sidecar bullet; on `V5d` / `V5d-T` and `V5e` / `V5e-T` it adds the citation to an existing tests bullet.

## Solution constraints

- Reuse the existing `cka-10` token — do not mint a new `cka-<n>` token (the token convention is append-only).
- Out of scope: the `cka-10` `V15j` co-witness citation owned by F-1373.

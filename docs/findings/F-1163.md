## Finding

The opening blockquote on `session-only-degraded-state.md` reads *"The obligations on this page are blocked pending resolution of the open contradiction recorded at Host prerequisites … clause (a); … The block clears only when item (a)'s authoritative-determination record resolves to the `provably-differs` verdict."* Read literally, "the obligations on this page" sweeps every normative clause on the page, of which there are many: the *Drain-state tag write* clause (defining `LoomRegistry.markRuntimeDegraded` semantics), the *Recovery path and prohibitions* MUST/MUST NOT set, the *Diagnostic emission* rule (including the `Isolation` paragraph and the *Inline triplet is normative* clause), the *Predicate split* table and its *Seam minimality (normative)* paragraph, the *Co-emission rules*, the *Idempotent re-entry* rule, the three documented *residual gaps*, and the thirteen acceptance-criteria fixtures (a)–(m).

Two sibling PIC pages already encode portions of this same surface as concrete, buildable contract — and the plan ships them on leaves that are **not** blocked. `drain-state-contract.md` defines `LoomRegistry.markRuntimeDegraded()` as the unconditional `"degraded-needs-reload"` setter in the closed four-method call surface and defines `readDrainState`'s three-arm enumeration (including the `"degraded-needs-reload"` arm that returns the *"degraded; /reload to recover"* slash note); both ship under `V9m`. `patch-skew-degradation.md` defines the *interstitial-seam-tag-write-swallow* rule (the dedicated `try`/`catch` wrap around the post-sub-step-4 tag write, the silent-swallow contract, and the `loomRegistry.markRuntimeDegraded` label) — fixture rule (k) on the degraded-state page references this rule by anchor, and the sub-step-3 settle-all obligation ships under `V9g`. The degraded-state page is the cross-reference target for both — the *Drain-state tag write*, *Isolation*, and *Predicate split* clauses are the canonical definitions the siblings rely on — yet the page-wide gate marks all of those definitions blocked. An implementer cannot tell whether the `markRuntimeDegraded` setter contract and the `"degraded-needs-reload"` tag arm (consumed by `V9m`/`V9g`) are inside or outside the gate, nor which of the fixtures (a)–(m) may be written before clause (a) resolves, nor — should clause (a) resolve `governed-by-rebind` — which page-local definitions sibling pages will need to relocate.

Compounding this, `session-model-and-appendix.md` now carries per-SM "*Implementation gate:*" markers on SM-3b/SM-4/SM-5/SM-6 explicitly; the PIC page's blockquote is the only gate that scopes by file rather than by clause and is therefore the only one whose scope is in doubt.

## Solution Space

### Recommendation

Rewrite the *Implementation gate* blockquote on `session-only-degraded-state.md` so it enumerates, by anchor, exactly which page-local clauses are inside the gate and which are not.

The **gated** set (blocked pending the clause-(a) authoritative determination) is: the *Recovery path and prohibitions* MUST/MUST NOT block, the *Diagnostic emission* rule for `loom/host/session-shutdown-runtime-degraded`, the *Predicate split* / *Seam minimality* clauses, the *Co-emission rules*, the *Idempotent re-entry* corner case, the *Accepted residual gaps* enumeration, and all acceptance-criteria fixtures (a)–(m). Classify fixtures (k) and (l)/(m) with the rest of (a)–(m) as **gated** — they are branch-behaviour acceptance even though they exercise a non-gated definition (they cite the `markRuntimeDegraded` setter contract by behaviour, not by branch).

The **non-gated** set is: the *Drain-state tag write* paragraph as the **method-level setter contract** for `LoomRegistry.markRuntimeDegraded` (the four-method-surface entry `V9m` ships against this), and the cross-reference targets the *Isolation* paragraph provides to `interstitial-seam-tag-write-swallow` consumers in `patch-skew-degradation.md`. Add a one-sentence rider stating that the non-gated definitions are the **API/seam contracts** the siblings consume, while the **branch behaviour** that invokes them on the session-only reasons is gated.

Author the enumeration as two flat lists of anchors (`#substep-2-stamp-throw-residual-gap`-style) so the gate's scope is grep-able from any leaf reading the page. Add page-anchors (`<a id="…"></a>`) on each clause the blockquote references so the enumeration is link-checkable and stable across editorial reflow. Include a one-line normative rider in the blockquote that any newly-added clause defaults to **gated** unless explicitly enumerated as non-gated, so the scope cannot drift back into ambiguity.

Make no edits on `drain-state-contract.md` or `patch-skew-degradation.md`; their existing inbound references resolve into the non-gated subset. Leave `V9m` and `V9g` unchanged. This keeps the `governed-by-rebind` rework scope mechanical: the rework set is exactly the enumerated gated anchors. This edit is independent of the clause-(a) authoritative determination — the enumeration shape does not depend on the verdict, so it can be applied before, after, or alongside that determination.

## Consequence

**Severity:** blocking

An implementer working from `V9m` or `V9g` cannot tell whether they may consume the `markRuntimeDegraded` setter contract and the `interstitial-seam-tag-write-swallow` cross-reference — both canonical on the degraded-state page — or whether those definitions are inside the page-wide gate and must be deferred or re-homed. The same ambiguity blocks a `V9l-T` author from deciding which of fixtures (a)–(m) may be written speculatively under TDD discipline. The defect propagates: if clause (a) eventually resolves `governed-by-rebind`, the rework scope is undefined because the gate did not separate definitions sibling pages already depend on from branch-behaviour obligations that would be reworked.

## Problem

The family-(4) `createRequire(...)` CJS-reach prohibition in *Recognised import/access shapes* (anchor `audit-recognised-shapes`) on `audit-target-categories.md` closes with the clause "no such call MUST exist anywhere in the audited source tree". Under RFC 2119 / RFC 8174, `MUST` expresses a positive requirement on the sentence subject; with the subject "no such call" the literal reading is a double-negative that a strict reader scanning for prohibitions (`MUST NOT` / `SHALL NOT`) will not match, and reads more naturally as "no such call is required to exist" — the opposite of a prohibition. Every neighbouring family-(4) clause in the same paragraph already uses `MUST NOT`; this lone deviation governs a non-exemptible discriminator, so the two RFC-2119 readings disagree on whether `createRequire(...)("@earendil-works/<peer>")` is forbidden, and a strict reader's V18b audit would ship green on a `createRequire`-via-rebinding bypass the author intends to fail.

The defective `MUST` (rather than `MUST NOT`) is present in the introducing commit's diff verbatim; every subsequent edit to the surrounding paragraph (notably `d8d5ad0`, the F-0632/F-0637/F-0649 Step 0(d) reconciliation that inserted the `.resolve()` carve-out into the same sentence) preserved the wording on either side of its insertion. The defect entered the corpus on its first authoring and has been carried unchanged ever since.

## Solution approach

Rewrite the closing CJS-reach prohibition clause so the family-(4) `createRequire` prohibition uses the RFC-2119 `MUST NOT` form (e.g. "any such call MUST NOT exist anywhere in the audited source tree") instead of the double-negative "no such call MUST exist". The defect is the RFC-2119 keyword itself, so the corrected form aligns this clause with every other family-(4) prohibition in the same paragraph.

## Solution constraints

- Edit only the closing CJS-reach prohibition clause; the rule-level `.resolve(<specifier>)` carve-out and the companion "MUST NOT be read as authorising the \"module\"-package shapes above by omission" sentence in the same paragraph remain unchanged.

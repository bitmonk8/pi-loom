## Finding

`docs/plan_topics/V4f-no-rollback.md`'s **Ships when** field opens with a concrete, externally observable gate — "`npm test` proves the enumerated `ERR-13` cases hold — a completed callee stays final after a downstream terminal event across the sampled authoring sites, with no side effect unwound and no compensating turn injected" — which is exactly what `conventions.md` §Leaf format requires of a Ships-when ("A concrete, externally observable change").

It then appends a second sentence — "The no-rollback guarantee is architectural: it rests on the runtime containing no compensating/rollback path, and the tests witness it on the enumerated cases rather than exhaustively." — that is design rationale, not a pass/fail condition. The same rationale is already stated in the leaf's **Adds** field ("The no-rollback guarantee is architectural: the runtime contains no compensating path, so a completed callee's side effect survives a downstream terminal event by construction rather than by an undo being suppressed"). The Ships-when gate's pass/fail boundary is fully carried by its first sentence; the trailing rationale is duplicative and belongs only in Adds.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** 098ec80 ("pi-loom plan: resolve \"V4c bundles the ERR-8...ERR-12 trichotomy and the ERR-13 no-rollback proof\"")
**History:** `docs/plan_topics/V4f-no-rollback.md` did not exist before 098ec80 (`git cat-file -e 098ec80~1:docs/plan_topics/V4f-no-rollback.md` fails). That commit split the no-rollback leaf out of V4c and authored the file; `git log -L '/Ships when/,+1:docs/plan_topics/V4f-no-rollback.md'` shows the Ships-when line — including the redundant architectural-rationale sentence — was introduced whole in that single commit and never subsequently edited. The duplication was therefore born with the leaf file rather than introduced by a later edit.

## Solution Space

**Shape:** single

### Recommendation

In `docs/plan_topics/V4f-no-rollback.md`, strike the trailing rationale sentence from the **Ships when** field, leaving only the pass/fail gate. Remove the literal text:

> The no-rollback guarantee is architectural: it rests on the runtime containing no compensating/rollback path, and the tests witness it on the enumerated cases rather than exhaustively.

The first sentence ("`npm test` proves the enumerated `ERR-13` cases hold … with no side effect unwound and no compensating turn injected") remains as the complete Ships-when criterion. The architectural rationale is preserved in the **Adds** field, which already states it; no rationale is lost. Do not touch the spec — this fix is internal to the plan leaf file.

## Consequence

**Severity:** cosmetic

The Ships-when already names a valid, observable gate, so no implementer is blocked or misdirected; the trailing rationale sentence merely duplicates Adds and dilutes the gate's pass/fail focus. Leaving it costs nothing but readability.

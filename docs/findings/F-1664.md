## Problem

The step-2 editorial-review checklist requires the contributor to record `pass` / `fail` / `N/A` / `not-performed` with a one-line rationale for each SHOULD-level item (f)–(aq), but defines no *disposition* for a recorded `fail` on most of those items — i.e. what the contributor is obligated to do once a falsified host presupposition is positively detected. The obligation is specified for only a minority: items (g), (j), (q), (v), (ad), and (ah) each close with an explicit "surface the divergence on the bump commit so the … paragraph can be amended in the same edit" clause, and item (e) additionally gates the merge. The remaining items state what to confirm and what a silent regression would cause but say nothing about what a recorded `fail` obligates, and the **Outcome recording conventions** block — the natural home for a default rule — is silent. The asymmetry means a detected falsification can be recorded and then silently ignored, defeating the audit's purpose.

## Solution approach

Add a default-disposition statement to the **Outcome recording conventions** block (anchor `bump-step-2-outcome-recording-conventions`) so it governs the whole (f)–(aq) range from that one site: a recorded `fail` on an item obligates the contributor to surface the divergence on the bump commit and amend the affected presupposition paragraph in the same edit, unless the item carries its own narrower disposition. Locating it in the conventions block mirrors how that block already collects the item-(e)/(a) recording shapes apart from the inline item bodies.

## Solution constraints

- Out of scope: authoring loom-side runtime recovery for the falsified Pi behaviour — the default rule governs only the contributor's spec-text and bump-commit obligation, consistent with the items' existing "PIC does not author the loom-side recovery here".

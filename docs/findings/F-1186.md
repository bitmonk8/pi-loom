## Problem

GOV-30 (`#gov-30`) and GOV-31 (`#gov-31`) frame their lock-step convention around "`spec.md` aggregator paragraphs" — GOV-30's lock-step MUST says to "update the corresponding `spec.md` aggregator", and GOV-31 references "the orientation prose literal that names the count on `spec.md`". After the spec.md split, `docs/spec.md` is a three-link table of contents, and the aggregators these rules govern (Scope, Prerequisites Host-runtime, Hard-ceilings, cross-cutting NFR, SDK-capabilities, loom 1.0 non-goals) live on the orientation topic pages — exactly where GOV-30/31's own inline links already point. The prose names a host file that no longer carries the aggregators while the embedded links name the right ones, an internal contradiction within the same rule. A maintainer who reads only the prose searches `spec.md` for an aggregator that is not there; one who follows the links discovers the prose is wrong, leaving the lock-step co-edit MUST with no single authoritative locus.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `f5e89f4` (2026-06-04, `docs: enforce 100KB hard / 30KB recommended size cap on spec set`) interacting with `126ff8a` (2026-05-06, `pi-loom spec: Step 0 capability probe rewrite + V1.0 deferrals + Choice B`) and `fe8aa5c` (2026-06-05, `pi-loom spec: resolve "GOV-12 bundles ≥3 independently-testable sub-obligations"`).
**History:** The aggregator lock-step convention was authored in `126ff8a` as a single rule (`GOV-12`) whose prose was correct at the time — `docs/spec.md` was then a monolith that hosted the Scope, Prerequisites, Hard-ceilings, NFR, and loom 1.0 non-goals paragraphs the rule governed. The size-cap split in `f5e89f4` (later refined by `81bf52a` on 2026-06-11, which renamed `spec/` to `spec_topics/`) moved those aggregator paragraphs off `spec.md` into `spec/overview-and-orientation.md` and `spec/language-and-architecture.md`, reducing `spec.md` to a three-link TOC. The split repointed 1571 cross-references mechanically but did not rewrite the GOV-12 prose itself, so the rule's natural-language framing — "`spec.md` aggregator paragraphs", "the corresponding `spec.md` aggregator", "the orientation prose literal that names the count on `spec.md`" — was left referring to a host that no longer exists. The subsequent GOV-12 → GOV-30/31 split in `fe8aa5c` (2026-06-05) reflowed the prose into two rules but preserved the obsolete `spec.md` framing in both. Neither commit was individually wrong: `f5e89f4`'s scope was layout, and `fe8aa5c`'s scope was a GOV-12 sub-obligation split; the defect is the unmet co-edit obligation between them — GOV-30/31's own lock-step rule applied to itself.

## Solution approach

Rewrite GOV-30 (`#gov-30`) and GOV-31 (`#gov-31`) prose in `req-id-prefix-table-active-b.md` so every "`spec.md` aggregator paragraph(s)", "corresponding `spec.md` aggregator", and "orientation prose literal that names the count on `spec.md`" phrasing names the orientation host page that actually carries the aggregator: `overview-and-orientation.md` for the Scope, Prerequisites Host-runtime, Hard-ceilings, cross-cutting NFR, and SDK-capabilities aggregators, `language-and-architecture.md` for the loom 1.0 non-goals aggregator, and `session-model-and-appendix.md` for the Session Model forward-link inventory. Align the prose with the inline link targets these rules already carry. Sweep the same "`spec.md` aggregator" phrasing in `stable-inline-labels.md`, in `anchor-scheme-and-retired.md` (GOV-23, whose Session Model forward-link inventory lives on `session-model-and-appendix.md`), and in the GOV-N entry in `glossary.md`.

## Solution constraints

- Out of scope: GOV-29's `docs/spec.md` + `docs/spec_topics/*.md` phrasing — it denotes the union and remains accurate; do not extend the rename to it unless its normative-owner / aggregator-copy framing is reconciled in the same pass.
- Out of scope: the "Orientation aggregator (per [Governance — GOV-30])" subhead annotations on the orientation pages — correct as written; do not reword them.

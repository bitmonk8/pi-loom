## Problem

DISC-5's `pi.looms` glob pipeline defines its candidate universe as "When no plain (un-prefixed) include pattern is present, the starting set is every path under the package root." The phrase "every path under the package root" never states whether "under" means the full recursive subtree or only the package root's immediate children. The depth is load-bearing: it determines the surviving set for exclusion-only manifests (e.g. `pi.looms: ["!test/**"]`) and whether a plain `**/*.loom` include can match below the top level. Surrounding text pulls both ways — the page emphasises non-recursion (`looms/` fallback, directory-match expansion, a "global non-recursion rule") while the matcher is `minimatch` with `**` support claiming to mirror Pi's recursive `applyPatterns` — so two conforming implementers silently diverge on the discovered loom set.

**History:** `git log -S "every path under the package root" -- docs/spec_topics/discovery/package-and-settings.md` returns exactly one commit, `1cb54c7`. `git show 1cb54c7` confirms that commit authored the entire DISC-5 matcher/override-precedence machinery — including the "the starting set is every path under the package root" sentence — wholesale; the prior DISC-5 carried no starting-set or matcher-engine prose. The depth ambiguity therefore entered with the contract it lives in, not before it: the sentence specified the override ordering and matcher engine but never qualified the recursion depth of the unbounded starting set.

## Solution approach

Rewrite the DISC-5 no-plain-include starting-set sentence (anchor `disc-5`) to pin that the starting set is the recursive enumeration of files at any depth below the package root. Add a clarifying clause distinguishing this recursive candidate universe from the two non-recursion rules it sits beside — the non-recursive `*.loom` expansion of a directory *match* and the non-recursive `looms/` convention fallback — so the patterns select from the full tree while a matched directory still contributes only its immediate `*.loom` children.

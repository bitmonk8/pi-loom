## Finding

Four normative MUSTs in the Shard-08 PIC topics ship without enumerated acceptance criteria or fixture inventory, so a conformant implementation cannot be discriminated from a non-conformant one on first-principles reading. The MUSTs in question:

1. **Construction-site catch-arm fallback forms** (`diagnostic-emission-isolation.md` — *Construction-site wrap*, *Construction-site catch-arm self-wrap*): on a payload-construction throw for the two nested-shape codes, the catch arm MUST emit byte-exact `` console.error(`${code} <unreadable>`) `` for `loom/host/session-shutdown-runtime-degraded` and `` console.error(`${code} ${entry.loom} <unreadable>`) `` for `loom/runtime/cancelled-by-session-shutdown`, and the construction-site `console.error` MUST itself be self-wrapped so its own throw is silently swallowed. The page pins the wire form down to token count but lists no fixture that drives a construction-site throw (frozen target, throwing setter on a host-shimmed `Object` prototype, etc.) and asserts on the emitted strings or on continuation past the swallowed inner throw.

2. **PIC-17 active-set isolation — empty/respond-only sub-cases** (`tool-registration-lifetime.md` — PIC-17): step 2's call is `pi.setActiveTools([...loomCallableSetNames, respondToolName?])`, and the "ambient tools are deliberately not inherited" invariant requires that the union over the user-session snapshot from step 1 is *not* installed. The two boundary sub-cases — (a) a loom whose callable set is empty on the untyped query (the installed list is `[]`), and (b) the typed-query forced-respond turn whose installed list is exactly `[respondToolName]` — are the load-bearing witnesses of the non-inheritance invariant but are not pinned as acceptance fixtures. A runtime that silently unioned `snapshot` into the installed list would still pass any test that only asserts "the loom's tools are visible" because the loom's tools are visible either way.

3. **`details.failure` `"throw:<…>"` truncation and control-character escaping** (`unknown-reason-rule.md` — *`"throw:<String(error)>"` discriminator's grammar*): the `String(error)` payload MUST be truncated to at most 256 UTF-16 code units before concatenation, control characters in U+0000–U+001F and U+007F–U+009F MUST be escaped (with the named two-character forms for `\n`/`\r`/`\t` and the six-character `\uXXXX` form otherwise), the prefix colon is positional rather than delimiter-based, and an interior literal `"throw:"` substring receives no special handling. No vector inventory is enumerated for the truncation boundary (255/256/257 UTF-16 units; surrogate-pair-split-at-boundary), the escape categories (each control class), the interior-colon case, or the inner-`String()`-throw `"throw:<unreadable>"` coercion-failure fallback — all of which are operator-visible discriminator forms.

4. **`(drained: true, tag: undefined)` post-handler tuple on `quit`/`reload`** (`drain-state-contract.md` — *Methods*, `readDrainState` arm (b) third antecedent): the six-tuple state-space enumeration routes `(true, undefined)` to the `"shutting-down"` arm, covering the case where sub-step 1's `LoomRegistry.drain` succeeded but `LoomRegistry.initDrainStateTag` threw (caught by *Per-step isolation*, leaving the tag unset). On the `quit`/`reload` reasons this is reachable but `session-only-degraded-state.md`'s acceptance criteria (i)/(j) only enumerate the `{new, resume, fork}` triplet for tag-init-throw and drain-throw fixtures. The slash-handler reader behaviour for the `(true, undefined)` tuple on `quit`/`reload` (i.e. the *unset tag fallback* sub-step 1 pins, routing through arm (b)'s `"loom /<name>: extension shutting down"` note) is therefore not witnessed by any fixture in the corpus; the closest neighbours (i)/(j) explicitly exclude `quit`/`reload` slash-arm assertions on the grounds that Pi tears the runtime down before any subsequent slash dispatch — but the tuple's reachability is what wants witnessing, not the post-tear-down slash dispatch.

The four gaps share a single shape — normative MUST + observable wire/state form + no enumerated fixture set — so a runtime implementer can ship spec-compliant prose-checked code that silently violates the contract on the un-fixtured boundary.

## Solution Space

**Shape:** single

### Recommendation

Add four enumerated fixture / acceptance-criteria blocks, each colocated with the obligation it witnesses. Each fixture inventory is normative-by-reference; the implementation owns `<file>` placement and exact harness shape, matching the existing per-page convention (e.g. patch-skew (P-a)/(P-b)/(P-c)).

1. **Construction-site catch-arm fixtures** (extend `diagnostic-emission-isolation.md` — *Construction-site wrap* paragraph with an `Acceptance criteria` sub-section). Two fixtures, one per nested-shape code. For `loom/host/session-shutdown-runtime-degraded`: force the `details.event` construction site to throw (e.g. a frozen target object whose property-set throws, or a host-shimmed `Object.defineProperty` setter), dispatch a session-only `session_shutdown`, and assert (a) exactly one `console.error` was invoked with byte-exact argument `` `${code} <unreadable>` ``, (b) no structured serialisation-and-emission attempt was made for this event, (c) the handler proceeded past the construction-site catch arm into sub-step 5, (d) on an additional fixture variant where the construction-site `console.error` itself throws, the handler still progresses past the self-wrap. For `loom/runtime/cancelled-by-session-shutdown`: same shape, byte-exact argument `` `${code} ${entry.loom} <unreadable>` ``, asserted at the per-invocation `finally` site with `entry.loom` interpolated as the canonical-key name.

2. **PIC-17 empty/respond-only active-set fixtures** (extend `tool-registration-lifetime.md` PIC-17 with an `Acceptance criteria` sub-section, or extend the conformance-test sentence in `V9c-T` and `V9f-T`). Three fixtures driving spy-based assertions on the exact argument vector passed to `pi.setActiveTools`: (a) untyped query against a loom whose `tools:` callable set is empty — assert `pi.setActiveTools` was called with `[]` (not the snapshot, not `undefined`); (b) typed-query forced-respond turn — assert the install vector is exactly `[respondToolName]` (one element, not unioned with snapshot, not unioned with the loom's free-phase callable set); (c) typed-query free phase against a non-empty callable set — assert the install vector is exactly `[...loomCallableSetNames]` (no `respondToolName` and no snapshot union).

3. **Throw-discriminator truncation / escaping fixtures** (extend `unknown-reason-rule.md` *Sub-anchors and pinned substrings* with a new sub-anchor `(iv) Throw-discriminator grammar witnesses`). Five fixture categories: (a) boundary — payloads at 255, 256, and 257 UTF-16 code units, asserting the post-truncation discriminator length and that the 256th code unit is preserved verbatim (no off-by-one); (b) surrogate-pair-at-boundary — a payload whose code-unit 256 is the high-surrogate of an astral pair, asserting the spec-defined behaviour (the rule pins UTF-16 code units, so the high surrogate is preserved as a lone code unit per `String.prototype.length` semantics, but the fixture must witness the chosen interpretation); (c) one escape vector per control-character class — `\n`, `\r`, `\t` as the named two-character escapes, one representative non-named character from U+0000–U+001F and one from U+007F–U+009F as `\uXXXX`, and one printable character from outside both ranges asserting passthrough; (d) interior-colon and interior-`"throw:"` payloads, asserting the discriminator round-trips through `startsWith("throw:")` correctly; (e) coercion-failure — an `error` whose `Symbol.toPrimitive` throws, asserting the discriminator is byte-exact `"throw:<unreadable>"`.

4. **`(drained:true, tag:undefined)` quit/reload fixtures** (add an acceptance branch to `session-only-degraded-state.md` *Acceptance criteria*, e.g. `(n) Tag-init throw — quit/reload state-tuple coverage`). For each `reason ∈ {"quit", "reload"}` (2 fixtures) where sub-step 1's `LoomRegistry.drain` succeeded but `LoomRegistry.initDrainStateTag` threw (caught; emits one `teardown-step-failed` under `details.call = "loomRegistry.initDrainStateTag"`): assert the post-handler `LoomRegistry.readDrainState()` snapshot is exactly `{ drained: true, tag: undefined }`, that a synthetic in-process slash-handler readout against that tuple selects arm (b) per the *unset tag fallback* clause (returning the `"loom /<name>: extension shutting down"` note), and that zero `runtime-degraded` rows are emitted (per existing branch (c) for `{quit, reload}`). The fixture witnesses the third antecedent of arm (b)'s composite predicate without contradicting the *no subsequent slash dispatch on `quit`/`reload`* rationale that excluded the case from (i)/(j) — the slash-handler readout is exercised against an injected fake, not Pi.

Cross-cut: each new fixture block keys on the closed `details.call` / discriminator-literal sets the relevant pages already pin; no new wire surface is introduced, and the *Count semantics* invocation-site framing of `diagnostic-emission-isolation.md` already governs the assertions on emission counts the fixtures will produce.

## Consequence

**Severity:** correctness

Two reasonable implementers will diverge on each of the four boundaries: one ships the byte-exact two-token / three-token construction-site form, the other interpolates `String(undefined)` or omits the `<unreadable>` sentinel; one truncates at 256 UTF-16 code units, the other at 256 bytes (or escapes the prefix colon, or splits a surrogate pair mid-boundary); one routes the empty/respond-only active-set installs through `pi.setActiveTools([])` / `pi.setActiveTools([respondToolName])`, the other silently unions the snapshot in; one services the `(drained:true, tag:undefined)` slash-handler tuple as the `"shutting-down"` arm per *unset tag fallback*, the other treats it as a fresh-runtime steady state and dispatches. All four divergences pass the prose-level reading of the obligations but split the wire and operator-visible behaviour the spec is otherwise pinning down to the byte.

## Problem

Four normative MUSTs across the Shard-08 PIC topics ship without enumerated acceptance criteria or fixture inventory, so a conformant implementation cannot be discriminated from a non-conformant one on first-principles reading: (1) the construction-site catch-arm fallback forms in `diagnostic-emission-isolation.md` (*Construction-site wrap* / *Construction-site catch-arm self-wrap*); (2) the PIC-17 empty / respond-only active-set sub-cases in `tool-registration-lifetime.md` that witness the "ambient tools are deliberately not inherited" invariant; (3) the `details.failure` `"throw:<String(error)>"` truncation and control-character escaping grammar in `unknown-reason-rule.md`; and (4) the `(drained: true, tag: undefined)` post-handler tuple on `quit`/`reload` routed to `readDrainState` arm (b) in `drain-state-contract.md`. Each gap shares one shape — a normative MUST plus an observable wire/state form with no enumerated fixture set — so a runtime can ship prose-compliant code that silently violates the contract on the un-fixtured boundary.

## Solution approach

Add an enumerated acceptance-criteria / fixture block colocated with each of the four obligations, each witnessing the un-fixtured boundary against the wire/state form the page already pins. For the construction-site catch arm (`diagnostic-emission-isolation.md` *Construction-site wrap* / *self-wrap*), witness the byte-exact two-token and three-token fallback emissions and continuation past the self-wrapped inner throw. For PIC-17, witness the exact `pi.setActiveTools` install vector on the empty-callable-set and forced-respond sub-cases, asserting the step-1 snapshot is not unioned in. For the throw-discriminator grammar (`unknown-reason-rule.md`), witness the 256-UTF-16-code-unit truncation boundary, each control-character escape class, the interior-colon case, and the `"throw:<unreadable>"` coercion-failure fallback. For the drain-state tuple (`drain-state-contract.md` `readDrainState` arm (b)), witness `(drained: true, tag: undefined)` reachability and arm-(b) routing on `quit`/`reload`. Match the existing per-page fixture-inventory convention; the implementation owns placement and harness shape.

## Solution constraints

- Fixtures key on the closed `details.call` / discriminator-literal sets the four pages already pin; introduce no new diagnostic code or wire field.
- Out of scope: the mechanism shapes F-1216 may demote — fixtures assert the surviving behavioural invariants, not the code shapes under that finding's review.

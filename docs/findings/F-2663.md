## Problem

The REQ-ID projection (`ids.md`) records every bold `**PREFIX-N**` span as a definition site, with no way to distinguish a definition (a bolded label co-located with an `<a id>` anchor) from a back-reference (a bolded mention citing an obligation another page owns). As a result three bold back-references — `**GOV-7**` / `**GOV-8**` in `placeholder-rendering-a.md:7` and `placeholder-rendering-b.md:131`, and `**ERR-7**` in `package-and-settings.md:98` — are indexed as canonical definition sites, so conformance auditors are sent to the wrong section; the real GOV-7/GOV-8 anchors (`#gov-7`, `#gov-8` in `req-id-prefix-table-active-b.md`) never appear in the index at all. The cause is a convention gap: GOV-1 reserves the bold-with-trailing-period form for anchored definitions, but nothing prohibits the bare bold form for prose back-references, so the projection conflates the two.

## Solution approach

Rewrite the three cited bold back-references as non-bold tokens, each hyperlinked to its canonical anchor: GOV-7 → `../governance/req-id-prefix-table-active-b.md#gov-7`, GOV-8 → `../governance/req-id-prefix-table-active-b.md#gov-8`, ERR-7 → `../errors-and-results/error-model.md#err-7` (the ERR-7 site already carries the link). Add a normative clause to the GOV-9 cross-link contract in `req-id-prefix-table-active-b.md` requiring REQ-ID back-references to use a non-bold hyperlinked token and reserving the bold label form for the co-located definition anchor.

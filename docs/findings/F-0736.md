## Finding

DISC-5 specifies that each glob is matched by "attempting each pattern against ‚Ä¶ the package-root-relative path, its basename, and its POSIX-normalised absolute path" but never states the combining rule for the three attempts. Two readings are observably distinct: a disjunctive rule (the candidate matches if the pattern matches **any** of the three forms) and a conjunctive rule (the candidate matches only if the pattern matches **all** three). The two diverge on virtually every realistic pattern ‚Äî e.g. a bare basename pattern like `foo.loom` matches the basename form for every candidate but matches the relative or absolute form only when the file sits at the package root, so conjunctive semantics would silently drop deeper matches that the disjunctive reading would register.

The matching paragraph adjacent to this one (the `+` / `-` exact-path operands) already commits to the disjunctive reading ‚Äî it says exact-path operands compare "against the relative path, the basename, and the POSIX-normalised absolute path, mirroring Pi's `matchesAnyExactPattern`" (the symbol name itself encodes "any"). The glob sentence omits an analogous statement, leaving the two halves of the same paragraph inconsistent about whether "attempt against three forms" combines with OR or AND.

The same opacity propagates to the settings-side `loomPaths` rules: their *Glob patterns and exclusions* bullet defers the matcher contract to DISC-5, so the conjunction gap silently leaks into `loomPaths` as well.

## Issue introduction

**Verdict:** single-commit
**Introducing commit:** `1cb54c7` ‚Äî *pi-loom spec: resolve "Glob !/+/- precedence and matcher engine unspecified for pi.looms / loomPaths"* (2026-06-06)
**History:** The DISC-5 bullet first entered the corpus in `f5e89f4` (2026-06-04) without the three-form matching prose ‚Äî at that point the spec said only that glob patterns were supported and resolved against the package root, with no enumeration of attempt targets. Commit `1cb54c7` added the present sentence ("attempting each pattern against ‚Ä¶ the package-root-relative path, its basename, and its POSIX-normalised absolute path") in the course of pinning the matcher engine to `minimatch` and citing Pi's `applyPatterns` / `matchesAnyExactPattern`. The same commit's adjacent edit to the `+` / `-` operand rule explicitly used the phrase "mirroring Pi's `matchesAnyExactPattern`", which records the disjunctive intent for the exact-path arm; the glob arm received the three-form enumeration but not the matching conjunction marker, so the defect is a single-commit omission rather than a multi-commit interaction.

## Solution Space

**Shape:** single

### Recommendation

Amend the DISC-5 glob sentence to state explicitly that a candidate matches a pattern when the pattern matches **any one** of the three rendered forms (package-root-relative path, basename, POSIX-normalised absolute path). Mirror the wording of the adjacent exact-path sentence so the two halves of the paragraph read symmetrically ‚Äî e.g. extend "attempting each pattern against the candidate's package-root-relative path, its basename, and its POSIX-normalised absolute path" to "‚Ä¶; a candidate matches the pattern when the pattern matches **any one** of those three forms, mirroring Pi's `matchesAnyPattern`". No change is needed to the `loomPaths` *Glob patterns and exclusions* bullet ‚Äî its existing back-reference to DISC-5 pulls the clarified contract through automatically.

Implementer-relevant edge cases the new wording must leave unambiguous:

- The disjunctive rule applies per pattern, not across the pattern set: a single pattern is admitted (or excluded, in the `!` arm) iff it matches at least one of the three forms for that candidate. The outer "pattern class" ordering (plain ‚Üí `!` ‚Üí `+` ‚Üí `-`) defined later in DISC-5 is unaffected.
- The same any-of-three-forms rule applies to `!` patterns (a path is dropped if the `!` pattern matches any one of its three rendered forms), since `!` reuses the same glob matcher.
- The `+` / `-` exact-path operands already state their any-of-three-forms rule via the `matchesAnyExactPattern` cite; the new glob sentence must not accidentally re-state that rule in a way that contradicts the exact-path wording.

## Consequence

**Severity:** correctness

Two reasonable implementers will diverge on every glob whose three rendered forms are not interchangeable ‚Äî most notably bare basename patterns (`foo.loom`, `*.loom`) against nested candidates. The disjunctive implementation registers them; the conjunctive implementation silently drops them. The divergence is silent (no diagnostic fires either way) and surfaces only as a missing slash command at runtime, which is precisely the discovery-layer bug class DISC-5 is meant to foreclose.

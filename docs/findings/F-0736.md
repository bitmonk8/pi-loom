## Problem

DISC-5's glob sentence (anchor `id="disc-5"`) specifies that each pattern is attempted against three rendered forms — the package-root-relative path, the basename, and the POSIX-normalised absolute path — but never states whether a candidate matches when the pattern matches **any one** form (disjunctive) or **all three** (conjunctive). The two readings diverge on common patterns: a bare basename pattern like `foo.loom` matches the basename form for every candidate but matches the relative or absolute form only when the file sits at the package root, so conjunctive semantics would silently drop deeper matches the disjunctive reading registers. The adjacent `+`/`-` exact-path operand sentence already commits to the disjunctive reading via its `matchesAnyExactPattern` cite, leaving the two halves of the same paragraph inconsistent. The `loomPaths` *Glob patterns and exclusions* bullet defers its matcher contract to DISC-5, so the gap propagates there too.

## Issue introduction

**Verdict:** single-commit
**Introducing commit:** `1cb54c7` — *pi-loom spec: resolve "Glob !/+/- precedence and matcher engine unspecified for pi.looms / loomPaths"* (2026-06-06)
**History:** The DISC-5 bullet first entered the corpus in `f5e89f4` (2026-06-04) without the three-form matching prose — at that point the spec said only that glob patterns were supported and resolved against the package root, with no enumeration of attempt targets. Commit `1cb54c7` added the present sentence ("attempting each pattern against … the package-root-relative path, its basename, and its POSIX-normalised absolute path") in the course of pinning the matcher engine to `minimatch` and citing Pi's `applyPatterns` / `matchesAnyExactPattern`. The same commit's adjacent edit to the `+` / `-` operand rule explicitly used the phrase "mirroring Pi's `matchesAnyExactPattern`", which records the disjunctive intent for the exact-path arm; the glob arm received the three-form enumeration but not the matching conjunction marker, so the defect is a single-commit omission rather than a multi-commit interaction.

## Solution approach

Clarify DISC-5's glob sentence (anchor `id="disc-5"`) to state explicitly that a candidate matches a pattern when the pattern matches **any one** of the three rendered forms (package-root-relative path, basename, POSIX-normalised absolute path), making the glob arm read symmetrically with the adjacent `+`/`-` exact-path operand sentence's existing disjunctive rule. The same any-of-three-forms rule covers `!` exclusion patterns, which reuse the glob matcher.

## Solution constraints

- Out of scope: the `loomPaths` *Glob patterns and exclusions* bullet — its existing back-reference to DISC-5 pulls the clarified contract through automatically; do not restate the conjunction rule there.

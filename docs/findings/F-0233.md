## Problem

`V2c` performs inbound wire-name translation by reattaching enum tags via a per-schema sidecar (`V2c` **Adds.**), and that two-map sidecar is produced by `V5f`'s lowering pass (`V5f` **Adds.**, **Tests.**). Yet the **Deps.** of both `V2c` and `V2c-T` name `V5d` — the schema-subset reject gate, whose own **Adds.** disclaims ownership of the lowering pass — instead of `V5f`, the actual producer. Because no edge points at the producer, the build graph permits `V2c` to be scheduled before `V5f` exists. Separately, no leaf is designated owner of the shared sidecar *type*, so `V5f` (producer) and `V2c` (consumer) can independently settle on divergent two-map shapes.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** c6a664e ("pi-loom plan: build/update plan for spec.md + review", 2026-06-10)
**History:** `git log --follow` shows `V2c-value-model.md` and `V2c-T-value-model.md` were both created in `c6a664e` and have no later commits. `git show c6a664e:docs/plan_topics/V2c-value-model.md` confirms the inception revision already carried both halves of the defect: **Adds.** named "enum-tag reattach via sidecar" and **Deps.** were `V2c-T, V2a, V5d` (V5d, not V5f). `git log -G "Deps.*V5d" -- docs/plan_topics/V2c-value-model.md` returns only `c6a664e`, so the V5d edge has never been the producer edge. Later `V5f`/`V5d` commits (`ba7da00`, `cb8737a`, `fdccb5d`) refined where the sidecar is described but never corrected `V2c`'s edge or assigned a type owner. The mis-targeted edge and the undesignated type owner have been latent since the plan's first build.

## Solution approach

In the **Deps.** of `docs/plan_topics/V2c-value-model.md` and `docs/plan_topics/V2c-T-value-model.md`, replace `V5d` with `V5f` so each consumer depends on the leaf that produces the sidecar it reads (`V5f` already deps `V5d`, so `V5d` stays transitively reachable). In `docs/plan_topics/V5f-subset-lowering-hash.md` **Adds.**, designate `V5f` as the owner of the shared per-schema sidecar type. Rephrase `docs/plan_topics/V2c-value-model.md` **Adds.** so the enum-tag reattach names the `V5f`-produced sidecar as the consumed seam.

## Solution constraints

- If the related `V5f`-split finding lands first, retarget the edge and the type-ownership statement to the sub-leaf that carries the lowering pass emitting the sidecar.

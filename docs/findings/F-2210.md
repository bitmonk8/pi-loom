## Problem

PIC-49 (`registration-steps.md#pic-49`) states a cross-window serialization property: a debounce timer that fires while a prior window's rebuild is still awaiting its asynchronous steps MUST defer rather than start a concurrent rebuild. But it pins no determinism seam capable of constructing the precondition the rule is about — an in-flight rebuild suspended at one of its `await` points. The documented seams cover only half the scenario: `FakeClock` (PIC-12) fires the second window's debounce timer synchronously, but cannot hold the first window's rebuild paused at an awaited re-parse / recompile / `pi.registerTool` step. Unlike PIC-10's `Checkpoint` seam for cancellation races, no seam exposes a controllable async boundary for the rebuild path, so the deferral cannot be deterministically exercised and a conformance test cannot distinguish PIC-49 from PIC-36's intra-publish atomicity.

**Verdict:** present-since-inception
**Introducing commits:** `443ee4e` (2026-06-25, "pi-loom spec-plan: fix F-1159 — coin PIC-49 cross-window rebuild serialization rule")
**History:** PIC-49 entered the corpus whole in a single coining commit (`443ee4e`), which added the rule to `registration-steps.md` to resolve finding F-1159. `git log -S 'PIC-49'`, `git log -G 'Cross-window rebuild serialization'`, and `git log -S 'concurrent rebuild'` over that file each return only `443ee4e`; the diff of that commit shows the rule was authored with its deferral prose but with no determinism seam for suspending the in-flight rebuild and no acceptance criteria. The rule has never carried the determinism-seam discipline applied to the corpus's other interleaving-sensitive properties (PIC-10/PIC-12), so the testability gap is original to the rule, not introduced by a later edit.

## Solution approach

Add a normative testability clause to PIC-49 modelled on PIC-10's `Checkpoint` discipline: a runtime-internal hook the rebuild `await`s at its asynchronous steps, whose production wiring resolves immediately with no observable production effect and whose test wiring can hold a prior window's rebuild suspended while `FakeClock` fires the next window's debounce timer. Keep the seam internal (not on `ExtensionAPI`), wired through the same constructor-injection skeleton as the other seams. State PIC-49's observable pass/fail conditions against that seam plus `FakeClock`. Per GOV-18 arm (a), whether the suspension point is a dedicated rebuild hook or a gated already-injected async boundary may be left open — what must be pinned is that some deterministic suspension point exists.

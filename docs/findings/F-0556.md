## Problem

The depth-6 co-fire Tests bullet in `V13c-query-tool-loop.md` (and mirrored verbatim in `V13c-T-query-tool-loop.md`) reads "ceiling #4 surfaces ... as `Err(QueryError {...})` ... and the surface's `masked` enumerates the co-satisfied-but-masked ceiling #2". The anaphor "the surface's `masked`" binds to the immediately preceding `Err(QueryError{...})`, so a natural reading makes `masked` a field of `QueryError`. The spec places `masked` exclusively on the operator-facing `RuntimeEvent` at `details.event.masked`, never on `QueryError` — no `QueryError` variant carries it. An implementer working the bullet alone can wire `masked` into the `QueryError` shape, producing an out-of-spec result type and a V13c-T assertion that contradicts the spec's worked example.

## Issue introduction

**Verdict:** single-commit-introduction
**Introducing commit:** `0dd2436` — "pi-loom plan: resolve \"V13c exhaustion bullet leaves depth-6 surfaced ceiling unstated\"" (2026-06-11)
**History:** The depth-6 co-fire Tests bullet did not exist before `0dd2436`; that commit added it (in both V13c and V13c-T) while resolving an earlier finding that the exhaustion bullet left the depth-6 surfaced ceiling unstated. The added bullet introduced the "the surface's `masked`" anaphor immediately after the `Err(QueryError{...})` clause. A later edit (`9d880b8`) reworded the surrounding bullet but preserved the ambiguous phrasing. `git log -S"the surface's \`masked\`"` over both leaf files identifies `0dd2436` as the sole introducing commit.

## Solution approach

Rewrite the depth-6 co-fire Tests bullet in both `V13c-query-tool-loop.md` and `V13c-T-query-tool-loop.md` so the two observables are separated and `masked` is unambiguously attributed to the `RuntimeEvent` rather than the `QueryError`. State the loom-code-visible `Err(QueryError {...})` return surface on its own, then in a separate clause attribute the `masked` enumeration to the `RuntimeEvent` at the wire location `details.event.masked`. Cite that wire location explicitly so the field cannot be read as a `QueryError` member.

## Solution constraints

- Apply the identical rewrite to both leaves; keep the V13c and V13c-T bullets mirrored.

## Problem

`V9i`'s final Tests bullet (the `ERR-8` delegated live-carrier witness) and its **Ships when** gate require an `npm test` assertion that a mid-stream cancellation inside the *real* subagent `AgentSession` leaves the session's committed turns unmutated. Observing a mid-stream cancellation requires the spawned real session to be producing an interruptible token stream, but the only model source in the spawn path is `ctx.modelRegistry.getAvailable()` feeding a live provider. Neither `V9i`'s Adds nor Deps names an offline stub-model or transport seam that would feed a real `createAgentSession` session a scripted, cancellable stream under `npm test`. The gate is therefore unsatisfiable offline without infrastructure the plan never enumerates, forcing an implementer to either invent an undeclared seam or read the bullet as a real-host-only assertion.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:** `7bd65a7` (pi-loom plan: resolve "V4c/V4f defer ERR-8 to ERR-13 live-surface confirmation to downstream owners that never re-assert it", Thu Jun 18 2026) introduced the `ERR-8` delegated live-carrier witness bullet on `V9i`, naming "the real subagent `AgentSession`" and adding `V4c` to Deps but no stream-source seam; `f892019` (pi-loom spec-plan: fix F-1243 — add ERR-8 red-first witness to V9i-T + Ships-when gate, Tue Jun 23 2026) added the paired `V9i-T` bullet and rewrote `V9i`'s **Ships when** to assert the real-session ERR-8 outcome under `npm test`.
**History:** `git log --follow` / `git log -S` over `V9i-subagent-isolation.md` show the bullet did not exist before `7bd65a7`. That commit's purpose was to give `V4c`'s deferred ERR-8/ERR-12 live-surface confirmation a downstream owner on the real `V9i` surface; the latent gap — a real-session witness with no enumerated offline stream — entered there. `f892019` then promoted the latent assumption into an explicit `npm test` Ships-when obligation, which is the point the gap became an unsatisfiable gate. Neither commit added a stub-model/transport seam to Adds or Deps.

## Solution approach

Treat the real-`AgentSession` mid-stream-cancellation outcome (ERR-8) as a real-host-only behaviour: remove the real-session ERR-8 outcome from `V9i`'s **Ships when**, and re-cast the `ERR-8` Tests bullet so its live-surface confirmation is asserted at the manual real-host smoke gate (`real-host-smoke-gate.md`) rather than under `npm test`. Mirror the same change in the paired `V9i-T` `ERR-8` Tests bullet. Re-state `V4c`'s deferred ERR-12 live-surface re-assertion to point at the smoke-gate witness, and reconcile the `coverage-matrix.md` `ERR-8 … ERR-12` row's `V9i` co-witness annotation with the relocated assertion so the matrix no longer claims a live-carrier witness under `npm test`.

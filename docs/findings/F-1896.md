## Finding

The closing paragraph of the manual real-host smoke governance note (`docs/plan_topics/real-host-smoke-gate.md`) states: "A confirmed behavioural-divergence finding **from that smoke** is a **merge blocker**." The demonstrative "that smoke" has no single clear antecedent. The two immediately preceding sentences are about the *skipped*-run / not-CI-enforced case — "a skipped run … is **not blocked from merging**; such an omission is detectable only **after the fact**, by reading the triggering commit message for the recorded run" — so the nearest run-referents in the prose are "a skipped run" and "the recorded run", not the gate the note defines at its top.

A reader can garden-path "that smoke" onto the skipped run, which is incoherent — a skipped run produces no finding to begin with. The intended referent is the manual real-host smoke run conducted under this protocol (the gate defined at the head of the note), whose pass/fail criteria (a)–(e) are what produce a "confirmed behavioural-divergence finding". The meaning is recoverable on reflection, but the demonstrative forces the reader to back out of the local context to resolve it.

## Solution Space

**Shape:** single

### Recommendation

In the final paragraph of `docs/plan_topics/real-host-smoke-gate.md`, replace the phrase `from that smoke` in the sentence "A confirmed behavioural-divergence finding from that smoke is a **merge blocker**" with an explicit referent to the protocol's defined gate, e.g. `from any pre-merge smoke run conducted under this protocol`. This pins the antecedent to the manual real-host smoke run the note defines (the run scored on criteria (a)–(e)), so the sentence no longer competes with the adjacent "skipped run" / "recorded run" phrases. The fix is confined to this one sentence; the spec is read-only for this finding, and no leaf bullet is touched.

## Consequence

**Severity:** cosmetic

A contributor or the `H6a` release-gate auditor reading the closing sentence must momentarily disambiguate "that smoke" against the wrong nearby referent (the skipped run) before settling on the protocol's defined smoke run. No reader would actually conclude a skipped run is a merge blocker — a skip yields no finding — so the gate's behaviour is not at risk; the cost is purely a reading-clarity bump in governance prose.

## Problem

The compact-transcript format in `binder-model-and-context.md` is a byte-exact, MUST-reproduce-exactly contract whose BNDR-7 reference renderings are the conformance oracle for each input shape, and the rendering function is required to be total over the `AgentMessage` union. Rule 4's `assistant` bullet pins the thin sub-case of no `TextContent` but at least one `ToolCall` (oracle BNDR-7f), but never pins the sibling case where an assistant message's `content` is only `ThinkingContent` — no `TextContent`, no `ToolCall`. This is a real input shape (for example a turn aborted mid-thinking before any text or tool call), and under rule 4 its text concatenation is empty, all thinking blocks drop, and there are no tool-call lines, so two conforming implementers diverge: one emits a bare `[assistant]: ` line, the other omits the message entirely. No BNDR-7 rendering covers the case, so there is no byte-exact oracle to adjudicate and the input-reproducibility contract these rules exist to guarantee is undermined.

**Introducing commit:** `20569f8` — *pi-loom spec: resolve "Compact-transcript format for the session-context block is unspecified"* (2026-05-07)
**History:** The corpus is git-tracked. `git log -S` traces every defect token (the `ThinkingContent`/`thinking` omission rule, the no-text-with-tool-calls clause, and the "role boundary stays detectable" phrasing) back to `20569f8`, the commit that first specified the compact-transcript format. From its introduction the section pinned the thinking-omission rule and the empty-text-with-tool-calls sub-case but never the thinking-only sub-case. A later hardening pass — `1a41db2` (*"Compact-transcript BNDR-7 oracle coverage and assistant byte-determinism (T034/T106)"*, 2026-06-06) — reworked rule 4 onto the `AssistantMessage.content` union and added the BNDR-7e/7f and BNDR-8 oracles, explicitly covering the adjacent empty-text-with-`ToolCall` case while leaving this sibling case uncovered. The file split at `f5e89f4` only relocated the prose; it did not introduce the gap. The defect has therefore been present since the section's inception.

## Solution approach

Clarify rule 4's `assistant` bullet to cover the case where `content` has no `TextContent` and no `ToolCall` — thinking-only, and degenerately empty `content` — folding both into one clause: the message emits the `[assistant]: ` prefix line with empty body and the rule-3 trailing U+0020, and no further lines, following the BNDR-7f precedent. Add a BNDR-7 reference rendering for a thinking-only assistant turn with its byte-exact transcript body so the conformance oracle covers it, taking the next free sub-letter per the BNDR-7 append rule and not renumbering existing renderings.

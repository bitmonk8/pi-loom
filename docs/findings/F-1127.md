## Finding

The `#### System-prompt structure (normative)` sub-section of `docs/spec_topics/binder/binder-bypass-and-envelope.md` enumerates eleven distinct normative obligations, each carrying RFC-2119 MUST/MUST NOT modal tokens: the eight numbered structural items (Loom-identity line, Description line, Argument-hint line, Parameters block, User-arguments line, Session-context block, Envelope-kinds enumeration, No-invent-defaults instruction), the *Type display* reference-rendering table, the *Default-literal rendering* rule, and the *Parameter-line reference renderings* table (which itself includes a fourth description-omitted normative rendering in the trailing prose). None carries a `BNDR-N` REQ-ID anchor in the GOV-1 dual form, even though the same page already uses `BNDR-1`…`BNDR-3` and the spec corpus uses `BNDR-4`…`BNDR-9` elsewhere under the page-prefix table entry for `binder/`.

The consequence is twofold. First, GOV-9's `#prefix-n` cross-link contract is unsatisfiable for any other page or finding that needs to depend on a single structural item — citations must name the heading slug `#system-prompt-structure-normative` and disambiguate by item number, by *Type display* / *Default-literal* / *Parameter-line* sub-section name, or by quoting the obligation. Second, downstream artefacts already feel the pressure: coverage-matrix row `cka-45` enumerates the whole eleven-obligation cluster as one row tagged `(un-anchored; GOV-22 residue)`, and the `V11d` / `V11d-T` Tests bullets must list "all eight structured items (1–8)" plus each reference-rendering table by name rather than cite individual REQ-IDs. The section is standing GOV-22 progressive-coinage residue and the consuming surfaces already advertise that.

## Solution Space

**Shape:** single

### Recommendation

Coin one `BNDR-N` REQ-ID per distinct structural obligation in `#### System-prompt structure (normative)`, in declaration order, each anchored in the GOV-1 dual form (`<a id="bndr-n"></a> **BNDR-N.**`) at the start of its numbered item or reference-rendering sub-section. Allocate the numeric tails under the page's already-registered `BNDR` prefix per GOV-3 (next free integers starting from `BNDR-10`). The eleven distinct obligations to coin are:

1. Item 1 — Loom-identity line.
2. Item 2 — Description line (conditional presence, both halves).
3. Item 3 — Argument-hint line (conditional presence, both halves).
4. Item 4 — Parameters block (header, indent bytes, per-field token order, em-dash separator, conditional-description omission, no-`params` omission of the whole block).
5. Item 5 — User-arguments line.
6. Item 6 — Session-context block (opening line, body framing, terminating blank line, zero-included-turns omission).
7. Item 7 — Envelope-kinds enumeration.
8. Item 8 — No-invent-defaults instruction.
9. *Type display* — declared-Loom-type table.
10. *Default-literal rendering* — Loom-literal-sublanguage surface form for `default=<literal>`.
11. *Parameter-line reference renderings* — the three-row example table plus the description-omitted-form rendering pinned in the trailing paragraph.

Item 4 is the natural sub-letter candidate: its six bundled obligations (header line, two-space indent, fixed token order, em-dash separator bytes, conditional description omission, no-`params` block-level omission) can each carry a page-local sub-letter anchor (`bndr-Na`…`bndr-Nf`) under the same scheme `BNDR-6` and `BNDR-7` already use elsewhere in the binder shard, if the implementer prefers per-clause addressability over a single composite ID. The same option is available for item 6's four sub-obligations.

After coining, retarget the consuming citations in the same diff: replace the heading-slug-plus-item-number enumeration in `V11d-T-binder-system-prompt.md` and `V11d-binder-system-prompt.md` Tests bullets with the new `BNDR-N` anchors, and remove the `(un-anchored; GOV-22 residue)` annotation from `cka-45` in `docs/plan_topics/coverage-matrix.md` (retargeting its prose onto the new anchors).

Edge cases: the *Parameter-line reference renderings* table sits inside a markdown table, which triggers GOV-1's *table-cell leading-token* mechanics — the dual-form anchor must be the cell's leading token sequence, or the rule must be lifted out of the table per GOV-1 *Edge cases*. The description-omitted-form paragraph that follows the table is a separate normative rendering and should carry its own anchor in body-paragraph context.

## Consequence

**Severity:** advisory

A test failing any one item — say the indent-bytes MUST in item 4 or the third row of *Parameter-line reference renderings* — cannot cite a `BNDR-N` REQ-ID in its assertion message; failure reports collapse to the heading slug plus an item number or row index, and consumers (the `V11d` Tests bullet, `cka-45`, any future page that needs to depend on a single item) must enumerate the obligation rather than link to it. Implementers can still produce a conforming renderer because the obligations themselves are precise; the gap is in traceability, not in correctness.

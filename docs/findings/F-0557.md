## Problem

The depth-6 co-fire Tests bullet in both `docs/plan_topics/V13c-query-tool-loop.md` and `docs/plan_topics/V13c-T-query-tool-loop.md` ends with the parenthetical "*(the event shape carries only `kind` plus the optional `masked` field)*". This is a false closed-shape claim: the normative `RuntimeEvent` shape in `spec_topics/pi-integration-contract/runtime-event-channel.md` (PIC-1) carries multiple required members beyond `kind` — `loom`, `invocation_id`, `message`, `occurred_at` — and the spec's own depth-6 worked example that V13c cites surfaces a seven-field payload. An implementer trusting the plan over the cited spec could write an over-narrow conformance assertion that rejects a conformant event.

The false "carries only `kind` plus the optional `masked` field" closed-shape claim entered as part of that clarification. The defect is collateral from a prior plan-review remediation, not present since inception.

## Solution approach

Rewrite the trailing parenthetical of the depth-6 co-fire Tests bullet to strike the false "carries only `kind` plus the optional `masked` field" closed-shape claim while retaining the valid point that there is no literal `surfaced:` wire key. Defer the full `RuntimeEvent` payload shape to its normative owner, `runtime-event-channel.md` (PIC-1, exercised by V9d's `RuntimeEvent`-conformance test). Apply the identical edit to both mirrored leaf files.

## Problem

The `ToolDefinition.label` field-derivation MUSTs (basename with interior hyphens preserved and the leading character capitalised, e.g. `code-review.loom` → `"Code-review"`; the synthesised typed-query one-shot tool's literal label `"Loom typed-query response"`) are carried by `V9f` in its **Adds** and a dedicated **Tests** bullet, but the paired test driver `V9f-T` has no label-derivation bullet — its **Tests** list only `PIC-8`, `PIC-19`, and the visibility / `registration-cache-collision` bullet. Under the per-phase TDD ritual (`V9f-T` red first, then `V9f`), a behaviour absent from `V9f-T` is never written red-first and can ship green-from-birth or be omitted. Neither leaf's **Ships when** names the label derivation: `V9f` Ships when asserts only the `PIC-8` restore-failure and `PIC-19` install-failure paths. The coverage-matrix row pointing at `V9f` is therefore satisfied vacuously — a wrong or missing label materialisation would ship undetected.

## Issue introduction

**Verdict:** introduced-by-edit
**Introducing commits:** cfd76f7 ("pi-loom plan: resolve \"Per-loom registration ToolDefinition field-derivation MUSTs unnamed by any leaf\"", 2026-06-12)
**History:** Both leaves were created in `c6a664e` (initial plan build) without any label-derivation content. Commit `cfd76f7` resolved a prior review finding that the `ToolDefinition.label` field-derivation MUSTs were unnamed by any leaf: it added the label derivation to `V9f`'s **Adds** and a new `V9f` **Tests** bullet, and added the closing-leaf row to `coverage-matrix.md`. That same edit did not add the corresponding red-first bullet to `V9f-T` and did not name the assertion in `V9f`'s **Ships when**. The asymmetry — closing-leaf obligation present in the implementation leaf and the matrix, absent from the test driver and the ship gate — was introduced wholly by `cfd76f7`.

## Solution approach

Add a label-derivation test bullet to `V9f-T`'s **Tests** mirroring the existing `V9f` label-derivation bullet, citing `extension-bootstrap-and-per-loom.md` §Per-loom registration. Name the label-derivation assertion in `V9f`'s **Ships when** alongside the existing `PIC-8` / `PIC-19` clauses so it becomes a completion criterion.

## Solution constraints

- Out of scope: `coverage-matrix.md` (its row already points at `V9f` correctly) and the spec (the spec MUST already exists; the spec is read-only for this fix).

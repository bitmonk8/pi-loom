## Problem

The §Leaf format **Deps.** field in `conventions.md` restates the transitive-completeness obligation using the same three-item "or" trigger structure as the authoritative *Transitive-completeness plan-maintenance* clause in §REQ-ID discipline, but its first arm ("carry an executable REQ-ID") omits the disambiguating parenthetical that narrows it — "a leaf that merely re-cites an already-closed REQ-ID does not qualify." Because the Deps field is reference text read at point-of-use before the author follows the pointer down to §REQ-ID discipline, an author acting on its literal first arm concludes that any leaf citing any executable REQ-ID must be added to `H5b`'s `Deps.`, contradicting the authoritative rule. The summary defers to the authoritative clause for the exact triggers, so a careful author who chases the pointer recovers the correct rule; the defect is that the summary reads as self-contained and is acted on first.

## Issue introduction

**Verdict:** single-commit
**Introducing commits:** ab8e297 — pi-loom plan: resolve "Transitive-completeness obligation invisible at leaf authoring" (2026-06-11, Thomas Andersen)
**History:** The §Leaf format **Deps.** field carried no transitive-completeness restatement until ab8e297 added the "A new leaf … is additionally added to `H5b`'s `Deps.`" summary, reusing the three-item "or" trigger structure. Six minutes earlier the same day, d7173b3 ("resolve 'Transitive-completeness rule trigger broader than H5b coverage set'") had already added the disambiguating parenthetical — "a leaf that merely re-cites an already-closed REQ-ID does not qualify" — to the authoritative §REQ-ID discipline clause. ab8e297's summary restatement did not carry that disambiguation through, so the Deps-field summary was born divergent from the authoritative clause it points to.

## Solution approach

In the §Leaf format **Deps.** field of `conventions.md`, delete the three-item trigger enumeration and replace it with a single alerting clause that signals a new leaf may need adding to `H5b`'s `Deps.` and defers to the *Transitive-completeness plan-maintenance* clause in §REQ-ID discipline for the exact triggers, without restating them. This leaves §REQ-ID discipline as the sole owner of the trigger conditions and decouples the fix from how the authoritative clause is worded.

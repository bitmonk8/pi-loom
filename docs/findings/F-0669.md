## Finding

The normative *Failure-mode templates* table in `docs/spec_topics/binder/determinism-cancellation-failure.md` enumerates six user-facing system notes that "Renderers MUST emit … verbatim". Five of the six rows phrase the facility as **argument binding** (`argument binding needs more info`, `ambiguous arguments` — which is a stylistic outlier but reads consistently with the binding framing, `argument binding failed`, `argument binding produced invalid args`, `argument binding cancelled`). The transport-failure row, alone, uses **argument binder** (`argument binder unavailable (<provider>: <message>)`). The surrounding paragraph treats wording changes as spec-versioned breaking changes, so both spellings are now pinned obligations in the same table with no note that they refer to the same facility.

The split has corpus-wide reach because operators read the system notes as a single product surface, the glossary defines `binder` and `binder model` (noun) distinctly from the action `argument binding` (gerund), and the rest of the binder topic group consistently prefers `argument binding` for the operator-facing phrase. Carrier plan leaves (`V11f`, `V11j`) repeat the templates verbatim, so any harmonisation must propagate through them.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commits:** 83802db (`spec: collapse to single typed argument style; add LLM-driven slash-command binder`, 2026-05-04)
**History:** The LLM-driven binder feature was introduced in 83802db as a single block inside the monolithic `spec.md`, already containing the six failure-mode rows in their current split form — five rows phrasing the facility as `argument binding` and the transport row alone using `argument binder unavailable`. `git log -S "argument binder unavailable" -- '*.md'` confirms 83802db is the unique introducing commit for the noun form; `git log -S "argument binding needs more info" -- '*.md'` shows the gerund forms entered in the same commit. The fecb504 split (2026-05-04) moved the table into `docs/spec_topics/binder/determinism-cancellation-failure.md` unchanged. Subsequent edits to that file (43bef30, b884cda, d0bcdb2, 47c0950, 2fb5783, 1f4f536) touched adjacent concerns — `<provider>` derivation, retry timing, classifier folding — but left both spellings as authored. The inconsistency has therefore been a stable property of the feature from its first appearance, not a regression introduced by any later edit.

## Solution Space

**Shape:** multiple

### Option A — Harmonise the transport row to `argument binding unavailable`

**Approach.** Rewrite the transport-failure row to `loom /<name>: argument binding unavailable (<provider>: <message>)`. Update BNDR-3, `V11f`'s six-template assertion, and `V11j`'s cancelled-note assertion only insofar as they cite the transport row (BNDR-3 names only the `needs_info` and `ambiguous` prefixes, so it is unaffected; `V11f`'s "six failure templates render verbatim" bullet picks up the new wording automatically; `V11j` cites only the cancelled-binder row and is unaffected).

**Spec edits.**
- One literal change in `determinism-cancellation-failure.md` (the `Binder model transport failure` row).
- No glossary edit (glossary already uses `binder` for the noun and `argument binding` for the action; the table simply stops mixing them).

**Pros.** Single canonical phrase across all six operator-visible templates; no aliasing footnote to maintain; the table reads as one consistent product surface; downstream code that parses the leading phrase no longer special-cases one row.

**Cons.** Changes a user-facing literal. The spec's "Wording changes are spec-versioned breaking changes" clause applies; pre-1.0, that clause's audience is hypothetical, but if any review-stage demo notes are already in the wild they shift.

**Risks.** Low. Carrier plan leaves consume the templates by reference, so the change propagates automatically. Reviewers of `V11f` should re-confirm the six-template fixture string after the spec edit lands.

### Option B — Keep the transport row as authored, add an alias preamble

**Approach.** Leave all six row literals unchanged. Add a sentence to the paragraph immediately above the table (or to the glossary's `binder` entry, cross-referenced from the table) stating that the transport-failure row's `argument binder` and the other rows' `argument binding` refer to the same facility — the former naming the LLM-driven mechanism as a noun and the latter naming the action it performs.

**Spec edits.**
- One added sentence in `determinism-cancellation-failure.md` (preamble to the table) or one sentence appended to `glossary.md`'s `binder` entry.

**Pros.** Non-breaking; preserves every authored literal; minimal diff; respects the "wording changes are breaking" clause without invoking a pre-1.0 carve-out.

**Cons.** Persists the surface inconsistency permanently — every future reader still hits the outlier and now also has to read the aliasing note. Glossary debt: a third surface (the table) starts depending on a glossary disambiguation that previously was internal to the glossary itself.

**Risks.** Low; the aliasing prose itself does not change any normative obligation.

### Recommendation

Option A. The spec is pre-1.0, no implementation exists, and the "wording changes are breaking" clause exists to protect post-release implementers — exactly the audience the harmonisation serves by giving them one canonical phrase to parse. The diff is one literal in the spec and propagates through `V11f` automatically because that leaf cites the templates by reference rather than by transcription. Edge cases for the implementer: when updating the `V11f` test fixture, refresh the verbatim string for the transport-failure row only; do not touch the other five rows, BNDR-3, or the BNDR-7c example (none of those carry the `argument binder unavailable` literal).

## Consequence

**Severity:** advisory

Two implementers following the table verbatim will produce both spellings correctly — there is no behavioural divergence. The cost is conceptual: a reader scanning the six rows perceives the transport-failure row as referring to a distinct entity ("the argument binder", a noun) while the other five treat the failure as occurring during *argument binding* (a process). Glossary readers who index on `binder` will follow a different mental path than readers who index on `argument binding`. Operator-facing diagnostics that group these notes (e.g. by parsing the leading phrase after `loom /<name>: `) get a one-off outlier to special-case.

## Finding

`V9f`'s **Tests** list three behavioural vectors: the `PIC-8` restore-failure path, the `PIC-19` install-failure path, and a third bullet — "The callable set's visibility tracks the invocation; `registration-cache-collision` disambiguates a slug clash." The paired `V9f-T` driver carries the same three vectors and gates them red-first via its own **Ships when** ("the tests above exist, compile, and fail red").

`V9f`'s **Ships when**, however, names only the first two: "`npm test` asserts the restore-failure path (`PIC-8`) and the install-failure path (`PIC-19`)." The visibility / `registration-cache-collision` assertion has no green gate in the implementation leaf, so the leaf's done-condition undercounts its own Tests list.

Edit `docs/plan_topics/V9f-tool-registration-lifetime.md`, **Ships when** field, to include the third Tests vector alongside `PIC-8`/`PIC-19` — e.g. extend the existing sentence so it also asserts that the callable set's visibility tracks the invocation and that `registration-cache-collision` disambiguates a slug clash.

## Problem

GOV-23 (`#gov-23` in `governance/anchor-scheme-and-retired.md`) issues SHOULD-level citation guidance keyed on the classifier "orientation-level inbound link," but that term is never defined and has no glossary entry or anchor. Two readings compete: page-scoped (a link is orientation-level when it originates on an orientation/aggregator page) versus purpose-scoped (a link is orientation-level when its purpose is wayfinding / aggregate citation of the cluster as a unit). The readings diverge on a real case — a normative topic page citing the session model as a whole prescribes the umbrella anchor under the purpose-scoped reading but a specific `sm-N-…` anchor under the page-scoped reading. GOV-23 binds the corpus, so the ambiguity governs how every author targets session-model anchors.

## Issue introduction
**Verdict:** multi-commit-interaction
**Introducing commits:** `40ffdc5` (precursor — informal coinage), `af9288d` (defect — normative elevation)
**History:** The spec corpus is git-tracked. `git log -S "orientation-level inbound link"` and `git log -G` show the phrase first appearing in `40ffdc5` ("pi-loom spec: resolve T19a — Replace session-model paragraph with eight SM-N sub-units", 2026-05-27), which coined "orientation-level inbound links" informally in the `session-model` umbrella-anchor prose without defining it. `af9288d` ("pi-loom spec: resolve editorial and governance meta-commentary misplaced in orientation", 2026-06-03) then added GOV-23 as a new rule (`git show af9288d` shows the `<a id="gov-23">` block as an added line), promoting the same undefined phrase into a normative SHOULD where the page-scoped vs. purpose-scoped ambiguity actually bites. The defect is the interaction: an informally-coined term (`40ffdc5`) was elevated to a normative classifier (`af9288d`) without an intervening definition step.

## Solution approach

Add a defining clause for *orientation-level inbound link* to GOV-23 (`#gov-23`), resolving it on the purpose-scoped reading: a link whose purpose is to reach the session model — or an SM-3 / SM-7 group — as a unit for wayfinding or aggregate reference, as opposed to one that pins a single sub-obligation. Keep the classification binary against the existing "pins a single sub-obligation" arm so no inbound link is left unclassified by the SHOULD.

## Problem

`V4d`'s **Adds** claims ownership of the nine-variant `QueryError` union's `kind`/`cause` wire forms, and its **Ships when** asserts `npm test` proves "the nine-variant shape". But `V4d`'s **Tests** (mirrored in `V4d-T`) name only `ERR-14`, `ERR-15`, `ERR-17`, and `ERR-19`, which assert wire forms only for the `ValidationError` family and `ToolLoopExhaustedError`. The remaining variants — `TransportError`, `ModelToolError`, `ContextOverflowError`, `CancelledError`, `CodeToolError`, `InvokeInfraError`, `InvokeCalleeError` — have no asserting Tests bullet, so the gate behind "asserts the nine-variant shape" is vacuous for them and a wrong `kind` discriminator or missing/renamed field can ship green. `ContextOverflowError` is the sharpest case: `V4d`'s Adds singles it out as the sole NOCEIL-2 surface and delegates its wire shape to `V4d`, yet no Tests bullet asserts its `kind`/field set.

## Issue introduction

**Verdict:** present-since-inception
**Introducing commit:** `c6a664e` (2026-06-10) "pi-loom plan: build/update plan for spec.md + review"
**History:** The scope mismatch was authored at the initial plan build. `git show c6a664e:docs/plan_topics/V4d-queryerror-variants.md` shows the Adds already claimed "the nine-variant `QueryError` union (its `kind`/`cause` wire forms)", the Tests already named only `ERR-14`/`ERR-15`/`ERR-17`/`ERR-19`, and Ships-when already asserted "the nine-variant shape" — `ContextOverflowError` has carried no Tests bullet from the start. `git log --follow` over the leaf shows three later commits that touched adjacent text without closing the gap: `eaa2893` (2026-06-10, narrowed the `ERR-19` firing-at-the-cap scope, confirmed via `git log -G 'asserts the nine-variant'`), `e8c1d65` (2026-06-10, added the `ContextOverflowError` NOCEIL-2 sole-surface annotation in Adds, confirmed via `git log -S`), and `0b8a0f8` (2026-06-11, reworked the Tests bullets' non-assertion notes). None added a wire-form assertion for the un-named variants.

## Solution approach

Add Tests bullets to `V4d-T` and the mirroring entry in `V4d` that assert each of the nine variants' wire form — the `kind` discriminator string and the field set declared in `queryerror-variants.md` — explicitly covering `ContextOverflowError` (`kind: "context_overflow"` and its field set). Keep `V4d`'s **Ships when** "nine-variant shape" wording, which then matches what the gate proves.

## Solution constraints

- The spec is read-only for this fix; `queryerror-variants.md` is the canonical wire-shape declaration and is not edited.
- Do not retighten `ERR-15`'s `kind` open seam: the added assertions cover the closed loom 1.0.0 variant set as the runtime's emission set while the `kind` *type* stays `string`.

## Problem

In `runtime-event-channel.md`, the Group B bullet's exclusion-list maintenance sentence directs that "a new diagnostic added to **either of those `console.error` sinks** MUST be added to this exclusion list." The phrase "either of those … sinks" implies two distinct delivery sinks, but there is only one: all five carved-out teardown-handler codes emit through the single `console.error` route in the `session_shutdown` handler. The two items the sentence calls "those sinks" are not sinks — they are the two documentation locations (the **Edge cases** bullet under **Extension entry point** sub-step 4, and the **Persistent diagnostics** paragraph in Diagnostics) where the single carve-out is described. A maintainer cannot map the binary framing onto any real two-way distinction, so the directive's trigger condition is unclear.

## Issue introduction
**Verdict:** single-commit
**Introducing commit:** `75a9626` (2026-05-09) — "pi-loom spec: resolve \"SessionShutdownEvent.reason closed set has no build-time pin against the SDK type\""
**History:** The group-B exclusion-maintenance sentence was added in `75a9626` with the phrasing "A new diagnostic added to **either of those sinks** MUST be added to this exclusion list in the same edit" already present (then four exclusions, referencing the same two documentation locations). `2e1888c` (2026-06-05, resolving "Group B routing self-contradicts, risking double-emission") added the fifth code and inserted the `console.error` qualifier — "either of those `console.error` sinks" — but preserved the two-sink mislabel; `78a2f94` (2026-06-04) relocated the section into `runtime-event-channel.md` unchanged. The defect entered as incidental prose in a finding-resolution commit and has been carried forward verbatim through every later edit; no commit ever introduced two genuinely distinct sinks for the phrasing to track.

## Solution approach

Rewrite the Group B exclusion-list maintenance sentence so it refers to the single `console.error`-routed teardown-handler set rather than to two sinks, and frames the two parentheticals (**Edge cases** bullet and **Persistent diagnostics** paragraph) as the documentation locations where that single carve-out is described. Preserve the maintenance obligation and both cross-references.

## Finding

The page's `## Idempotent re-entry and the all-three-throw corner case` section, together with the *Slash-dispatch interaction* paragraph of `drain-state-contract.md` it implicitly cross-references, pins two distinct observable behaviours on the loom 1.0-accepted all-three-throw corner case (sub-step 1 `LoomRegistry.drain`, sub-step 1 `LoomRegistry.initDrainStateTag`, and the post-sub-step-4 `LoomRegistry.markRuntimeDegraded` all throw on a session-only `session_shutdown`, leaving the post-handler tuple `(drained: false, tag: undefined)`):

1. Subsequent slash invocations consult `LoomRegistry.readDrainState()`, observe the steady-state tuple, route through arm (a) — i.e. the loom is **dispatched** (neither the `"shutting down"` nor the `"degraded; /reload to recover"` system note is returned) into a half-torn-down runtime whose watchers and listeners are already gone.
2. The runtime MUST NOT introduce a third boolean drain-state field, an arm-specific "post-failed-handler" gate, or any other suppression mechanism that would conditionally short-circuit slash dispatch on this post-handler tuple.

The acceptance criteria (a)–(m) are framed as the exhaustive per-branch decomposition the implementation MUST satisfy, but none of them exercises this arm. (g) tests dispatch after `/reload`; (h) tests idempotent re-entry from a *non-steady-state* tuple; (j) tests drain-throw recovery where the second sub-step-1 act succeeds (so the post-handler tuple is `(false, "shutting-down")`, not `(false, undefined)`). None gives the implementer a fixture that observes "post-all-three-throw → next slash dispatches the loom, no system note returned" or that catches an implementation which silently added a fourth `readDrainState` arm or a sentinel flag to suppress dispatch on this tuple. The MUST NOT in the *Slash-dispatch interaction* paragraph therefore has no observable witness.

The arm is also non-trivial to derive from the steady-state criterion (a): the dispatch fact is the same, but the load-bearing assertion is the **negative** — that no degraded note is fronted despite three swallowed throws — and that the runtime did not grow a suppression seam since the spec was last audited.

## Solution Space

**Shape:** single

### Recommendation

Add a criterion **(n) All-three-throw corner case — slash-dispatch arm** to the per-branch acceptance list, parallel in shape to (i)/(j)/(k) (the other "sub-step-1 / interstitial-seam throw" branches). Required content:

- **Setup, per `event.reason ∈ {"new","resume","fork"}` (3 fixtures):** pinned-constant snapshot read succeeds; sub-step-1 `LoomRegistry.drain` throws (per-call `try`/`catch` catches it, emits one `teardown-step-failed` row under `details.call = "loomRegistry.drain"`); sub-step-1 `LoomRegistry.initDrainStateTag` throws (caught, one row under `details.call = "loomRegistry.initDrainStateTag"`); sub-steps 2–4 run; interstitial-seam `LoomRegistry.markRuntimeDegraded` throws (silently swallowed per the interstitial-seam swallow rule, no `teardown-step-failed` row). Post-handler tuple: `(drained: false, tag: undefined)`.
- **Diagnostic-emission assertions:** exactly one `loom/host/session-shutdown-runtime-degraded` row whose `details.event.reason` equals the supplied reason (the diagnostic-emission predicate still fires; the emission runs after the swallowed seam under its own isolation wrap); plus the two `teardown-step-failed` rows above; no other teardown-handler rows for this event.
- **Slash-dispatch assertion (the load-bearing addition):** the next slash invocation against the still-live extension instance MUST dispatch its loom normally — neither the `"loom /<name>: extension shutting down"` note nor the `"loom /<name>: extension degraded; /reload to recover"` note is returned, on the same footing as (g)'s post-`/reload` arm; this confirms `readDrainState` returns the steady-state tuple `(false, undefined)` and arm (a) is selected. The fixture explicitly forbids any conformant implementation from growing a fourth `readDrainState` arm or a "post-failed-handler" sentinel flag that would front the degraded note instead — an implementation observing either non-dispatch outcome on this fixture is non-conformant against the *Slash-dispatch interaction* paragraph's MUST NOT.
- **Re-entry sub-assertion:** a follow-up session-only `session_shutdown` against the same instance (with a successful follow-up snapshot read) re-runs the full handler (the broadened short-circuit predicate `snapshot.drained === true || snapshot.tag !== undefined` does not fire on `(false, undefined)`) and emits one additional `runtime-degraded` row — the loom 1.0-accepted duplicate-emission path the all-three-throw paragraph already pins, on the same footing as (m)'s read-failure re-entry; operator dedup on `(code, details.event.reason)` collapses the duplicates.

Place (n) immediately after (m) so the criterion list keeps its current order and the cross-references from (b), (m), and the *Idempotent re-entry and the all-three-throw corner case* section line up. Update the `V9l-T` "Tests" bullets to add a §all-three-throw entry parallel to the existing four `(un-anchored; GOV-22 residue)` bullets.

Edge cases the implementer must watch:

- The `{"quit","reload"}` variant is **out of scope** for (n) — Pi tears the runtime down before any subsequent slash can reach it (the same scoping (j) applies for its quit/reload variant), and the diagnostic-emission predicate suppresses the `runtime-degraded` row anyway. Mention this explicitly to forestall a 5-fixture grid.
- The chokidar acknowledged-unpinned double-close behaviour is on the load path of the re-entry sub-assertion (sub-step 4 re-runs against already-closed watchers); any extra `teardown-step-failed` rows that surfaces are loom 1.0-accepted per the existing all-three-throw prose and must not be asserted-against in the fixture.

## Consequence

**Severity:** correctness

Two reasonable implementers will diverge on the all-three-throw arm: one reads the *Slash-dispatch interaction* paragraph and dispatches; the other reads "degraded for the remainder of its lifetime" elsewhere on the page and adds a sentinel flag or a fourth `readDrainState` arm to front the degraded note on the steady-state tuple too, silently violating the explicit MUST NOT. With no fixture, the leaf's "ships when" criteria pass either way and the divergence ships. The duplicate-emission half of the corner case is covered indirectly by (h), but the dispatch arm — the operator-visible consequence — is not.

## Problem

The all-three-throw corner case (sub-step 1's `LoomRegistry.drain`, sub-step 1's `LoomRegistry.initDrainStateTag`, and the post-sub-step-4 `LoomRegistry.markRuntimeDegraded` all throw on a session-only `session_shutdown`) leaves the post-handler tuple `(drained: false, tag: undefined)`. The `## Acceptance criteria` list (a)–(m) is framed as the exhaustive per-branch decomposition the implementation MUST satisfy, but none exercises this arm's slash-dispatch behaviour — that the next slash invocation dispatches the loom normally (no system note), and that the runtime did not grow a fourth `readDrainState` arm or a post-failed-handler sentinel to front the degraded note on this steady-state tuple. The `## Predicate split` seam-minimality MUST NOT therefore has no observable witness; (g) tests post-`/reload` dispatch, (h) tests idempotent re-entry, and (j) tests drain-throw recovery from a `(false, "shutting-down")` tuple, but none covers `(false, undefined)` dispatch.

The 2026-06-04 file-size split (`f5e89f4`) only moved the section into the standalone subpage and is irrelevant to the gap. The 2026-06-05 commit `2775382` extended the grid with (l)/(m) for `readDrainState` read failures but did not touch the all-three-throw dispatch arm.

## Solution approach

Add an acceptance criterion to the `## Acceptance criteria` list in `session-only-degraded-state.md`, parallel in shape to (i)/(j)/(m), that exercises the all-three-throw corner case's `(drained: false, tag: undefined)` slash-dispatch arm. The criterion asserts the next slash invocation dispatches the loom normally — neither the `"shutting down"` nor the `"degraded; /reload to recover"` note returned, on the same footing as (g)'s post-`/reload` arm — confirming `readDrainState` selects arm (a) (`drain-state-contract.md#idempotent-handler-clause`), and witnesses the `## Predicate split` MUST NOT against any fourth `readDrainState` arm or post-failed-handler sentinel. Cover the follow-up duplicate-emission re-entry the `## Idempotent re-entry and the all-three-throw corner case` section already pins, parallel to (m)'s read-failure re-entry.

## Solution constraints

- Out of scope: the `{"quit","reload"}` variant — Pi tears the runtime down before any subsequent slash dispatch, so the criterion carries no quit/reload dispatch fixture (the same scoping (j) applies to its quit/reload variant).
- Out of scope: asserting against extra `teardown-step-failed` rows from chokidar's acknowledged-unpinned double-close on the re-entry path — those are loom 1.0-accepted per the all-three-throw prose.

## Problem

`.pi/project-config.md`'s `## Spec rules` section reproduces the live inline-label prefixes as a fixer quick-reference, but its inline-label bullet lists only `HC3` and `NOCEIL` and omits the third live prefix `SNK` (bound to `slash-invocation.md`, registered in GOV-16's per-page table for the `SNK-a`…`SNK-k` rows under SLSH-4). Unlike the REQ-ID bullet, it carries no non-authoritative disclaimer, so its two-prefix list reads as exhaustive. A fixer consulting only project-config can conclude `SNK` is unregistered and either raise a spurious GOV-16 *Add* request or refuse a legitimate `SNK-l` allocation, when the prefix is live and the next free alphabetic tail is the correct mint.

## Issue introduction

**Verdict:** multi-commit-interaction
**Introducing commits:**
- `899b8fa` (2026-05-22) — authored the inline-label bullet as "Live inline-label prefixes: `HC3`, `NOCEIL`" with no staleness disclaimer.
- `6c75128` (2026-06-05) — added the "this snapshot is not authoritative" disclaimer to the REQ-ID bullet only, establishing the asymmetry.
- `27b2576` (2026-06-05) — added `SNK` to the GOV-16 per-page inline-label prefix table on `stable-inline-labels.md` without mirroring the addition into `.pi/project-config.md`.

**History:** `git log --follow -- .pi/project-config.md` confirms the file has been touched four times since `899b8fa` and that no commit has touched the inline-label bullet since its original authoring. `git log -S "SNK" -- docs/spec_topics/governance/stable-inline-labels.md` returns exactly `27b2576`. The exhaustive-reading defect was latent from `899b8fa`, became a missing-disclaimer asymmetry at `6c75128`, and became a stale-content defect at `27b2576`. The durable fix targets the interaction (snapshot ↔ authoritative table linkage) rather than any single commit.

## Solution approach

Rewrite the Stable inline labels bullet in `.pi/project-config.md`'s `## Spec rules` to add `SNK` alongside `HC3`/`NOCEIL`, pairing each prefix with its registered page (`HC3`/`NOCEIL` → `hard-ceilings.md`, `SNK` → `slash-invocation.md`). Append a non-authoritative disclaimer mirroring the REQ-ID bullet's, deferring to the GOV-16 per-page inline-label prefix table on `docs/spec_topics/governance/stable-inline-labels.md` as the authoritative live-prefix set.

## Solution constraints

- Out of scope: GOV-16's per-page inline-label prefix table on `stable-inline-labels.md` (owned by F-1328/F-1329) — do not edit it or change which prefixes are live.
- `.pi/project-config.md` lives outside `docs/` and is not covered by `commitAddPaths` — the fix must `git add .pi/project-config.md` explicitly to land.

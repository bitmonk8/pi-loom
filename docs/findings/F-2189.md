## Finding

The blockquote note at the top of the *Settings file reads* section in `discovery/package-and-settings.md` points readers to where the write-back key-preservation rule for the loom-owned `loomPaths` / `looms.*` keys now resides:

> The **write-back key preservation** rule for the loom-owned `loomPaths` / `looms.*` keys is a host-behaviour presupposition about Pi's serialiser and **now lives** alongside the other Pi presuppositions at [Pi Integration Contract — Host prerequisites — Settings write-back key preservation](../pi-integration-contract/host-prerequisites.md#settings-write-back-preservation-presupposition).

The word "now" encodes editorial history — it tells the reader the rule *used to* live elsewhere and was relocated. An implementer (or auditor) consulting this note needs only the present fact: the rule lives at the cited anchor. "now lives" is decision-log residue from the move; the temporal qualifier carries no normative content and will read as stale once the relocation is no longer recent.

## Issue introduction
**Verdict:** single-commit-introduction
**Introducing commit:** `6caacfe` — "pi-loom spec: resolve \"Write-back preservation MUST NOT in wrong place\"" (2026-06-05)
**History:** `git log -L 38,38:docs/spec_topics/discovery/package-and-settings.md` shows the prior commit (`257c545`) introduced the full inline `**Write-back preservation presupposition.**` rule at this site. Commit `6caacfe` then relocated that rule to `pi-integration-contract/host-prerequisites.md` and replaced the in-place text with the present stub blockquote, introducing the "now lives" phrasing in the same edit. The migration wording is an artefact of that relocation commit; it has been carried verbatim since.

## Solution Space

**Shape:** single

### Recommendation

Replace "now lives" with "lives" in the blockquote note. The sentence then reads: "…is a host-behaviour presupposition about Pi's serialiser and **lives** alongside the other Pi presuppositions at [Pi Integration Contract — Host prerequisites — Settings write-back key preservation](…)." No other text in the note changes; the anchor target is unaffected.

## Consequence

**Severity:** cosmetic

Pure wording. No implementer behaviour changes either way; the cross-reference resolves to the same anchor. The only cost is the minor reading friction of process-history phrasing embedded in a forward-pointing note.

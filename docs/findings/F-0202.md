## Finding

`coverage-matrix.md` maps the executable REQ-ID `DISC-4` to two closing leaves: `V10a` (collision-detection closure) and `V9m` (superseded-entry-dispatch closure). `conventions.md` §REQ-ID discipline requires that "a test asserting a numbered REQ-ID MUST cite that REQ-ID inline in the test source," and §Leaf format restates it for Tests bullets ("one bullet per REQ-ID the leaf claims to implement; cite the ID inline"). `V10a`'s collision-detection bullet complies — it opens with `DISC-4: …`. `V9m`'s superseded-entry-dispatch bullet (and its paired `V9m-T` bullet) cite only the spec page anchor `drain-state-contract.md#superseded-entry-dispatch`; neither carries the inline `DISC-4` token.

Because `DISC-4` already has one citing test (V10a's facet), the citing-test reconciliation arm of the closing gate stays green — the gate certifies that *a* citing test exists per mapped REQ-ID, not that every closing leaf the matrix names actually cites it. The result is a silent traceability hole: an auditor (or the split-row residue walk that finding 28 proposes) tracing `DISC-4` across the test corpus finds only the V10a collision facet, with the V9m superseded-entry-dispatch facet uncited and therefore invisible to the matrix→test trace.

Commit 9d61b21 relocated the superseded-entry-dispatch facet to a new matrix row and authored its assertion on `V9b` citing only the `drain-state-contract.md#superseded-entry-dispatch` spec anchor — never the inline `DISC-4` token. The omission then carried verbatim through cb8737a (which extracted the drain-state contract from `V9b` into the new `V9m`/`V9m-T` leaves) and b61322f (which consolidated the two `DISC-4` rows into the single multi-close row), both preserving the uncited bullet.

## Solution Space

**Shape:** single

### Recommendation

Add an inline `DISC-4` citation to the superseded-entry-dispatch Tests bullet in both `V9m-T-drain-state-contract.md` and `V9m-drain-state-contract.md`, so the authored test source emits the `DISC-4` token and V9m's facet joins the matrix→test trace. Mirror `V10a`'s compliant form — open the bullet with `DISC-4` and a disambiguating qualifier, e.g. `DISC-4 (superseded-entry dispatch): after a session_start supersession pass …`, retaining the existing spec-anchor link and assertion text.

Constraints for the fixer: this fix is internal to the two V9m leaf files — the spec is read-only, and the `coverage-matrix.md` `DISC-4` row (including the `V10a` mapping and the `V9m` superseded-entry-dispatch mapping) is already correct and must not be altered. Cite only the existing `DISC-4` ID; introduce no new leaf or REQ-ID.

## Consequence

**Severity:** advisory

The closing gate does not redden — V10a's citation satisfies the per-REQ-ID citing-test check — so no leaf is blocked and the implementer can still produce a working V9m. The cost is degraded traceability: `DISC-4`'s superseded-entry-dispatch facet carries no inline REQ-ID token in the test corpus, so a matrix→test trace (and the split-row residue walk finding 28 proposes) cannot locate V9m's coverage of `DISC-4`.

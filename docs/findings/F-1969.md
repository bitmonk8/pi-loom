## Finding

*"The implementation MAY use a type-aware variant (e.g. a TypeScript-program symbol-table walk) provided the canonical-`pi`-carrier and canonical-`ctx`-carrier rules and the prohibition above continue to be enforced with the same per-discriminator routing — the audit's behavioural contract is invariant under tooling choice; what changes is only the implementation."* This is implementation guidance (an informative MAY clause about permissible tooling strategies), not a normative prohibition on import/access shapes. It appears as the closing sentence of a dense normative prohibition list.

## Finding

The `loom/load/unreadable-source` row of the diagnostic registry carries, in its *Trigger* column, a sentence that records the editorial history of a prior code split rather than describing when the diagnostic fires:

> "The per-read-timeout sub-trigger that previously rode this row now has its own code, `loom/load/package-read-timeout` (see the row below)."

The *Trigger* column's job is to state the condition under which the code is emitted. This sentence states nothing about `loom/load/unreadable-source`'s own trigger condition — it is a back-reference explaining that a cause which once lived here has since moved elsewhere. The `loom/load/package-read-timeout` row immediately below is self-contained and fully specifies its own trigger, so the back-reference adds no information an implementer or auditor needs to emit or interpret either code. It is decision-log residue left behind by the split.

## Issue introduction
**Verdict:** single-commit
**Introducing commit:** 65989e2
**History:** The per-`package.json` read-timeout cause was first folded into the `loom/load/unreadable-source` row inline at commit 25e8a60 (2026-05-07, "resolve T08 — Per-package.json read timeout: overrun is unbounded"), as a "Also fires …" clause carrying `details.kind = "package-read-timeout"`. Commit 65989e2 (2026-06-01, "resolve 'unreadable-source row carries two message formats'") split that cause out into its own `loom/load/package-read-timeout` code and, in the same edit, replaced the inline timeout clause with the back-reference sentence flagged here. The defect is therefore an artefact of that split rather than present since the row's inception.

## Solution Space

**Shape:** single

### Recommendation

Delete the sentence "The per-read-timeout sub-trigger that previously rode this row now has its own code, `loom/load/package-read-timeout` (see the row below)." from the `loom/load/unreadable-source` *Trigger* cell. Leave the rest of the cell (permission denied, ACL, symlink loop, transient I/O error; severity-by-source rule) unchanged. The `loom/load/package-read-timeout` row below already states its own trigger completely and needs no back-reference to be discoverable.

## Consequence

**Severity:** cosmetic

No observer is affected: both codes' trigger conditions remain fully specified without the sentence. The cost is purely that a reader of the `unreadable-source` trigger must parse historical commentary that does not bear on current behaviour.

## Problem

The closing edge-case bullet of `cancellation.md`'s Race-semantics section claims the `Checkpoint` seam's production wiring is "an already-resolved promise per checkpoint" and that the seam is "purely a test surface … imposing no observable behaviour on production code beyond the always-`await`ed no-op". That blanket claim contradicts both the page's own **Granularity** rule, under which the `loop-iter` checkpoint yields a macrotask turn before its signal-check, and PIC-10's precise scoping (`host-interfaces-services.md#pic-10`), which defines a macrotask yield for `loop-iter` and a microtask-resolved no-op for all other checkpoint kinds. An implementer reading the bullet in isolation would wire every checkpoint — including `loop-iter` — as an already-resolved promise, defeating cancellation of compute-bound loops.

## Solution approach

Rewrite the closing edge-case bullet of `cancellation.md` so it cross-references PIC-10's production-wiring rule (`host-interfaces-services.md#pic-10`) instead of asserting the blanket "already-resolved promise per checkpoint" / "purely a test surface" framing. Retain the bullet's substantive point — that the seam is the deterministic-test substrate for the two race rules.

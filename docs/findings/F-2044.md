## Problem

The six-file inventory-closure audit cluster (`inventory-audit-intro.md`, `audit-target-categories.md`, `audit-recognised-shapes.md`, `audit-resolution.md`, `audit-failures.md`, `audit-wire-and-canary.md`) carries no consolidated non-goals / out-of-scope statement. Its loom 1.0 deferrals are stated, but scattered inline at their normative sites: the optional `generated-source-in-tree` check and citation-resolution validation in `audit-resolution.md`, the baseline-snapshot floor left as a MAY in `audit-wire-and-canary.md`, and the CI-parser implementation plus `npm`-script wiring left as implementation choices in `inventory-audit-intro.md`. This is a discoverability and consistency gap: the sibling version-bump cluster in the same Pi-Integration-Contract family already publishes an explicit `### Non-goals` section, and the corpus-wide `## loom 1.0 non-goals` aggregator does not mention any audit-cluster deferral — so the cluster's boundaries live in neither established non-goals home.

## Issue introduction
**Verdict:** present-since-inception
**Introducing commits:** `e5cd514` (`pi-loom spec: resolve "T35 SDK capability inventory closed-set negative-direction audit"`) — first authored the audit content into `docs/spec_topics/pi-integration-contract.md`; `f5e89f4` (`docs: enforce 100KB hard / 30KB recommended size cap on spec set`) — split that content into the six present cluster files.
**History:** The corpus is git-tracked. `git log --follow --diff-filter=A` shows all six cluster files were created at the size-cap split `f5e89f4`; tracing the audit prose back through `git log -S 'allow-pi-surface'` shows it originated at `e5cd514`. A `git log -S 'Non-goals' -- spec_topics/pi-integration-contract/` finds non-goals sections only in the version-bump siblings, never in the audit cluster. The consolidated non-goals statement was therefore never present in the audit content from its first authoring; the gap is original to the cluster, not a regression introduced by a later edit.

## Solution approach

Add a `### Non-goals` section to `inventory-audit-intro.md`, the cluster entry page, modelled on the sibling `### Non-goals` section in `version-bump-intro.md`. Consolidate the cluster's existing inline loom 1.0 deferrals by reference — the deferred `generated-source-in-tree` check and citation-resolution validation in `audit-resolution.md` (`#audit-scope`), the baseline-snapshot floor MAY in `audit-wire-and-canary.md` (`#audit-non-empty-scan-canary`), and the CI-parser / `npm`-script implementation choices in this file. Defer genuinely cross-cutting non-goals to the corpus-wide aggregator (`future-considerations/model-changes-and-non-goals.md#v1-non-goals`) rather than restating them.

## Solution constraints

- The new section must point at the inline deferral sites rather than restate the normative detail they own — it must not become a second normative owner that can drift from those sites.
